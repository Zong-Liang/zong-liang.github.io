## 题目：0550. 游戏玩法分析 IV

**Table:** `Activity`

| Column Name  | Type |
| ------------ | ---- |
| player_id    | int  |
| device_id    | int  |
| event_date   | date |
| games_played | int  |

- `(player_id, event_date)` 是此表的主键（具有唯一值的列的组合）。
- 这张表显示了某些游戏的玩家的活动情况。
- 每一行是一个玩家的记录，他在某一天使用某个设备注销之前登录并玩了很多游戏（可能是 0）。

**问题描述:**

编写解决方案，报告在首次登录的第二天再次登录的玩家的 **比率**，**四舍五入到小数点后两位**
。换句话说，你需要计算从首次登录后的第二天登录的玩家数量，并将其除以总玩家数。

**示例 1:**

**输入:**
`Activity` table:

| player_id | device_id | event_date | games_played |
| --------- | --------- | ---------- | ------------ |
| 1         | 2         | 2016-03-01 | 5            |
| 1         | 2         | 2016-03-02 | 6            |
| 2         | 3         | 2017-06-25 | 1            |
| 3         | 1         | 2016-03-02 | 0            |
| 3         | 4         | 2018-07-03 | 5            |

**输出:**

| fraction |
| -------- |
| 0.33     |

**解释:**
只有 ID 为 1 的玩家在第一天登录后才重新登录，所以答案是 1/3 = 0.33

## 题解

这个问题的核心是首先找出每个玩家的首次登录日期，然后检查这些玩家是否在第二天也登录了。

```sql
SELECT ROUND(
               COUNT(T.player_id) / (SELECT COUNT(DISTINCT player_id) FROM Activity),
               2
       ) AS fraction
FROM (
         -- 步骤 1: 找出每个玩家的首次登录日期
         SELECT player_id,
                MIN(event_date) AS first_login_date
         FROM Activity
         GROUP BY player_id) AS T
WHERE
    -- 步骤 2: 检查该玩家是否在首次登录的第二天也登录了
    (T.player_id, DATE_ADD(T.first_login_date, INTERVAL 1 DAY))
        IN
    (SELECT player_id, event_date FROM Activity);
```

**查询逻辑分解:**

1. **子查询 `(SELECT ... ) AS T` (步骤 1):**

   - `SELECT player_id, MIN(event_date) AS first_login_date`: 我们首先需要确定每个玩家的首次登录日期。
   - `GROUP BY player_id`: 通过按 `player_id` 分组，`MIN(event_date)` 会返回每个玩家的最早（即首次）登录日期。
   - 这个子查询的结果是一个临时表 `T`，包含两列：`player_id` 和 `first_login_date`。

2. **主查询的 `WHERE` 子句 (步骤 2):**

   - `WHERE (T.player_id, DATE_ADD(T.first_login_date, INTERVAL 1 DAY)) IN (...)`: 这是筛选的核心。
   - `DATE_ADD(T.first_login_date, INTERVAL 1 DAY)`: 计算出每个玩家首次登录日期的 **第二天**。
   - 我们构造一个元组 `(player_id, next_day_date)`。
   - `IN (SELECT player_id, event_date FROM Activity)`: 然后我们检查这个元组是否存在于原始的 `Activity`
     表中。如果存在，就意味着这个玩家确实在首次登录的第二天再次登录了。

3. **主查询的 `SELECT` 部分 (步骤 3):**
   - `COUNT(T.player_id)`: 计算通过了 `WHERE` 子句筛选的玩家数量，这就是次日留存的玩家数（分子）。
   - `(SELECT COUNT(DISTINCT player_id) FROM Activity)`: 计算总的独立玩家数量（分母）。
   - `... / ...`: 执行除法运算得到比率。
   - `ROUND(..., 2) AS fraction`: 将计算出的比率四舍五入到小数点后两位，并重命名为 `fraction`。
