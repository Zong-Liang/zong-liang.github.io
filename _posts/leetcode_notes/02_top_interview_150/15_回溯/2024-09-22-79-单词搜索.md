## 题目

**描述：**

给定一个 `m x n` 二维字符网格 `board` 和一个字符串单词 `word`。如果 `word` 存在于网格中，返回 `true`；否则，返回 `false`。

单词必须按照字母顺序，通过相邻的单元格内的字母构成，其中“相邻”单元格是那些水平相邻或垂直相邻的单元格。同一个单元格内的字母不允许被重复使用。

**示例 1:**

![示例1图片](https://assets.leetcode.com/uploads/2020/11/04/word2.jpg)

- **输入:** `board = [['A','B','C','E'],['S','F','C','S'],['A','D','E','E']]`, `word = "ABCCED"`
- **输出:** `true`

**示例 2:**

![示例2图片](https://assets.leetcode.com/uploads/2020/11/04/word-1.jpg)

- **输入:** `board = [['A','B','C','E'],['S','F','C','S'],['A','D','E','E']]`, `word = "SEE"`
- **输出:** `true`

**示例 3:**

![示例3图片](https://assets.leetcode.com/uploads/2020/10/15/word3.jpg)

- **输入:** `board = [['A','B','C','E'],['S','F','C','S'],['A','D','E','E']]`, `word = "ABCB"`
- **输出:** `false`

**提示:**

- `m == board.length`
- `n = board[i].length`
- `1 <= m, n <= 6`
- `1 <= word.length <= 15`
- `board` 和 `word` 仅由大小写英文字母组成

**进阶：** 你可以使用搜索剪枝的技术来优化解决方案，使其在 `board` 更大的情况下可以更快解决问题？

## 题解

### 这题不是在网格里“找字母”，而是在网格里“找一条路径”

如果只是判断某个字母在不在网格里，那直接遍历一遍就够了。  
但这题要找的是一个完整单词，而且这个单词必须按顺序由**相邻格子**拼出来。

这就意味着，题目真正问的是：

**能不能在网格里找到一条路径，使路径上的字符依次组成 `word`。**

比如 `word = "ABCCED"`，那你不能只验证这些字母都出现过，还得保证：

- 第一个字母和第二个字母相邻
- 第二个和第三个也相邻
- 并且同一个格子不能重复使用

所以这题的本质，其实是一个二维网格上的路径搜索问题。

### 为什么这题适合用回溯

这题很适合用 DFS + 回溯。

因为当你站在某个格子上，已经匹配到 `word[k]` 时，下一步只有四种可能：

- 上
- 下
- 左
- 右

你需要从这四个方向里继续找 `word[k + 1]`。  
如果某个方向走得通，就能一路匹配成功；如果四个方向都走不通，就说明当前这条路径选错了，需要退回上一步，换别的方向继续试。

这正是回溯最标准的场景：

**试一个方向，不行就退回来，再试下一个。**

### 为什么不能重复使用同一个格子

题目明确要求：同一个单元格不能被重复使用。

这个限制非常关键。  
否则很多本来不成立的单词也会被错误判成 `true`。

比如示例 3：

```text
word = "ABCB"
```

在板子上确实能找到 `A -> B -> C`，但最后那个 `B` 只能回头再用一次之前的格子。  
而这是不允许的，所以答案是 `false`。

也就是说，搜索过程中除了要看“字符对不对”，还得记住“这个格子当前这条路径里有没有用过”。

### 最自然的做法：从每个格子都试一次

因为我们并不知道单词会从哪里开始，所以最直接的办法就是：

**把每个格子都当成起点试一遍。**

如果某个格子正好等于 `word[0]`，那就从这里启动 DFS，看看能不能把后面的字符也顺着找出来。  
只要有一个起点能成功，整个答案就是 `true`。

如果所有起点都试完了还是不行，那答案就是 `false`。

### DFS 里到底在检查什么

假设递归函数写成：

```java
dfs(i, j, k)
```

它表示的意思就是：

**当前站在 `board[i][j]`，正在尝试匹配 `word[k]`。**

那它要做的事很明确：

1. 如果当前位置越界，直接失败
2. 如果当前位置字符和 `word[k]` 对不上，直接失败
3. 如果已经匹配到最后一个字符，并且当前字符也对上了，那就成功
4. 否则，先把当前位置标记为“已访问”
5. 然后继续递归尝试四个方向，看能不能匹配下一个字符
6. 无论成功还是失败，返回前都要把当前位置恢复回来

这最后一步“恢复现场”，就是回溯的核心。

### 为什么要先标记、再搜索、最后恢复

因为同一个格子在一条路径里不能重复使用。  
所以当你走到某个格子时，必须先把它标记掉，避免下一层递归又走回这里。

最常见的做法，是把当前字符临时改成一个不会和字母冲突的特殊值，比如 `'\0'` 或 `'#'`。

但这个修改只是为了当前路径服务。  
如果这一条路没走通，回到上一层后，别的路径仍然可能需要用到这个格子，所以必须把它恢复回原来的字符。

所以流程一定是：

- 暂时标记
- 递归搜索
- 恢复原值

少了最后一步，后面的搜索结果就会被污染。

### 用示例 1 走一下过程

示例 1：

```text
board = [
['A','B','C','E'],
['S','F','C','S'],
['A','D','E','E']
]
word = "ABCCED"
```

先从左上角的 `A` 开始，因为它和 `word[0]` 相同。

接下来要找 `B`，右边正好是 `B`，可以继续。  
再找 `C`，继续往右。  
再找一个 `C`，往下。  
再找 `E`，往下。  
最后找 `D`，往左。

整条路径能连起来，所以答案是 `true`。

而示例 3 的 `ABCB`，前面虽然也能找到 `A -> B -> C`，但最后那个 `B` 只能回头复用已经走过的格子，所以这条路径不合法，最终返回 `false`。

### 标准回溯写法

```java
public boolean exist(char[][] board, String word) {
    int m = board.length;
    int n = board[0].length;
    char[] chars = word.toCharArray();

    for (int i = 0; i < m; i++) {
        for (int j = 0; j < n; j++) {
            if (dfs(board, chars, i, j, 0)) {
                return true;
            }
        }
    }

    return false;
}

private boolean dfs(char[][] board, char[] word, int i, int j, int k) {
    if (i < 0 || i >= board.length || j < 0 || j >= board[0].length) {
        return false;
    }

    if (board[i][j] != word[k]) {
        return false;
    }

    if (k == word.length - 1) {
        return true;
    }

    char temp = board[i][j];
    board[i][j] = '\0';

    boolean found = dfs(board, word, i + 1, j, k + 1) ||
                    dfs(board, word, i - 1, j, k + 1) ||
                    dfs(board, word, i, j + 1, k + 1) ||
                    dfs(board, word, i, j - 1, k + 1);

    board[i][j] = temp;
    return found;
}
```

### 这段代码为什么是对的

可以从两个角度理解。

第一，它不会漏解。  
因为每个格子都被当作起点试过了，而每次进入 DFS 后，也把上下左右四个方向都尝试了一遍，所以所有可能路径都覆盖到了。

第二，它不会误判。  
因为只有当前位置字符和 `word[k]` 一致时才会继续往下搜，而且同一个格子一旦被当前路径用过，就会被暂时标记，下一层不可能再重复使用它。

所以只要函数返回 `true`，一定存在一条合法路径；  
如果所有搜索都失败，那就说明这条单词确实拼不出来。

### 还能做哪些剪枝

标准回溯已经能通过这题，但还可以做一些更聪明的预处理。

#### 1. 频次剪枝

先统计 `board` 里每个字符出现了多少次，再统计 `word` 里每个字符出现了多少次。  
如果 `word` 中某个字符需要的数量，比 `board` 里实际拥有的还多，那根本不可能拼出来，可以直接返回 `false`。

比如板子里只有一个 `A`，而 `word` 里有两个 `A`，那就不用搜了。

#### 2. 从更稀缺的一端开始搜

有时候 `word` 的首字符在棋盘里到处都是，而尾字符只出现一两次。  
那从头开始搜，会有很多无效起点。

这时可以反过来想：  
既然路径本身是可以正着搜也可以倒着搜的，那不如从更少见的一端开始，这样分支会更少。

比如如果 `word[0]` 在板上出现很多次，而 `word[word.length - 1]` 只出现很少次，就可以先把 `word` 反转，再去做同样的 DFS。

这个优化不改变最坏复杂度，但在一些构造数据上会快很多。

### 优化后的思路

完整优化版一般会这样做：

1. 先判断棋盘总格子数是否小于 `word.length()`，如果是，直接返回 `false`
2. 统计字符频次，不够就直接返回 `false`
3. 比较首尾字符在棋盘中的出现次数，必要时把单词翻转
4. 再执行标准 DFS

这几步本质上都是在尽量减少无意义搜索。

### 再看复杂度

设棋盘大小是 `m x n`，单词长度是 `L`。

最坏情况下，我们会从每个格子出发做一次 DFS。  
而在 DFS 过程中，第一步最多有 4 个方向，之后因为不能立刻走回头路，通常最多只会继续扩展出 3 个方向。

所以时间复杂度通常写成：

$$
O(m \cdot n \cdot 3^L)
$$

这是一个常见写法，表达的是搜索树的规模。

空间复杂度主要来自递归栈深度，最多为 `L`，所以是：

$$
O(L)
$$

因为我们是直接在原数组上做标记，没有额外开 `visited` 矩阵，所以辅助空间比较省。

### 这题真正要记住的点

这题最值得记住的，不是“单词搜索”这四个字，而是这种模型：

**在二维网格中，从某个起点出发，按规则走路径，并且一条路径里不能重复使用格子。**

这类题十有八九都要往 DFS + 回溯上想。  
关键通常就是三件事：

1. 递归参数表示什么
2. 什么时候剪掉当前分支
3. 如何标记当前格子已使用，以及如何恢复现场

把这三件事想清楚，很多网格搜索题都会变得很像。

### 完整代码

```java
import java.util.*;

public class Main {

    public static boolean exist(char[][] board, String word) {
        int m = board.length;
        int n = board[0].length;

        if (m * n < word.length()) {
            return false;
        }

        int[] boardCounts = new int[128];
        for (char[] row : board) {
            for (char c : row) {
                boardCounts[c]++;
            }
        }

        char[] wordChars = word.toCharArray();
        int[] wordCounts = new int[128];
        for (char c : wordChars) {
            wordCounts[c]++;
            if (wordCounts[c] > boardCounts[c]) {
                return false;
            }
        }

        if (boardCounts[wordChars[0]] > boardCounts[wordChars[word.length() - 1]]) {
            word = new StringBuilder(word).reverse().toString();
            wordChars = word.toCharArray();
        }

        for (int i = 0; i < m; i++) {
            for (int j = 0; j < n; j++) {
                if (board[i][j] == wordChars[0] && dfs(board, wordChars, i, j, 0)) {
                    return true;
                }
            }
        }

        return false;
    }

    private static boolean dfs(char[][] board, char[] word, int i, int j, int k) {
        if (i < 0 || i >= board.length || j < 0 || j >= board[0].length) {
            return false;
        }

        if (board[i][j] != word[k]) {
            return false;
        }

        if (k == word.length - 1) {
            return true;
        }

        char temp = board[i][j];
        board[i][j] = '\0';

        boolean found = dfs(board, word, i + 1, j, k + 1) ||
                        dfs(board, word, i - 1, j, k + 1) ||
                        dfs(board, word, i, j + 1, k + 1) ||
                        dfs(board, word, i, j - 1, k + 1);

        board[i][j] = temp;
        return found;
    }

    public static void main(String[] args) {
        char[][] board1 = {
            {'A','B','C','E'},
            {'S','F','C','S'},
            {'A','D','E','E'}
        };
        String word1 = "ABCCED";
        System.out.println("示例 1 输入: board = " + Arrays.deepToString(board1) + ", word = " + word1);
        System.out.println("示例 1 输出: " + exist(board1, word1));
        System.out.println("-----------------------------");

        char[][] board2 = {
            {'A','B','C','E'},
            {'S','F','C','S'},
            {'A','D','E','E'}
        };
        String word2 = "SEE";
        System.out.println("示例 2 输入: board = " + Arrays.deepToString(board2) + ", word = " + word2);
        System.out.println("示例 2 输出: " + exist(board2, word2));
        System.out.println("-----------------------------");

        char[][] board3 = {
            {'A','B','C','E'},
            {'S','F','C','S'},
            {'A','D','E','E'}
        };
        String word3 = "ABCB";
        System.out.println("示例 3 输入: board = " + Arrays.deepToString(board3) + ", word = " + word3);
        System.out.println("示例 3 输出: " + exist(board3, word3));
    }
}
```
