## 题目

**描述：**

给定一个仅包含数字 `2-9` 的字符串，返回所有它能表示的字母组合。答案可以按 **任意顺序** 返回。

给出数字到字母的映射如下（与电话按键相同）。注意 1 不对应任何字母。

![电话按键映射](https://pic.leetcode.cn/1752723054-mfIHZs-image.png)

**示例 1:**

- **输入:** `digits = "23"`
- **输出:** `["ad","ae","af","bd","be","bf","cd","ce","cf"]`

**示例 2:**

- **输入:** `digits = "2"`
- **输出:** `["a","b","c"]`

**提示:**

- `1 <= digits.length <= 4`
- `digits[i]` 是范围 `['2', '9']` 的一个数字。

## 题解

### 这题本质上是在做组合枚举

题目给你一个数字字符串，每个数字都对应若干个字母，让你把所有可能的组合都列出来。

比如：

```text
digits = "23"
```

其中：

- `2` 对应 `"abc"`
- `3` 对应 `"def"`

那结果其实就是：

- 先从 `"abc"` 里选一个字母
- 再从 `"def"` 里选一个字母
- 把所有选法都列出来

也就是：

```text
ad ae af bd be bf cd ce cf
```

所以这题本质上不是在考映射，而是在考：

**如何把每一位的所有选择，按顺序一层一层组合起来。**

### 为什么这是典型的回溯题

这题特别适合用回溯，因为它的结构非常像一棵决策树。

以 `"23"` 为例，可以这样理解：

- 第一层，从数字 `2` 对应的 `"abc"` 里选一个
- 第二层，从数字 `3` 对应的 `"def"` 里再选一个

整棵树长这样：

```text
           ""
        /   |   \
       a    b    c
     / | \ / | \ / | \
    d  e  f d  e  f d  e  f
```

从根走到叶子的一条路径，就是一个合法答案。

比如：

- `a -> d` 得到 `"ad"`
- `b -> e` 得到 `"be"`
- `c -> f` 得到 `"cf"`

所以这题最自然的思路就是：

**用 DFS 把这棵树完整走一遍。**

### 回溯到底在做什么

回溯其实就是“尝试一种选择，往下走；走完之后撤销这次选择，再试下一种”。

放到这题里，就是：

1. 先处理当前这一位数字
2. 遍历它能对应的所有字母
3. 选一个字母拼到当前结果后面
4. 递归处理下一位数字
5. 下一层返回后，把刚才加上的字母删掉
6. 再试当前数字对应的下一个字母

这个“加上去，再删掉”的动作，就是回溯最核心的地方。

### 用 `"23"` 走一遍过程

还是看 `digits = "23"`。

一开始路径是空串 `""`。

先处理第一位数字 `2`，它对应 `"abc"`。

先选 `'a'`，当前路径变成：

```text
"a"
```

然后递归去处理第二位数字 `3`，它对应 `"def"`。

依次尝试：

- 加 `'d'`，得到 `"ad"`，这是一个完整答案
- 回退
- 加 `'e'`，得到 `"ae"`，这是一个完整答案
- 回退
- 加 `'f'`，得到 `"af"`，这是一个完整答案
- 回退到 `"a"`

接着再回到第一层，撤销 `'a'`，改选 `'b'`，继续重复这个过程，最后再处理 `'c'`。

你会发现，整件事其实就是：

**固定前面一部分，递归生成后面所有可能。**

这正是回溯的标准套路。

### 递归函数怎么设计

这题写递归时，最重要的是想清楚函数参数分别表示什么。

通常需要两个状态：

- `index`：当前处理到第几位数字
- `sb`：当前已经拼出来的字母组合

递归含义就是：

**从 `digits[index]` 开始，把后面的所有组合继续补全。**

终止条件也很简单：

如果 `index == digits.length()`，说明每一位都处理完了，当前 `sb` 就是一组完整答案，可以加入结果集。

### 回溯写法

```java
private static final String[] MAPPING = {
    "", "", "abc", "def", "ghi", "jkl", "mno", "pqrs", "tuv", "wxyz"
};

public List<String> letterCombinations(String digits) {
    List<String> result = new ArrayList<>();
    if (digits == null || digits.length() == 0) {
        return result;
    }

    backtrack(result, digits, 0, new StringBuilder());
    return result;
}

private void backtrack(List<String> result, String digits, int index, StringBuilder sb) {
    if (index == digits.length()) {
        result.add(sb.toString());
        return;
    }

    String letters = MAPPING[digits.charAt(index) - '0'];

    for (int i = 0; i < letters.length(); i++) {
        sb.append(letters.charAt(i));
        backtrack(result, digits, index + 1, sb);
        sb.deleteCharAt(sb.length() - 1);
    }
}
```

### 这段代码为什么是对的

可以把这段代码拆成三件事来看。

第一件事，找到当前数字能选哪些字母。  
比如当前是 `'2'`，那就拿到 `"abc"`。

第二件事，枚举这一位的每一种可能。  
也就是依次选 `'a'`、`'b'`、`'c'`。

第三件事，把当前这次选择交给下一层继续处理。  
如果下一层处理完了，就撤销这次选择，再试别的字母。

因为每一位都完整枚举了，而每种选择都会继续递归到最后，所以不会漏答案；  
又因为每条路径只会生成一次，也不会重复。

### 为什么要用 StringBuilder

这里用 `StringBuilder`，主要是为了方便做“加一个字符”和“删最后一个字符”这两件事。

如果每次都用字符串拼接，虽然也能写，但会频繁创建新字符串。  
这题数据范围不大，性能差距不明显，但从写法上说，`StringBuilder` 更符合回溯的习惯。

因为回溯本来就是在同一条路径上反复“试”和“撤销”，`StringBuilder` 很适合这种场景。

### 再看复杂度

假设输入长度为 `n`。

每一位数字最多对应 4 个字母，所以最坏情况下，总组合数接近 $4^n$。  
而每生成一个结果，长度最多是 `n`。

因此时间复杂度可以写成：

$$
O(4^n \cdot n)
$$

如果写得更细一点，也可以像题解里那样区分：

- 某些数字对应 3 个字母
- 某些数字对应 4 个字母

那就是 $O(3^m \times 4^k)$ 的形式。

空间复杂度主要来自递归深度和当前路径长度，都是 $O(n)$。  
如果把最终结果集占用的空间也算上，那当然会更大，因为答案本身就有很多组。

### 队列也能做，但没有回溯直观

这题除了回溯，还可以用队列迭代。

思路是：

1. 初始时队列里放一个空串
2. 每处理一位数字，就把队列中当前层的所有字符串取出来
3. 给它们分别拼上这一位数字对应的每个字母
4. 再放回队列

这样一轮一轮扩展，最后队列里剩下的就是所有答案。

这种写法本质上是按层生成组合，也没有问题。  
但从“为什么这样做”这个角度来看，回溯更贴合这题的决策树结构，所以通常会更容易讲清楚。

### 队列写法

```java
public List<String> letterCombinations(String digits) {
    LinkedList<String> queue = new LinkedList<>();
    if (digits == null || digits.length() == 0) {
        return queue;
    }

    String[] mapping = {"", "", "abc", "def", "ghi", "jkl", "mno", "pqrs", "tuv", "wxyz"};
    queue.add("");

    for (int i = 0; i < digits.length(); i++) {
        String letters = mapping[digits.charAt(i) - '0'];

        while (queue.peek().length() == i) {
            String cur = queue.remove();
            for (char ch : letters.toCharArray()) {
                queue.add(cur + ch);
            }
        }
    }

    return queue;
}
```

### 这题真正要记住的点

这题最值得记住的，不是电话按键映射，而是这种判断方式：

**只要题目让你“列出所有可能的组合”，并且每一位都有若干个可选项，通常就可以往回溯上想。**

因为这类题天然都可以抽象成一棵决策树：

- 每一层处理一个位置
- 每个分支代表当前位置的一种选择
- 从根走到叶子的路径，就是一个答案

以后再遇到这种“字符串组合”“路径枚举”“每一步有多个选项”的题，都可以先问自己一句：

**这是不是一棵决策树？**

如果答案是是的，那大概率就能用回溯来写。

### 完整代码

```java
import java.util.ArrayList;
import java.util.List;

public class Main {

    private static final String[] MAPPING = new String[] {
        "", "", "abc", "def", "ghi", "jkl", "mno", "pqrs", "tuv", "wxyz"
    };

    public static List<String> letterCombinations(String digits) {
        List<String> result = new ArrayList<>();
        if (digits == null || digits.length() == 0) {
            return result;
        }

        backtrack(result, digits, 0, new StringBuilder());
        return result;
    }

    private static void backtrack(List<String> result, String digits, int index, StringBuilder sb) {
        if (index == digits.length()) {
            result.add(sb.toString());
            return;
        }

        int digit = digits.charAt(index) - '0';
        String letters = MAPPING[digit];

        for (int i = 0; i < letters.length(); i++) {
            sb.append(letters.charAt(i));
            backtrack(result, digits, index + 1, sb);
            sb.deleteCharAt(sb.length() - 1);
        }
    }

    public static void main(String[] args) {
        String digits1 = "23";
        System.out.println("示例 1 输入: digits = \"23\"");
        List<String> res1 = letterCombinations(digits1);
        System.out.println("示例 1 输出: " + res1);
        System.out.println("-----------------------------");

        String digits2 = "2";
        System.out.println("示例 2 输入: digits = \"2\"");
        List<String> res2 = letterCombinations(digits2);
        System.out.println("示例 2 输出: " + res2);
        System.out.println("-----------------------------");

        String digits3 = "";
        System.out.println("示例 3 输入: digits = \"\"");
        List<String> res3 = letterCombinations(digits3);
        System.out.println("示例 3 输出: " + res3);
    }
}
```
