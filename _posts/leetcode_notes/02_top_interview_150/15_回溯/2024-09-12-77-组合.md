## 题目

**描述：**

给定两个整数 `n` 和 `k`，返回范围 `[1, n]` 中所有可能的 `k` 个数的组合。

你可以按 **任何顺序** 返回答案。

**示例 1:**

- **输入:** `n = 4`, `k = 2`
- **输出:**

  ```
  [
    [2,4],
    [3,4],
    [2,3],
    [1,2],
    [1,3],
    [1,4],
  ]
  ```

**示例 2:**

- **输入:** `n = 1`, `k = 1`
- **输出:** `[[1]]`

**提示:**

- `1 <= n <= 20`
- `1 <= k <= n`

## 题解

### 这题本质上是在“从 1 到 n 里选 k 个数”

题目要求的是组合，不是排列。

这句话非常关键。

比如 `n = 4, k = 2`，那么 `[1,2]` 和 `[2,1]` 在这题里算同一种结果，不能重复统计。  
所以这题并不是“把数字排出所有顺序”，而是：

**从 `[1, n]` 里挑出 `k` 个数。**

一旦这么理解，思路就很清楚了：  
我们要做的，是把所有合法的选法都枚举出来。

### 为什么这题适合用回溯

这题很适合画成一棵决策树。

还是拿 `n = 4, k = 2` 来看。

第一步，你可以先选：

- 1
- 2
- 3
- 4

如果第一步选了 1，那么第二步只能从 2、3、4 里继续选。  
如果第一步选了 2，那么第二步只能从 3、4 里继续选。

也就是说，每往下一层，选择范围都会缩小，而且后面的数字只能比前面的大。  
这其实就是一个很典型的“路径枚举”问题，天然适合回溯。

### 为什么后面只能从更大的数开始选

这是这题最关键的地方。

因为组合不看顺序，所以一旦我们已经选了 `1`，下一层就没必要再去选 `1` 前面的数字了。  
否则就会出现重复情况，比如：

- 先选 `1` 再选 `2`，得到 `[1,2]`
- 先选 `2` 再选 `1`，又得到 `[2,1]`

这两种在组合里其实是同一个答案。

所以为了去重，我们可以规定：

**下一层只能从当前数字的后一个位置开始选。**

这样做之后，每一组答案天然就是升序的，也就不会重复。

### 回溯里到底维护什么

这题写回溯时，核心就两个状态：

- `path`：当前已经选了哪些数字
- `startIndex`：这一层从哪个数字开始选

`path` 表示当前路径。  
比如现在 `path = [1,3]`，就说明我们已经选了 1 和 3。

`startIndex` 表示下一层搜索的起点。  
比如当前最后选的是 3，那下一层就只能从 4 开始继续选。

终止条件也很直接：

**当 `path` 的长度等于 `k` 时，说明已经凑够了一个合法组合，可以加入结果集。**

### 用 `n = 4, k = 2` 走一遍

我们从 1 开始。

先选 1，路径变成：

```text
[1]
```

这时候还差一个数，所以第二层可以从 2 开始选：

- 选 2，得到 `[1,2]`
- 回退
- 选 3，得到 `[1,3]`
- 回退
- 选 4，得到 `[1,4]`

然后回到第一层，撤销 1，改选 2：

```text
[2]
```

接着第二层只能从 3 开始：

- 选 3，得到 `[2,3]`
- 回退
- 选 4，得到 `[2,4]`

再回到第一层，选 3：

- 第二层只能选 4，得到 `[3,4]`

这样所有组合就都出来了。

你会发现，这个过程本质上就是：

**每一层选一个数，递归去补齐后面的数。**

### 基础回溯写法

```java
List<List<Integer>> result = new ArrayList<>();
LinkedList<Integer> path = new LinkedList<>();

public List<List<Integer>> combine(int n, int k) {
    backtrack(n, k, 1);
    return result;
}

private void backtrack(int n, int k, int startIndex) {
    if (path.size() == k) {
        result.add(new ArrayList<>(path));
        return;
    }

    for (int i = startIndex; i <= n; i++) {
        path.add(i);
        backtrack(n, k, i + 1);
        path.removeLast();
    }
}
```

这个版本已经能正确做出来了。

逻辑很简单：

- 当前层从 `startIndex` 开始枚举可选数字
- 选一个，加入路径
- 递归处理下一层
- 回来以后撤销这次选择

这是最标准的回溯模板。

### 为什么还可以剪枝

虽然上面的写法已经对了，但有些分支其实根本不用走。

比如 `n = 4, k = 2`。

如果第一层已经走到 `4`，那路径是 `[4]`。  
这时候后面已经没有数字可选了，但我们还差一个数，显然不可能凑成长度为 2 的组合。

也就是说：

**如果“当前已经选的个数 + 后面剩余可选的个数”仍然不够 `k`，那这一支就没必要继续搜了。**

这就是剪枝。

### 剪枝上界怎么来的

假设当前路径长度是 `path.size()`，那么还需要选的数字个数就是：

```text
k - path.size()
```

如果当前从 `i` 开始选，那么从 `i` 到 `n` 一共还剩：

```text
n - i + 1
```

为了让这一支还有机会凑够 `k` 个数，必须满足：

```text
n - i + 1 >= k - path.size()
```

整理一下就是：

```text
i <= n - (k - path.size()) + 1
```

所以 for 循环的上界就可以直接写成这个式子。  
超过这个范围的起点，后面剩下的数字根本不够，不用再试。

### 加上剪枝后的写法

```java
List<List<Integer>> result = new ArrayList<>();
LinkedList<Integer> path = new LinkedList<>();

public List<List<Integer>> combine(int n, int k) {
    backtrack(n, k, 1);
    return result;
}

private void backtrack(int n, int k, int startIndex) {
    if (path.size() == k) {
        result.add(new ArrayList<>(path));
        return;
    }

    int limit = n - (k - path.size()) + 1;
    for (int i = startIndex; i <= limit; i++) {
        path.add(i);
        backtrack(n, k, i + 1);
        path.removeLast();
    }
}
```

这版和前面的区别，只是把循环上界从 `n` 缩小成了 `limit`。  
逻辑不变，但会少走很多无意义分支。

### 为什么存结果时要 new 一个新列表

这也是回溯题里一个很容易踩的点。

`path` 在整个递归过程中是不断变化的：

- 进去时加元素
- 回来时删元素

如果你直接把 `path` 本身放进结果集，后面继续回溯时，这个对象里的内容也会跟着变，最后结果就全乱了。

所以每次找到一组合法答案时，都必须这样写：

```java
result.add(new ArrayList<>(path));
```

意思是把当前路径内容拷贝一份保存下来，而不是保存同一个引用。

### 再看复杂度

这题最终要输出的答案数量就是组合数：

$$
\binom{n}{k}
$$

而每找到一组答案，都要把当前路径复制一份，复制长度是 `k`。  
所以时间复杂度通常写成：

$$
O\left(\binom{n}{k} \cdot k\right)
$$

空间复杂度主要来自递归栈和当前路径，都是 $O(k)$。  
如果按更宽松的方式写成 $O(n)$ 也可以，因为递归深度不会超过 `n`，不过从这题实际状态来看，说 $O(k)$ 更贴切一些。

当然，如果把最终结果集本身占的空间也算进去，那会更大，因为答案本来就有这么多组。

### 这题真正要记住的点

这题最值得记住的，不是组合数公式，也不是剪枝公式本身，而是这几个判断：

第一，这题是组合，不是排列，所以必须想办法避免顺序不同造成的重复。  
第二，避免重复最自然的方法，就是让每一层都从上一次选择的后面开始搜。  
第三，回溯里一旦能判断“后面剩下的数已经不够用了”，就可以提前停掉，这就是剪枝。

以后再碰到这种“从一个范围里选若干个数”的题，基本都可以先往这个模板上套：

- 用 `path` 记录当前选了什么
- 用 `startIndex` 控制下一层从哪开始
- 用 `path.size() == k` 作为终止条件
- 能剪枝就尽量剪枝

这类题本质上都很像。

### 完整代码

```java
import java.util.ArrayList;
import java.util.LinkedList;
import java.util.List;

public class Main {

    static List<List<Integer>> result = new ArrayList<>();
    static LinkedList<Integer> path = new LinkedList<>();

    public static List<List<Integer>> combine(int n, int k) {
        result.clear();
        path.clear();
        backtracking(n, k, 1);
        return result;
    }

    private static void backtracking(int n, int k, int startIndex) {
        if (path.size() == k) {
            result.add(new ArrayList<>(path));
            return;
        }

        int limit = n - (k - path.size()) + 1;
        for (int i = startIndex; i <= limit; i++) {
            path.add(i);
            backtracking(n, k, i + 1);
            path.removeLast();
        }
    }

    public static void main(String[] args) {
        int n1 = 4;
        int k1 = 2;
        System.out.println("示例 1 输入: n = " + n1 + ", k = " + k1);
        List<List<Integer>> res1 = combine(n1, k1);
        System.out.println("示例 1 输出: " + res1);
        System.out.println("-----------------------------");

        int n2 = 1;
        int k2 = 1;
        System.out.println("示例 2 输入: n = " + n2 + ", k = " + k2);
        List<List<Integer>> res2 = combine(n2, k2);
        System.out.println("示例 2 输出: " + res2);
        System.out.println("-----------------------------");

        int n3 = 5;
        int k3 = 4;
        System.out.println("示例 3 输入: n = " + n3 + ", k = " + k3);
        List<List<Integer>> res3 = combine(n3, k3);
        System.out.println("示例 3 输出: " + res3);
    }
}
```
