## 题目

**描述：**

给你一个 **无重复元素** 的整数数组 `candidates` 和一个目标整数 `target`，找出 `candidates` 中可以使数字和为目标数 `target` 的所有 **不同组合**，并以列表形式返回。你可以按 **任意顺序** 返回这些组合。

`candidates` 中的 **同一个** 数字可以 **无限制重复被选取**。如果至少一个数字的被选数量不同，则两种组合是不同的。

对于给定的输入，保证和为 `target` 的不同组合数少于 `150` 个。

**示例 1:**

- **输入:** `candidates = [2,3,6,7]`, `target = 7`
- **输出:** `[[2,2,3],[7]]`
- **解释:**

  2 和 3 可以形成一组候选，2 + 2 + 3 = 7。注意 2 可以使用多次。

  7 也是一个候选， 7 = 7。

  仅有这两种组合。

**示例 2:**

- **输入:** `candidates = [2,3,5]`, `target = 8`
- **输出:** `[[2,2,2,2],[2,3,3],[3,5]]`

**示例 3:**

- **输入:** `candidates = [2]`, `target = 1`
- **输出:** `[]`

**提示:**

- `1 <= candidates.length <= 30`
- `2 <= candidates[i] <= 40`
- `candidates` 的所有元素 **互不相同**
- `1 <= target <= 40`

## 题解

### 这题不是在“挑不同数字”，而是在“凑目标和”

这题第一眼看上去和组合题有点像，但核心其实不一样。

它不是单纯让你从数组里选几个数，而是要求你：

**选出若干个数字，使它们的和刚好等于 `target`。**

而且这里还有两个很关键的条件：

- 每个数字可以重复使用
- 结果里不能出现重复组合

这两个条件决定了，这题虽然还是回溯，但写法和普通组合题不完全一样。

### 这题为什么还是回溯

因为本质上你还是在做“枚举所有可能选择”。

假设 `candidates = [2,3,6,7]`，`target = 7`。

那你可以这样想：

- 先试着选 2
- 选完以后，剩下还要凑 5
- 接着继续选 2、3、6、7 中能用的数
- 如果某条路径最后刚好凑成 7，那就是一个答案
- 如果某条路径已经超过 7 了，那这条路就不用继续走了

这就是一个非常典型的回溯过程：

- 路径：当前已经选了哪些数
- 目标：当前还剩多少没凑出来
- 选择：接下来还可以从哪些数字里继续选

### 这题最关键的地方有两个

第一，**同一个数字可以重复使用**。  
这意味着如果这层选了 `2`，下一层仍然可以继续选 `2`。

第二，**组合不能重复**。  
也就是说 `[2,2,3]` 和 `[2,3,2]` 在这题里其实是同一个结果，不能都算进去。

所以这题看起来有点矛盾：

- 一方面允许重复选
- 另一方面又不允许结果重复

解决办法就是：

**递归时不回头选前面的数。**

也就是说，当前层从下标 `i` 开始选，下一层仍然从 `i` 开始，而不是从 `0` 重新开始。  
这样既能保证一个数字可以重复使用，又能保证组合始终按固定顺序生成，不会出现顺序不同但内容相同的重复答案。

### 为什么递归时传 `i`，不是 `i + 1`

这是这题最容易写错的地方。

如果某一层你选了 `candidates[i]`，下一层：

- 传 `i`，表示这个数还能继续选
- 传 `i + 1`，表示这个数以后不能再选

而题目明确说了“同一个数字可以无限制重复被选取”，所以这里必须传 `i`。

比如：

```text
candidates = [2,3,6,7], target = 7
```

当你第一步选了 2，后面还可以继续选 2，才能得到 `[2,2,3]`。  
如果下一层直接传 `i + 1`，那就永远凑不出这个答案了。

所以这题和“组合”那题最大的区别之一就是：

- 普通组合：下一层传 `i + 1`
- 组合总和：下一层传 `i`

### 用示例 1 走一遍

示例 1：

```text
candidates = [2,3,6,7], target = 7
```

先从 2 开始。

路径变成：

```text
[2]
```

剩余目标是 5。  
下一层仍然可以从 2 开始选。

再选一个 2：

```text
[2,2]
```

剩余目标变成 3。  
继续从 2 开始选：

- 选 2，剩余目标变成 1，这时后面所有数都太大了，这条路走不通
- 回退
- 选 3，剩余目标刚好变成 0，得到一个合法答案 `[2,2,3]`

然后继续回退，尝试别的分支。  
比如第一层直接选 7，也能得到另一个答案 `[7]`。

所以最终结果是：

```text
[[2,2,3],[7]]
```

你会发现，这题其实就是不断做这件事：

**选一个数，把 target 减掉，然后递归去解决更小的子问题。**

### 为什么排序以后可以剪枝

这题还有一个很常见的优化：先排序。

排序之后，数组是递增的。  
这样在某一层遍历时，如果发现当前数字已经大于剩余目标值了，那后面的数字只会更大，肯定也不可能满足条件。

比如当前剩余目标是 3，而你已经看到 `6` 了，那后面的 `7`、`8` 更不可能行。  
这时就没必要继续往后试，直接 `break` 就可以。

这就是排序带来的剪枝效果。

所以这题常见写法里，第一步都会先：

```java
Arrays.sort(candidates);
```

### 回溯写法

```java
public List<List<Integer>> combinationSum(int[] candidates, int target) {
    List<List<Integer>> result = new ArrayList<>();
    Arrays.sort(candidates);
    backtrack(result, new ArrayList<>(), candidates, target, 0);
    return result;
}

private void backtrack(List<List<Integer>> result, List<Integer> path,
                       int[] candidates, int remainingTarget, int start) {
    if (remainingTarget == 0) {
        result.add(new ArrayList<>(path));
        return;
    }

    for (int i = start; i < candidates.length; i++) {
        if (candidates[i] > remainingTarget) {
            break;
        }

        path.add(candidates[i]);
        backtrack(result, path, candidates, remainingTarget - candidates[i], i);
        path.remove(path.size() - 1);
    }
}
```

### 这段代码到底在做什么

可以拆成三件事来看。

第一件事，`remainingTarget` 表示还差多少。  
每选一个数，就把它减掉。

第二件事，`start` 控制这一层从哪开始选。  
这样可以保证组合按固定顺序生成，避免 `[2,3]` 和 `[3,2]` 这种重复。

第三件事，找到答案或者提前停止：

- 如果 `remainingTarget == 0`，说明刚好凑出来了，加入结果集
- 如果当前数已经大于剩余目标，后面不用再试，直接剪枝

整套逻辑其实很统一：

**不断尝试一个选择，递归去凑剩余值；不行就回退。**

### 为什么不会出现重复组合

因为我们始终规定：

**下一层只能从当前下标及其后面继续选。**

这会让路径中的数字天然保持非递减顺序。

比如 `[2,2,3]` 可以生成，但 `[2,3,2]` 不会生成。  
因为一旦你已经走到 3 这个位置，后面就不会再回去选 2 了。

也正因为这个限制，重复组合会被直接挡在搜索树之外，而不是生成完以后再去重。

这比用 `Set` 去重更自然，也更高效。

### 再看复杂度

这题的时间复杂度很难像排列组合那样写成一个特别整齐的式子，因为它取决于搜索树实际展开的情况。

从本质上说，它还是指数级搜索。  
但因为：

- `target` 不大
- `candidates` 数值不大
- 结果数量本身也被题目限制得比较小
- 还做了排序剪枝

所以实际运行通常很快。

空间复杂度主要来自递归栈和当前路径。  
最深情况下，如果一直选最小的数，递归深度大约是：

$$
O\left(\frac{target}{\min(candidates)}\right)
$$

如果按题目这种规模来看，这个开销是完全可接受的。

### 这题真正要记住的点

这题最值得记住的，其实是回溯题里两个非常常见的判断：

第一，题目是否允许一个数重复使用。  
如果允许，递归时通常就要考虑“下一层还能不能继续从当前下标开始”。

第二，题目虽然是组合，但是否需要靠固定搜索顺序来避免重复。  
这题就是典型例子：不用额外去重结构，只要控制好 `start`，重复答案自然不会出现。

所以以后再遇到类似题，可以先问自己三件事：

1. 数字能不能重复选
2. 顺序算不算不同答案
3. 能不能通过排序做剪枝

这三件事想清楚了，回溯框架基本就定下来了。

### 完整代码

```java
import java.util.ArrayList;
import java.util.Arrays;
import java.util.List;

public class Main {

    public static List<List<Integer>> combinationSum(int[] candidates, int target) {
        List<List<Integer>> result = new ArrayList<>();
        if (candidates == null || candidates.length == 0) {
            return result;
        }

        Arrays.sort(candidates);
        backtrack(result, new ArrayList<>(), candidates, target, 0);
        return result;
    }

    private static void backtrack(List<List<Integer>> result, List<Integer> path,
                                  int[] candidates, int remainingTarget, int start) {
        if (remainingTarget == 0) {
            result.add(new ArrayList<>(path));
            return;
        }

        for (int i = start; i < candidates.length; i++) {
            if (candidates[i] > remainingTarget) {
                break;
            }

            path.add(candidates[i]);
            backtrack(result, path, candidates, remainingTarget - candidates[i], i);
            path.remove(path.size() - 1);
        }
    }

    public static void main(String[] args) {
        int[] candidates1 = {2, 3, 6, 7};
        int target1 = 7;
        System.out.println("示例 1 输入: candidates = " + Arrays.toString(candidates1) + ", target = " + target1);
        List<List<Integer>> result1 = combinationSum(candidates1, target1);
        System.out.println("示例 1 输出: " + result1);
        System.out.println("-----------------------------");

        int[] candidates2 = {2, 3, 5};
        int target2 = 8;
        System.out.println("示例 2 输入: candidates = " + Arrays.toString(candidates2) + ", target = " + target2);
        List<List<Integer>> result2 = combinationSum(candidates2, target2);
        System.out.println("示例 2 输出: " + result2);
        System.out.println("-----------------------------");

        int[] candidates3 = {2};
        int target3 = 1;
        System.out.println("示例 3 输入: candidates = " + Arrays.toString(candidates3) + ", target = " + target3);
        List<List<Integer>> result3 = combinationSum(candidates3, target3);
        System.out.println("示例 3 输出: " + result3);
    }
}
```
