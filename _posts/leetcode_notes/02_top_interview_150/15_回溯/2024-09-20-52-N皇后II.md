## 题目

**描述：**

**n 皇后问题** 研究的是如何将 `n` 个皇后放置在 `n x n` 的棋盘上，并且使皇后彼此之间不能相互攻击。

给你一个整数 `n`，返回 **n 皇后问题** 不同的解决方案的数量。

(注：皇后可以攻击同一行、同一列、以及两条对角线上的任何棋子。)

**示例 1:**

![示例1图片](https://assets.leetcode.com/uploads/2020/11/13/queens.jpg)

- **输入:** `n = 4`
- **输出:** 2
- **解释:** 如上图所示，4 皇后问题存在两个不同的解法。

**示例 2:**

- **输入:** `n = 1`
- **输出:** 1

**提示:**

- `1 <= n <= 9`

## 题解

### 这题和 N 皇后不一样，它只要你数方案数

如果是经典的 N 皇后题，通常会让你把每一种棋盘摆法都输出出来。  
但这题不需要你真的把棋盘保存下来，它只问：

**一共有多少种合法摆法。**

这意味着我们不需要维护整张棋盘，也不需要在找到答案时把当前布局转成字符串列表。  
只要在搜索过程中判断当前位置能不能放皇后，并在放满 `n` 行时把答案计数加一就够了。

所以这题本质上还是回溯，只不过目标从“构造所有解”变成了“统计解的数量”。

### 为什么可以按“逐行放皇后”来做

皇后的攻击范围是：

- 同一行
- 同一列
- 两条对角线

而每一行最终都必须恰好放一个皇后。  
那最自然的做法就是：

**一行一行地放。**

比如现在正在处理第 `row` 行，我们只需要尝试这一行的每一列：

- 如果当前位置不会和前面行的皇后冲突，就放下去
- 然后递归处理下一行
- 如果冲突，就跳过这一列

这样做有一个很大的好处：

因为我们是“逐行放”，所以“同一行冲突”这个条件天然就已经满足了，不需要额外检查。  
真正需要处理的，只剩下：

- 当前列有没有皇后
- 两条对角线有没有皇后

### 这题真正关键的是怎么判断冲突

假设我们要在 `(row, col)` 放一个皇后。

那它会和前面已经放过的皇后冲突，只可能发生在三种地方：

1. 同列
2. 主对角线（左上到右下）
3. 副对角线（右上到左下）

列很好理解，直接记录哪一列已经被占用就行。

对角线稍微抽象一点，但规律其实很固定：

- 同一条主对角线上的格子，`row - col` 相同
- 同一条副对角线上的格子，`row + col` 相同

所以只要我们能记录哪些 `row - col` 和 `row + col` 已经出现过，就能快速判断当前位置是否合法。

### 为什么 `row - col` 要加偏移量

因为 `row - col` 可能是负数。

比如在 `4 x 4` 棋盘里：

- `(0, 3)` 的 `row - col = -3`
- `(3, 0)` 的 `row - col = 3`

数组下标不能是负数，所以通常会给它加一个偏移量，比如加 `n`。  
这样：

```text
d1 = row - col + n
```

就能把所有主对角线映射到合法下标范围里。

而 `row + col` 本身不会是负数，所以副对角线可以直接用：

```text
d2 = row + col
```

### 用 `n = 4` 体会一下搜索过程

以 `n = 4` 为例。

第一行可以尝试把皇后放在第 0、1、2、3 列中的任意一列。  
假设第一行放在 `(0,1)`，那么：

- 第 1 列不能再放
- 和 `(0,1)` 同主对角线、同副对角线的位置也都不能放

接着递归到第二行，只尝试剩下合法的位置。

如果某一行发现没有任何位置可放，说明前面的选择走错了，就要回退到上一行，换一个列重新试。  
如果一直成功放到了第 `n` 行之后，说明前面 `0` 到 `n-1` 行都已经合法放完了，这就是一种有效方案，答案加一。

所以这题的本质其实非常标准：

**试一个位置，递归；不行就回退，继续试下一个位置。**

这就是回溯最典型的样子。

### 回溯里到底维护什么

这题常见写法里，会维护三个布尔数组：

- `cols[col]`：第 `col` 列是否已经有皇后
- `diag1[d1]`：某条主对角线是否已经有皇后
- `diag2[d2]`：某条副对角线是否已经有皇后

递归函数通常写成：

```java
backtrack(row, ...)
```

意思是：

**现在要处理第 `row` 行，返回从这一行开始往下能找到多少种解。**

终止条件也很明确：

如果 `row == n`，说明前 `n` 行都已经合法放好了，这就是一种完整方案，直接返回 1。

### 回溯写法

```java
public int totalNQueens(int n) {
    boolean[] cols = new boolean[n];
    boolean[] diag1 = new boolean[2 * n];
    boolean[] diag2 = new boolean[2 * n];
    return backtrack(0, n, cols, diag1, diag2);
}

private int backtrack(int row, int n, boolean[] cols, boolean[] diag1, boolean[] diag2) {
    if (row == n) {
        return 1;
    }

    int count = 0;
    for (int col = 0; col < n; col++) {
        int d1 = row - col + n;
        int d2 = row + col;

        if (cols[col] || diag1[d1] || diag2[d2]) {
            continue;
        }

        cols[col] = true;
        diag1[d1] = true;
        diag2[d2] = true;

        count += backtrack(row + 1, n, cols, diag1, diag2);

        cols[col] = false;
        diag1[d1] = false;
        diag2[d2] = false;
    }

    return count;
}
```

### 这段代码为什么是对的

可以从“每一层递归在干什么”这个角度去理解。

每一层递归对应棋盘的一行。  
在这一行里，我们尝试所有列：

- 如果某列冲突，就跳过
- 如果不冲突，就把皇后放下去，并更新列和对角线的占用状态
- 然后递归到下一行
- 下一行处理完之后，再把当前这一步撤销掉

因为每一行都尝试了所有可能位置，所以不会漏掉合法解；  
因为每次放置前都检查了列和对角线，所以不会把非法摆法算进去；  
因为到达 `row == n` 时，说明每一行都放了一个合法皇后，所以此时返回 1 是合理的。

### 为什么这里不用真的维护棋盘

因为题目只要求方案数，不要求输出具体布局。

那我们真正关心的其实不是“这一格长什么样”，而是：

**这个位置能不能放。**

而判断能不能放，只需要知道列和对角线是否被占用。  
所以没必要真的维护一个 `char[][] board`。

这也是为什么这题可以写得比 N 皇后 I 更轻一些：  
状态更少，逻辑也更集中。

### 再看复杂度

这题的时间复杂度通常写成 $O(N!)$。

因为第一行最多有 `N` 种选择，第二行在冲突约束下会更少，后面继续递减，本质上仍然是一个指数级搜索问题。  
当然，实际运行会比简单的 `N!` 好一些，因为列和对角线限制会剪掉很多分支。

空间复杂度是 $O(N)$。

主要来自：

- 三个状态数组
- 递归调用栈深度最多为 `N`

如果不把最终答案存成具体棋盘，这题的辅助空间其实很省。

### 位运算为什么更快

这题还有一个非常经典的优化版本，就是位运算。

核心思路没变，还是在记录：

- 哪些列被占了
- 哪些主对角线被占了
- 哪些副对角线被占了

只不过普通写法是用布尔数组，位运算写法是直接用整数的二进制位来表示这些状态。

比如某一位是 1，表示该列或该对角线已被占用；  
某一位是 0，表示当前位置还可以尝试。

这样做的优势是：

- 状态压缩得更紧
- 判断冲突、更新状态都变成了位运算
- 常数开销会更小

但从教学角度说，第一次写这题时，数组版更容易理解。

### 位运算写法

```java
public int totalNQueensBitmap(int n) {
    return solve(n, 0, 0, 0, 0);
}

private int solve(int n, int row, int col, int ld, int rd) {
    if (row == n) {
        return 1;
    }

    int count = 0;
    int availableBits = ~(col | ld | rd) & ((1 << n) - 1);

    while (availableBits != 0) {
        int position = availableBits & -availableBits;
        availableBits &= (availableBits - 1);

        count += solve(n, row + 1,
                col | position,
                (ld | position) << 1,
                (rd | position) >> 1);
    }

    return count;
}
```

这段代码的核心思想还是一样的：  
先算出当前行所有能放的位置，然后逐个尝试。

只是这里“能放的位置”不再靠 for 循环逐列检查，而是直接通过二进制位一次性算出来。

### 这题真正要记住的点

这题最值得记住的，不是 N 皇后本身，而是这种回溯建模方式：

**如果题目要求在一个结构里逐步放元素，并且每放一步都会影响后面的合法性，那么通常可以按“行 / 层”递归，用若干状态结构快速判断冲突。**

这类题的关键永远不是暴力枚举，而是：

1. 递归的层代表什么
2. 当前选择会影响哪些约束
3. 如何用尽量轻的状态快速判断是否合法

N 皇后就是这套思路的典型代表。

### 完整代码

```java
public class Main {

    public static int totalNQueens(int n) {
        boolean[] cols = new boolean[n];
        boolean[] diag1 = new boolean[2 * n];
        boolean[] diag2 = new boolean[2 * n];

        return backtrack(0, n, cols, diag1, diag2);
    }

    private static int backtrack(int row, int n, boolean[] cols, boolean[] diag1, boolean[] diag2) {
        if (row == n) {
            return 1;
        }

        int count = 0;
        for (int col = 0; col < n; col++) {
            int d1 = row - col + n;
            int d2 = row + col;

            if (cols[col] || diag1[d1] || diag2[d2]) {
                continue;
            }

            cols[col] = true;
            diag1[d1] = true;
            diag2[d2] = true;

            count += backtrack(row + 1, n, cols, diag1, diag2);

            cols[col] = false;
            diag1[d1] = false;
            diag2[d2] = false;
        }

        return count;
    }

    public static void main(String[] args) {
        int n1 = 4;
        System.out.println("示例 1 输入: n = " + n1);
        int result1 = totalNQueens(n1);
        System.out.println("示例 1 输出: " + result1);
        System.out.println("-----------------------------");

        int n2 = 1;
        System.out.println("示例 2 输入: n = " + n2);
        int result2 = totalNQueens(n2);
        System.out.println("示例 2 输出: " + result2);
        System.out.println("-----------------------------");

        int n3 = 8;
        System.out.println("额外测试 输入: n = " + n3);
        int result3 = totalNQueens(n3);
        System.out.println("额外测试 输出: " + result3);
    }
}
```
