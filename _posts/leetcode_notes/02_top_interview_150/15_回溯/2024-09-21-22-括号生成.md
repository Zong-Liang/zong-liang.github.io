## 题目

**描述：**

数字 `n` 代表生成括号的对数，请你设计一个函数，用于能够生成所有可能的并且 **有效** 的括号组合。

**示例 1:**

- **输入:** `n = 3`
- **输出:** `["((()))","(()())","(())()","()(())","()()()"]`

**示例 2:**

- **输入:** `n = 1`
- **输出:** `["()"]`

**提示:**

- `1 <= n <= 8`

## 题解

### 这题不是“枚举所有括号串”，而是“只生成合法的括号串”

如果题目只是让你把长度为 `2n` 的括号字符串全部枚举出来，那做法会很简单：每个位置放 `(` 或 `)`，一共 `2^(2n)` 种可能。

但这题真正要求的是：

**只返回有效的括号组合。**

也就是说，像下面这些是合法的：

```text
(())
()()
((()))
```

而这些是不合法的：

```text
)(
(()
())(
```

所以这题真正的关键，不是“怎么枚举”，而是：

**怎么在构造过程中就把不合法的情况提前挡掉。**

这也是它为什么特别适合用回溯。

### 为什么这题适合回溯

这题可以理解成一个逐步填空的过程。

我们要构造一个长度为 `2n` 的字符串。  
每一步只有两种选择：

- 放一个左括号 `(`
- 放一个右括号 `)`

如果完全不加限制，搜索树确实很大。  
但问题在于，很多分支其实走到一半就已经注定不合法了，根本没必要继续。

比如刚开始第一个字符如果放 `)`，这条路就一定不对。  
因为一个合法括号串的任意前缀里，左括号数量都不能少于右括号数量。

所以这题最自然的思路就是：

**一边构造，一边检查当前状态是否合法；只继续扩展那些还有希望成为答案的分支。**

这正是回溯最典型的用法。

### 合法括号串最关键的约束是什么

这题只要抓住两个约束，代码就很顺。

第一个约束：

**左括号最多只能放 `n` 个。**

因为总共就只有 `n` 对括号。

第二个约束：

**任意时刻，右括号数量都不能超过左括号数量。**

这一条才是核心。  
因为只要某个前缀里右括号比左括号还多，那么这个前缀就已经不可能被补救成合法串了。

比如：

```text
")"
"())"
"())("
```

这些一出现就已经没救了，所以可以直接剪掉，不用继续往下搜。

### 递归时到底维护什么

这题写回溯时，通常维护三个状态就够了：

- `cur`：当前已经拼出来的字符串
- `open`：已经用了多少个左括号
- `close`：已经用了多少个右括号

递归的含义就是：

**基于当前这段前缀，继续往后补全所有可能的合法括号串。**

那什么时候结束呢？

当 `cur.length() == 2 * n` 时，说明长度已经够了。  
而且因为前面的递归过程始终都在遵守合法性约束，所以此时得到的一定是一个有效答案，可以直接加入结果集。

### 为什么“右括号只能在 `close < open` 时放”

这是整题最关键的一句判断。

假设当前已经用了：

- `open = 2`
- `close = 1`

那么当前前缀里还有一个左括号没有被匹配，这时候可以继续放右括号。

但如果当前是：

- `open = 2`
- `close = 2`

说明前缀已经完全配平了。  
这时候如果你再放一个右括号，就会出现右括号数量超过左括号数量的情况，立刻非法。

所以放右括号的前提必须是：

```java
close < open
```

这句判断，其实就是在保证：

**任何前缀都不会非法。**

### 用 `n = 3` 走一遍

我们从空串开始：

```text
""
```

此时：

- `open = 0`
- `close = 0`

因为左括号还没用满，所以可以先放 `(`：

```text
"("
```

现在：

- `open = 1`
- `close = 0`

这时候两种选择都可能合法：

- 继续放 `(`，得到 `"(("`
- 或者放 `)`，得到 `"()"`

先看 `"(("` 这条路。  
再继续放一个 `(`，得到：

```text
"((("
```

这时左括号已经用满 3 个了，后面只能不断补右括号，最终得到：

```text
"((()))"
```

而如果从 `"(()"` 这类前缀继续扩展，又会得到：

```text
"(()())"
"(())()"
```

从 `"()"` 这条路继续往下走，则会生成：

```text
"()(())"
"()()()"
```

所以最终所有结果就是：

```text
["((()))","(()())","(())()","()(())","()()()"]
```

你会发现，这题本质上就是：

**每一步尝试加括号，但只允许继续走那些还合法的前缀。**

### 回溯写法

```java
public List<String> generateParenthesis(int n) {
    List<String> result = new ArrayList<>();
    backtrack(result, "", 0, 0, n);
    return result;
}

private void backtrack(List<String> result, String cur, int open, int close, int n) {
    if (cur.length() == 2 * n) {
        result.add(cur);
        return;
    }

    if (open < n) {
        backtrack(result, cur + "(", open + 1, close, n);
    }

    if (close < open) {
        backtrack(result, cur + ")", open, close + 1, n);
    }
}
```

### 这段代码为什么是对的

可以分成两部分理解。

第一部分，是“什么时候能放左括号”。

只要左括号还没用满，也就是 `open < n`，就可以继续放。  
因为最终总共要放 `n` 个左括号，这一点不会错。

第二部分，是“什么时候能放右括号”。

只有当前还有没匹配完的左括号时，才允许放右括号，也就是 `close < open`。  
这确保了任何时刻都不会出现非法前缀。

而一旦长度达到 `2n`，说明括号总数已经够了。  
又因为在整个构造过程中我们始终没有允许非法状态出现，所以这个结果必然是合法括号串。

换句话说，这段代码不是“先生成，再筛选”，而是“从一开始就只生成合法答案”。  
这也是它高效的原因。

### 为什么这题不需要额外判重

因为整个搜索树里，每条路径都是靠“当前前缀 + 下一步放什么括号”唯一确定的。

比如：

- 从 `"("` 走到 `"()"`
- 再走到 `"()("`
- 再走到 `"()()"`

这条路径不会用别的方式再次生成一遍。  
所以结果天然不会重复，不需要像有些排列组合题那样再去额外去重。

### 再看复杂度

这题最终生成的合法括号串数量，其实就是第 `n` 个卡特兰数。  
严格写复杂度会比较复杂，但对做题来说，记住这件事就够了：

**时间复杂度取决于最终合法答案的数量。**

因为我们本来就要把所有答案都生成出来，所以不可能比“答案总数”更低。

空间复杂度主要来自递归深度。  
字符串长度最多是 `2n`，所以递归层数也是 `O(n)` 量级。

如果把结果集本身占用的空间也算上，那当然会更大，因为答案本身就有很多个。

### 还能怎么优化写法

你原文用的是字符串拼接：

```java
cur + "("
cur + ")"
```

这个写法完全没问题，题目数据范围也很小，`n <= 8`，性能不是瓶颈。

如果更讲究一点，也可以改成 `StringBuilder` 版本，像常规回溯那样“追加一个字符，再删掉最后一个字符”。  
但这题里直接用字符串会更直观，教学上反而更容易解释，所以保留这种写法是合理的。

### 这题真正要记住的点

这题最值得记住的，不是括号本身，而是这种回溯思路：

**如果题目要求你生成所有合法结果，那重点往往不是“怎么把所有东西都枚举出来”，而是“怎么在构造过程中就保证合法”。**

这类题和普通枚举题的区别就在这里：

- 普通枚举：先全列出来，再筛
- 回溯剪枝：只走那些还有希望成为答案的分支

而这题最核心的合法性条件，就是：

**任意前缀中，左括号数量都不能少于右括号数量。**

以后再遇到类似“生成所有合法状态”的题，都可以先想一想：

**有没有什么前缀条件，一旦不满足，就可以立刻剪枝。**

很多时候，关键就藏在这里。

### 完整代码

```java
import java.util.ArrayList;
import java.util.List;

public class Main {

    public static List<String> generateParenthesis(int n) {
        List<String> result = new ArrayList<>();
        if (n <= 0) {
            return result;
        }
        backtrack(result, "", 0, 0, n);
        return result;
    }

    private static void backtrack(List<String> result, String cur, int open, int close, int n) {
        if (cur.length() == 2 * n) {
            result.add(cur);
            return;
        }

        if (open < n) {
            backtrack(result, cur + "(", open + 1, close, n);
        }

        if (close < open) {
            backtrack(result, cur + ")", open, close + 1, n);
        }
    }

    public static void main(String[] args) {
        int n1 = 3;
        System.out.println("示例 1 输入: n = " + n1);
        List<String> result1 = generateParenthesis(n1);
        System.out.println("示例 1 输出: " + result1);
        System.out.println("-----------------------------");

        int n2 = 1;
        System.out.println("示例 2 输入: n = " + n2);
        List<String> result2 = generateParenthesis(n2);
        System.out.println("示例 2 输出: " + result2);
    }
}
```
