## 题目

**描述：**

给你一个变量对数组 `equations` 和一个实数值数组 `values` 作为已知条件，其中 `equations[i] = [Ai, Bi]` 和 `values[i]` 共同表示等式 `Ai / Bi = values[i]`。每个 `Ai` 或 `Bi` 是一个表示单个变量的字符串。

另有一些以数组 `queries` 表示的问题，其中 `queries[j] = [Cj, Dj]` 表示第 `j` 个问题，请你根据已知条件找出 `Cj / Dj = ?` 的结果作为答案。

返回 **所有问题的答案**。如果存在某个无法确定的答案，则用 `-1.0` 替代这个答案。如果问题中出现了给定的已知条件中没有出现的字符串，也需要用 `-1.0` 替代这个答案。

**注意：** 输入总是有效的。你可以假设除法运算中不会出现除数为 0 的情况，且不存在任何矛盾的结果。

**注意：** 未在等式列表中出现的变量是未定义的，因此无法确定它们的答案。

**示例 1:**

- **输入:** `equations = [["a","b"],["b","c"]], values = [2.0,3.0], queries = [["a","c"],["b","a"],["a","e"],["a","a"],["x","x"]]`
- **输出:** `[6.00000,0.50000,-1.00000,1.00000,-1.00000]`

**示例 2:**

- **输入:** `equations = [["a","b"],["b","c"],["bc","cd"]], values = [1.5,2.5,5.0], queries = [["a","c"],["c","b"],["bc","cd"],["cd","bc"]]`
- **输出:** `[3.75000,0.40000,5.00000,0.20000]`

**提示:**

- `1 <= equations.length <= 20`
- `1 <= queries.length <= 20`
- 变量由小写英文字母与数字组成

## 题解

### 这题表面是除法，实际上是找路径

这题第一次看，容易把注意力放在“怎么算除法”上。  
但其实题目真正给你的，不是一堆独立的算式，而是一组**变量之间的相对关系**。

比如已知：

```text
a / b = 2.0
b / c = 3.0
```

那你想求 `a / c`，本质上并不是直接算，而是顺着这两条关系接起来：

```text
a / c = (a / b) * (b / c) = 2.0 * 3.0 = 6.0
```

所以这题的关键不是“做除法”，而是：

**能不能从起点变量一路找到终点变量，并把沿途的权重乘起来。**

一旦这样理解，题目就很自然地变成图论问题了。

### 为什么这题可以建图

每个变量都可以看成图里的一个节点。  
如果题目给出：

```text
a / b = 2.0
```

那就说明：

- 从 `a` 到 `b` 有一条边，权重是 `2.0`
- 从 `b` 到 `a` 也有一条边，权重是 `1 / 2.0 = 0.5`

因为知道了一个方向的比值，反方向其实也就跟着确定了。

所以整张图里：

- 节点表示变量
- 边表示变量之间的除法关系
- 边权表示这个除法结果

这样一来，原题里的每个查询 `C / D`，就变成了：

**图中是否存在一条从 `C` 到 `D` 的路径。**

如果存在，路径上所有边权的乘积就是答案；如果不存在，答案就是 `-1.0`。

### 用示例 1 看一下

已知：

```text
a / b = 2.0
b / c = 3.0
```

对应的图可以理解成：

```text
a --2.0--> b --3.0--> c
a <--0.5-- b <--1/3-- c
```

那几个查询就很好理解了：

- `a / c`：从 `a -> b -> c`，结果是 `2.0 * 3.0 = 6.0`
- `b / a`：直接走 `b -> a`，结果是 `0.5`
- `a / e`：图里根本没有 `e`，所以返回 `-1.0`
- `a / a`：只要 `a` 这个变量存在，自己除以自己就是 `1.0`
- `x / x`：`x` 压根没出现过，所以不是 `1.0`，而是 `-1.0`

这里最后一种情况特别容易写错。  
**变量自己除以自己等于 1.0 的前提，是这个变量必须已经在已知关系里出现过。**

### 这题其实就分两步

第一步，先把图建出来。  
遍历 `equations` 和 `values`，把双向边都加进去。

第二步，逐个处理查询。  
对于每个查询：

- 如果起点或终点不在图里，直接返回 `-1.0`
- 如果起点和终点相同，并且这个变量存在，返回 `1.0`
- 否则从起点出发做一次搜索，看看能不能走到终点

这题用 DFS 或 BFS 都能做。  
因为我们要的不是最短路，也不是最优解，只要找到任意一条可达路径，把沿途权重乘起来就行。

### 为什么 DFS 很适合这题

DFS 很适合做这种“从一个点出发，看看能不能走到另一个点”的问题。

搜索时我们需要维护两个东西：

1. 当前走到了哪个变量
2. 从起点一路乘到这里的结果是多少

比如从 `a` 出发，初始乘积就是 `1.0`。  
如果第一步走到 `b`，边权是 `2.0`，那当前累积值就变成 `2.0`。  
接着从 `b` 走到 `c`，边权是 `3.0`，那累积值就变成 `2.0 * 3.0 = 6.0`。

当搜索到终点时，直接返回这个累积值就行。

### 搜索时为什么一定要用 visited

因为图里可能有环。

比如：

```text
a / b = 2.0
b / c = 3.0
c / a = ...
```

你从 `a` 出发搜索时，如果不记录访问过的节点，就可能在 `a -> b -> c -> a` 这种路径里一直绕圈。

所以 DFS 时必须带一个 `visited` 集合。  
每到一个新节点，先把它记下来，后面再遇到就不往下走了。

这不是为了剪枝优化，而是为了避免死循环。

### DFS 写法

```java
private double dfs(String curr, String target, double value,
                   Set<String> visited,
                   Map<String, Map<String, Double>> graph) {
    if (curr.equals(target)) {
        return value;
    }

    visited.add(curr);

    for (Map.Entry<String, Double> entry : graph.get(curr).entrySet()) {
        String next = entry.getKey();
        double weight = entry.getValue();

        if (!visited.contains(next)) {
            double result = dfs(next, target, value * weight, visited, graph);
            if (result != -1.0) {
                return result;
            }
        }
    }

    return -1.0;
}
```

这段代码的逻辑很直接：

- 如果当前节点已经是目标节点，返回当前累积乘积
- 否则继续遍历邻居
- 每走一条边，就把乘积乘上对应的权重
- 只要某条路找到了答案，就直接返回
- 所有路都走不通，才返回 `-1.0`

### 主流程怎么配合 DFS

主流程主要负责三件事：

1. 构图
2. 处理一些明显的边界情况
3. 真正需要搜索时，再调用 DFS

这样分层之后，代码会比较清楚：

- 构图负责准备数据
- 主循环负责回答每个查询
- DFS 只负责做路径搜索

这类题只要把职责拆开，写起来通常都不会乱。

### 再看复杂度

设等式数量为 `E`，图中变量数量为 `V`，查询数量为 `Q`。

构图的时间复杂度是 $O(E)$。  
每个查询最坏情况下都可能把整张图搜一遍，所以单次查询是 $O(V + E)$。  
总时间复杂度就是：

$$
O(E + Q \cdot (V + E))
$$

空间复杂度是 $O(V + E)$，主要来自图的存储，以及 DFS 过程中使用的 `visited` 集合和递归栈。

这题数据规模本来就不大，所以这种做法完全够用。

### BFS 也能做

如果不想写递归，也可以用 BFS。

思路其实完全一样：

- 先建图
- 查询时从起点出发做搜索
- 队列里同时保存“当前节点”和“从起点到这里的累积乘积”
- 找到终点就返回结果

只是 DFS 用的是递归，BFS 用的是队列。  
在这题里，两者本质没有区别，都是在图上找一条可达路径。

### 这题真正值得记住的点

这题最值得记住的，不是“遇到除法就用图”，而是这种抽象方式：

**当题目给的是一组相对关系，并且查询的是两个点之间能否推出结果时，往往可以把它转成图上的路径问题。**

这里的“路径”不一定非得是距离最短，也不一定非得求条数。  
有时候像这题一样，路径的意义是“把沿途关系连乘起来”。

以后再遇到“已知 A 和 B 的关系，问 A 和 C 的关系”这类题，可以优先往图建模这个方向想。

### 完整代码

```java
import java.util.*;

public class Main {

    public static double[] calcEquation(List<List<String>> equations, double[] values, List<List<String>> queries) {
        Map<String, Map<String, Double>> graph = new HashMap<>();

        for (int i = 0; i < equations.size(); i++) {
            String u = equations.get(i).get(0);
            String v = equations.get(i).get(1);
            double val = values[i];

            graph.putIfAbsent(u, new HashMap<>());
            graph.putIfAbsent(v, new HashMap<>());

            graph.get(u).put(v, val);
            graph.get(v).put(u, 1.0 / val);
        }

        double[] result = new double[queries.size()];

        for (int i = 0; i < queries.size(); i++) {
            String start = queries.get(i).get(0);
            String end = queries.get(i).get(1);

            if (!graph.containsKey(start) || !graph.containsKey(end)) {
                result[i] = -1.0;
            } else if (start.equals(end)) {
                result[i] = 1.0;
            } else {
                Set<String> visited = new HashSet<>();
                result[i] = dfs(start, end, 1.0, visited, graph);
            }
        }

        return result;
    }

    private static double dfs(String curr, String target, double accValue,
                              Set<String> visited,
                              Map<String, Map<String, Double>> graph) {
        if (curr.equals(target)) {
            return accValue;
        }

        visited.add(curr);

        for (Map.Entry<String, Double> entry : graph.get(curr).entrySet()) {
            String next = entry.getKey();
            double weight = entry.getValue();

            if (!visited.contains(next)) {
                double result = dfs(next, target, accValue * weight, visited, graph);
                if (result != -1.0) {
                    return result;
                }
            }
        }

        return -1.0;
    }

    public static void main(String[] args) {
        List<List<String>> equations1 = new ArrayList<>();
        equations1.add(Arrays.asList("a", "b"));
        equations1.add(Arrays.asList("b", "c"));
        double[] values1 = {2.0, 3.0};

        List<List<String>> queries1 = new ArrayList<>();
        queries1.add(Arrays.asList("a", "c"));
        queries1.add(Arrays.asList("b", "a"));
        queries1.add(Arrays.asList("a", "e"));
        queries1.add(Arrays.asList("a", "a"));
        queries1.add(Arrays.asList("x", "x"));

        System.out.println("示例 1 输入:");
        System.out.println("equations = " + equations1);
        System.out.println("values = " + Arrays.toString(values1));
        System.out.println("queries = " + queries1);

        double[] res1 = calcEquation(equations1, values1, queries1);
        System.out.println("示例 1 输出: " + Arrays.toString(res1));
        System.out.println("-----------------------------");

        List<List<String>> equations2 = new ArrayList<>();
        equations2.add(Arrays.asList("a", "b"));
        equations2.add(Arrays.asList("b", "c"));
        equations2.add(Arrays.asList("bc", "cd"));
        double[] values2 = {1.5, 2.5, 5.0};

        List<List<String>> queries2 = new ArrayList<>();
        queries2.add(Arrays.asList("a", "c"));
        queries2.add(Arrays.asList("c", "b"));
        queries2.add(Arrays.asList("bc", "cd"));
        queries2.add(Arrays.asList("cd", "bc"));

        System.out.println("示例 2 输入:");
        System.out.println("equations = " + equations2);
        System.out.println("values = " + Arrays.toString(values2));
        System.out.println("queries = " + queries2);

        double[] res2 = calcEquation(equations2, values2, queries2);
        System.out.println("示例 2 输出: " + Arrays.toString(res2));
    }
}
```
