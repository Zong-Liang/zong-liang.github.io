## 题目

**描述：**

现在你总共有 `numCourses` 门课需要选，记为 `0` 到 `numCourses - 1`。给你一个数组 `prerequisites`，其中 `prerequisites[i] = [ai, bi]`，表示在选修课程 `ai` 前 **必须** 先选修 `bi`。

- 例如，想要学习课程 `0`，你需要先完成课程 `1`，我们用一个匹配来表示：`[0,1]`。

返回你为了学完所有课程所安排的学习顺序。可能会有多个正确的顺序，你只要返回 **任意一种** 就可以了。如果不可能完成所有课程，返回一个 **空数组**。

**示例 1:**

- **输入:** `numCourses = 2`, `prerequisites = [[1,0]]`
- **输出:** `[0,1]`
- **解释:** 总共有 2 门课程。要学习课程 1，你需要先完成课程 0。因此，正确的课程顺序为 `[0,1]`。

**示例 2:**

- **输入:** `numCourses = 4`, `prerequisites = [[1,0],[2,0],[3,1],[3,2]]`
- **输出:** `[0,2,1,3]`
- **解释:** 总共有 4 门课程。要学习课程 3，你应该先完成课程 1 和课程 2。并且课程 1 和课程 2 都应该排在课程 0 之后。因此，一个正确的课程顺序是 `[0,1,2,3]`。另一个正确的排序是 `[0,2,1,3]`。

**示例 3:**

- **输入:** `numCourses = 1`, `prerequisites = []`
- **输出:** `[0]`

**提示:**

- `1 <= numCourses <= 2000`
- `0 <= prerequisites.length <= numCourses * (numCourses - 1)`
- `prerequisites[i].length == 2`
- `0 <= ai, bi < numCourses`
- `ai != bi`
- 所有 `[ai, bi]` **互不相同**

## 题解

### 这题和课程表 I 的区别，只多了一步

上一题问的是：能不能学完所有课程。  
这一题问的是：如果能学完，那具体顺序是什么。

所以本质并没有变，底层还是同一个问题：

**先修关系能不能构成一张没有环的有向图。**

如果有环，那就说明有些课程互相依赖，根本排不出合法顺序，直接返回空数组。  
如果没有环，那在判断“可以完成”的同时，顺手把课程被处理的先后顺序记下来，就是答案。

也就是说，这题比 207 只多了一件事：

**把拓扑排序的结果保存下来。**

### 为什么这题还是图论题

每门课都看成一个节点。  
如果 `prerequisites[i] = [a, b]`，意思是学 `a` 之前必须先学 `b`，那就画一条边：

```text
b -> a
```

含义还是一样：

- `b` 在前
- `a` 在后

题目要你返回一种学习顺序，本质上就是要你给出这张图的一种**拓扑序**。

只要这张图没有环，就至少存在一种合法顺序；  
一旦有环，拓扑序就不存在，只能返回空数组。

### 怎么理解“合法顺序”

所谓合法顺序，就是：

**每门课都出现在它所有先修课的后面。**

比如示例 2：

```text
numCourses = 4
prerequisites = [[1,0],[2,0],[3,1],[3,2]]
```

对应的边就是：

```text
0 -> 1
0 -> 2
1 -> 3
2 -> 3
```

那课程 `0` 一定要在 `1` 和 `2` 前面，  
而 `1` 和 `2` 又都要在 `3` 前面。

所以：

- `[0,1,2,3]` 是对的
- `[0,2,1,3]` 也是对的
- 但 `[1,0,2,3]` 就不行，因为 `1` 跑到 `0` 前面去了

这也是题目为什么说“返回任意一种正确顺序都可以”。  
因为拓扑序通常不止一种。

### 还是从入度为 0 的课程开始

这题最自然的做法，还是 Kahn 算法，也就是 BFS 版拓扑排序。

思路和上一题完全一致：

1. 先统计每门课的入度
2. 把所有入度为 0 的课放进队列
3. 这些课说明当前就能学
4. 学完之后，去更新它们后续课程的入度
5. 某门后续课程入度一旦减到 0，就说明它的先修课都已经满足，可以入队

不同点只在于：

**这次每从队列里取出一门课，都要把它按顺序放进结果数组。**

最后如果结果数组长度刚好等于 `numCourses`，说明所有课程都排出来了；  
如果长度不够，说明中间卡住了，图里有环，返回空数组。

### 为什么这个顺序一定合法

因为只有入度为 0 的课程才会进队列。  
而入度为 0 的含义就是：它当前已经没有任何未完成的前置课程。

所以每次从队列里拿出来的课程，都是“现在确实可以学”的课程。  
把它放进结果数组，肯定不会违反依赖关系。

换句话说，这个算法不是先拍脑袋给顺序，再验证对不对；  
而是每一步都只挑当前合法的课程加入答案，所以天然就是正确的。

### 用示例 2 走一遍

还是看这个例子：

```text
numCourses = 4
prerequisites = [[1,0],[2,0],[3,1],[3,2]]
```

先统计入度：

- `0` 的入度是 0
- `1` 的入度是 1
- `2` 的入度是 1
- `3` 的入度是 2

所以一开始队列里只有 `0`。

先取出 `0`，结果数组变成：

```text
[0]
```

然后处理 `0` 的后续课程 `1` 和 `2`：

- `1` 的入度从 1 变成 0
- `2` 的入度从 1 变成 0

这时队列里就会有 `1` 和 `2`。  
它们谁先出队都可以，所以最终结果可能不同。

假设先出 `1`，再出 `2`，结果就是：

```text
[0,1,2]
```

然后处理 `1` 和 `2` 对 `3` 的影响，`3` 的入度会从 2 变成 0。  
最后再把 `3` 放进去，得到：

```text
[0,1,2,3]
```

如果队列里当时先取的是 `2`，也可能得到 `[0,2,1,3]`。  
两种都对。

### BFS 写法

```java
public int[] findOrder(int numCourses, int[][] prerequisites) {
    List<List<Integer>> graph = new ArrayList<>();
    for (int i = 0; i < numCourses; i++) {
        graph.add(new ArrayList<>());
    }

    int[] indegree = new int[numCourses];

    for (int[] p : prerequisites) {
        int course = p[0];
        int pre = p[1];
        graph.get(pre).add(course);
        indegree[course]++;
    }

    Queue<Integer> queue = new LinkedList<>();
    for (int i = 0; i < numCourses; i++) {
        if (indegree[i] == 0) {
            queue.offer(i);
        }
    }

    int[] result = new int[numCourses];
    int idx = 0;

    while (!queue.isEmpty()) {
        int cur = queue.poll();
        result[idx++] = cur;

        for (int next : graph.get(cur)) {
            indegree[next]--;
            if (indegree[next] == 0) {
                queue.offer(next);
            }
        }
    }

    return idx == numCourses ? result : new int[0];
}
```

### 这段代码在做什么

前半部分在建图：

- `graph.get(pre)` 里存的是“学完 `pre` 之后，哪些课会被解锁”
- `indegree[course]` 记录的是“`course` 还有几门先修课没完成”

接着把所有入度为 0 的课程放进队列。  
这些课程就是最早可以开始学的课。

后面的 while 循环，每次做三件事：

1. 取出一门当前能学的课
2. 把它写进答案数组
3. 更新它对后续课程的影响

整个过程其实就是在模拟“课程一门一门被解锁”的过程。

最后检查 `idx` 是否等于 `numCourses`：

- 相等，说明所有课都成功排出来了
- 不相等，说明还有课永远进不了队列，本质上就是有环

### 再看复杂度

时间复杂度是 $O(N + M)$。

其中：

- `N` 是课程数
- `M` 是依赖关系数

建图要遍历所有边，拓扑排序过程中每个节点和每条边也都只会处理一次，所以整体是线性的。

空间复杂度也是 $O(N + M)$。

主要来自：

- 邻接表
- 入度数组
- 队列
- 结果数组

这和课程表 I 是同一套复杂度。

### DFS 也能做，而且也能输出顺序

除了 BFS 之外，这题也可以用 DFS。

DFS 的思路是：

- 遍历某个节点时，先去递归处理它的所有后续节点
- 等这些节点都处理完，再把当前节点加入结果

这其实相当于后序遍历。  
因为一个节点只有在它后面的依赖关系都检查完成后，才会被放进结果里。

为了检测环，还是要用三种状态：

- `0`：没访问过
- `1`：正在访问
- `2`：已经访问完成

如果 DFS 过程中又遇到状态为 `1` 的节点，就说明走回了当前递归路径，图里有环，直接失败。

由于 DFS 是“处理完子节点再放当前节点”，所以结果通常要倒着填，或者最后再反转一次。

### DFS 写法

```java
int[] visited;
List<List<Integer>> graph;
int[] result;
int index;
boolean valid = true;

public int[] findOrder(int numCourses, int[][] prerequisites) {
    graph = new ArrayList<>();
    for (int i = 0; i < numCourses; i++) {
        graph.add(new ArrayList<>());
    }

    for (int[] p : prerequisites) {
        graph.get(p[1]).add(p[0]);
    }

    visited = new int[numCourses];
    result = new int[numCourses];
    index = numCourses - 1;

    for (int i = 0; i < numCourses && valid; i++) {
        if (visited[i] == 0) {
            dfs(i);
        }
    }

    return valid ? result : new int[0];
}

private void dfs(int u) {
    visited[u] = 1;

    for (int v : graph.get(u)) {
        if (visited[v] == 1) {
            valid = false;
            return;
        }
        if (visited[v] == 0) {
            dfs(v);
            if (!valid) {
                return;
            }
        }
    }

    visited[u] = 2;
    result[index--] = u;
}
```

### 这题真正要记住的地方

这题和课程表 I 放在一起看，其实特别适合建立一个整体认识：

- 课程表 I：问你能不能完成，本质是**判断有向图是否有环**
- 课程表 II：问你怎么完成，本质是**输出一组拓扑序**

所以两题不是两种题，而是一题的两个版本。  
前一题偏判断，后一题偏构造。

以后再遇到这种“任务有先后依赖”的题，可以先问自己两件事：

1. 这是不是一张有向图
2. 题目要的是“能不能”，还是“具体顺序”

如果只是判断可不可行，检测环就够了；  
如果还要你给出顺序，那基本就是拓扑排序。

import java.util.ArrayList;
import java.util.Arrays;
import java.util.LinkedList;
import java.util.List;
import java.util.Queue;

public class Main {

    public static int[] findOrder(int numCourses, int[][] prerequisites) {
        List<List<Integer>> graph = new ArrayList<>();
        for (int i = 0; i < numCourses; i++) {
            graph.add(new ArrayList<>());
        }

        int[] indegree = new int[numCourses];

        for (int[] p : prerequisites) {
            int course = p[0];
            int pre = p[1];
            graph.get(pre).add(course);
            indegree[course]++;
        }

        Queue<Integer> queue = new LinkedList<>();
        for (int i = 0; i < numCourses; i++) {
            if (indegree[i] == 0) {
                queue.offer(i);
            }
        }

        int[] result = new int[numCourses];
        int idx = 0;

        while (!queue.isEmpty()) {
            int cur = queue.poll();
            result[idx++] = cur;

            for (int next : graph.get(cur)) {
                indegree[next]--;
                if (indegree[next] == 0) {
                    queue.offer(next);
                }
            }
        }

        return idx == numCourses ? result : new int[0];
    }

    public static void main(String[] args) {
        int numCourses1 = 2;
        int[][] prerequisites1 = {{1, 0}};
        System.out.println("示例 1 输入: numCourses = 2, prerequisites = [[1,0]]");
        int[] res1 = findOrder(numCourses1, prerequisites1);
        System.out.println("示例 1 输出: " + Arrays.toString(res1));
        System.out.println("-----------------------------");

        int numCourses2 = 4;
        int[][] prerequisites2 = {{1, 0}, {2, 0}, {3, 1}, {3, 2}};
        System.out.println("示例 2 输入: numCourses = 4, prerequisites = [[1,0],[2,0],[3,1],[3,2]]");
        int[] res2 = findOrder(numCourses2, prerequisites2);
        System.out.println("示例 2 输出: " + Arrays.toString(res2));
        System.out.println("-----------------------------");

        int numCourses3 = 1;
        int[][] prerequisites3 = {};
        System.out.println("示例 3 输入: numCourses = 1, prerequisites = []");
        int[] res3 = findOrder(numCourses3, prerequisites3);
        System.out.println("示例 3 输出: " + Arrays.toString(res3));
        System.out.println("-----------------------------");

        int numCourses4 = 2;
        int[][] prerequisites4 = {{1, 0}, {0, 1}};
        System.out.println("示例 4 输入: numCourses = 2, prerequisites = [[1,0],[0,1]]");
        int[] res4 = findOrder(numCourses4, prerequisites4);
        System.out.println("示例 4 输出: " + Arrays.toString(res4));
    }
}
