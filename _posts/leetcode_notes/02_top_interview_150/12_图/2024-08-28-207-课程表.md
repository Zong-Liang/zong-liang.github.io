## 题目

**描述：**

你这个学期必须选修 `numCourses` 门课程，记为 `0` 到 `numCourses - 1`。

在选修某些课程之前需要一些先修课程。 先修课程按数组 `prerequisites` 给出，其中 `prerequisites[i] = [ai, bi]`，表示如果要学习课程 `ai` 则 **必须** 先学习课程 `bi`。

- 例如，先修课程对 `[0, 1]` 表示：想要学习课程 `0`，你需要先完成课程 `1`。

请你判断是否可能完成所有课程的学习？如果可以，返回 `true`；否则，返回 `false`。

**示例 1:**

- **输入:** `numCourses = 2`, `prerequisites = [[1,0]]`
- **输出:** `true`
- **解释:** 总共有 2 门课程。学习课程 1 之前，你需要完成课程 0。这是可能的。

**示例 2:**

- **输入:** `numCourses = 2`, `prerequisites = [[1,0],[0,1]]`
- **输出:** `false`
- **解释:** 总共有 2 门课程。学习课程 1 之前，你需要先完成课程 0；并且学习课程 0 之前，你还应先完成课程 1。这是不可能的。

**提示:**

- `1 <= numCourses <= 2000`
- `0 <= prerequisites.length <= 5000`
- `prerequisites[i].length == 2`
- `0 <= ai, bi < numCourses`
- `prerequisites[i]` 中的所有课程对 **互不相同**

## 题解

### 这题本质上不是排课，而是判断图里有没有环

题目表面上在说“课程安排”，但核心问题其实很直接：

**这些先修关系里，会不会出现互相依赖。**

比如：

```text
[1, 0]
[0, 1]
```

意思就是：

- 学 1 之前要先学 0
- 学 0 之前又要先学 1

这样就形成了一个闭环，不管从哪门课开始都走不下去，所以答案一定是 `false`。

所以这题真正要判断的，其实不是“怎么安排顺序”，而是：

**这张有向图里有没有环。**

只要有环，就不可能修完所有课；没有环，就一定可以。

### 为什么这题可以转成图

每门课都可以看成图里的一个节点。  
如果题目给出一组先修关系 `[a, b]`，表示想学课程 `a`，必须先学课程 `b`，那我们就可以画一条有向边：

```text
b -> a
```

意思很明确：  
只有先完成 `b`，后面才有资格学 `a`。

这样整道题就变成了：

- 节点：课程
- 有向边：先修关系
- 问题：这张图能不能按某种顺序把所有节点都处理完

而这正是拓扑排序最经典的应用场景。

### 怎么理解“可以完成所有课程”

如果一门课当前没有任何前置要求，那它就可以直接学。  
换成图里的语言，就是：

**这门课的入度为 0。**

所以我们可以这样想：

1. 先把所有入度为 0 的课程找出来
2. 这些课一定可以先学
3. 学完之后，它们对后续课程造成的依赖就消失了
4. 于是有些原本入度不为 0 的课，可能会变成入度为 0
5. 再继续学这些新解锁的课

如果最后所有课程都能这样被依次“解锁”，说明没有环，答案就是 `true`。  
如果中途卡住了，还有课程始终无法变成入度为 0，那就说明这些课程之间形成了环，答案就是 `false`。

### 这其实就是拓扑排序

上面这个过程，其实就是拓扑排序里的 Kahn 算法。

做法可以分成三步：

第一步，建图并统计每门课的入度。  
对于 `[a, b]`，表示有一条 `b -> a` 的边，同时 `a` 的入度加一。

第二步，把所有入度为 0 的课程放进队列。  
这些课程就是当前立刻可以学的课。

第三步，不断从队列里取课：

- 取出一门课，说明它已经学完
- 去看哪些课程依赖它
- 这些后续课程的入度减一
- 如果某门后续课程入度变成 0，就把它加入队列

最后统计一下，实际“学完”的课程数量是否等于总课程数即可。

### 用示例 2 看一下为什么会失败

示例 2 是：

```text
numCourses = 2
prerequisites = [[1,0],[0,1]]
```

建图后是：

```text
0 -> 1
1 -> 0
```

此时：

- 课程 0 的入度是 1
- 课程 1 的入度也是 1

也就是说，一开始根本没有任何课程的入度为 0。  
那队列就是空的，算法一步都走不出去。

这就说明：  
没有任何一门课能作为起点先学，这两个节点互相卡住了，所以一定存在环，答案自然是 `false`。

### 用示例 1 看一下为什么能成功

示例 1 是：

```text
numCourses = 2
prerequisites = [[1,0]]
```

图就是：

```text
0 -> 1
```

初始时：

- 课程 0 的入度是 0
- 课程 1 的入度是 1

所以先把课程 0 放进队列。  
学完课程 0 后，课程 1 的入度减到 0，于是课程 1 也能进队列。

最后两门课都能被处理掉，说明没有环，答案就是 `true`。

### BFS 写法为什么这么自然

这题用 BFS 做，其实一点都不勉强。  
因为整个过程本来就是“先处理当前所有可以学的课，再不断解锁下一批课”。

队列里装的不是普通的遍历顺序，而是：

**当前所有已经满足先修条件的课程。**

每次出队一门课，都等价于“这门课现在可以学，而且已经学完了”。  
然后它会给后续课程腾出条件，这些后续课程一旦没有剩余前置要求，就进入队列。

所以 BFS 在这里不是为了“按层遍历”，而是为了模拟“课程逐步解锁”的过程。

### BFS 写法

```java
public boolean canFinish(int numCourses, int[][] prerequisites) {
    int[] indegrees = new int[numCourses];
    List<List<Integer>> graph = new ArrayList<>();
    for (int i = 0; i < numCourses; i++) {
        graph.add(new ArrayList<>());
    }

    for (int[] edge : prerequisites) {
        int course = edge[0];
        int pre = edge[1];
        graph.get(pre).add(course);
        indegrees[course]++;
    }

    Queue<Integer> queue = new LinkedList<>();
    for (int i = 0; i < numCourses; i++) {
        if (indegrees[i] == 0) {
            queue.offer(i);
        }
    }

    int count = 0;
    while (!queue.isEmpty()) {
        int cur = queue.poll();
        count++;

        for (int next : graph.get(cur)) {
            indegrees[next]--;
            if (indegrees[next] == 0) {
                queue.offer(next);
            }
        }
    }

    return count == numCourses;
}
```

### 这段代码在做什么

先看前半部分。

`graph` 用来存储邻接表，也就是“某门课学完后，会影响哪些后续课程”。  
`indegrees` 用来记录每门课还有多少门前置课没完成。

接着把所有入度为 0 的课程放进队列。  
这些课程就是当前可以直接学的起点。

后面的 while 循环则是在不断做一件事：

- 学掉一门当前能学的课
- 更新它对后续课程的影响
- 把新解锁的课程加入队列

最后如果 `count == numCourses`，说明所有课程都被顺利处理过；否则说明有一部分课程一直卡着出不来，本质上就是图里有环。

### 再看复杂度

时间复杂度是 $O(N + M)$。

其中：

- `N` 是课程数
- `M` 是先修关系数

建图要遍历所有边，BFS 过程中每个节点、每条边也都只会处理一次，所以整体就是线性的。

空间复杂度也是 $O(N + M)$。

主要来自：

- 邻接表存图
- 入度数组
- 队列

这也是拓扑排序的标准复杂度。

### DFS 也可以做

除了 BFS 之外，这题还可以用 DFS 来做环检测。

思路是：  
从每个节点出发递归搜索，并记录每个节点当前处于什么状态。

通常会用三个状态：

- `0`：还没访问过
- `1`：正在访问，也就是当前递归路径上
- `-1`：已经访问完成，并且确定这条路没有环

DFS 时如果你从某个节点继续往下搜，又遇到了一个状态为 `1` 的节点，就说明绕回了当前递归路径上的点，也就是发现了环。

这种写法也很经典，只是从教学和直观性上看，很多人第一次接触这题时会更容易接受 BFS 的“逐步解锁课程”模型。

### DFS 写法

```java
public boolean canFinish(int numCourses, int[][] prerequisites) {
    List<List<Integer>> graph = new ArrayList<>();
    for (int i = 0; i < numCourses; i++) {
        graph.add(new ArrayList<>());
    }

    for (int[] edge : prerequisites) {
        graph.get(edge[1]).add(edge[0]);
    }

    int[] flags = new int[numCourses];
    for (int i = 0; i < numCourses; i++) {
        if (!dfs(graph, flags, i)) {
            return false;
        }
    }

    return true;
}

private boolean dfs(List<List<Integer>> graph, int[] flags, int i) {
    if (flags[i] == 1) {
        return false;
    }
    if (flags[i] == -1) {
        return true;
    }

    flags[i] = 1;
    for (int next : graph.get(i)) {
        if (!dfs(graph, flags, next)) {
            return false;
        }
    }
    flags[i] = -1;
    return true;
}
```

### 这题真正要记住的点

这题最重要的，不是背拓扑排序模板，而是先建立这个判断：

**只要先修关系里出现环，就一定无法完成所有任务。**

课程表只是外壳，底层其实是在问一张有向图是否有环。  
一旦看出这一点，后面不管用 BFS 拓扑排序，还是用 DFS 检测环，思路都会顺很多。

以后再遇到这种“任务之间有前后依赖，问能不能全部完成”的题，可以优先往这两个方向想：

- 拓扑排序
- 有向图判环

大多数同类题，本质都绕不开这两个东西。

### 完整代码

```java
import java.util.ArrayList;
import java.util.LinkedList;
import java.util.List;
import java.util.Queue;

public class Main {

    public static boolean canFinish(int numCourses, int[][] prerequisites) {
        int[] indegrees = new int[numCourses];
        List<List<Integer>> graph = new ArrayList<>();
        for (int i = 0; i < numCourses; i++) {
            graph.add(new ArrayList<>());
        }

        for (int[] edge : prerequisites) {
            int course = edge[0];
            int pre = edge[1];
            graph.get(pre).add(course);
            indegrees[course]++;
        }

        Queue<Integer> queue = new LinkedList<>();
        for (int i = 0; i < numCourses; i++) {
            if (indegrees[i] == 0) {
                queue.offer(i);
            }
        }

        int count = 0;
        while (!queue.isEmpty()) {
            int cur = queue.poll();
            count++;

            for (int next : graph.get(cur)) {
                indegrees[next]--;
                if (indegrees[next] == 0) {
                    queue.offer(next);
                }
            }
        }

        return count == numCourses;
    }

    public static void main(String[] args) {
        int numCourses1 = 2;
        int[][] prerequisites1 = {{1, 0}};

        System.out.println("示例 1 输入: numCourses = 2, prerequisites = [[1,0]]");
        boolean res1 = canFinish(numCourses1, prerequisites1);
        System.out.println("示例 1 输出: " + res1);
        System.out.println("-----------------------------");

        int numCourses2 = 2;
        int[][] prerequisites2 = {{1, 0}, {0, 1}};

        System.out.println("示例 2 输入: numCourses = 2, prerequisites = [[1,0],[0,1]]");
        boolean res2 = canFinish(numCourses2, prerequisites2);
        System.out.println("示例 2 输出: " + res2);
        System.out.println("-----------------------------");

        int numCourses3 = 3;
        int[][] prerequisites3 = {{1, 0}, {2, 1}, {0, 2}};

        System.out.println("示例 3 输入: numCourses = 3, prerequisites = [[1,0],[2,1],[0,2]]");
        boolean res3 = canFinish(numCourses3, prerequisites3);
        System.out.println("示例 3 输出: " + res3);
        System.out.println("-----------------------------");

        int numCourses4 = 3;
        int[][] prerequisites4 = {};

        System.out.println("示例 4 输入: numCourses = 3, prerequisites = []");
        boolean res4 = canFinish(numCourses4, prerequisites4);
        System.out.println("示例 4 输出: " + res4);
    }
}
```
