## 题目

**描述：**

给你二叉树的根节点 `root` 和一个表示目标的整数 `targetSum`。判断该树中是否存在 **根节点到叶子节点** 的路径，这条路径上所有节点值相加等于目标和 `targetSum`。如果存在，返回 `true`；否则，返回 `false`。

**叶子节点** 是指没有子节点的节点。

**示例 1:**

![示例1图片](https://assets.leetcode.com/uploads/2021/01/18/pathsum1.jpg)

- **输入:** `root = [5,4,8,11,null,13,4,7,2,null,null,null,1]`, `targetSum = 22`
- **输出:** `true`
- **解释:** 等于目标和的根节点到叶节点路径如上图所示。

**示例 2:**

![示例2图片](https://assets.leetcode.com/uploads/2021/01/18/pathsum2.jpg)

- **输入:** `root = [1,2,3]`, `targetSum = 5`
- **输出:** `false`

**示例 3:**

- **输入:** `root = []`, `targetSum = 0`
- **输出:** `false`
- **解释:** 由于树是空的，所以不存在根节点到叶子节点的路径。

**提示:**

- 树中节点的数目在范围 `[0, 5000]` 内
- `-1000 <= Node.val <= 1000`
- `-1000 <= targetSum <= 1000`

## 题解

### 先别急着套模板，先看题目到底在找什么

这题问的是：  
二叉树里是否存在一条 **从根节点到叶子节点** 的路径，使得这条路径上的节点值之和恰好等于 `targetSum`。

这里有两个限制一定要先看清：

- 路径必须从 **根节点开始**
- 路径必须到 **叶子节点结束**

所以它并不是在问：

- 树里有没有任意一段路径和等于目标值
- 走到中间某个节点时和刚好等于目标值可不可以

这些都不算。

这题真正要找的，是一条完整的“根到叶”的路径。

### 这题为什么适合递归

对于当前节点 `root` 来说，如果这条路径最终要凑出 `targetSum`，那么走到当前节点之后，剩下要凑出的值其实就变成了：

```text
targetSum - root.val
```

也就是说，问题可以自然地变成：

**在左子树或右子树里，是否存在一条从当前节点往下走到叶子节点的路径，它们的和等于 `targetSum - root.val`。**

这就是一个很典型的递归结构。

所以这题最自然的想法不是“把所有路径都列出来”，而是：

**每往下一层，就把目标值减掉当前节点的值。**

### 什么时候算找到答案

这题还有一个很重要的判断点：  
不是和变成 0 就立刻算成功，而是必须走到叶子节点再判断。

比如某个中间节点正好让累计和等于目标值，如果它下面还有孩子，那这条路径仍然不算完成。

所以真正的成功条件是：

- 当前节点是叶子节点
- 并且当前节点的值刚好等于剩余的目标值

这两个条件必须同时满足。

### 递归到底在做什么

递归逻辑其实很直接。

#### 1. 如果当前节点为空

那说明这条路走不通，直接返回 `false`。

#### 2. 如果当前节点是叶子节点

那就看它的值是不是刚好等于当前要凑的目标值。  
如果相等，说明找到了；否则说明这条路径不满足要求。

#### 3. 如果当前节点不是叶子节点

那就继续去左右子树里找，目标值改成：

```text
targetSum - root.val
```

只要左边或右边有一条路能找到答案，整棵树就返回 `true`。

### 用示例看一下

看示例 1：

```text
root = [5,4,8,11,null,13,4,7,2,null,null,null,1]
targetSum = 22
```

从根节点 `5` 出发，目标值先减去 `5`，剩下要找的是 `17`。

走到 `4`，再减去 `4`，剩下 `13`。

走到 `11`，再减去 `11`，剩下 `2`。

接着有两个叶子节点：

- `7`
- `2`

其中走到 `2` 时，刚好满足“叶子节点的值等于当前剩余目标值”，所以返回 `true`。

也就是说，这条路径：

```text
5 -> 4 -> 11 -> 2
```

它的和正好是 `22`。

### 递归写法

```java
public boolean hasPathSum(TreeNode root, int targetSum) {
    if (root == null) {
        return false;
    }

    if (root.left == null && root.right == null) {
        return root.val == targetSum;
    }

    return hasPathSum(root.left, targetSum - root.val)
        || hasPathSum(root.right, targetSum - root.val);
}
```

这段代码本质上就是把刚才那套思路直接翻译成程序。

每深入一层，就把当前节点的值从目标值里扣掉。  
真正到叶子节点时，再判断这条路径是否刚好凑满。

### 这题最容易错的地方

这题最常见的错误，不是递归不会写，而是条件看错。

#### 1. 把“根到叶”看成“任意路径”

这是最容易出错的地方。  
题目要的是从根出发，而且必须走到叶子节点，中间停下都不算。

#### 2. 看到和已经等于目标值就提前返回

这也不对。  
只有当前节点是叶子节点时，才能判断当前路径是否成立。

#### 3. 以为空树且目标值为 0 就应该返回 true

题目明确要求的是“存在一条根到叶路径”。  
空树连根节点都没有，自然不存在这样的路径，所以应该返回 `false`。

### 广度优先搜索也能做

除了递归，这题也可以用 BFS 来做。

思路也不复杂：

- 一个队列存当前节点
- 另一个队列存“从根走到这个节点时的路径和”

每次取出一个节点时：

- 如果它是叶子节点，就检查当前路径和是否等于 `targetSum`
- 如果不是叶子节点，就把它的孩子加入队列，同时把新的路径和也一起加入

这样一层一层扫过去，也能找到答案。

### BFS 写法

```java
public boolean hasPathSum(TreeNode root, int targetSum) {
    if (root == null) {
        return false;
    }

    Queue<TreeNode> nodeQueue = new LinkedList<>();
    Queue<Integer> valQueue = new LinkedList<>();

    nodeQueue.offer(root);
    valQueue.offer(root.val);

    while (!nodeQueue.isEmpty()) {
        TreeNode curr = nodeQueue.poll();
        int currVal = valQueue.poll();

        if (curr.left == null && curr.right == null) {
            if (currVal == targetSum) {
                return true;
            }
        }

        if (curr.left != null) {
            nodeQueue.offer(curr.left);
            valQueue.offer(currVal + curr.left.val);
        }

        if (curr.right != null) {
            nodeQueue.offer(curr.right);
            valQueue.offer(currVal + curr.right.val);
        }
    }

    return false;
}
```

### 递归和 BFS 怎么选

如果只是做这道题，递归通常更自然一些。

因为这题本身就是在不断往下走，同时维护“还差多少”这个量，和递归的结构非常贴合，代码也更短。

BFS 当然也能做，但需要额外维护一个路径和队列，写起来会稍微啰嗦一点。

所以这题一般优先掌握递归写法就够了。

### 复杂度分析

递归写法的时间复杂度是 $O(n)$，因为最坏情况下需要把整棵树都走一遍。

空间复杂度是 $O(h)$，这里的 `h` 是树的高度。  
主要来自递归调用栈。

如果树比较平衡，通常是 $O(\log n)$；  
如果树退化成链表，最坏就是 $O(n)$。

如果用 BFS：

- 时间复杂度同样是 $O(n)$
- 空间复杂度是 $O(n)$，因为队列最坏情况下可能存下一整层节点

### 这题真正值得记住的点

这题最值得记住的，其实不是某个模板，而是这种处理方式：

**当题目让你判断“从根往下的路径是否满足某个和”时，可以把问题改写成“走到当前节点后，剩下还差多少”。**

这样每往下一层，问题都会自然变小，递归就顺理成章了。

而且一定记住，这类题只要题目写了“根到叶”，那叶子节点条件就不能漏。

### 完整代码

```java
import java.util.LinkedList;
import java.util.Queue;

public class Main {

    public static class TreeNode {
        int val;
        TreeNode left;
        TreeNode right;

        TreeNode() {}

        TreeNode(int val) {
            this.val = val;
        }

        TreeNode(int val, TreeNode left, TreeNode right) {
            this.val = val;
            this.left = left;
            this.right = right;
        }
    }

    public static boolean hasPathSum(TreeNode root, int targetSum) {
        if (root == null) {
            return false;
        }

        if (root.left == null && root.right == null) {
            return root.val == targetSum;
        }

        return hasPathSum(root.left, targetSum - root.val)
            || hasPathSum(root.right, targetSum - root.val);
    }

    public static void main(String[] args) {
        TreeNode root1 = new TreeNode(5);
        root1.left = new TreeNode(4);
        root1.right = new TreeNode(8);
        root1.left.left = new TreeNode(11);
        root1.left.left.left = new TreeNode(7);
        root1.left.left.right = new TreeNode(2);
        root1.right.left = new TreeNode(13);
        root1.right.right = new TreeNode(4);
        root1.right.right.right = new TreeNode(1);

        int target1 = 22;
        System.out.println("示例 1 (target=22): " + hasPathSum(root1, target1));
        System.out.println("-----------------------------");

        TreeNode root2 = new TreeNode(1);
        root2.left = new TreeNode(2);
        root2.right = new TreeNode(3);

        int target2 = 5;
        System.out.println("示例 2 (target=5): " + hasPathSum(root2, target2));
        System.out.println("-----------------------------");

        TreeNode root3 = null;
        int target3 = 0;
        System.out.println("示例 3 (空树): " + hasPathSum(root3, target3));
    }
}
```
