## 题目

**描述：**

给你一个二叉树的根节点 `root`，树中每个节点都存放有一个 `0` 到 `9` 之间的数字。

每条从根节点到叶节点的路径都代表一个数字：

- 例如，从根节点到叶节点的路径 `1 -> 2 -> 3` 表示数字 `123`。

计算从根节点到叶节点生成的 **所有数字之和**。

**叶节点** 是指没有子节点的节点。

**示例 1:**

![示例1图片](https://assets.leetcode.com/uploads/2021/02/19/pathsum1.jpg)

- **输入:** `root = [1,2,3]`
- **输出:** 25
- **解释:**

  从根到叶子节点路径 `1->2` 代表数字 `12`

  从根到叶子节点路径 `1->3` 代表数字 `13`

  因此，数字总和 = `12 + 13 = 25`

**示例 2:**

![示例2图片](https://assets.leetcode.com/uploads/2021/02/19/pathsum2.jpg)

- **输入:** `root = [4,9,0,5,1]`
- **输出:** 1026
- **解释:**

  从根到叶子节点路径 `4->9->5` 代表数字 `495`

  从根到叶子节点路径 `4->9->1` 代表数字 `491`

  从根到叶子节点路径 `4->0` 代表数字 `40`

  因此，数字总和 = `495 + 491 + 40 = 1026`

**提示:**

- 树中节点的数目在范围 `[1, 1000]` 内
- `0 <= Node.val <= 9`
- 树的深度不超过 `10`

## 题解

### 先看这题到底在算什么

这题表面上也是“根到叶子节点路径”类问题，但和上一题不一样的地方在于：

这里不是判断路径和，而是把每条路径当成一个数字。

比如：

```text
1 -> 2 -> 3
```

它表示的不是 `1 + 2 + 3 = 6`，而是数字 `123`。

所以这题真正要做的事情是：

- 找出所有从根到叶子的路径
- 把每条路径按顺序拼成一个数字
- 最后把这些数字全部加起来

这一步一定要先想清楚。  
因为如果一开始还用“普通路径和”的思路去看，这题就容易写偏。

### 为什么会出现 `prev * 10 + node.val`

这题最核心的一步，其实就是“拼数字”。

假设我们从根走到当前节点之前，已经拼出了一个数字 `prev`。  
那再往下走到当前节点 `node` 时，新的数字应该怎么得到？

答案很直接：

```text
新数字 = 原数字 * 10 + 当前节点值
```

原因也很简单。

比如之前已经走出了 `12`，现在又走到一个值为 `3` 的节点，那新数字就应该是：

```text
12 * 10 + 3 = 123
```

这就是这题最重要的状态转移。  
只要这句话真正想明白，后面的递归基本就是顺着写出来的。

### 为什么递归特别自然

对于当前节点来说，它只需要知道两件事：

- 我前面那段路径已经拼成了多少
- 我是不是已经走到叶子节点了

如果还没到叶子节点，那就把当前拼出来的新数字继续传给左右孩子。  
如果已经到叶子节点，那这一条路径对应的数字就已经完整了，可以直接返回。

所以这题特别适合递归，因为每往下一层，本质上都只是在做同一件事：

**把当前节点值接到前面的数字后面。**

### 递归到底在做什么

可以把递归过程拆成下面几步。

#### 1. 如果当前节点为空

说明这条路不存在，不会对答案产生贡献，直接返回 `0`。

#### 2. 先更新当前路径对应的数字

假设传下来的数字是 `prevSum`，那走到当前节点之后，新的数字就是：

```text
currentSum = prevSum * 10 + node.val
```

#### 3. 如果当前节点是叶子节点

说明这条路径已经完整走完了，直接返回 `currentSum`。

#### 4. 如果当前节点不是叶子节点

那就继续递归去左右子树，把它们返回的结果加起来。

因为题目要求的是“所有路径数字之和”，所以左右子树的结果天然就是做加法。

### 用示例看一下

先看示例 1：

```text
root = [1,2,3]
```

树的结构就是：

```text
   1
  / \
 2   3
```

从根节点 `1` 出发：

- 走到左子节点 `2`，数字变成 `1 * 10 + 2 = 12`
- 走到右子节点 `3`，数字变成 `1 * 10 + 3 = 13`

这两个节点又都是叶子节点，所以最终答案就是：

```text
12 + 13 = 25
```

再看示例 2：

```text
root = [4,9,0,5,1]
```

对应的路径有三条：

- `4 -> 9 -> 5`，表示 `495`
- `4 -> 9 -> 1`，表示 `491`
- `4 -> 0`，表示 `40`

所以总和就是：

```text
495 + 491 + 40 = 1026
```

### 递归写法

```java
public int sumNumbers(TreeNode root) {
    return dfs(root, 0);
}

private int dfs(TreeNode node, int prevSum) {
    if (node == null) {
        return 0;
    }

    int currentSum = prevSum * 10 + node.val;

    if (node.left == null && node.right == null) {
        return currentSum;
    }

    return dfs(node.left, currentSum) + dfs(node.right, currentSum);
}
```

这段代码其实就是把刚才的思路一字不差地翻译成程序：

- 空节点返回 `0`
- 当前节点更新数字
- 到叶子节点就返回当前完整数字
- 否则继续递归左右子树，并把结果相加

### 这题和“路径总和”有什么区别

这两题看起来都在处理“根到叶”的路径，所以很容易混。

但本质区别很大：

- “路径总和”是在做普通加法，每走一步是 `target - node.val`
- 这题是在拼十进制数字，每走一步是 `prev * 10 + node.val`

也就是说，上一题维护的是“还差多少”，这一题维护的是“已经拼到了多少”。

这层区别最好单独记住。

### BFS 也能做

除了递归，这题也可以用 BFS。

思路和上一题有点像：

- 一个队列存节点
- 一个队列存从根到当前节点拼出来的数字

每次取出一个节点和它对应的数字：

- 如果是叶子节点，就把这个数字加到答案里
- 如果不是叶子节点，就把左右孩子和新的数字一起入队

这样也能把所有路径都处理出来。

### BFS 写法

```java
public int sumNumbers(TreeNode root) {
    if (root == null) {
        return 0;
    }

    int totalSum = 0;
    Queue<TreeNode> nodeQueue = new LinkedList<>();
    Queue<Integer> numQueue = new LinkedList<>();

    nodeQueue.offer(root);
    numQueue.offer(root.val);

    while (!nodeQueue.isEmpty()) {
        TreeNode node = nodeQueue.poll();
        int num = numQueue.poll();

        if (node.left == null && node.right == null) {
            totalSum += num;
        } else {
            if (node.left != null) {
                nodeQueue.offer(node.left);
                numQueue.offer(num * 10 + node.left.val);
            }

            if (node.right != null) {
                nodeQueue.offer(node.right);
                numQueue.offer(num * 10 + node.right.val);
            }
        }
    }

    return totalSum;
}
```

### 递归和 BFS 怎么选

如果只是做这道题，递归通常更自然。

因为这题本身就是沿着路径不断往下，把当前数字传给下一层，和递归结构非常贴合，代码也更简洁。

BFS 当然也能做，但要额外维护一个数值队列，写起来会稍微繁一点。

所以这题一般优先掌握递归写法就够了。

### 复杂度分析

递归写法的时间复杂度是 $O(n)$，因为每个节点都会被访问一次。

空间复杂度是 $O(h)$，这里的 `h` 是树的高度。  
主要开销来自递归调用栈。

如果树比较平衡，通常是 $O(\log n)$；  
如果树退化成链表，最坏就是 $O(n)$。

如果用 BFS：

- 时间复杂度同样是 $O(n)$
- 空间复杂度是 $O(n)$，最坏情况下队列会存下一整层节点

### 这题最容易错的地方

这题常见的错误主要有两个。

#### 1. 把它当成普通路径和

一旦这么想，状态转移就会写成简单加法，那就不是这题了。  
这题要维护的是“路径形成的数字”，不是节点值总和。

#### 2. 没有等到叶子节点就把当前值加进答案

这也不对。  
只有当路径真正走到叶子节点时，这条路径对应的数字才算完整，才能加入答案。

中间节点即使已经拼出了一个数，也还不能算最终结果。

### 这题真正值得记住的点

这题最值得记住的，不是递归模板，而是这种处理思路：

**当路径本身代表一个数时，每往下一层，其实是在给已有数字“补一位”。**

所以状态更新自然就是：

```text
prev * 10 + 当前节点值
```

把这句话彻底理解了，这题就真的做透了。

### 完整代码

```java
import java.util.LinkedList;
import java.util.Queue;

public class Main {

    public static class TreeNode {
        int val;
        TreeNode left;
        TreeNode right;

        TreeNode() {}

        TreeNode(int val) {
            this.val = val;
        }

        TreeNode(int val, TreeNode left, TreeNode right) {
            this.val = val;
            this.left = left;
            this.right = right;
        }
    }

    public static int sumNumbers(TreeNode root) {
        return dfs(root, 0);
    }

    private static int dfs(TreeNode node, int prevSum) {
        if (node == null) {
            return 0;
        }

        int currentSum = prevSum * 10 + node.val;

        if (node.left == null && node.right == null) {
            return currentSum;
        }

        return dfs(node.left, currentSum) + dfs(node.right, currentSum);
    }

    public static void main(String[] args) {
        TreeNode root1 = new TreeNode(1);
        root1.left = new TreeNode(2);
        root1.right = new TreeNode(3);

        System.out.println("示例 1 输入: [1,2,3]");
        int result1 = sumNumbers(root1);
        System.out.println("示例 1 输出: " + result1);
        System.out.println("-----------------------------");

        TreeNode root2 = new TreeNode(4);
        root2.left = new TreeNode(9);
        root2.right = new TreeNode(0);
        root2.left.left = new TreeNode(5);
        root2.left.right = new TreeNode(1);

        System.out.println("示例 2 输入: [4,9,0,5,1]");
        int result2 = sumNumbers(root2);
        System.out.println("示例 2 输出: " + result2);
        System.out.println("-----------------------------");

        TreeNode root3 = new TreeNode(5);
        System.out.println("示例 3 输入: [5]");
        int result3 = sumNumbers(root3);
        System.out.println("示例 3 输出: " + result3);
    }
}
```
