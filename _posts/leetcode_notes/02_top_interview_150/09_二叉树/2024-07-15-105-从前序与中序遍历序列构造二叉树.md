## 题目

**描述：**

给定两个整数数组 `preorder` 和 `inorder`，其中 `preorder` 是二叉树的 **先序遍历**，`inorder` 是同一棵树的 **中序遍历**，请构造二叉树并返回其根节点。

**示例 1:**

![示例1图片](https://assets.leetcode.com/uploads/2021/02/19/tree.jpg)

- **输入:** `preorder = [3,9,20,15,7]`, `inorder = [9,3,15,20,7]`
- **输出:** `[3,9,20,null,null,15,7]`

**示例 2:**

- **输入:** `preorder = [-1]`, `inorder = [-1]`
- **输出:** `[-1]`

**提示:**

- `1 <= preorder.length <= 3000`
- `inorder.length == preorder.length`
- `-3000 <= preorder[i], inorder[i] <= 3000`
- `preorder` 和 `inorder` 均 **无重复元素**
- `inorder` 均出现在 `preorder`
- `preorder` **保证** 为二叉树的前序遍历序列
- `inorder` **保证** 为二叉树的中序遍历序列

## 题解

### 先想清楚，这题为什么能建树

题目给了两种遍历结果：

- 前序遍历 `preorder`
- 中序遍历 `inorder`

要求根据这两个序列把原来的二叉树还原出来。

这题最核心的地方，不是“怎么写递归”，而是先想明白：

**为什么光靠前序和中序，就能把树确定下来？**

关键就在于这两种遍历各自提供的信息不一样：

- 前序遍历的顺序是：**根 -> 左子树 -> 右子树**
- 中序遍历的顺序是：**左子树 -> 根 -> 右子树**

也就是说：

- 在前序遍历里，当前子树的第一个节点，一定是这棵子树的根
- 在中序遍历里，只要找到这个根的位置，就能知道左边哪些属于左子树，右边哪些属于右子树

这两条信息一结合，树就可以一层一层还原出来了。

### 先看根节点是怎么确定的

对于当前这棵子树来说，前序遍历的第一个元素一定是根节点。

比如：

```text
preorder = [3,9,20,15,7]
inorder  = [9,3,15,20,7]
```

先看前序遍历，第一个数是 `3`，所以整棵树的根节点就是 `3`。

接下来去中序遍历里找 `3`：

```text
[9, 3, 15, 20, 7]
```

可以看到，`3` 左边只有一个 `9`，右边是 `[15,20,7]`。

这就说明：

- 左子树的节点是 `[9]`
- 右子树的节点是 `[15,20,7]`

到这里，整棵树已经被拆成了：

- 根节点：`3`
- 左子树
- 右子树

而左右子树本身又是同样的问题，所以自然就想到递归。

### 为什么递归是最自然的写法

一旦确定了根节点，剩下的工作其实就是：

- 用左子树那部分前序和中序序列去建左子树
- 用右子树那部分前序和中序序列去建右子树

也就是说，原问题可以直接拆成两个规模更小、形式完全一样的子问题。

这就是典型的递归结构。

所以这题真正的主线其实很简单：

1. 从前序遍历里拿到根节点
2. 在中序遍历里找到根节点的位置
3. 切出左子树和右子树的范围
4. 递归去建左右子树

### 难点其实在“范围怎么切”

这题代码写起来不算长，但很多人会在一个地方绕住：  
**左右子树的区间到底怎么确定？**

这里要抓住一个点：

在中序遍历中，根节点左边有多少个元素，左子树就有多少个节点。

假设当前处理中序区间是 `[inLeft, inRight]`，根节点在中序里的位置是 `inRootIndex`，那么：

```text
左子树节点个数 = inRootIndex - inLeft
```

这个值特别重要，因为它会直接帮我们确定前序遍历里左子树的范围。

### 左右子树区间怎么推出来

假设当前递归处理的是：

- 前序区间 `[preLeft, preRight]`
- 中序区间 `[inLeft, inRight]`

那么：

- 根节点值就是 `preorder[preLeft]`
- 根节点在中序中的位置是 `inRootIndex`
- 左子树大小是 `leftSubtreeSize = inRootIndex - inLeft`

接下来区间就能推出来了。

#### 左子树

前序遍历里：

- 第一个位置 `preLeft` 已经被根节点占掉了
- 后面连续 `leftSubtreeSize` 个节点属于左子树

所以左子树的前序范围是：

```text
[preLeft + 1, preLeft + leftSubtreeSize]
```

中序遍历里：

- 根节点左边整段都属于左子树

所以左子树的中序范围是：

```text
[inLeft, inRootIndex - 1]
```

#### 右子树

前序遍历里：

- 左子树占掉了 `leftSubtreeSize` 个位置
- 所以右子树要从它后面开始

右子树的前序范围是：

```text
[preLeft + leftSubtreeSize + 1, preRight]
```

中序遍历里：

- 根节点右边整段都属于右子树

右子树的中序范围是：

```text
[inRootIndex + 1, inRight]
```

这四个区间一旦想清楚，这题基本就通了。

### 为什么要用哈希表

还有一个容易忽略的点是：  
每次确定了根节点之后，我们都要去中序遍历里找它的位置。

如果每次都线性查找，那么一层层递归下来，整体复杂度可能会退化成 $O(n^2)$。

所以更好的做法是先用一个哈希表，把 `inorder` 里每个值对应的位置提前存起来。

这样后面每次找根节点位置，时间复杂度就是 $O(1)$。

题目还特别说明了**没有重复元素**，这也是为什么可以放心用“值 -> 下标”的映射。

### 结合示例完整走一遍

还是看这个例子：

```text
preorder = [3,9,20,15,7]
inorder  = [9,3,15,20,7]
```

第一步：

- 前序第一个数是 `3`，所以根是 `3`
- 在中序里找到 `3`，它左边是 `[9]`，右边是 `[15,20,7]`

于是可以确定：

- 左子树前序是 `[9]`
- 左子树中序是 `[9]`
- 右子树前序是 `[20,15,7]`
- 右子树中序是 `[15,20,7]`

再递归处理右子树：

- 前序第一个数是 `20`，所以右子树根是 `20`
- 在中序里找到 `20`，左边是 `[15]`，右边是 `[7]`

于是继续拆成：

- `20.left = 15`
- `20.right = 7`

整棵树就建出来了。

所以这题本质上就是不断重复一句话：

**前序定根，中序分左右。**

### 代码

```java
private Map<Integer, Integer> indexMap;

public TreeNode buildTree(int[] preorder, int[] inorder) {
    int n = preorder.length;
    indexMap = new HashMap<>();

    for (int i = 0; i < n; i++) {
        indexMap.put(inorder[i], i);
    }

    return myBuildTree(preorder, 0, n - 1, inorder, 0, n - 1);
}

private TreeNode myBuildTree(int[] preorder, int preLeft, int preRight,
                             int[] inorder, int inLeft, int inRight) {
    if (preLeft > preRight) {
        return null;
    }

    int rootVal = preorder[preLeft];
    TreeNode root = new TreeNode(rootVal);

    int inRootIndex = indexMap.get(rootVal);
    int leftSubtreeSize = inRootIndex - inLeft;

    root.left = myBuildTree(preorder, preLeft + 1, preLeft + leftSubtreeSize,
                            inorder, inLeft, inRootIndex - 1);

    root.right = myBuildTree(preorder, preLeft + leftSubtreeSize + 1, preRight,
                             inorder, inRootIndex + 1, inRight);

    return root;
}
```

### 复杂度分析

时间复杂度是 $O(n)$。

原因有两部分：

- 先遍历一次 `inorder` 建哈希表，花 $O(n)$
- 递归建树时，每个节点只会真正创建一次，每次通过哈希表找位置是 $O(1)$

所以总复杂度是 $O(n)$。

空间复杂度也是 $O(n)$。

- 哈希表要存 `n` 个节点的位置
- 递归调用栈在最坏情况下可能达到 $O(n)$

如果树比较平衡，递归深度通常是 $O(\log n)$；  
如果树退化成链表，最坏就是 $O(n)$。

### 这题最容易写错的地方

这题常见的错误主要有两个。

#### 1. 区间边界算错

尤其是右子树前序区间，很多人会少写或多写一个位置。

最稳的办法不是死记公式，而是先记住：

- `preLeft` 是根
- 左子树占了 `leftSubtreeSize` 个节点
- 右子树就从根和左子树之后开始

这样不容易错。

#### 2. 每次都去中序里线性找根

这样代码虽然也能写出来，但复杂度会变差。  
这题数据范围不算很小，哈希表优化最好直接带上。

### 这题真正值得记住的点

这题最值得记住的不是某个递归模板，而是这句判断依据：

**前序遍历负责告诉你“谁是根”，中序遍历负责告诉你“左右子树怎么分”。**

只要这句话真的理解了，后面再碰到“根据遍历结果还原二叉树”这类题，思路就会非常清楚。

### 完整代码

```java
import java.util.HashMap;
import java.util.Map;
import java.util.LinkedList;
import java.util.Queue;

public class Main {

    public static class TreeNode {
        int val;
        TreeNode left;
        TreeNode right;

        TreeNode() {}

        TreeNode(int val) {
            this.val = val;
        }

        TreeNode(int val, TreeNode left, TreeNode right) {
            this.val = val;
            this.left = left;
            this.right = right;
        }
    }

    private static Map<Integer, Integer> indexMap;

    public static TreeNode buildTree(int[] preorder, int[] inorder) {
        int n = preorder.length;
        indexMap = new HashMap<>();

        for (int i = 0; i < n; i++) {
            indexMap.put(inorder[i], i);
        }

        return myBuildTree(preorder, 0, n - 1, inorder, 0, n - 1);
    }

    private static TreeNode myBuildTree(int[] preorder, int preLeft, int preRight,
                                        int[] inorder, int inLeft, int inRight) {
        if (preLeft > preRight) {
            return null;
        }

        int rootVal = preorder[preLeft];
        TreeNode root = new TreeNode(rootVal);

        int inRootIndex = indexMap.get(rootVal);
        int leftSubtreeSize = inRootIndex - inLeft;

        root.left = myBuildTree(preorder, preLeft + 1, preLeft + leftSubtreeSize,
                                inorder, inLeft, inRootIndex - 1);

        root.right = myBuildTree(preorder, preLeft + leftSubtreeSize + 1, preRight,
                                 inorder, inRootIndex + 1, inRight);

        return root;
    }

    public static void printTree(TreeNode root) {
        if (root == null) {
            System.out.println("[]");
            return;
        }

        StringBuilder sb = new StringBuilder();
        sb.append("[");
        Queue<TreeNode> queue = new LinkedList<>();
        queue.offer(root);

        while (!queue.isEmpty()) {
            TreeNode node = queue.poll();
            if (node == null) {
                sb.append("null, ");
            } else {
                sb.append(node.val).append(", ");
                if (node.left != null || node.right != null) {
                    queue.offer(node.left);
                    queue.offer(node.right);
                }
            }
        }

        if (sb.lastIndexOf(", ") > 0) {
            sb.setLength(sb.length() - 2);
        }

        sb.append("]");
        System.out.println(sb.toString());
    }

    public static void main(String[] args) {
        int[] preorder1 = {3, 9, 20, 15, 7};
        int[] inorder1 = {9, 3, 15, 20, 7};

        System.out.println("示例 1 输入:");
        System.out.println("preorder = [3,9,20,15,7]");
        System.out.println("inorder  = [9,3,15,20,7]");

        TreeNode root1 = buildTree(preorder1, inorder1);

        System.out.print("示例 1 输出: ");
        printTree(root1);
        System.out.println("-----------------------------");

        int[] preorder2 = {-1};
        int[] inorder2 = {-1};

        System.out.println("示例 2 输入:");
        System.out.println("preorder = [-1]");
        System.out.println("inorder  = [-1]");

        TreeNode root2 = buildTree(preorder2, inorder2);

        System.out.print("示例 2 输出: ");
        printTree(root2);
    }
}
```
