## 题目

**描述**

给你两棵二叉树的根节点 `p` 和 `q` ，编写一个函数来检验这两棵树是否相同。

如果两个树在结构上相同，并且节点具有相同的值，则认为它们是相同的。

**示例 1:**

- **输入:** `p = [1,2,3]`, `q = [1,2,3]`
- **输出:** `true`

**示例 2:**

- **输入:** `p = [1,2]`, `q = [1,null,2]`
- **输出:** `false`

**示例 3:**

- **输入:** `p = [1,2,1]`, `q = [1,1,2]`
- **输出:** `false`

**提示:**

- 两棵树上的节点数目都在范围 `[0, 100]` 内
- `-10^4 <= Node.val <= 10^4`

## 题解

### 先想清楚，什么叫两棵树相同

这题看起来是在判断两棵二叉树是否相同，但“相同”其实不是只看节点值。

要想两棵树相同，必须同时满足两件事：

- 结构相同
- 对应位置上的节点值也相同

也就是说，不能只看一层，也不能只看节点值的集合，而是要看两棵树在每个对应位置上是否完全一致。

比如：

- `[1,2]` 和 `[1,null,2]` 虽然都有 `1` 和 `2`，但结构不同，所以不是同一棵树
- `[1,2,1]` 和 `[1,1,2]` 结构一样，但对应位置的值不同，也不是同一棵树

所以这题本质上是在做一件事：

**同步检查两棵树的对应节点。**

### 递归为什么最自然

树的问题里，只要题目要求“比较整棵树”，递归通常都会很顺手。

因为对于当前这两个节点 `p` 和 `q` 来说，要判断它们为根的两棵树是否相同，其实只需要看三件事：

- 当前两个节点本身是否一致
- 左子树是否一致
- 右子树是否一致

只要这三部分都没问题，整棵树就是相同的。

这个思路和“最大深度”那题不太一样。  
那题是“把左右子树的结果算出来，再合成当前节点的答案”；  
这题则是“当前节点先过关，然后继续检查左右子树”。

### 先把几种情况分清楚

递归题最重要的，永远是先把边界想明白。

这里一共就三种情况。

#### 1. 两个节点都为空

说明这两个位置都没有节点，而且目前没有发现差异，那这一部分显然是相同的，返回 `true`。

#### 2. 只有一个节点为空

这说明结构已经不一样了，直接返回 `false`。

#### 3. 两个节点都不为空，但值不同

结构虽然对上了，但值不一样，还是返回 `false`。

只有在这三种情况都排除之后，才有继续往下比较的必要。

### 真正的判断逻辑是什么

当 `p` 和 `q` 都不为空，而且 `p.val == q.val` 时，接下来要做的事情就很明确了：

- 比较 `p.left` 和 `q.left`
- 比较 `p.right` 和 `q.right`

只有左右两边都相同，当前整棵树才相同。

所以递归关系可以直接写成：

```text
当前两棵树是否相同
=
当前节点值相同
并且左子树相同
并且右子树相同
```

这就是整道题最核心的判断。

### 结合示例看一下

先看示例 1：

```text
p = [1,2,3]
q = [1,2,3]
```

根节点都是 `1`，继续往下看：

- 左子节点都是 `2`
- 右子节点都是 `3`

再往下走，左右孩子都为空，也都能对应上，所以最后返回 `true`。

再看示例 2：

```text
p = [1,2]
q = [1,null,2]
```

根节点虽然都是 `1`，但继续往左比较时：

- `p.left` 是 `2`
- `q.left` 是 `null`

一个为空，一个不为空，结构不一致，直接返回 `false`。

示例 3 也是类似的道理，结构虽然一样，但对应位置的值对不上，结果同样是 `false`。

### 递归写法

```java
public boolean isSameTree(TreeNode p, TreeNode q) {
    if (p == null && q == null) {
        return true;
    }

    if (p == null || q == null || p.val != q.val) {
        return false;
    }

    return isSameTree(p.left, q.left) && isSameTree(p.right, q.right);
}
```

这段代码其实就是把刚才那套判断原样翻译成程序。

先处理最基础的几种情况，剩下的部分交给左右子树递归完成。

### 复杂度分析

时间复杂度是 $O(n)$，这里的 `n` 可以理解为需要比较的节点数。  
最坏情况下，两棵树完全相同，就要把所有对应节点都检查一遍。

空间复杂度是 $O(h)$，`h` 是树的高度。  
这是递归调用栈带来的开销。

如果树比较平衡，递归深度大致是 $O(\log n)$`；  
如果树退化成链表，最坏就是 $O(n)$。

### 迭代也可以做

除了递归，这题也可以用迭代来写。

思路也不复杂：既然要同步比较两棵树，那就用两个队列，按层序遍历的顺序同时往下走。

每次从两个队列里各取出一个节点，然后检查：

- 当前节点值是否相同
- 左孩子的空和非空状态是否一致
- 右孩子的空和非空状态是否一致

只要有一处对不上，就返回 `false`。  
如果一路比较到最后都没问题，那就返回 `true`。

### 迭代写法

```java
public boolean isSameTree(TreeNode p, TreeNode q) {
    if (p == null && q == null) {
        return true;
    }
    if (p == null || q == null) {
        return false;
    }

    Queue<TreeNode> queueP = new LinkedList<>();
    Queue<TreeNode> queueQ = new LinkedList<>();
    queueP.offer(p);
    queueQ.offer(q);

    while (!queueP.isEmpty() && !queueQ.isEmpty()) {
        TreeNode nodeP = queueP.poll();
        TreeNode nodeQ = queueQ.poll();

        if (nodeP.val != nodeQ.val) {
            return false;
        }

        if (nodeP.left != null && nodeQ.left != null) {
            queueP.offer(nodeP.left);
            queueQ.offer(nodeQ.left);
        } else if (nodeP.left != null || nodeQ.left != null) {
            return false;
        }

        if (nodeP.right != null && nodeQ.right != null) {
            queueP.offer(nodeP.right);
            queueQ.offer(nodeQ.right);
        } else if (nodeP.right != null || nodeQ.right != null) {
            return false;
        }
    }

    return true;
}
```

### 这题该怎么选写法

如果只是做这道题，递归写法通常更自然一些。

因为这题本来就是在问“当前两棵树是否相同”，而这个问题本身就可以直接拆成“当前节点是否相同 + 左子树是否相同 + 右子树是否相同”。递归和题意是天然对上的。

迭代写法也没问题，只是代码会稍微绕一点，因为你不仅要比较值，还要额外处理左右孩子的空节点情况。

所以这题更推荐先把递归彻底吃透。  
只要你能真正想明白“为什么要同时判断结构和数值”，这题就不会写错。

### 这题真正值得记住的点

这题最值得记住的，不是递归模板本身，而是判断树是否相同的时候，检查的是“对应位置”而不是“出现过哪些值”。

也就是说，树的问题里一旦涉及“是否相同”“是否镜像”“是否对称”这类判断，脑子里要先有一个意识：

**比较的一定是节点之间的对应关系。**

这道题就是最典型的一题。

### 完整代码

```java
import java.util.LinkedList;
import java.util.Queue;

public class Main {

    public static class TreeNode {
        int val;
        TreeNode left;
        TreeNode right;

        TreeNode() {}

        TreeNode(int val) {
            this.val = val;
        }

        TreeNode(int val, TreeNode left, TreeNode right) {
            this.val = val;
            this.left = left;
            this.right = right;
        }
    }

    public static boolean isSameTree(TreeNode p, TreeNode q) {
        if (p == null && q == null) {
            return true;
        }

        if (p == null || q == null || p.val != q.val) {
            return false;
        }

        return isSameTree(p.left, q.left) && isSameTree(p.right, q.right);
    }

    public static void main(String[] args) {
        TreeNode p1 = new TreeNode(1, new TreeNode(2), new TreeNode(3));
        TreeNode q1 = new TreeNode(1, new TreeNode(2), new TreeNode(3));

        System.out.println("示例 1 测试:");
        System.out.println("Result: " + isSameTree(p1, q1));
        System.out.println("-----------------------------");

        TreeNode p2 = new TreeNode(1);
        p2.left = new TreeNode(2);

        TreeNode q2 = new TreeNode(1);
        q2.right = new TreeNode(2);

        System.out.println("示例 2 测试:");
        System.out.println("Result: " + isSameTree(p2, q2));
        System.out.println("-----------------------------");

        TreeNode p3 = new TreeNode(1, new TreeNode(2), new TreeNode(1));
        TreeNode q3 = new TreeNode(1, new TreeNode(1), new TreeNode(2));

        System.out.println("示例 3 测试:");
        System.out.println("Result: " + isSameTree(p3, q3));
    }
}
```
