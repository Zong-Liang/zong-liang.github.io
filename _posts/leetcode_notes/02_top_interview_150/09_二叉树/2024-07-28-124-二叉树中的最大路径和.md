## 题目

**描述：**

二叉树中的 **路径** 被定义为一条从树中任意节点出发，沿父节点-子节点连接，达到任意节点的序列。同一个节点在一条路径序列中 **至多出现一次**。该路径 **至少包含一个** 节点，且不一定经过根节点。

**路径和** 是路径中各节点值的总和。

给你一个二叉树的根节点 `root`，返回其 **最大路径和**。

**示例 1:**

![示例1图片](https://assets.leetcode.com/uploads/2020/10/13/exx1.jpg)

- **输入:** `root = [1,2,3]`
- **输出:** 6
- **解释:** 最优路径是 `2 -> 1 -> 3`，路径和为 `2 + 1 + 3 = 6`

**示例 2:**

![示例2图片](https://assets.leetcode.com/uploads/2020/10/13/exx2.jpg)

- **输入:** `root = [-10,9,20,null,null,15,7]`
- **输出:** 42
- **解释:** 最优路径是 `15 -> 20 -> 7`，路径和为 `15 + 20 + 7 = 42`

**提示:**

- 树中节点数目范围是 `[1, 3 * 10^4]`
- `-1000 <= Node.val <= 1000`

## 题解

### 先把题意看准，这题和普通路径题不一样

这题最容易做错的地方，不是递归不会写，而是题意一开始就看偏了。

这里说的“路径”有几个特点：

- 路径可以从 **任意节点** 开始
- 也可以在 **任意节点** 结束
- 不要求经过根节点
- 也不要求一定到叶子节点
- 但路径必须沿着父子关系走，而且同一个节点不能重复经过

也就是说，这题要找的并不是“根到叶”的路径，也不是“某一条向下的链”，而是树里任意一条合法路径的最大和。

这一点一定要先立住。  
因为如果脑子里还在套“根到叶”那类题的模板，后面基本都会偏。

### 为什么这题不能只看“从当前节点往下的最大和”

很多树题里，我们会定义“从当前节点出发，往下走的最大路径和”，这当然也能在这里用，但它**不能直接当作最终答案**。

原因在于，这题的最优路径可能会经过当前节点，并且同时用到它的左子树和右子树。

比如这棵树：

```text
    1
   / \
  2   3
```

如果只看“从当前节点往下走的一条路径”，那从 `1` 出发最多只能选：

- `1 + 2 = 3`
- 或 `1 + 3 = 4`

但这题真正的最大路径其实是：

```text
2 -> 1 -> 3
```

路径和是：

```text
2 + 1 + 3 = 6
```

所以这题必须把两个概念分开看。

### 这题真正要分清的，是两种值

对于任意一个节点 `node`，我们其实会同时关心两件事。

#### 第一种：经过当前节点时，能组成的最大路径和

这种路径把当前节点当成“拐点”或者“最高点”，可能长这样：

```text
左边一路上来 -> 当前节点 -> 右边一路下去
```

它可以同时用到左子树和右子树。  
这个值可以用来更新整棵树的答案。

#### 第二种：当前节点能给父节点提供的最大贡献

这个值表示：  
如果父节点想把当前节点接进它自己的路径里，那当前节点最多能贡献多少。

这里就只能选一边，不能左右都要。  
因为一旦左右都接上，路径就在当前节点处分叉了，没法继续往上延伸。

所以返回给父节点的值，只能是：

```text
当前节点值 + max(左贡献, 右贡献)
```

这两个值看起来很像，但意义完全不同。  
这题真正难的地方，就在这里。

### 为什么负数贡献要直接丢掉

再往下想一步。

如果某个子树给出的最大贡献是负数，那把它接进当前路径只会让总和变小。  
既然如此，不如干脆不要它。

所以对于左右子树贡献，我们真正用的是：

```text
max(子树贡献, 0)
```

也就是：

- 能帮忙就接上
- 如果只会拖后腿，那就当它不存在

这一点非常关键，也正是为什么这题能正确处理全负数和混合正负数的情况。

### 递归函数到底应该返回什么

这里最稳的定义是：

**递归函数返回：以当前节点为起点，向下延伸的一条路径所能取得的最大和。**

注意，是“向下延伸的一条路径”。

这句话里有两个关键词：

- **向下**
- **一条**

所以它返回给父节点的时候，不能左右都选，只能保留一边。

但与此同时，在当前节点这里，我们又可以拿左右两边都试一次，看看“经过当前节点的完整路径”能不能刷新全局最大值。

这就是这题的完整逻辑。

### 递归过程怎么理解

设递归函数叫 `maxGain(node)`。

它做的事情可以拆成下面几步。

#### 1. 空节点返回 0

空节点没有贡献，直接返回 `0`。

#### 2. 递归求左右子树的最大贡献

分别去算：

- 左子树最多能提供多少
- 右子树最多能提供多少

但如果结果是负数，就直接当成 `0`。

也就是：

```text
leftGain = max(maxGain(node.left), 0)
rightGain = max(maxGain(node.right), 0)
```

#### 3. 计算“经过当前节点”的最大路径和

当前节点作为路径中间点时，可以把左右两边都接上：

```text
node.val + leftGain + rightGain
```

这个值不能返回给父节点，但可以用来更新全局答案。

#### 4. 返回给父节点的值

返回时只能选一边继续往上延伸，所以返回：

```text
node.val + max(leftGain, rightGain)
```

这就是当前节点对父节点的最大贡献。

### 用示例 1 看一下

示例 1：

```text
root = [1,2,3]
```

树的结构是：

```text
    1
   / \
  2   3
```

看节点 `2`：

- 左右孩子都没有
- 它给父节点的贡献就是 `2`

看节点 `3`：

- 同理，贡献是 `3`

看节点 `1`：

- 左贡献是 `2`
- 右贡献是 `3`

于是：

- 经过 `1` 的最大路径和是 `1 + 2 + 3 = 6`
- 返回给父节点的贡献是 `1 + max(2, 3) = 4`

因为 `1` 已经是根了，不需要再往上返回，但全局答案被更新成了 `6`。

这就是最终结果。

### 再看示例 2 为什么答案不是经过根节点

示例 2：

```text
root = [-10,9,20,null,null,15,7]
```

树的结构大致是：

```text
      -10
      /  \
     9   20
        /  \
       15   7
```

先看节点 `20`：

- 左贡献是 `15`
- 右贡献是 `7`

所以经过 `20` 的最大路径和是：

```text
15 + 20 + 7 = 42
```

这已经非常大了。

再看根节点 `-10`：

- 左贡献是 `9`
- 右贡献是 `20 + max(15, 7) = 35`

于是经过根节点的路径和最多是：

```text
-10 + 9 + 35 = 34
```

反而比 `42` 小。

这也正好说明了一点：

**最大路径不一定经过根节点。**

这句话对这题非常重要。

### 代码

```java
int maxSum = Integer.MIN_VALUE;

public int maxPathSum(TreeNode root) {
    maxGain(root);
    return maxSum;
}

public int maxGain(TreeNode node) {
    if (node == null) {
        return 0;
    }

    int leftGain = Math.max(maxGain(node.left), 0);
    int rightGain = Math.max(maxGain(node.right), 0);

    int priceNewPath = node.val + leftGain + rightGain;
    maxSum = Math.max(maxSum, priceNewPath);

    return node.val + Math.max(leftGain, rightGain);
}
```

### 这段代码真正该怎么记

这题不建议死记代码，更建议记住这两句话：

**更新答案时，可以左右都要。**  
**返回父节点时，只能选一边。**

对应到代码里就是：

- `node.val + leftGain + rightGain` 用来更新全局最大值
- `node.val + max(leftGain, rightGain)` 用来返回给父节点

只要这个区别没混，这题基本就稳了。

### 复杂度分析

时间复杂度是 $O(n)$，因为每个节点只会被访问一次。

空间复杂度是 $O(h)$，这里的 `h` 是树的高度。  
主要开销来自递归调用栈。

如果树比较平衡，通常是 $O(\log n)$；  
如果树退化成链表，最坏就是 $O(n)$。

### 这题最容易错的地方

这题最常见的错误主要有三个。

#### 1. 把返回值当成答案

递归函数返回的是“单边最大贡献”，不是“当前子树里的最大路径和”。  
真正的答案要靠全局变量在每个节点处不断更新。

#### 2. 返回父节点时把左右两边都加上

这不对。  
因为返回给父节点的路径必须还能继续往上走，所以只能保留一边。

#### 3. 没有把负贡献截成 0

如果某个子树贡献是负数，还强行加上去，只会让结果更差。  
这一题里，“不要这条支路”本身就是一种选择。

### 这题真正值得记住的点

这题最值得记住的，不是某个公式，而是这种树上 DP 的思路：

**一个节点处，局部最优路径和“向上可提供的最优贡献”可能是两个不同的量。**

这题里：

- 局部最优路径可以左右都选
- 向上贡献只能选一边

把这个差别彻底想明白，这题就真的吃透了。

### 完整代码

```java
import java.util.LinkedList;
import java.util.Queue;

public class Main {

    public static class TreeNode {
        int val;
        TreeNode left;
        TreeNode right;

        TreeNode() {}

        TreeNode(int val) {
            this.val = val;
        }

        TreeNode(int val, TreeNode left, TreeNode right) {
            this.val = val;
            this.left = left;
            this.right = right;
        }
    }

    static int maxSum = Integer.MIN_VALUE;

    public static int maxPathSum(TreeNode root) {
        maxSum = Integer.MIN_VALUE;
        maxGain(root);
        return maxSum;
    }

    public static int maxGain(TreeNode node) {
        if (node == null) {
            return 0;
        }

        int leftGain = Math.max(maxGain(node.left), 0);
        int rightGain = Math.max(maxGain(node.right), 0);

        int priceNewPath = node.val + leftGain + rightGain;
        maxSum = Math.max(maxSum, priceNewPath);

        return node.val + Math.max(leftGain, rightGain);
    }

    public static void main(String[] args) {
        TreeNode root1 = new TreeNode(1);
        root1.left = new TreeNode(2);
        root1.right = new TreeNode(3);

        System.out.println("示例 1 输入: [1,2,3]");
        System.out.println("示例 1 输出: " + maxPathSum(root1));
        System.out.println("-----------------------------");

        TreeNode root2 = new TreeNode(-10);
        root2.left = new TreeNode(9);
        root2.right = new TreeNode(20);
        root2.right.left = new TreeNode(15);
        root2.right.right = new TreeNode(7);

        System.out.println("示例 2 输入: [-10,9,20,null,null,15,7]");
        System.out.println("示例 2 输出: " + maxPathSum(root2));
        System.out.println("-----------------------------");

        TreeNode root3 = new TreeNode(-3);
        System.out.println("示例 3 输入: [-3]");
        System.out.println("示例 3 输出: " + maxPathSum(root3));
    }
}
```
