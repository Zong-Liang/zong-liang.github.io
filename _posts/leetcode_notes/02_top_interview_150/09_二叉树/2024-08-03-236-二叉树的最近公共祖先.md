## 题目

**描述：**

给定一个二叉树, 找到该树中两个指定节点的最近公共祖先。

**百度百科中最近公共祖先的定义为：** “对于有根树 T 的两个节点 p、q，最近公共祖先表示为一个节点 x，满足 x 是 p、q 的祖先且 x 的深度尽可能大（一个节点也可以是它自己的祖先）。”

**示例 1:**

![示例1图片](https://assets.leetcode.com/uploads/2018/12/14/binarytree.png)

- **输入:** `root = [3,5,1,6,2,0,8,null,null,7,4]`, `p = 5`, `q = 1`
- **输出:** `3`
- **解释:** 节点 `5` 和节点 `1` 的最近公共祖先是节点 `3`。

**示例 2:**

![示例2图片](https://assets.leetcode.com/uploads/2018/12/14/binarytree.png)

- **输入:** `root = [3,5,1,6,2,0,8,null,null,7,4]`, `p = 5`, `q = 4`
- **输出:** `5`
- **解释:** 节点 `5` 和节点 `4` 的最近公共祖先是节点 `5`。因为根据定义最近公共祖先节点可以为节点本身。

**示例 3:**

- **输入:** `root = [1,2]`, `p = 1`, `q = 2`
- **输出:** `1`

**提示:**

- 树中节点数目在范围 `[2, 10^5]` 内。
- `-10^9 <= Node.val <= 10^9`
- 所有 `Node.val` **互不相同**。
- `p != q`
- `p` 和 `q` 均存在于给定的二叉树中。

## 题解

### 先把“最近公共祖先”这几个字想明白

这题问的是二叉树里两个节点 `p` 和 `q` 的最近公共祖先。

这里有两个关键词：

- **公共祖先**：这个节点既是 `p` 的祖先，也是 `q` 的祖先
- **最近**：在满足上面条件的前提下，它离 `p` 和 `q` 尽可能近，也就是深度尽可能大

可以把它理解成这样：

如果从 `p` 和 `q` 同时往上走，它们第一次汇合的那个节点，就是最近公共祖先。

这里还要特别注意一点：

**一个节点也可以是它自己的祖先。**

所以如果 `p` 本身就在 `q` 的祖先链上，那答案可能就是 `p` 自己，而不是它上面的某个节点。

### 这题最适合从“子树里找没找到”来想

这题看起来是在找“祖先”，但递归时其实不用真的去维护父节点路径。

更自然的想法是：

对于当前节点 `root`，我去它的左子树里找 `p` 和 `q`，再去它的右子树里找 `p` 和 `q`，然后根据查找结果判断：

- 如果左边找到了一个，右边也找到了一个，那当前节点就是分叉点
- 如果两者都在左边，那答案就在左子树里
- 如果两者都在右边，那答案就在右子树里

也就是说，这题本质上是在做一件事：

**看 `p` 和 `q` 分别落在当前节点的哪一边。**

### 为什么递归特别合适

因为对于当前节点来说，答案只有几种可能：

- 当前节点自己就是答案
- 答案在左子树里
- 答案在右子树里

这就是很典型的递归结构。

所以递归函数的含义可以这样理解：

**在以当前节点为根的这棵子树里，寻找 `p` 和 `q` 的最近公共祖先；如果只找到了其中一个，也把那个节点返回上去。**

这句“如果只找到一个，也返回上去”特别重要。  
因为后面父节点正是靠左右两边的返回结果，来判断当前是不是最近公共祖先。

### 先看递归什么时候停

这题的递归终止条件其实非常关键。

#### 1. 当前节点是 `null`

说明这条路已经走到底了，还没找到，返回 `null`。

#### 2. 当前节点就是 `p` 或者 `q`

那也直接返回当前节点。

这里很多人第一次看会疑惑：  
为什么只是找到 `p` 或 `q`，就直接返回，不继续往下找？

原因很简单：

如果当前节点就是 `p` 或 `q`，那这个信息本身就已经足够重要了，必须先往上汇报。  
父节点后面要靠这个返回值，去判断另一边有没有另外一个目标节点。

而且如果一个节点本身就是另一个节点的祖先，这个写法也正好能自然处理。

### 递归回来之后，到底看什么

假设当前节点不是 `null`，也不是 `p` 或 `q`，那我们就递归去左右子树里找：

```text
left = 在左子树中查找结果
right = 在右子树中查找结果
```

然后分情况讨论。

#### 情况 1：`left` 和 `right` 都不为空

这说明什么？

说明 `p` 和 `q` 分别在当前节点的左右两边。  
那当前节点就是它们第一次汇合的地方，也就是最近公共祖先。

所以这时候直接返回 `root`。

#### 情况 2：只有 `left` 不为空

说明两个目标节点都在左子树里，或者左子树里找到了其中一个，而另一个在这个节点更下面。  
总之答案一定在左边，所以直接返回 `left`。

#### 情况 3：只有 `right` 不为空

同理，直接返回 `right`。

#### 情况 4：左右都为空

说明当前这棵子树里根本没有 `p` 和 `q`，返回 `null`。

### 这题最核心的一句话

整道题其实可以压成一句话：

**如果一个节点的左子树和右子树分别找到了 `p`、`q`，那它就是最近公共祖先。**

这就是整个递归成立的核心。

### 用示例 1 看一下

示例 1：

```text
root = [3,5,1,6,2,0,8,null,null,7,4]
p = 5
q = 1
```

树的结构大致是：

```text
        3
      /   \
     5     1
    / \   / \
   6   2 0   8
      / \
     7   4
```

从根节点 `3` 开始看：

- 去左子树找，能找到 `5`
- 去右子树找，能找到 `1`

左右两边都找到了目标节点，所以 `3` 就是最近公共祖先。

这题里“最近”其实很好理解：  
再往下走，无论是 `5` 那边还是 `1` 那边，都已经没法同时覆盖两个目标节点了，所以最近的分叉点就是 `3`。

### 为什么示例 2 的答案是 `5`

示例 2 里：

```text
p = 5
q = 4
```

而 `4` 是 `5` 的后代。

这时候递归走到节点 `5` 时，因为当前节点正好等于 `p`，会直接返回 `5`。  
后面往上回溯时，虽然 `4` 在 `5` 的子树里，但因为“一个节点可以是自己的祖先”，所以最近公共祖先就是 `5` 本身。

这也是为什么前面那个终止条件必须写成：

```java
if (root == null || root == p || root == q) {
    return root;
}
```

少了这一句，这种情况就不好自然处理了。

### 代码

```java
public TreeNode lowestCommonAncestor(TreeNode root, TreeNode p, TreeNode q) {
    if (root == null || root == p || root == q) {
        return root;
    }

    TreeNode left = lowestCommonAncestor(root.left, p, q);
    TreeNode right = lowestCommonAncestor(root.right, p, q);

    if (left != null && right != null) {
        return root;
    }

    return left != null ? left : right;
}
```

### 这段代码真正该怎么理解

这段代码虽然短，但它返回的东西不是固定一种含义，而是分层次的。

递归函数的返回值可能表示三种情况：

- 找到了 `p`
- 找到了 `q`
- 找到了 `p` 和 `q` 的最近公共祖先

而父节点并不需要区分得那么细。  
它只需要知道：左边有没有有效结果，右边有没有有效结果。

只要左右两边都有，那当前就是答案；否则就把那一个非空结果继续往上交。

所以这题的递归写法，真正厉害的地方就在于：

**用很少的返回信息，就把整件事串起来了。**

### 复杂度分析

时间复杂度是 $O(n)$，因为最坏情况下需要把整棵树都遍历一遍。

空间复杂度是 $O(h)$，这里的 `h` 是树的高度。  
主要来自递归调用栈。

如果树比较平衡，通常是 $O(\log n)$；  
如果树退化成链表，最坏就是 $O(n)$。

### 这题最容易错的地方

这题常见的错误主要有几个。

#### 1. 没理解“节点本身也可以是祖先”

这会导致示例 2 这种情况想不通。  
一定要记住，`p` 可以是 `q` 的最近公共祖先，反过来也一样。

#### 2. 递归找到 `p` 或 `q` 后还想继续深挖

其实没必要。  
找到目标节点之后，最重要的是先把这个信息往上返回，让父节点去判断另一边有没有另一个目标节点。

#### 3. 把“最近公共祖先”理解成“离根更近的公共祖先”

这就完全反了。  
“最近”指的是离 `p`、`q` 更近，也就是深度更大，不是离根更近。

### 这题真正值得记住的点

这题最值得记住的，不是代码模板，而是这种递归判断方式：

**在树上找某个“分叉点”时，可以让左右子树分别返回查找结果，再由当前节点判断自己是不是汇合处。**

这类思路以后在很多树题里都会再次出现。

### 完整代码

```java
public class Main {

    public static class TreeNode {
        int val;
        TreeNode left;
        TreeNode right;

        TreeNode(int x) {
            val = x;
        }
    }

    public static TreeNode lowestCommonAncestor(TreeNode root, TreeNode p, TreeNode q) {
        if (root == null || root == p || root == q) {
            return root;
        }

        TreeNode left = lowestCommonAncestor(root.left, p, q);
        TreeNode right = lowestCommonAncestor(root.right, p, q);

        if (left != null && right != null) {
            return root;
        }

        return left != null ? left : right;
    }

    public static void main(String[] args) {
        TreeNode root = new TreeNode(3);
        TreeNode n5 = new TreeNode(5);
        TreeNode n1 = new TreeNode(1);
        TreeNode n6 = new TreeNode(6);
        TreeNode n2 = new TreeNode(2);
        TreeNode n0 = new TreeNode(0);
        TreeNode n8 = new TreeNode(8);
        TreeNode n7 = new TreeNode(7);
        TreeNode n4 = new TreeNode(4);

        root.left = n5;
        root.right = n1;
        n5.left = n6;
        n5.right = n2;
        n1.left = n0;
        n1.right = n8;
        n2.left = n7;
        n2.right = n4;

        System.out.println("示例 1 输入: root=[3...], p=5, q=1");
        TreeNode result1 = lowestCommonAncestor(root, n5, n1);
        System.out.println("示例 1 输出: " + (result1 != null ? result1.val : "null"));
        System.out.println("-----------------------------");

        System.out.println("示例 2 输入: root=[3...], p=5, q=4");
        TreeNode result2 = lowestCommonAncestor(root, n5, n4);
        System.out.println("示例 2 输出: " + (result2 != null ? result2.val : "null"));
        System.out.println("-----------------------------");

        TreeNode root3 = new TreeNode(1);
        TreeNode node2 = new TreeNode(2);
        root3.left = node2;

        System.out.println("示例 3 输入: root=[1,2], p=1, q=2");
        TreeNode result3 = lowestCommonAncestor(root3, root3, node2);
        System.out.println("示例 3 输出: " + (result3 != null ? result3.val : "null"));
    }
}
```
