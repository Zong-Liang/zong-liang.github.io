## 题目

**描述：**

给你一棵 **完全二叉树** 的根节点 `root`，求出该树的节点个数。

**完全二叉树** 的定义如下：在完全二叉树中，除了最底层节点可能没填满外，其余每层节点数都达到最大值，并且最下面一层的节点都集中在该层最左边的若干位置。若最底层为第 `h` 层（从第 0 层开始），则该层包含 `1`~`2^h` 个节点。

**示例 1:**

![示例1图片](https://assets.leetcode.com/uploads/2021/01/14/complete.jpg)

- **输入:** `root = [1,2,3,4,5,6]`
- **输出:** 6

**示例 2:**

- **输入:** `root = []`
- **输出:** 0

**示例 3:**

- **输入:** `root = [1]`
- **输出:** 1

**提示:**

- 树中节点的数目范围是 `[0, 5 * 10^4]`
- `0 <= Node.val <= 5 * 10^4`
- 题目数据保证输入的树是 **完全二叉树**

**进阶：** 遍历树来统计节点是一种时间复杂度为 `O(n)` 的简单解决方案。你可以设计一个更快的算法吗？

## 题解

### 先说最直接的做法

如果不管题目里“完全二叉树”这几个字，最直接的思路当然就是把它当普通二叉树处理。

对于任意节点来说：

- 总节点数 = 左子树节点数 + 右子树节点数 + 1

直接递归下去就行。

```java
public int countNodes(TreeNode root) {
    if (root == null) {
        return 0;
    }
    return 1 + countNodes(root.left) + countNodes(root.right);
}
```

这个方法没有任何问题，逻辑也最简单，但时间复杂度是 $O(n)$，因为每个节点都要访问一遍。

如果题目只是让你求节点个数，到这里其实已经做完了。  
但这题专门强调了“完全二叉树”，还问你能不能设计更快的算法，那就说明这道题真正想考的不是普通遍历。

### 这题真正想利用的，是“完全二叉树”的结构

完全二叉树和普通二叉树最大的区别在于，它的形状很规整：

- 除了最后一层，前面的每一层都是满的
- 最后一层的节点一定从左往右连续出现

这就带来一个很关键的性质：

**对于任意一棵完全二叉树，它的左右子树里，至少有一棵一定是满二叉树。**

而一旦某一边是满二叉树，我们就不用真的去遍历它了，直接套公式就能把那一整块的节点数算出来。

这就是这题能比 $O(n)$ 更快的原因。

### 为什么比较左右子树高度就够了

这里先统一一下“高度”的定义。  
我们在代码里计算的，不是整棵树的严格数学高度，而是：

**从当前节点开始，一直沿着左孩子往下走，能走多少层。**

比如：

```text
    1
   /
  2
 /
3
```

它的这个“左链高度”就是 `3`。

对于完全二叉树来说，这样算高度是很有用的，因为树的缺口只可能出现在最底层的右边，不会影响左边这条链的完整性。

所以我们只要分别算：

- 左子树的左链高度
- 右子树的左链高度

就能判断哪一边一定是满的。

### 为什么高度相等时，左子树一定是满二叉树

这是整道题最核心的一步。

假设当前节点的左右子树左链高度相等，也就是：

```text
leftHeight == rightHeight
```

这说明右子树都已经长到了和左子树一样深。  
而完全二叉树最后一层的节点是从左往右填的，如果右边都能达到这个高度，那左边一定早就填满了。

所以这时候可以确定：

**左子树一定是一棵满二叉树。**

既然左子树是满的，那它的节点数就能直接算：

```text
2^leftHeight - 1
```

再加上当前根节点这 `1` 个，一共就是：

```text
2^leftHeight
```

于是当前整棵树的节点数就变成：

```text
2^leftHeight + 右子树递归结果
```

### 为什么高度不相等时，右子树一定是满二叉树

如果：

```text
leftHeight > rightHeight
```

那说明右子树比左子树少一层。

在完全二叉树里，这种情况意味着最后一层还没填到右子树那边去，所以右子树一定是一个高度更小的满二叉树。

于是这时候能直接确定：

**右子树一定是满二叉树。**

同样地，右子树节点数加上当前根节点，就是：

```text
2^rightHeight
```

所以当前整棵树的节点数就是：

```text
2^rightHeight + 左子树递归结果
```

### 这题真正的思路可以压成一句话

每次递归时都做下面这件事：

- 先看左右子树的左链高度
- 判断哪一边一定是满二叉树
- 满的那一边直接用公式算
- 另一边继续递归

这样每次都能省掉一大块遍历。

### 用示例看一下

比如示例 1：

```text
root = [1,2,3,4,5,6]
```

树的结构大致是：

```text
        1
      /   \
     2     3
    / \   /
   4   5 6
```

先看根节点 `1`：

- 左子树一路往左是 `2 -> 4`，高度是 `2`
- 右子树一路往左是 `3 -> 6`，高度也是 `2`

高度相等，所以左子树一定是满二叉树。

左子树节点数加上根节点一共是：

```text
2^2 = 4
```

也就是：

- 根节点 `1`
- 左子树 `2,4,5`

接下来只需要递归去算右子树 `3` 那一边。

再看节点 `3`：

- 左子树高度是 `1`
- 右子树高度是 `0`

不相等，所以右子树一定是满二叉树，而且这棵“满右子树”其实是空树。

于是：

```text
2^0 = 1
```

表示节点 `3` 自己这一部分。

再加上左子树 `6` 的递归结果 `1`，得到右边总共 `2` 个节点。

最后整棵树就是：

```text
4 + 2 = 6
```

### 代码

```java
public int countNodes(TreeNode root) {
    if (root == null) {
        return 0;
    }

    int leftHeight = getHeight(root.left);
    int rightHeight = getHeight(root.right);

    if (leftHeight == rightHeight) {
        return (1 << leftHeight) + countNodes(root.right);
    } else {
        return (1 << rightHeight) + countNodes(root.left);
    }
}

private int getHeight(TreeNode node) {
    int height = 0;
    while (node != null) {
        height++;
        node = node.left;
    }
    return height;
}
```

### 这段代码该怎么理解

这段代码里最容易看不顺的，其实是这两句：

```java
return (1 << leftHeight) + countNodes(root.right);
return (1 << rightHeight) + countNodes(root.left);
```

这里的 `(1 << h)` 不只是“满子树节点数”，而是：

**满子树节点数 + 当前根节点**

因为：

```text
满二叉树节点数 = 2^h - 1
再加上根节点 1
正好就是 2^h
```

而 `1 << h` 在 Java 里就是 `2^h`。

所以这两句虽然写得短，但背后的意思其实是完整的。

### 时间复杂度为什么是 O(log² n)

每次递归都会做两件事：

- 调一次 `getHeight(root.left)`
- 调一次 `getHeight(root.right)`

而 `getHeight` 本身是一直往左走，所以复杂度是 $O(\log n)$。

同时，递归最多也只会往下走树高这么多层，也就是大约 $O(\log n)$ 层。

所以总复杂度就是：

```text
O(log n) * O(log n) = O(log^2 n)
```

这比普通遍历的 $O(n)$ 快得多。

### 空间复杂度

空间复杂度是 $O(\log n)$，主要来自递归调用栈。

因为每次递归都会把问题缩小到一边子树里，而完全二叉树的高度本身就是对数级别，所以递归深度也是 $O(\log n)$。

### 这题最容易卡住的地方

这题最容易卡住的，不是代码，而是这句判断：

**为什么左右高度相等时，左子树一定是满的？**

这里一定要从“完全二叉树最后一层从左往右填”去理解，而不是死记结论。

只要右子树都能达到这个高度，说明左边不可能还没填满。  
反过来，如果右边比左边矮，那说明最后一层还没填到右边，所以右子树一定是满的。

这才是这题真正的逻辑支点。

### 这题真正值得记住的点

这题最值得记住的，不是公式本身，而是这种利用结构性质加速的思路：

**当题目明确给出一种特殊树结构时，不要急着遍历全部节点，先想这种结构能不能帮你直接确定一大块答案。**

这题里，完全二叉树的规整性，正好让我们能在每一层都直接算掉一棵满二叉树。

这才是这题的核心价值。

### 完整代码

```java
import java.util.*;

// 二叉树节点定义
class TreeNode {
    int val;
    TreeNode left;
    TreeNode right;

    TreeNode() {}

    TreeNode(int val) {
        this.val = val;
    }

    TreeNode(int val, TreeNode left, TreeNode right) {
        this.val = val;
        this.left = left;
        this.right = right;
    }
}

public class Main {

    public static int countNodes(TreeNode root) {
        if (root == null) {
            return 0;
        }

        int leftH = getHeight(root.left);
        int rightH = getHeight(root.right);

        if (leftH == rightH) {
            return (1 << leftH) + countNodes(root.right);
        } else {
            return (1 << rightH) + countNodes(root.left);
        }
    }

    private static int getHeight(TreeNode node) {
        int height = 0;
        while (node != null) {
            height++;
            node = node.left;
        }
        return height;
    }

    public static TreeNode buildTree(Integer[] arr) {
        if (arr.length == 0) {
            return null;
        }

        TreeNode root = new TreeNode(arr[0]);
        Queue<TreeNode> q = new LinkedList<>();
        q.add(root);
        int i = 1;

        while (i < arr.length) {
            TreeNode curr = q.poll();

            if (i < arr.length && arr[i] != null) {
                curr.left = new TreeNode(arr[i]);
                q.add(curr.left);
            }
            i++;

            if (i < arr.length && arr[i] != null) {
                curr.right = new TreeNode(arr[i]);
                q.add(curr.right);
            }
            i++;
        }

        return root;
    }

    public static void main(String[] args) {
        Integer[] input1 = {1, 2, 3, 4, 5, 6};
        TreeNode root1 = buildTree(input1);

        System.out.println("示例 1 输入: root = [1,2,3,4,5,6]");
        int res1 = countNodes(root1);
        System.out.println("示例 1 输出: " + res1);
        System.out.println("-----------------------------");

        Integer[] input2 = {};
        TreeNode root2 = buildTree(input2);

        System.out.println("示例 2 输入: root = []");
        int res2 = countNodes(root2);
        System.out.println("示例 2 输出: " + res2);
        System.out.println("-----------------------------");

        Integer[] input3 = {1};
        TreeNode root3 = buildTree(input3);

        System.out.println("示例 3 输入: root = [1]");
        int res3 = countNodes(root3);
        System.out.println("示例 3 输出: " + res3);

        Integer[] input4 = {1, 2, 3, 4, 5};
        TreeNode root4 = buildTree(input4);
        System.out.println("-----------------------------");
        System.out.println("额外测试输入: root = [1,2,3,4,5]");
        System.out.println("额外测试输出: " + countNodes(root4));
    }
}
```
