## 题目

**描述：**

给你二叉树的根节点 `root`，请你将它展开为一个单链表：

- 展开后的单链表应该同样使用 `TreeNode`，其中 `right` 子指针指向链表中下一个节点，而左子指针始终为 `null`。
- 展开后的单链表应该与二叉树 **先序遍历** 顺序相同。

**示例 1:**

![示例1图片](https://assets.leetcode.com/uploads/2021/01/14/flaten.jpg)

- **输入:** `root = [1,2,5,3,4,null,6]`
- **输出:** `[1,null,2,null,3,null,4,null,5,null,6]`

**示例 2:**

- **输入:** `root = []`
- **输出:** `[]`

**示例 3:**

- **输入:** `root = [0]`
- **输出:** `[0]`

**提示:**

- 树中节点数在范围 `[0, 2000]` 内
- `-100 <= Node.val <= 100`

**进阶：** 你可以使用原地算法（`O(1)` 额外空间）展开这棵树吗？

## 题解

### 先把题目要求翻译成人话

这题要求把一棵二叉树“拉直”成一条链表。

但这里的链表并不是新建出来的，而是直接在原来的树上改指针：

- 每个节点的 `left` 都要变成 `null`
- 每个节点的 `right` 指向链表中的下一个节点
- 而这条链表的顺序，必须和二叉树的**先序遍历**一致

也就是说，最终结果要满足的顺序其实就是：

```text
根 -> 左 -> 右
```

所以这题真正的核心，不是“怎么展开”，而是：

**怎样在原树上改指针，让整棵树最后沿着 `right` 走出来的顺序，恰好就是先序遍历顺序。**

### 先序遍历顺序为什么这么重要

先序遍历的顺序是：

```text
根 -> 左子树 -> 右子树
```

所以对于当前节点 `root` 来说，展开之后它后面应该接谁，其实很明确：

1. 先接左子树展开后的结果
2. 再接右子树展开后的结果

也就是说，如果当前节点原来长这样：

```text
    root
   /    \
 left   right
```

那么展开之后，应该变成：

```text
root -> 左子树展开后的链表 -> 右子树展开后的链表
```

只要你一直记着这个顺序，后面的指针调整就不会乱。

### 最容易想到的做法：先递归把左右子树展开

一个很自然的思路是：

1. 先把左子树展开
2. 再把右子树展开
3. 然后把左子树接到当前节点右边
4. 再把原来的右子树接到这段链表后面

这个想法本身没有问题，而且很好理解。

因为左右子树一旦各自都已经被拉平成链表了，那么当前节点要做的事就只剩下“拼接”。

### 递归做法到底在拼什么

假设左右子树都已经处理完了，那么此时：

- `root.left` 指向的是左子树形成的链表头
- `root.right` 指向的是右子树形成的链表头

为了满足先序顺序，我们应该：

1. 先把原来的右子树存起来
2. 把左子树移到 `root.right`
3. 把 `root.left` 置空
4. 找到这段新右链表的末尾
5. 把原来的右子树接到末尾后面

这样顺序就变成了：

```text
root -> 左链表 -> 右链表
```

这正好就是先序遍历顺序。

### 递归写法

```java
public void flatten(TreeNode root) {
    if (root == null) {
        return;
    }

    flatten(root.left);
    flatten(root.right);

    TreeNode left = root.left;
    TreeNode right = root.right;

    root.left = null;
    root.right = left;

    TreeNode p = root;
    while (p.right != null) {
        p = p.right;
    }
    p.right = right;
}
```

### 这个递归写法有什么问题

这个写法思路很直，也确实能做出来，但它有一个明显缺点：

**每次拼接时，都要重新去找左链表的尾巴。**

这一步：

```java
while (p.right != null) {
    p = p.right;
}
```

在最坏情况下会反复走很多遍。

比如树如果本身很偏，反复找尾部这件事就会让整体效率变差。  
所以这个递归思路适合帮助理解，但不算这题最漂亮的解法。

### 进阶解法为什么能做到 O(1) 空间

题目进阶要求使用原地算法，也就是尽量不要额外开栈或容器。

这时候更好的思路是：  
**不要等左右子树都完全展开后再拼，而是在遍历过程中直接改结构。**

对于当前节点 `curr` 来说，如果它有左子树，那么根据先序遍历顺序：

```text
curr -> 左子树 -> 右子树
```

那我们其实只需要做三件事：

1. 把左子树整体挪到右边
2. 把原来的右子树接到左子树最右边那个节点后面
3. 把左指针清空

这样一改，当前节点这一段的先序顺序就已经对了。

### 为什么要找左子树里最右边的节点

这里最关键的一步，就是找到左子树里最右边的那个节点。

原因并不复杂。

因为我们准备把：

- 原来的左子树放到 `curr.right`
- 原来的右子树接到左子树展开结果的最后面

那“最后面”是谁呢？

就是左子树这条展开路径上的最后一个节点。  
在这种原地改指针的过程中，我们可以通过一直往右走，找到左子树当前最右边的节点，把原来的右子树挂到它后面。

你可以把它理解成：  
**先给左子树腾位置，再把右子树续在左子树尾巴上。**

### 整个过程怎么走

对于每个节点 `curr`：

#### 如果没有左子树

那它本来就不影响先序顺序，直接往右走，处理下一个节点。

#### 如果有左子树

那就做下面几步：

1. 记下左子树的根 `next = curr.left`
2. 在左子树里找到最右边的节点 `predecessor`
3. 让 `predecessor.right = curr.right`
4. 让 `curr.right = next`
5. 让 `curr.left = null`

做完之后，当前节点这一段就被改成了符合先序顺序的链表结构。

然后继续让 `curr = curr.right`，处理后面的节点。

### 用示例走一下

比如这棵树：

```text
        1
      /   \
     2     5
    / \     \
   3   4     6
```

先看节点 `1`。

它有左子树，所以先找左子树里最右边的节点，也就是 `4`。  
然后把原来的右子树 `5` 接到 `4` 后面：

```text
4 -> 5
```

再把左子树整体移到 `1` 的右边，变成：

```text
1
 \
  2
 / \
3   4
     \
      5
       \
        6
```

然后把 `1.left` 置空。

接着继续处理节点 `2`。  
它也有左子树，于是把 `3`、`4` 这部分再整理成：

```text
2 -> 3 -> 4
```

一路做下去，最后整棵树就会变成：

```text
1 -> 2 -> 3 -> 4 -> 5 -> 6
```

这条链正好就是原树的先序遍历顺序。

### 原地写法

```java
public void flatten(TreeNode root) {
    TreeNode curr = root;

    while (curr != null) {
        if (curr.left != null) {
            TreeNode next = curr.left;
            TreeNode predecessor = next;

            while (predecessor.right != null) {
                predecessor = predecessor.right;
            }

            predecessor.right = curr.right;
            curr.left = null;
            curr.right = next;
        }

        curr = curr.right;
    }
}
```

### 这段代码真正该怎么理解

这段代码表面上是在不停找“左子树最右节点”，但本质上它一直在维护同一件事：

**让当前节点往后的结构，逐步变成先序遍历顺序的一条右链。**

其中：

- `curr.left` 这一块本来应该排在前面
- `curr.right` 这一块本来应该排在后面
- 所以要把左边先搬到右边，再把原右子树接到后面

只要这个顺序想明白，代码就不会显得绕。

### 复杂度分析

如果用前面的递归拼接写法：

- 时间复杂度最坏可能到 $O(n^2)$，因为反复找尾节点
- 空间复杂度是 $O(h)$，来自递归栈

如果用这种原地写法：

- 时间复杂度是 $O(n)$
- 空间复杂度是 $O(1)$

因为整个过程中只用了有限几个指针变量，而且每条边被处理的次数是有限的。

### 这题最容易错的地方

这题最常见的错误，不是忘记把 `left` 置空，而是没想清楚顺序。

很多人会机械地记“找前驱、接右边、挪左边”，但一旦顺序稍微乱了，树结构就会被改坏。

真正稳妥的方式，是一直从先序遍历去看：

```text
根 -> 左 -> 右
```

当前节点后面，本来就应该先接左子树，再接右子树。  
原地做法只是把这个顺序直接翻译成指针操作而已。

### 这题真正值得记住的点

这题最值得记住的，不是哪几个变量名，而是这种结构改造思路：

**当题目要求把树原地改成某种遍历顺序时，本质上是在重新安排子树接到当前节点后的先后次序。**

这题里，目标顺序是先序，所以当前节点后面一定是：

```text
左子树展开结果 -> 右子树展开结果
```

把这一层想透，这题就真的吃透了。

### 完整代码

```java
import java.util.*;

// 二叉树节点定义
class TreeNode {
    int val;
    TreeNode left;
    TreeNode right;

    TreeNode() {}

    TreeNode(int val) {
        this.val = val;
    }

    TreeNode(int val, TreeNode left, TreeNode right) {
        this.val = val;
        this.left = left;
        this.right = right;
    }
}

public class Main {

    public static void flatten(TreeNode root) {
        TreeNode curr = root;

        while (curr != null) {
            if (curr.left != null) {
                TreeNode next = curr.left;
                TreeNode predecessor = next;

                while (predecessor.right != null) {
                    predecessor = predecessor.right;
                }

                predecessor.right = curr.right;
                curr.left = null;
                curr.right = next;
            }

            curr = curr.right;
        }
    }

    public static void printFlattenedTree(TreeNode root) {
        List<String> res = new ArrayList<>();
        TreeNode curr = root;

        while (curr != null) {
            res.add(String.valueOf(curr.val));
            if (curr.left != null) {
                System.err.println("Error: Node " + curr.val + " has left child!");
            }
            curr = curr.right;
        }

        System.out.println(res);
    }

    public static void main(String[] args) {
        TreeNode root = new TreeNode(1);
        root.left = new TreeNode(2);
        root.right = new TreeNode(5);
        root.left.left = new TreeNode(3);
        root.left.right = new TreeNode(4);
        root.right.right = new TreeNode(6);

        System.out.println("示例 1 输入: root = [1,2,5,3,4,null,6]");
        flatten(root);
        System.out.print("示例 1 输出: ");
        printFlattenedTree(root);
        System.out.println("-----------------------------");

        TreeNode root2 = null;
        System.out.println("示例 2 输入: root = []");
        flatten(root2);
        System.out.print("示例 2 输出: ");
        printFlattenedTree(root2);
        System.out.println("-----------------------------");

        TreeNode root3 = new TreeNode(0);
        System.out.println("示例 3 输入: root = [0]");
        flatten(root3);
        System.out.print("示例 3 输出: ");
        printFlattenedTree(root3);
    }
}
```
