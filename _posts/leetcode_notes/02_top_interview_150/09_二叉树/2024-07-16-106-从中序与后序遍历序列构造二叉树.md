## 题目

**描述：**

给定两个整数数组 `inorder` 和 `postorder`，其中 `inorder` 是二叉树的中序遍历，`postorder` 是同一棵树的后序遍历，请你构造并返回这颗 **二叉树**。

**示例 1:**

![示例1图片](https://assets.leetcode.com/uploads/2021/02/19/tree.jpg)

- **输入:** `inorder = [9,3,15,20,7]`, `postorder = [9,15,7,20,3]`
- **输出:** `[3,9,20,null,null,15,7]`

**示例 2:**

- **输入:** `inorder = [-1]`, `postorder = [-1]`
- **输出:** `[-1]`

**提示:**

- `1 <= inorder.length <= 3000`
- `postorder.length == inorder.length`
- `-3000 <= inorder[i], postorder[i] <= 3000`
- `inorder` 和 `postorder` 都由 **不同** 的值组成
- `postorder` 中每一个值都在 `inorder` 中
- `inorder` **保证**是树的中序遍历
- `postorder` **保证**是树的后序遍历

## 题解

### 先抓住这题的核心

这题和“从前序与中序遍历序列构造二叉树”本质上是同一类题，都是根据两种遍历结果去还原原树。

区别只在于，这次给的是：

- 中序遍历 `inorder`
- 后序遍历 `postorder`

所以这题真正要先想明白的，不是代码怎么写，而是这两种遍历各自能告诉我们什么。

其中最关键的一句是：

**后序遍历的最后一个元素，一定是当前子树的根节点。**

而中序遍历依然负责做另一件事：

**找到根节点之后，把左右子树分开。**

只要这两句话能连起来，这题就能做。

### 为什么后序遍历能确定根节点

后序遍历的顺序是：

```text
左子树 -> 右子树 -> 根
```

所以对于当前这棵子树来说，后序遍历区间里的最后一个元素，一定就是它的根节点。

比如：

```text
inorder   = [9,3,15,20,7]
postorder = [9,15,7,20,3]
```

看 `postorder` 最后一个元素，是 `3`，所以整棵树的根节点就是 `3`。

接下来再去 `inorder` 里找 `3`：

```text
[9, 3, 15, 20, 7]
```

可以看到：

- `3` 左边是 `[9]`
- `3` 右边是 `[15,20,7]`

这就说明：

- 左子树的节点是 `[9]`
- 右子树的节点是 `[15,20,7]`

到这里，根节点和左右子树就都拆出来了。

### 这题为什么还是递归

一旦当前根节点确定下来，剩下要做的事情其实和原问题完全一样：

- 用左子树对应的中序和后序区间去建左子树
- 用右子树对应的中序和后序区间去建右子树

也就是说，大问题会自然拆成两个更小、形式完全相同的问题。

这就是标准的递归结构。

所以这题主线其实很简单：

1. 从后序遍历里拿到当前根节点
2. 在中序遍历里找到根节点位置
3. 切出左子树和右子树的区间
4. 递归构造左右子树

### 难点还是在区间怎么切

这类题最容易绕住人的地方，永远不是“根是谁”，而是：

**左右子树在两个数组中的范围到底怎么确定。**

先设当前递归处理的是：

- 中序区间 `[inLeft, inRight]`
- 后序区间 `[postLeft, postRight]`

那么：

- 当前根节点值就是 `postorder[postRight]`
- 它在中序中的位置是 `inRootIndex`

接着就能算出左子树有多少个节点：

```text
leftSubtreeSize = inRootIndex - inLeft
```

这个值一出来，左右子树的区间就都能推出来了。

### 左右子树区间怎么推

#### 左子树

在中序遍历里，根节点左边那一段全是左子树，所以左子树的中序范围是：

```text
[inLeft, inRootIndex - 1]
```

在后序遍历里，顺序是“左子树 -> 右子树 -> 根”。  
既然当前区间最后一个位置已经被根占掉了，那么前面那一段里，最左边这 `leftSubtreeSize` 个节点就是左子树。

所以左子树的后序范围是：

```text
[postLeft, postLeft + leftSubtreeSize - 1]
```

#### 右子树

中序遍历里，根节点右边那一段就是右子树，所以右子树的中序范围是：

```text
[inRootIndex + 1, inRight]
```

后序遍历里，右子树紧跟在左子树后面，并且在根节点前面，所以右子树的后序范围是：

```text
[postLeft + leftSubtreeSize, postRight - 1]
```

这里的 `postRight - 1` 很关键，因为 `postRight` 已经是当前根节点了，不能再算进右子树里。

### 和上一题最大的区别在哪里

如果你刚做过上一题，会发现这两题几乎是一个模子刻出来的。

真正的区别只有一句：

- 前序遍历：**第一个元素是根**
- 后序遍历：**最后一个元素是根**

中序遍历的作用完全没变，还是负责把左右子树切开。

所以这两题最好对着记：

- 105 题：前序定根，中序分左右
- 106 题：后序定根，中序分左右

这样不容易混。

### 为什么还要用哈希表

和上一题一样，这里每次确定根节点之后，都要去中序遍历里找它的位置。

如果每次都线性查找，那么总复杂度会退化，最坏可能到 $O(n^2)$。

所以最好先用一个哈希表，把中序遍历中每个值对应的下标存起来。  
这样后面每次找根节点位置都能做到 $O(1)$。

题目也明确说了元素**互不重复**，所以这种“值 -> 下标”的映射是成立的。

### 用示例完整走一遍

还是看这个例子：

```text
inorder   = [9,3,15,20,7]
postorder = [9,15,7,20,3]
```

第一步：

- 后序最后一个元素是 `3`，所以根节点是 `3`
- 去中序里找 `3`，它左边是 `[9]`，右边是 `[15,20,7]`

于是可以确定：

- 左子树的中序是 `[9]`
- 右子树的中序是 `[15,20,7]`

再看后序：

- 去掉最后的根 `3`
- 剩下 `[9,15,7,20]`
- 其中左子树有 1 个节点，所以左子树后序是 `[9]`
- 右子树后序是 `[15,7,20]`

接着递归处理右子树：

- 后序最后一个元素是 `20`，所以右子树根是 `20`
- 去中序里找 `20`，左边是 `[15]`，右边是 `[7]`

于是继续拆出：

- `20.left = 15`
- `20.right = 7`

整棵树就建出来了。

所以这题本质上也是反复做同一件事：

**后序定根，中序分左右。**

### 代码

```java
Map<Integer, Integer> indexMap;

public TreeNode buildTree(int[] inorder, int[] postorder) {
    int n = inorder.length;
    indexMap = new HashMap<>();

    for (int i = 0; i < n; i++) {
        indexMap.put(inorder[i], i);
    }

    return myBuildTree(inorder, 0, n - 1, postorder, 0, n - 1);
}

private TreeNode myBuildTree(int[] inorder, int inLeft, int inRight,
                             int[] postorder, int postLeft, int postRight) {
    if (inLeft > inRight) {
        return null;
    }

    int rootVal = postorder[postRight];
    TreeNode root = new TreeNode(rootVal);

    int inRootIndex = indexMap.get(rootVal);
    int leftSubtreeSize = inRootIndex - inLeft;

    root.left = myBuildTree(inorder, inLeft, inRootIndex - 1,
                            postorder, postLeft, postLeft + leftSubtreeSize - 1);

    root.right = myBuildTree(inorder, inRootIndex + 1, inRight,
                             postorder, postLeft + leftSubtreeSize, postRight - 1);

    return root;
}
```

### 复杂度分析

时间复杂度是 $O(n)$。

因为：

- 建哈希表需要一次遍历，花 $O(n)$
- 递归建树时，每个节点只会处理一次
- 每次通过哈希表找根节点位置是 $O(1)$

所以总时间复杂度就是 $O(n)$。

空间复杂度也是 $O(n)$。

主要来自两部分：

- 哈希表存储节点值到下标的映射
- 递归调用栈

如果树比较平衡，递归深度通常是 $O(\log n)$；  
如果树退化成链表，最坏情况下递归栈会到 $O(n)$。

### 这题最容易写错的地方

这题最常见的错误，主要还是区间边界。

尤其是右子树的后序范围，很多人容易把根节点也一起算进去，或者起点写错一位。

最稳的办法不是硬背公式，而是先记住后序遍历的结构：

```text
左子树 -> 右子树 -> 根
```

只要脑子里这条顺序不乱，区间基本就不会切错。

另外还有一个常见问题，就是把这题和上一题混掉。  
上一题取根节点看的是前序最左边，这题取根节点看的是后序最右边，这个地方一定要分开记。

### 这题真正值得记住的点

这题最值得记住的，不是哪一条公式，而是这类建树题的统一思路：

**先找根，再分左右，最后递归。**

只不过不同遍历方式里，“根在哪”不一样：

- 前序的根在开头
- 后序的根在结尾

而中序始终负责切开左右子树。

把这一层吃透，105 和 106 这两题其实就是一回事。

### 完整代码

```java
import java.util.HashMap;
import java.util.Map;
import java.util.LinkedList;
import java.util.Queue;

public class Main {

    public static class TreeNode {
        int val;
        TreeNode left;
        TreeNode right;

        TreeNode() {}

        TreeNode(int val) {
            this.val = val;
        }

        TreeNode(int val, TreeNode left, TreeNode right) {
            this.val = val;
            this.left = left;
            this.right = right;
        }
    }

    private static Map<Integer, Integer> indexMap;

    public static TreeNode buildTree(int[] inorder, int[] postorder) {
        int n = inorder.length;
        indexMap = new HashMap<>();

        for (int i = 0; i < n; i++) {
            indexMap.put(inorder[i], i);
        }

        return myBuildTree(inorder, 0, n - 1, postorder, 0, n - 1);
    }

    private static TreeNode myBuildTree(int[] inorder, int inLeft, int inRight,
                                        int[] postorder, int postLeft, int postRight) {
        if (inLeft > inRight) {
            return null;
        }

        int rootVal = postorder[postRight];
        TreeNode root = new TreeNode(rootVal);

        int inRootIndex = indexMap.get(rootVal);
        int leftSubtreeSize = inRootIndex - inLeft;

        root.left = myBuildTree(inorder, inLeft, inRootIndex - 1,
                                postorder, postLeft, postLeft + leftSubtreeSize - 1);

        root.right = myBuildTree(inorder, inRootIndex + 1, inRight,
                                 postorder, postLeft + leftSubtreeSize, postRight - 1);

        return root;
    }

    public static void printTree(TreeNode root) {
        if (root == null) {
            System.out.println("[]");
            return;
        }

        StringBuilder sb = new StringBuilder();
        sb.append("[");
        Queue<TreeNode> queue = new LinkedList<>();
        queue.offer(root);

        while (!queue.isEmpty()) {
            TreeNode node = queue.poll();
            if (node == null) {
                sb.append("null, ");
            } else {
                sb.append(node.val).append(", ");
                if (node.left != null || node.right != null) {
                    queue.offer(node.left);
                    queue.offer(node.right);
                }
            }
        }

        if (sb.lastIndexOf(", ") > 0) {
            sb.setLength(sb.length() - 2);
        }

        sb.append("]");
        System.out.println(sb.toString());
    }

    public static void main(String[] args) {
        int[] inorder1 = {9, 3, 15, 20, 7};
        int[] postorder1 = {9, 15, 7, 20, 3};

        System.out.println("示例 1 输入:");
        System.out.println("inorder   = [9,3,15,20,7]");
        System.out.println("postorder = [9,15,7,20,3]");

        TreeNode root1 = buildTree(inorder1, postorder1);

        System.out.print("示例 1 输出: ");
        printTree(root1);
        System.out.println("-----------------------------");

        int[] inorder2 = {-1};
        int[] postorder2 = {-1};

        System.out.println("示例 2 输入:");
        System.out.println("inorder   = [-1]");
        System.out.println("postorder = [-1]");

        TreeNode root2 = buildTree(inorder2, postorder2);

        System.out.print("示例 2 输出: ");
        printTree(root2);
    }
}
```
