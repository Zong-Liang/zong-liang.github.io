## 题目

**描述：**

实现一个二叉搜索树迭代器类 `BSTIterator`，表示一个按 **中序遍历** 二叉搜索树（BST）的迭代器：

- `BSTIterator(TreeNode root)` 初始化 `BSTIterator` 类的一个对象。BST 的根节点 `root` 会作为构造函数的一部分给出。指针应初始化为一个不存在于 BST 中的数字，且该数字小于 BST 中的任何元素。
- `boolean hasNext()` 如果指针向右侧遍历存在数字，则返回 `true`；否则返回 `false`。
- `int next()` 将指针向右移动，然后返回指针处的数字。

**注意**，指针初始化为一个不存在于 BST 中的数字，所以对 `next()` 的首次调用将返回 BST 中的最小元素。

你可以假设 `next()` 调用总是有效的，也就是说，当调用 `next()` 时，BST 的中序遍历中至少存在一个下一个数字。

**示例:**
![示例图片](https://assets.leetcode.com/uploads/2018/12/25/bst-iterator.png)

- **输入**
  `["BSTIterator", "next", "next", "hasNext", "next", "hasNext", "next", "hasNext", "next", "hasNext"]`
  `[[[7, 3, 15, null, null, 9, 20]], [], [], [], [], [], [], [], [], []]`
- **输出**
  `[null, 3, 7, true, 9, true, 15, true, 20, false]`

**提示:**

- 树中节点数目在范围 `[1, 10^5]` 内
- `0 <= Node.val <= 10^6`
- 最多调用 `10^5` 次 `hasNext` 和 `next` 操作

**进阶:**

- 你可以设计一个满足下述条件的解决方案吗？`next()` 和 `hasNext()` 操作均摊时间复杂度为 `O(1)`，并使用 `O(h)` 内存，其中 `h` 是树的高度。

## 题解

### 先看这题到底在模拟什么

这题不是单纯让你做一次中序遍历，而是要你实现一个“迭代器”。

也就是说，它希望你像操作数组迭代器那样去操作一棵二叉搜索树：

- 调用 `next()`，返回下一个更小的元素
- 调用 `hasNext()`，判断后面还有没有元素

所以这题本质上是在做一件事：

**把 BST 的中序遍历过程，拆成一连串可暂停、可继续的操作。**

这个理解很重要。  
因为如果一开始只把它看成“遍历树”，就不容易意识到题目真正想考的是“如何维护遍历到一半的状态”。

### 为什么答案一定是中序遍历

题目已经明确说了，迭代器要按中序遍历顺序返回节点值。

而对二叉搜索树来说，中序遍历有个特别关键的性质：

**得到的结果是递增序列。**

所以这题说白了，就是要你从小到大一个一个取出 BST 里的元素。

这也是为什么第一次调用 `next()` 时，返回的一定是整棵树中的最小值。

### 最直观的做法：先全部拍平

最容易想到的办法，是在构造函数里先对整棵树做一次完整的中序遍历，把结果存进一个数组或列表里。

这样后面：

- `next()` 就是返回当前下标对应的值
- `hasNext()` 就是判断下标有没有越界

这个思路当然没问题，而且实现也很直接。

### 预处理写法为什么能做出来

因为中序遍历顺序已经是有序的，所以只要我们在初始化时把所有节点值按中序顺序收集起来，后面整个迭代器就退化成了“读数组”。

这个做法的优点是非常直观：

- 构造的时候一次性算完
- 后面每次 `next()` 和 `hasNext()` 都是 $O(1)$

但缺点也很明显：

**你把整棵树都存下来了。**

这会占用 $O(n)$ 的额外空间，不满足题目进阶要求的 $O(h)$。

### 预处理写法

```java
class BSTIterator {
    private List<Integer> nodes;
    private int index;

    public BSTIterator(TreeNode root) {
        nodes = new ArrayList<>();
        index = 0;
        inorder(root);
    }

    private void inorder(TreeNode root) {
        if (root == null) {
            return;
        }
        inorder(root.left);
        nodes.add(root.val);
        inorder(root.right);
    }

    public int next() {
        return nodes.get(index++);
    }

    public boolean hasNext() {
        return index < nodes.size();
    }
}
```

### 进阶要求真正想让你做什么

题目进阶要求：

- `next()` 和 `hasNext()` 均摊 $O(1)$
- 空间复杂度 $O(h)$，其中 `h` 是树的高度

这意味着你不能把整棵树都存下来，而是应该：

**只保留“为了找到下一个节点，当前必须记住的那部分状态”。**

而这部分状态，其实正好就是中序遍历递归过程中那条“还没处理完的路径”。

所以这题更优的做法，本质上是在用栈模拟中序遍历。

### 为什么栈里只需要放左链

先想一件事：

中序遍历顺序是：

```text
左 -> 根 -> 右
```

所以任何时候，下一个要访问的节点，一定是当前这部分树里**最靠左的那个还没访问过的节点**。

这就说明：

- 初始化时，我们只需要一路往左，把沿途节点全部压栈
- 这样栈顶就是当前最小的元素，也就是下一次 `next()` 应该返回的值

换句话说，栈里维护的其实不是“所有还没访问的节点”，而是：

**从当前搜索位置出发，通往下一个答案的那条左链。**

这就是为什么只压左链就够了。

### `next()` 为什么弹出后还要处理右子树

这也是这题最容易卡住的地方。

当我们弹出栈顶节点 `cur` 时，说明：

- 它左边已经没有更小的节点了
- 所以它自己就是当前应该返回的下一个值

但返回完 `cur` 之后，并不代表遍历结束了。  
接下来中序遍历该去哪里？

答案是：

- 如果 `cur` 有右子树，那么下一批要访问的节点一定在它的右子树里
- 而右子树里最先访问的，又一定是那棵右子树中最左边的节点

所以这时候要做的就是：

**从 `cur.right` 出发，再一路往左压栈。**

这样栈顶又会重新变成“当前未访问部分中的最小值”。

### 整个过程到底在维护什么

可以把整个迭代器理解成这样：

- 栈里始终保存着“接下来可能成为答案的节点路径”
- 栈顶永远是当前最小的未访问节点
- 每调用一次 `next()`，就弹出一个节点
- 如果这个节点有右子树，就把右子树的左链补进来

整个过程，本质上是在按需推进中序遍历，而不是一次性做完。

这就是这题最核心的思想。

### 用示例走一遍

题目里的树是：

```text
        7
       / \
      3   15
         /  \
        9    20
```

初始化时，从根节点一路往左压栈：

```text
7 -> 3
```

所以栈顶是 `3`。

#### 第一次调用 `next()`

弹出 `3`，返回 `3`。  
因为 `3` 没有右子树，所以不需要额外压栈。

此时栈顶变成 `7`。

#### 第二次调用 `next()`

弹出 `7`，返回 `7`。  
因为 `7` 有右子树 `15`，所以要从 `15` 开始一路往左压栈：

```text
15 -> 9
```

此时栈顶变成 `9`。

#### 第三次调用 `next()`

弹出 `9`，返回 `9`。  
`9` 没有右子树，栈顶变成 `15`。

接下来依次就是 `15`、`20`。

整个输出顺序正好是：

```text
3, 7, 9, 15, 20
```

这就是 BST 的中序遍历结果。

### 进阶写法

```java
class BSTIterator {
    private Deque<TreeNode> stack;

    public BSTIterator(TreeNode root) {
        stack = new ArrayDeque<>();
        pushLeft(root);
    }

    private void pushLeft(TreeNode node) {
        while (node != null) {
            stack.push(node);
            node = node.left;
        }
    }

    public int next() {
        TreeNode node = stack.pop();

        if (node.right != null) {
            pushLeft(node.right);
        }

        return node.val;
    }

    public boolean hasNext() {
        return !stack.isEmpty();
    }
}
```

### 这段代码该怎么记

这题不建议死记流程，更建议记住两句话：

**栈顶永远是当前最小的未访问节点。**  
**访问完一个节点后，如果它有右子树，就去处理右子树的最左链。**

只要这两句话没丢，代码很容易自己写出来。

### 为什么空间复杂度是 O(h)

因为栈里存的不是整棵树，而只是某一条从上到下的路径。

最深的时候，栈里最多也就是树高这么多个节点，所以空间复杂度是 $O(h)$。

这正好满足题目进阶要求。

如果树比较平衡，`h` 大约是 $O(\log n)`；  
如果树退化成链表，最坏就是 $O(n)$。

### 为什么 `next()` 是均摊 O(1)

乍一看，`next()` 里可能会调用 `pushLeft`，而 `pushLeft` 里有一个 `while` 循环，好像最坏会到 $O(h)$。

但均摊分析不是这么看。

在整个遍历过程中：

- 每个节点最多只会被压栈一次
- 每个节点最多只会被弹栈一次

所以总的栈操作次数是 $O(n)$。  
既然一共只会处理这么多次，而 `next()` 总共也只会被调用这么多次，那均摊下来每次就是 $O(1)$。

这也是题目进阶条件里最关键的一点。

### 这题最容易错的地方

这题常见的错误主要有两个。

#### 1. 初始化时只压根节点，不一路压左

这样第一次 `next()` 返回的就不一定是最小值了，整个顺序都会错。

#### 2. 弹出节点后直接返回，不处理右子树

这样会漏掉中序遍历里“右子树的最左节点”这一段，后续顺序就断了。

所以要一直记住：

- 初始化要把最左链准备好
- 每访问完一个节点，如果有右子树，就把右子树的最左链准备好

### 这题真正值得记住的点

这题最值得记住的，不是某个类模板，而是这种设计思路：

**当题目要求你把一次完整遍历改造成“按需取下一个”的迭代器时，本质上是在保存遍历进行到一半的状态。**

这题里，那个状态就是中序遍历尚未处理完的栈。

把这个思路吃透，后面类似的“树迭代器”“懒遍历”题也会顺很多。

### 完整代码

```java
import java.util.*;

// 二叉树节点定义
class TreeNode {
    int val;
    TreeNode left;
    TreeNode right;

    TreeNode() {}

    TreeNode(int val) {
        this.val = val;
    }

    TreeNode(int val, TreeNode left, TreeNode right) {
        this.val = val;
        this.left = left;
        this.right = right;
    }
}

public class Main {

    static class BSTIterator {
        private Deque<TreeNode> stack;

        public BSTIterator(TreeNode root) {
            stack = new ArrayDeque<>();
            pushLeft(root);
        }

        private void pushLeft(TreeNode node) {
            while (node != null) {
                stack.push(node);
                node = node.left;
            }
        }

        public int next() {
            TreeNode curr = stack.pop();

            if (curr.right != null) {
                pushLeft(curr.right);
            }

            return curr.val;
        }

        public boolean hasNext() {
            return !stack.isEmpty();
        }
    }

    public static void main(String[] args) {
        TreeNode root = new TreeNode(7);
        root.left = new TreeNode(3);
        root.right = new TreeNode(15);
        root.right.left = new TreeNode(9);
        root.right.right = new TreeNode(20);

        System.out.println("示例输入: root = [7, 3, 15, null, null, 9, 20]");

        BSTIterator iterator = new BSTIterator(root);

        System.out.println("iterator.next()    -> " + iterator.next());
        System.out.println("iterator.next()    -> " + iterator.next());
        System.out.println("iterator.hasNext() -> " + iterator.hasNext());
        System.out.println("iterator.next()    -> " + iterator.next());
        System.out.println("iterator.hasNext() -> " + iterator.hasNext());
        System.out.println("iterator.next()    -> " + iterator.next());
        System.out.println("iterator.hasNext() -> " + iterator.hasNext());
        System.out.println("iterator.next()    -> " + iterator.next());
        System.out.println("iterator.hasNext() -> " + iterator.hasNext());

        System.out.println("-----------------------------");
        System.out.println("预期输出应为: 3, 7, true, 9, true, 15, true, 20, false");
    }
}
```
