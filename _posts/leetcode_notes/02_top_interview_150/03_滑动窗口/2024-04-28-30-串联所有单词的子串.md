## 题目

**描述：**

给定一个字符串 `s` 和一个字符串数组 `words` 。`words` 中所有字符串 **长度相同** 。

`s` 中的 **串联子串** 是指一个包含 `words` 中所有字符串以任意顺序排列连接起来的子串。

- 例如，如果 `words = ["ab","cd","ef"]`， 那么 `"abcdef"`， `"abefcd"`，`"cdabef"`， `"cdefab"`，`"efabcd"`， 和 `"efcdab"` 都是串联子串。`"acdbef"` 不是串联子串，因为它不是 `words` 中任何排列的连接。

返回所有串联子串在 `s` 中的开始索引。你可以以 **任意顺序** 返回答案。

**示例 1:**

- **输入:** `s = "barfoothefoobarman"`, `words = ["foo","bar"]`
- **输出:** `[0,9]`
- **解释:**

  因为 `words.length == 2` 同时 `words[i].length == 3`，连接的子字符串的长度必须为 6。

  子串 `"barfoo"` 开始位置是 0。它是 `words` 中以 `["bar","foo"]` 顺序排列的连接。

  子串 `"foobar"` 开始位置是 9。它是 `words` 中以 `["foo","bar"]` 顺序排列的连接。

  输出顺序无关紧要。返回 `[9,0]` 也是可以的。

**示例 2:**

- **输入:** `s = "wordgoodgoodgoodbestword"`, `words = ["word","good","best","word"]`
- **输出:** `[]`

**示例 3:**

- **输入:** `s = "barfoofoobarthefoobarman"`, `words = ["bar","foo","the"]`
- **输出:** `[6,9,12]`

**提示:**

- `1 <= s.length <= 10^4`
- `1 <= words.length <= 5000`
- `1 <= words[i].length <= 30`
- `words[i]` 和 `s` 由小写英文字母组成

## 题解

这题第一眼也像滑动窗口，但如果按字符级别去套普通窗口模板，通常很快就会卡住。它难的地方在于：窗口里装的不是单个字符，而是一块一块等长的单词；而且这些单词不要求顺序固定，只要求出现次数和 `words` 完全一致。

所以做这题时，最重要的不是“窗口”这两个字，而是先抓住题目给的那个特殊条件：`words` 里的所有单词长度都相同。只要这个条件存在，我们就不应该再按字符移动窗口，而应该按“单词块”来移动。假设每个单词长度是 `wordLen`，单词个数是 `wordNum`，那么一个合法答案的总长度就固定是 `wordLen * wordNum`，而窗口每次也应该按 `wordLen` 为步长前进。

这一步想清楚以后，结构就出来了。先用一个 `targetMap` 统计 `words` 里每个单词应出现多少次。然后不要只从下标 `0` 开始扫，而是把字符串按模 `wordLen` 的不同余数分成几组：从 `0` 开始一组，从 `1` 开始一组，一直到 `wordLen - 1`。因为只有在同一组里，切出来的块边界才和单词边界对齐。对每一组，我们再用滑动窗口维护当前区间里这些单词块的词频。

窗口向右扩时，每次读入一个长度为 `wordLen` 的单词块。如果这个块根本不在 `targetMap` 里，说明当前这整段窗口都没法继续用了，直接清空窗口状态，把左右边界一起跳到下一个位置。如果这个块在目标表里，就把它加入 `windowMap`。一旦某个单词在窗口里出现次数超标，就不断从左边弹出单词，直到它的次数恢复正常为止。

这里最容易想通的一点是：窗口失效时，问题只会出在刚刚新加入的这个单词上。因为在它进来之前，窗口一直都是合法推进的，所以收缩条件只需要盯着“当前这个单词是不是超标了”。这也是为什么代码里会写成 `while (windowMap.get(word) > targetMap.get(word))`，而不是重新去检查整张表。

当窗口里的单词数刚好等于 `wordNum` 时，说明当前窗口已经是一组合法答案了，这时把左边界记下来即可。但题目要找的是所有起点，所以不能停，还要继续往后滑。最自然的做法就是顺手把最左边那个单词弹出去，让窗口继续保持按块滑动的状态，这样后面新的单词才有机会和剩余部分组成下一组答案。

拿 `s = "barfoofoobarthefoobarman"`、`words = ["bar", "foo", "the"]` 来看，从对齐的某一组开始按三个字符一块一块切，会依次读到 `bar`、`foo`、`foo`、`bar`、`the`、`foo` ... 当第二个 `foo` 进来时，它在窗口里超标了，于是左边界开始按单词长度往右弹，直到多出来的 `foo` 被消化掉；当窗口恰好含有 `bar`、`foo`、`the` 各一个时，就记录一个起点。整个过程本质上还是滑动窗口，只不过窗口单位从字符换成了等长单词块。

代码如下：

```java
public List<Integer> findSubstring(String s, String[] words) {
    List<Integer> res = new ArrayList<>();

    if (s == null || s.length() == 0 || words == null || words.length == 0) {
        return res;
    }

    int wordLen = words[0].length();
    int wordNum = words.length;
    int totalLen = wordLen * wordNum;

    if (s.length() < totalLen) {
        return res;
    }

    Map<String, Integer> targetMap = new HashMap<>();
    for (String word : words) {
        targetMap.put(word, targetMap.getOrDefault(word, 0) + 1);
    }

    for (int i = 0; i < wordLen; i++) {
        int left = i;
        int right = i;
        int count = 0;
        Map<String, Integer> windowMap = new HashMap<>();

        while (right + wordLen <= s.length()) {
            String word = s.substring(right, right + wordLen);
            right += wordLen;

            if (!targetMap.containsKey(word)) {
                windowMap.clear();
                count = 0;
                left = right;
            } else {
                windowMap.put(word, windowMap.getOrDefault(word, 0) + 1);
                count++;

                while (windowMap.get(word) > targetMap.get(word)) {
                    String leftWord = s.substring(left, left + wordLen);
                    windowMap.put(leftWord, windowMap.get(leftWord) - 1);
                    count--;
                    left += wordLen;
                }

                if (count == wordNum) {
                    res.add(left);

                    String leftWord = s.substring(left, left + wordLen);
                    windowMap.put(leftWord, windowMap.get(leftWord) - 1);
                    count--;
                    left += wordLen;
                }
            }
        }
    }

    return res;
}
```

时间复杂度通常可以理解为 `O(n * wordLen)`，因为外层有 `wordLen` 组起点偏移，每一组里左右边界都按单词长度向前走且不会回退；空间复杂度主要来自目标词频表和窗口词频表。

这题很适合记住的，不只是“多起点滑动窗口”，而是一个切换视角的动作：当题目里出现固定长度块、频次匹配、连续区间这几个特征时，应该尽快把问题从字符级别切到块级别去想，很多看起来混乱的细节就会一下子规整起来。
