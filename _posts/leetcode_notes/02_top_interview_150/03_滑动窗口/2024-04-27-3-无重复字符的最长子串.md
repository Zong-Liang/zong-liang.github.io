## 题目

**描述：**

给定一个字符串 `s` ，请你找出其中不含有重复字符的 **最长子串** 的长度。

**示例 1:**

- **输入:** `s = "abcabcbb"`
- **输出:** 3
- **解释:** 因为无重复字符的最长子串是 `"abc"`，所以其长度为 3。注意 `"bca"` 和 `"cab"` 也是正确答案。

**示例 2:**

- **输入:** `s = "bbbbb"`
- **输出:** 1
- **解释:** 因为无重复字符的最长子串是 `"b"`，所以其长度为 1。

**示例 3:**

- **输入:** `s = "pwwkew"`
- **输出:** 3
- **解释:** 因为无重复字符的最长子串是 `"wke"`，所以其长度为 3。请注意，你的答案必须是 **子串** 的长度，`"pwke"` 是一个子序列，不是子串。

**提示:**

- `0 <= s.length <= 5 * 10^4`
- `s` 由英文字母、数字、符号和空格组成

## 题解

### 先看这题到底在找什么

这题要找的是**最长子串**，而不是子序列。

“子串”这两个字很关键，它意味着答案必须是字符串里一段**连续**的区间。只要题目同时出现了“连续”和“最值”这两个特征，就应该先想一想能不能用滑动窗口。

不过这题和上一道“长度最小的子数组”还有一点不一样。那道题的窗口是否合法，取决于区间和是否达到目标值；而这道题的窗口是否合法，取决于窗口里**有没有重复字符**。

所以这题的核心不是“怎么枚举所有子串”，而是：

**如何在向右扩展窗口的过程中，始终保证窗口内没有重复字符。**

### 最直接的想法为什么不够好

最容易想到的方法，是枚举每个起点，然后不断向右扩展，直到遇到重复字符为止，顺便更新最长长度。

这个思路当然能做出来，但效率不高。因为每次换一个起点，后面的很多字符都要重新检查一遍，存在大量重复计算。

这题真正更好的做法，不是每次都重新开始，而是维护一个“当前不重复的窗口”，让它在字符串上持续滑动。

### 窗口该怎么维护

我们用两个指针维护一个窗口 `[left, right]`，表示当前这段子串没有重复字符。

处理时让 `right` 逐个向右走，把字符一个个加入窗口。问题在于：  
如果新加入的字符已经在窗口里出现过，当前窗口就不合法了，必须把左边界往右挪。

这里还需要一个哈希表，记录**每个字符最近一次出现的位置**。

为什么一定要记“最近一次出现的位置”？  
因为当我们在 `right` 位置再次遇到某个字符时，窗口左边界应该直接跳到它上一次出现位置的后一位，这样才能一次性把重复问题处理干净。

也就是说，遇到重复字符时，不是慢慢一点点挪，而是直接跳。

### 用一个例子把过程走清楚

先看示例：

```text
s = "abcabcbb"
```

一开始：

- `left = 0`
- `maxLen = 0`
- 哈希表为空

#### `right = 0`，字符是 `a`

当前窗口是 `"a"`，没有重复，长度为 `1`。

记录：

```text
a -> 0
```

更新 `maxLen = 1`。

#### `right = 1`，字符是 `b`

当前窗口是 `"ab"`，没有重复，长度为 `2`。

记录：

```text
a -> 0
b -> 1
```

更新 `maxLen = 2`。

#### `right = 2`，字符是 `c`

当前窗口是 `"abc"`，没有重复，长度为 `3`。

记录：

```text
a -> 0
b -> 1
c -> 2
```

更新 `maxLen = 3`。

#### `right = 3`，字符是 `a`

这时发现 `a` 已经出现过，而且上一次位置是 `0`。  
说明当前窗口 `"abca"` 出现了重复字符。

为了让窗口重新合法，`left` 应该直接跳到 `0 + 1 = 1`。

跳完之后，窗口变成 `"bca"`，依然没有重复，长度还是 `3`。

然后把 `a` 的最新位置更新成 `3`。

后面继续这样做，最终最长长度就是 `3`。

这个过程里最关键的一点是：

**右指针负责不断尝试扩张窗口，左指针只在出现重复时才往右跳，用来恢复窗口的合法性。**

### 这题最容易写错的地方

真正容易出错的，不是“要不要用哈希表”，而是这一句：

```java
left = Math.max(left, map.get(c) + 1);
```

很多人第一次写时，会直接写成：

```java
left = map.get(c) + 1;
```

这样在某些情况下会出错，因为 `map` 里记录的是字符上一次出现的位置，但那个位置未必还在当前窗口里。

看一个最典型的例子：

```text
s = "abba"
```

我们简单走一下：

- 读到第一个 `a`，窗口是 `"a"`
- 读到 `b`，窗口是 `"ab"`
- 读到第二个 `b`，发现重复，`left` 跳到前一个 `b` 的后一位，也就是 `2`，窗口变成 `"b"`
- 再读到最后一个 `a`

这时 `a` 上一次出现的位置是 `0`，但那个 `a` 其实早就被窗口甩在左边了，已经不属于当前窗口了。  
如果你直接写：

```java
left = map.get('a') + 1 = 1
```

那就相当于把 `left` 从 `2` 拉回了 `1`，窗口反而变大了，这显然不对。

所以这里必须写成：

```java
left = Math.max(left, map.get(c) + 1);
```

意思就是：  
**左边界只能向右走，不能后退。**

这一点是整题最关键的细节。

### 代码

```java
public int lengthOfLongestSubstring(String s) {
    Map<Character, Integer> map = new HashMap<>();
    int maxLen = 0;
    int left = 0;

    for (int right = 0; right < s.length(); right++) {
        char c = s.charAt(right);

        if (map.containsKey(c)) {
            left = Math.max(left, map.get(c) + 1);
        }

        map.put(c, right);
        maxLen = Math.max(maxLen, right - left + 1);
    }

    return maxLen;
}
```

### 这段代码在做什么

整个过程其实就三件事：

第一，判断当前字符是否在哈希表里出现过。  
如果出现过，说明它曾经来过，我们就要考虑窗口里会不会产生重复。

第二，更新左边界。  
如果这个字符上一次出现的位置仍然落在当前窗口内，就把 `left` 跳到那个位置的后一位；如果它上一次出现的位置已经在窗口左边之外，那 `left` 不动。

第三，更新当前字符的最新位置，并计算当前窗口长度。  
因为此时窗口已经是合法的，所以 `right - left + 1` 就是当前这一步能得到的不重复子串长度。

### 为什么时间复杂度是 O(n)

虽然代码里既有循环，又有哈希表操作，但整体仍然是线性的。

因为：

- `right` 从左到右只遍历一遍字符串
- `left` 也只会不断向右移动，不会回退
- 哈希表的查询和更新平均都是 `O(1)`

所以总时间复杂度是 `O(n)`，空间复杂度是 `O(Σ)`，其中 `Σ` 表示字符集大小。

如果题目限定的是 ASCII 字符，也可以用数组代替哈希表，把常数再压小一点；但在题解里，`HashMap` 更直观，也更通用。

### 这题真正值得记住的点

这题真正值得记住的，不是“滑动窗口 + 哈希表”这几个字，而是这种处理方式：

**当窗口因为某个新字符变得不合法时，不一定非要一步一步缩；如果能够准确定位冲突位置，就可以让左边界直接跳过去。**

本题里，哈希表记录的是字符最近一次出现的位置，所以一旦发现重复，我们就能立刻知道左边界至少要跳到哪里。

以后再遇到“最长连续区间”“窗口内元素不能重复”“字符最近出现位置”这类题，基本都可以往这个方向想。

### 完整代码

```java
import java.util.HashMap;
import java.util.Map;

public class Main {

    public static int lengthOfLongestSubstring(String s) {
        Map<Character, Integer> map = new HashMap<>();
        int maxLen = 0;
        int left = 0;

        for (int right = 0; right < s.length(); right++) {
            char c = s.charAt(right);

            if (map.containsKey(c)) {
                left = Math.max(left, map.get(c) + 1);
            }

            map.put(c, right);
            maxLen = Math.max(maxLen, right - left + 1);
        }

        return maxLen;
    }

    public static void main(String[] args) {
        String s1 = "abcabcbb";
        System.out.println("示例 1 输入: s = \"" + s1 + "\"");
        System.out.println("示例 1 输出: " + lengthOfLongestSubstring(s1));
        System.out.println("-----------------------------");

        String s2 = "bbbbb";
        System.out.println("示例 2 输入: s = \"" + s2 + "\"");
        System.out.println("示例 2 输出: " + lengthOfLongestSubstring(s2));
        System.out.println("-----------------------------");

        String s3 = "pwwkew";
        System.out.println("示例 3 输入: s = \"" + s3 + "\"");
        System.out.println("示例 3 输出: " + lengthOfLongestSubstring(s3));
    }
}
```
