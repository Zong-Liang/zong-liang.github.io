## 题目

**描述：**

给你一个字符串 `s` 、一个字符串 `t` 。返回 `s` 中涵盖 `t` 所有字符的最小子串。如果 `s` 中不存在涵盖 `t` 所有字符的子串，则返回空字符串 `""` 。

**注意：**

- 对于 `t` 中重复字符，我们寻找的子字符串中该字符数量必须不少于 `t` 中该字符数量。
- 如果 `s` 中存在这样的子串，我们保证它是唯一的答案。

**示例 1:**

- **输入:** `s = "ADOBECODEBANC"`, `t = "ABC"`
- **输出:** `"BANC"`
- **解释:** 最小覆盖子串 "BANC" 包含来自字符串 t 的 'A'、'B' 和 'C'。

**示例 2:**

- **输入:** `s = "a"`, `t = "a"`
- **输出:** `"a"`
- **解释:** 整个字符串 s 是最小覆盖子串。

**示例 3:**

- **输入:** `s = "a"`, `t = "aa"`
- **输出:** `""`
- **解释:** t 中两个字符 'a' 均应包含在 s 的子串中，因此没有符合条件的子字符串，返回空字符串。

**提示:**

- `m == s.length`
- `n == t.length`
- `1 <= m, n <= 10^5`
- `s` 和 `t` 由英文字母组成

**进阶：** 你能设计一个在 `o(m+n)` 时间内解决此问题的算法吗？

## 题解

### 这题表面是找子串，本质是在找“刚好够用”的窗口

这题要求在 `s` 里找到一个最短子串，使它能够覆盖 `t` 的所有字符。

这里“覆盖”两个字不能只按集合来理解。  
如果 `t = "AABC"`，那么窗口里不只是要有 `A`、`B`、`C`，还必须至少有两个 `A`。所以这题真正要维护的，不是“哪些字符出现过”，而是：

**每个字符还差多少个。**

这也是它和很多普通滑动窗口题不一样的地方。  
普通题可能只关心“是否存在重复”“和是否达标”，而这题关心的是一整套字符频次是否全部满足。

### 为什么这题一定是滑动窗口

题目要找的是一个**最短的连续子串**。

只要题目同时出现“连续”和“最短”这两个特征，通常都要先考虑滑动窗口。因为这类问题天然适合这样处理：

- 右边界不断扩张，直到窗口第一次满足条件
- 一旦满足条件，就立刻收缩左边界，逼出更短的可行解
- 在扩张和收缩之间反复切换，最后得到全局最优答案

这题也是一样。  
难点不在于“要不要用滑动窗口”，而在于：

**窗口什么时候算满足条件，以及收缩时怎么判断它会不会失效。**

### 先把“满足条件”这件事说清楚

假设 `t = "ABC"`，那一个合法窗口至少要包含：

- `A` 一个
- `B` 一个
- `C` 一个

如果 `t = "AABC"`，那合法窗口至少要包含：

- `A` 两个
- `B` 一个
- `C` 一个

所以我们需要先统计 `t` 中每个字符需要出现多少次。  
之后在窗口扩张时，再统计当前窗口里这些字符已经出现了多少次。

只要窗口内每个目标字符的数量都不少于需求，这个窗口就是合法的。

然后我们要做的就只有一件事：

**每次窗口刚一合法，就尽量从左边缩，直到它马上要失效为止。**

因为这样得到的，才是当前右边界下的最短可行窗口。

### 用示例走一遍过程

看示例 1：

```text
s = "ADOBECODEBANC"
t = "ABC"
```

目标很明确，窗口里必须至少有 `A`、`B`、`C` 各一个。

一开始窗口是空的，右边界不断向右扩：

- 扫到 `A`，还不够
- 扫到 `D`、`O`，都不是关键字符，先放着
- 扫到 `B`，还不够
- 扫到 `E`，还是不够
- 扫到 `C`，这时窗口 `"ADOBEC"` 第一次已经覆盖了 `ABC`

既然已经覆盖，就不能满足于“有答案了”，而要马上尝试缩小：

- 去掉左边的 `A`，窗口就不再覆盖 `A` 了
- 所以 `"ADOBEC"` 已经是以当前右边界结尾时的最短可行窗口

接着继续向右扩张，直到再次补齐缺失的字符。  
后面会经历很多中间窗口，最后当窗口滑到 `"BANC"` 时，再次满足覆盖条件，而且长度更短，于是更新答案。

这个过程最关键的节奏就是：

- **右指针负责把窗口补到合法**
- **左指针负责把合法窗口压到最短**

### 最容易想明白的写法：哈希表统计需求和窗口

最直接的做法，是用两个哈希表：

- `need`：记录 `t` 中每个字符需要多少个
- `window`：记录当前窗口中每个字符有多少个

再维护一个变量 `valid`，表示当前有多少种字符已经满足了需求。

比如 `need` 里有 3 种字符：`A`、`B`、`C`。  
那么只要 `valid == 3`，就说明当前窗口已经覆盖了 `t`。

这个写法逻辑最清楚，很适合先把思路想顺。

### 哈希表写法的代码

```java
public String minWindow(String s, String t) {
    if (s == null || t == null || s.length() == 0 || t.length() == 0) {
        return "";
    }

    Map<Character, Integer> need = new HashMap<>();
    Map<Character, Integer> window = new HashMap<>();

    for (char c : t.toCharArray()) {
        need.put(c, need.getOrDefault(c, 0) + 1);
    }

    int left = 0, right = 0;
    int valid = 0;
    int start = 0, len = Integer.MAX_VALUE;

    while (right < s.length()) {
        char c = s.charAt(right);
        right++;

        if (need.containsKey(c)) {
            window.put(c, window.getOrDefault(c, 0) + 1);
            if (window.get(c).equals(need.get(c))) {
                valid++;
            }
        }

        while (valid == need.size()) {
            if (right - left < len) {
                start = left;
                len = right - left;
            }

            char d = s.charAt(left);
            left++;

            if (need.containsKey(d)) {
                if (window.get(d).equals(need.get(d))) {
                    valid--;
                }
                window.put(d, window.get(d) - 1);
            }
        }
    }

    return len == Integer.MAX_VALUE ? "" : s.substring(start, start + len);
}
```

这个写法的优点是可读性强。  
不过它在 Java 里会频繁进行 `HashMap` 操作和装箱拆箱，性能常数不算小。对于这道题，其实还能再往前走一步。

### 更常用的写法：直接用数组代替哈希表

这题里字符是英文字母，用数组来做频次统计会更直接。

这里有一个很巧的写法：  
我们只维护一个 `need` 数组，不再单独开 `window`。

先把 `t` 中所有字符的需求量统计进去：

```text
need[c] = t 中字符 c 还需要多少个
```

然后开始扩张窗口。

每读入一个字符 `c`：

- 如果 `need[c] > 0`，说明它是当前仍然需要的字符，那么匹配数 `count` 加一
- 然后不管它是不是需要的字符，都执行 `need[c]--`

这样一来：

- `need[c] > 0` 表示这个字符还缺
- `need[c] == 0` 表示这个字符刚刚好
- `need[c] < 0` 表示这个字符在窗口里已经多了

这套写法的好处是：  
**“还差多少”和“多出来多少”都被压进同一个数组里了。**

### 为什么 `count == t.length()` 就说明窗口合法

这个地方很多人第一次看会有点不放心。

比如 `t = "ABC"`，窗口里混进了很多别的字符，为什么只要 `count == 3` 就说明覆盖完成了？

原因在于：  
`count` 只会在读入“当前仍然需要的字符”时增加。

也就是说，只有当某个字符对补齐 `t` 真有贡献时，它才会让 `count++`。  
多余字符、无关字符，都不会增加 `count`。

因此，当 `count == t.length()` 时，表示 `t` 里的每一个字符需求都已经被满足了，窗口就是合法的。

### 收缩窗口时为什么这样写

当窗口已经合法后，我们开始移动 `left`。

每移出一个字符 `d`，先执行：

```java
need[d]++;
```

这表示把这个字符“还回去”。

如果还回去之后：

```java
need[d] > 0
```

那就说明这个字符原本正好够用，现在少了一个，窗口立刻失效，所以必须让 `count--`。

反过来说，如果 `need[d] <= 0`，说明这个字符本来就有富余，移出去一个也不影响合法性，窗口还能继续缩。

这一点就是整个数组写法的核心。

### 数组写法的代码

```java
public String minWindow(String s, String t) {
    if (s == null || s.length() == 0 || t == null || t.length() == 0) {
        return "";
    }

    int[] need = new int[128];
    for (char c : t.toCharArray()) {
        need[c]++;
    }

    int left = 0, right = 0;
    int count = 0;
    int minLen = Integer.MAX_VALUE;
    int start = 0;

    while (right < s.length()) {
        char c = s.charAt(right);

        if (need[c] > 0) {
            count++;
        }
        need[c]--;
        right++;

        while (count == t.length()) {
            if (right - left < minLen) {
                minLen = right - left;
                start = left;
            }

            char d = s.charAt(left);
            need[d]++;

            if (need[d] > 0) {
                count--;
            }
            left++;
        }
    }

    return minLen == Integer.MAX_VALUE ? "" : s.substring(start, start + minLen);
}
```

### 这题最容易写错的地方

这题真正容易错的，通常不是框架，而是几个细节判断。

#### 第一处：不能只判断字符是否出现过

这题不是集合匹配，而是频次匹配。  
如果 `t` 里有重复字符，窗口里也必须补齐对应数量。单纯判断“字符在不在”是不够的。

#### 第二处：`count` 统计的是“匹配到的字符总数”，不是“匹配到的字符种类数”

数组写法里，`count` 的目标是和 `t.length()` 比较，所以它统计的是总匹配字符数。  
比如 `t = "AABC"`，那最终 `count` 要达到 `4`，而不是 `3`。

#### 第三处：先判断再减，先加再判断，顺序不能乱

扩张窗口时要先看：

```java
if (need[c] > 0) {
    count++;
}
need[c]--;
```

收缩窗口时要先：

```java
need[d]++;
if (need[d] > 0) {
    count--;
}
```

这两个顺序都不能随便换。  
因为这里判断的不是字符本身，而是“操作前后是否还满足需求”。

### 复杂度怎么理解

无论是哈希表写法还是数组写法，本质上都是同一个滑动窗口。

- `right` 从左到右只走一遍
- `left` 也从左到右只走一遍
- 每个字符最多进窗口一次、出窗口一次

所以时间复杂度是 `O(m + n)`，其中：

- `m` 是 `s` 的长度
- `n` 是 `t` 的长度

空间复杂度：

- 哈希表写法是 `O(C)`
- 数组写法是 `O(1)`，因为数组大小固定

在 Java 里，实际写题时更常见的是数组版，速度和常数都会更漂亮一些。

### 这题真正值得记住的地方

这题真正值得记住的，不是“最小覆盖子串”这个名字，而是这一类题的共性：

**当题目要求找到最短连续区间，并且区间是否合格取决于一组频次约束时，就要考虑滑动窗口配合“需求表”来做。**

这里的关键不是把窗口撑大，而是：

- 知道什么时候窗口第一次合法
- 知道窗口合法后如何安全地缩
- 知道窗口失效的临界点在哪里

把这几点想明白之后，这题其实就是一套很标准的“扩张到合法，再收缩到极限”的过程。

### 完整代码

```java
public class Main {

    public static String minWindow(String s, String t) {
        if (s == null || s.length() == 0 || t == null || t.length() == 0) {
            return "";
        }

        int[] need = new int[128];
        for (char c : t.toCharArray()) {
            need[c]++;
        }

        int left = 0, right = 0;
        int count = 0;
        int minLen = Integer.MAX_VALUE;
        int start = 0;
        int tLen = t.length();

        while (right < s.length()) {
            char c = s.charAt(right);

            if (need[c] > 0) {
                count++;
            }
            need[c]--;
            right++;

            while (count == tLen) {
                if (right - left < minLen) {
                    minLen = right - left;
                    start = left;
                }

                char d = s.charAt(left);
                need[d]++;

                if (need[d] > 0) {
                    count--;
                }
                left++;
            }
        }

        return minLen == Integer.MAX_VALUE ? "" : s.substring(start, start + minLen);
    }

    public static void main(String[] args) {
        String s1 = "ADOBECODEBANC";
        String t1 = "ABC";
        System.out.println("示例 1 输入: s = \"" + s1 + "\", t = \"" + t1 + "\"");
        System.out.println("示例 1 输出: \"" + minWindow(s1, t1) + "\"");
        System.out.println("-----------------------------");

        String s2 = "a";
        String t2 = "a";
        System.out.println("示例 2 输入: s = \"" + s2 + "\", t = \"" + t2 + "\"");
        System.out.println("示例 2 输出: \"" + minWindow(s2, t2) + "\"");
        System.out.println("-----------------------------");

        String s3 = "a";
        String t3 = "aa";
        System.out.println("示例 3 输入: s = \"" + s3 + "\", t = \"" + t3 + "\"");
        System.out.println("示例 3 输出: \"" + minWindow(s3, t3) + "\"");
    }
}
```
