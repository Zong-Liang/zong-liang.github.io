## 题目

**描述：**

给你一个字符串 `s` 、一个字符串 `t` 。返回 `s` 中涵盖 `t` 所有字符的最小子串。如果 `s` 中不存在涵盖 `t` 所有字符的子串，则返回空字符串 `""` 。

**注意：**

- 对于 `t` 中重复字符，我们寻找的子字符串中该字符数量必须不少于 `t` 中该字符数量。
- 如果 `s` 中存在这样的子串，我们保证它是唯一的答案。

**示例 1:**

- **输入:** `s = "ADOBECODEBANC"`, `t = "ABC"`
- **输出:** `"BANC"`
- **解释:** 最小覆盖子串 "BANC" 包含来自字符串 t 的 'A'、'B' 和 'C'。

**示例 2:**

- **输入:** `s = "a"`, `t = "a"`
- **输出:** `"a"`
- **解释:** 整个字符串 s 是最小覆盖子串。

**示例 3:**

- **输入:** `s = "a"`, `t = "aa"`
- **输出:** `""`
- **解释:** t 中两个字符 'a' 均应包含在 s 的子串中，因此没有符合条件的子字符串，返回空字符串。

**提示:**

- `m == s.length`
- `n == t.length`
- `1 <= m, n <= 10^5`
- `s` 和 `t` 由英文字母组成

**进阶：** 你能设计一个在 `o(m+n)` 时间内解决此问题的算法吗？

## 题解

这题表面上是在找子串，更该盯住的不是“有没有这些字符”，而是“当前窗口是不是已经把 `t` 里每个字符的需求量都补齐了”。这点很重要，因为覆盖不是按集合算的，而是按频次算的。比如 `t = "AABC"` 时，窗口里不仅要有 `A`、`B`、`C`，还必须至少有两个 `A`。

一旦把问题理解成“补齐需求量”，滑动窗口的节奏就比较明确了。右边界不断扩张，把字符往窗口里加；只要窗口还没补齐所有需求，就继续扩。等到窗口第一次合法以后，左边界就开始尽量往右缩，因为题目要的是最短覆盖子串，不是随便找到一个可行窗口就结束。

比较顺手的一种写法，是直接用一个 `need` 数组同时表示“还差多少”和“多出来多少”。先把 `t` 中每个字符需要的次数统统加进 `need` 里。之后每读入 `s` 中的一个字符 `c`，如果 `need[c] > 0`，说明它对补齐需求有贡献，那么匹配数 `count` 就加一；无论它是不是有贡献，都执行 `need[c]--`。这样一来，`need[c] > 0` 表示这个字符还缺，`need[c] == 0` 表示刚好够，`need[c] < 0` 则表示窗口里这个字符已经多出来了。

这时候 `count == t.length()` 就意味着窗口已经合法，因为它表示 `t` 中每一个需要的字符都已经被完整补齐了，而不是只统计了种类数。比如 `t = "AABC"`，这里要等 `count` 变成 `4`，而不是 `3`。一旦窗口合法，就开始收缩左边界：每移出一个字符 `d`，先做 `need[d]++`，相当于把它“还回需求表”里；如果这一步之后 `need[d] > 0`，说明这个字符本来刚好够用，现在少了一个，窗口立刻失效，于是 `count--`，收缩停止。否则说明它原本就是富余的，还可以继续缩。

拿 `s = "ADOBECODEBANC"`、`t = "ABC"` 来看，窗口一开始会一路扩到 `"ADOBEC"`，这是第一个合法窗口，因为它第一次凑齐了 `A`、`B`、`C`。但它显然还不够短，所以左边界继续收缩；一旦把最左边那个 `A` 移出去，窗口就失效了，只能重新向右扩。后面同样的过程会在 `"BANC"` 这个窗口上再次发生，而这一次它的长度更短，于是成为最终答案。整道题的核心节奏就是：先扩到合法，再缩到极限。

代码如下：

```java
public String minWindow(String s, String t) {
    if (s == null || s.length() == 0 || t == null || t.length() == 0) {
        return "";
    }

    int[] need = new int[128];
    for (char c : t.toCharArray()) {
        need[c]++;
    }

    int left = 0, right = 0;
    int count = 0;
    int minLen = Integer.MAX_VALUE;
    int start = 0;

    while (right < s.length()) {
        char c = s.charAt(right);

        if (need[c] > 0) {
            count++;
        }
        need[c]--;
        right++;

        while (count == t.length()) {
            if (right - left < minLen) {
                minLen = right - left;
                start = left;
            }

            char d = s.charAt(left);
            need[d]++;

            if (need[d] > 0) {
                count--;
            }
            left++;
        }
    }

    return minLen == Integer.MAX_VALUE ? "" : s.substring(start, start + minLen);
}
```

时间复杂度是 `O(m + n)`，其中 `m` 是 `s` 的长度，`n` 是 `t` 的长度，因为左右指针都只会单向走一遍；空间复杂度是 `O(1)`，这里数组大小固定。

这题值得顺手记住的，不只是一个最小覆盖模板，而是“需求表”这个视角：当窗口是否合法取决于一组频次约束时，问题往往不是“窗口里有什么”，而是“窗口离满足目标还差多少”。一旦换成这个口径，很多判断顺序就会自然很多。

完整代码如下：

```java
public class Main {

    public static String minWindow(String s, String t) {
        if (s == null || s.length() == 0 || t == null || t.length() == 0) {
            return "";
        }

        int[] need = new int[128];
        for (char c : t.toCharArray()) {
            need[c]++;
        }

        int left = 0, right = 0;
        int count = 0;
        int minLen = Integer.MAX_VALUE;
        int start = 0;
        int tLen = t.length();

        while (right < s.length()) {
            char c = s.charAt(right);

            if (need[c] > 0) {
                count++;
            }
            need[c]--;
            right++;

            while (count == tLen) {
                if (right - left < minLen) {
                    minLen = right - left;
                    start = left;
                }

                char d = s.charAt(left);
                need[d]++;

                if (need[d] > 0) {
                    count--;
                }
                left++;
            }
        }

        return minLen == Integer.MAX_VALUE ? "" : s.substring(start, start + minLen);
    }

    public static void main(String[] args) {
        String s1 = "ADOBECODEBANC";
        String t1 = "ABC";
        System.out.println("示例 1 输入: s = \"" + s1 + "\", t = \"" + t1 + "\"");
        System.out.println("示例 1 输出: \"" + minWindow(s1, t1) + "\"");
        System.out.println("-----------------------------");

        String s2 = "a";
        String t2 = "a";
        System.out.println("示例 2 输入: s = \"" + s2 + "\", t = \"" + t2 + "\"");
        System.out.println("示例 2 输出: \"" + minWindow(s2, t2) + "\"");
        System.out.println("-----------------------------");

        String s3 = "a";
        String t3 = "aa";
        System.out.println("示例 3 输入: s = \"" + s3 + "\", t = \"" + t3 + "\"");
        System.out.println("示例 3 输出: \"" + minWindow(s3, t3) + "\"");
    }
}
```
