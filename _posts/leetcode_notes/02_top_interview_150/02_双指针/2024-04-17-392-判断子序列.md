## 题目

**描述：**

给定字符串 **s** 和 **t** ，判断 **s** 是否为 **t** 的子序列。

字符串的一个子序列是原始字符串删除一些（也可以不删除）字符而不改变剩余字符相对位置形成的新字符串。（例如，`"ace"`是`"abcde"`的一个子序列，而`"aec"`不是）。

**进阶:**
如果有大量输入的 S，称作 S1, S2, ..., Sk 其中 k >= 10 亿，你需要依次检查它们是否为 T 的子序列。在这种情况下，你会怎样改变代码？

**示例 1:**

- **输入:** `s = "abc"`, `t = "ahbgdc"`
- **输出:** `true`

**示例 2:**

- **输入:** `s = "axc"`, `t = "ahbgdc"`
- **输出:** `false`

**提示:**

- `0 <= s.length <= 100`
- `0 <= t.length <= 10^4`
- 两个字符串都只由小写字母组成。

## 题解

### 先把题目意思想透

这题问的是：`s` 能不能在 `t` 里按顺序找到。

注意这里说的是**按顺序找到**，不是要求连续。  
比如 `"abc"` 是 `"ahbgdc"` 的子序列，因为可以依次找到 `a`、`b`、`c`；但 `"axc"` 就不行，因为虽然有 `a` 和 `c`，中间那个 `x` 根本找不到。

所以这题真正要判断的，不是两个字符串像不像，而是：

**能不能顺着扫一遍 `t`，把 `s` 里的字符按顺序匹配完。**

### 原题本身，其实就是一次顺着找

如果只看原题，不考虑后面的进阶问题，那么最自然的做法就是双指针。

- 一个指针 `i` 指向 `s`
- 一个指针 `j` 指向 `t`

然后让 `j` 一路往后扫：

- 如果 `s[i] == t[j]`，说明当前这个字符匹配上了，`i` 往后走一步
- 不管匹不匹配，`j` 都继续往后走

最后如果 `i` 走到了 `s` 的末尾，就说明 `s` 的所有字符都已经按顺序在 `t` 中找到；否则就不是子序列。

这题最关键的点就在于：  
**`t` 可以跳过字符，但 `s` 不行。**

因为我们是在 `t` 里“寻找”`s`，所以 `t` 当前这个字符如果用不上，直接略过就行；但 `s` 只有当前字符匹配成功了，才能继续往后走。

### 为什么这样做不会漏

这个写法其实带一点很朴素的贪心。

每次在 `t` 中遇到一个和 `s[i]` 相同的字符时，直接拿来匹配，一定不会吃亏。因为子序列只要求相对顺序，不要求你“晚一点再选会更优”。既然当前就能匹配上，那越早用掉越好，后面还能给剩下的字符留出更多空间。

所以这题不需要回溯，也不需要动态规划。顺着扫过去就够了。

### 用示例走一遍

看示例 1：

```text
s = "abc"
t = "ahbgdc"
```

一开始：

- `i` 指向 `a`
- `j` 指向 `a`

两个字符相同，说明 `a` 找到了，`i` 往后走，开始找 `b`。

接着 `j` 继续往后扫，经过 `h`，没匹配上；再看到 `b`，匹配成功，`i` 再往后走，开始找 `c`。

后面同理，扫到 `c` 时又匹配成功。  
这时候 `s` 已经全部匹配完，所以返回 `true`。

再看示例 2：

```text
s = "axc"
t = "ahbgdc"
```

`a` 能找到，但往后再也找不到 `x`，最后 `t` 已经扫完了，`s` 还没匹配完，所以返回 `false`。

### 代码

```java
public boolean isSubsequence(String s, String t) {
    int i = 0, j = 0;
    while (i < s.length() && j < t.length()) {
        if (s.charAt(i) == t.charAt(j)) {
            i++;
        }
        j++;
    }
    return i == s.length();
}
```

时间复杂度是 $O(n)$，这里的 $n$ 是 `t` 的长度，因为本质上就是顺着扫一遍 `t`。  
空间复杂度是 $O(1)$，只用了两个指针。

### 进阶问题为什么不能还这么做

题目后面还有一句进阶：

如果有大量输入的 `s`，比如 `S1, S2, ..., Sk`，而 `t` 是固定的，该怎么办？

这时候如果还用上面的双指针，每来一个 `s` 都要把 `t` 从头扫一遍。单次看没问题，但如果查询次数特别大，总耗时就会非常夸张。

问题不在单次匹配，而在于**同一个 `t` 被反复扫描了太多次**。

既然 `t` 是固定的，那更合理的做法就是先把 `t` 预处理好，后面每次查询都直接利用这份结果。

### 进阶解法的核心：先存位置，再二分查找

因为题目里说字符串只由小写字母组成，所以我们完全可以先把 `t` 中每个字符出现的位置全部记下来。

比如：

```text
t = "bahbgdca"
```

那么可以得到：

- `'a'` 出现在 `[1, 7]`
- `'b'` 出现在 `[0, 3]`
- `'c'` 出现在 `[6]`
- 其他字符也类似

这样一来，判断某个 `s` 是否是子序列时，就不用再把整个 `t` 扫一遍了，而是改成下面这种方式：

假设当前已经匹配到了 `t` 中的位置 `currentIdx`，接下来要找字符 `c`，那就去 `c` 对应的位置列表里，找到**第一个大于 `currentIdx` 的位置**。

只要能找到，说明这个字符还能接在后面继续匹配；如果找不到，就说明后面已经没有合适的位置了，直接返回 `false`。

而这些位置列表本身就是递增的，所以很适合用二分查找。

### 进阶解法为什么更适合大量查询

这样做的代价是：

- 先花一次 $O(n)$ 的时间预处理 `t`
- 再花 $O(n)$ 的空间把各个字符的位置存下来

但后面每次查询时，就不用完整扫描 `t` 了。  
假设 `s` 的长度是 $m$，那么每个字符只需要在对应列表里做一次二分查找，单次查询大约是 $O(m \log n)$。

当查询次数非常多时，这样会比反复扫整个 `t` 高效得多。

### 进阶代码片段

```java
// 预处理 t
List<Integer>[] indexMap = new ArrayList[26];
for (int i = 0; i < 26; i++) {
    indexMap[i] = new ArrayList<>();
}
for (int i = 0; i < t.length(); i++) {
    indexMap[t.charAt(i) - 'a'].add(i);
}

// 查询某一个 s
int currentIdx = -1;
for (char c : s.toCharArray()) {
    List<Integer> list = indexMap[c - 'a'];
    int left = 0, right = list.size() - 1;
    int foundIdx = -1;

    while (left <= right) {
        int mid = left + (right - left) / 2;
        if (list.get(mid) > currentIdx) {
            foundIdx = list.get(mid);
            right = mid - 1;
        } else {
            left = mid + 1;
        }
    }

    if (foundIdx == -1) {
        return false;
    }

    currentIdx = foundIdx;
}
return true;
```

### 这题真正值得记住的地方

这题本身不难，关键是要分清楚两层：

第一层是原题。  
原题只需要判断一次，直接双指针顺着扫就够了，简单而且已经是最合适的写法。

第二层是进阶。  
一旦变成“同一个 `t` 要被查询很多很多次”，思路就得变。这个时候重点已经不是“怎么匹配一次”，而是“怎么把重复利用的信息提前存起来”。

所以这题真正值得记住的，不只是双指针本身，而是这种判断：

**单次查询，直接扫；重复查询，先预处理。**

### 完整代码

```java
public class Main {

    public static boolean isSubsequence(String s, String t) {
        if (s.length() > t.length()) {
            return false;
        }

        if (s.length() == 0) {
            return true;
        }

        int i = 0;
        int j = 0;

        while (i < s.length() && j < t.length()) {
            if (s.charAt(i) == t.charAt(j)) {
                i++;
            }
            j++;
        }

        return i == s.length();
    }

    public static void main(String[] args) {
        String s1 = "abc";
        String t1 = "ahbgdc";

        System.out.println("示例 1 输入: s = \"" + s1 + "\", t = \"" + t1 + "\"");
        System.out.println("示例 1 输出: " + isSubsequence(s1, t1));
        System.out.println("-----------------------------");

        String s2 = "axc";
        String t2 = "ahbgdc";

        System.out.println("示例 2 输入: s = \"" + s2 + "\", t = \"" + t2 + "\"");
        System.out.println("示例 2 输出: " + isSubsequence(s2, t2));
    }
}
```
