## 题目

**描述：**

如果在将所有大写字符转换为小写字符、并移除所有非字母数字字符之后，短语正着读和反着读都一样。则可以认为该短语是一个 **回文串** 。

字母和数字都属于字母数字字符。

给你一个字符串 `s`，如果它是 **回文串** ，返回 `true` ；否则，返回 `false` 。

**示例 1:**

- **输入:** `s = "A man, a plan, a canal: Panama"`
- **输出:** `true`
- **解释:** `"amanaplanacanalpanama"` 是回文串。

**示例 2:**

- **输入:** `s = "race a car"`
- **输出:** `false`
- **解释:** `"raceacar"` 不是回文串。

**示例 3:**

- **输入:** `s = " "`
- **输出:** `true`
- **解释:** 在移除除非字母数字字符之后，`s` 是一个空字符串 `""` 。由于空字符串正着反着读都一样，所以是回文串。

**提示:**

- `1 <= s.length <= 2 * 10^5`
- `s` 仅由可打印的 ASCII 字符组成

## 题解

### 这题麻烦的地方，不在回文本身

如果题目只是判断一个字符串是不是回文，其实很简单，左右两端往中间比就行了。

这题多了两个限制：

- 要忽略所有非字母数字字符
- 要忽略大小写

也就是说，真正要比较的并不是原字符串，而是“处理之后”的结果。

最直接的想法当然是先把字符串清洗一遍：把无关字符去掉，把字母都转成小写，再判断它是不是回文。这个思路没问题，也很好想到，但它会额外开一段新空间。

这题其实没必要这么做。因为我们并不真的关心“处理后的字符串长什么样”，只关心它从两边往中间看，是否始终对应相等。既然这样，就可以直接在原字符串上判断。

### 先看最容易想到的做法

先遍历一遍字符串，把所有字母和数字提取出来，同时统一转成小写，拼成一个新字符串；然后再用双指针判断这个新字符串是不是回文。

这个做法逻辑很顺，也容易写，但空间复杂度是 $O(n)$。

如果只是做题，这么写也完全可以接受；但既然这题本身就适合原地判断，那就没有必要专门再构造一个新串。

### 这题更自然的写法，是在原串上直接判断

既然非字母数字字符本来就不用参与比较，那遇到它们时直接跳过就行。

于是可以用两个指针：

- `left` 从左往右走
- `right` 从右往左走

每一轮先做两件事：

1. `left` 跳过所有非字母数字字符
2. `right` 跳过所有非字母数字字符

等两个指针都停在有效字符上之后，再把这两个字符统一转成小写进行比较：

- 如果不相等，直接返回 `false`
- 如果相等，就继续往中间收缩

一直比较到两个指针交错为止。

这样做的好处是，整个过程只是在原字符串上移动指针，不需要额外构造新字符串。

### 为什么双指针在这里正合适

回文判断本来就天然适合双指针，因为它只关心“对称位置是否相同”。

而这题虽然多了“忽略无关字符”和“忽略大小写”两个条件，但本质并没有变。无非就是在真正比较之前，多做一步筛选而已。

换句话说，这题不是把字符串处理完再去判断回文，而是一边移动指针，一边完成“过滤”和“比较”这两件事。

这样写出来，逻辑反而更紧凑。

### 用示例走一遍

看示例 1：

```text
s = "A man, a plan, a canal: Panama"
```

一开始：

- `left` 指向 `'A'`
- `right` 指向 `'a'`

两边都是字母，转成小写后分别是 `'a'` 和 `'a'`，相等，继续往中间走。

接下来会遇到空格、逗号、冒号这些字符。它们都不是字母数字，所以直接跳过，不参与比较。

最后真正参与比较的其实是这个序列：

```text
amanaplanacanalpanama
```

它正着读和反着读都一样，所以返回 `true`。

再看示例 2：

```text
s = "race a car"
```

跳过空格之后，前面和后面会比较到 `'e'` 和 `'a'`，这时不相等，直接返回 `false`。

### 这题里有个细节很容易忽略

像示例 3 这种情况：

```text
s = " "
```

字符串里只有空格，没有任何字母或数字。按题意把无关字符去掉之后，剩下的是空串。

而空串也算回文串，因为它正着读反着读都一样。

所以这题里，“清洗之后为空”并不是特殊错误情况，照样应该返回 `true`。双指针写法天然就能处理这一点，因为两个指针在跳过无关字符后，很快就会结束循环。

### 代码

```java
public boolean isPalindrome(String s) {
    if (s == null || s.length() == 0) {
        return true;
    }

    int left = 0;
    int right = s.length() - 1;

    while (left < right) {
        while (left < right && !Character.isLetterOrDigit(s.charAt(left))) {
            left++;
        }

        while (left < right && !Character.isLetterOrDigit(s.charAt(right))) {
            right--;
        }

        if (Character.toLowerCase(s.charAt(left)) != Character.toLowerCase(s.charAt(right))) {
            return false;
        }

        left++;
        right--;
    }

    return true;
}
```

时间复杂度是 $O(n)$。虽然看起来有两层 `while`，但两个指针都只会单向移动，每个字符最多被访问一次。

空间复杂度是 $O(1)$，因为整个过程只用了几个额外变量。

### 这题真正值得记住的地方

这题不难，关键是别把它写重了。

很多时候，题目里要求“忽略某些字符”或者“统一大小写”，第一反应会是先把字符串处理干净，再做判断。这个思路没有错，但不一定是最顺手的写法。

如果最终目标本身就适合双指针，那么更自然的做法，往往是让指针在移动过程中顺手把这些额外条件一起处理掉。这样代码不但更省空间，思路也更贴近问题本身。

### 完整代码

```java
public class Main {

    public static boolean isPalindrome(String s) {
        if (s == null || s.isEmpty()) {
            return true;
        }

        int left = 0;
        int right = s.length() - 1;

        while (left < right) {
            while (left < right && !Character.isLetterOrDigit(s.charAt(left))) {
                left++;
            }

            while (left < right && !Character.isLetterOrDigit(s.charAt(right))) {
                right--;
            }

            if (Character.toLowerCase(s.charAt(left)) != Character.toLowerCase(s.charAt(right))) {
                return false;
            }

            left++;
            right--;
        }

        return true;
    }

    public static void main(String[] args) {
        String s1 = "A man, a plan, a canal: Panama";
        System.out.println("示例 1 输入: s = \"" + s1 + "\"");
        System.out.println("示例 1 输出: " + isPalindrome(s1));
        System.out.println("-----------------------------");

        String s2 = "race a car";
        System.out.println("示例 2 输入: s = \"" + s2 + "\"");
        System.out.println("示例 2 输出: " + isPalindrome(s2));
        System.out.println("-----------------------------");

        String s3 = " ";
        System.out.println("示例 3 输入: s = \"" + s3 + "\"");
        System.out.println("示例 3 输出: " + isPalindrome(s3));
    }
}
```
