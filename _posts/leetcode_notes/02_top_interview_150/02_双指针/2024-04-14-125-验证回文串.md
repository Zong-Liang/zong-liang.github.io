## 题目

**描述：**

如果在将所有大写字符转换为小写字符、并移除所有非字母数字字符之后，短语正着读和反着读都一样。则可以认为该短语是一个 **回文串** 。

字母和数字都属于字母数字字符。

给你一个字符串 `s`，如果它是 **回文串** ，返回 `true` ；否则，返回 `false` 。

**示例 1:**

- **输入:** `s = "A man, a plan, a canal: Panama"`
- **输出:** `true`
- **解释:** `"amanaplanacanalpanama"` 是回文串。

**示例 2:**

- **输入:** `s = "race a car"`
- **输出:** `false`
- **解释:** `"raceacar"` 不是回文串。

**示例 3:**

- **输入:** `s = " "`
- **输出:** `true`
- **解释:** 在移除除非字母数字字符之后，`s` 是一个空字符串 `""` 。由于空字符串正着反着读都一样，所以是回文串。

**提示:**

- `1 <= s.length <= 2 * 10^5`
- `s` 仅由可打印的 ASCII 字符组成

## 题解

如果这题只是普通的回文判断，那几乎没什么难点，左右两个指针往中间比就行了。它稍微绕一点的地方在于，题目要求忽略所有非字母数字字符，而且比较时还不区分大小写。也就是说，我们要判断的，不是原字符串本身，而是“过滤并统一大小写之后”的那一串字符是不是回文。

最直观的做法当然是先把字符串清洗一遍：把字母和数字提取出来，顺手转成小写，拼成一个新串，再去判断这个新串是不是回文。这个办法没有问题，也很好写，但它会额外开一段 `O(n)` 的空间。其实这题没必要这么做，因为我们并不关心过滤后的字符串到底长什么样，只关心左右两边最终参与比较的有效字符能不能一一对应。

所以更贴题的写法，是直接在原字符串上做双指针。`left` 从左边出发，`right` 从右边出发。每一轮先让它们各自跳过无关字符，停在要参与比较的字母或数字上；然后把这两个字符统一转成小写，再看它们是否相同。相同就继续往中间收缩，不同就可以立刻返回 `false`。整个过程里，“过滤”和“比较”是同时完成的，不需要先专门做一遍预处理。

拿 `"A man, a plan, a canal: Panama"` 这个例子来看，两个指针一开始会分别停在左右两端的 `A` 和 `a` 上，转成小写以后可以匹配；接着它们会不断跳过空格、逗号、冒号这些无关字符，最后参与比较的其实就是 `amanaplanacanalpanama`。而像 `"race a car"` 这种例子，左右收缩到中间时很快就会遇到不相等的字符，于是可以直接判定不是回文。

这里还有个小边界顺手想一下会更安心：如果字符串里全是空格或标点，过滤完之后其实是空串。空串同样可以看成回文，所以像示例里的 `" "` 也应该返回 `true`。双指针写法天然就能把这种情况处理掉，因为两个指针跳过所有无关字符以后，循环会直接结束。

代码如下：

```java
public boolean isPalindrome(String s) {
    if (s == null || s.length() == 0) {
        return true;
    }

    int left = 0;
    int right = s.length() - 1;

    while (left < right) {
        while (left < right && !Character.isLetterOrDigit(s.charAt(left))) {
            left++;
        }

        while (left < right && !Character.isLetterOrDigit(s.charAt(right))) {
            right--;
        }

        if (Character.toLowerCase(s.charAt(left)) != Character.toLowerCase(s.charAt(right))) {
            return false;
        }

        left++;
        right--;
    }

    return true;
}
```

时间复杂度是 `O(n)`，因为两个指针都只会单向移动，每个字符最多被访问一次；空间复杂度是 `O(1)`，因为只用了几个额外变量。

这题可以顺手记住一个挺常见的处理方式：如果题目要求“忽略某些字符之后再判断”，而最终目标又本来就适合双指针，那通常可以让指针在移动时顺手把这些过滤条件一起处理掉，不一定非要先构造一个干净的新串。

完整代码如下：

```java
public class Main {

    public static boolean isPalindrome(String s) {
        if (s == null || s.isEmpty()) {
            return true;
        }

        int left = 0;
        int right = s.length() - 1;

        while (left < right) {
            while (left < right && !Character.isLetterOrDigit(s.charAt(left))) {
                left++;
            }

            while (left < right && !Character.isLetterOrDigit(s.charAt(right))) {
                right--;
            }

            if (Character.toLowerCase(s.charAt(left)) != Character.toLowerCase(s.charAt(right))) {
                return false;
            }

            left++;
            right--;
        }

        return true;
    }

    public static void main(String[] args) {
        String s1 = "A man, a plan, a canal: Panama";
        System.out.println("示例 1 输入: s = \"" + s1 + "\"");
        System.out.println("示例 1 输出: " + isPalindrome(s1));
        System.out.println("-----------------------------");

        String s2 = "race a car";
        System.out.println("示例 2 输入: s = \"" + s2 + "\"");
        System.out.println("示例 2 输出: " + isPalindrome(s2));
        System.out.println("-----------------------------");

        String s3 = " ";
        System.out.println("示例 3 输入: s = \"" + s3 + "\"");
        System.out.println("示例 3 输出: " + isPalindrome(s3));
    }
}
```
