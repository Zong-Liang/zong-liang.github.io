## 题目

**描述：**

给你一个下标从 **1** 开始的整数数组 `numbers` ，该数组已按 **非递减顺序排列** ，请你从数组中找出满足相加之和等于目标数 `target` 的两个数。如果设这两个数分别是 `numbers[index1]` 和 `numbers[index2]` ，则 `1 <= index1 < index2 <= numbers.length` 。

以长度为 2 的整数数组 `[index1, index2]` 的形式返回这两个整数的下标 `index1` 和 `index2`。

你可以假设每种输入 **只对应唯一的答案** ，而且你 **不可以** 重复使用相同的元素。

你所设计的解决方案必须只使用常数级的额外空间。

**示例 1:**

- **输入:** `numbers = [2,7,11,15]`, `target = 9`
- **输出:** `[1,2]`
- **解释:** 2 与 7 之和等于目标数 9 。因此 index₁ = 1, index₂ = 2 。返回 `[1, 2]` 。

**示例 2:**

- **输入:** `numbers = [2,3,4]`, `target = 6`
- **输出:** `[1,3]`
- **解释:** 2 与 4 之和等于目标数 6 。因此 index₁ = 1, index₂ = 3 。返回 `[1, 3]` 。

**示例 3:**

- **输入:** `numbers = [-1,0]`, `target = -1`
- **输出:** `[1,2]`
- **解释:** -1 与 0 之和等于目标数 -1 。因此 index₁ = 1, index₂ = 2 。返回 `[1, 2]` 。

**提示:**

- `2 <= numbers.length <= 3 * 10^4`
- `-1000 <= numbers[i] <= 1000`
- `numbers` 按 **非递减顺序** 排列
- `-1000 <= target <= 1000`
- **仅存在一个有效答案**

## 题解

### 这题和经典“两数之和”不是一回事

看到“两数之和”，很多人第一反应还是哈希表：一边遍历，一边查 `target - x` 是否出现过。

但这题和经典版有一个很关键的区别：**数组已经有序**，而且题目还明确要求只能使用常数级额外空间。

这两个条件一出来，哈希表其实就不是最合适的方向了。  
这题真正该抓住的点，不是“怎么快速找另一个数”，而是：

**既然数组有序，能不能直接利用顺序关系把范围不断缩小？**

### 有序数组最适合用对撞指针

既然数组是非递减顺序排列的，那最左边是当前最小的数，最右边是当前最大的数。

于是可以直接把这两个位置拿来试：

- 用 `left` 指向数组开头
- 用 `right` 指向数组结尾

然后看 `numbers[left] + numbers[right]` 和 `target` 的关系。

如果刚好等于 `target`，答案就找到了。  
如果和偏小，说明当前这个最小值太小了；如果和偏大，说明当前这个最大值太大了。

这样一来，下一步该移动哪个指针，其实是可以直接判断出来的。

### 为什么和小了就只能动左指针

这是这题最核心的一步，想清楚这里，整个双指针思路就顺了。

假设当前：

```text
sum = numbers[left] + numbers[right]
```

如果 `sum < target`，说明现在这两个数加起来还不够大。

这时候右边已经是当前范围内最大的数了，继续保留右边不动，想让和变大，就只能把左边换成一个更大的数。由于数组有序，左指针往右走，值才会变大。

所以这时候只能让 `left++`。

反过来，如果 `sum > target`，说明当前和太大了。  
左边已经是当前范围内最小的数，再保留左边不动，想让和变小，就只能把右边换成一个更小的数，也就是让 `right--`。

这就是为什么两个指针的移动方向不是拍脑袋决定的，而是由数组有序这个条件直接推出的。

### 用示例走一遍

看示例 1：

```text
numbers = [2,7,11,15], target = 9
```

一开始：

- `left = 0`，指向 `2`
- `right = 3`，指向 `15`

此时和是 `2 + 15 = 17`，比 `9` 大。  
既然和太大，就说明右边这个数太大了，右指针左移。

然后变成：

- `left = 0`，指向 `2`
- `right = 2`，指向 `11`

此时和是 `13`，还是大，继续让右指针左移。

再变成：

- `left = 0`，指向 `2`
- `right = 1`，指向 `7`

此时和是 `9`，刚好等于目标值。  
因为题目要求返回的是 **从 1 开始的下标**，所以结果是 `[1, 2]`。

### 这题为什么不需要回头

双指针好用的关键，在于每次移动一个指针时，都能排掉一整部分不可能的情况。

比如当前 `sum < target`，我们决定让 `left++`。这意味着当前位置的 `left` 已经不可能再参与任何正确答案了，因为它和当前最大的 `numbers[right]` 搭配都还嫌小，换成更靠左的任何数只会更小，更不可能满足要求。

同理，当前 `sum > target` 时，当前位置的 `right` 也可以直接排除，因为它和当前最小的 `numbers[left]` 搭配都已经太大了。

正因为每一步都能排除掉确定不可能的一侧，所以整个过程只需要线性扫描一遍，不需要回头。

### 代码

```java
public int[] twoSum(int[] numbers, int target) {
    int left = 0;
    int right = numbers.length - 1;

    while (left < right) {
        int sum = numbers[left] + numbers[right];

        if (sum == target) {
            return new int[]{left + 1, right + 1};
        } else if (sum < target) {
            left++;
        } else {
            right--;
        }
    }

    return new int[]{-1, -1};
}
```

时间复杂度是 $O(n)$，因为两个指针只会朝中间各走一次。  
空间复杂度是 $O(1)$，只用了常数级额外变量。

### 这题真正值得记住的地方

这题的重点不是“双指针”这三个字本身，而是背后的判断方式：

**一旦数组有序，很多原本需要“查找”的问题，就可以改成“缩范围”的问题。**

经典“两数之和”里没有顺序信息，所以哈希表更自然；这题因为数组已经排好序，而且还限制额外空间，于是对撞指针就成了更贴题的解法。

以后只要看到这种特征：

- 数组有序
- 目标和、目标差、区间比较这类问题
- 允许从两端逐步缩小范围

基本都可以优先想一想双指针。

### 完整代码

```java
import java.util.Arrays;

public class Main {

    public static int[] twoSum(int[] numbers, int target) {
        int left = 0;
        int right = numbers.length - 1;

        while (left < right) {
            int sum = numbers[left] + numbers[right];

            if (sum == target) {
                return new int[]{left + 1, right + 1};
            } else if (sum < target) {
                left++;
            } else {
                right--;
            }
        }

        return new int[]{-1, -1};
    }

    public static void main(String[] args) {
        int[] numbers1 = new int[]{2, 7, 11, 15};
        int target1 = 9;

        System.out.println("示例 1 输入:");
        System.out.println("numbers = " + Arrays.toString(numbers1) + ", target = " + target1);

        int[] result1 = twoSum(numbers1, target1);

        System.out.println("示例 1 输出: " + Arrays.toString(result1));
        System.out.println("-----------------------------");

        int[] numbers2 = new int[]{2, 3, 4};
        int target2 = 6;

        System.out.println("示例 2 输入:");
        System.out.println("numbers = " + Arrays.toString(numbers2) + ", target = " + target2);

        int[] result2 = twoSum(numbers2, target2);

        System.out.println("示例 2 输出: " + Arrays.toString(result2));
        System.out.println("-----------------------------");

        int[] numbers3 = new int[]{-1, 0};
        int target3 = -1;

        System.out.println("示例 3 输入:");
        System.out.println("numbers = " + Arrays.toString(numbers3) + ", target = " + target3);

        int[] result3 = twoSum(numbers3, target3);

        System.out.println("示例 3 输出: " + Arrays.toString(result3));
    }
}
```
