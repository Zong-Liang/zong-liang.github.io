## 题目

**描述：**

给你一个下标从 **1** 开始的整数数组 `numbers` ，该数组已按 **非递减顺序排列** ，请你从数组中找出满足相加之和等于目标数 `target` 的两个数。如果设这两个数分别是 `numbers[index1]` 和 `numbers[index2]` ，则 `1 <= index1 < index2 <= numbers.length` 。

以长度为 2 的整数数组 `[index1, index2]` 的形式返回这两个整数的下标 `index1` 和 `index2`。

你可以假设每种输入 **只对应唯一的答案** ，而且你 **不可以** 重复使用相同的元素。

你所设计的解决方案必须只使用常数级的额外空间。

**示例 1:**

- **输入:** `numbers = [2,7,11,15]`, `target = 9`
- **输出:** `[1,2]`
- **解释:** 2 与 7 之和等于目标数 9 。因此 index₁ = 1, index₂ = 2 。返回 `[1, 2]` 。

**示例 2:**

- **输入:** `numbers = [2,3,4]`, `target = 6`
- **输出:** `[1,3]`
- **解释:** 2 与 4 之和等于目标数 6 。因此 index₁ = 1, index₂ = 3 。返回 `[1, 3]` 。

**示例 3:**

- **输入:** `numbers = [-1,0]`, `target = -1`
- **输出:** `[1,2]`
- **解释:** -1 与 0 之和等于目标数 -1 。因此 index₁ = 1, index₂ = 2 。返回 `[1, 2]` 。

**提示:**

- `2 <= numbers.length <= 3 * 10^4`
- `-1000 <= numbers[i] <= 1000`
- `numbers` 按 **非递减顺序** 排列
- `-1000 <= target <= 1000`
- **仅存在一个有效答案**

## 题解

这题和经典的“两数之和”看起来题型一样，落点其实不太一样。经典版因为数组无序，所以最自然的思路通常是哈希表；但这题专门告诉你数组已经升序排列，而且还要求额外空间尽量省。这两个条件一摆出来，思路就应该从“查找另一个数”切到“利用有序性缩小范围”。

最顺手的写法是对撞指针。`left` 指向最左边，`right` 指向最右边，当前这两个位置就是还没排除掉的范围里的最小值和最大值。先看 `numbers[left] + numbers[right]` 与 `target` 的关系：如果刚好相等，答案就找到了；如果和偏小，说明当前最小值太小，需要把 `left` 往右挪，让和变大；如果和偏大，说明当前最大值太大，需要把 `right` 往左挪，让和变小。

这里最重要的其实是“为什么和小了只能动左边，和大了只能动右边”。比如现在 `sum < target`，右边已经是当前范围内能拿到的最大值了，再继续保留右边不动，唯一能让和变大的办法就是把左边换成更大的数，也就是让左指针右移。反过来，如果 `sum > target`，左边已经是当前范围里最小的数了，想让和变小，就只能把右边换成更小的数。

拿 `numbers = [2, 7, 11, 15]`、`target = 9` 这个例子来看，一开始左右两端是 `2` 和 `15`，和是 `17`，太大了，所以右指针左移；再看 `2` 和 `11`，和是 `13`，还是大，继续左移；最后来到 `2` 和 `7`，和正好是 `9`，于是答案就是它们对应的下标。因为题目要求返回的是从 `1` 开始的下标，所以最后要记得把两个位置都加一。

双指针之所以好用，不只是因为“它能试出答案”，而是因为每移动一次，都能排掉一整侧不可能再参与答案的情况。比如当前和已经太小了，那以当前 `left` 为左端、配上任何更靠内的右端，得到的和只会更小，所以当前这个 `left` 直接就可以淘汰掉。这种“每一步都在缩小合法范围”的感觉，正是有序数组最适合拿来做双指针的原因。

代码如下：

```java
public int[] twoSum(int[] numbers, int target) {
    int left = 0;
    int right = numbers.length - 1;

    while (left < right) {
        int sum = numbers[left] + numbers[right];

        if (sum == target) {
            return new int[]{left + 1, right + 1};
        } else if (sum < target) {
            left++;
        } else {
            right--;
        }
    }

    return new int[]{-1, -1};
}
```

时间复杂度是 `O(n)`，因为两个指针只会朝中间各走一次；空间复杂度是 `O(1)`，只用了常数个额外变量。

这题更适合记住的，不是“双指针”这三个字本身，而是这个判断：一旦数组有序，很多原本需要额外查找结构的问题，都可以改写成从两端逐步缩范围的问题。

完整代码如下：

```java
import java.util.Arrays;

public class Main {

    public static int[] twoSum(int[] numbers, int target) {
        int left = 0;
        int right = numbers.length - 1;

        while (left < right) {
            int sum = numbers[left] + numbers[right];

            if (sum == target) {
                return new int[]{left + 1, right + 1};
            } else if (sum < target) {
                left++;
            } else {
                right--;
            }
        }

        return new int[]{-1, -1};
    }

    public static void main(String[] args) {
        int[] numbers1 = new int[]{2, 7, 11, 15};
        int target1 = 9;

        System.out.println("示例 1 输入:");
        System.out.println("numbers = " + Arrays.toString(numbers1) + ", target = " + target1);

        int[] result1 = twoSum(numbers1, target1);

        System.out.println("示例 1 输出: " + Arrays.toString(result1));
        System.out.println("-----------------------------");

        int[] numbers2 = new int[]{2, 3, 4};
        int target2 = 6;

        System.out.println("示例 2 输入:");
        System.out.println("numbers = " + Arrays.toString(numbers2) + ", target = " + target2);

        int[] result2 = twoSum(numbers2, target2);

        System.out.println("示例 2 输出: " + Arrays.toString(result2));
        System.out.println("-----------------------------");

        int[] numbers3 = new int[]{-1, 0};
        int target3 = -1;

        System.out.println("示例 3 输入:");
        System.out.println("numbers = " + Arrays.toString(numbers3) + ", target = " + target3);

        int[] result3 = twoSum(numbers3, target3);

        System.out.println("示例 3 输出: " + Arrays.toString(result3));
    }
}
```
