## 题目

**描述：**

给你一个整数数组 `nums` ，判断是否存在三元组 `[nums[i], nums[j], nums[k]]` 满足 `i != j`、`i != k` 且 `j != k` ，同时还满足 `nums[i] + nums[j] + nums[k] == 0` 。请你返回所有和为 `0` 且不重复的三元组。

**注意：** 答案中不可以包含重复的三元组。

**示例 1:**

- **输入:** `nums = [-1,0,1,2,-1,-4]`
- **输出:** `[[-1,-1,2],[-1,0,1]]`
- **解释:**
  - `nums[0] + nums[1] + nums[2] = (-1) + 0 + 1 = 0` 。
  - `nums[1] + nums[2] + nums[4] = 0 + 1 + (-1) = 0` 。
  - `nums[0] + nums[3] + nums[4] = (-1) + 2 + (-1) = 0` 。
  - 不同的三元组是 `[-1,0,1]` 和 `[-1,-1,2]` 。
  - 注意，输出的顺序和三元组的顺序并不重要。

**示例 2:**

- **输入:** `nums = [0,1,1]`
- **输出:** `[]`
- **解释:** 唯一可能的三元组和不为 0 。

**示例 3:**

- **输入:** `nums = [0,0,0]`
- **输出:** `[[0,0,0]]`
- **解释:** 唯一可能的三元组和为 0 。

**提示:**

- `3 <= nums.length <= 3000`
- `-10^5 <= nums[i] <= 10^5`

## 题解

### 这题难的不是找三个数，而是去重

如果只要求找出有没有三元组满足和为 0，其实问题还没那么麻烦。  
真正把这题拉难的，是题目还要求：

- 返回所有满足条件的三元组
- 结果里不能有重复答案

也就是说，这题不是单纯的“求和”，而是“**在不漏解的前提下，把重复结果处理干净**”。

这也是为什么很多人第一次做这题时，明明思路差不多对了，最后却总是在去重上出问题。

### 暴力能想到，但肯定不够用

最直接的办法当然是三层循环，把所有三元组都枚举出来，再挑出和为 0 的组合。

这个思路没有问题，但时间复杂度是 $O(n^3)$。  
题目里 `nums.length` 最大能到 `3000`，三层枚举肯定会超时。

所以这题真正需要解决的是：

**能不能在少枚举一层的同时，还把重复答案处理掉？**

### 排序之后，问题会清楚很多

这题最关键的一步，是先排序。

排序的作用不只是“让数组更整齐”，而是直接解决了两个核心问题：

第一，方便用双指针找两数之和。  
第二，方便去重，因为相同的数会挨在一起。

排完序之后，数组从小到大排列。  
这时候我们可以先固定第一个数 `nums[i]`，再在它后面的区间里找另外两个数，使三者之和为 0。

换句话说，这题其实是在做一层转换：

**三数之和 = 固定一个数 + 在剩下部分里做两数之和。**

### 固定一个数之后，剩下的就变成双指针

当第一个数固定为 `nums[i]` 之后，我们要找的是：

```text
nums[L] + nums[R] = -nums[i]
```

这就变成了一个非常典型的有序数组双指针问题。

- `L` 从 `i + 1` 开始
- `R` 从数组末尾开始

然后计算：

```text
sum = nums[i] + nums[L] + nums[R]
```

如果：

- `sum == 0`，说明找到一组答案
- `sum < 0`，说明和太小，需要把左指针右移
- `sum > 0`，说明和太大，需要把右指针左移

这里的移动逻辑，和前面“两数之和 II”其实是一脉相承的，本质上还是在利用排序后的单调性缩范围。

### 这题真正麻烦的地方，是去重分三层

这题去重不能只做一次，而是要分清楚三处。

#### 第一处：固定第一个数时去重

假设排完序后，当前 `nums[i]` 和前一个数相同，那就没必要再以它作为起点重新做一遍。

因为前一个相同的数已经把所有可能情况处理过了，再来一次只会得到重复答案。

所以要写：

```java
if (i > 0 && nums[i] == nums[i - 1]) continue;
```

这是第一层去重。

#### 第二处：找到答案后，左指针要跳过重复值

比如当前已经找到一组：

```text
[nums[i], nums[L], nums[R]]
```

这时如果 `nums[L]` 后面还是相同的值，那继续拿它参与组合，只会得到重复答案，所以左指针要一直往后跳，直到走到一个新值。

#### 第三处：找到答案后，右指针也要跳过重复值

右边同理。  
如果 `nums[R]` 左边还是相同的值，也要继续跳过，不然还是会重复。

很多人做这题时，主逻辑其实没问题，最后结果不对，往往就是因为这三层去重没有分清楚：  
**固定值要去重，找到答案之后左右两边也都要去重。**

### 为什么可以提前结束

还有一个很常见、也很自然的优化：

如果当前固定的 `nums[i] > 0`，那后面就没必要继续了。

因为数组已经有序，`nums[i]` 后面的数只会更大。  
三个数都大于 0，不可能再加出 0。

所以一旦出现 `nums[i] > 0`，可以直接 `break`。

这个优化不影响正确性，只是少做一些无意义的扫描。

### 用示例走一遍

看示例 1：

```text
nums = [-1,0,1,2,-1,-4]
```

先排序，得到：

```text
[-4,-1,-1,0,1,2]
```

先固定 `i = 0`，也就是 `nums[i] = -4`。  
这时候目标变成在后面找两个数，让它们的和等于 `4`。

用双指针扫一遍，会发现没有答案。

接着固定 `i = 1`，也就是 `nums[i] = -1`。  
这时候目标变成找两个数，让它们的和等于 `1`。

- 左指针在第二个 `-1`
- 右指针在 `2`

三数之和正好是 `0`，得到第一组：

```text
[-1,-1,2]
```

然后继续收缩，后面还能找到：

```text
[-1,0,1]
```

再往后，`i = 2` 时，当前值还是 `-1`，和前一个相同，所以直接跳过，不然结果就重复了。

最终答案就是：

```text
[[-1,-1,2],[-1,0,1]]
```

### 代码

```java
public List<List<Integer>> threeSum(int[] nums) {
    List<List<Integer>> ans = new ArrayList<>();
    if (nums == null || nums.length < 3) {
        return ans;
    }

    Arrays.sort(nums);

    for (int i = 0; i < nums.length; i++) {
        if (nums[i] > 0) {
            break;
        }

        if (i > 0 && nums[i] == nums[i - 1]) {
            continue;
        }

        int L = i + 1;
        int R = nums.length - 1;

        while (L < R) {
            int sum = nums[i] + nums[L] + nums[R];

            if (sum == 0) {
                ans.add(Arrays.asList(nums[i], nums[L], nums[R]));
                L++;
                R--;

                while (L < R && nums[L] == nums[L - 1]) {
                    L++;
                }
                while (L < R && nums[R] == nums[R + 1]) {
                    R--;
                }
            } else if (sum < 0) {
                L++;
            } else {
                R--;
            }
        }
    }

    return ans;
}
```

时间复杂度是 $O(n^2)$。  
排序是 $O(n \log n)$，外层枚举一个数，内层双指针扫一遍，整体还是 $O(n^2)$。

空间复杂度取决于排序实现，一般记为 $O(\log n)$`或`O(n)`；如果不算结果集本身，额外思路空间不大。

### 这题真正值得记住的地方

这题最值得记住的，不只是“排序 + 双指针”这套写法，而是这个拆解方式：

**遇到三数问题，先想能不能固定一个数，把它降成两数问题。**

很多看起来复杂的多数求和题，本质上都是这么做的：

- 先固定一部分
- 再在剩下的区间里解决更小规模的问题

而这题另外一个必须记住的点，就是：

**去重一定要跟着搜索过程走，不能最后再临时补。**

因为这题的重复，不只来自结果相同，更来自搜索过程中会反复遇到相同的值。排序之后顺手处理掉，才是最自然、也最稳的写法。

### 完整代码

```java
import java.util.ArrayList;
import java.util.Arrays;
import java.util.List;

public class Main {

    public static List<List<Integer>> threeSum(int[] nums) {
        List<List<Integer>> ans = new ArrayList<>();
        if (nums == null || nums.length < 3) {
            return ans;
        }

        Arrays.sort(nums);

        for (int i = 0; i < nums.length; i++) {
            if (nums[i] > 0) {
                break;
            }

            if (i > 0 && nums[i] == nums[i - 1]) {
                continue;
            }

            int L = i + 1;
            int R = nums.length - 1;

            while (L < R) {
                int sum = nums[i] + nums[L] + nums[R];

                if (sum == 0) {
                    ans.add(Arrays.asList(nums[i], nums[L], nums[R]));
                    L++;
                    R--;

                    while (L < R && nums[L] == nums[L - 1]) {
                        L++;
                    }
                    while (L < R && nums[R] == nums[R + 1]) {
                        R--;
                    }
                } else if (sum < 0) {
                    L++;
                } else {
                    R--;
                }
            }
        }

        return ans;
    }

    public static void main(String[] args) {
        int[] nums1 = {-1, 0, 1, 2, -1, -4};
        System.out.println("示例 1 输入: " + Arrays.toString(nums1));
        List<List<Integer>> res1 = threeSum(nums1);
        System.out.println("示例 1 输出: " + res1);
        System.out.println("-----------------------------");

        int[] nums2 = {0, 1, 1};
        System.out.println("示例 2 输入: " + Arrays.toString(nums2));
        List<List<Integer>> res2 = threeSum(nums2);
        System.out.println("示例 2 输出: " + res2);
        System.out.println("-----------------------------");

        int[] nums3 = {0, 0, 0};
        System.out.println("示例 3 输入: " + Arrays.toString(nums3));
        List<List<Integer>> res3 = threeSum(nums3);
        System.out.println("示例 3 输出: " + res3);
    }
}
```
