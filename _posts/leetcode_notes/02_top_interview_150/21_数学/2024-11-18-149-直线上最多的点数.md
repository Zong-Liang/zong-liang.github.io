## 题目

**描述：**

给你一个数组 `points`，其中 `points[i] = [xi, yi]` 表示 **X-Y** 平面上的一个点。求最多有多少个点在同一条直线上。

**示例 1:**

![示例1图片](https://assets.leetcode.com/uploads/2021/02/25/plane1.jpg)

- **输入:** `points = [[1,1],[2,2],[3,3]]`
- **输出:** 3

**示例 2:**

![示例2图片](https://assets.leetcode.com/uploads/2021/02/25/plane2.jpg)

- **输入:** `points = [[1,1],[3,2],[5,3],[4,1],[2,3],[1,4]]`
- **输出:** 4

**提示:**

- `1 <= points.length <= 300`
- `points[i].length == 2`
- `-10^4 <= xi, yi <= 10^4`
- `points` 中的所有点 **互不相同**

## 题解

### 这题表面上是几何题，实质上是在统计“斜率”

两点可以确定一条直线。  
如果多个点在同一条直线上，那么站在其中某个点往外看，这些点和它连出来的直线方向一定是一样的。

换句话说，如果我们固定一个点 `points[i]` 作为基准点，那么：

- 和它共线的其他点
- 在“斜率”这个意义下
- 都会落到同一类里

所以这题最自然的思路其实是：

**枚举每个点作为起点，统计它和其他点形成的斜率，看看哪种斜率出现得最多。**

最多的那一类，再加上基准点自己，就是经过这个点的最大共线点数。

### 为什么不能直接用 `double` 算斜率

如果直接写：

```java
double slope = (double) dy / dx;
```

表面上很方便，但这里其实有精度风险。

比如某些分数斜率，像 `1/3`、`2/6`、`-4/-12`，理论上表示的是同一个方向，但转成浮点数之后，可能会因为精度问题导致比较不稳定。

这类题里，更稳的做法不是算小数，而是直接保存一个“约分后的方向”：

```text
dy / dx
```

比如：

- `2/4` 约成 `1/2`
- `-3/-6` 也约成 `1/2`
- `4/-2` 约成 `-2/1`

只要把它们统一成最简形式，再做好符号规范化，就能作为哈希表的 Key 来计数。

### 所以关键不是斜率值，而是“斜率的唯一表示”

对每一对点，我们先算：

```text
dx = x1 - x2
dy = y1 - y2
```

然后把 `dx` 和 `dy` 同时除以它们的最大公约数，化成最简形式。

比如：

```text
dx = 4, dy = 2   ->   2, 1
dx = -6, dy = -3 ->   2, 1
```

这样，同一条直线方向就会被归到同一个表示上。

但这里还有一步不能省：**符号统一**。

因为下面这两种写法本质上表示的是同一条斜率：

```text
1 / -1
-1 / 1
```

如果不统一，哈希表会把它们当成两种不同情况，结果就错了。

所以通常会约定：

- `dx` 尽量保持非负
- 如果 `dx == 0`，那就让 `dy` 保持正数

这样同一个方向最终只会落到一种表示上。

### 整体做法怎么想最顺

这题的标准思路其实很直接：

#### 第一步：枚举基准点

外层循环枚举每个点 `points[i]`，把它当成“起点”。

#### 第二步：看它和后面每个点形成什么斜率

内层循环枚举 `points[j]`，计算：

- `dx`
- `dy`
- 约分后的最简斜率表示

然后放到 `HashMap` 里计数。

这个 `HashMap` 的含义就是：

```text
某种斜率 -> 以 points[i] 为基准时，这种斜率出现了多少次
```

#### 第三步：更新答案

如果某个斜率出现了 `count` 次，说明除基准点外，还有 `count` 个点和它共线。  
所以经过这个基准点的共线点总数就是：

```text
count + 1
```

最后对所有基准点取最大值即可。

### 用例子看会更清楚

先看最简单的例子：

```text
[[1,1],[2,2],[3,3]]
```

固定 `[1,1]` 作为基准点。

它和 `[2,2]` 的方向是：

```text
dx = -1, dy = -1
```

约分后可以表示成同一种斜率。

它和 `[3,3]` 的方向是：

```text
dx = -2, dy = -2
```

约分之后，还是同一个表示。

所以对于基准点 `[1,1]` 来说，这个斜率出现了两次。  
再加上基准点自己，总共就是 `3` 个点共线。

这正好就是答案。

### 为什么时间复杂度是 `O(n^2)`

因为每个点都要和后面的点比较一次。

如果有 `n` 个点，那么大致要做：

```text
n + (n-1) + (n-2) + ...
```

数量级就是 `O(n^2)`。

这题里 `points.length <= 300`，所以这个复杂度是完全可以接受的。  
真正要注意的不是枚举本身，而是**斜率表示必须准确**。

### 剪枝为什么能提前结束

你原来的代码里有一个常见优化：

```java
if (maxCount >= n - i || maxCount > n / 2) {
    break;
}
```

这个优化的意思是：

- 如果当前最优答案已经不小于“剩下点数的上限”，后面不可能再超过它
- 或者当前答案已经足够大，后面的枚举也没必要继续

它不是理解题目的关键，可以当成性能上的顺手优化。  
真正的主线还是“枚举基准点 + 用哈希表统计斜率”。

### 代码

```java
public int maxPoints(int[][] points) {
    int n = points.length;
    if (n <= 2) {
        return n;
    }

    int maxCount = 0;

    for (int i = 0; i < n; i++) {
        if (maxCount >= n - i || maxCount > n / 2) {
            break;
        }

        Map<String, Integer> slopeMap = new HashMap<>();

        for (int j = i + 1; j < n; j++) {
            int deltaX = points[i][0] - points[j][0];
            int deltaY = points[i][1] - points[j][1];

            int gcdVal = gcd(deltaX, deltaY);
            deltaX /= gcdVal;
            deltaY /= gcdVal;

            if (deltaX < 0) {
                deltaX = -deltaX;
                deltaY = -deltaY;
            } else if (deltaX == 0) {
                deltaY = Math.abs(deltaY);
            }

            String key = deltaX + "_" + deltaY;
            slopeMap.put(key, slopeMap.getOrDefault(key, 0) + 1);
        }

        for (int count : slopeMap.values()) {
            maxCount = Math.max(maxCount, count + 1);
        }
    }

    return maxCount;
}
```

这里最关键的三步其实是：

- 用 `gcd` 约分
- 统一符号
- 用哈希表统计每种方向出现次数

只要这三步不乱，这题就能做对。

### 复杂度分析

- 时间复杂度：$O(n^2 \cdot \log M)$

外层和内层枚举点对，主复杂度是 `O(n^2)`；  
每次还要做一次 `gcd`，它的代价可以记成 `O(\log M)`，这里 `M` 是坐标差值的数量级。

- 空间复杂度：$O(n)$

对于某个固定基准点，哈希表里最多记录 `O(n)` 种不同斜率。

### 这题真正值得记住的地方

这题最值得记住的，不是“共线点怎么求”，而是这种建模方式：

**当几何关系不好直接判断时，先把它转成可计数的离散特征。**

这里的几何关系是“是否共线”，转出来的离散特征就是“相对于基准点的斜率”。

再进一步，为了避免精度问题，又把“斜率”转成了：

- `dy / dx` 的最简整数比
- 再做统一规范化

这样一个看起来像几何题的问题，最后就变成了“枚举 + 哈希计数”的题。

这种转化很典型，比单独记住代码更有价值。

### 完整代码

```java
import java.util.HashMap;
import java.util.Map;

public class Main {

    public static int maxPoints(int[][] points) {
        int n = points.length;
        if (n <= 2) {
            return n;
        }

        int maxCount = 0;

        for (int i = 0; i < n; i++) {
            if (maxCount >= n - i || maxCount > n / 2) {
                break;
            }

            Map<String, Integer> slopeMap = new HashMap<>();

            for (int j = i + 1; j < n; j++) {
                int deltaX = points[i][0] - points[j][0];
                int deltaY = points[i][1] - points[j][1];

                int gcdVal = gcd(deltaX, deltaY);
                deltaX /= gcdVal;
                deltaY /= gcdVal;

                if (deltaX < 0) {
                    deltaX = -deltaX;
                    deltaY = -deltaY;
                } else if (deltaX == 0) {
                    deltaY = Math.abs(deltaY);
                }

                String key = deltaX + "_" + deltaY;
                slopeMap.put(key, slopeMap.getOrDefault(key, 0) + 1);
            }

            for (int count : slopeMap.values()) {
                maxCount = Math.max(maxCount, count + 1);
            }
        }

        return maxCount;
    }

    private static int gcd(int a, int b) {
        while (b != 0) {
            int temp = a % b;
            a = b;
            b = temp;
        }
        return a;
    }

    public static void main(String[] args) {
        int[][] points1 = {{1, 1}, {2, 2}, {3, 3}};
        System.out.println("示例 1 输入: [[1,1],[2,2],[3,3]]");
        System.out.println("示例 1 输出: " + maxPoints(points1));
        System.out.println("-----------------------------");

        int[][] points2 = {{1, 1}, {3, 2}, {5, 3}, {4, 1}, {2, 3}, {1, 4}};
        System.out.println("示例 2 输入: [[1,1],[3,2],[5,3],[4,1],[2,3],[1,4]]");
        System.out.println("示例 2 输出: " + maxPoints(points2));
        System.out.println("-----------------------------");

        int[][] points3 = {{0, 0}, {0, 1}, {0, 2}};
        System.out.println("特殊测试 输入: [[0,0],[0,1],[0,2]]");
        System.out.println("特殊测试 输出: " + maxPoints(points3));
    }
}
```
