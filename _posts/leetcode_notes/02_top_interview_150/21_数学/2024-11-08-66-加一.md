## 题目

**描述：**

给定一个表示 **大整数** 的整数数组 `digits`，其中 `digits[i]` 是整数的第 `i` 位数字。这些数字按从左到右，从最高位到最低位排列。这个大整数不包含任何前导 0。

将大整数加 1，并返回结果的数字数组。

**示例 1:**

- **输入:** `digits = [1,2,3]`
- **输出:** `[1,2,4]`
- **解释:** 输入数组表示数字 123。加 1 后得到 123 + 1 = 124。因此，结果应该是 `[1,2,4]`。

**示例 2:**

- **输入:** `digits = [4,3,2,1]`
- **输出:** `[4,3,2,2]`
- **解释:** 输入数组表示数字 4321。加 1 后得到 4321 + 1 = 4322。因此，结果应该是 `[4,3,2,2]`。

**示例 3:**

- **输入:** `digits = [9]`
- **输出:** `[1,0]`
- **解释:** 输入数组表示数字 9。加 1 后得到 9 + 1 = 10。因此，结果应该是 `[1,0]`。

**提示:**

- `1 <= digits.length <= 100`
- `0 <= digits[i] <= 9`
- `digits` 不包含任何前导 0。

## 题解

### 这题本质上就是在模拟加法

这道题看起来是数组题，其实本质上做的事情很简单：  
把一个用数组表示的大整数，加上 `1`。

比如：

```text
[1,2,3]  表示 123
```

加一之后变成：

```text
[1,2,4]
```

所以这题真正要做的，不是什么复杂运算，而是把我们平时手算加法时的“进位过程”用代码写出来。

### 为什么要从后往前处理

数组里的数字是从高位到低位存的。

也就是说：

- 开头是高位
- 末尾是个位

而“加一”这件事，一定是从个位开始影响的。  
因为你给一个数加 `1`，最先变化的永远是最后一位。

所以这题最自然的做法，就是**从数组末尾往前扫**。

这一步其实就是把纸笔加法直接翻译成代码。

### 什么时候可以立刻结束

从后往前处理时，最关键的是看当前这一位是不是 `9`。

#### 如果当前位不是 9

比如最后一位是 `3`，那它加一之后直接变成 `4`，不会产生进位。

例如：

```text
[1,2,3] -> [1,2,4]
```

这种情况下，前面的数字完全不用动，处理到这里就可以直接返回结果。

这也是为什么很多测试数据其实一轮就结束了。

#### 如果当前位是 9

比如最后一位是 `9`，那它加一之后会变成 `10`。

但数组里每一位只能存一个数字，所以这一位要变成 `0`，同时把进位交给前一位继续处理。

例如：

```text
[1,2,9] -> [1,3,0]
```

这里最后一位从 `9` 变成 `0`，前一位 `2` 因为进位变成 `3`。

所以一旦遇到 `9`，当前位先置为 `0`，然后继续往前看。

### 为什么最坏情况是“全是 9”

这题真正需要单独处理的特殊情况，是数组所有位都是 `9`。

比如：

```text
[9]     -> [1,0]
[9,9]   -> [1,0,0]
[9,9,9] -> [1,0,0,0]
```

原因很直接。

如果每一位都是 `9`，那从后往前加的时候，每一位都会产生进位，于是原数组最后会全部变成 `0`。  
但最高位还会再多出一个新的进位 `1`，这时候原数组长度已经不够了，只能新开一个长度加一的数组。

这一点非常关键：  
**只有“全是 9”时，答案的位数才会增加。**

除此之外，原数组长度都够用。

### 用几个例子走一遍

#### 例 1：`[1,2,3]`

从后往前看，最后一位是 `3`，不是 `9`，直接加一：

```text
[1,2,4]
```

结束。

#### 例 2：`[4,3,2,1]`

最后一位是 `1`，不是 `9`，直接加一：

```text
[4,3,2,2]
```

结束。

#### 例 3：`[1,9,9]`

最后一位是 `9`，变成 `0`，进位继续往前。  
倒数第二位也是 `9`，也变成 `0`。  
再往前看，第一位是 `1`，不是 `9`，加一变成 `2`。

所以结果是：

```text
[2,0,0]
```

#### 例 4：`[9,9,9]`

最后一位 `9 -> 0`  
倒数第二位 `9 -> 0`  
第一位 `9 -> 0`

扫完整个数组之后，发现还在进位，说明原数全是 `9`。  
这时候只能新建数组：

```text
[1,0,0,0]
```

### 代码为什么能写得这么短

因为这题其实只有两种情况：

- 某一位不是 `9`，那就加一后直接返回
- 一路都是 `9`，那就一路置零，最后新开数组

也就是说，循环里一旦遇到“能停下来的地方”，就立刻结束。  
只有在最极端的情况下，才会完整扫完整个数组。

代码如下：

```java
public int[] plusOne(int[] digits) {
    int n = digits.length;

    for (int i = n - 1; i >= 0; i--) {
        if (digits[i] < 9) {
            digits[i]++;
            return digits;
        }

        digits[i] = 0;
    }

    int[] newNumber = new int[n + 1];
    newNumber[0] = 1;
    return newNumber;
}
```

这段代码短，是因为它非常贴近加法本身的规则，没有多余分支。

### 复杂度分析

- 时间复杂度：$O(n)$

最坏情况下需要从后往前把整个数组扫一遍，比如 `[9,9,9,9]` 这种情况。  
如果末尾不是 `9`，那其实一轮就结束了，但整体复杂度还是记作 $O(n)$。

- 空间复杂度：$O(1)$

大多数情况下都是直接在原数组上改。  
只有全是 `9` 的时候，才需要新建一个长度为 `n+1` 的数组。通常这类题的额外空间仍然记作 $O(1)$。

### 这题真正值得记住的地方

这题最值得记住的，不是“加一怎么写”，而是这种处理方式：

**当题目是在数组里模拟数值运算时，通常要从最低位开始，按真实运算规则逐位处理。**

这里的规则就是：

- 不是 `9`，直接加一结束
- 是 `9`，当前位置变成 `0`，继续向前传递进位

这种“从后往前处理进位”的思路，在加法类题目里非常常见。  
这道题本身很基础，但它其实是在练你把纸笔运算翻译成代码的能力。

### 完整代码

```java
import java.util.Arrays;

public class Main {

    public static int[] plusOne(int[] digits) {
        int n = digits.length;

        for (int i = n - 1; i >= 0; i--) {
            if (digits[i] < 9) {
                digits[i]++;
                return digits;
            }

            digits[i] = 0;
        }

        int[] newNumber = new int[n + 1];
        newNumber[0] = 1;
        return newNumber;
    }

    public static void main(String[] args) {
        int[] digits1 = {1, 2, 3};
        System.out.println("示例 1 输入: " + Arrays.toString(digits1));
        int[] res1 = plusOne(digits1);
        System.out.println("示例 1 输出: " + Arrays.toString(res1));
        System.out.println("-----------------------------");

        int[] digits2 = {4, 3, 2, 1};
        System.out.println("示例 2 输入: " + Arrays.toString(digits2));
        int[] res2 = plusOne(digits2);
        System.out.println("示例 2 输出: " + Arrays.toString(res2));
        System.out.println("-----------------------------");

        int[] digits3 = {9};
        System.out.println("示例 3 输入: " + Arrays.toString(digits3));
        int[] res3 = plusOne(digits3);
        System.out.println("示例 3 输出: " + Arrays.toString(res3));
        System.out.println("-----------------------------");

        int[] digits4 = {1, 9, 9};
        System.out.println("额外示例 输入: " + Arrays.toString(digits4));
        int[] res4 = plusOne(digits4);
        System.out.println("额外示例 输出: " + Arrays.toString(res4));
    }
}
```
