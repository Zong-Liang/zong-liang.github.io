## 题目

**描述：**

给你一个非负整数 `x`，计算并返回 `x` 的 **算术平方根**。

由于返回类型是整数，结果只保留 **整数部分**，小数部分将被 **舍去**。

**注意：** 不允许使用任何内置指数函数和算符，例如 `pow(x, 0.5)` 或者 `x ** 0.5`。

**示例 1:**

- **输入:** `x = 4`
- **输出:** 2

**示例 2:**

- **输入:** `x = 8`
- **输出:** 2
- **解释:** 8 的算术平方根是 2.82842...，由于返回类型是整数，小数部分将被舍去。

**提示:**

- `0 <= x <= 2^31 - 1`

## 题解

### 这题不是在求精确平方根

先把题目要求看清楚。

它不是让你返回真正的平方根，而是让你返回平方根的**整数部分**。也就是说：

- `x = 4`，答案是 `2`
- `x = 8`，平方根大约是 `2.828...`，但答案仍然是 `2`

所以这题真正要找的，不是某个精确小数，而是这样一个整数 `k`：

```text
k * k <= x
(k + 1) * (k + 1) > x
```

换句话说，我们要找的是：

**最后一个满足“平方不超过 x”的整数。**

这个转化一旦想清楚，二分思路就会很自然。

### 为什么可以用二分查找

因为平方这个操作，在非负整数范围内是单调的。

也就是说，随着 `k` 变大，`k * k` 也一定越来越大。  
于是就会出现一个很明显的分界点：

- 分界点左边，`k * k <= x`
- 分界点右边，`k * k > x`

像这种“前一段满足条件，后一段不满足条件”的题型，本质上就是在一个单调区间里找边界，非常适合用二分查找。

所以这题可以理解成：

**在 `[0, x]` 这个范围里，找最后一个满足 `mid * mid <= x` 的数。**

### 二分时到底在判断什么

二分每次会取一个中间值 `mid`，然后看 `mid * mid` 和 `x` 的关系。

#### 如果 `mid * mid <= x`

说明 `mid` 还不大，它可能就是答案。  
但也有可能答案在更右边，比如：

```text
x = 8
mid = 2
2 * 2 = 4 <= 8
```

这时候 `2` 是合法答案，但还得继续往右找，看有没有更大的数也满足条件。

所以遇到这种情况时：

- 先把 `mid` 记下来
- 再去右半边继续搜索

#### 如果 `mid * mid > x`

说明 `mid` 太大了，不可能是答案。  
而且它右边的数只会更大，所以也都不用看了，直接去左边找。

整个二分过程，其实就是不断逼近这个“最后一个合法值”。

### 为什么要额外注意溢出

这题有一个非常典型的坑：  
`x` 最大能到 `2^31 - 1`，这时候如果你直接用 `int` 计算 `mid * mid`，是可能溢出的。

比如某些较大的 `mid`，平方以后已经超过 `int` 范围，结果就会变成错误的负数或者其他异常值，导致判断失真。

所以这里一般有两种处理方式：

- 用 `long` 来做乘法
- 或者不用乘法，改成判断 `mid <= x / mid`

这两种都可以。  
你原来的写法是第一种，用 `long`，很直观，也好理解。

### 用例子走一遍更容易明白

以 `x = 8` 为例。

我们要找的是最后一个满足：

```text
k * k <= 8
```

的整数。

显然：

- `2 * 2 = 4 <= 8`
- `3 * 3 = 9 > 8`

所以答案是 `2`。

如果用二分来找，大概会经历这样的判断：

- 先猜一个中间值
- 如果平方太小，说明还能往右走
- 如果平方太大，说明得往左收缩

最后停下来的位置，就是那个“最接近但不超过平方根”的整数。

你会发现，这个过程和“找某个精确值”不一样，它更像是在找一个边界。

### 代码

```java
public int mySqrt(int x) {
    int left = 0;
    int right = x;
    int ans = -1;

    while (left <= right) {
        int mid = left + (right - left) / 2;

        if ((long) mid * mid <= x) {
            ans = mid;
            left = mid + 1;
        } else {
            right = mid - 1;
        }
    }

    return ans;
}
```

这段代码里最关键的其实就两点：

- `mid * mid <= x` 时，不是直接返回，而是继续往右找
- 用 `ans` 记录当前最后一个合法值

因为这题找的不是“刚好等于平方根”的数，而是“平方不超过 `x` 的最大整数”。

### 为什么最后返回的是 `ans`

因为很多时候，平方根不是整数。

比如 `x = 8`，你永远不会遇到某个 `mid` 满足：

```text
mid * mid == 8
```

但题目仍然要求你返回 `2`。  
所以二分过程中必须一直记录“当前合法的最好答案”。

每当发现某个 `mid` 满足 `mid * mid <= x`，就说明它至少是一个候选答案。  
之后再试着往右走，看看能不能找到更大的。

循环结束时，`ans` 存的就是那个最大的合法值，也就是最终答案。

### 牛顿迭代法为什么也能做

这题除了二分，其实还有一个更偏数学的方法：**牛顿迭代**。

它本质上是在求方程：

```text
k^2 - x = 0
```

的正根。

推导之后会得到一个迭代公式：

```text
k = (k + x / k) / 2
```

从一个初始值开始，不断更新，最终会非常快地逼近平方根。

代码也不长：

```java
public int mySqrtNewton(int x) {
    if (x == 0) return 0;

    long k = x;
    while (k * k > x) {
        k = (k + x / k) / 2;
    }

    return (int) k;
}
```

这个方法收敛很快，也很经典。  
不过如果是面试或者刷题场景，二分通常更容易讲清楚，也更稳。

### 复杂度分析

二分查找：

- 时间复杂度：$O(\log x)$
- 空间复杂度：$O(1)$

因为每次都会把搜索区间缩小一半，所以时间复杂度是对数级。

牛顿迭代法：

- 时间复杂度：通常也可以看作 $O(\log x)$
- 空间复杂度：$O(1)$

虽然牛顿迭代在实践里往往更快，但这题里二分已经完全够用，而且更容易直接从题意推出。

### 这题真正值得记住的地方

这题最值得记住的，不只是“平方根可以二分”，而是这种判断方式：

**如果题目要找的是某个整数边界，而这个边界满足明显的单调性，那么就可以考虑把问题改写成“找最后一个满足条件的数”或“找第一个不满足条件的数”。**

这题里，那个条件就是：

```text
mid * mid <= x
```

一旦把题目理解成“找最后一个满足条件的整数”，整个二分模板就很顺了。

这类题以后还会反复遇到，比单独记代码更重要。

### 完整代码

```java
public class Main {

    public static int mySqrt(int x) {
        int left = 0;
        int right = x;
        int ans = -1;

        while (left <= right) {
            int mid = left + (right - left) / 2;

            if ((long) mid * mid <= x) {
                ans = mid;
                left = mid + 1;
            } else {
                right = mid - 1;
            }
        }

        return ans;
    }

    public static int mySqrtNewton(int x) {
        if (x == 0) {
            return 0;
        }

        long k = x;
        while (k * k > x) {
            k = (k + x / k) / 2;
        }

        return (int) k;
    }

    public static void main(String[] args) {
        int x1 = 4;
        System.out.println("示例 1 输入: x = " + x1);
        System.out.println("示例 1 输出: " + mySqrt(x1));
        System.out.println("-----------------------------");

        int x2 = 8;
        System.out.println("示例 2 输入: x = " + x2);
        System.out.println("示例 2 输出: " + mySqrt(x2));
        System.out.println("-----------------------------");

        int x3 = 0;
        System.out.println("示例 3 输入: x = " + x3);
        System.out.println("示例 3 输出: " + mySqrt(x3));
        System.out.println("-----------------------------");

        int x4 = 1;
        System.out.println("示例 4 输入: x = " + x4);
        System.out.println("示例 4 输出: " + mySqrt(x4));
        System.out.println("-----------------------------");

        int x5 = 2147395600;
        System.out.println("示例 5 输入: x = " + x5);
        System.out.println("示例 5 输出: " + mySqrt(x5));
    }
}
```
