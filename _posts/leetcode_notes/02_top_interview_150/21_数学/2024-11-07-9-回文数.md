## 题目

**描述：**

给你一个整数 `x`，如果 `x` 是一个回文整数，返回 `true`；否则，返回 `false`。

**回文数** 是指正序（从左向右）和倒序（从右向左）读都是一样的整数。

- 例如，121 是回文，而 123 不是。

**示例 1:**

- **输入:** `x = 121`
- **输出:** `true`

**示例 2:**

- **输入:** `x = -121`
- **输出:** `false`
- **解释:** 从左向右读, 为 -121。 从右向左读, 为 121-。因此它不是一个回文数。

**示例 3:**

- **输入:** `x = 10`
- **输出:** `false`
- **解释:** 从右向左读, 为 01。因此它不是一个回文数。

**提示:**

- `-2^31 <= x <= 2^31 - 1`

**进阶:** 你能不将整数转为字符串来解决这个问题吗？

## 题解
这题最常见的坑是直接把整数整体反转。虽然可做，但有溢出风险，而且做了不必要的工作。更稳的办法是只反转后半段数字。

先处理两类必定不是回文的情况：负数一定不行，末位是 `0` 但数本身不是 `0` 也不行。然后循环里每次把 `x` 的末位“搬”到 `revertedNumber`，直到前半段不再长于后半段。

循环结束后分两种比较：偶数位时 `x == revertedNumber`；奇数位时中间那位可以忽略，所以比较 `x == revertedNumber / 10`。这样就把回文判断完整覆盖了。

代码如下：

```java
public class Main {

    public static boolean isPalindromeStr(int x) {
        String s = String.valueOf(x);
        int left = 0, right = s.length() - 1;

        while (left < right) {
            if (s.charAt(left) != s.charAt(right)) {
                return false;
            }
            left++;
            right--;
        }

        return true;
    }

    public static boolean isPalindrome(int x) {
        if (x < 0 || (x % 10 == 0 && x != 0)) {
            return false;
        }

        int revertedNumber = 0;

        while (x > revertedNumber) {
            revertedNumber = revertedNumber * 10 + x % 10;
            x /= 10;
        }

        return x == revertedNumber || x == revertedNumber / 10;
    }

    public static void main(String[] args) {
        int x1 = 121;
        System.out.println("示例 1 输入: " + x1);
        System.out.println("示例 1 输出: " + isPalindrome(x1));
        System.out.println("-----------------------------");

        int x2 = -121;
        System.out.println("示例 2 输入: " + x2);
        System.out.println("示例 2 输出: " + isPalindrome(x2));
        System.out.println("-----------------------------");

        int x3 = 10;
        System.out.println("示例 3 输入: " + x3);
        System.out.println("示例 3 输出: " + isPalindrome(x3));
    }
}
```

复杂度分析：按数字位数记，时间复杂度 `O(log10 x)`，空间复杂度 `O(1)`。

数学题里这种“只做一半”的处理很常用，既安全又能减少不必要的计算。
