## 题目

**描述：**

给你一个整数 `x`，如果 `x` 是一个回文整数，返回 `true`；否则，返回 `false`。

**回文数** 是指正序（从左向右）和倒序（从右向左）读都是一样的整数。

- 例如，121 是回文，而 123 不是。

**示例 1:**

- **输入:** `x = 121`
- **输出:** `true`

**示例 2:**

- **输入:** `x = -121`
- **输出:** `false`
- **解释:** 从左向右读, 为 -121。 从右向左读, 为 121-。因此它不是一个回文数。

**示例 3:**

- **输入:** `x = 10`
- **输出:** `false`
- **解释:** 从右向左读, 为 01。因此它不是一个回文数。

**提示:**

- `-2^31 <= x <= 2^31 - 1`

**进阶:** 你能不将整数转为字符串来解决这个问题吗？

## 题解

### 先看这题在判断什么

回文数的定义其实很直接：  
一个数如果从左往右读和从右往左读都一样，那它就是回文数。

比如：

- `121` 是回文数
- `123` 不是回文数
- `-121` 也不是，因为倒过来会变成 `121-`

题目本身不难，真正需要注意的是后面的进阶要求：  
**不要把整数转成字符串。**

所以这题可以分成两个层次来看：

- 先用最直观的方法把思路想明白
- 再理解不转字符串时，为什么只反转一半就够了

### 最直观的做法就是转成字符串

如果不考虑进阶要求，这题最顺手的写法就是把整数转成字符串，然后判断这个字符串是不是回文串。

判断字符串回文也很经典：  
用两个指针，一个从左边往右走，一个从右边往左走，只要有一对字符不相等，就不是回文。

代码很短：

```java
public boolean isPalindrome(int x) {
    String s = String.valueOf(x);
    int left = 0, right = s.length() - 1;

    while (left < right) {
        if (s.charAt(left) != s.charAt(right)) {
            return false;
        }
        left++;
        right--;
    }

    return true;
}
```

这个方法没有任何理解门槛，写起来也快。  
但题目既然专门问了“能不能不转字符串”，那显然更想考的是数学做法。

### 真正值得掌握的是数学解法

不转字符串时，我们还是想做同一件事：  
比较这个数的前半部分和后半部分是否对称。

一个很自然的念头是：  
那我把整个数字反转一下，再和原数比较不就行了？

思路本身没问题，但这样做有两个隐患：

- 代码没有很好体现“只需要比较对称性”这件事
- 反转整个整数时，可能遇到溢出问题

所以更合适的做法是：

**只反转后半部分数字。**

这样既够用，也更稳。

### 先排除两类不可能的情况

在正式反转之前，有两种情况可以直接返回 `false`。

#### 负数一定不是回文数

比如 `-121`，从左往右是 `-121`，从右往左是 `121-`，显然不一样。

#### 末尾是 0，但这个数又不是 0

比如 `10`。  
如果一个数最后一位是 `0`，那它如果是回文数，第一位也必须是 `0`。但整数的开头不可能写前导零，所以这种情况不成立。

只有 `0` 自己是例外，因为它本身就是回文数。

所以这一步可以先写成：

```java
if (x < 0 || (x % 10 == 0 && x != 0)) {
    return false;
}
```

这一步不是细节，而是整个思路里很重要的一层剪枝。

### 为什么只反转一半就够了

这题最关键的地方就在这里。

假设 `x = 1221`。

我们每次取出末尾数字，拼到 `revertedNumber` 后面：

- 第一次取出 `1`，`revertedNumber = 1`
- 第二次取出 `2`，`revertedNumber = 12`

这时原来的 `x` 也被去掉了两位，变成 `12`。

可以看到，前半部分和后半部分已经正好分开了：

```text
x = 12
revertedNumber = 12
```

如果它们相等，说明原数是回文数。

再看奇数位的情况，比如 `12321`。

同样反转后半部分：

- 第一次取出 `1`，`revertedNumber = 1`
- 第二次取出 `2`，`revertedNumber = 12`
- 第三次取出 `3`，`revertedNumber = 123`

这时原来的 `x` 变成了 `12`。

现在会发现：

```text
x = 12
revertedNumber = 123
```

看起来不相等，但其实多出来的那个 `3` 正好是中间位。  
而回文判断里，中间这一位是不用参与比较的，所以把它去掉就行：

```text
revertedNumber / 10 = 12
```

于是只要判断：

- `x == revertedNumber`
- 或者 `x == revertedNumber / 10`

就够了。

### 循环为什么写成 `x > revertedNumber`

这句也值得单独说一下。

我们每次都在做两件事：

- 从 `x` 末尾拿出一位
- 拼到 `revertedNumber` 后面

随着循环进行：

- `x` 会越来越短
- `revertedNumber` 会越来越长

当 `x <= revertedNumber` 时，就说明后半部分已经反转完了，甚至可能已经多反转了一位。  
这正好对应前面说的偶数位和奇数位两种情况。

所以循环条件写成：

```java
while (x > revertedNumber)
```

刚好合适。

### 代码

```java
public boolean isPalindrome(int x) {
    if (x < 0 || (x % 10 == 0 && x != 0)) {
        return false;
    }

    int revertedNumber = 0;

    while (x > revertedNumber) {
        revertedNumber = revertedNumber * 10 + x % 10;
        x /= 10;
    }

    return x == revertedNumber || x == revertedNumber / 10;
}
```

整段代码虽然不长，但背后其实对应了三个关键点：

- 先排除一定不可能的情况
- 只反转后半部分
- 奇数位时去掉中间那个数字再比较

这三个点一旦顺了，这题就不会觉得绕。

### 复杂度分析

- 时间复杂度：$O(\log_{10} x)$

每次循环都会去掉一位数字，循环次数和数字位数成正比。严格来说这里只处理了一半位数，但复杂度量级仍然记作 $O(\log_{10} x)$。

- 空间复杂度：$O(1)$

只用了几个整型变量，没有额外开辟与输入规模相关的空间。

### 这题真正值得记住的地方

这题最值得记住的，不只是“回文数怎么写”，而是这种处理方式：

**当一个问题只要求比较两边是否对称时，很多时候没必要把整个对象都完整反转，只处理一半就够了。**

这里“只反转一半”有两个好处：

- 避免了反转整个整数可能带来的溢出问题
- 也更贴近题目本身要做的事，本质上只是比较左右两边是否一致

这个思路在数字题里很常见，比单独记住代码更重要。

### 完整代码

```java
public class Main {

    public static boolean isPalindromeStr(int x) {
        String s = String.valueOf(x);
        int left = 0, right = s.length() - 1;

        while (left < right) {
            if (s.charAt(left) != s.charAt(right)) {
                return false;
            }
            left++;
            right--;
        }

        return true;
    }

    public static boolean isPalindrome(int x) {
        if (x < 0 || (x % 10 == 0 && x != 0)) {
            return false;
        }

        int revertedNumber = 0;

        while (x > revertedNumber) {
            revertedNumber = revertedNumber * 10 + x % 10;
            x /= 10;
        }

        return x == revertedNumber || x == revertedNumber / 10;
    }

    public static void main(String[] args) {
        int x1 = 121;
        System.out.println("示例 1 输入: " + x1);
        System.out.println("示例 1 输出: " + isPalindrome(x1));
        System.out.println("-----------------------------");

        int x2 = -121;
        System.out.println("示例 2 输入: " + x2);
        System.out.println("示例 2 输出: " + isPalindrome(x2));
        System.out.println("-----------------------------");

        int x3 = 10;
        System.out.println("示例 3 输入: " + x3);
        System.out.println("示例 3 输出: " + isPalindrome(x3));
    }
}
```
