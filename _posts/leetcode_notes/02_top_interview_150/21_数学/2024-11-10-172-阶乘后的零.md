## 题目

**描述：**

给定一个整数 `n`，返回 `n!` 结果中尾随零的数量。

提示：`n! = n * (n - 1) * (n - 2) * ... * 3 * 2 * 1`

**示例 1:**

- **输入:** `n = 3`
- **输出:** 0
- **解释:** `3! = 6`，不含尾随 0。

**示例 2:**

- **输入:** `n = 5`
- **输出:** 1
- **解释:** `5! = 120`，有一个尾随 0。

**示例 3:**

- **输入:** `n = 0`
- **输出:** 0

**提示:**

- `0 <= n <= 10^4`

**进阶:** 你可以设计并实现对数时间复杂度的算法来解决此问题吗？

## 题解

### 先别真的去算阶乘

这题如果按字面意思来做，很容易先想到两步：

1. 先把 `n!` 算出来
2. 再看结果末尾有多少个 `0`

但这个方向其实一开始就偏了。

因为这题真正关心的，不是 `n!` 这个数本身，而是它**末尾为什么会出现 0**。  
只要把这个问题想明白，就会发现根本没必要真的把阶乘算出来。

### 末尾为什么会出现 0

一个数末尾有一个 `0`，本质上说明它能被 `10` 整除。  
而：

```text
10 = 2 × 5
```

所以，`n!` 里每出现一组 `2 × 5`，末尾就会多一个 `0`。

问题就变成了：

**`n!` 的质因数分解里，一共能凑出多少个 2 和 5。**

接下来再往前想一步，就会发现真正关键的不是 `2`，而是 `5`。

因为在阶乘里，`2` 的数量远远多于 `5`。  
随便看几个数就知道：

- 每隔 2 个数就会出现一个偶数，能提供因子 `2`
- 但每隔 5 个数才会出现一个能提供因子 `5` 的数

所以决定尾随零个数的，不是 `2` 有多少，而是 **`5` 有多少**。

也就是说，这题最后其实就是在问：

**从 1 到 n 里，一共能贡献多少个因子 5。**

### 最直接的想法：一个一个数去数 5

如果先按最朴素的思路写，可以遍历 `1` 到 `n` 中所有 `5` 的倍数，统计每个数里含有多少个 `5`。

比如：

- `5` 贡献 1 个 `5`
- `10` 贡献 1 个 `5`
- `15` 贡献 1 个 `5`
- `25` 贡献 2 个 `5`
- `125` 贡献 3 个 `5`

像这样把它们全加起来，也能得到答案。

代码可以写成：

```java
public int trailingZeroesNaive(int n) {
    int count = 0;

    for (int i = 5; i <= n; i += 5) {
        int current = i;
        while (current % 5 == 0) {
            count++;
            current /= 5;
        }
    }

    return count;
}
```

这个方法思路没有问题，而且很适合拿来帮助理解。  
但题目后面专门问了进阶：**能不能做到对数时间复杂度。**

所以这题真正要掌握的，还是下面这个数学统计方法。

### 真正关键的是“5 被重复算了几次”

我们换个角度，不去看每个数本身，而是直接统计：

- 有多少个数至少贡献了 1 个 `5`
- 有多少个数额外再贡献了 1 个 `5`
- 有多少个数还会再额外贡献 1 个 `5`

这样就顺了。

#### 第 1 层：统计 5 的倍数

从 `1` 到 `n` 里，每隔 5 个数就有一个能提供至少一个因子 `5`。

所以第一层贡献是：

```text
n / 5
```

比如 `n = 26` 时：

```text
5, 10, 15, 20, 25
```

一共 5 个数，说明至少有 5 个因子 `5`。

#### 第 2 层：统计 25 的倍数

但这里有个容易漏掉的点：  
像 `25` 这样的数，不是只贡献一个 `5`，而是贡献两个。

因为：

```text
25 = 5 × 5
```

所以除了在上一层已经算过一次之外，所有 `25` 的倍数还得再额外贡献一次。

这一层的数量就是：

```text
n / 25
```

#### 第 3 层：统计 125 的倍数

同理，`125` 还能再多贡献一个 `5``：

```text
125 = 5 × 5 × 5
```

所以所有 `125` 的倍数还要再补一层：

```text
n / 125
```

后面也是一样的道理。

所以总答案就是：

```text
n / 5 + n / 25 + n / 125 + n / 625 + ...
```

一直加到某一项变成 0 为止。

### 用例子看就很清楚

以 `n = 26` 为例。

先看第一层：

```text
26 / 5 = 5
```

对应的是：

```text
5, 10, 15, 20, 25
```

这 5 个数至少各自贡献一个 `5`。

再看第二层：

```text
26 / 25 = 1
```

对应的是 `25`。  
因为 `25` 其实含有两个 `5`，上一轮只算了一个，这里要把多出来的那个补上。

再往下：

```text
26 / 125 = 0
```

结束。

所以总数是：

```text
5 + 1 = 6
```

也就是说，`26!` 末尾有 `6` 个 `0`。

### 为什么代码可以写得这么短

因为上面的过程完全可以压成一个循环。

每次都把 `n` 除以 `5`，把结果加到答案里：

- 第一次加的是 `n / 5`
- 第二次加的是 `n / 25`
- 第三次加的是 `n / 125`

……

直到 `n` 变成 `0` 为止。

代码如下：

```java
public int trailingZeroes(int n) {
    int count = 0;

    while (n > 0) {
        n /= 5;
        count += n;
    }

    return count;
}
```

这段代码之所以短，不是因为技巧多，而是因为它直接抓住了题目的本质：  
**统计 5 的个数，而不是计算阶乘。**

### 复杂度分析

- 时间复杂度：$O(\log_5 n)$

因为每次循环都会把 `n` 缩小为原来的五分之一，所以循环次数是对数级的。

- 空间复杂度：$O(1)$

只用了常数个变量。

### 这题真正值得记住的地方

这题最值得记住的，不是公式本身，而是这种转化思路：

**不要被题目的表面表达带着走，要先想它真正统计的对象是什么。**

这里题目表面上问的是“阶乘后的零”，但本质上统计的是：

- 一个尾随零来自一个 `10`
- 一个 `10` 来自一组 `2 × 5`
- `2` 足够多，所以最后只需要数 `5`

一旦把问题拆到这里，原本看起来像大数运算的问题，就会变成一道很干净的数学计数题。

这种“先换问题，再做题”的能力，比单独记住这道题更重要。

### 完整代码

```java
public class Main {

    public static int trailingZeroesNaive(int n) {
        int count = 0;

        for (int i = 5; i <= n; i += 5) {
            int current = i;
            while (current % 5 == 0) {
                count++;
                current /= 5;
            }
        }

        return count;
    }

    public static int trailingZeroes(int n) {
        int count = 0;

        while (n > 0) {
            n /= 5;
            count += n;
        }

        return count;
    }

    public static void main(String[] args) {
        int n1 = 3;
        System.out.println("示例 1 输入: n = " + n1);
        System.out.println("示例 1 输出: " + trailingZeroes(n1));
        System.out.println("-----------------------------");

        int n2 = 5;
        System.out.println("示例 2 输入: n = " + n2);
        System.out.println("示例 2 输出: " + trailingZeroes(n2));
        System.out.println("-----------------------------");

        int n3 = 0;
        System.out.println("示例 3 输入: n = " + n3);
        System.out.println("示例 3 输出: " + trailingZeroes(n3));
        System.out.println("-----------------------------");

        int n4 = 125;
        System.out.println("额外测试 输入: n = " + n4);
        System.out.println("Naive 解法: " + trailingZeroesNaive(n4));
        System.out.println("优化 解法: " + trailingZeroes(n4));
    }
}
```
