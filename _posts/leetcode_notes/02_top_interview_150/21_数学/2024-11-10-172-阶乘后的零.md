## 题目

**描述：**

给定一个整数 `n`，返回 `n!` 结果中尾随零的数量。

提示：`n! = n * (n - 1) * (n - 2) * ... * 3 * 2 * 1`

**示例 1:**

- **输入:** `n = 3`
- **输出:** 0
- **解释:** `3! = 6`，不含尾随 0。

**示例 2:**

- **输入:** `n = 5`
- **输出:** 1
- **解释:** `5! = 120`，有一个尾随 0。

**示例 3:**

- **输入:** `n = 0`
- **输出:** 0

**提示:**

- `0 <= n <= 10^4`

**进阶:** 你可以设计并实现对数时间复杂度的算法来解决此问题吗？

## 题解
末尾有多少个 `0`，本质是在问乘积里有多少个因子 `10`。而 `10 = 2 * 5`，在阶乘里 `2` 的数量远多于 `5`，所以问题就变成“统计 `1..n` 里一共能提供多少个 `5`”。

第一层是 `n/5`，表示有多少个数至少贡献一个 `5`；第二层是 `n/25`，因为 `25`、`50` 这类数会再多贡献一个；再往后是 `n/125`、`n/625`……一直到商变成 `0`。

因此循环写成不断 `n /= 5` 并累计结果即可，不需要真的去算阶乘，也不会有大数溢出问题。

代码如下：

```java
public class Main {

    public static int trailingZeroesNaive(int n) {
        int count = 0;

        for (int i = 5; i <= n; i += 5) {
            int current = i;
            while (current % 5 == 0) {
                count++;
                current /= 5;
            }
        }

        return count;
    }

    public static int trailingZeroes(int n) {
        int count = 0;

        while (n > 0) {
            n /= 5;
            count += n;
        }

        return count;
    }

    public static void main(String[] args) {
        int n1 = 3;
        System.out.println("示例 1 输入: n = " + n1);
        System.out.println("示例 1 输出: " + trailingZeroes(n1));
        System.out.println("-----------------------------");

        int n2 = 5;
        System.out.println("示例 2 输入: n = " + n2);
        System.out.println("示例 2 输出: " + trailingZeroes(n2));
        System.out.println("-----------------------------");

        int n3 = 0;
        System.out.println("示例 3 输入: n = " + n3);
        System.out.println("示例 3 输出: " + trailingZeroes(n3));
        System.out.println("-----------------------------");

        int n4 = 125;
        System.out.println("额外测试 输入: n = " + n4);
        System.out.println("Naive 解法: " + trailingZeroesNaive(n4));
        System.out.println("优化 解法: " + trailingZeroes(n4));
    }
}
```

复杂度分析：时间复杂度 `O(log_5 n)`，空间复杂度 `O(1)`。

这题很典型：先把现象翻译成因子计数，复杂度会从“不可算”直接降到很轻。
