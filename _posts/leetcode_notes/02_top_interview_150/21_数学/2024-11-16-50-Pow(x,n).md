## 题目

**描述：**

实现 `pow(x, n)`，即计算 `x` 的整数 `n` 次幂函数（即，`x^n`）。

**示例 1:**

- **输入:** `x = 2.00000`, `n = 10`
- **输出:** `1024.00000`

**示例 2:**

- **输入:** `x = 2.10000`, `n = 3`
- **输出:** `9.26100`

**示例 3:**

- **输入:** `x = 2.00000`, `n = -2`
- **输出:** `0.25000`
- **解释:** `2^-2 = 1/2^2 = 1/4 = 0.25`

**提示:**

- `-100.0 < x < 100.0`
- `-2^31 <= n <= 2^31 - 1`
- `n` 是一个整数
- 要么 `x` 不为零，要么 `n > 0`。
- `-10^4 <= x^n <= 10^4`

## 题解

### 这题如果老老实实乘，会慢得离谱

题目要实现的是：

```text
x^n
```

最直接的想法当然是把 `x` 连乘 `n` 次。  
比如 `2^10`，那就乘 10 次；`2^100`，那就乘 100 次。

但问题在于，`n` 的范围非常大，最大能到 `2^31 - 1`。  
如果还按“乘一次算一步”的思路来做，时间复杂度就是 $O(n)$，这种写法在数据一大时基本扛不住。

所以这题真正要解决的，不是“怎么算幂”，而是：

**怎么把重复乘法的次数大幅压缩下来。**

### 关键不在一下一下乘，而在“拆指数”

这题最核心的转折点，是别把 `n` 看成“要乘多少次”，而是把它看成“能不能拆”。

比如：

```text
x^10 = x^5 × x^5
```

如果继续拆：

```text
x^5 = x × x^4
x^4 = x^2 × x^2
```

会发现指数并不一定非得一格一格减，而是可以通过“折半”的方式快速往下压。

这就是快速幂的核心：

- 如果 `n` 是偶数，`x^n = (x^(n/2))^2`
- 如果 `n` 是奇数，`x^n = x × (x^((n-1)/2))^2`

也就是说，每处理一步，指数都能缩小一半。  
这和普通做法最大的区别就在这里。

### 为什么它叫“快速幂”

因为普通做法每次只把指数减 1，而快速幂每次直接把指数减半。

比如算 `x^10`：

- 普通做法要连续乘很多次
- 快速幂会把过程变成：

```text
10 -> 5 -> 2 -> 1 -> 0
```

指数下降得非常快，所以整体时间复杂度会从 $O(n)$ 变成 $O(\log n)$。

这就是它真正快的地方。

### 从二进制角度看，会更容易理解

快速幂其实还可以从二进制理解。

比如 `10` 的二进制是：

```text
1010
```

这意味着：

```text
x^10 = x^(8+2) = x^8 × x^2
```

再比如 `13` 的二进制是：

```text
1101
```

也就是：

```text
x^13 = x^(8+4+1) = x^8 × x^4 × x
```

所以我们完全可以一边不断让底数平方：

```text
x -> x^2 -> x^4 -> x^8 -> ...
```

一边看指数当前这一位是不是 `1`：

- 如果是 `1`，就把当前底数乘进答案
- 如果是 `0`，就跳过

这样就能只用很少的乘法，把整个结果算出来。

### 用一个具体例子走一遍

以 `x = 2, n = 10` 为例。

`10` 的二进制是 `1010`，所以答案其实是：

```text
2^10 = 2^8 × 2^2
```

我们在循环里维护两个东西：

- `ans`：当前已经确定要乘进去的结果
- `x`：当前这一轮对应的底数权重，依次变成 `x, x^2, x^4, x^8...`

过程大概是这样：

#### 初始状态

```text
ans = 1
x = 2
n = 10
```

#### 第 1 轮

`10` 是偶数，当前最低位不是 `1`，所以这轮不用把 `x` 乘进答案。

然后底数平方：

```text
x = 4
n = 5
```

#### 第 2 轮

`5` 是奇数，说明当前这一位是 `1`，要把当前的 `x = 4` 乘进答案：

```text
ans = 4
```

再继续：

```text
x = 16
n = 2
```

#### 第 3 轮

`2` 是偶数，这一位不是 `1`，跳过。

继续：

```text
x = 256
n = 1
```

#### 第 4 轮

`1` 是奇数，把当前的 `x = 256` 乘进答案：

```text
ans = 4 × 256 = 1024
```

然后：

```text
x = 65536
n = 0
```

循环结束，答案就是 `1024`。

整个过程中，我们并没有乘 10 次，而是通过不断平方和按位选择，很快就做完了。

### 负指数怎么处理

如果 `n` 是负数，比如：

```text
2^-2
```

那其实不用单独发明新算法，只要先做一个变形：

```text
x^(-n) = 1 / x^n
```

也就是说：

```text
2^-2 = 1 / 2^2
```

所以处理方法很简单：

- 如果 `n < 0`
- 就先把 `x` 变成 `1 / x`
- 再把 `n` 变成正数继续算

这样正指数和负指数就统一了。

### 这里有个最容易漏掉的坑：`Integer.MIN_VALUE`

这题真正的边界坑，不在 `x`，而在 `n`。

因为 `n` 是 `int`，范围是：

```text
[-2^31, 2^31 - 1]
```

如果 `n = -2^31`，也就是 `Integer.MIN_VALUE`，你直接写：

```java
n = -n;
```

会溢出。

原因是 `int` 正数最大只能到 `2^31 - 1`，装不下 `2^31`。  
所以这种情况下，取反之后还是负数，后面逻辑就会出错。

正确做法是先把 `n` 转成 `long`，再处理：

```java
long N = n;
```

这样 `-N` 才是安全的。

这一步不是细节，而是这题代码能不能写对的关键。

### 代码

```java
public double myPow(double x, int n) {
    long N = n;

    if (N < 0) {
        x = 1 / x;
        N = -N;
    }

    double ans = 1.0;

    while (N > 0) {
        if ((N & 1) == 1) {
            ans *= x;
        }

        x *= x;
        N >>= 1;
    }

    return ans;
}
```

这段代码真正要读懂的是三件事：

- `N & 1` 用来判断当前这一位是不是 `1`
- `x *= x` 表示底数权重不断翻倍
- `N >>= 1` 表示指数右移一位，也就是除以 `2`

一旦把它和“二进制拆指数”这件事对应起来，整段代码就很好理解。

### 复杂度分析

- 时间复杂度：$O(\log n)$

每次循环都会把指数缩小一半，所以循环次数和指数的二进制位数有关，是对数级的。

- 空间复杂度：$O(1)$

整个过程只用了几个变量，没有额外开辟和输入规模相关的空间。

### 这题真正值得记住的地方

这题最值得记住的，不只是“快速幂怎么写”，而是这种思路：

**当一个问题表面上是在做很多次重复运算时，先别急着按次数硬做，看看能不能利用数学结构把次数折半。**

这里利用的结构就是：

- 偶数次幂可以对半拆
- 奇数次幂可以先拆出一个，再继续对半处理
- 指数还能用二进制来表达

所以它最终不再是“乘很多次”，而是“按位决定哪些权重要乘进去”。

这类思路在很多题里都会出现，比单独记一份模板更重要。

### 完整代码

```java
public class Main {

    public static double myPow(double x, int n) {
        long N = n;

        if (N < 0) {
            x = 1 / x;
            N = -N;
        }

        double ans = 1.0;

        while (N > 0) {
            if ((N & 1) == 1) {
                ans *= x;
            }

            x *= x;
            N >>= 1;
        }

        return ans;
    }

    public static void main(String[] args) {
        double x1 = 2.00000;
        int n1 = 10;
        System.out.println("示例 1 输入: x = " + x1 + ", n = " + n1);
        System.out.println("示例 1 输出: " + myPow(x1, n1));
        System.out.println("-----------------------------");

        double x2 = 2.10000;
        int n2 = 3;
        System.out.println("示例 2 输入: x = " + x2 + ", n = " + n2);
        System.out.println("示例 2 输出: " + String.format("%.5f", myPow(x2, n2)));
        System.out.println("-----------------------------");

        double x3 = 2.00000;
        int n3 = -2;
        System.out.println("示例 3 输入: x = " + x3 + ", n = " + n3);
        System.out.println("示例 3 输出: " + myPow(x3, n3));
        System.out.println("-----------------------------");

        double x4 = 2.0;
        int n4 = -2147483648;
        System.out.println("边界测试 输入: x = " + x4 + ", n = " + n4);
        System.out.println("边界测试 输出: " + myPow(x4, n4));
    }
}
```
