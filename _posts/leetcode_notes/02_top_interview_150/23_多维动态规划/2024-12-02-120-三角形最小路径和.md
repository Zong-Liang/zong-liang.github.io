## 题目

**描述：**

给定一个三角形 `triangle`，找出自顶向下的最小路径和。

每一步只能移动到下一行中相邻的结点上。**相邻的结点** 在这里指的是 **下标** 与上一层结点下标 **相同** 或者等于 **上一层结点下标 + 1** 的两个结点。也就是说，如果正位于当前行的下标 `i`，那么下一步可以移动到下一行的下标 `i` 或 `i + 1`。

**示例 1:**

- **输入:** `triangle = [[2],[3,4],[6,5,7],[4,1,8,3]]`
- **输出:** 11
- **解释:** 如下面简图所示：

  ```
     2
    3 4
   6 5 7
  4 1 8 3
  ```

  自顶向下的最小路径和为 11 (即, `2 + 3 + 5 + 1 = 11`)。

**示例 2:**

- **输入:** `triangle = [[-10]]`
- **输出:** -10

**提示:**

- `1 <= triangle.length <= 200`
- `triangle[0].length == 1`
- `triangle[i].length == triangle[i - 1].length + 1`
- `-10^4 <= triangle[i][j] <= 10^4`

**进阶:**

- 你可以只使用 O(n) 的额外空间（n 为三角形的总行数）来解决这个问题吗？

## 题解

### 先看这题为什么不是贪心

这题要求的是从顶到底的最小路径和，而且每一步只能走到下一行中“正下方”或者“右下方”的位置。

看到“每一步都选更小的那个”时，很多人会先想到贪心。  
但这个思路其实不稳，因为当前位置下面较小的那个数，不一定能带来全局最优解。

也就是说，这题不是“当前看谁小就走谁”的问题，而是：

**站在某个位置时，往下走的最小总代价到底是多少。**

一旦这么想，这题就很明显是动态规划。

### 关键在于：每个位置只依赖下面两个位置

假设我们现在站在位置 `(i, j)`，它下一步只能走到：

- `(i + 1, j)`
- `(i + 1, j + 1)`

所以如果我们已经知道这两个位置到底部的最小路径和，那么当前位置的答案就很容易写出来：

```text
当前位置的值 + min(正下方的最小路径和, 右下方的最小路径和)
```

这就是整道题的状态转移核心。

也就是说，`(i, j)` 的最优解，只取决于它下面相邻的两个位置，而不需要关心更远的细节。

### 为什么自底向上更顺

这题既可以自顶向下想，也可以自底向上想。

如果自顶向下做，通常会写成：

- 到达当前位置的最小路径和是多少

这种写法当然也能做，但边界会稍微麻烦一点，因为最左边和最右边都只有一个来源。

而如果改成**自底向上**，就会顺很多。

因为站在当前位置时，我们只需要看它下面两个位置里谁更优，直接取最小值即可。  
最底下一行本身也很好处理：它们到底部的最小路径和，就是它们自己。

所以这题从下往上推，会比从上往下推更自然。

### 二维 DP 怎么定义最清楚

我们可以定义：

```text
dp[i][j] 表示从位置 (i, j) 出发，到达底部的最小路径和
```

有了这个定义之后，转移方程就非常直接：

```text
dp[i][j] = triangle[i][j] + min(dp[i + 1][j], dp[i + 1][j + 1])
```

含义也很直白：

- 当前位置的值一定要算上
- 下一步只能去下面两个位置中的一个
- 那当然选总代价更小的那条路

### 初始化为什么从最后一行开始

因为最后一行已经没有下一步可走了。

所以对于最后一行来说：

```text
dp[n - 1][j] = triangle[n - 1][j]
```

它本身就是从这个位置到底部的最小路径和。

有了这一层作为起点，往上推的时候，上一层每个位置都能直接用下面两个已经算好的结果来更新。

这就是标准的自底向上动态规划。

### 用题目的例子走一遍

看示例 1：

```text
[
     [2],
    [3,4],
   [6,5,7],
  [4,1,8,3]
]
```

先看最后一行，它们的最小路径和就是自己：

```text
[4,1,8,3]
```

再往上一层推：

- `6` 下面是 `4` 和 `1`，取较小的是 `1`，所以得到 `6 + 1 = 7`
- `5` 下面是 `1` 和 `8`，取较小的是 `1`，所以得到 `5 + 1 = 6`
- `7` 下面是 `8` 和 `3`，取较小的是 `3`，所以得到 `7 + 3 = 10`

这一层更新后，相当于变成：

```text
[7,6,10]
```

再往上一层：

- `3` 下面是 `7` 和 `6`，取较小的是 `6`，得到 `9`
- `4` 下面是 `6` 和 `10`，取较小的是 `6`，得到 `10`

这一层变成：

```text
[9,10]
```

最后最顶层：

- `2` 下面是 `9` 和 `10`，取较小的是 `9`
- 所以答案是 `2 + 9 = 11`

这就是最终结果。

### 代码

```java
public int minimumTotal(List<List<Integer>> triangle) {
    int n = triangle.size();
    int[][] dp = new int[n][n];

    for (int i = 0; i < n; i++) {
        dp[n - 1][i] = triangle.get(n - 1).get(i);
    }

    for (int i = n - 2; i >= 0; i--) {
        for (int j = 0; j <= i; j++) {
            dp[i][j] = triangle.get(i).get(j) + Math.min(dp[i + 1][j], dp[i + 1][j + 1]);
        }
    }

    return dp[0][0];
}
```

这段代码真正要抓住的就两点：

- `dp[i][j]` 表示“从这里出发到底部”的最小路径和
- 更新当前位置时，只看下一行相邻两个位置

如果这两个点想清楚了，这题就不会绕。

### 为什么可以压成一维数组

这题的进阶要求是只用 `O(n)` 的额外空间。

其实看状态转移就能发现：

```text
dp[i][j]
```

只依赖下一行的：

```text
dp[i + 1][j] 和 dp[i + 1][j + 1]
```

也就是说，计算当前行时，我们根本不需要保留整个二维表，只需要保留“下一行的结果”就够了。

这就说明二维 DP 是可以压缩的。

### 一维 DP 为什么不会覆盖出错

我们用一个一维数组 `dp[j]` 来表示“当前正在处理的下一行结果”。

从底往上更新时：

```java
dp[j] = triangle[i][j] + Math.min(dp[j], dp[j + 1]);
```

这里：

- 旧的 `dp[j]` 表示正下方
- 旧的 `dp[j + 1]` 表示右下方

更新完之后，新的 `dp[j]` 就表示当前这一行位置 `(i, j)` 的最小路径和。

因为我们是一行一行往上推，而每一行内部又是从左到右更新，所以当前计算 `dp[j]` 时，`dp[j]` 和 `dp[j+1]` 都还是上一行留下来的值，逻辑上是安全的。

这就是空间优化能成立的原因。

### 一维写法为什么这么简洁

一维优化版通常会把 `dp` 数组开成 `n + 1` 长度，并初始化为 0。

这样从最后一行开始更新时：

```java
dp[j] = triangle[i][j] + Math.min(dp[j], dp[j + 1]);
```

对于最后一行来说，`dp[j]` 和 `dp[j+1]` 一开始都是 0，于是就相当于直接把最后一行的值写进去，不需要专门额外初始化。

代码会很简洁：

```java
public int minimumTotalOptimized(List<List<Integer>> triangle) {
    int n = triangle.size();
    int[] dp = new int[n + 1];

    for (int i = n - 1; i >= 0; i--) {
        List<Integer> row = triangle.get(i);
        for (int j = 0; j < row.size(); j++) {
            dp[j] = row.get(j) + Math.min(dp[j], dp[j + 1]);
        }
    }

    return dp[0];
}
```

最后 `dp[0]` 就是答案。

### 复杂度分析

二维 DP：

- 时间复杂度：$O(n^2)$
- 空间复杂度：$O(n^2)$

这里的 `n` 是行数。三角形一共有大约 `n(n+1)/2` 个元素，所以整体仍然是平方级。

一维优化版：

- 时间复杂度：$O(n^2)$
- 空间复杂度：$O(n)$

时间不会变，因为所有位置还是都要算一遍；但空间可以从二维压到一维。

### 这题真正值得记住的地方

这题最值得记住的，不只是“自底向上 DP”这个结论，而是这种判断方式：

**如果一个位置的最优解，只依赖下一层很少几个状态，那么通常就可以考虑倒着推。**

这里之所以适合从下往上，是因为：

- 当前答案只依赖下一行两个位置
- 最后一行天然就是边界
- 倒推时不需要再额外处理复杂边界

而进一步观察“只依赖下一行”，又能继续把二维压成一维。

这才是这道题真正有价值的地方。

### 完整代码

```java
import java.util.ArrayList;
import java.util.Arrays;
import java.util.List;

public class Main {

    public static int minimumTotal(List<List<Integer>> triangle) {
        if (triangle == null || triangle.size() == 0) {
            return 0;
        }

        int n = triangle.size();
        int[][] dp = new int[n][n];

        for (int i = 0; i < n; i++) {
            dp[n - 1][i] = triangle.get(n - 1).get(i);
        }

        for (int i = n - 2; i >= 0; i--) {
            for (int j = 0; j <= i; j++) {
                dp[i][j] = triangle.get(i).get(j) + Math.min(dp[i + 1][j], dp[i + 1][j + 1]);
            }
        }

        return dp[0][0];
    }

    public static int minimumTotalOptimized(List<List<Integer>> triangle) {
        if (triangle == null || triangle.size() == 0) {
            return 0;
        }

        int n = triangle.size();
        int[] dp = new int[n + 1];

        for (int i = n - 1; i >= 0; i--) {
            List<Integer> currentRow = triangle.get(i);
            for (int j = 0; j < currentRow.size(); j++) {
                dp[j] = currentRow.get(j) + Math.min(dp[j], dp[j + 1]);
            }
        }

        return dp[0];
    }

    public static void main(String[] args) {
        List<List<Integer>> triangle = new ArrayList<>();
        triangle.add(Arrays.asList(2));
        triangle.add(Arrays.asList(3, 4));
        triangle.add(Arrays.asList(6, 5, 7));
        triangle.add(Arrays.asList(4, 1, 8, 3));

        System.out.println("示例 1 输入: " + triangle);
        System.out.println("标准解法输出: " + minimumTotal(triangle));
        System.out.println("优化解法输出: " + minimumTotalOptimized(triangle));
        System.out.println("-----------------------------");

        List<List<Integer>> triangle2 = new ArrayList<>();
        triangle2.add(Arrays.asList(-10));

        System.out.println("示例 2 输入: " + triangle2);
        System.out.println("标准解法输出: " + minimumTotal(triangle2));
        System.out.println("优化解法输出: " + minimumTotalOptimized(triangle2));
    }
}
```
