## 题目

**描述：**

给定三个字符串 `s1`、`s2`、`s3`，请你帮忙验证 `s3` 是否是由 `s1` 和 `s2` **交错** 组成的。

两个字符串 `s` 和 `t` **交错** 的定义与过程如下，其中每个字符串都会被分割成若干 **非空** 子字符串：

- `s = s1 + s2 + ... + sn`
- `t = t1 + t2 + ... + tm`
- `|n - m| <= 1`
- 交错是 `s1 + t1 + s2 + t2 + s3 + t3 + ...` 或者 `t1 + s1 + t2 + s2 + t3 + s3 + ...`

**注意：** `a + b` 意味着字符串 `a` 和 `b` 连接。

（简而言之，交错意味着 `s1` 和 `s2` 中的字符相对顺序保持不变。）

**示例 1:**

![示例1图片](https://assets.leetcode.com/uploads/2020/09/02/interleave.jpg)

- **输入:** `s1 = "aabcc"`, `s2 = "dbbca"`, `s3 = "aadbbcbcac"`
- **输出:** `true`

**示例 2:**

- **输入:** `s1 = "aabcc"`, `s2 = "dbbca"`, `s3 = "aadbbbaccc"`
- **输出:** `false`

**示例 3:**

- **输入:** `s1 = ""`, `s2 = ""`, `s3 = ""`
- **输出:** `true`

**提示:**

- `0 <= s1.length, s2.length <= 100`
- `0 <= s3.length <= 200`
- `s1`、`s2`、和 `s3` 都由小写英文字母组成

**进阶:** 您能否仅使用 O(s2.length) 额外的内存空间来解决它？

## 题解

### 先把“交错”这件事想清楚

这题看起来像字符串匹配，但它真正考的不是“字符能不能凑出来”，而是：

**`s3` 能不能由 `s1` 和 `s2` 按顺序交替拼出来，同时又不打乱 `s1` 和 `s2` 各自内部的相对顺序。**

这里最关键的是“相对顺序不能乱”。

比如：

- `s1 = "abc"`
- `s2 = "def"`

那么 `"adbcef"` 是可能的，因为：

- `a b c` 的顺序没变
- `d e f` 的顺序也没变

但如果某个结果字符串要求你把 `s1` 里的字符顺序打乱，那就一定不行。

所以这题的难点，不是判断字符有没有出现，而是判断：

**走到 `s3` 的某个位置时，前面这部分能不能正好由 `s1` 的前缀和 `s2` 的前缀拼出来。**

一旦从“前缀”这个角度去想，动态规划就很自然了。

### 为什么长度一定要先判断

这是这题最应该先做的一步。

如果 `s1.length() + s2.length() != s3.length()`，那根本不需要继续想，答案一定是 `false`。

因为交错的本质还是把 `s1` 和 `s2` 的字符全部用完，只是交替方式不同而已。  
长度对不上，说明字符总数都不一致，那不可能拼成同一个字符串。

所以这题一开始先写：

```java
if (s1.length() + s2.length() != s3.length()) {
    return false;
}
```

这不是优化，而是逻辑上必须先排掉的情况。

### 用“前缀能不能拼出来”来定义状态

这题最顺手的状态定义是：

```text
dp[i][j] 表示 s1 的前 i 个字符和 s2 的前 j 个字符，
能不能交错组成 s3 的前 i + j 个字符
```

这个定义有两个好处：

- 它和题目本身贴得很近
- 它天然带有“顺序不能乱”的限制，因为我们始终只在用前缀

比如 `dp[2][3] = true` 的意思就是：

- `s1` 前 2 个字符
- `s2` 前 3 个字符
- 这 5 个字符能按某种交错方式，拼成 `s3` 的前 5 个字符

只要这个定义立住，后面的转移就很好推。

### 为什么只看最后一个字符就够了

这是整道题最核心的一步。

如果我们现在想判断 `dp[i][j]`，也就是：

```text
s1 的前 i 个 + s2 的前 j 个
能不能组成 s3 的前 i+j 个
```

那么 `s3` 的第 `i + j` 个字符，最后只可能来自两个地方之一：

- 来自 `s1` 的第 `i` 个字符
- 或者来自 `s2` 的第 `j` 个字符

不可能有第三种来源。

所以判断 `dp[i][j]` 时，只需要看两种情况：

#### 情况一：最后一个字符来自 `s1`

那就要求：

- `s1[i - 1] == s3[i + j - 1]`
- 并且前面的部分已经能拼出来，也就是 `dp[i - 1][j] == true`

#### 情况二：最后一个字符来自 `s2`

那就要求：

- `s2[j - 1] == s3[i + j - 1]`
- 并且前面的部分已经能拼出来，也就是 `dp[i][j - 1] == true`

只要这两种情况里有一种成立，`dp[i][j]` 就成立。

所以状态转移就是：

```text
dp[i][j] =
( dp[i - 1][j] 且 s1[i - 1] == s3[i + j - 1] )
或
( dp[i][j - 1] 且 s2[j - 1] == s3[i + j - 1] )
```

这一步本质上是在问：

**当前位置这个字符，是从 `s1` 接过来的，还是从 `s2` 接过来的。**

### 边界为什么这样初始化

#### `dp[0][0]`

空串和空串当然可以组成空串，所以：

```text
dp[0][0] = true
```

#### 第一列 `dp[i][0]`

这里表示只用 `s1`，不用 `s2`。

那它是否成立，就取决于：

- `s1` 的前 `i` 个字符
- 能不能和 `s3` 的前 `i` 个字符完全一致

所以第一列的转移是：

```java
dp[i][0] = dp[i - 1][0] && s1.charAt(i - 1) == s3.charAt(i - 1);
```

#### 第一行 `dp[0][j]`

同理，这里表示只用 `s2`，不用 `s1`。

所以第一行的转移是：

```java
dp[0][j] = dp[0][j - 1] && s2.charAt(j - 1) == s3.charAt(j - 1);
```

可以看出来，初始化本质上就是在处理“只从一个字符串里拿字符”的特殊情况。

### 用题目的例子带一下思路

看示例 1：

```text
s1 = "aabcc"
s2 = "dbbca"
s3 = "aadbbcbcac"
```

先看开头几步会更容易理解。

#### `dp[1][0]`

表示：

```text
"a" 能不能组成 "a"
```

显然可以，所以是 `true`。

#### `dp[2][0]`

表示：

```text
"aa" 能不能组成 "aa"
```

也成立，所以还是 `true`。

#### `dp[2][1]`

表示：

```text
s1 前 2 个字符 "aa"
s2 前 1 个字符 "d"
能不能组成 s3 前 3 个字符 "aad"
```

最后一个字符是 `'d'`，只能来自 `s2`。  
而 `s2[0]` 也确实是 `'d'`，再加上前面的 `dp[2][0]` 已经成立，所以 `dp[2][1] = true`。

这样一格一格推下去，最后如果 `dp[n][m] = true`，就说明整个 `s3` 可以由 `s1` 和 `s2` 交错组成。

所以这题本质上不是在“猜一条交错路径”，而是在一张二维表里，把所有前缀组合的可能性都推出来。

### 代码

```java
public boolean isInterleave(String s1, String s2, String s3) {
    int n = s1.length(), m = s2.length(), t = s3.length();
    if (n + m != t) {
        return false;
    }

    boolean[][] dp = new boolean[n + 1][m + 1];
    dp[0][0] = true;

    for (int i = 1; i <= n; i++) {
        dp[i][0] = dp[i - 1][0] && s1.charAt(i - 1) == s3.charAt(i - 1);
    }

    for (int j = 1; j <= m; j++) {
        dp[0][j] = dp[0][j - 1] && s2.charAt(j - 1) == s3.charAt(j - 1);
    }

    for (int i = 1; i <= n; i++) {
        for (int j = 1; j <= m; j++) {
            char c3 = s3.charAt(i + j - 1);
            boolean fromS1 = s1.charAt(i - 1) == c3 && dp[i - 1][j];
            boolean fromS2 = s2.charAt(j - 1) == c3 && dp[i][j - 1];
            dp[i][j] = fromS1 || fromS2;
        }
    }

    return dp[n][m];
}
```

这段代码真正要抓住的只有一句话：

**`dp[i][j]` 成不成立，只看最后一个字符来自哪一边。**

这个想通了，整道题就顺了。

### 为什么可以压成一维数组

题目进阶要求只用 `O(s2.length)` 的额外空间。

这其实也不难，因为观察二维转移就会发现：

```text
dp[i][j]
```

只依赖两个位置：

- 上边的 `dp[i - 1][j]`
- 左边的 `dp[i][j - 1]`

也就是说，当前这一行只需要：

- 上一行同列的值
- 当前行前一列的值

所以二维表不一定要完整保留，只要一行一行滚动更新就够了。

### 一维数组里，`dp[j]` 到底表示什么

在一维写法里，`dp[j]` 表示：

**当前处理到 `s1` 的第 `i` 层时，`s1` 前 `i` 个字符和 `s2` 前 `j` 个字符，能不能组成 `s3` 的前 `i+j` 个字符。**

而在每一轮更新前后：

- 更新前的 `dp[j]`，相当于二维里的“上一行”
- 更新后的 `dp[j]`，相当于二维里的“当前行”

所以状态更新时：

- `dp[j]` 旧值代表 `dp[i - 1][j]`
- `dp[j - 1]` 新值代表 `dp[i][j - 1]`

于是就能写成：

```java
dp[j] = (dp[j] && s1.charAt(i - 1) == s3.charAt(i + j - 1))
     || (dp[j - 1] && s2.charAt(j - 1) == s3.charAt(i + j - 1));
```

这其实和二维版本完全一样，只是把空间压掉了。

### 一维优化代码

```java
public boolean isInterleaveOptimized(String s1, String s2, String s3) {
    int n = s1.length(), m = s2.length(), t = s3.length();
    if (n + m != t) {
        return false;
    }

    boolean[] dp = new boolean[m + 1];
    dp[0] = true;

    for (int j = 1; j <= m; j++) {
        dp[j] = dp[j - 1] && s2.charAt(j - 1) == s3.charAt(j - 1);
    }

    for (int i = 1; i <= n; i++) {
        dp[0] = dp[0] && s1.charAt(i - 1) == s3.charAt(i - 1);

        for (int j = 1; j <= m; j++) {
            boolean fromS1 = dp[j] && s1.charAt(i - 1) == s3.charAt(i + j - 1);
            boolean fromS2 = dp[j - 1] && s2.charAt(j - 1) == s3.charAt(i + j - 1);

            dp[j] = fromS1 || fromS2;
        }
    }

    return dp[m];
}
```

一维写法的难点不在公式，而在你得分清楚：

- `dp[j]` 什么时候还是旧值
- `dp[j - 1]` 什么时候已经是新值

把这件事理顺，一维优化就不会乱。

### 复杂度分析

二维 DP：

- 时间复杂度：$O(n \times m)$
- 空间复杂度：$O(n \times m)$

一维优化版：

- 时间复杂度：$O(n \times m)$
- 空间复杂度：$O(m)$

时间上不会变，因为所有状态还是都要算一遍；  
优化掉的是空间。

### 这题真正值得记住的地方

这题最值得记住的，不只是“交错字符串怎么判”，而是这种建模方式：

**当一个字符串问题要求你判断“前面这部分能不能由另外两部分按规则组成”时，很适合从前缀状态入手。**

这里的关键不是去模拟某一条具体的交错路径，而是定义：

```text
前 i 个 + 前 j 个，能不能组成前 i+j 个
```

一旦把问题转成这种前缀 DP，原本看起来有点抽象的“交错”就会变得非常具体。

这类题很多时候难的不是公式，而是状态定义。  
把状态定义对了，后面往往就只是正常递推。

public class Main {

    public static boolean isInterleave(String s1, String s2, String s3) {
        int n = s1.length();
        int m = s2.length();
        if (n + m != s3.length()) {
            return false;
        }

        boolean[][] dp = new boolean[n + 1][m + 1];

        for (int i = 1; i <= n; i++) {
            dp[i][0] = dp[i - 1][0] && s1.charAt(i - 1) == s3.charAt(i - 1);
        }

        for (int j = 1; j <= m; j++) {
            dp[0][j] = dp[0][j - 1] && s2.charAt(j - 1) == s3.charAt(j - 1);
        }

        for (int i = 1; i <= n; i++) {
            for (int j = 1; j <= m; j++) {
                char c3 = s3.charAt(i + j - 1);
                boolean fromS1 = s1.charAt(i - 1) == c3 && dp[i - 1][j];

        }

        return dp[n][m];
    }

    public static boolean isInterleaveOptimized(String s1, String s2, String s3) {
        int n = s1.length();
        int m = s2.length();
        if (n + m != s3.length()) {
            return false;
        }

        boolean[] dp = new boolean[m + 1];
        dp[0] = true;

        for (int j = 1; j <= m; j++) {
            dp[j] = dp[j - 1] && s2.charAt(j - 1) == s3.charAt(j - 1);
        }

        for (int i = 1; i <= n; i++) {
            dp[0] = dp[0] && s1.charAt(i - 1) == s3.charAt(i - 1);

            for (int j = 1; j <= m; j++) {

                boolean fromS2 = dp[j - 1] && s2.charAt(j - 1) == s3.charAt(i + j - 1);

                dp[j] = fromS1 || fromS2;
            }
        }

        return dp[m];
    }

    public static void main(String[] args) {
        String s1_1 = "aabcc", s2_1 = "dbbca", s3_1 = "aadbbcbcac";
        System.out.println("示例 1 输入: s1 = " + s1_1 + ", s2 = " + s2_1 + ", s3 = " + s3_1);
        System.out.println("示例 1 输出: " + isInterleaveOptimized(s1_1, s2_1, s3_1));
        System.out.println("-----------------------------");

        String s1_2 = "aabcc", s2_2 = "dbbca", s3_2 = "aadbbbaccc";
        System.out.println("示例 2 输入: s1 = " + s1_2 + ", s2 = " + s2_2 + ", s3 = " + s3_2);
        System.out.println("示例 2 输出: " + isInterleaveOptimized(s1_2, s2_2, s3_2));
        System.out.println("-----------------------------");

        String s1_3 = "", s2_3 = "", s3_3 = "";
        System.out.println("示例 3 输入: s1 = \"\", s2 = \"\", s3 = \"\"");
        System.out.println("示例 3 输出: " + isInterleaveOptimized(s1_3, s2_3, s3_3));
    }
}
