## 题目

**描述：**

给你一个字符串 `s`，找到 `s` 中最长的**回文子串**。

**示例 1:**

- **输入:** `s = "babad"`
- **输出:** `"bab"`
- **解释:** `"aba"` 同样是符合题意的答案。

**示例 2:**

- **输入:** `s = "cbbd"`
- **输出:** `"bb"`

**提示:**

- `1 <= s.length <= 1000`
- `s` 仅由数字和英文字母组成

## 题解

### 这题最容易走偏的地方，是先去枚举所有子串

题目要找的是**最长回文子串**。

一看到“子串”，很多人第一反应就是：

1. 先枚举所有子串
2. 再逐个判断是不是回文
3. 最后取最长的那个

这个思路当然能做，但效率很差。  
因为子串本身就很多，再对每个子串做回文判断，整体复杂度会很难看。

所以这题真正要抓住的，不是“怎么枚举子串”，而是：

**回文串本身有什么结构特点，可以反过来利用。**

### 回文串最核心的性质就是“对称”

一个字符串之所以是回文，根本原因就是它绕着某个中心左右对称。

比如：

- `"aba"` 的中心是中间那个 `'b'`
- `"abba"` 的中心不是某个字符，而是中间两个 `'b'` 之间的空隙

这一步很关键，因为它说明：

**我们没必要先枚举所有子串，再去判断它是不是回文。**  
更自然的办法其实是：

**先枚举回文的中心，再向两边扩散。**

只要扩散时左右字符一直相等，这个回文就能不断变长；一旦不相等，当前这个中心能形成的最长回文就结束了。

这就是中心扩散法的核心。

### 为什么中心会有两种

这题里最容易漏掉的点，就是回文中心不只有一种。

#### 第一种：单个字符作为中心

这种情况对应奇数长度回文，比如：

```text
aba
racecar
```

它们都有一个明确的中间字符。

#### 第二种：两个字符之间的空隙作为中心

这种情况对应偶数长度回文，比如：

```text
abba
bb
aaaa
```

它们的中心不是某个字符，而是中间那条缝。

所以对于字符串中的每个位置 `i`，我们都要检查两种扩散方式：

- 以 `i` 为中心扩散
- 以 `i` 和 `i + 1` 之间为中心扩散

少掉任何一种，答案都有可能漏掉。

### 为什么“从中心出发”比“枚举子串”更自然

因为回文不是靠区间本身定义的，而是靠对称性定义的。

比如你拿到一个区间 `[l, r]`，想判断它是不是回文，还是得一层层往中间比。  
但如果一开始就站在中心，往外扩的时候，整个过程天然就在利用回文的结构，不用多绕一层。

这也是为什么这题最常见、最好讲清楚的写法，就是中心扩散，而不是上来就动态规划。

### 用例子看会更直观

先看示例 1：

```text
s = "babad"
```

我们依次把每个位置当作中心。

#### 以第一个 `'b'` 为中心

往两边扩只能得到：

```text
"b"
```

#### 以 `'a'` 为中心

往两边扩：

- 左边是 `'b'`
- 右边是 `'b'`

相等，所以得到：

```text
"bab"
```

再继续往外就越界了，结束。

#### 以中间那个 `'b'` 为中心

往两边扩：

- 左边是 `'a'`
- 右边是 `'a'`

相等，所以得到：

```text
"aba"
```

再往外继续时字符不匹配，结束。

所以在这个例子里，最长回文可能是 `"bab"`，也可能是 `"aba"`，两种都对。

这也正好对应题目说明。

### 再看偶数长度的情况

比如示例 2：

```text
s = "cbbd"
```

如果你只检查“单字符中心”，最多只能找到 `"c"`、`"b"`、`"b"`、`"d"` 这种长度为 1 的回文。

真正的答案 `"bb"`，其实来自：

```text
第二个字符和第三个字符之间的空隙
```

也就是说，偶数长度回文必须靠“双中心”才能找到。

所以这题里每轮都做两次扩散，不是技巧，而是必须。

### 扩散函数到底在做什么

扩散函数其实就一件事：

给你一个左边界 `left` 和右边界 `right`，然后不断向两边走，只要：

- `left` 没越界
- `right` 没越界
- `s.charAt(left) == s.charAt(right)`

就继续扩。

一旦条件不满足，说明上一次满足时的区间，才是真正的回文区间。

所以函数最后返回的不是当前位置，而是：

**以这个中心能扩出来的最长回文长度。**

这也是为什么它最后会写成：

```java
return right - left - 1;
```

因为循环退出时，`left` 和 `right` 已经各自多走了一步，真正的回文区间其实是：

```text
[left + 1, right - 1]
```

长度自然就是：

```text
right - left - 1
```

### 为什么更新答案时公式是这样

外层遍历到某个位置 `i` 时，我们会得到当前中心扩出来的最大长度 `len`。

这时需要把“长度”转换成“起点和终点”。

代码里常见的写法是：

```java
start = i - (len - 1) / 2;
end = i + len / 2;
```

这套公式的好处是：  
奇数长度和偶数长度都能统一处理。

比如：

#### 奇数长度

如果当前回文是 `"aba"`，长度 `len = 3`，中心在 `i = 1`：

```text
start = 1 - (3 - 1) / 2 = 0
end   = 1 + 3 / 2 = 2
```

正好对应 `"aba"`。

#### 偶数长度

如果当前回文是 `"abba"`，长度 `len = 4`，中心在左侧那个中间位置附近，也能正确算出边界。

所以这两行看起来像技巧，实际上只是把“中心 + 长度”还原成区间而已。

### 代码

```java
public String longestPalindrome(String s) {
    if (s == null || s.length() < 1) {
        return "";
    }

    int start = 0, end = 0;

    for (int i = 0; i < s.length(); i++) {
        int len1 = expandAroundCenter(s, i, i);
        int len2 = expandAroundCenter(s, i, i + 1);

        int len = Math.max(len1, len2);

        if (len > end - start) {
            start = i - (len - 1) / 2;
            end = i + len / 2;
        }
    }

    return s.substring(start, end + 1);
}

private int expandAroundCenter(String s, int left, int right) {
    while (left >= 0 && right < s.length() && s.charAt(left) == s.charAt(right)) {
        left--;
        right++;
    }
    return right - left - 1;
}
```

这段代码真正要抓住的其实就三点：

- 每个位置要做两次扩散
- 一次处理奇数长度，一次处理偶数长度
- 扩散到不能扩为止，当前中心的答案就出来了

理解了这三点，这题基本就顺了。

### 为什么这题不一定非要写 DP

这题当然也能用动态规划做，但中心扩散法通常更适合这道题。

因为它更贴近回文串的定义，而且：

- 思路更直观
- 代码更短
- 空间复杂度更低

动态规划通常适合那种“区间状态天然需要复用”的场景；  
而这题里，中心扩散已经足够好，没必要为了“写 DP”而硬上 DP。

尤其是做教学型题解时，中心扩散更容易让读者真正理解为什么这么做。

### 复杂度分析

- 时间复杂度：$O(n^2)$

因为一共有大约 `2n - 1` 个中心，每个中心最坏情况下都可能向两边扩到整个字符串长度，所以整体最坏是平方级。

- 空间复杂度：$O(1)$

只用了几个变量记录边界和长度，没有额外开与字符串长度相关的空间。

### 这题真正值得记住的地方

这题最值得记住的，不是“最长回文子串怎么写”，而是这种转换思路：

**如果一个问题的对象本身有很强的结构特征，就不要先暴力枚举结果，再回头验证；应该直接围绕这个结构去做。**

这里的结构特征就是“对称中心”。

所以这题真正的转折，不是从暴力法优化到某种模板，而是从“枚举子串”换到了“枚举中心”。

这类思路以后会很常见：  
先抓对象的本质结构，再决定怎么枚举，往往比直接套模板更重要。

### 完整代码

```java
public class Main {

    public static String longestPalindrome(String s) {
        if (s == null || s.length() < 1) {
            return "";
        }

        int start = 0, end = 0;

        for (int i = 0; i < s.length(); i++) {
            int len1 = expandAroundCenter(s, i, i);
            int len2 = expandAroundCenter(s, i, i + 1);

            int len = Math.max(len1, len2);

            if (len > end - start) {
                start = i - (len - 1) / 2;
                end = i + len / 2;
            }
        }

        return s.substring(start, end + 1);
    }

    private static int expandAroundCenter(String s, int left, int right) {
        while (left >= 0 && right < s.length() && s.charAt(left) == s.charAt(right)) {
            left--;
            right++;
        }

        return right - left - 1;
    }

    public static void main(String[] args) {
        String s1 = "babad";
        System.out.println("示例 1 输入: s = \"" + s1 + "\"");
        System.out.println("示例 1 输出: \"" + longestPalindrome(s1) + "\"");
        System.out.println("-----------------------------");

        String s2 = "cbbd";
        System.out.println("示例 2 输入: s = \"" + s2 + "\"");
        System.out.println("示例 2 输出: \"" + longestPalindrome(s2) + "\"");
        System.out.println("-----------------------------");

        String s3 = "a";
        System.out.println("示例 3 输入: s = \"" + s3 + "\"");
        System.out.println("示例 3 输出: \"" + longestPalindrome(s3) + "\"");
        System.out.println("-----------------------------");

        String s4 = "aaaa";
        System.out.println("示例 4 输入: s = \"" + s4 + "\"");
        System.out.println("示例 4 输出: \"" + longestPalindrome(s4) + "\"");
    }
}
```
