## 题目

**描述：**

给你两个单词 `word1` 和 `word2`，请返回将 `word1` 转换成 `word2` 所使用的最少操作数。

你可以对一个单词进行如下三种操作：

- 插入一个字符
- 删除一个字符
- 替换一个字符

**示例 1:**

- **输入:** `word1 = "horse"`, `word2 = "ros"`
- **输出:** 3
- **解释:**

  horse -> rorse (将 'h' 替换为 'r')

  rorse -> rose (删除 'r')

  rose -> ros (删除 'e')

**示例 2:**

- **输入:** `word1 = "intention"`, `word2 = "execution"`
- **输出:** 5
- **解释:**

  intention -> inention (删除 't')

  inention -> enention (将 'i' 替换为 'e')

  enention -> exention (将 'n' 替换为 'x')

  exention -> exection (将 'n' 替换为 'c')

  exection -> execution (插入 'u')

**提示:**

- `0 <= word1.length, word2.length <= 500`
- `word1` 和 `word2` 由小写英文字母组成

## 题解

### 这题本质上是在问：把一个前缀变成另一个前缀，最少要几步

“编辑距离”这个名字听起来有点抽象，但题目真正问的事情其实很具体：

给你两个字符串，允许你做三种操作：

- 插入一个字符
- 删除一个字符
- 替换一个字符

然后问你，最少做多少步，才能把 `word1` 变成 `word2`。

这类题最容易卡住的地方，是一上来就盯着整个字符串看。  
但如果直接从整体思考，很难判断当前这一步到底该删、该插，还是该替换。

更自然的做法，是把问题拆小：

**如果只看 `word1` 的前 `i` 个字符和 `word2` 的前 `j` 个字符，它们之间的最小编辑距离是多少？**

一旦改成“前缀和前缀”的问题，这题就很明显是动态规划。

### 为什么要用前缀来定义状态

因为无论最后一步做什么操作，本质上都只会影响字符串的“末尾”。

也就是说，当我们在考虑：

```text
word1 的前 i 个字符 -> word2 的前 j 个字符
```

时，最后一步一定只和下面这些位置有关：

- `word1[i - 1]`
- `word2[j - 1]`

这说明问题天然适合写成前缀状态。

最顺手的定义是：

```text
dp[i][j] 表示 word1 的前 i 个字符转换成 word2 的前 j 个字符所需的最少操作数
```

这个定义一立住，后面的转移就比较自然了。

### 先把边界想清楚

动态规划里，边界往往决定你有没有真正把状态想明白。这题也是一样。

#### `dp[i][0]` 表示什么

表示：

```text
word1 的前 i 个字符 -> 空字符串
```

那除了把这 `i` 个字符全删掉，没有别的办法。  
所以：

```text
dp[i][0] = i
```

#### `dp[0][j]` 表示什么

表示：

```text
空字符串 -> word2 的前 j 个字符
```

那就只能不断插入字符。  
所以：

```text
dp[0][j] = j
```

这两个边界其实很重要，因为它们正好对应题目给出的两种基本操作：

- 删除
- 插入

### 转移为什么只分“字符相同”和“字符不同”两种情况

这是整道题最关键的地方。

我们现在要算的是：

```text
dp[i][j]
```

也就是把 `word1` 前 `i` 个字符变成 `word2` 前 `j` 个字符的最少步数。

这时候只需要看两个字符串当前最后一个字符：

- `word1[i - 1]`
- `word2[j - 1]`

#### 情况一：这两个字符相同

如果它们本来就一样，那最后这个字符根本不用动。

也就是说，问题会直接缩小成：

```text
word1 前 i-1 个字符 -> word2 前 j-1 个字符
```

所以：

```text
dp[i][j] = dp[i - 1][j - 1]
```

这里没有额外代价，因为最后这个字符已经匹配好了。

#### 情况二：这两个字符不同

这时候就必须做一次操作，而可选的操作正好有三种。

### 三种操作，分别对应什么状态

这里不能只背公式，最好把三种操作和状态对应起来理解。

#### 1. 替换

如果把 `word1[i - 1]` 直接替换成 `word2[j - 1]`，那么这两个末尾字符就对齐了。

剩下的问题就是：

```text
word1 前 i-1 个字符 -> word2 前 j-1 个字符
```

所以替换对应的是：

```text
dp[i - 1][j - 1] + 1
```

#### 2. 删除

如果删除 `word1[i - 1]`，那就相当于先把 `word1` 末尾这个字符扔掉，再去匹配：

```text
word1 前 i-1 个字符 -> word2 前 j 个字符
```

所以删除对应的是：

```text
dp[i - 1][j] + 1
```

#### 3. 插入

如果在 `word1` 末尾插入一个字符，让它去匹配 `word2[j - 1]`，那么插完之后，这个新插入的字符就和 `word2[j - 1]` 对上了。

于是剩下的问题变成：

```text
word1 前 i 个字符 -> word2 前 j-1 个字符
```

所以插入对应的是：

```text
dp[i][j - 1] + 1
```

### 所以完整转移式就是这样来的

如果末尾字符相同：

```text
dp[i][j] = dp[i - 1][j - 1]
```

如果末尾字符不同：

```text
dp[i][j] = min(
    dp[i - 1][j - 1],   // 替换
    dp[i - 1][j],       // 删除
    dp[i][j - 1]        // 插入
) + 1
```

注意，这里的 `+1` 不是随便加的，而是因为这三种情况都对应“当前这一步做了一次操作”。

### 用一个小例子感受一下

以题目里的 `horse -> ros` 为例。

我们不用把整张表全推完，只看几个关键位置就能明白意思。

假设我们在比较：

```text
word1 = "h"
word2 = "r"
```

最后一个字符不同。  
那这里有三种想法：

- 把 `h` 替换成 `r`
- 删除 `h`
- 插入一个 `r`

显然最优是替换，所以答案是 `1`。

再往后，比如：

```text
word1 = "ho"
word2 = "ro"
```

最后一个字符 `'o'` 和 `'o'` 相同，那这一位不用动。  
问题就直接退化成：

```text
"h" -> "r"
```

所以这里的答案会继承左上角状态。

这就是为什么这题的 DP 表，其实一直都在做同一件事：

**看末尾这一位要不要处理，如果要处理，该选哪种操作更划算。**

### 代码

```java
public int minDistance(String word1, String word2) {
    int m = word1.length();
    int n = word2.length();

    int[][] dp = new int[m + 1][n + 1];

    for (int i = 0; i <= m; i++) {
        dp[i][0] = i;
    }

    for (int j = 0; j <= n; j++) {
        dp[0][j] = j;
    }

    for (int i = 1; i <= m; i++) {
        for (int j = 1; j <= n; j++) {
            if (word1.charAt(i - 1) == word2.charAt(j - 1)) {
                dp[i][j] = dp[i - 1][j - 1];
            } else {
                dp[i][j] = Math.min(
                    dp[i - 1][j - 1],
                    Math.min(dp[i - 1][j], dp[i][j - 1])
                ) + 1;
            }
        }
    }

    return dp[m][n];
}
```

这段代码里最应该真正吃透的，不是公式本身，而是这句话：

**`dp[i][j]` 的最后一步，只可能是替换、删除、插入中的一种。**

理解了这一点，公式其实就是顺着题意写出来的。

### 为什么这题是经典 DP

因为它有非常标准的动态规划特征：

- 大问题可以拆成更小的前缀问题
- 当前状态只依赖于左边、上边、左上角
- 每个状态的最优解可以由更小状态推出

而且这题的状态设计很有代表性。  
以后碰到字符串 DP，只要题目里出现“前几个字符”“最少操作”“是否能匹配”这类字眼，就很容易往前缀 DP 的方向靠。

### 复杂度分析

- 时间复杂度：$O(m \times n)$

因为整张 `dp` 表的每个位置都要计算一次。

- 空间复杂度：$O(m \times n)$

用了一个二维数组保存状态。

这题其实还能继续做空间优化，因为当前状态只依赖上一行和当前行左边的位置，所以理论上可以压成一维数组。  
不过这题用二维写法更好理解，也更适合讲清楚思路。

### 这题真正值得记住的地方

这题最值得记住的，不只是“编辑距离怎么写”，而是这种思考方式：

**当题目要求把一个对象变成另一个对象，而且每一步操作都只影响局部时，通常可以从“最后一步做了什么”来反推状态转移。**

这里的“最后一步”就是：

- 替换
- 删除
- 插入

一旦把最后一步拆清楚，整个 DP 转移就会非常自然。

这也是很多动态规划题的共同套路：  
不是先背公式，而是先问自己——**当前这个状态，最后一步可能怎么来的？**

### 完整代码

```java
public class Main {

    public static int minDistance(String word1, String word2) {
        int m = word1.length();
        int n = word2.length();

        int[][] dp = new int[m + 1][n + 1];

        for (int i = 0; i <= m; i++) {
            dp[i][0] = i;
        }

        for (int j = 0; j <= n; j++) {
            dp[0][j] = j;
        }

        for (int i = 1; i <= m; i++) {
            for (int j = 1; j <= n; j++) {
                if (word1.charAt(i - 1) == word2.charAt(j - 1)) {
                    dp[i][j] = dp[i - 1][j - 1];
                } else {
                    dp[i][j] = Math.min(
                        dp[i - 1][j - 1],
                        Math.min(dp[i - 1][j], dp[i][j - 1])
                    ) + 1;
                }
            }
        }

        return dp[m][n];
    }

    public static void main(String[] args) {
        String w1_ex1 = "horse";
        String w2_ex1 = "ros";
        System.out.println("示例 1 输入: word1 = \"" + w1_ex1 + "\", word2 = \"" + w2_ex1 + "\"");
        System.out.println("示例 1 输出: " + minDistance(w1_ex1, w2_ex1));
        System.out.println("-----------------------------");

        String w1_ex2 = "intention";
        String w2_ex2 = "execution";
        System.out.println("示例 2 输入: word1 = \"" + w1_ex2 + "\", word2 = \"" + w2_ex2 + "\"");
        System.out.println("示例 2 输出: " + minDistance(w1_ex2, w2_ex2));
        System.out.println("-----------------------------");

        String w1_ex3 = "";
        String w2_ex3 = "a";
        System.out.println("示例 3 输入: word1 = \"" + w1_ex3 + "\", word2 = \"" + w2_ex3 + "\"");
        System.out.println("示例 3 输出: " + minDistance(w1_ex3, w2_ex3));
    }
}
```
