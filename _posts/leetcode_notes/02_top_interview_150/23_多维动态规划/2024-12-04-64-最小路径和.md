## 题目

**描述：**

给定一个包含非负整数的 `m x n` 网格 `grid`，请找出一条从左上角到右下角的路径，使得路径上的数字总和为最小。

**说明：** 每次只能向下或者向右移动一步。

**示例 1:**

![示例1图片](https://assets.leetcode.com/uploads/2020/11/05/minpath.jpg)

- **输入:** `grid = [[1,3,1],[1,5,1],[4,2,1]]`
- **输出:** 7
- **解释:** 因为路径 1→3→1→1→1 的总和最小。

**示例 2:**

- **输入:** `grid = [[1,2,3],[4,5,6]]`
- **输出:** 12

**提示:**

- `m == grid.length`
- `n == grid[i].length`
- `1 <= m, n <= 200`
- `0 <= grid[i][j] <= 200`

## 题解

### 这题本质上是在问：走到每个位置的最小代价是多少

从左上角走到右下角，每次只能向右或者向下。

这类题的关键，不是去枚举所有路径，而是先想清楚一个更小的问题：

**如果我现在想走到某个格子 `(i, j)`，最小路径和应该怎么算？**

只要这个问题能回答出来，整张表就可以一步步推完，最后右下角那个位置自然就是答案。

这就是典型的动态规划思路。

### 为什么当前位置只和上边、左边有关

因为题目限制得很死：

- 只能向右走
- 只能向下走

所以想走到 `(i, j)`，最后一步只可能有两种来源：

- 从上面的 `(i - 1, j)` 走下来
- 从左边的 `(i, j - 1)` 走过来

不会有第三种可能。

于是，走到 `(i, j)` 的最小路径和，就只能是：

```text
上边的最小路径和 和 左边的最小路径和 里较小的那个
再加上当前格子的值
```

这就是整道题最核心的状态转移。

### 状态该怎么定义最顺

我们可以定义：

```text
dp[i][j] 表示从左上角走到 (i, j) 的最小路径和
```

有了这个定义，转移就很自然了：

```text
dp[i][j] = min(dp[i - 1][j], dp[i][j - 1]) + grid[i][j]
```

含义非常直接：

- 先看从哪里过来更便宜
- 再把当前格子的代价加上

这类“从起点走到当前位置最优值”的网格题，很多都是这个套路。

### 第一行和第一列为什么要单独处理

虽然中间位置可以直接套公式，但第一行和第一列是特殊的，因为它们少一个来源。

#### 第一行

第一行的格子不可能从上面过来，只能一路从左边走过来。  
所以第一行的值只能不断累加：

```text
grid[0][j] = grid[0][j] + grid[0][j - 1]
```

#### 第一列

同理，第一列的格子不可能从左边过来，只能一路从上面走下来。  
所以第一列也是不断累加：

```text
grid[i][0] = grid[i][0] + grid[i - 1][0]
```

把这两条边界处理完，剩下的位置就都能统一按状态转移公式来做了。

### 用题目的例子走一遍

看示例 1：

```text
[
  [1,3,1],
  [1,5,1],
  [4,2,1]
]
```

先处理第一行：

```text
[1,4,5]
```

因为：

- `1`
- `1 + 3 = 4`
- `4 + 1 = 5`

再处理第一列：

```text
[
  [1,4,5],
  [2,5,1],
  [6,2,1]
]
```

因为：

- 第二行第一列：`1 + 1 = 2`
- 第三行第一列：`2 + 4 = 6`

接着处理中间位置。

#### 位置 `(1,1)`，也就是数字 `5`

它可以从上面的 `4` 过来，也可以从左边的 `2` 过来。  
显然左边更小，所以这里变成：

```text
5 + min(4, 2) = 7
```

#### 位置 `(1,2)`，也就是数字 `1`

它可以从上面的 `5` 过来，也可以从左边的 `7` 过来。  
取较小的是 `5`，所以这里变成：

```text
1 + min(5, 7) = 6
```

#### 位置 `(2,1)`，也就是数字 `2`

它可以从上面的 `7` 过来，也可以从左边的 `6` 过来。  
取较小的是 `6`，所以这里变成：

```text
2 + min(7, 6) = 8
```

#### 位置 `(2,2)`，也就是数字 `1`

它可以从上面的 `6` 过来，也可以从左边的 `8` 过来。  
取较小的是 `6`，所以这里变成：

```text
1 + min(6, 8) = 7
```

最后右下角得到的就是 `7`，也就是答案。

### 为什么可以直接在原数组上改

如果严格按 DP 写，我们当然可以另外开一个二维数组 `dp`。

但这题里，`dp[i][j]` 一旦算出来，后面其实只会被右边或者下边继续使用，不再需要保留原始的 `grid[i][j]`。

所以完全可以直接把最小路径和写回 `grid[i][j]` 里：

```java
grid[i][j] += Math.min(grid[i - 1][j], grid[i][j - 1]);
```

这样就相当于把 `grid` 本身当成了 `dp` 数组来用。

这么写有两个好处：

- 代码更短
- 不需要额外空间

这也是这题最常见的写法。

### 代码

```java
public int minPathSum(int[][] grid) {
    int m = grid.length;
    int n = grid[0].length;

    for (int j = 1; j < n; j++) {
        grid[0][j] += grid[0][j - 1];
    }

    for (int i = 1; i < m; i++) {
        grid[i][0] += grid[i - 1][0];
    }

    for (int i = 1; i < m; i++) {
        for (int j = 1; j < n; j++) {
            grid[i][j] += Math.min(grid[i - 1][j], grid[i][j - 1]);
        }
    }

    return grid[m - 1][n - 1];
}
```

这段代码真正要抓住的就是三步：

- 先处理第一行
- 再处理第一列
- 最后处理中间所有格子

因为中间格子的状态转移依赖左边和上边，所以遍历顺序必须保证它们已经算好。按从上到下、从左到右的顺序更新，刚好满足这个条件。

### 复杂度分析

- 时间复杂度：$O(m \times n)$

因为每个格子都只会被访问一次。

- 空间复杂度：$O(1)$

这里直接在原数组上修改，没有额外开辟和网格大小相关的空间。  
如果不允许修改原数组，那么就需要额外的二维 `dp` 数组，空间复杂度会变成 $O(m \times n)$。

### 这题真正值得记住的地方

这题最值得记住的，不是“最小路径和”这四个字，而是这种判断方式：

**如果一个位置的最优解，只取决于它前面有限几个位置的最优解，那么大概率就是动态规划。**

这里的“前面”指的是：

- 上边
- 左边

因为移动方向被限制成了只能向右或向下，所以路径不会绕回去，状态之间也就形成了很清楚的递推关系。

这类二维网格 DP 题非常常见。  
一旦你能熟练抓住“当前位置由谁转移而来”，后面很多题都会顺很多。

### 完整代码

```java
import java.util.Arrays;

public class Main {

    public static int minPathSum(int[][] grid) {
        if (grid == null || grid.length == 0 || grid[0].length == 0) {
            return 0;
        }

        int m = grid.length;
        int n = grid[0].length;

        for (int j = 1; j < n; j++) {
            grid[0][j] += grid[0][j - 1];
        }

        for (int i = 1; i < m; i++) {
            grid[i][0] += grid[i - 1][0];
        }

        for (int i = 1; i < m; i++) {
            for (int j = 1; j < n; j++) {
                grid[i][j] += Math.min(grid[i - 1][j], grid[i][j - 1]);
            }
        }

        return grid[m - 1][n - 1];
    }

    public static void main(String[] args) {
        int[][] grid1 = {
            {1, 3, 1},
            {1, 5, 1},
            {4, 2, 1}
        };
        System.out.println("示例 1 输入: " + Arrays.deepToString(grid1));
        System.out.println("示例 1 输出: " + minPathSum(grid1));
        System.out.println("-----------------------------");

        int[][] grid2 = {
            {1, 2, 3},
            {4, 5, 6}
        };
        System.out.println("示例 2 输入: " + Arrays.deepToString(grid2));
        System.out.println("示例 2 输出: " + minPathSum(grid2));
    }
}
```
