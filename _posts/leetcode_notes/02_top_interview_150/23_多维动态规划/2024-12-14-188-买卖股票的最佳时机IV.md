## 题目

**描述：**

给你一个整数数组 `prices` 和一个整数 `k`，其中 `prices[i]` 是某支给定的股票在第 `i` 天的价格。

设计一个算法来计算你所能获取的最大利润。你最多可以完成 **k** 笔交易。也就是说，你最多可以买 `k` 次，卖 `k` 次。

**注意：** 你不能同时参与多笔交易（你必须在再次购买前出售掉之前的股票）。

**示例 1:**

- **输入:** `k = 2`, `prices = [2,4,1]`
- **输出:** 2
- **解释:** 在第 1 天（股票价格 = 2）的时候买入，在第 2 天（股票价格 = 4）的时候卖出，这笔交易所能获得利润 = 4-2 = 2。

**示例 2:**

- **输入:** `k = 2`, `prices = [3,2,6,5,0,3]`
- **输出:** 7
- **解释:**

  在第 2 天（股票价格 = 2）的时候买入，在第 3 天（股票价格 = 6）的时候卖出，这笔交易所能获得利润 = 6-2 = 4。

  随后，在第 5 天（股票价格 = 0）的时候买入，在第 6 天（股票价格 = 3）的时候卖出，这笔交易所能获得利润 = 3-0 = 3。

**提示:**

- `1 <= k <= 100`
- `1 <= prices.length <= 1000`
- `0 <= prices[i] <= 1000`

## 题解

### 这题和“最多两笔交易”本质上一样，只是把 2 改成了 k

如果你前面已经做过“买卖股票的最佳时机 III”，那这题其实就是它的通用版。

区别不在思路变了，而在于：

- 之前最多做 2 笔交易
- 现在最多做 `k` 笔交易

所以真正要解决的问题还是那个：

**随着天数往后推进，如果我现在正处于第几次买入或第几次卖出状态，我手里最多还能剩下多少利润。**

只不过这次不能再手写 `buy1、sell1、buy2、sell2` 了，而是要把它推广成数组。

### 为什么不能再用“一次交易”的思路

如果只允许做一笔交易，那只需要维护：

- 一个最低买入价
- 一个最大利润

但一旦允许做多笔交易，问题就不再是“找一个最低点和一个最高点”这么简单了。

因为：

- 第 2 次买入，必须建立在第 1 次卖出之后
- 第 3 次买入，必须建立在第 2 次卖出之后
- 每一次买入和卖出，都会影响后面状态的最优值

所以这题本质上仍然是状态机 DP。  
区别只是：之前状态数固定，现在状态数跟 `k` 有关。

### 把每一笔交易都拆成“买入状态”和“卖出状态”

对于第 `j` 笔交易，我们只需要关心两个状态：

#### `buy[j]`

表示：

**完成第 `j` 次买入之后，当前手里持有股票时的最大利润。**

因为买入会花钱，所以它通常是负数，或者说“利润比较小”。

#### `sell[j]`

表示：

**完成第 `j` 次卖出之后，当前手里不持有股票时的最大利润。**

这才是真正实现了一笔完整交易之后的收益。

这样一来，第 1 笔、第 2 笔、……、第 `k` 笔，都能统一用这两组状态去表示。

### 每个状态是怎么转移的

设今天的价格是 `p`。

#### `buy[j]`

今天结束后，处于“第 `j` 次买入”状态，有两种可能：

- 昨天已经买过第 `j` 次了，今天什么都不做
- 今天才进行第 `j` 次买入，而这次买入之前，必须已经完成第 `j-1` 次卖出

所以：

```text
buy[j] = max(buy[j], sell[j - 1] - p)
```

这句很好理解：

- `sell[j - 1]` 是前 `j-1` 笔交易都完成后的最大利润
- 再拿这部分利润去买当前价格的股票，就变成了 `sell[j - 1] - p`

#### `sell[j]`

今天结束后，处于“第 `j` 次卖出”状态，也有两种可能：

- 昨天已经卖过第 `j` 次了，今天不动
- 今天把第 `j` 次买入的股票卖掉

所以：

```text
sell[j] = max(sell[j], buy[j] + p)
```

也就是说，把当前持股状态兑现成现金。

这两条式子就是整道题最核心的转移。

### 为什么第 `j` 次买入依赖第 `j-1` 次卖出

这里很值得强调一下，因为这是整题里最容易混的地方。

题目明确说了：

**不能同时参与多笔交易。**

所以你不可能：

- 第一笔还没卖掉
- 就直接开始第二笔买入

这意味着：

- 第 1 次买入之前，不需要依赖任何卖出
- 第 2 次买入之前，必须已经完成第 1 次卖出
- 第 3 次买入之前，必须已经完成第 2 次卖出

所以 `buy[j]` 只能从 `sell[j - 1]` 转移，而不能乱接别的状态。

这一点其实就是“交易不能重叠”在状态转移里的体现。

### 初始化为什么这样写

假设第一天价格是 `prices[0]`。

我们通常会这样初始化：

```java
Arrays.fill(buy, -prices[0]);
Arrays.fill(sell, 0);
```

意思是：

- 对于任意一次买入状态，如果第一天就买，利润都是 `-prices[0]`
- 任意卖出状态一开始都可以看作 `0`，因为还没开始交易时，利润至少不会小于 `0`

这里看起来像是把所有 `buy[j]` 都初始化成一样的值，但其实没问题，因为后续状态转移会自然把无效状态修正掉。

本质上你只要抓住一点就够了：

**买入状态表示“持股后的最大利润”，卖出状态表示“空仓后的最大利润”。**

初始化就是在给这两个含义一个合理起点。

### 为什么这题还是 `O(nk)` 而不是更复杂

因为每天我们只需要更新：

- 第 1 次买入 / 卖出
- 第 2 次买入 / 卖出
- …
- 第 `k` 次买入 / 卖出

每一天就是扫一遍 `1...k`。

所以总复杂度就是：

```text
天数 × 交易次数 = O(nk)
```

在这道题的数据范围里：

- `k <= 100`
- `prices.length <= 1000`

这个复杂度是完全可接受的。

### 为什么 `k >= n / 2` 时可以直接当成无限次交易

这是这题一个很经典的优化。

因为完成一笔交易，最少也得占两天：

- 一天买
- 一天卖

如果总共有 `n` 天，那最多能完成的有效交易数其实不会超过 `n / 2`。

所以如果题目给的 `k` 已经大到：

```text
k >= n / 2
```

那“最多 `k` 笔交易”这个限制其实已经没有实际约束力了。  
因为你根本用不完这么多交易次数。

这时候问题就退化成：

**你可以做无限次交易，但不能同时持有多股。**

而这个问题的最优策略很简单：  
只要今天比昨天贵，就把这段上涨利润加进来。

也就是：

```java
if (prices[i] > prices[i - 1]) {
    profit += prices[i] - prices[i - 1];
}
```

这个优化不是这题的主线，但很值得提，因为它体现了你对题目上界的理解，而不是机械套模板。

### 用题目的例子带一下思路

看示例 2：

```text
k = 2
prices = [3,2,6,5,0,3]
```

最优答案是 7，对应两笔交易：

- `2 -> 6`，赚 4
- `0 -> 3`，再赚 3

合起来 7。

如果从状态角度看，大概会这样变化：

#### 当价格降到 2 时

第一笔买入状态会更优，因为买得更便宜：

```text
buy[1] = max(buy[1], -2)
```

#### 当价格涨到 6 时

第一笔卖出状态更新：

```text
sell[1] = max(sell[1], buy[1] + 6)
```

于是第一笔交易利润来到 4。

#### 当价格降到 0 时

第二笔买入状态更新：

```text
buy[2] = max(buy[2], sell[1] - 0)
```

这表示：  
第一笔已经赚了 4，现在第二次买入成本几乎最低，所以第二笔持股状态变得很优。

#### 当价格涨到 3 时

第二笔卖出更新：

```text
sell[2] = max(sell[2], buy[2] + 3)
```

最终拿到总利润 7。

你会发现，这和“最多两笔交易”的写法本质上是一模一样的，只不过把 4 个变量变成了两组数组。

### 代码

```java
public int maxProfit(int k, int[] prices) {
    if (k <= 0 || prices == null || prices.length == 0) {
        return 0;
    }

    int n = prices.length;
    if (k >= n / 2) {
        return maxProfitUnlimited(prices);
    }

    int[] buy = new int[k + 1];
    int[] sell = new int[k + 1];

    Arrays.fill(buy, -prices[0]);
    Arrays.fill(sell, 0);

    for (int p : prices) {
        for (int j = 1; j <= k; j++) {
            buy[j] = Math.max(buy[j], sell[j - 1] - p);
            sell[j] = Math.max(sell[j], buy[j] + p);
        }
    }

    return sell[k];
}
```

这里真正要理解的只有两件事：

- `buy[j]` 表示第 `j` 次买入后的最大利润
- `sell[j]` 表示第 `j` 次卖出后的最大利润

其余公式，其实都只是这两个定义顺着题意推出来的。

### 无限次交易的辅助函数

```java
private int maxProfitUnlimited(int[] prices) {
    int maxProfit = 0;
    for (int i = 1; i < prices.length; i++) {
        if (prices[i] > prices[i - 1]) {
            maxProfit += prices[i] - prices[i - 1];
        }
    }
    return maxProfit;
}
```

这段逻辑的本质是：  
把所有连续上涨区间的利润都吃掉。因为在无限次交易的前提下，这样做一定不吃亏。

### 复杂度分析

- 时间复杂度：$O(n \cdot k)$

每天都要更新 `1..k` 这 `k` 组买卖状态。

- 空间复杂度：$O(k)$

只用了两个长度为 `k + 1` 的数组。

### 这题真正值得记住的地方

这题最值得记住的，不是代码本身，而是这个推广过程：

- 固定 2 笔交易时，可以写成 `buy1、sell1、buy2、sell2`
- 当交易次数变成任意 `k` 时，本质没有变，只是把状态从“固定变量”推广成“数组”

也就是说，股票题真正的关键，从来不是背某一道题，而是抓住状态含义：

- 第几次买入
- 第几次卖出
- 当前持股还是空仓

一旦这个框架搭起来，很多股票题其实都只是这个模型的不同参数版本。

### 完整代码

```java
import java.util.Arrays;

public class Main {

    public static int maxProfit(int k, int[] prices) {
        if (k <= 0 || prices == null || prices.length == 0) {
            return 0;
        }

        int n = prices.length;
        if (k >= n / 2) {
            return maxProfitUnlimited(prices);
        }

        int[] buy = new int[k + 1];
        int[] sell = new int[k + 1];

        Arrays.fill(buy, -prices[0]);
        Arrays.fill(sell, 0);

        for (int p : prices) {
            for (int j = 1; j <= k; j++) {
                buy[j] = Math.max(buy[j], sell[j - 1] - p);
                sell[j] = Math.max(sell[j], buy[j] + p);
            }
        }

        return sell[k];
    }

    private static int maxProfitUnlimited(int[] prices) {
        int maxProfit = 0;
        for (int i = 1; i < prices.length; i++) {
            if (prices[i] > prices[i - 1]) {
                maxProfit += prices[i] - prices[i - 1];
            }
        }
        return maxProfit;
    }

    public static void main(String[] args) {
        int k1 = 2;
        int[] prices1 = {2, 4, 1};
        System.out.println("示例 1 输入: k = " + k1 + ", prices = " + Arrays.toString(prices1));
        System.out.println("示例 1 输出: " + maxProfit(k1, prices1));
        System.out.println("-----------------------------");

        int k2 = 2;
        int[] prices2 = {3, 2, 6, 5, 0, 3};
        System.out.println("示例 2 输入: k = " + k2 + ", prices = " + Arrays.toString(prices2));
        System.out.println("示例 2 输出: " + maxProfit(k2, prices2));
        System.out.println("-----------------------------");

        int k3 = 100;
        int[] prices3 = {1, 2, 3, 4, 5};
        System.out.println("示例 3 输入: k = " + k3 + ", prices = " + Arrays.toString(prices3));
        System.out.println("示例 3 输出: " + maxProfit(k3, prices3));
    }
}
```
