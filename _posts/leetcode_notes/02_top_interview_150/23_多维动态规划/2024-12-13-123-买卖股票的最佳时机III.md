## 题目

**描述：**

给定一个数组，它的第 `i` 个元素是一支给定的股票在第 `i` 天的价格。

设计一个算法来计算你所能获取的最大利润。你最多可以完成 **两笔** 交易。

**注意：** 你不能同时参与多笔交易（你必须在再次购买前出售掉之前的股票）。

**示例 1:**

- **输入:** `prices = [3,3,5,0,0,3,1,4]`
- **输出:** 6
- **解释:**

  在第 4 天（股票价格 = 0）的时候买入，在第 6 天（股票价格 = 3）的时候卖出，这笔交易所能获得利润 = 3-0 = 3。

  随后，在第 7 天（股票价格 = 1）的时候买入，在第 8 天 （股票价格 = 4）的时候卖出，这笔交易所能获得利润 = 4-1 = 3。总利润 = 3 + 3 = 6。

**示例 2:**

- **输入:** `prices = [1,2,3,4,5]`
- **输出:** 4
- **解释:**

  在第 1 天（股票价格 = 1）的时候买入，在第 5 天 （股票价格 = 5）的时候卖出, 这笔交易所能获得利润 = 5-1 = 4。

  注意你不能在第 1 天和第 2 天接连购买股票，之后再将它们卖出。
  因为这样属于同时参与了多笔交易，你必须在再次购买前出售掉之前的股票。

**示例 3:**

- **输入:** `prices = [7,6,4,3,1]`
- **输出:** 0
- **解释:** 在这个情况下, 没有交易完成, 所以最大利润为 0。

**示例 4:**

- **输入:** `prices = [1]`
- **输出:** 0

**提示:**

- `1 <= prices.length <= 10^5`
- `0 <= prices[i] <= 10^5`

## 题解

### 这题难的不是买卖股票，而是“最多两笔”怎么管

如果这题只允许做一笔交易，那思路很简单：

- 维护一个历史最低价
- 再维护当前能得到的最大利润

但这题麻烦的地方在于：**最多可以做两笔交易**，而且两笔交易不能重叠，必须先卖掉第一笔，才能开始第二笔。

这一下，问题就不再是“找一个最低点和一个最高点”了，而是：

**在时间推进的过程中，我现在到底处于哪一步，手里有没有股票，此时的最大利润是多少。**

一旦从“状态”这个角度去看，这题就会顺很多。

### 别把它想成“两次独立求最大利润”

很多人第一次做这题时，会本能地想：

- 先找一段最赚钱的区间
- 再找另一段最赚钱的区间
- 最后把两段加起来

这个想法的问题在于，两笔交易之间不是独立的。  
第一笔什么时候结束，会直接影响第二笔什么时候才能开始；而且第一笔赚到的钱，也会影响第二次买入时“剩下多少成本”。

所以这题更合适的思路，不是先切两段，而是：

**每天结束时，分别记录几种关键状态下的最大利润。**

这就是典型的状态机 DP。

### 这题其实只需要盯住 4 个状态

对于任意一天结束时，我们真正关心的只有下面 4 种状态。

#### 1. `buy1`

表示：

**做完第一次买入后，手里持有股票时的最大利润。**

因为买股票要花钱，所以这个值通常是负数。

#### 2. `sell1`

表示：

**完成第一次卖出后，手里不持股时的最大利润。**

这对应“一笔完整交易已经结束”。

#### 3. `buy2`

表示：

**完成第一笔交易后，又做了第二次买入，此时手里持有股票时的最大利润。**

注意，这里已经进入第二笔交易了。

#### 4. `sell2`

表示：

**完成第二次卖出后，手里不持股时的最大利润。**

这就是“最多两笔交易”最终想要的状态。

这 4 个状态一旦想清楚，整题就几乎只剩翻译了。

### 为什么 `buy` 状态是负数

这点单独说一下会更容易理解。

比如第一天价格是 `3`，如果你买入了一次，那利润其实就是：

```text
-3
```

因为你还没卖，只是先花出去了 3 块钱。

同理，第二次买入时也一样。  
它不是“已经赚了多少”，而是“在做完当前动作后，你账户上的净收益是多少”。  
所以买入状态可以是负数，这很正常。

### 每个状态是怎么转移来的

设今天的股价是 `p`。

#### `buy1`

今天结束后，第一次买入状态有两种可能：

- 昨天就已经处于第一次买入状态，今天什么都不做
- 今天才第一次买入，那利润就是 `-p`

所以：

```text
buy1 = max(buy1, -p)
```

#### `sell1`

今天结束后，第一次卖出状态也有两种可能：

- 昨天已经卖完第一笔了，今天不动
- 今天把第一次买入的股票卖掉，利润就是 `buy1 + p`

所以：

```text
sell1 = max(sell1, buy1 + p)
```

#### `buy2`

今天结束后，第二次买入状态有两种可能：

- 昨天已经买完第二次了，今天不动
- 今天在第一笔交易完成后，再买入一次，利润就是 `sell1 - p`

所以：

```text
buy2 = max(buy2, sell1 - p)
```

#### `sell2`

今天结束后，第二次卖出状态也有两种可能：

- 昨天已经完成第二笔交易，今天不动
- 今天把第二次买入的股票卖掉，利润就是 `buy2 + p`

所以：

```text
sell2 = max(sell2, buy2 + p)
```

这 4 个式子就是整道题的核心。

### 初始化为什么这么写

假设第一天价格是 `prices[0]`。

那么一开始可以这样初始化：

```java
int buy1 = -prices[0];
int sell1 = 0;
int buy2 = -prices[0];
int sell2 = 0;
```

解释一下：

- `buy1 = -prices[0]`：第一天就买入第一笔
- `sell1 = 0`：什么都不做，利润至少是 0
- `buy2 = -prices[0]`：这套写法里把第二次买入也先初始化成同样的值，后面转移会自然修正
- `sell2 = 0`：两笔交易都不做，利润也是 0

这里最重要的不是死记初始化，而是明白：  
**这些变量表示的是“截至今天，处于某个状态时的最大利润”。**

### 用题目的例子看一遍会更直观

看示例 1：

```text
prices = [3,3,5,0,0,3,1,4]
```

这组数据的最优答案是 6，对应两笔交易：

- `0 -> 3`，赚 3
- `1 -> 4`，再赚 3

合起来是 6。

如果用状态来理解，关键的几天大概是这样：

#### 前几天价格较低时，`buy1` 会不断更新

比如价格降到 `0` 时：

```text
buy1 = max(之前的 buy1, -0) = 0
```

这表示：  
如果现在才做第一笔买入，那成本最低，相当于“持股后的最大利润”变成了 0。

#### 当价格涨到 3 时，`sell1` 会更新

因为这时可以把第一次买的股票卖掉：

```text
sell1 = max(sell1, buy1 + 3)
```

于是第一笔交易的收益就被记下来了。

#### 接着第二次买入时，`buy2` 会利用 `sell1`

比如后面价格来到 `1`，就有：

```text
buy2 = max(buy2, sell1 - 1)
```

这表示：  
第一笔已经赚到的钱，可以作为第二次买入的基础。

#### 最后价格到 4 时，`sell2` 更新成最终答案

```text
sell2 = max(sell2, buy2 + 4)
```

于是两笔交易的总利润就算出来了。

你会发现，这 4 个状态其实一直在做同一件事：

**每天根据今天的价格，重新评估“如果我此刻正处于某个阶段，最多能赚多少”。**

### 为什么最后返回 `sell2`

因为 `sell2` 表示的是：

**在最多两笔交易都考虑进去后，手里不持股时的最大利润。**

这就是题目想要的最终状态。

而且这套状态设计里，即使最优解实际上只做了一笔交易，`sell2` 也不会出问题，因为“不做第二笔”本身就会被状态转移自然包含进去。

所以最后直接返回：

```java
return sell2;
```

就可以了。

### 代码

```java
public int maxProfit(int[] prices) {
    if (prices == null || prices.length == 0) {
        return 0;
    }

    int buy1 = -prices[0];
    int sell1 = 0;
    int buy2 = -prices[0];
    int sell2 = 0;

    for (int i = 1; i < prices.length; i++) {
        int p = prices[i];

        buy1 = Math.max(buy1, -p);
        sell1 = Math.max(sell1, buy1 + p);
        buy2 = Math.max(buy2, sell1 - p);
        sell2 = Math.max(sell2, buy2 + p);
    }

    return sell2;
}
```

这段代码看起来像在硬背 4 个变量，但其实本质上就是把“最多两笔交易”的过程拆成了 4 个阶段。

只要你能把每个变量的含义讲清楚，这段代码就一点也不玄。

### 复杂度分析

- 时间复杂度：$O(n)$

只遍历了一次价格数组。

- 空间复杂度：$O(1)$

只用了 4 个变量保存状态。

### 这题真正值得记住的地方

这题最值得记住的，不是这 4 个变量的名字，而是这种思考方式：

**当题目对操作次数有限制时，可以把“做了第几次操作、当前是否持有”拆成状态。**

股票题里这类模型非常常见。  
一旦题目里出现：

- 最多做 `k` 笔交易
- 不能同时持有多笔
- 买和卖必须成对出现

基本都可以往“状态机 DP”这个方向想。

这比单独记住这道题更重要，因为很多股票题其实都只是这个模型的不同变形。

### 完整代码

```java
import java.util.Arrays;

public class Main {

    public static int maxProfit(int[] prices) {
        if (prices == null || prices.length == 0) {
            return 0;
        }

        int buy1 = -prices[0];
        int sell1 = 0;
        int buy2 = -prices[0];
        int sell2 = 0;

        for (int i = 1; i < prices.length; i++) {
            int p = prices[i];

            buy1 = Math.max(buy1, -p);
            sell1 = Math.max(sell1, buy1 + p);
            buy2 = Math.max(buy2, sell1 - p);
            sell2 = Math.max(sell2, buy2 + p);
        }

        return sell2;
    }

    public static void main(String[] args) {
        int[] prices1 = {3, 3, 5, 0, 0, 3, 1, 4};
        System.out.println("示例 1 输入: " + Arrays.toString(prices1));
        System.out.println("示例 1 输出: " + maxProfit(prices1));
        System.out.println("-----------------------------");

        int[] prices2 = {1, 2, 3, 4, 5};
        System.out.println("示例 2 输入: " + Arrays.toString(prices2));
        System.out.println("示例 2 输出: " + maxProfit(prices2));
        System.out.println("-----------------------------");

        int[] prices3 = {7, 6, 4, 3, 1};
        System.out.println("示例 3 输入: " + Arrays.toString(prices3));
        System.out.println("示例 3 输出: " + maxProfit(prices3));
        System.out.println("-----------------------------");

        int[] prices4 = {1};
        System.out.println("示例 4 输入: " + Arrays.toString(prices4));
        System.out.println("示例 4 输出: " + maxProfit(prices4));
    }
}
```
