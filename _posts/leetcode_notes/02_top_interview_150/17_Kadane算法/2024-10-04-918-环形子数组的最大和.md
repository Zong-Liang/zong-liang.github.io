## 题目

**描述：**

给定一个长度为 `n` 的 **环形** 整数数组 `nums`，返回 `nums` 的非空 **子数组** 的最大可能和。

**环形数组** 意味着数组的末端将会与开头相连呈环状。形式上，`nums[i]` 的下一个元素是 `nums[(i + 1) % n]`，`nums[i]` 的前一个元素是 `nums[(i - 1 + n) % n]`。

**子数组** 最多只能包含固定缓冲区 `nums` 中的每个元素一次。形式上，对于子数组 `nums[i], nums[i + 1], ..., nums[j]`，不存在 `i <= k1, k2 <= j` 其中 `k1 % n == k2 % n`。

**示例 1:**

- **输入:** `nums = [1,-2,3,-2]`
- **输出:** 3
- **解释:** 从子数组 `[3]` 得到最大和 3

**示例 2:**

- **输入:** `nums = [5,-3,5]`
- **输出:** 10
- **解释:** 从子数组 `[5,5]` 得到最大和 5 + 5 = 10

**示例 3:**

- **输入:** `nums = [3,-2,2,-3]`
- **输出:** 3
- **解释:** 从子数组 `[3]` 和 `[3,-2,2]` 都可以得到最大和 3

**提示:**

- `n == nums.length`
- `1 <= n <= 3 * 10^4`
- `-3 * 10^4 <= nums[i] <= 3 * 10^4`

## 题解

### 这题比上一题多出来的，只有一件事

上一题「最大子数组和」里，子数组只能老老实实在数组中间连续出现。

但这题是**环形数组**，所以最大子数组除了普通情况之外，还可能出现一种新情况：

**子数组从数组尾部接到数组头部。**

比如：

```text
[5, -3, 5]
```

最大和不是中间某一段，而是首尾拼起来的：

```text
[5, 5]
```

所以这题真正要解决的，不是重新发明一套新算法，而是先把情况分清：

- 不成环时怎么办
- 成环时怎么办

只要这两种情况都想明白，题目其实就顺了。

### 第一种情况：最大子数组根本没有绕回去

如果最大子数组没有跨越数组边界，那这题就和普通的最大子数组和一模一样。

也就是说，直接套上一题的 Kadane 算法就行。

我们照样维护：

- `curMax`：以当前位置结尾的最大子数组和
- `maxSum`：遍历过程中出现过的最大值

这一部分没有任何新东西。

### 第二种情况：最大子数组跨越了边界

这才是这题真正新增的部分。

如果一个最大子数组是“成环”的，那它看起来会像这样：

- 取了数组尾部一段
- 又取了数组开头一段
- 中间有一段没取

换句话说，**成环的最大子数组，其实等价于“整个数组减去中间一段最小子数组”。**

这是这题最核心的转换。

比如：

```text
[5, -3, 5]
```

总和是：

```text
7
```

如果我们把中间最小那段 `[-3]` 去掉，剩下的就是首尾两边的 `5 + 5 = 10`。

所以成环情况下的最大和，其实可以写成：

```text
总和 - 最小子数组和
```

这一步一旦想通，这题就和上一题连起来了。

### 为什么“总和减最小子数组”是对的

因为成环子数组看上去虽然断成了两段，但它在整个数组里恰好对应“留下了两边，挖掉了中间”。

而要让“留下的部分”尽可能大，就等价于让“挖掉的那一段”尽可能小。

所以成环最大和并不是直接去找“两段拼起来最大的值”，而是换一个角度：

**找一段最小的连续子数组，把它从总和里扣掉。**

这就是这题最巧的地方。

### 所以整题其实只有两种答案来源

最后答案只可能来自下面两种情况之一：

1. **不成环**  
   直接就是普通最大子数组和 `maxSum`

2. **成环**  
   等于 `totalSum - minSum`

所以最终答案自然就是：

```java
Math.max(maxSum, totalSum - minSum)
```

但这里还有一个必须单独处理的坑。

### 为什么全负数时不能直接套这个式子

看这个例子：

```text
[-3, -2, -3]
```

普通最大子数组和是：

```text
-2
```

这是对的，因为子数组必须非空，最大值就是最大的那个负数。

但如果你去算：

- `totalSum = -8`
- `minSum = -8`

那么：

```text
totalSum - minSum = 0
```

这就错了。

为什么会出错？

因为这里的“最小子数组”其实是整个数组本身。  
如果你把整个数组都减掉，剩下的就是空数组了，而题目明确说子数组必须非空。

所以一旦数组全是负数，就不能走“总和减最小子数组”这条路。

这个特殊情况的判断方法也很简单：

**如果普通最大子数组和 `maxSum < 0`，说明所有数都是负的，直接返回 `maxSum`。**

### 用示例把思路走一遍

看示例 1：

```text
nums = [1, -2, 3, -2]
```

不成环时，最大子数组和是：

```text
[3] -> 3
```

成环时：

- 总和是 `0`
- 最小子数组和是 `-2`
- `totalSum - minSum = 2`

所以答案取两者最大值：

```text
max(3, 2) = 3
```

再看示例 2：

```text
nums = [5, -3, 5]
```

不成环时最大子数组和是：

```text
7
```

成环时：

- 总和是 `7`
- 最小子数组和是 `-3`
- `7 - (-3) = 10`

所以答案是：

```text
10
```

这两个例子正好把两种情况都说明白了。

### 代码

```java
public int maxSubarraySumCircular(int[] nums) {
    int totalSum = 0;
    int curMax = 0, maxSum = nums[0];
    int curMin = 0, minSum = nums[0];

    for (int num : nums) {
        curMax = Math.max(curMax + num, num);
        maxSum = Math.max(maxSum, curMax);

        curMin = Math.min(curMin + num, num);
        minSum = Math.min(minSum, curMin);

        totalSum += num;
    }

    if (maxSum < 0) {
        return maxSum;
    }

    return Math.max(maxSum, totalSum - minSum);
}
```

### 这段代码最关键的地方

真正关键的是你要看出，这段代码其实同时在做三件事：

1. 用 Kadane 算普通最大子数组和
2. 用 Kadane 的反向版本算最小子数组和
3. 顺手统计数组总和

也就是说，这题并没有脱离 Kadane，只是在它的基础上多维护了一组“最小和”状态。

核心转移仍然很熟悉：

```java
curMax = Math.max(curMax + num, num);
curMin = Math.min(curMin + num, num);
```

前者表示：

- 当前最大和是“接着累加”还是“重新开始”

后者表示：

- 当前最小和是“接着累加”还是“重新开始”

所以这题本质上是把“最大子数组和”这个思路同时用了两遍。

### 为什么最小子数组和也能用 Kadane

因为 Kadane 的本质不是“只能求最大值”，而是：

**每走到一个位置，判断前面的状态还有没有继续保留的价值。**

求最大和时，我们判断的是：

- 前面的和如果有增益，就接着用
- 没增益就重新开始

求最小和时，其实完全一样，只是判断方向反过来了：

- 前面的和如果会让结果更小，就接着用
- 否则就重新开始

所以最小子数组和本质上只是 Kadane 的镜像版本。

### 这题真正值得记住的地方

这题最值得记住的，不是“环形数组最大和 = 两种情况取 max”这句结论本身。

真正值得记住的是这个转换能力：

**当一个环形问题不好直接处理时，可以尝试把“选中哪一段”转化成“舍弃哪一段”。**

这题里就是：

- 成环子数组不好直接算
- 那就改成“总和减去中间最小连续段”

这个思路很重要，因为很多题真正难的地方，不在计算，而在于能不能把问题换个角度重写。

### 复杂度分析

整个过程只遍历一次数组，所以：

- 时间复杂度：$O(n)$

只用了几个变量维护当前状态，所以：

- 空间复杂度：$O(1)$

这也是这题最优、最标准的写法。

### 完整代码

```java
import java.util.Arrays;

public class Main {

    public static int maxSubarraySumCircular(int[] nums) {
        int totalSum = 0;

        int curMax = 0;
        int maxSum = nums[0];

        int curMin = 0;
        int minSum = nums[0];

        for (int num : nums) {
            curMax = Math.max(num, curMax + num);
            maxSum = Math.max(maxSum, curMax);

            curMin = Math.min(num, curMin + num);
            minSum = Math.min(minSum, curMin);

            totalSum += num;
        }

        if (maxSum < 0) {
            return maxSum;
        }

        return Math.max(maxSum, totalSum - minSum);
    }

    public static void main(String[] args) {
        int[] nums1 = {1, -2, 3, -2};
        System.out.println("示例 1 输入: " + Arrays.toString(nums1));
        System.out.println("示例 1 输出: " + maxSubarraySumCircular(nums1));
        System.out.println("-----------------------------");

        int[] nums2 = {5, -3, 5};
        System.out.println("示例 2 输入: " + Arrays.toString(nums2));
        System.out.println("示例 2 输出: " + maxSubarraySumCircular(nums2));
        System.out.println("-----------------------------");

        int[] nums3 = {3, -2, 2, -3};
        System.out.println("示例 3 输入: " + Arrays.toString(nums3));
        System.out.println("示例 3 输出: " + maxSubarraySumCircular(nums3));
        System.out.println("-----------------------------");

        int[] nums4 = {-3, -2, -3};
        System.out.println("示例 4 输入: " + Arrays.toString(nums4));
        System.out.println("示例 4 输出: " + maxSubarraySumCircular(nums4));
    }
}
```
