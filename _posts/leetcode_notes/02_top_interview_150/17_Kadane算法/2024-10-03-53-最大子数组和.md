## 题目

**描述：**

给你一个整数数组 `nums`，请你找出一个具有最大和的连续子数组（子数组最少包含一个元素），返回其最大和。

**子数组** 是数组中的一个连续部分。

**示例 1:**

- **输入:** `nums = [-2,1,-3,4,-1,2,1,-5,4]`
- **输出:** 6
- **解释:** 连续子数组 `[4,-1,2,1]` 的和最大，为 6。

**示例 2:**

- **输入:** `nums = [1]`
- **输出:** 1

**示例 3:**

- **输入:** `nums = [5,4,-1,7,8]`
- **输出:** 23

**提示:**

- `1 <= nums.length <= 10^5`
- `-10^4 <= nums[i] <= 10^4`

**进阶：** 如果你已经实现复杂度为 `O(n)` 的解法，尝试使用更为精妙的 **分治法** 求解。

## 题解

### 这题最关键的，不是“枚举所有子数组”，而是想清楚一件事

假设我们正在看某个位置 `nums[i]`，现在有一个问题：

**以 `nums[i]` 结尾的最大子数组和，到底是应该接上前面的子数组，还是从自己重新开始？**

这其实就是整道题的核心。

因为连续子数组一旦固定了右端点，左边怎么选，其实只有两种可能：

- 要么把前面那段接上
- 要么前面那段不要了，从当前元素重新开始

所以这题真正要解决的，不是“所有区间里哪个最大”这么笼统的问题，而是：

**每走到一个新位置，当前这段连续和还有没有继续保留的价值。**

### 为什么负数前缀应该果断扔掉

这是这题最核心的判断。

如果我们前面已经累加出来的和是正数，那么把它接到当前元素前面，显然会让结果更大。  
但如果前面的累加和是负数，那它对后面只会起反作用。

比如当前元素是 `4`：

- 如果前面的和是 `3`，那接上后得到 `7`，当然更好
- 如果前面的和是 `-2`，那接上后只会变成 `2`，还不如直接从 `4` 自己开始

所以每到一个位置时，我们其实都在做一个非常直接的选择：

**前面的累加如果有贡献，就保留；如果已经成了负担，就直接丢掉。**

这就是 Kadane 算法的核心直觉。

### 先定义状态，题目就顺了

我们用一个变量 `pre` 表示：

**以当前位置结尾的最大连续子数组和。**

注意这句话很重要。  
它不是“全局最大值”，而是“必须以当前元素结尾”。

为什么要这样定义？

因为这样一来，转移就特别自然：

对于当前元素 `x`，新的 `pre` 只有两种来源：

- 当前元素自己单独开一个新子数组，也就是 `x`
- 把前面的最优结果接上当前元素，也就是 `pre + x`

所以状态转移就是：

```java
pre = Math.max(x, pre + x);
```

然后我们再额外用一个变量记录遍历过程中出现过的最大值，就得到了最终答案。

### 用示例把这个过程走一遍

看示例 1：

```text
nums = [-2,1,-3,4,-1,2,1,-5,4]
```

我们从左到右看：

- 到 `-2` 时，`pre = -2`
- 到 `1` 时，前面的 `-2` 是拖累，不如重新开始，所以 `pre = 1`
- 到 `-3` 时，`pre = max(-3, 1 + -3) = -2`
- 到 `4` 时，前面的 `-2` 又成了拖累，所以 `pre = 4`
- 到 `-1` 时，`pre = 3`
- 到 `2` 时，`pre = 5`
- 到 `1` 时，`pre = 6`

这时候我们已经得到了最大的连续和 `6`，对应子数组就是：

```text
[4,-1,2,1]
```

你会发现，这个过程里最重要的不是“算和”，而是始终在判断：

**之前那一段到底还值不值得带着走。**

### Kadane 算法代码

```java
public int maxSubArray(int[] nums) {
    int pre = 0;
    int maxAns = nums[0];

    for (int x : nums) {
        pre = Math.max(pre + x, x);
        maxAns = Math.max(maxAns, pre);
    }

    return maxAns;
}
```

### 这段代码里最关键的一句

核心就是这一句：

```java
pre = Math.max(pre + x, x);
```

它的含义非常直接：

- `pre + x`：说明继续接着前面的子数组
- `x`：说明前面的不要了，从当前元素重新开始

而这两个里面谁更大，就说明“以当前元素结尾”的最优选择是谁。

所以这题本质上并不是在暴力找区间，而是在维护一个非常局部但很有用的信息：

**当前位置结尾的最优结果。**

全局答案，只是这个局部最优在遍历过程中出现过的最大值而已。

### 为什么初始化要写成这样

这里有个细节很容易写错。

很多人会把答案初始化成 `0`，但这在全负数组时会出问题。

比如：

```text
[-3, -2, -5]
```

最大子数组和应该是 `-2`，而不是 `0`。  
因为题目要求子数组至少包含一个元素，不能什么都不选。

所以正确写法应该是：

```java
int maxAns = nums[0];
```

这样无论数组里全是负数，还是有正有负，都不会出错。

### 为什么这题能用动态规划，但又没必要把它写得很重

严格来说，这题当然是动态规划。

因为我们定义了状态 `pre`，也写出了状态转移。  
但这题最好的地方就在于，它根本不需要开数组去存一整串 `dp[i]`。

因为 `dp[i]` 只依赖 `dp[i-1]`，所以前一个状态用一个变量就够了。

也就是说，这题虽然是 DP，但代码呈现出来其实非常轻：

- 一个变量维护当前位置结尾的最优值
- 一个变量维护全局答案

这也是为什么 Kadane 算法会这么经典。  
它把一个动态规划问题压缩到了常数空间。

### 进阶为什么会提分治法

题目最后提到，如果你已经会 `O(n)` 的解法，可以再试试分治。

这不是因为分治更优，而是因为它能训练你从另一个角度理解“最大子数组和”这个问题。

分治法会把一个区间拆成左右两半，然后思考：

对于一个区间 `[l, r]`，最大子数组和可能出现在哪？

其实只有三种情况：

- 完全在左半边
- 完全在右半边
- 横跨中点

所以只要我们能把左右子区间的信息总结出来，就能把整个区间的信息合并出来。

### 分治法为什么要维护四个量

对于一个区间，我们通常维护下面四个值：

- `iSum`：整个区间的总和
- `lSum`：从左端点开始的最大子数组和
- `rSum`：到右端点结束的最大子数组和
- `mSum`：这个区间里的最大子数组和

为什么是这四个？

因为当左右两个区间要合并时，跨中点的最大子数组和，正好可以由：

```text
左区间的 rSum + 右区间的 lSum
```

拼出来。

而剩下几个量，也都能通过左右子区间推出来，所以这四个值正好够用。

### 分治法代码

```java
class Status {
    int lSum, rSum, mSum, iSum;

    Status(int lSum, int rSum, int mSum, int iSum) {
        this.lSum = lSum;
        this.rSum = rSum;
        this.mSum = mSum;
        this.iSum = iSum;
    }
}

public int maxSubArrayDivideConquer(int[] nums) {
    return getInfo(nums, 0, nums.length - 1).mSum;
}

private Status getInfo(int[] nums, int left, int right) {
    if (left == right) {
        return new Status(nums[left], nums[left], nums[left], nums[left]);
    }

    int mid = left + (right - left) / 2;
    Status leftSub = getInfo(nums, left, mid);
    Status rightSub = getInfo(nums, mid + 1, right);

    return pushUp(leftSub, rightSub);
}

private Status pushUp(Status left, Status right) {
    int iSum = left.iSum + right.iSum;
    int lSum = Math.max(left.lSum, left.iSum + right.lSum);
    int rSum = Math.max(right.rSum, right.iSum + left.rSum);
    int mSum = Math.max(Math.max(left.mSum, right.mSum), left.rSum + right.lSum);

    return new Status(lSum, rSum, mSum, iSum);
}
```

### 这两种写法怎么选

如果只是正常做题，这题毫无疑问优先写 Kadane。

原因很简单：

- 时间复杂度已经是最优的 `O(n)`
- 空间复杂度是 `O(1)`
- 代码短，而且好解释

分治法更多是用来帮你理解这个问题还可以怎么拆。  
它在这题里不是主流解法，但它背后的区间信息维护思想很重要，后面做线段树或者区间 DP 时会经常遇到。

所以这题的选择非常明确：

- **面试/刷题主写 Kadane**
- **想拓展思路时再看分治**

### 复杂度分析

Kadane 算法：

- 时间复杂度：$O(n)$
- 空间复杂度：$O(1)$

分治法：

- 时间复杂度：$O(n)$
- 空间复杂度：$O(\log n)$，主要来自递归栈

虽然两者时间复杂度一样，但 Kadane 的常数更小、写法更简洁，所以实际使用里基本都会优先选它。

### 这题真正值得记住的地方

这题最值得记住的，不是“最大子数组和 = Kadane”这句结论本身。

真正值得记住的是这个判断：

**当你在维护一个连续区间时，如果前面的贡献已经变成负担，就应该果断丢掉。**

这是这题最核心的思维方式。

很多后续题目虽然形式不同，但底层都会有类似判断：

- 前面的状态还有没有保留价值
- 什么时候应该重新开始
- 什么时候应该把历史贡献全部清空

这才是这题真正有价值的地方。

### 完整代码

```java
import java.util.Arrays;

public class Main {

    public static int maxSubArray(int[] nums) {
        int pre = 0;
        int maxAns = nums[0];

        for (int x : nums) {
            pre = Math.max(pre + x, x);
            maxAns = Math.max(maxAns, pre);
        }

        return maxAns;
    }

    static class Status {
        int lSum, rSum, mSum, iSum;

        public Status(int lSum, int rSum, int mSum, int iSum) {
            this.lSum = lSum;
            this.rSum = rSum;
            this.mSum = mSum;
            this.iSum = iSum;
        }
    }

    public static int maxSubArrayDivideConquer(int[] nums) {
        return getInfo(nums, 0, nums.length - 1).mSum;
    }

    private static Status getInfo(int[] nums, int left, int right) {
        if (left == right) {
            return new Status(nums[left], nums[left], nums[left], nums[left]);
        }

        int mid = left + (right - left) / 2;
        Status leftSub = getInfo(nums, left, mid);
        Status rightSub = getInfo(nums, mid + 1, right);

        return pushUp(leftSub, rightSub);
    }

    private static Status pushUp(Status left, Status right) {
        int iSum = left.iSum + right.iSum;
        int lSum = Math.max(left.lSum, left.iSum + right.lSum);
        int rSum = Math.max(right.rSum, right.iSum + left.rSum);
        int mSum = Math.max(Math.max(left.mSum, right.mSum), left.rSum + right.lSum);

        return new Status(lSum, rSum, mSum, iSum);
    }

    public static void main(String[] args) {
        int[] nums1 = {-2, 1, -3, 4, -1, 2, 1, -5, 4};
        System.out.println("示例 1 输入: " + Arrays.toString(nums1));
        System.out.println("Kadane 输出: " + maxSubArray(nums1));
        System.out.println("分治输出: " + maxSubArrayDivideConquer(nums1));
        System.out.println("-----------------------------");

        int[] nums2 = {1};
        System.out.println("示例 2 输入: " + Arrays.toString(nums2));
        System.out.println("Kadane 输出: " + maxSubArray(nums2));
        System.out.println("分治输出: " + maxSubArrayDivideConquer(nums2));
        System.out.println("-----------------------------");

        int[] nums3 = {5, 4, -1, 7, 8};
        System.out.println("示例 3 输入: " + Arrays.toString(nums3));
        System.out.println("Kadane 输出: " + maxSubArray(nums3));
        System.out.println("分治输出: " + maxSubArrayDivideConquer(nums3));
    }
}
```
