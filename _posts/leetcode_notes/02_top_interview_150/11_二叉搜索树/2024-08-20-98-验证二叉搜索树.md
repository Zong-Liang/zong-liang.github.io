## 题目

**描述：**

给你一个二叉树的根节点 `root`，判断其是否是一个 **有效** 的二叉搜索树。

**有效** 二叉搜索树定义如下：

- 节点的左子树只包含 **严格小于** 当前节点的数。
- 节点的右子树只包含 **严格大于** 当前节点的数。
- 所有左子树和右子树自身必须也是二叉搜索树。

**示例 1:**

![示例1图片](https://assets.leetcode.com/uploads/2020/12/01/tree1.jpg)

- **输入:** `root = [2,1,3]`
- **输出:** `true`

**示例 2:**

![示例2图片](https://assets.leetcode.com/uploads/2020/12/01/tree2.jpg)

- **输入:** `root = [5,1,4,null,null,3,6]`
- **输出:** `false`
- **解释:** 根节点的值是 5，但是右子节点的值是 4。

**提示:**

- 树中节点数目范围在 `[1, 10^4]` 内
- `-2^31 <= Node.val <= 2^31 - 1`

## 题解

### 这题最容易想错的地方，不是“怎么遍历”，而是“怎么理解 BST”

很多人一看到“验证二叉搜索树”，第一反应会写成这样：

- 当前节点比左孩子大
- 当前节点比右孩子小

然后觉得这样就够了。

其实这还不够。

因为二叉搜索树的要求不是只和“直接相连的左右孩子”比较，而是：

- 左子树里 **所有节点** 都要严格小于当前节点
- 右子树里 **所有节点** 都要严格大于当前节点

注意这里的关键词是“所有节点”。

也就是说，问题的难点不在遍历本身，而在于你有没有真正理解 BST 的定义。

### 为什么只比较左右孩子是不够的

看一个经典反例：

```text
    5
   / \
  1   4
     / \
    3   6
```

如果你只看每个节点和它的左右孩子：

- `5 > 1`
- `5 > 4`？这里其实已经不对了，但很多人写局部判断时只会继续往子树里看
- `4 > 3`
- `4 < 6`

表面上局部关系看着都还行，但整棵树并不是合法 BST。

原因在于：  
节点 `4` 在根节点 `5` 的右子树里，所以它本来应该 **大于 5**，可实际上 `4 < 5`，这就已经违背 BST 定义了。

所以这题真正要验证的，不是局部大小关系，而是整棵子树范围是否一直合法。

### 第一种思路：利用中序遍历结果应该严格递增

二叉搜索树有一个非常重要的性质：

**如果一棵树是合法 BST，那么它的中序遍历结果一定是严格递增的。**

这里要注意两个字：**严格**。

也就是说，如果中序遍历过程中出现了下面任意一种情况，都说明它不是合法 BST：

- 当前值小于前一个值
- 当前值等于前一个值

因为题目要求左子树节点必须严格小于根节点，右子树节点必须严格大于根节点，所以重复值也不允许出现。

这样一来，这题就可以转化成一个更直接的问题：

**中序遍历这棵树时，得到的序列是不是严格递增。**

### 为什么这个思路成立

因为中序遍历顺序是：

```text
左 -> 中 -> 右
```

而 BST 恰好满足：

- 左边都更小
- 中间是当前节点
- 右边都更大

所以只要树是合法 BST，中序遍历出来的结果就一定像一个严格升序数组。

反过来，如果中序遍历过程中出现了不递增的地方，那就说明至少有某个节点破坏了 BST 的大小关系。

所以判断“是否合法”，其实就变成了判断“中序结果是否严格递增”。

### 这题真正要维护的东西

既然只需要判断中序序列是不是严格递增，那我们根本没必要把整个序列都存下来。

只需要在遍历过程中维护一个变量：

- `pre`：记录中序遍历时上一个访问到的节点值

每访问当前节点时，只做一件事：

- 如果当前值 `<= pre`，直接返回 `false`
- 否则更新 `pre = 当前值`

整棵树遍历完都没有出问题，那就是合法 BST。

### 代码

```java
long pre = Long.MIN_VALUE;

public boolean isValidBST(TreeNode root) {
    pre = Long.MIN_VALUE;
    return inorder(root);
}

private boolean inorder(TreeNode node) {
    if (node == null) return true;

    if (!inorder(node.left)) {
        return false;
    }

    if (node.val <= pre) {
        return false;
    }
    pre = node.val;

    return inorder(node.right);
}
```

### 这里为什么要用 `long`

这个细节要注意一下。

题目里节点值范围是：

```text
-2^31 <= Node.val <= 2^31 - 1
```

也就是说，节点值有可能刚好等于 `Integer.MIN_VALUE`。

如果你用 `int` 类型的最小值去初始化 `pre`，那当树里第一个访问到的节点正好也是 `Integer.MIN_VALUE` 时，比较逻辑就会出问题。

所以这里更稳妥的做法是用：

```java
long pre = Long.MIN_VALUE;
```

这样边界值就不会撞上。

### 这段代码最关键的地方

关键其实就在这一句：

```java
if (node.val <= pre) {
    return false;
}
```

因为题目要求的是**严格递增**，不是非递减。  
所以只要当前节点值小于等于前一个节点值，就已经不合法了。

这里很多人会不小心写成 `<`，那就会把有重复值的情况误判成合法 BST，这是不对的。

### 第二种思路：直接按定义递归检查区间范围

虽然中序遍历法已经很好用了，但这题还有一种非常经典、也更贴定义的写法：

**给每个节点一个合法取值区间。**

什么意思？

如果当前节点在某个区间 `(lower, upper)` 内才算合法，那么：

- 它的左子树所有节点都必须落在 `(lower, 当前节点值)` 里
- 它的右子树所有节点都必须落在 `(当前节点值, upper)` 里

这样一路递归下去，就能把祖先节点的约束一层层传递给后代节点。

这其实比“只看左右孩子”更接近 BST 的真正定义。

### 为什么区间法能解决那个经典反例

还是刚才那棵树：

```text
    5
   / \
  1   4
     / \
    3   6
```

当我们走到节点 `4` 时，它位于节点 `5` 的右子树中，所以它的合法范围应该是：

```text
(5, +∞)
```

但 `4` 明显不在这个范围里，所以可以直接判定这棵树不是 BST。

这就是区间法厉害的地方：

**它不会只盯着当前节点和父节点，而是会把祖先节点的限制一直传下来。**

### 区间法怎么写

定义一个递归函数：

```java
validate(node, lower, upper)
```

表示当前这棵以 `node` 为根的子树，所有节点值都必须严格落在 `(lower, upper)` 这个开区间里。

递归逻辑就是：

1. 如果节点为空，返回 `true`
2. 如果当前节点值不在合法范围内，返回 `false`
3. 递归检查左子树，更新上界为当前节点值
4. 递归检查右子树，更新下界为当前节点值

### 区间法代码

```java
public boolean isValidBST(TreeNode root) {
    return validate(root, Long.MIN_VALUE, Long.MAX_VALUE);
}

private boolean validate(TreeNode node, long lower, long upper) {
    if (node == null) return true;

    if (node.val <= lower || node.val >= upper) {
        return false;
    }

    return validate(node.left, lower, node.val)
        && validate(node.right, node.val, upper);
}
```

### 这两种写法怎么选

如果是正常做题，我觉得两种都值得会。

但如果只说“第一次写这题，哪种更容易写稳”，我会更偏向下面这个判断：

- **中序遍历法** 更容易和 BST 的性质联系起来，代码也短
- **区间法** 更贴近 BST 的定义，思路更本质

如果你已经很熟悉“BST 中序结果严格递增”这个性质，那中序法会写得很顺。

如果你想真正把“为什么局部比较不够”这件事想透，那区间法更值得掌握。因为它直接对应题目定义，没有任何中间转换。

### 复杂度分析

无论是中序遍历法，还是区间递归法，每个节点都只访问一次，所以时间复杂度都是 $O(n)$，其中 $n$ 是树中节点总数。

空间复杂度都和树的高度有关：

- 平均情况下是 $O(\log n)$
- 最坏情况下树退化成链表，会变成 $O(n)$

主要消耗都来自递归调用栈。

### 这题真正值得记住的地方

这题最值得记住的，不是“再背一个验证 BST 模板”，而是下面这两个判断：

第一，**验证 BST 不能只看当前节点和左右孩子。**  
真正要满足的是整棵左子树都小于当前节点，整棵右子树都大于当前节点。

第二，**BST 相关题，优先想两个方向：**

- 中序遍历之后是否有序
- 当前节点是否始终落在合法区间内

很多 BST 题，本质上都逃不开这两个切入口。

### 完整代码

```java
public class Main {

    public static class TreeNode {
        int val;
        TreeNode left;
        TreeNode right;

        TreeNode() {}

        TreeNode(int val) {
            this.val = val;
        }

        TreeNode(int val, TreeNode left, TreeNode right) {
            this.val = val;
            this.left = left;
            this.right = right;
        }
    }

    static long preValue = Long.MIN_VALUE;

    public static boolean isValidBST(TreeNode root) {
        preValue = Long.MIN_VALUE;
        return inorder(root);
    }

    private static boolean inorder(TreeNode node) {
        if (node == null) {
            return true;
        }

        if (!inorder(node.left)) {
            return false;
        }

        if (node.val <= preValue) {
            return false;
        }
        preValue = node.val;

        return inorder(node.right);
    }

    public static boolean isValidBSTRange(TreeNode root) {
        return validate(root, Long.MIN_VALUE, Long.MAX_VALUE);
    }

    private static boolean validate(TreeNode node, long lower, long upper) {
        if (node == null) {
            return true;
        }

        if (node.val <= lower || node.val >= upper) {
            return false;
        }

        return validate(node.left, lower, node.val)
            && validate(node.right, node.val, upper);
    }

    public static void main(String[] args) {
        TreeNode root1 = new TreeNode(2);
        root1.left = new TreeNode(1);
        root1.right = new TreeNode(3);

        System.out.println("示例 1 输入: [2,1,3]");
        System.out.println("输出（中序法）: " + isValidBST(root1));
        System.out.println("输出（区间法）: " + isValidBSTRange(root1));

        TreeNode root2 = new TreeNode(5);
        root2.left = new TreeNode(1);
        root2.right = new TreeNode(4);
        root2.right.left = new TreeNode(3);
        root2.right.right = new TreeNode(6);

        System.out.println("示例 2 输入: [5,1,4,null,null,3,6]");
        System.out.println("输出（中序法）: " + isValidBST(root2));
        System.out.println("输出（区间法）: " + isValidBSTRange(root2));

        TreeNode root3 = new TreeNode(Integer.MAX_VALUE);

        System.out.println("示例 3 输入: [2147483647]");
        System.out.println("输出（中序法）: " + isValidBST(root3));
        System.out.println("输出（区间法）: " + isValidBSTRange(root3));
    }
}
```
