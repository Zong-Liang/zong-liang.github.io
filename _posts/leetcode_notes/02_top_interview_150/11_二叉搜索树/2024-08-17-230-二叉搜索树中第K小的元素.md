## 题目

**描述：**

给定一个二叉搜索树的根节点 `root`，和一个整数 `k`，请你设计一个算法查找其中第 `k` 小的元素（从 1 开始计数）。

**示例 1:**

![示例1图片](https://assets.leetcode.com/uploads/2021/01/28/kthtree1.jpg)

- **输入:** `root = [3,1,4,null,2]`, `k = 1`
- **输出:** 1

**示例 2:**

![示例2图片](https://assets.leetcode.com/uploads/2021/01/28/kthtree2.jpg)

- **输入:** `root = [5,3,6,2,4,null,null,1]`, `k = 3`
- **输出:** 3

**提示:**

- 树中的节点数为 `n`。
- `1 <= k <= n <= 10^4`
- `0 <= Node.val <= 10^4`

**进阶：** 如果二叉搜索树经常被修改（插入/删除操作）并且你需要频繁地查找第 `k` 小的值，你将如何优化算法？

## 题解

### 这题的关键，还是先抓住 BST 的性质

这题表面上是在问“第 `k` 小的元素”，但真正决定解法的，不是 `k`，而是这棵树是一棵 **二叉搜索树**。

二叉搜索树最重要的性质就是：

- 左子树所有节点值都小于当前节点
- 右子树所有节点值都大于当前节点

而这个性质一旦和 **中序遍历** 放在一起，就会得到一个直接结论：

**二叉搜索树的中序遍历结果就是一个递增序列。**

所以这题其实可以换一种说法来理解：

**在一组已经排好序的数里，找第 `k` 个。**

这样一来，思路就清楚很多了。

### 为什么中序遍历正好能解决这题

既然中序遍历得到的是递增序列，那么第 1 个访问到的节点就是最小值，第 2 个访问到的节点就是第 2 小的值，第 3 个访问到的节点就是第 3 小的值……

也就是说，我们根本不需要把所有节点值都取出来排个序。  
只要按中序遍历的顺序访问节点，同时数一数当前访问到了第几个节点，就够了。

一旦数到第 `k` 个，当前节点值就是答案，可以直接返回。

这就是这题最核心的思路。

### 这题真正要做的事情

整道题其实就两步：

1. 按中序遍历的顺序访问 BST
2. 访问到一个节点，就把计数减 1；当 `k` 减到 0 时，说明当前节点就是第 `k` 小的元素

所以重点并不是“怎么找第 `k` 小”，而是：

**怎么稳定地按 BST 的升序顺序访问节点。**

而这件事，中序遍历天然就会帮我们做好。

### 为什么这里更适合用迭代写法

这题用递归当然也能做，但迭代写法有一个很实际的好处：

**可以在找到答案之后立刻停下来。**

如果你先递归遍历完整棵树，再把结果存到列表里，当然也能做对，但会多做很多没必要的工作。

而用栈模拟中序遍历时，我们是一边遍历、一边计数：

- 一路往左走，把沿途节点入栈
- 弹出当前最小的那个节点
- 看看是不是第 `k` 个
- 如果不是，再转向它的右子树

这样当答案出现时，直接返回就行，不需要把整棵树都走完。

### 具体怎么做

准备一个栈，专门用来模拟中序遍历。

然后重复下面的过程：

1. 从当前节点开始，一路向左走，把沿途所有节点都压入栈中
2. 弹出栈顶节点，这个节点就是当前还没访问过的最小节点
3. 访问它，把 `k` 减 1
4. 如果这时 `k == 0`，说明当前节点就是第 `k` 小的值，直接返回
5. 否则转向这个节点的右子树，继续同样的过程

整个流程其实就是标准的中序遍历，只不过多了一步计数。

### 用示例看一遍

比如示例 2：

```text
root = [5,3,6,2,4,null,null,1], k = 3
```

这棵 BST 中序遍历的结果是：

```text
[1,2,3,4,5,6]
```

题目问第 3 小的元素，那答案显然就是 `3`。

而在代码里，我们不需要真的把这个序列完整存下来。  
只要按中序顺序依次访问：

- 访问到 `1`，`k` 变成 `2`
- 访问到 `2`，`k` 变成 `1`
- 访问到 `3`，`k` 变成 `0`

这时直接返回 `3` 就可以了。

### 代码

```java
public int kthSmallest(TreeNode root, int k) {
    Deque<TreeNode> stack = new ArrayDeque<>();

    while (root != null || !stack.isEmpty()) {
        while (root != null) {
            stack.push(root);
            root = root.left;
        }

        root = stack.pop();
        k--;

        if (k == 0) {
            return root.val;
        }

        root = root.right;
    }

    return -1;
}
```

### 这段代码最关键的地方

关键其实是下面这几步连在一起理解：

```java
while (root != null) {
    stack.push(root);
    root = root.left;
}
```

这一步的作用，是把当前子树里最左边那条路径全部压栈。  
因为在 BST 里，越往左值越小，所以这样做之后，栈顶元素就是当前还没访问到的最小节点。

接着：

```java
root = stack.pop();
k--;
```

弹出的这个节点，就是当前序列里的下一个最小值。  
每弹出一个，说明我们又按升序访问到了一个节点，所以 `k` 就减 1。

当 `k == 0` 的时候，说明当前节点正好是第 `k` 小的元素。

所以这题本质上是在做一件很简单的事：

**按升序一个一个取数，取到第 `k` 个就停。**

### 为什么不需要真的把所有节点存进数组

当然也可以先中序遍历，把结果存到列表里，然后返回 `list.get(k - 1)`。

这种写法也没问题，但它其实没有利用好这题的特点。

因为题目只要第 `k` 小的元素，不需要完整有序序列。  
既然如此，那就没必要把所有节点都存下来。访问到第 `k` 个时直接返回，会更省事，也更符合这题本身的要求。

### 进阶问题该怎么理解

题目最后问了一句：

> 如果二叉搜索树经常被修改，并且需要频繁查找第 `k` 小的值，怎么优化？

这句话的意思其实是：  
如果查询只有一两次，那每次中序遍历一下问题不大；但如果树会频繁插入、删除，而且还要反复查询第 `k` 小`，那每次都重新遍历，代价就高了。

这时候，常见做法是在每个节点里额外维护一个信息：

**以这个节点为根的子树一共有多少个节点。**

有了这个字段之后，查第 `k` 小的过程就会很像二分查找。

### 为什么维护子树大小之后会更快

假设当前来到某个节点 `curr`，它左子树的节点数是 `leftSize`。

那就分三种情况：

- 如果 `k == leftSize + 1`，说明当前节点正好就是第 `k` 小的，直接返回
- 如果 `k <= leftSize`，说明答案一定在左子树里，继续往左找
- 如果 `k > leftSize + 1`，说明答案在右子树里，但要把左子树和当前节点这部分数量扣掉，也就是去右子树里找第 `k - leftSize - 1` 小的元素

这样每次都能明确缩小范围，查询效率就会高很多。

代价是：  
插入和删除节点时，你得顺手把沿途节点的子树大小也维护好。

所以这是一种典型的“空间换时间”。

### 进阶写法的逻辑示意

```java
public int kthSmallest(TreeNode root, int k) {
    TreeNode curr = root;

    while (curr != null) {
        int leftSize = (curr.left != null) ? curr.left.size : 0;

        if (k == leftSize + 1) {
            return curr.val;
        } else if (k <= leftSize) {
            curr = curr.left;
        } else {
            k = k - leftSize - 1;
            curr = curr.right;
        }
    }

    return -1;
}
```

这段代码的重点不在于你现在就得把它背下来，而在于你要知道：

**如果查询很多次，就不要每次都从头遍历；可以考虑给节点增加额外信息，换取更快的查询。**

### 这题该怎么选写法

如果只是在 LeetCode 正常做题，直接写迭代版中序遍历就够了。

因为它有几个优点：

- 思路直接
- 不需要额外存整个有序序列
- 找到答案就能立刻返回
- 代码也不长

进阶那部分更适合放在“数据结构设计”这个层面去理解，而不是这题的常规解法。

### 复杂度分析

对于普通的中序遍历解法：

- 时间复杂度是 $O(H + k)$  
  其中 `H` 是树的高度。我们先一路往左走到最小值，然后按中序顺序访问到第 `k` 个节点就停止了。  
  最坏情况下树退化成链表，复杂度会变成 $O(n)$)。

- 空间复杂度是 $O(H)$  
  主要来自栈中存储的节点数量。

如果是进阶做法，也就是每个节点维护子树大小，那么单次查询可以做到 $O(H)$，对于平衡 BST 来说通常就是 $O(\log n)$。

### 这题真正值得记住的地方

这题最值得记住的，不是“第 `k` 小”这个表面问法，而是下面这个转换：

**只要题目给的是二叉搜索树，又和大小顺序有关，就优先想中序遍历。**

因为 BST 的很多题，本质上都能转成“有序序列上的问题”：

- 第 `k` 小
- 最小绝对差
- 众数
- 是否递增有序

一旦你能主动想到这一步，题目往往会简单很多。

### 完整代码

```java
import java.util.ArrayDeque;
import java.util.Deque;
import java.util.LinkedList;
import java.util.Queue;

class TreeNode {
    int val;
    TreeNode left;
    TreeNode right;

    TreeNode() {}

    TreeNode(int val) {
        this.val = val;
    }

    TreeNode(int val, TreeNode left, TreeNode right) {
        this.val = val;
        this.left = left;
        this.right = right;
    }
}

public class Main {

    public static int kthSmallest(TreeNode root, int k) {
        Deque<TreeNode> stack = new ArrayDeque<>();
        TreeNode curr = root;

        while (curr != null || !stack.isEmpty()) {
            while (curr != null) {
                stack.push(curr);
                curr = curr.left;
            }

            curr = stack.pop();
            k--;

            if (k == 0) {
                return curr.val;
            }

            curr = curr.right;
        }

        return -1;
    }

    public static TreeNode buildTree(Integer[] arr) {
        if (arr.length == 0) return null;

        TreeNode root = new TreeNode(arr[0]);
        Queue<TreeNode> queue = new LinkedList<>();
        queue.offer(root);

        int i = 1;
        while (i < arr.length) {
            TreeNode node = queue.poll();

            if (i < arr.length && arr[i] != null) {
                node.left = new TreeNode(arr[i]);
                queue.offer(node.left);
            }
            i++;

            if (i < arr.length && arr[i] != null) {
                node.right = new TreeNode(arr[i]);
                queue.offer(node.right);
            }
            i++;
        }

        return root;
    }

    public static void main(String[] args) {
        Integer[] nums1 = {3, 1, 4, null, 2};
        TreeNode root1 = buildTree(nums1);
        int k1 = 1;

        System.out.println("示例 1 输入: root = [3,1,4,null,2], k = 1");
        System.out.println("示例 1 输出: " + kthSmallest(root1, k1));

        Integer[] nums2 = {5, 3, 6, 2, 4, null, null, 1};
        TreeNode root2 = buildTree(nums2);
        int k2 = 3;

        System.out.println("示例 2 输入: root = [5,3,6,2,4,null,null,1], k = 3");
        System.out.println("示例 2 输出: " + kthSmallest(root2, k2));
    }
}
```
