## 题目

**描述：**

编写一个算法来判断一个数 `n` 是不是快乐数。

**「快乐数」** 定义为：

- 对于一个正整数，每一次将该数替换为它每个位置上的数字的平方和。
- 然后重复这个过程直到这个数变为 1，也可能是 **无限循环** 但始终变不到 1。
- 如果这个过程 **结果为 1**，那么这个数就是快乐数。

如果 `n` 是快乐数就返回 `true` ；不是，则返回 `false` 。

**示例 1:**

- **输入:** `n = 19`
- **输出:** `true`
- **解释:**
  - 1² + 9² = 82
  - 8² + 2² = 68
  - 6² + 8² = 100
  - 1² + 0² + 0² = 1

**示例 2:**

- **输入:** `n = 2`
- **输出:** `false`

**提示:**

- `1 <= n <= 2^31 - 1`

## 题解

### 先看这题到底在反复做什么

这题的规则其实很简单：

给一个正整数，每次把它替换成“各位数字平方和”，然后继续重复这个过程。

比如 `19`：

```text
1² + 9² = 82
8² + 2² = 68
6² + 8² = 100
1² + 0² + 0² = 1
```

所以题目真正要问的是：

**这个过程最终会不会走到 1。**

如果能走到 `1`，它就是快乐数；  
如果永远走不到 `1`，那它迟早会在某些数字之间来回打转。

也就是说，这题本质上只有两种结局：

- 最后变成 `1`
- 掉进循环

真正的关键，不是怎么算“下一步”，而是怎么判断它有没有进入循环。

### 先把“下一步怎么求”单独拆出来

无论后面用什么方法判断，第一步都得先会算一个数的“下一状态”。

做法很直接：

- 每次取出个位数字
- 平方后加入总和
- 去掉个位，继续处理下一位

比如对 `82`：

- 取 `2`，平方后加 `4`
- 再取 `8`，平方后加 `64`
- 最终得到 `68`

这一步建议单独写成一个辅助函数，因为后面会反复用到。

```java
private int getNext(int n) {
    int sum = 0;
    while (n > 0) {
        int digit = n % 10;
        n /= 10;
        sum += digit * digit;
    }
    return sum;
}
```

这段代码本身不难，真正需要往下想的是：

**一个数不断做这个变换，到底怎么判断它会不会陷入死循环。**

### 最容易想到的办法：用集合记录出现过的数字

既然问题在于“会不会重复”，那最自然的办法就是把已经出现过的数字记下来。

处理过程可以这样理解：

- 当前数字不是 `1`，就继续往下算
- 每算出一个新数字，先看它以前有没有出现过
- 如果以前出现过，说明已经绕回来了，后面只会继续重复，直接返回 `false`
- 如果没出现过，就把它加入集合，继续计算下一个数

这就是最直观的判环方式。

### 为什么这样一定能判断出来

因为这个过程不可能无限变大。

虽然初始输入 `n` 可能很大，但各位平方和很快就会掉进一个比较小的范围里。  
比如一个 10 位数，即使每一位都是 `9`，下一步的最大值也只是：

```text
10 * 9² = 810
```

也就是说，不管一开始多大，算几轮之后，数字都会落到一个有限的小范围内。  
在这个有限范围里继续反复变换，不可能一直不重复。

所以最后一定只有两种情况：

- 遇到 `1`
- 遇到之前已经出现过的数字

这也正是哈希集合解法成立的原因。

### 用示例走一遍

看示例 1：

```text
n = 19
```

过程是：

```text
19 -> 82 -> 68 -> 100 -> 1
```

一路往下算，从来没有重复过，而且最后到了 `1`，所以它是快乐数。

再看示例 2：

```text
n = 2
```

往下推：

```text
2 -> 4 -> 16 -> 37 -> 58 -> 89 -> 145 -> 42 -> 20 -> 4
```

这里又回到了 `4`。  
既然 `4` 已经出现过了，说明从这一刻开始，后面的过程会无限重复，永远不可能到 `1`，所以返回 `false`。

这题的判断核心其实就是这一句：

**只要某个中间结果第二次出现，就说明进入了环。**

### 先保留一个最容易理解的写法

```java
public boolean isHappy(int n) {
    Set<Integer> seen = new HashSet<>();

    while (n != 1 && !seen.contains(n)) {
        seen.add(n);
        n = getNext(n);
    }

    return n == 1;
}

private int getNext(int n) {
    int sum = 0;
    while (n > 0) {
        int digit = n % 10;
        n /= 10;
        sum += digit * digit;
    }
    return sum;
}
```

这个写法的好处是非常直观。  
它几乎就是把题目的过程原样翻译成了代码：

- 不是 `1`
- 也没重复过
- 那就继续算下一轮

如果读者第一次接触这题，这个版本是最容易理解的。

### 这题其实还能换一种眼光来看

如果把每个数字都看成一个节点，把“各位平方和”看成“指向下一个节点”的规则，那整个过程其实很像一条链：

```text
n -> getNext(n) -> getNext(getNext(n)) -> ...
```

这样一来，问题就变成了：

**这条链最后会走到 1，还是会进入一个环。**

这和链表判环其实是同一个模型。

既然是判环，自然就能想到快慢指针。

### 为什么快慢指针在这里也能用

快慢指针通常用来判断链表有没有环。  
这题虽然不是链表，但“每个数都唯一指向下一个数”这件事，本质上就是在构造一条隐式链表。

所以可以这样设：

- `slow` 每次走一步：`slow = getNext(slow)`
- `fast` 每次走两步：`fast = getNext(getNext(fast))`

那么只会出现两种结果：

- 如果这条链最终能到 `1`，那 `fast` 会先碰到 `1`
- 如果这条链进了环，`fast` 和 `slow` 迟早会在环里相遇

这就是 Floyd 判圈算法在这题里的应用。

### 快慢指针写法为什么更省空间

和哈希集合相比，快慢指针不需要记录所有出现过的数字。  
它只维护两个变量：

- 一个慢指针
- 一个快指针

所以空间复杂度能从“记录一串状态”降到常数级。

这也是这题更常被拿来当标准解的原因。

### 推荐掌握的写法

```java
public boolean isHappy(int n) {
    int slow = n;
    int fast = getNext(n);

    while (fast != 1 && slow != fast) {
        slow = getNext(slow);
        fast = getNext(getNext(fast));
    }

    return fast == 1;
}

private int getNext(int n) {
    int sum = 0;
    while (n > 0) {
        int digit = n % 10;
        n /= 10;
        sum += digit * digit;
    }
    return sum;
}
```

### 这段代码到底在做什么

先看初始化：

```java
int slow = n;
int fast = getNext(n);
```

相当于让快指针先比慢指针多走一步。

然后进入循环：

```java
while (fast != 1 && slow != fast)
```

只有两种情况会停下来：

- `fast == 1`：说明已经找到快乐数了
- `slow == fast`：说明快慢指针相遇，进入了环

循环体里：

```java
slow = getNext(slow);
fast = getNext(getNext(fast));
```

慢指针每次走一步，快指针每次走两步。

如果最后是因为 `fast == 1` 退出，返回 `true`；  
否则就是相遇在环里，返回 `false`。

### 这题最容易想偏的地方

这题不难，但有几个点很容易一开始没想透。

#### 第一个误区：以为数字可能一直变大

实际上不会。  
各位平方和会迅速把数字压到一个较小范围里，所以最终只可能到 `1` 或进入循环。

#### 第二个误区：把它当普通数学模拟题写到底

如果只是机械地一轮轮算下去，但没有“判环”的意识，就很容易写成一个没有停机保障的循环。

这题真正的核心不是模拟，而是：

**模拟 + 判断是否重复。**

#### 第三个误区：觉得快慢指针只能用在链表里

这题其实很好地说明了一点：  
只要问题能抽象成“每个状态唯一指向下一个状态”的过程，就可以把它当成隐式链表，再考虑判环。

### 复杂度怎么理解

设数字 `n` 的位数是 `k`。

每次调用 `getNext(n)`，都需要把这个数按位拆开，所以单次计算大约是 `O(k)`。

但由于序列很快就会落到一个小范围里，后续链条长度其实非常有限。  
所以整体时间复杂度通常可以看作与按位处理相关，实际运行也很快。

对于两种写法：

- 哈希集合法的空间复杂度不是常数，因为要存已经出现过的状态
- 快慢指针法只用两个变量，空间复杂度是 `O(1)`

因此如果是作为最终答案，通常更推荐快慢指针版本。

### 这题真正值得记住的地方

这题真正值得记住的，不是“快乐数”这个名字，而是这种抽象方式：

**当一个问题表现为“当前状态会唯一地变成下一个状态”时，就可以把整个过程看成一条链，再去想它是终止，还是成环。**

本题里：

- 状态是当前数字
- 转移规则是“各位平方和”
- 终点是 `1`
- 失败情况是进入循环

把这一层抽象出来之后，这题就不只是一个数字模拟题了，而是一道很典型的判环题。

### 完整代码

```java
import java.util.HashSet;
import java.util.Set;

public class Main {

    private static int getNext(int n) {
        int sum = 0;
        while (n > 0) {
            int digit = n % 10;
            n /= 10;
            sum += digit * digit;
        }
        return sum;
    }

    public static boolean isHappy(int n) {
        int slow = n;
        int fast = getNext(n);

        while (fast != 1 && slow != fast) {
            slow = getNext(slow);
            fast = getNext(getNext(fast));
        }

        return fast == 1;
    }

    public static boolean isHappySet(int n) {
        Set<Integer> seen = new HashSet<>();

        while (n != 1 && !seen.contains(n)) {
            seen.add(n);
            n = getNext(n);
        }

        return n == 1;
    }

    public static void main(String[] args) {
        int n1 = 19;
        System.out.println("示例 1 输入: n = " + n1);
        System.out.println("示例 1 输出: " + isHappy(n1));
        System.out.println("-----------------------------");

        int n2 = 2;
        System.out.println("示例 2 输入: n = " + n2);
        System.out.println("示例 2 输出: " + isHappy(n2));
    }
}
```
