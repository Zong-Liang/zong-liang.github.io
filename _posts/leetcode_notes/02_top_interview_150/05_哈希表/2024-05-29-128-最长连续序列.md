## 题目

**描述：**

给定一个未排序的整数数组 `nums` ，找出数字连续的最长序列（不要求序列元素在原数组中连续）的长度。

请你设计并实现时间复杂度为 `O(n)` 的算法解决此问题。

**示例 1:**

- **输入:** `nums = [100,4,200,1,3,2]`
- **输出:** 4
- **解释:** 最长数字连续序列是 `[1, 2, 3, 4]`。它的长度为 4。

**示例 2:**

- **输入:** `nums = [0,3,7,2,5,8,4,6,0,1]`
- **输出:** 9

**示例 3:**

- **输入:** `nums = [1,0,1,2]`
- **输出:** 3

**提示:**

- `0 <= nums.length <= 10^5`
- `-10^9 <= nums[i] <= 10^9`

## 题解

### 先看这题到底在找什么

这题要找的不是“数组里连续出现的一段”，而是**数值上连续**的一段。

比如：

```text
[100,4,200,1,3,2]
```

虽然 `1,2,3,4` 在原数组里并不是挨着放的，但它们的值是连续的，所以答案是 `4`。

也就是说，这题和“最长连续子数组”完全不是一回事。  
这里不关心原下标顺序，只关心一个数字的前一个、后一个值在不在数组里。

这一点先想清楚，后面思路才不会跑偏。

### 最自然的想法是排序，但题目偏偏不让你停在这里

如果只是先求一个能过的解法，那排序几乎是本能反应。

数组一旦排好序，所有数值连续的元素就会排在一起。接下来只要从左到右扫一遍：

- 如果当前数和前一个数相同，说明是重复值，跳过
- 如果当前数刚好等于前一个数加一，说明连续段还在延长
- 否则，说明连续段断了，重新开始统计

这个思路非常直观，也很适合作为第一反应。

问题在于，排序的时间复杂度是 `O(n log n)`，而题目明确要求 `O(n)`。  
所以排序法可以帮助我们理解题意，但不是这题真正想考的东西。

### 先保留一个最直观的写法

```java
public int longestConsecutive(int[] nums) {
    if (nums.length == 0) {
        return 0;
    }

    Arrays.sort(nums);

    int longest = 1;
    int currentStreak = 1;

    for (int i = 1; i < nums.length; i++) {
        if (nums[i] == nums[i - 1]) {
            continue;
        }

        if (nums[i] == nums[i - 1] + 1) {
            currentStreak++;
        } else {
            longest = Math.max(longest, currentStreak);
            currentStreak = 1;
        }
    }

    return Math.max(longest, currentStreak);
}
```

这个版本的好处是思路平。  
但它真正的瓶颈也很明确：排序。

所以接下来关键不是怎么把遍历写得更花，而是怎么**在不排序的情况下，仍然快速知道某个数的前后邻居是否存在**。

### 真正的突破口：把“找相邻数字”变成“查集合”

题目要求的是数值连续。  
那对任意一个数字 `x` 来说，我们真正关心的其实只有两件事：

- `x - 1` 在不在
- `x + 1` 在不在

如果能很快回答这两个问题，这题就会顺很多。

而哈希集合 `HashSet` 恰好最适合做这种事情。  
因为它能在平均 `O(1)` 的时间里判断一个数是否存在。

所以这题的核心转变就是：

**不要再想怎么把数组排好，而是把所有数字先丢进一个集合里，然后直接查。**

### 但只会查还不够，还得避免重复计算

如果只是把所有数放进集合，然后对每个数都一路往后查：

- `1` 查 `2,3,4`
- `2` 再查 `3,4`
- `3` 再查 `4`

虽然也能做出来，但会有大量重复计算。

所以这题真正关键的一步，不只是“用集合查”，而是：

**只从连续序列的起点开始往后扩。**

那什么叫起点？

如果一个数 `x` 的前一个数 `x - 1` 不在集合里，那它就是某段连续序列的起点。  
反过来，如果 `x - 1` 在集合里，那它就只是某段序列中间的一个元素，没必要从它重新开始数。

这一步想明白，整题就通了。

### 为什么“只从起点开始”能避免重复

还是拿序列 `1,2,3,4` 来看。

如果你从每个数都开始扩展，那会这样：

- 从 `1` 开始，查到 `2,3,4`
- 从 `2` 开始，又查到 `3,4`
- 从 `3` 开始，再查到 `4`

很浪费。

但如果加一个条件：

```java
if (!set.contains(num - 1))
```

那就只有 `1` 会进入扩展逻辑。因为：

- `1 - 1 = 0` 不在集合里，所以 `1` 是起点
- `2 - 1 = 1` 在集合里，所以 `2` 不是起点
- `3 - 1 = 2` 在集合里，所以 `3` 不是起点
- `4 - 1 = 3` 在集合里，所以 `4` 不是起点

这样一来，每一段连续序列只会被完整计算一次。

这就是这题能做到 `O(n)` 的真正原因。

### 用示例走一遍过程

看示例 1：

```text
nums = [100,4,200,1,3,2]
```

先把所有数字放进集合：

```text
{100,4,200,1,3,2}
```

接下来遍历集合里的每个数。

#### 看 `100`

先判断 `99` 在不在集合里。  
不在，所以 `100` 是一个起点。

从 `100` 开始往后查：

- `101` 不在

所以这一段长度是 `1`。

#### 看 `4`

先判断 `3` 在不在集合里。  
在，所以 `4` 不是起点，跳过。

#### 看 `200`

先判断 `199` 在不在集合里。  
不在，所以 `200` 是起点。

往后查：

- `201` 不在

这一段长度也是 `1`。

#### 看 `1`

先判断 `0` 在不在集合里。  
不在，所以 `1` 是起点。

从 `1` 开始往后查：

- `2` 在
- `3` 在
- `4` 在
- `5` 不在

所以这一段长度是 `4`。

这时更新答案为 `4`。

后面的 `2`、`3` 因为前一个数都存在，所以都不是起点，直接跳过。  
最终答案就是 `4`。

这个过程里最重要的一点是：

**不是每个数都值得往后查，只有序列起点才值得。**

### 真正推荐掌握的写法

```java
public int longestConsecutive(int[] nums) {
    Set<Integer> numSet = new HashSet<>();
    for (int num : nums) {
        numSet.add(num);
    }

    int longestStreak = 0;

    for (int num : numSet) {
        if (!numSet.contains(num - 1)) {
            int currentNum = num;
            int currentStreak = 1;

            while (numSet.contains(currentNum + 1)) {
                currentNum++;
                currentStreak++;
            }

            longestStreak = Math.max(longestStreak, currentStreak);
        }
    }

    return longestStreak;
}
```

### 这段代码到底在做什么

可以把这段代码分成三部分来看。

第一部分，建集合：

```java
Set<Integer> numSet = new HashSet<>();
for (int num : nums) {
    numSet.add(num);
}
```

这一步做了两件事：

- 去重
- 提供快速查询能力

去重很重要，因为重复数字对连续长度没有贡献。  
比如 `[1,2,2,3]`，真正连续的还是 `1,2,3`，长度是 `3`，不是 `4`。

第二部分，找起点：

```java
if (!numSet.contains(num - 1))
```

只有当前一个数不存在时，当前数才是某段连续序列的开头。

第三部分，从起点向后延伸：

```java
while (numSet.contains(currentNum + 1)) {
    currentNum++;
    currentStreak++;
}
```

这一步就是不断试探下一个数是否还在集合里，直到断掉为止。

最后用：

```java
longestStreak = Math.max(longestStreak, currentStreak);
```

去更新全局答案。

### 这题最容易想错的地方

这题不算难，但第一次做时有几个地方很容易绕一下。

#### 第一个误区：把“数组里连续”理解成“下标连续”

这题和子数组完全没关系。  
它找的是数值连续，不要求这些数字在原数组中挨着出现。

#### 第二个误区：用了集合之后，还是从每个数都往后扩

这样虽然也能做，但会把很多数字反复查很多次。  
这题之所以能做到 `O(n)`，关键不只是哈希集合，而是“只从起点开始扩展”。

#### 第三个误区：忽略重复值

如果用排序法，重复值一定要单独跳过；  
如果用集合，重复值天然已经被去掉了。

这点虽然不复杂，但很容易在排序写法里漏掉。

### 为什么这题能做到 O(n)

表面上看，这段代码里有一个 `for`，里面又套了一个 `while`，很多人第一眼会担心是不是 `O(n^2)`。

其实不是。

原因在于：

- `for` 会把每个不同的数字看一遍
- `while` 只会从序列起点触发
- 每个数字至多只会在某一条连续链里被向后扩展时访问一次

换句话说，那些 `2、3、4` 虽然会在 `for` 里被扫到，但因为它们不是起点，不会再次进入完整扩展流程。

所以整体仍然是线性的。

可以这样理解：

- 建集合：`O(n)`
- 遍历集合并扩展连续段：总共还是 `O(n)`

因此总时间复杂度是 `O(n)`，空间复杂度是 `O(n)`。

### 这题真正值得记住的地方

这题真正值得记住的，不是“最长连续序列”这几个字，而是这种处理方式：

**当题目要求你快速判断某个值在不在，而且顺序本身不重要时，先考虑把数据放进哈希集合。**

但只会“放进集合”还不够。  
这题更关键的一层是：

**想办法找到每段序列的唯一入口，只从入口开始计算。**

本题里，这个入口就是“前一个数不存在”的位置。  
以后再遇到类似“区间扩展”“连续值判断”“避免重复统计”的题，也可以先想有没有这种唯一入口。

### 完整代码

```java
import java.util.*;

public class Main {

    public static int longestConsecutive(int[] nums) {
        Set<Integer> numSet = new HashSet<>();
        for (int num : nums) {
            numSet.add(num);
        }

        int longestStreak = 0;

        for (int num : numSet) {
            if (!numSet.contains(num - 1)) {
                int currentNum = num;
                int currentStreak = 1;

                while (numSet.contains(currentNum + 1)) {
                    currentNum++;
                    currentStreak++;
                }

                longestStreak = Math.max(longestStreak, currentStreak);
            }
        }

        return longestStreak;
    }

    public static void main(String[] args) {
        int[] nums1 = {100, 4, 200, 1, 3, 2};
        System.out.println("示例 1 输入: " + Arrays.toString(nums1));
        System.out.println("示例 1 输出: " + longestConsecutive(nums1));
        System.out.println("-----------------------------");

        int[] nums2 = {0, 3, 7, 2, 5, 8, 4, 6, 0, 1};
        System.out.println("示例 2 输入: " + Arrays.toString(nums2));
        System.out.println("示例 2 输出: " + longestConsecutive(nums2));
        System.out.println("-----------------------------");

        int[] nums3 = {1, 0, 1, 2};
        System.out.println("示例 3 输入: " + Arrays.toString(nums3));
        System.out.println("示例 3 输出: " + longestConsecutive(nums3));
    }
}
```
