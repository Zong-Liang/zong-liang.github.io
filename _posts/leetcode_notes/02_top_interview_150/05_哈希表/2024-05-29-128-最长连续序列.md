## 题目

**描述：**

给定一个未排序的整数数组 `nums` ，找出数字连续的最长序列（不要求序列元素在原数组中连续）的长度。

请你设计并实现时间复杂度为 `O(n)` 的算法解决此问题。

**示例 1:**

- **输入:** `nums = [100,4,200,1,3,2]`
- **输出:** 4
- **解释:** 最长数字连续序列是 `[1, 2, 3, 4]`。它的长度为 4。

**示例 2:**

- **输入:** `nums = [0,3,7,2,5,8,4,6,0,1]`
- **输出:** 9

**示例 3:**

- **输入:** `nums = [1,0,1,2]`
- **输出:** 3

**提示:**

- `0 <= nums.length <= 10^5`
- `-10^9 <= nums[i] <= 10^9`

## 题解

这题最容易想偏的地方，是把“最长连续序列”理解成原数组里一段相邻的区间。其实它根本不关心原下标顺序，只关心数值能不能连起来。像 `[100,4,200,1,3,2]` 里，`1,2,3,4` 在原数组中并没有挨着放，但它们数值连续，所以答案仍然是 `4`。

如果先不考虑题目的线性复杂度要求，排序几乎是最自然的起点。排完序以后，数值连续的元素会凑到一起，再从左到右扫一遍就能统计出最长长度。这个思路非常直观，也很适合先帮助自己把题意看清。但题目明确要 `O(n)`，所以排序法只能当对照，不是最终解法。

更好的突破口是：这题只在乎某个数的前一个值和后一个值在不在。既然如此，就没必要真的把数组排好，只要能快速判断“`x - 1` 在不在、`x + 1` 在不在”就够了。哈希集合正适合做这件事，因为它能在平均 `O(1)` 的时间里判断一个值是否存在。

不过只会查集合还不够。要是对每个数都从头往后扩，比如从 `1` 查到 `2,3,4`，再从 `2` 继续查 `3,4`，重复计算会很多。所以这题关键的一步是：只从连续序列的起点开始扩展。所谓起点，就是“前一个数不存在”的那个数。比如在 `1,2,3,4` 这一段里，只有 `1` 满足 `0` 不在集合里，所以只有它值得往后一路扩。

这样一来，每段连续序列都只会被完整计算一次。拿 `nums = [100,4,200,1,3,2]` 来看，把所有数先放进集合后，`100` 会被识别为一个长度为 `1` 的起点，`200` 也是；而 `1` 因为前一个数 `0` 不在集合里，所以会从它开始一路扩到 `4`，最终得到长度 `4`。至于 `2、3、4`，因为它们前一个数都在集合里，所以直接跳过，不会重复统计。

代码如下：

```java
public int longestConsecutive(int[] nums) {
    Set<Integer> numSet = new HashSet<>();
    for (int num : nums) {
        numSet.add(num);
    }

    int longestStreak = 0;

    for (int num : numSet) {
        if (!numSet.contains(num - 1)) {
            int currentNum = num;
            int currentStreak = 1;

            while (numSet.contains(currentNum + 1)) {
                currentNum++;
                currentStreak++;
            }

            longestStreak = Math.max(longestStreak, currentStreak);
        }
    }

    return longestStreak;
}
```

时间复杂度是 `O(n)`，因为建集合和按起点扩展总共都是线性的；空间复杂度是 `O(n)`。

这题比较适合带走的，不只是“哈希集合查连续值”，而是那个更通用的思路：当题目要求你快速判断某个值在不在，而且顺序本身不重要时，先想哈希集合；如果又能进一步找到“每段结构的唯一入口”，就尽量只从入口开始算，避免重复扩展。

完整代码如下：

```java
import java.util.*;

public class Main {

    public static int longestConsecutive(int[] nums) {
        Set<Integer> numSet = new HashSet<>();
        for (int num : nums) {
            numSet.add(num);
        }

        int longestStreak = 0;

        for (int num : numSet) {
            if (!numSet.contains(num - 1)) {
                int currentNum = num;
                int currentStreak = 1;

                while (numSet.contains(currentNum + 1)) {
                    currentNum++;
                    currentStreak++;
                }

                longestStreak = Math.max(longestStreak, currentStreak);
            }
        }

        return longestStreak;
    }

    public static void main(String[] args) {
        int[] nums1 = {100, 4, 200, 1, 3, 2};
        System.out.println("示例 1 输入: " + Arrays.toString(nums1));
        System.out.println("示例 1 输出: " + longestConsecutive(nums1));
        System.out.println("-----------------------------");

        int[] nums2 = {0, 3, 7, 2, 5, 8, 4, 6, 0, 1};
        System.out.println("示例 2 输入: " + Arrays.toString(nums2));
        System.out.println("示例 2 输出: " + longestConsecutive(nums2));
        System.out.println("-----------------------------");

        int[] nums3 = {1, 0, 1, 2};
        System.out.println("示例 3 输入: " + Arrays.toString(nums3));
        System.out.println("示例 3 输出: " + longestConsecutive(nums3));
    }
}
```
