## 题目

**描述：**

给定一个整数数组 `nums` 和一个整数目标值 `target`，请你在该数组中找出 **和为目标值 `target`** 的那 **两个** 整数，并返回它们的数组下标。

你可以假设每种输入只会对应一个答案。但是，数组中同一个元素在答案里不能重复出现。

你可以按任意顺序返回答案。

**示例 1:**

- **输入:** `nums = [2,7,11,15]`, `target = 9`
- **输出:** `[0,1]`
- **解释:** 因为 `nums[0] + nums[1] == 9` ，返回 `[0, 1]` 。

**示例 2:**

- **输入:** `nums = [3,2,4]`, `target = 6`
- **输出:** `[1,2]`

**示例 3:**

- **输入:** `nums = [3,3]`, `target = 6`
- **输出:** `[0,1]`

**提示:**

- `2 <= nums.length <= 10^4`
- `-10^9 <= nums[i] <= 10^9`
- `-10^9 <= target <= 10^9`
- **只会存在一个有效答案**

**进阶：** 你可以想出一个时间复杂度小于 `O(n^2)` 的算法吗？

## 题解

这题如果顺着题面去想，最自然的办法就是枚举第一个数，再去后面找能和它凑成 `target` 的第二个数。这个思路本身没问题，但它的代价也很明显：每处理一个位置，都要把后面的很多元素再看一遍，所以时间复杂度会变成 `O(n^2)`。

更顺手的角度，是把问题换成“补数”来想。遍历到 `nums[i]` 时，我们想知道的不是“后面有没有合适的数”，而是：为了和当前这个数凑成 `target`，我还差谁？这个“还差的数”就是 `target - nums[i]`。如果这个补数已经在前面出现过，那答案立刻就出来了。

所以哈希表在这里的作用非常直接：它负责记住前面已经看过的数，以及这些数出现的位置。遍历数组时，先算补数，再去哈希表里查它有没有出现过；如果出现过，就返回补数的位置和当前位置；如果没出现过，再把当前值和当前下标存进去，留给后面的人来配对。

这里顺序很重要，一定是先查再存。如果反过来先把当前元素存进哈希表，再去查补数，就有可能把元素和自己配成一对。像 `nums = [3,3]`、`target = 6` 这种输入，就很能看出这个顺序的重要性。

拿 `nums = [2,7,11,15]`、`target = 9` 来看，遍历到 `2` 时，补数是 `7`，前面还没人，先把 `2 -> 0` 存起来；遍历到 `7` 时，补数变成 `2`，而 `2` 已经在表里了，于是答案就是 `[0, 1]`。整个过程都不需要回头找，数组只扫一遍就够了。

代码如下：

```java
public int[] twoSum(int[] nums, int target) {
    Map<Integer, Integer> map = new HashMap<>();

    for (int i = 0; i < nums.length; i++) {
        int complement = target - nums[i];

        if (map.containsKey(complement)) {
            return new int[]{map.get(complement), i};
        }

        map.put(nums[i], i);
    }

    return new int[0];
}
```

时间复杂度是 `O(n)`，因为整个数组只遍历一遍，哈希表查询和插入平均是 `O(1)`；空间复杂度是 `O(n)`。

这题比较适合记住的，不只是“两数之和”本身，而是那个非常常见的动作：当题目要求你在遍历过程中快速判断“某个目标值之前出现过没有”时，优先考虑哈希表。本题里这个目标值就是补数。

完整代码如下：

```java
import java.util.Arrays;
import java.util.HashMap;
import java.util.Map;

public class Main {

    public static int[] twoSum(int[] nums, int target) {
        Map<Integer, Integer> map = new HashMap<>();

        for (int i = 0; i < nums.length; i++) {
            int complement = target - nums[i];

            if (map.containsKey(complement)) {
                return new int[]{map.get(complement), i};
            }

            map.put(nums[i], i);
        }

        return new int[0];
    }

    public static void main(String[] args) {
        int[] nums1 = new int[]{2, 7, 11, 15};
        int target1 = 9;
        System.out.println("示例 1 输入: nums = " + Arrays.toString(nums1) + ", target = " + target1);
        System.out.println("示例 1 输出: " + Arrays.toString(twoSum(nums1, target1)));
        System.out.println("-----------------------------");

        int[] nums2 = new int[]{3, 2, 4};
        int target2 = 6;
        System.out.println("示例 2 输入: nums = " + Arrays.toString(nums2) + ", target = " + target2);
        System.out.println("示例 2 输出: " + Arrays.toString(twoSum(nums2, target2)));
        System.out.println("-----------------------------");

        int[] nums3 = new int[]{3, 3};
        int target3 = 6;
        System.out.println("示例 3 输入: nums = " + Arrays.toString(nums3) + ", target = " + target3);
        System.out.println("示例 3 输出: " + Arrays.toString(twoSum(nums3, target3)));
    }
}
```
