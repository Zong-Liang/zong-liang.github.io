## 题目

**描述：**

给你一个整数数组 `nums` 和一个整数 `k` ，判断数组中是否存在两个 **不同** 的索引 `i` 和 `j` ，满足 `nums[i] == nums[j]` 且 `abs(i - j) <= k` 。如果存在，返回 `true` ；否则，返回 `false` 。

**示例 1:**

- **输入:** `nums = [1,2,3,1], k = 3`
- **输出:** `true`

**示例 2:**

- **输入:** `nums = [1,0,1,1], k = 1`
- **输出:** `true`

**示例 3:**

- **输入:** `nums = [1,2,3,1,2,3], k = 2`
- **输出:** `false`

**提示:**

- `1 <= nums.length <= 10^5`
- `-10^9 <= nums[i] <= 10^9`
- `0 <= k <= 10^5`

## 题解

这题不是单纯在问“数组里有没有重复元素”，而是在问：有没有两个相同的值，它们的位置还得足够近。也就是说，重复本身不够，只有下标差不超过 `k` 的重复才算答案。

最自然的想法，是遍历数组时顺手记住每个数上一次出现的位置。这样当同一个数再次出现时，就可以立刻算出它和上一次的距离。如果这个距离不超过 `k`，答案就出来了；如果超过 `k`，那也没关系，只要把它的最新位置更新进去，继续往后看就行。

这里为什么只记最近一次出现的位置就够了？因为我们关心的是“有没有一对足够近的重复值”。如果当前数和最近一次出现的位置都已经超过了 `k`，那和更早的那些位置只会更远，更不可能满足条件。所以哈希表里保留最新下标，正好就是这题最有用的信息。

拿 `nums = [1,2,3,1]`、`k = 3` 来看，遍历到最后一个 `1` 时，哈希表里已经记着前一个 `1` 出现在下标 `0`，当前距离是 `3 - 0 = 3`，刚好满足要求，于是直接返回 `true`。而在 `nums = [1,2,3,1,2,3]`、`k = 2` 里，虽然每个数都会重复出现，但它们和上一次出现的位置距离都是 `3`，所以都不满足条件，最后返回 `false`。

这题其实还可以换个角度理解成滑动窗口：遍历到 `nums[i]` 时，我们关心的，只是它左边最近的 `k` 个元素里有没有和它相同的值。这个视角可以写成一个大小最多为 `k` 的 `HashSet` 窗口。不过从贴着题意解释的角度来看，哈希表记录最近下标会更直观一些，因为它直接对应“值相同 + 位置距离”这两个条件。

代码如下：

```java
public boolean containsNearbyDuplicate(int[] nums, int k) {
    Map<Integer, Integer> map = new HashMap<>();

    for (int i = 0; i < nums.length; i++) {
        int num = nums[i];

        if (map.containsKey(num) && i - map.get(num) <= k) {
            return true;
        }

        map.put(num, i);
    }

    return false;
}
```

时间复杂度是 `O(n)`，因为只遍历了一次数组，哈希表查询和更新平均是 `O(1)`；空间复杂度最坏是 `O(n)`。

这题比较适合记住的，不是题名本身，而是那个判断模型：当题目同时要求“值相同”和“位置受限”时，只看值是不够的，得把位置信息也一起记下来。记最近位置，是最常见也最自然的一种办法。

完整代码如下：

```java
import java.util.Arrays;
import java.util.HashMap;
import java.util.Map;

public class Main {

    public static boolean containsNearbyDuplicate(int[] nums, int k) {
        Map<Integer, Integer> map = new HashMap<>();

        for (int i = 0; i < nums.length; i++) {
            int num = nums[i];

            if (map.containsKey(num)) {
                int prevIndex = map.get(num);
                if (i - prevIndex <= k) {
                    return true;
                }
            }

            map.put(num, i);
        }

        return false;
    }

    public static void main(String[] args) {
        int[] nums1 = {1, 2, 3, 1};
        int k1 = 3;
        System.out.println("示例 1 输入: nums = " + Arrays.toString(nums1) + ", k = " + k1);
        System.out.println("示例 1 输出: " + containsNearbyDuplicate(nums1, k1));
        System.out.println("-----------------------------");

        int[] nums2 = {1, 0, 1, 1};
        int k2 = 1;
        System.out.println("示例 2 输入: nums = " + Arrays.toString(nums2) + ", k = " + k2);
        System.out.println("示例 2 输出: " + containsNearbyDuplicate(nums2, k2));
        System.out.println("-----------------------------");

        int[] nums3 = {1, 2, 3, 1, 2, 3};
        int k3 = 2;
        System.out.println("示例 3 输入: nums = " + Arrays.toString(nums3) + ", k = " + k3);
        System.out.println("示例 3 输出: " + containsNearbyDuplicate(nums3, k3));
    }
}
```
