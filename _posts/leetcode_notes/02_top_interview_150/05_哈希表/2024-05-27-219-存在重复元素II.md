## 题目

**描述：**

给你一个整数数组 `nums` 和一个整数 `k` ，判断数组中是否存在两个 **不同** 的索引 `i` 和 `j` ，满足 `nums[i] == nums[j]` 且 `abs(i - j) <= k` 。如果存在，返回 `true` ；否则，返回 `false` 。

**示例 1:**

- **输入:** `nums = [1,2,3,1], k = 3`
- **输出:** `true`

**示例 2:**

- **输入:** `nums = [1,0,1,1], k = 1`
- **输出:** `true`

**示例 3:**

- **输入:** `nums = [1,2,3,1,2,3], k = 2`
- **输出:** `false`

**提示:**

- `1 <= nums.length <= 10^5`
- `-10^9 <= nums[i] <= 10^9`
- `0 <= k <= 10^5`

## 题解

### 先看这题真正要判断的是什么

这题不是单纯问“数组里有没有重复元素”，而是问：

**有没有两个相同的元素，它们的位置还不能离得太远。**

也就是说，要同时满足两个条件：

- 值相同：`nums[i] == nums[j]`
- 下标距离不超过 `k`：`abs(i - j) <= k`

所以这题的关键不只是“查重”，还得顺手知道这个元素上一次出现在哪。

### 最直接的想法：一边遍历，一边记住每个数最近出现的位置

如果当前遍历到 `nums[i]`，那么我们最关心的其实不是它以前出现过多少次，而是：

**它最近一次出现的位置在哪里。**

因为如果当前这个数之前出现过，那么和它距离最近的那个相同元素，一定最有可能满足 `<= k`。  
更早之前的相同元素只会更远，不会更有优势。

所以这题最自然的做法，就是用一个哈希表记录：

```text
数值 -> 最近一次出现的下标
```

遍历到当前位置时：

1. 先看这个数之前有没有出现过
2. 如果出现过，就拿当前下标减去上一次下标
3. 只要差值不超过 `k`，直接返回 `true`
4. 不管这次能不能成功，都把它的最新位置更新成当前下标

这就是这题最推荐掌握的主解法。

### 为什么只记录“最近一次出现的位置”就够了

这个点是整题最关键的地方之一。

假设某个数字 `x` 在这些位置出现过：

```text
2, 5, 9
```

现在我们正在看位置 `9`。  
如果要判断它和之前的某个 `x` 是否满足距离不超过 `k`，那显然先看位置 `5` 就够了，因为 `5` 离 `9` 最近。

如果 `9 - 5 > k`，那 `9 - 2` 只会更大，更不可能满足。  
所以我们根本不需要把所有出现位置都记下来，只保留最近一次的位置就够用了。

这也是为什么哈希表里每次都要更新成最新下标。

### 用示例走一遍

先看示例 1：

```text
nums = [1,2,3,1], k = 3
```

我们从左到右遍历。

#### 第 0 位：`1`

哈希表里还没有 `1`，先记下来：

```text
1 -> 0
```

#### 第 1 位：`2`

哈希表里没有 `2`，继续记录：

```text
1 -> 0
2 -> 1
```

#### 第 2 位：`3`

哈希表里没有 `3`，继续记录：

```text
1 -> 0
2 -> 1
3 -> 2
```

#### 第 3 位：`1`

这时候发现 `1` 之前出现过，而且上一次位置是 `0`。  
当前距离就是：

```text
3 - 0 = 3
```

满足 `<= k`，所以直接返回 `true`。

再看示例 3：

```text
nums = [1,2,3,1,2,3], k = 2
```

虽然数组里确实有重复元素，但：

- 第 0 位和第 3 位的 `1` 相差 `3`
- 第 1 位和第 4 位的 `2` 相差 `3`
- 第 2 位和第 5 位的 `3` 相差 `3`

都超过了 `2`，所以最后返回 `false`。

这个例子也很能说明问题：

**这题不是看有没有重复，而是看重复得够不够近。**

### 代码

```java
public boolean containsNearbyDuplicate(int[] nums, int k) {
    Map<Integer, Integer> map = new HashMap<>();

    for (int i = 0; i < nums.length; i++) {
        int num = nums[i];

        if (map.containsKey(num) && i - map.get(num) <= k) {
            return true;
        }

        map.put(num, i);
    }

    return false;
}
```

### 这段代码到底在做什么

整段代码其实就是把前面的思路直接翻译出来了。

先开一个哈希表：

```java
Map<Integer, Integer> map = new HashMap<>();
```

它负责保存每个数最近一次出现的位置。

然后遍历数组：

```java
for (int i = 0; i < nums.length; i++)
```

每到一个位置，先检查当前值 `num` 是否已经在表里。

如果在，说明它以前出现过，就可以直接算距离：

```java
i - map.get(num)
```

只要这个距离不超过 `k`，就说明找到了满足条件的一对，立刻返回 `true`。

如果不满足，或者这个数之前没出现过，那也没关系，最后都要执行这句：

```java
map.put(num, i);
```

意思是：  
从现在开始，这个数最新一次出现的位置是当前下标 `i`。

所以这段代码做的事非常集中：

**对每个元素，只关心它离上一次相同元素有多远。**

### 这题其实还能换个角度理解

除了“记录最近位置”之外，这题还有一种也很自然的想法：

既然只关心距离不超过 `k`，那就没必要盯着整个数组看，完全可以只盯着“当前元素前面的 `k` 个位置”。

换句话说，遍历到 `nums[i]` 时，我们真正关心的是：

**在下标范围 `[i-k, i-1]` 里，有没有和 `nums[i]` 相同的元素。**

如果有，就返回 `true`；  
如果没有，就继续往后走。

这个想法其实已经很接近滑动窗口了。

### 滑动窗口为什么也能做

如果我们维护一个大小最多为 `k` 的窗口，里面放的是“当前元素之前最近的 `k` 个元素”，那么每次只要判断当前元素是否已经在这个窗口里出现过就可以了。

做法是：

- 用一个哈希集合 `set` 存窗口内的元素
- 遍历到 `nums[i]` 时，如果窗口大小超过了 `k`，就把最左边那个元素移出去
- 然后检查当前元素能不能加入集合
- 如果加入失败，说明集合里已经有相同元素了，也就说明当前这个数和窗口内某个数距离不超过 `k`

这个写法的好处是空间复杂度更接近 `O(k)`，当 `k` 远小于 `n` 时会比较省空间。

### 滑动窗口写法

```java
public boolean containsNearbyDuplicate(int[] nums, int k) {
    Set<Integer> set = new HashSet<>();

    for (int i = 0; i < nums.length; i++) {
        if (i > k) {
            set.remove(nums[i - k - 1]);
        }

        if (!set.add(nums[i])) {
            return true;
        }
    }

    return false;
}
```

### 这段滑动窗口代码怎么理解

这里最容易看懂的方式，是把 `set` 视为“当前元素左边最近的 `k` 个数”。

比如当前来到位置 `i`：

- 如果 `i <= k`，那前面元素还不到 `k` 个，窗口里就把前面的都留着
- 如果 `i > k`，说明前面已经超过 `k` 个元素了，这时候最老的那个，也就是 `nums[i - k - 1]`，已经不可能再和当前位置组成合法答案，因为它离得太远了，所以要把它删掉

删完以后，窗口里留下的，正好就是距离当前位置不超过 `k` 的那些值。

这时再执行：

```java
if (!set.add(nums[i])) {
    return true;
}
```

如果加入失败，说明窗口里已经有一个相同的值了，而窗口本身又保证距离不超过 `k`，所以可以直接返回 `true`。

这个写法本质上是在做：

**只保留还来得及形成答案的那部分元素。**

### 两种写法该怎么选

如果只是从讲解和思路清晰度来看，哈希表记录最近下标的写法更直观。

因为它直接贴着题意：

- 发现重复
- 看距离
- 更新最近位置

这条线非常顺。

如果更在意“我其实只关心最近 `k` 个元素”这个视角，那滑动窗口也很好，而且空间上通常会更省一些。

不过作为这题的主解法，还是更建议优先掌握哈希表版本。  
因为它更容易一眼对应到题目条件。

### 这题最容易写错的地方

这题本身不难，但有几个细节很容易在第一次写时出错。

#### 第一个误区：发现重复就直接返回 `true`

这不对。  
题目要求的不是“有重复”，而是“有重复并且距离不超过 `k`”。

所以只看到重复值还不够，还必须算一下下标差。

#### 第二个误区：不更新为最新位置

比如某个数之前在位置 `1` 出现过，现在又在位置 `4` 出现了一次。  
如果你不把记录更新成 `4`，后面再遇到同样的数时，算出来的距离就可能不是最近的，逻辑会变笨，甚至容易绕。

正确做法是：  
**每次都用最新位置覆盖旧位置。**

#### 第三个误区：滑动窗口里删错元素

在集合写法里，移除的必须是：

```java
nums[i - k - 1]
```

因为它刚好是窗口左边那个已经超出允许范围的元素。  
这个下标如果写错，窗口范围就不对了，结果也会跟着出错。

### 复杂度怎么理解

对于哈希表解法：

- 时间复杂度：`O(n)`  
  只遍历一次数组，每次哈希表查询和更新平均是 `O(1)`

- 空间复杂度：`O(n)`  
  最坏情况下所有元素都不重复，哈希表要记录所有值

对于滑动窗口解法：

- 时间复杂度：`O(n)`
- 空间复杂度：`O(k)`  
  因为集合里最多只保留最近 `k` 个元素

所以如果 `k` 很小，滑动窗口在空间上会更紧凑一些；  
但从教学表达和直观程度来说，哈希表记最近下标仍然是更稳的主写法。

### 这题真正值得记住的地方

这题真正值得记住的，不是“存在重复元素 II”这个题名，而是这种判断模型：

**当题目同时要求“值相同”和“位置距离受限”时，除了看值，还要顺手记录位置信息。**

有时记录的是最近一次出现的下标，  
有时记录的是一个有限范围内的窗口内容。

本题里，这两种思路其实都围绕着同一个核心：

**重复值本身不够，只有足够近的重复值才有意义。**

### 完整代码

```java
import java.util.Arrays;
import java.util.HashMap;
import java.util.Map;

public class Main {

    public static boolean containsNearbyDuplicate(int[] nums, int k) {
        Map<Integer, Integer> map = new HashMap<>();

        for (int i = 0; i < nums.length; i++) {
            int num = nums[i];

            if (map.containsKey(num)) {
                int prevIndex = map.get(num);
                if (i - prevIndex <= k) {
                    return true;
                }
            }

            map.put(num, i);
        }

        return false;
    }

    public static void main(String[] args) {
        int[] nums1 = {1, 2, 3, 1};
        int k1 = 3;
        System.out.println("示例 1 输入: nums = " + Arrays.toString(nums1) + ", k = " + k1);
        System.out.println("示例 1 输出: " + containsNearbyDuplicate(nums1, k1));
        System.out.println("-----------------------------");

        int[] nums2 = {1, 0, 1, 1};
        int k2 = 1;
        System.out.println("示例 2 输入: nums = " + Arrays.toString(nums2) + ", k = " + k2);
        System.out.println("示例 2 输出: " + containsNearbyDuplicate(nums2, k2));
        System.out.println("-----------------------------");

        int[] nums3 = {1, 2, 3, 1, 2, 3};
        int k3 = 2;
        System.out.println("示例 3 输入: nums = " + Arrays.toString(nums3) + ", k = " + k3);
        System.out.println("示例 3 输出: " + containsNearbyDuplicate(nums3, k3));
    }
}
```
