## 题目

**描述：**

给你两个字符串：`ransomNote` 和 `magazine` ，判断 `ransomNote` 能不能由 `magazine` 里面的字符构成。

如果可以，返回 `true` ；否则返回 `false` 。

`magazine` 中的每个字符只能在 `ransomNote` 中使用一次。

**示例 1:**

- **输入:** `ransomNote = "a"`, `magazine = "b"`
- **输出:** `false`

**示例 2:**

- **输入:** `ransomNote = "aa"`, `magazine = "ab"`
- **输出:** `false`

**示例 3:**

- **输入:** `ransomNote = "aa"`, `magazine = "aab"`
- **输出:** `true`

**提示:**

- `1 <= ransomNote.length, magazine.length <= 10^5`
- `ransomNote` 和 `magazine` 由小写英文字母组成

## 题解

### 先把这题看成一次“库存是否够用”的判断

这题表面上是在比较两个字符串，实际上更像一道很典型的“资源够不够”的问题。

`magazine` 里每个字符都像库存，`ransomNote` 里每个字符都像需求。  
题目问的其实就是：

**`magazine` 里每种字符的数量，能不能覆盖 `ransomNote` 对应的需求量。**

而且还有一个限制条件：

**`magazine` 中的每个字符只能使用一次。**

这句话非常关键。它意味着我们不能只判断“某个字符在不在”，而必须判断“这个字符有几个”。

比如：

- `ransomNote = "aa"`
- `magazine = "ab"`

虽然 `magazine` 里确实有字符 `a`，但只有一个，不够拼出两个 `a`，所以答案仍然是 `false`。

所以这题真正要解决的，不是字符是否出现过，而是字符频次是否足够。

### 最自然的思路就是先统计，再消耗

处理这种“频次够不够”的问题，最直接的办法就是两步：

第一步，先统计 `magazine` 中每个字符出现了多少次。  
第二步，再去遍历 `ransomNote`，每遇到一个字符，就从这份统计里扣掉一次。

如果某次扣完之后变成了负数，就说明这个字符不够用了，可以立刻返回 `false`。

换句话说，这题并不需要什么复杂技巧，关键只是把思路理顺：

- `magazine` 负责提供字符
- `ransomNote` 负责消耗字符
- 只要中途出现库存不足，答案就是 `false`
- 如果整个 `ransomNote` 都能顺利消耗完，答案就是 `true`

### 为什么这里不用 HashMap，而用数组更合适

如果题目没有限制字符范围，那确实很容易先想到用 `HashMap<Character, Integer>` 来统计频次。

但这题已经明确说了：

- 两个字符串都只由**小写英文字母**组成

这就意味着字符总共只有 `26` 种。  
既然范围已经固定，那就没必要再用哈希表了，直接开一个长度为 `26` 的整型数组就够了。

比如：

- `cnt[0]` 表示字符 `'a'` 的数量
- `cnt[1]` 表示字符 `'b'` 的数量
- ...
- `cnt[25]` 表示字符 `'z'` 的数量

字符 `c` 对应的下标就是：

```java
c - 'a'
```

这样写的好处很明显：

- 代码更短
- 查找和修改都是直接下标访问
- 没有哈希计算，也没有装箱拆箱
- 空间始终固定

对于这道题，这就是最合适的写法。

### 先做一个长度判断，其实很自然

在正式统计之前，还可以先加一个很自然的判断：

如果 `ransomNote.length() > magazine.length()`，那答案一定是 `false`。

因为赎金信需要的总字符数比杂志里现有的字符数还多，不管字符怎么分布，都不可能拼出来。

这个判断不是必须的，因为后面的逻辑本身也能判断出来；  
但加上它以后，可以提前剪枝，也更符合直觉。

### 用示例走一遍过程

看示例 3：

```text
ransomNote = "aa"
magazine = "aab"
```

先统计 `magazine`：

- `'a'` 有 `2` 个
- `'b'` 有 `1` 个

也就是：

```text
a -> 2
b -> 1
```

接下来开始遍历 `ransomNote`：

#### 第一个字符是 `a`

把 `'a'` 的库存减一：

```text
a -> 1
```

还够，继续。

#### 第二个字符还是 `a`

再减一：

```text
a -> 0
```

还是没有小于 `0`，说明够用。

遍历结束，所以返回 `true`。

再看示例 2：

```text
ransomNote = "aa"
magazine = "ab"
```

先统计 `magazine`：

```text
a -> 1
b -> 1
```

遍历 `ransomNote`：

- 第一个 `a` 扣完后，`a -> 0`
- 第二个 `a` 再扣一次后，`a -> -1`

一旦出现负数，就说明这个字符已经不够了，直接返回 `false`。

这个过程其实非常直观：

**数组里存的是库存，赎金信里的每个字符都在向这份库存申请一次。**

### 代码

```java
public boolean canConstruct(String ransomNote, String magazine) {
    if (ransomNote.length() > magazine.length()) {
        return false;
    }

    int[] cnt = new int[26];

    for (char c : magazine.toCharArray()) {
        cnt[c - 'a']++;
    }

    for (char c : ransomNote.toCharArray()) {
        cnt[c - 'a']--;
        if (cnt[c - 'a'] < 0) {
            return false;
        }
    }

    return true;
}
```

### 这段代码到底在做什么

代码虽然短，但逻辑非常完整。

先看第一段：

```java
for (char c : magazine.toCharArray()) {
    cnt[c - 'a']++;
}
```

这一步是在给库存表补货。  
`magazine` 里出现一个字符，对应位置就加一。

再看第二段：

```java
for (char c : ransomNote.toCharArray()) {
    cnt[c - 'a']--;
    if (cnt[c - 'a'] < 0) {
        return false;
    }
}
```

这一步是在按需消耗库存。  
每处理一个 `ransomNote` 中的字符，就从对应库存里扣一次。

如果扣完之后小于 `0`，说明需求超过了库存，直接失败。

最后如果整个循环都走完了，说明每个字符都成功拿到了对应数量，自然就能构造出来。

### 这题最容易犯的误区

这题本身不难，但有两个误区很常见。

#### 第一个误区：只判断字符是否存在

比如有人会想：只要 `ransomNote` 里的每个字符都能在 `magazine` 里找到，不就行了吗？

这样想是不够的。  
因为题目不是问“能不能找到”，而是问“数量够不够”。

`"aa"` 和 `"ab"` 就是最典型的反例。

#### 第二个误区：用字符串删除来模拟消耗

有些人会想到，每找到一个字符，就从 `magazine` 里删掉一个对应字符。  
这种做法逻辑上能说通，但实现起来很麻烦，而且效率也差。

因为字符串本身不适合频繁删除，反复查找和构造新字符串的代价都很高。  
相比之下，直接做频次统计会更自然，也更高效。

### 复杂度为什么是 O(m + n)

设：

- `m` 是 `magazine` 的长度
- `n` 是 `ransomNote` 的长度

我们只做了两次线性遍历：

- 一次遍历 `magazine`
- 一次遍历 `ransomNote`

每次操作都是数组下标访问，时间复杂度是常数级。

所以总时间复杂度就是：

- 时间复杂度：`O(m + n)`

而空间方面只开了一个长度固定为 `26` 的数组，所以：

- 空间复杂度：`O(1)`

这是这题最标准、也最合适的解法。

### 这题真正值得记住的地方

这题真正值得记住的，不是“赎金信”这个题面，而是这种判断模型：

**当题目本质上是在问“一个集合的频次，能不能覆盖另一个集合的频次”时，优先考虑做计数统计。**

如果字符集范围固定，就直接用数组；  
如果字符范围不固定，再考虑哈希表。

把这个思路记住之后，后面再遇到字母异位词、字符覆盖、频次匹配这类题，都会顺很多。

### 完整代码

```java
public class Main {

    public static boolean canConstruct(String ransomNote, String magazine) {
        if (ransomNote.length() > magazine.length()) {
            return false;
        }

        int[] cnt = new int[26];

        for (char c : magazine.toCharArray()) {
            cnt[c - 'a']++;
        }

        for (char c : ransomNote.toCharArray()) {
            cnt[c - 'a']--;
            if (cnt[c - 'a'] < 0) {
                return false;
            }
        }

        return true;
    }

    public static void main(String[] args) {
        String ransomNote1 = "a";
        String magazine1 = "b";
        System.out.println("示例 1 输入: ransomNote = \"" + ransomNote1 + "\", magazine = \"" + magazine1 + "\"");
        System.out.println("示例 1 输出: " + canConstruct(ransomNote1, magazine1));
        System.out.println("-----------------------------");

        String ransomNote2 = "aa";
        String magazine2 = "ab";
        System.out.println("示例 2 输入: ransomNote = \"" + ransomNote2 + "\", magazine = \"" + magazine2 + "\"");
        System.out.println("示例 2 输出: " + canConstruct(ransomNote2, magazine2));
        System.out.println("-----------------------------");

        String ransomNote3 = "aa";
        String magazine3 = "aab";
        System.out.println("示例 3 输入: ransomNote = \"" + ransomNote3 + "\", magazine = \"" + magazine3 + "\"");
        System.out.println("示例 3 输出: " + canConstruct(ransomNote3, magazine3));
    }
}
```
