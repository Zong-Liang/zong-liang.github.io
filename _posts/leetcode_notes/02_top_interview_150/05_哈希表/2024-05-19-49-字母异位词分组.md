## 题目

**描述：**

给你一个字符串数组，请你将 **字母异位词** 组合在一起。可以按任意顺序返回结果列表。

**字母异位词** 是由相同字母按不同顺序排列形成的字符串。

**示例 1:**

- **输入:** `strs = ["eat","tea","tan","ate","nat","bat"]`
- **输出:** `[["bat"],["nat","tan"],["ate","eat","tea"]]`

**示例 2:**

- **输入:** `strs = [""]`
- **输出:** `[[""]]`

**示例 3:**

- **输入:** `strs = ["a"]`
- **输出:** `[["a"]]`

**提示:**

- `1 <= strs.length <= 10^4`
- `0 <= strs[i].length <= 100`
- `strs[i]` 仅包含小写字母

## 题解

### 这题不是在判断两个字符串，而是在给一批字符串“归类”

如果只是判断两个字符串是否互为字母异位词，那思路通常是比较它们的字符频次是否一致。  
但这题更进一步，它给的是一个字符串数组，要求把所有互为字母异位词的字符串分到同一组里。

所以这题真正要解决的问题，其实是：

**怎么为每个字符串找到一个“统一标识”，让所有异位词都能落到同一个桶里。**

只要这个标识设计得对，后面的事情就简单了：  
用哈希表分组，标识相同的放进同一个列表里，最后把所有列表取出来即可。

### 先想清楚，异位词之间到底有什么不变的东西

“字母异位词”虽然顺序不同，但字符种类和每种字符的数量一定完全一样。

比如：

- `"eat"`
- `"tea"`
- `"ate"`

它们字符顺序不同，但都由一个 `a`、一个 `e`、一个 `t` 组成。

所以如果我们能把一个字符串转换成某种**与顺序无关、只和字符组成有关**的形式，那么异位词转换后的结果一定一样。

最容易想到的办法，就是排序。

例如：

- `"eat"` 排序后是 `"aet"`
- `"tea"` 排序后也是 `"aet"`
- `"ate"` 排序后还是 `"aet"`

这样一来，排序后的字符串就可以作为这一组异位词的统一标识。

### 这题最核心的一步，就是给每个字符串生成 key

整题可以拆成两步看。

第一步，对每个字符串生成一个 key。  
这里的 key 不是原字符串本身，而是**排序后的结果**。

第二步，用哈希表把相同 key 的字符串放到一起。

比如输入：

```text
["eat","tea","tan","ate","nat","bat"]
```

遍历时会变成这样：

- `"eat"` -> key 是 `"aet"`
- `"tea"` -> key 是 `"aet"`
- `"tan"` -> key 是 `"ant"`
- `"ate"` -> key 是 `"aet"`
- `"nat"` -> key 是 `"ant"`
- `"bat"` -> key 是 `"abt"`

然后用 `Map<String, List<String>>` 来存：

- `"aet"` 对应 `["eat", "tea", "ate"]`
- `"ant"` 对应 `["tan", "nat"]`
- `"abt"` 对应 `["bat"]`

最后把 `map.values()` 取出来，就是答案。

所以这题真正的关键，不在哈希表，而在于你能不能找到一个正确的分组依据。

### 为什么排序后的字符串可以作为分组依据

因为排序会把“顺序不同”这个干扰因素彻底消掉，只保留字符组成。

只要两个字符串互为异位词，它们排序后一定相同。  
反过来，如果两个字符串排序后相同，那它们的字符种类和频次也一定一致，自然也互为异位词。

这说明：

**“排序后的字符串相同” 和 “互为字母异位词” 这两件事，在这题里是等价的。**

这也是为什么排序法这么适合拿来做分组 key。

### 用示例走一遍过程

看示例 1：

```text
strs = ["eat","tea","tan","ate","nat","bat"]
```

一开始哈希表是空的。

#### 处理 `"eat"`

排序后得到 `"aet"`。  
哈希表里还没有这个 key，于是新建一个列表：

```text
"aet" -> ["eat"]
```

#### 处理 `"tea"`

排序后还是 `"aet"`。  
说明它和 `"eat"` 属于同一组，直接追加进去：

```text
"aet" -> ["eat", "tea"]
```

#### 处理 `"tan"`

排序后是 `"ant"`。  
这是一个新的 key，新开一组：

```text
"ant" -> ["tan"]
```

#### 处理 `"ate"`

排序后还是 `"aet"`。  
继续加入 `"aet"` 对应的列表：

```text
"aet" -> ["eat", "tea", "ate"]
```

#### 处理 `"nat"`

排序后是 `"ant"`。  
加入 `"ant"` 这一组：

```text
"ant" -> ["tan", "nat"]
```

#### 处理 `"bat"`

排序后是 `"abt"`。  
新开一组：

```text
"abt" -> ["bat"]
```

遍历结束后，哈希表里就已经自然分好了组。  
最后把所有 value 取出来即可。

### 代码

```java
public List<List<String>> groupAnagrams(String[] strs) {
    if (strs == null || strs.length == 0) {
        return new ArrayList<>();
    }

    Map<String, List<String>> map = new HashMap<>();

    for (String s : strs) {
        char[] chars = s.toCharArray();
        Arrays.sort(chars);
        String key = new String(chars);

        if (!map.containsKey(key)) {
            map.put(key, new ArrayList<>());
        }

        map.get(key).add(s);
    }

    return new ArrayList<>(map.values());
}
```

### 这段代码到底在做什么

整段代码其实很直白，可以分成三步看。

第一步，把字符串转换成分组 key：

```java
char[] chars = s.toCharArray();
Arrays.sort(chars);
String key = new String(chars);
```

这里做的事情，就是把原字符串变成一个“只反映字符组成、不反映原始顺序”的标准形式。

第二步，把原字符串放到对应 key 的列表里：

```java
if (!map.containsKey(key)) {
    map.put(key, new ArrayList<>());
}
map.get(key).add(s);
```

这一步本质上就是“按 key 分桶”。

第三步，返回所有分组结果：

```java
return new ArrayList<>(map.values());
```

因为哈希表的 value 本身就是一组一组的异位词列表，所以直接取出来就行。

### 这题最容易想偏的地方

这题本身不难，但第一次做时有几个点很容易绕一下。

#### 第一个误区：直接拿原字符串做 key

这当然不行。  
因为 `"eat"` 和 `"tea"` 原字符串不同，如果直接拿原字符串当 key，它们永远不会进同一组。

问题的关键就在于：  
**key 必须能屏蔽顺序差异，只保留字符组成。**

#### 第二个误区：只看字符种类，不看字符个数

比如 `"abb"` 和 `"aab"`，字符种类看起来都是 `{a, b}`，但它们不是异位词，因为频次不同。

所以分组依据不能只是“包含哪些字符”，还必须把每种字符出现的次数也编码进去。

排序法之所以可靠，就是因为它天然把频次信息也保留住了。

#### 第三个误区：把这题当成两两比较

如果拿每个字符串去和别的字符串逐个比较，再决定归到哪组，逻辑当然也能做，但会变得很笨重，复杂度也高。

这题更自然的做法不是“比较谁和谁一样”，而是“给每个人发一个统一标签，再按标签归类”。

### 为什么这里不用上一题那种 26 长度计数数组直接做答案

上一题“有效的字母异位词”只需要判断两个字符串是否相等，所以开一个 26 长度的数组做频次统计很自然。

这题是要把很多字符串分组，理论上也可以用字符计数作为 key。  
比如把每个字符串统计成类似：

```text
#1#0#0#2#0...
```

这样的频次编码，然后把这个编码作为 key。

这种做法也能过，而且当字符串很长时，有时还会比排序更合适。  
但从题解表达和直观性来说，排序法更容易让读者一下子看懂：

**异位词排序后相同，所以可以直接拿排序结果分组。**

对这题来说，这就是最顺手的主解法。

### 复杂度怎么理解

假设：

- 数组里一共有 `N` 个字符串
- 每个字符串的平均长度是 `K`

对每个字符串，我们都要先排序。  
单个字符串排序的代价是 `O(K log K)`，总共要做 `N` 次，所以总时间复杂度是：

- 时间复杂度：`O(N * K log K)`

空间方面：

- 哈希表要存所有字符串的分组结果
- 每次排序还需要字符数组

整体可以记为：

- 空间复杂度：`O(N * K)`

这也是排序分组法的标准复杂度。

### 这题真正值得记住的地方

这题真正值得记住的，不是“哈希表 + 排序”这几个字，而是这种拆题方式：

**当题目要求把“本质相同、表现形式不同”的对象归类时，先想能不能为每个对象构造一个统一的标准表示。**

本题里：

- 表现形式不同的是字符顺序
- 本质相同的是字符组成
- 排序后的字符串，就是那个统一的标准表示

以后再遇到分组、归类、规范化表示这类题，都可以先往这个方向想。

### 完整代码

```java
import java.util.*;

public class Main {

    public static List<List<String>> groupAnagrams(String[] strs) {
        if (strs == null || strs.length == 0) {
            return new ArrayList<>();
        }

        Map<String, List<String>> map = new HashMap<>();

        for (String s : strs) {
            char[] chars = s.toCharArray();
            Arrays.sort(chars);
            String key = new String(chars);

            if (!map.containsKey(key)) {
                map.put(key, new ArrayList<>());
            }

            map.get(key).add(s);
        }

        return new ArrayList<>(map.values());
    }

    public static void main(String[] args) {
        String[] strs1 = {"eat", "tea", "tan", "ate", "nat", "bat"};
        System.out.println("示例 1 输入: " + Arrays.toString(strs1));
        System.out.println("示例 1 输出: " + groupAnagrams(strs1));
        System.out.println("-----------------------------");

        String[] strs2 = {""};
        System.out.println("示例 2 输入: " + Arrays.toString(strs2));
        System.out.println("示例 2 输出: " + groupAnagrams(strs2));
        System.out.println("-----------------------------");

        String[] strs3 = {"a"};
        System.out.println("示例 3 输入: " + Arrays.toString(strs3));
        System.out.println("示例 3 输出: " + groupAnagrams(strs3));
    }
}
```
