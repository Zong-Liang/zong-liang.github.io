## 题目

**描述：**

给定两个字符串 `s` 和 `t` ，判断它们是否是同构的。

如果 `s` 中的字符可以按某种映射关系替换得到 `t` ，那么这两个字符串是同构的。

每个出现的字符都应当映射到另一个字符，同时不改变字符的顺序。不同字符不能映射到同一个字符上，相同字符只能映射到同一个字符上，字符可以映射到自己本身。

**示例 1:**

- **输入:** `s = "egg"`, `t = "add"`
- **输出:** `true`

**示例 2:**

- **输入:** `s = "foo"`, `t = "bar"`
- **输出:** `false`

**示例 3:**

- **输入:** `s = "paper"`, `t = "title"`
- **输出:** `true`

**提示:**

- `1 <= s.length <= 5 * 10^4`
- `t.length == s.length`
- `s` 和 `t` 由任意有效的 ASCII 字符组成

## 题解

### 这题真正要判断的，不是“能不能对应上”，而是“对应关系会不会乱”

题目说的是“同构字符串”，听起来有点抽象，但本质并不复杂。

它要求的是：  
把 `s` 里的每个字符按某种规则替换后，能不能得到 `t`。

这里有两个限制必须同时满足：

- `s` 中相同的字符，必须始终映射成 `t` 中同一个字符
- `s` 中不同的字符，不能映射成 `t` 中同一个字符

也就是说，这不是单纯看两个字符串长得像不像，而是在判断它们之间能不能建立一种**一一对应**的关系。

比如：

- `"egg"` 和 `"add"` 是可以的，因为 `e -> a`，`g -> d`，关系始终一致
- `"foo"` 和 `"bar"` 不行，因为第二个和第三个字符都是 `o`，但在 `t` 里对应成了 `a` 和 `r`，同一个字符映射乱了
- `"ab"` 和 `"aa"` 也不行，因为 `a` 和 `b` 是两个不同字符，却都映射成了 `a`

所以这题真正要检查的是：

**当前位置这两个字符，是否延续了前面已经建立好的对应关系。**

### 最容易想到的做法：直接记录映射关系

最直观的思路，其实是开两个哈希表：

- 一个记录 `s` 中字符映射到 `t` 中哪个字符
- 一个反过来记录 `t` 中字符映射回 `s` 中哪个字符

这样双向检查，就能保证不是“单向一致”，而是真正的一一对应。

这个思路当然是对的，但这题还有一种更简洁的写法，不用真的把映射关系一个个存下来，也能完成判断。

### 这题更巧的地方，在于比较“上次出现的位置”

这题有个非常好用的观察：

如果两个字符串是同构的，那么在同一个位置上的两个字符，它们之前出现的位置模式也应该一致。

拿示例 `"egg"` 和 `"add"` 来看：

- 第 0 位：`e` 第一次出现，`a` 也是第一次出现
- 第 1 位：`g` 第一次出现，`d` 也是第一次出现
- 第 2 位：`g` 上一次出现在下标 `1`，`d` 上一次也出现在下标 `1`

这说明它们的“重复方式”是一致的，所以可以判定为同构。

再看 `"foo"` 和 `"bar"`：

- 第 0 位：`f` 和 `b` 都是第一次出现
- 第 1 位：`o` 和 `a` 都是第一次出现
- 第 2 位：`o` 上一次出现在下标 `1`，但 `r` 是第一次出现

这里模式就对不上了，所以不是同构。

换句话说，这题不一定非要显式记录“`a -> x`”，也可以改成判断：

**`s[i]` 和 `t[i]` 这两个字符，上一次出现的位置是否相同。**

只要有一次不同，映射关系就已经乱了。

### 为什么这个判断是成立的

因为同构的本质，本来就是“重复关系一致”。

如果 `s` 中某个字符之前已经出现过一次，那么它在 `t` 中对应的字符也必须在对应的位置出现过；  
如果 `s` 中这个字符是第一次出现，那么 `t` 中这个字符也必须是第一次出现。

只要两边的“历史位置”始终同步，就说明映射关系始终没有冲突。

所以这题可以转成一个非常紧凑的判断：

- 记录 `s` 中每个字符上一次出现的位置
- 记录 `t` 中每个字符上一次出现的位置
- 遍历时比较这两个位置是否相同

相同就继续，不同就直接返回 `false`

### 为什么这里用数组比 HashMap 更顺手

题目已经说了，字符串由**有效 ASCII 字符**组成。

这意味着字符范围是固定的，所以没必要用 `HashMap<Character, Integer>`，直接开数组就行。

比如：

- `mapS[c]` 表示字符 `c` 在 `s` 中上一次出现的位置
- `mapT[c]` 表示字符 `c` 在 `t` 中上一次出现的位置

因为 ASCII 字符范围固定，所以数组长度开到 `256` 就够用了。

这种写法比哈希表更直接，访问也更快。

### 为什么数组里要存 `i + 1`，而不是直接存 `i`

这是这题里最容易忽略的一个细节。

数组初始值默认是 `0`。  
如果你直接存下标 `i`，那就会出现一个问题：

- 字符从来没出现过，对应值是 `0`
- 字符第一次出现在下标 `0`，对应值也是 `0`

这两种情况就分不清了。

所以这里通常会把“上一次出现的位置”存成 `i + 1`。

这样一来：

- `0` 表示从未出现过
- `1` 表示上一次出现在下标 `0`
- `2` 表示上一次出现在下标 `1`
- 以此类推

这样就不会混淆。

### 用示例走一遍过程

看示例 3：

```text
s = "paper"
t = "title"
```

我们从左到右看：

#### 第 0 位

- `p` 第一次出现
- `t` 也是第一次出现

两边记录的上一次出现位置都是 `0`，说明没问题。

然后更新：

- `p` 的最新位置记为 `1`
- `t` 的最新位置记为 `1`

#### 第 1 位

- `a` 第一次出现
- `i` 也是第一次出现

两边仍然一致，继续。

更新后：

- `a -> 2`
- `i -> 2`

#### 第 2 位

- `p` 之前出现过，上次位置是第 `0` 位，对应数组里存的是 `1`
- `t` 之前也出现过，上次位置也是第 `0` 位，对应数组里存的也是 `1`

两边相同，继续。

#### 第 3 位

- `e` 第一次出现
- `l` 也是第一次出现

一致，继续。

#### 第 4 位

- `r` 第一次出现
- `e` 也是第一次出现

还是一致。

遍历结束，没有发现冲突，所以返回 `true`。

### 代码

```java
public boolean isIsomorphic(String s, String t) {
    int[] mapS = new int[256];
    int[] mapT = new int[256];

    for (int i = 0; i < s.length(); i++) {
        char c1 = s.charAt(i);
        char c2 = t.charAt(i);

        if (mapS[c1] != mapT[c2]) {
            return false;
        }

        mapS[c1] = i + 1;
        mapT[c2] = i + 1;
    }

    return true;
}
```

### 这段代码到底在做什么

整段代码的核心其实就两步。

第一步，比较当前位置两个字符之前的出现位置：

```java
if (mapS[c1] != mapT[c2]) {
    return false;
}
```

如果不相等，说明它们的历史模式不一致。  
可能是一边重复了，另一边没重复；也可能是一边对应的是老字符，另一边对应的是新字符。只要出现这种情况，就不可能同构。

第二步，更新这两个字符的最新位置：

```java
mapS[c1] = i + 1;
mapT[c2] = i + 1;
```

表示它们最近一次都出现在当前位置。

整个过程其实就是不断检查：

**这两个字符串在每个位置上的“重复节奏”是否一致。**

### 这题最容易想错的地方

这题本身不难，但有几个地方很容易误判。

#### 第一个误区：只检查单向映射

比如只记录 `s -> t`，看起来好像够了，其实不够。

因为这样会漏掉这种情况：

```text
s = "ab"
t = "aa"
```

如果只看 `a -> a`、`b -> a`，单向上似乎都能记下来，但这明显不是一一对应，因为两个不同字符不能映射成同一个字符。

所以这题必须保证双向一致，或者像这道题这样，直接比较出现位置模式。

#### 第二个误区：把“字符相同”当成判断标准

同构不要求字符本身相同，只要求结构相同。

所以：

- `"egg"` 和 `"add"` 虽然字符不同，但结构一致，是同构
- `"foo"` 和 `"bar"` 虽然长度相同，但结构不一致，不同构

真正要看的是模式，不是字面值。

#### 第三个误区：忘了处理索引 0 的问题

如果不用 `i + 1`，而直接存 `i`，就会把“没出现过”和“出现在第 0 位”混在一起，这会导致判断失真。

这个细节虽然小，但非常关键。

### 复杂度为什么是 O(n)

设字符串长度为 `n`。

我们只遍历了一次字符串，每个位置做的事情都很固定：

- 读两个字符
- 查两次数组
- 写两次数组

所以时间复杂度就是 `O(n)`。

空间方面只用了两个固定长度为 `256` 的数组，因此空间复杂度是 `O(1)`。

这也是这题最简洁、最稳的一种写法。

### 这题真正值得记住的地方

这题真正值得记住的，不是“同构字符串”这个名字，而是这样一种判断思路：

**当题目本质上是在比较两个序列的结构是否一致时，可以考虑比较它们的出现模式，而不是直接比较值本身。**

本题里，字符是否相同并不重要，重要的是：

- 第一次出现的位置是否同步
- 重复出现的节奏是否一致

以后再遇到“模式匹配”“结构一致”“映射关系是否冲突”这类题，也可以先往这个方向想。

### 完整代码

```java
public class Main {

    public static boolean isIsomorphic(String s, String t) {
        if (s.length() != t.length()) {
            return false;
        }

        int[] mapS = new int[256];
        int[] mapT = new int[256];

        for (int i = 0; i < s.length(); i++) {
            char c1 = s.charAt(i);
            char c2 = t.charAt(i);

            if (mapS[c1] != mapT[c2]) {
                return false;
            }

            mapS[c1] = i + 1;
            mapT[c2] = i + 1;
        }

        return true;
    }

    public static void main(String[] args) {
        String s1 = "egg";
        String t1 = "add";
        System.out.println("示例 1 输入: s = \"" + s1 + "\", t = \"" + t1 + "\"");
        System.out.println("示例 1 输出: " + isIsomorphic(s1, t1));
        System.out.println("-----------------------------");

        String s2 = "foo";
        String t2 = "bar";
        System.out.println("示例 2 输入: s = \"" + s2 + "\", t = \"" + t2 + "\"");
        System.out.println("示例 2 输出: " + isIsomorphic(s2, t2));
        System.out.println("-----------------------------");

        String s3 = "paper";
        String t3 = "title";
        System.out.println("示例 3 输入: s = \"" + s3 + "\", t = \"" + t3 + "\"");
        System.out.println("示例 3 输出: " + isIsomorphic(s3, t3));
    }
}
```
