## 题目

**描述：**

给定一种规律 `pattern` 和一个字符串 `s` ，判断 `s` 是否遵循相同的规律。

这里的 **遵循** 指完全匹配，例如，`pattern` 里的每个字母和字符串 `s` 中的每个非空单词之间存在着双向连接的对应规律。具体来说：

- `pattern` 中的每个字母都 **恰好** 映射到 `s` 中的一个唯一单词。
- `s` 中的每个唯一单词都 **恰好** 映射到 `pattern` 中的一个字母。
- 没有两个字母映射到同一个单词，也没有两个单词映射到同一个字母。

**示例 1:**

- **输入:** `pattern = "abba"`, `s = "dog cat cat dog"`
- **输出:** `true`

**示例 2:**

- **输入:** `pattern = "abba"`, `s = "dog cat cat fish"`
- **输出:** `false`

**示例 3:**

- **输入:** `pattern = "aaaa"`, `s = "dog cat cat dog"`
- **输出:** `false`

**提示:**

- `1 <= pattern.length <= 300`
- `pattern` 只包含小写英文字母
- `1 <= s.length <= 3000`
- `s` 只包含小写英文字母和 `' '`
- `s` **不包含** 任何前导或尾随空格
- `s` 中每个单词都被 **单个空格** 分隔

## 题解

### 先把这题的要求看准

这题不是简单地看 `pattern` 和 `s` 能不能一一对应上，而是要看它们之间能不能建立**双向唯一映射**。

题目里的约束其实有两层：

- `pattern` 里的同一个字符，必须始终对应同一个单词
- `s` 里的同一个单词，也必须始终对应同一个字符

也就是说，这不是“单向能对上就行”，而是两边都不能乱。

比如：

- `pattern = "abba"`，`s = "dog cat cat dog"`，这是可以的，因为 `a -> dog`，`b -> cat`，前后一直一致
- `pattern = "abba"`，`s = "dog cat cat fish"`，这是不行的，因为最后一个 `a` 本来应该还是对应 `dog`，结果变成了 `fish`
- `pattern = "ab"`，`s = "dog dog"`，这也不行，因为两个不同字符不能映射到同一个单词

所以这题真正要判断的是：

**字符和单词之间的对应关系，能不能从头到尾保持一一对应。**

### 第一步其实很简单：先看数量对不对

在谈映射之前，先要确认一件最基本的事：  
`pattern` 里有几个字符，`s` 里就应该有几个单词。

如果数量都对不上，后面根本不用看，直接返回 `false`。

所以拿到题以后，第一步就是先把 `s` 按空格切开，得到单词数组 `words`，然后比较：

```java
pattern.length() == words.length
```

如果不相等，说明连位置都对不上，自然不可能匹配成功。

### 为什么这题必须做双向检查

很多人第一次做这题时，会先想到只维护一张映射表：

- `a -> dog`
- `b -> cat`

看起来好像就够了，其实不够。

因为只看“字符映射到单词”这一边，会漏掉一种冲突：

```text
pattern = "ab"
s = "dog dog"
```

如果只看左边到右边：

- `a -> dog`
- `b -> dog`

代码表面上都能记下来，但这显然不满足题意。因为不同字符不能映射到同一个单词。

所以这里一定要同时保证两件事：

- 一个字符不能对应多个单词
- 一个单词也不能被多个字符占用

这就是为什么这题最稳妥的写法，是维护两张表：

- `charToWord`：字符到单词
- `wordToChar`：单词到字符

只有双向都不冲突，映射关系才算成立。

### 这题的核心，不是建表，而是一路校验

这题的思路其实很直：

遍历 `pattern` 和 `words` 的每个位置，把当前位置的字符和单词拿出来配对。

假设当前是：

- 字符 `c`
- 单词 `w`

那么只需要检查下面几种情况。

#### 情况一：`c` 以前出现过

那就说明它之前已经绑定过某个单词了。  
这时候只要看它之前绑定的是不是 `w`。

- 如果是，说明没问题，继续
- 如果不是，说明同一个字符想对应两个不同单词，直接返回 `false`

#### 情况二：`c` 没出现过，但 `w` 出现过

这说明当前字符是新的，但这个单词已经被别的字符占用了。

这种情况同样不合法，因为不同字符不能映射到同一个单词，直接返回 `false`。

#### 情况三：两边都没出现过

那就说明这是一个全新的合法配对，可以把映射关系建立起来。

整个过程本质上就是：

**一边遍历，一边检查旧关系有没有被破坏；如果没有，就在合适的时候建立新关系。**

### 用示例走一遍

看示例 1：

```text
pattern = "abba"
s = "dog cat cat dog"
```

先把 `s` 切成单词数组：

```text
["dog", "cat", "cat", "dog"]
```

长度和 `pattern` 一样，说明可以继续。

接下来逐个看：

#### 第 0 位

- 字符是 `a`
- 单词是 `dog`

两边都是第一次出现，所以建立映射：

```text
a -> dog
dog -> a
```

#### 第 1 位

- 字符是 `b`
- 单词是 `cat`

也都是第一次出现，继续建立：

```text
b -> cat
cat -> b
```

#### 第 2 位

- 字符还是 `b`
- 单词还是 `cat`

`b` 之前映射的本来就是 `cat`，没问题，继续。

#### 第 3 位

- 字符是 `a`
- 单词是 `dog`

`a` 之前映射的本来就是 `dog`，还是没问题。

遍历结束，没有冲突，所以返回 `true`。

再看示例 2：

```text
pattern = "abba"
s = "dog cat cat fish"
```

前 3 位都没问题。  
到了最后一位：

- 当前字符是 `a`
- 当前单词是 `fish`

但 `a` 之前明明已经映射到 `dog` 了，现在却想映射到 `fish`，这就破坏了原来的关系，所以直接返回 `false`。

### 代码

```java
public boolean wordPattern(String pattern, String s) {
    String[] words = s.split(" ");

    if (pattern.length() != words.length) {
        return false;
    }

    Map<Character, String> charToWord = new HashMap<>();
    Map<String, Character> wordToChar = new HashMap<>();

    for (int i = 0; i < pattern.length(); i++) {
        char c = pattern.charAt(i);
        String w = words[i];

        if (charToWord.containsKey(c)) {
            if (!charToWord.get(c).equals(w)) {
                return false;
            }
        } else {
            if (wordToChar.containsKey(w)) {
                return false;
            }

            charToWord.put(c, w);
            wordToChar.put(w, c);
        }
    }

    return true;
}
```

### 这段代码到底在做什么

整段代码可以分成三步来看。

第一步，先拆分单词并检查长度：

```java
String[] words = s.split(" ");
if (pattern.length() != words.length) {
    return false;
}
```

这是最基础的对位检查。  
因为后面所有映射，都是按下标一一对应来判断的。

第二步，遍历每一组字符和单词：

```java
char c = pattern.charAt(i);
String w = words[i];
```

把当前位置的配对拿出来。

第三步，检查和维护双向映射：

- 如果字符 `c` 以前出现过，就看它映射的是不是 `w`
- 如果字符 `c` 没出现过，那就得保证单词 `w` 也没被别的字符占用
- 两边都没问题，才建立新映射

所以这段代码真正做的事，不是单纯“存关系”，而是：

**不断验证已有关系是否一致，并防止新关系和旧关系冲突。**

### 这题最容易出错的地方

这题不难，但有几个地方很容易写偏。

#### 第一个误区：只维护一张映射表

只写 `char -> word` 往往不够，因为你还得防止多个字符映射到同一个单词。

这也是为什么一定要做双向检查。

#### 第二个误区：忘了先判断长度

如果 `pattern` 比单词数多，或者单词数比 `pattern` 多，后面根本不可能逐位匹配。  
这一步应该尽早做掉，不然逻辑会变得别扭。

#### 第三个误区：把“新字符 + 已有单词”当成合法情况

比如：

```text
pattern = "ab"
s = "dog dog"
```

这里第二个字符 `b` 是新的，但单词 `dog` 已经被 `a` 用过了，所以这不是新映射，而是冲突。

这个细节很常见，也最容易漏掉。

### 复杂度为什么是 O(N + M)

设：

- `N` 是 `pattern` 的长度
- `M` 是字符串 `s` 的长度

先执行一次 `split`，这一步需要扫描整个字符串，所以是 `O(M)`。  
之后再遍历一次 `pattern` 和单词数组，代价是 `O(N)`。

哈希表的查询和插入平均都可以看作 `O(1)`，所以总时间复杂度就是：

- 时间复杂度：`O(N + M)`

空间方面：

- `split` 之后需要保存单词数组
- 还要维护两张哈希表

所以空间复杂度是 `O(M)` 量级。

### 这题真正值得记住的地方

这题真正值得记住的，不是“单词规律”这个题面，而是这种判断模型：

**当题目要求两个序列之间建立一一对应关系时，一定要警惕“单向成立但双向冲突”的情况。**

很多看起来像映射的问题，真正难点都不在“能不能记下来”，而在：

- 旧关系会不会被打破
- 新关系会不会占用别人已经用过的位置

把这层想清楚，以后再遇到模式匹配、双射关系、结构对应这类题，思路都会更顺。

### 完整代码

```java
import java.util.HashMap;
import java.util.Map;

public class Main {

    public static boolean wordPattern(String pattern, String s) {
        String[] words = s.split(" ");

        if (pattern.length() != words.length) {
            return false;
        }

        Map<Character, String> charToWord = new HashMap<>();
        Map<String, Character> wordToChar = new HashMap<>();

        for (int i = 0; i < pattern.length(); i++) {
            char c = pattern.charAt(i);
            String w = words[i];

            if (charToWord.containsKey(c)) {
                if (!charToWord.get(c).equals(w)) {
                    return false;
                }
            } else {
                if (wordToChar.containsKey(w)) {
                    return false;
                }

                charToWord.put(c, w);
                wordToChar.put(w, c);
            }
        }

        return true;
    }

    public static void main(String[] args) {
        String pattern1 = "abba";
        String s1 = "dog cat cat dog";
        System.out.println("示例 1 输入: pattern = \"" + pattern1 + "\", s = \"" + s1 + "\"");
        System.out.println("示例 1 输出: " + wordPattern(pattern1, s1));
        System.out.println("-----------------------------");

        String pattern2 = "abba";
        String s2 = "dog cat cat fish";
        System.out.println("示例 2 输入: pattern = \"" + pattern2 + "\", s = \"" + s2 + "\"");
        System.out.println("示例 2 输出: " + wordPattern(pattern2, s2));
        System.out.println("-----------------------------");

        String pattern3 = "aaaa";
        String s3 = "dog cat cat dog";
        System.out.println("示例 3 输入: pattern = \"" + pattern3 + "\", s = \"" + s3 + "\"");
        System.out.println("示例 3 输出: " + wordPattern(pattern3, s3));
        System.out.println("-----------------------------");

        String pattern4 = "aaa";
        String s4 = "aa aa aa aa";
        System.out.println("额外示例 输入: pattern = \"" + pattern4 + "\", s = \"" + s4 + "\"");
        System.out.println("额外示例 输出: " + wordPattern(pattern4, s4));
    }
}
```
