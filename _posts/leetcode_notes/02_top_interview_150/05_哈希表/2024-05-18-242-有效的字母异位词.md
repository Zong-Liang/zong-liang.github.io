## 题目

**描述：**

给定两个字符串 `s` 和 `t` ，编写一个函数来判断 `t` 是否是 `s` 的字母异位词。

**字母异位词** 是由相同字母按不同顺序排列形成的字符串。

**示例 1:**

- **输入:** `s = "anagram"`, `t = "nagaram"`
- **输出:** `true`

**示例 2:**

- **输入:** `s = "rat"`, `t = "car"`
- **输出:** `false`

**提示:**

- `1 <= s.length, t.length <= 5 * 10^4`
- `s` 和 `t` 仅包含小写字母

**进阶:** 如果输入字符串包含 unicode 字符怎么办？你能否调整你的解法来应对这种情况？

## 题解

### 先看这题到底在比什么

这题看起来是在比较两个字符串，实际上比较的不是顺序，而是**每个字符出现的次数是否完全一致**。

“字母异位词”说白了就是：

- 用到的字符种类一样
- 每种字符出现的次数一样
- 顺序可以不同

比如：

- `"anagram"` 和 `"nagaram"` 是异位词，因为两边的字母种类和数量完全一致
- `"rat"` 和 `"car"` 不是，因为前者里有 `t`，后者里有 `c`

所以这题真正要判断的，不是谁排在前谁排在后，而是：

**两个字符串的字符频次表能不能完全对上。**

### 第一步其实很简单：长度不一样，直接不用看了

如果两个字符串长度都不相等，那它们不可能是字母异位词。

因为异位词只是重新排列，不会凭空多一个字符，也不会少一个字符。

所以第一步先判断：

```java
if (s.length() != t.length()) {
    return false;
}
```

这个判断非常自然，而且能提前排除很多不可能的情况。

### 这题最适合的思路，就是做频次统计

既然要比较字符出现次数，那最直接的办法就是统计。

做法很简单：

- 遍历 `s`，把每个字符出现次数加一
- 遍历 `t`，把每个字符出现次数减一
- 最后如果所有位置都刚好归零，说明两边完全匹配

题目已经限定字符串只包含**小写字母**，所以这里完全没必要用哈希表，直接开一个长度为 `26` 的数组就够了。

比如：

- `table[0]` 表示 `'a'` 的出现次数
- `table[1]` 表示 `'b'` 的出现次数
- ...
- `table[25]` 表示 `'z'` 的出现次数

字符对应的下标就是：

```java
c - 'a'
```

这种写法很直接，也正是这题最常规、最高效的解法。

### 为什么一边加、一边减就够了

这个思路的关键在于：  
我们并不需要分别把两个字符串的统计结果都算出来再比较。

因为最终要的只是“差值是否为 0”。

所以完全可以在同一轮循环里做两件事：

- 对 `s.charAt(i)` 对应的位置加一
- 对 `t.charAt(i)` 对应的位置减一

这样遍历结束后：

- 如果两个字符串里某个字符次数一样，那对应位置一定会被抵消成 `0`
- 如果某个字符次数不一样，那对应位置最后一定不是 `0`

这种写法比“先统计两个数组再比较”更紧凑一些。

### 用示例走一遍过程

看示例 1：

```text
s = "anagram"
t = "nagaram"
```

一开始数组全是 `0`。

我们从左到右同时处理两个字符串。

#### 第 0 位

- `s[0] = 'a'`，对应位置 `+1`
- `t[0] = 'n'`，对应位置 `-1`

#### 第 1 位

- `s[1] = 'n'`，对应位置 `+1`
- `t[1] = 'a'`，对应位置 `-1`

到这里你会发现，前两轮其实已经把 `a` 和 `n` 抵消掉了。

后面继续这样处理，整个字符串遍历结束后，所有字符的增减都会刚好相互抵消，所以数组里每个位置最终都是 `0`，返回 `true`。

再看示例 2：

```text
s = "rat"
t = "car"
```

处理完之后：

- `r` 可以抵消
- `a` 可以抵消
- 但 `t` 和 `c` 对不上

所以数组里一定会有非零项，结果就是 `false`。

### 代码

```java
public boolean isAnagram(String s, String t) {
    if (s.length() != t.length()) {
        return false;
    }

    int[] table = new int[26];

    for (int i = 0; i < s.length(); i++) {
        table[s.charAt(i) - 'a']++;
        table[t.charAt(i) - 'a']--;
    }

    for (int count : table) {
        if (count != 0) {
            return false;
        }
    }

    return true;
}
```

### 这段代码到底在做什么

可以把它分成两部分来看。

第一部分是长度检查：

```java
if (s.length() != t.length()) {
    return false;
}
```

只要长度不一样，就不可能互为异位词。

第二部分是统计差值：

```java
for (int i = 0; i < s.length(); i++) {
    table[s.charAt(i) - 'a']++;
    table[t.charAt(i) - 'a']--;
}
```

这一步不是在分别统计，而是在直接比较两边的频次差。

最后这一段：

```java
for (int count : table) {
    if (count != 0) {
        return false;
    }
}
```

就是在验证差值是否已经全部归零。

全部归零，说明两边频次完全一致；  
只要有一个位置不是 `0`，说明至少有一种字符的数量不同。

### 为什么这里不用排序

这题还有一种很常见的做法：先把两个字符串都转成字符数组，再排序，最后比较排序后的结果是否相同。

这个思路当然也能做，但不如计数法合适。

原因很简单：

- 排序的时间复杂度是 `O(n log n)`
- 计数法的时间复杂度是 `O(n)`

而且这题字符范围已经固定为 26 个小写字母，计数数组几乎就是为这种题准备的。  
所以这里最自然、也最推荐的就是频次统计。

### 进阶：如果字符串里不再只是小写字母怎么办

题目最后还问了一个进阶问题：  
如果字符串包含 Unicode 字符，该怎么处理？

这时候长度为 `26` 的数组就不够用了，因为字符集已经不再固定。  
比如中文、日文、emoji，都不可能简单地映射到 `0 ~ 25` 之间。

这时思路不变，还是统计频次，只是统计工具要从数组改成哈希表。

不过这里有个细节要注意：  
如果真想完整支持 Unicode，在 Java 里不能直接按 `char` 来处理。因为 `char` 只能表示 UTF-16 的一个代码单元，对于某些字符，比如 emoji，实际会占两个 `char`。

所以更稳妥的写法，是按**Unicode 码点**遍历字符串，再用 `Map<Integer, Integer>` 做统计。

### Unicode 写法的思路

处理方式还是一样：

- 先判断两个字符串的码点数量是否相同
- 遍历 `s` 的所有码点，做 `+1`
- 遍历 `t` 的所有码点，做 `-1`
- 中途如果某个码点减完后小于 `0`，说明 `t` 里这个字符比 `s` 多，直接返回 `false`

这里之所以用“码点数量”而不是 `length()`，就是为了避免代理对字符把长度判断搞错。

### Unicode 写法的代码

```java
public boolean isAnagramUnicode(String s, String t) {
    if (s.codePointCount(0, s.length()) != t.codePointCount(0, t.length())) {
        return false;
    }

    Map<Integer, Integer> map = new HashMap<>();

    s.codePoints().forEach(cp -> map.put(cp, map.getOrDefault(cp, 0) + 1));

    for (int cp : t.codePoints().toArray()) {
        int count = map.getOrDefault(cp, 0);
        if (count == 0) {
            return false;
        }
        map.put(cp, count - 1);
    }

    return true;
}
```

这个写法的核心变化只有一个：  
**统计单位从 `char` 变成了 Unicode 码点。**

### 这题最容易犯的误区

这题本身不难，但有几个地方很容易想偏。

#### 第一个误区：只比较字符种类，不比较次数

比如有人会觉得，只要两个字符串里出现过的字符集合一样就行。

这是不够的。  
因为 `"aab"` 和 `"abb"` 虽然字符种类都是 `{a, b}`，但数量明显不同，所以不是异位词。

真正要比较的是频次，不是集合。

#### 第二个误区：进阶题里继续用 `char` 处理所有 Unicode

如果只是普通英文字母，用 `char` 没问题。  
但如果真的扩展到完整 Unicode，尤其是涉及补充平面的字符，`char` 就不够严谨了。

所以进阶部分更稳妥的写法，应该是按码点处理。

#### 第三个误区：遇到这类题下意识先排序

排序能做，但不是最优。  
这类“频次完全一致”的题，优先应该想到计数。

### 复杂度怎么理解

对于题目原始条件下的数组解法：

- 时间复杂度：`O(n)`
- 空间复杂度：`O(1)`

因为只遍历了一遍字符串，额外空间只是固定大小的 `26` 个整型位置。

对于 Unicode 的哈希表解法：

- 时间复杂度：平均 `O(n)`
- 空间复杂度：`O(k)`，其中 `k` 是不同字符的种类数

两种写法的核心其实一样，区别只是字符集范围不同，所以选择的数据结构不同。

### 这题真正值得记住的地方

这题真正值得记住的，不是“字母异位词”这个名字，而是这样一种判断模型：

**当题目要比较两个序列是否只是顺序不同、本质组成相同时，优先考虑做频次统计。**

如果元素范围固定，就用数组；  
如果元素范围不固定，就用哈希表；  
如果题目还涉及完整 Unicode，要注意统计单位是不是应该提升到码点层面。

把这条思路记住，后面遇到字符计数、元素抵消、频次匹配这类题，基本都会很顺。

### 完整代码

```java
import java.util.HashMap;
import java.util.Map;

public class Main {

    // 适用于题目原始条件：仅包含小写字母
    public static boolean isAnagram(String s, String t) {
        if (s.length() != t.length()) {
            return false;
        }

        int[] table = new int[26];

        for (int i = 0; i < s.length(); i++) {
            table[s.charAt(i) - 'a']++;
            table[t.charAt(i) - 'a']--;
        }

        for (int count : table) {
            if (count != 0) {
                return false;
            }
        }

        return true;
    }

    // 适用于完整 Unicode 字符
    public static boolean isAnagramUnicode(String s, String t) {
        if (s.codePointCount(0, s.length()) != t.codePointCount(0, t.length())) {
            return false;
        }

        Map<Integer, Integer> map = new HashMap<>();

        s.codePoints().forEach(cp -> map.put(cp, map.getOrDefault(cp, 0) + 1));

        for (int cp : t.codePoints().toArray()) {
            int count = map.getOrDefault(cp, 0);
            if (count == 0) {
                return false;
            }
            map.put(cp, count - 1);
        }

        return true;
    }

    public static void main(String[] args) {
        String s1 = "anagram";
        String t1 = "nagaram";
        System.out.println("示例 1 输入: s = \"" + s1 + "\", t = \"" + t1 + "\"");
        System.out.println("示例 1 输出: " + isAnagram(s1, t1));
        System.out.println("-----------------------------");

        String s2 = "rat";
        String t2 = "car";
        System.out.println("示例 2 输入: s = \"" + s2 + "\", t = \"" + t2 + "\"");
        System.out.println("示例 2 输出: " + isAnagram(s2, t2));
        System.out.println("-----------------------------");

        String s3 = "你好世界";
        String t3 = "世界你好";
        System.out.println("进阶示例 输入: s = \"" + s3 + "\", t = \"" + t3 + "\"");
        System.out.println("进阶示例 输出: " + isAnagramUnicode(s3, t3));
        System.out.println("-----------------------------");

        String s4 = "a😊b";
        String t4 = "😊ab";
        System.out.println("补充示例 输入: s = \"" + s4 + "\", t = \"" + t4 + "\"");
        System.out.println("补充示例 输出: " + isAnagramUnicode(s4, t4));
    }
}
```
