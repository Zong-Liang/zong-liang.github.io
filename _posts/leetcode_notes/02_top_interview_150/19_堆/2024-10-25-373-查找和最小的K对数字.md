## 题目

**描述：**

给定两个以 **非递减顺序** 排列的整数数组 `nums1` 和 `nums2`，以及一个整数 `k`。

定义一对值 `(u,v)`，其中第一个元素来自 `nums1`，第二个元素来自 `nums2`。

请找到和最小的 `k` 个数对 `(u1,v1), (u2,v2) ... (uk,vk)`。

**示例 1:**

- **输入:** `nums1 = [1,7,11]`, `nums2 = [2,4,6]`, `k = 3`
- **输出:** `[[1,2],[1,4],[1,6]]`
- **解释:** 返回序列中的前 3 对数：

  `[1,2],[1,4],[1,6],[7,2],[7,4],[11,2],[7,6],[11,4],[11,6]`

**示例 2:**

- **输入:** `nums1 = [1,1,2]`, `nums2 = [1,2,3]`, `k = 2`
- **输出:** `[[1,1],[1,1]]`
- **解释:** 返回序列中的前 2 对数：

  `[1,1],[1,1],[1,2],[2,1],[1,2],[2,2],[1,3],[1,3],[2,3]`

**提示:**

- `1 <= nums1.length, nums2.length <= 10^5`
- `-10^9 <= nums1[i], nums2[i] <= 10^9`
- `nums1` 和 `nums2` 均为升序排列
- `1 <= k <= 10^4`
- `k <= nums1.length * nums2.length`

## 题解

### 先别急着把所有数对都列出来

这题最容易想到的办法，就是把 `nums1[i]` 和 `nums2[j]` 的所有组合都枚举出来，算出它们的和，再整体排序，最后取前 `k` 个。

思路当然没问题，但数据范围一摆出来，这个方法基本就不用想了。

因为两个数组长度都可能到 `10^5`，数对总数最多会到 `10^{10}`` 级别，不管是时间还是空间都扛不住。  
所以这题肯定不能真的把所有数对都生成出来，而是要想办法：

**只按从小到大的顺序，把前 `k` 个数对依次找出来。**

这才是这道题真正的方向。

### 把它想成很多行有序序列

这题两个数组都是升序的，这是最关键的信息。

假设：

```text
nums1 = [1,7,11]
nums2 = [2,4,6]
```

那么所有数对可以按 `nums1` 的下标拆成几行：

```text
第 0 行： (1,2), (1,4), (1,6)
第 1 行： (7,2), (7,4), (7,6)
第 2 行： (11,2), (11,4), (11,6)
```

如果只看每一行里的“和”：

```text
第 0 行： 3, 5, 7
第 1 行： 9, 11, 13
第 2 行： 13, 15, 17
```

可以发现，每一行本身都是递增的。  
原因很简单：`nums1[i]` 固定不变，而 `nums2[j]` 是升序，所以 `nums1[i] + nums2[j]` 也会随着 `j` 的增加而变大。

这样一来，这题就不再像“暴力枚举所有组合”，而更像是：

**从多行有序序列里，依次取出最小的前 `k` 个元素。**

这个角度一出来，解法就比较自然了。

### 这其实就是多路归并

每一行都已经有序，那我们每次只需要知道：

**当前所有行的行首元素里，哪个最小。**

拿到最小的那个之后，再把它所在行的下一个元素补进来，继续比较。

这和“合并多个有序链表”是一个套路，所以这题最常见的解法就是：

- 用一个**小顶堆**维护当前候选数对
- 每次弹出最小和的那个数对
- 再把它所在行的下一个数对加入堆中

整个过程，就是一个标准的多路归并。

### 堆里放什么最合适

这里没必要把真正的数对对象设计得太复杂，直接存下标就够了。

因为只要知道一对下标 `(i, j)`，就能立刻得到它对应的数对：

```text
(nums1[i], nums2[j])
```

也能算出它们的和：

```text
nums1[i] + nums2[j]
```

所以堆里通常会放一个长度为 3 的数组：

```text
[当前和, i, j]
```

这样比较堆顶时只看“当前和”，真正加入答案时再根据 `i` 和 `j` 取值即可。

### 为什么初始化时只放每一行的第一个

既然每一行都是递增的，那么每一行最有机会进入答案的，一定是它的第一个数对：

```text
(nums1[i], nums2[0])
```

所以初始化时，我们只需要把这些“每行的第一个元素”放进堆里。

接下来谁最小，谁先出堆；某一行的第一个元素出堆后，再把这一行的第二个元素补进去；第二个出堆后，再补第三个。

这样做的好处是，我们始终只维护“当前有资格竞争最小值”的那些候选，不会把一整行全塞进堆里。

另外，还有一个常见优化：

初始化时只需要放前 `min(nums1.length, k)` 行的第一个元素。

因为我们最多只取 `k` 个答案，如果某一行已经排到第 `k` 行之后，那它的首元素通常也不可能比前面这些更早进入前 `k` 小结果。就算不加这个优化也能做，但加上会更稳一点。

### 为什么每次只往右补一个就够了

假设当前弹出的是第 `i` 行第 `j` 列，也就是：

```text
(nums1[i], nums2[j])
```

那下一步，只需要把同一行右边那个数对：

```text
(nums1[i], nums2[j + 1])
```

加入堆里。

原因是这一行本来就是递增的。  
既然 `(i, j)` 已经被取走了，那么这一行里下一个最有资格参与竞争的，自然就是 `(i, j + 1)`。

至于 `(i, j + 2)`、`(i, j + 3)` 这些更靠后的元素，现在都还轮不到，它们一定不会比 `(i, j + 1)` 更早进入答案。

所以每次只补一个，不会漏，也不会乱。

### 用示例走一遍

看示例 1：

```text
nums1 = [1,7,11]
nums2 = [2,4,6]
k = 3
```

初始化时，把每一行的第一个数对加入小顶堆：

```text
(1,2)  和为 3
(7,2)  和为 9
(11,2) 和为 13
```

#### 第 1 次弹出

堆顶是 `(1,2)`，它的和最小，加入答案：

```text
[[1,2]]
```

然后把同一行的下一个数对 `(1,4)` 放入堆中。

#### 第 2 次弹出

现在堆里大概是：

```text
(1,4)  和为 5
(7,2)  和为 9
(11,2) 和为 13
```

堆顶变成 `(1,4)`，加入答案：

```text
[[1,2],[1,4]]
```

再把这一行的下一个 `(1,6)` 放进去。

#### 第 3 次弹出

现在堆里最小的是 `(1,6)`，加入答案：

```text
[[1,2],[1,4],[1,6]]
```

已经取满 `k = 3` 个，结束。

这个过程正好对应题目答案。

### 代码

```java
public List<List<Integer>> kSmallestPairs(int[] nums1, int[] nums2, int k) {
    List<List<Integer>> res = new ArrayList<>();

    if (nums1 == null || nums1.length == 0 || nums2 == null || nums2.length == 0 || k <= 0) {
        return res;
    }

    PriorityQueue<int[]> pq = new PriorityQueue<>((a, b) -> a[0] - b[0]);

    for (int i = 0; i < Math.min(nums1.length, k); i++) {
        pq.offer(new int[]{nums1[i] + nums2[0], i, 0});
    }

    while (k > 0 && !pq.isEmpty()) {
        int[] curr = pq.poll();
        int i = curr[1];
        int j = curr[2];

        res.add(Arrays.asList(nums1[i], nums2[j]));

        if (j + 1 < nums2.length) {
            pq.offer(new int[]{nums1[i] + nums2[j + 1], i, j + 1});
        }

        k--;
    }

    return res;
}
```

整段代码真正关键的地方，其实就两句：

- 初始化时，把每一行的第一个元素入堆
- 弹出一个之后，只补这一行右边的下一个元素

理解了这两步，这题就不会乱。

### 复杂度分析

- 时间复杂度：$O(k \log k)$

堆里最多维护 `min(k, nums1.length)` 个元素。  
每次弹出和插入的复杂度是 $O(\log k)$，总共最多进行 `k` 轮，所以整体是 $O(k \log k)$。

- 空间复杂度：$O(k)$

主要是优先队列占用的空间。

### 这题真正值得记住的地方

这题最值得记住的，不只是“小顶堆”这三个字，而是这种判断方式：

**如果题目要求从多个有序来源中，按顺序不断取最小值，那么往往可以往“多路归并 + 堆”这个方向去想。**

这类“来源有序、答案只取前 `k` 个、不需要把所有结果都展开”的题，很多都能套到这个模型里。

这比单独记住一道题更有用。

### 完整代码

```java
import java.util.ArrayList;
import java.util.Arrays;
import java.util.List;
import java.util.PriorityQueue;

public class Main {

    public static List<List<Integer>> kSmallestPairs(int[] nums1, int[] nums2, int k) {
        List<List<Integer>> res = new ArrayList<>();

        if (nums1 == null || nums1.length == 0 || nums2 == null || nums2.length == 0 || k <= 0) {
            return res;
        }

        PriorityQueue<int[]> pq = new PriorityQueue<>((a, b) -> a[0] - b[0]);

        for (int i = 0; i < Math.min(nums1.length, k); i++) {
            pq.offer(new int[]{nums1[i] + nums2[0], i, 0});
        }

        while (k > 0 && !pq.isEmpty()) {
            int[] curr = pq.poll();
            int i = curr[1];
            int j = curr[2];

            res.add(Arrays.asList(nums1[i], nums2[j]));

            if (j + 1 < nums2.length) {
                pq.offer(new int[]{nums1[i] + nums2[j + 1], i, j + 1});
            }

            k--;
        }

        return res;
    }

    public static void main(String[] args) {
        int[] nums1_1 = {1, 7, 11};
        int[] nums2_1 = {2, 4, 6};
        int k1 = 3;

        System.out.println("示例 1 输入: nums1 = " + Arrays.toString(nums1_1)
                + ", nums2 = " + Arrays.toString(nums2_1) + ", k = " + k1);
        List<List<Integer>> res1 = kSmallestPairs(nums1_1, nums2_1, k1);
        System.out.println("示例 1 输出: " + res1);
        System.out.println("-----------------------------");

        int[] nums1_2 = {1, 1, 2};
        int[] nums2_2 = {1, 2, 3};
        int k2 = 2;

        System.out.println("示例 2 输入: nums1 = " + Arrays.toString(nums1_2)
                + ", nums2 = " + Arrays.toString(nums2_2) + ", k = " + k2);
        List<List<Integer>> res2 = kSmallestPairs(nums1_2, nums2_2, k2);
        System.out.println("示例 2 输出: " + res2);
        System.out.println("-----------------------------");

        int[] nums1_3 = {1, 2};
        int[] nums2_3 = {3, 4};
        int k3 = 3;

        System.out.println("示例 3 输入: nums1 = " + Arrays.toString(nums1_3)
                + ", nums2 = " + Arrays.toString(nums2_3) + ", k = " + k3);
        List<List<Integer>> res3 = kSmallestPairs(nums1_3, nums2_3, k3);
        System.out.println("示例 3 输出: " + res3);
    }
}
```
