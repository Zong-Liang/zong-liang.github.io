## 题目

**描述：**

假设力扣（LeetCode）即将开始 **IPO**。为了以更高的价格将股票卖给风险投资公司，力扣希望在 IPO 之前开展一些项目以增加其资本。由于资源有限，它只能在 IPO 之前完成最多 `k` 个不同的项目。帮助力扣设计完成最多 `k` 个不同项目后得到最大总资本的方式。

给你 `n` 个项目。对于每个项目 `i`，它都有一个纯利润 `profits[i]` 和启动该项目需要的最小资本 `capital[i]`。

最初，你的资本为 `w`。当你完成一个项目时，你将获得纯利润，且利润将被添加到你的总资本中。

总而言之，从给定项目中选择 **最多** `k` 个不同项目的列表，以 **最大化最终资本**，并输出最终可获得的最多的资本。

答案保证在 32 位有符号整数范围内。

**示例 1:**

- **输入:** `k = 2`, `w = 0`, `profits = [1,2,3]`, `capital = [0,1,1]`
- **输出:** 4
- **解释:**
  - 由于你的初始资本为 0，你仅可以从 0 号项目开始。
  - 在完成后，你将获得 1 的利润，你的总资本将变为 1。
  - 此时你可以选择开始 1 号或 2 号项目。
  - 由于你最多可以选择两个项目，所以你需要完成 2 号项目以获得最大的资本。
  - 因此，输出最后最大化的资本，为 0 + 1 + 3 = 4。

**示例 2:**

- **输入:** `k = 3`, `w = 0`, `profits = [1,2,3]`, `capital = [0,1,2]`
- **输出:** 6

**提示:**

- `1 <= k <= 10^5`
- `0 <= w <= 10^9`
- `n == profits.length`
- `n == capital.length`
- `1 <= n <= 10^5`
- `0 <= profits[i] <= 10^4`
- `0 <= capital[i] <= 10^9`

## 题解

### 先把这题的选择逻辑想明白

这题每做完一个项目，手里的资本都会增加；而资本一增加，下一轮能选的项目范围也会跟着变大。

所以问题的核心其实是：

**在当前资金允许启动的项目里，怎么选，才能让后面的选择空间尽可能大？**

答案其实很直接：  
既然做完项目后，资本会立刻增加，那当前这一轮只要能做，就应该优先选**利润最大的项目**。

原因也不复杂。  
在同样做一个项目的前提下，利润越大，下一轮手里的钱就越多；钱越多，后面能解锁的项目就越多。也就是说，在当前可做项目里选利润最大的那个，不会让未来变差，只会让后面的选择更宽。

这就是这题的贪心核心。

### 但光知道贪心还不够

思路有了，接下来要解决的是怎么高效实现。

如果每一轮都做两件事：

1. 扫一遍所有项目，找出当前资金能做的项目
2. 再从里面挑利润最大的一个

那时间复杂度会比较难看。  
因为最多要做 `k` 轮，而每轮如果都扫完整个数组，最坏可能到 `O(k \cdot n)`，数据范围一大就不太行了。

所以这题真正的难点，不在“要不要贪心”，而在：

**怎么快速找到当前能做的项目里利润最大的那个。**

### 两个步骤配合起来就够了

这题常规写法里，会用到两个结构：

- 一个按启动资本排序的项目列表
- 一个大顶堆

分工很明确。

#### 第一部分：先按启动资本排序

我们先把每个项目整理成一对数据：

- 启动需要的资本
- 完成后能拿到的利润

然后按启动资本从小到大排序。

这样做的好处是，随着手里资本不断变多，我们就可以按顺序把“现在已经做得起的项目”逐个放进候选集合，而不需要每一轮都把所有项目重新扫描一遍。

#### 第二部分：用大顶堆维护当前可做项目

当某个项目的启动资本不超过当前资金时，就说明它已经进入“可以考虑”的范围了。

这些当前能做的项目里，我们每次只关心一件事：  
**利润最大的那个是谁。**

这正好适合用大顶堆维护。  
每解锁一个新项目，就把它的利润放进堆里；每轮真正做项目时，就从堆顶取出最大利润。

这样整个流程就顺了：

- 排序负责告诉我们“哪些项目现在已经买得起”
- 大顶堆负责告诉我们“买得起的项目里哪个最赚”

### 用题目的例子走一遍

看示例 1：

```text
k = 2, w = 0
profits = [1,2,3]
capital = [0,1,1]
```

先把项目整理一下：

```text
(0,1), (1,2), (1,3)
```

这里每一对数的含义是：

```text
(启动所需资本, 利润)
```

按启动资本排序后，其实顺序不变。

#### 第 1 轮

当前资金 `w = 0`。

只有启动资本不超过 `0` 的项目能做，也就是：

```text
(0,1)
```

把它的利润 `1` 放进大顶堆。  
这时堆里只有一个数，那就只能选它。

做完后：

```text
w = 0 + 1 = 1
```

#### 第 2 轮

现在资金变成了 `1`。

于是所有启动资本不超过 `1` 的项目都可以加入候选，也就是：

```text
(1,2), (1,3)
```

把利润 `2` 和 `3` 放进大顶堆。  
此时堆顶是 `3`，说明当前最该做的是利润为 `3` 的项目。

做完后：

```text
w = 1 + 3 = 4
```

总共最多做 `2` 个项目，结束。最终答案就是 `4`。

这个过程其实已经把整道题说明白了：

- 先不断解锁当前做得起的项目
- 再从这些项目里选利润最大的那个

### 为什么这样选一定合理

这题最容易让人犹豫的地方是：  
“当前可做项目里直接选利润最大，会不会太贪了？”

其实这里的贪心是成立的，因为这题里做项目带来的唯一影响，就是让资本变大，不存在什么额外代价，也没有“做了 A 就不能做 B”这种复杂限制。

换句话说，在同一轮里：

- 选利润较小的项目，只会让你下一轮的资本更少
- 选利润较大的项目，只会让你下一轮的资本更多

而资本更多，不会减少你的选择，只会让你能做的项目更多。

所以当前能做的项目里，优先拿最大利润，是完全合理的。

### 代码怎么写

先把项目和利润绑定起来，再排序：

```java
int[][] projects = new int[n][2];
for (int i = 0; i < n; i++) {
    projects[i][0] = capital[i];
    projects[i][1] = profits[i];
}

Arrays.sort(projects, (a, b) -> Integer.compare(a[0], b[0]));
```

这里 `projects[i][0]` 表示启动资本，`projects[i][1]` 表示利润。

然后用一个大顶堆保存当前能做项目的利润：

```java
PriorityQueue<Integer> maxHeap = new PriorityQueue<>((a, b) -> b - a);
```

接下来最多循环 `k` 次：

- 先把所有“当前资金能启动”的项目利润加入堆中
- 如果堆是空的，说明已经没有项目可做，直接结束
- 否则取出堆顶，把最大利润加到当前资金里

代码如下：

```java
public int findMaximizedCapital(int k, int w, int[] profits, int[] capital) {
    int n = profits.length;
    int[][] projects = new int[n][2];

    for (int i = 0; i < n; i++) {
        projects[i][0] = capital[i];
        projects[i][1] = profits[i];
    }

    Arrays.sort(projects, (a, b) -> Integer.compare(a[0], b[0]));

    PriorityQueue<Integer> maxHeap = new PriorityQueue<>((a, b) -> b - a);

    int i = 0;
    for (int step = 0; step < k; step++) {
        while (i < n && projects[i][0] <= w) {
            maxHeap.offer(projects[i][1]);
            i++;
        }

        if (maxHeap.isEmpty()) {
            break;
        }

        w += maxHeap.poll();
    }

    return w;
}
```

### 复杂度分析

- 时间复杂度：$O(n \log n + k \log n)$

排序需要 $O(n \log n)$。  
每个项目最多入堆一次，最多进行 `k` 次出堆操作，每次堆操作的复杂度是 $O(\log n)$。

- 空间复杂度：$O(n)$

主要是项目数组和优先队列占用的空间。

### 这题真正值得记住的点

这题表面上是堆题，但本质上是一个很典型的“**排序 + 动态解锁 + 堆维护最优候选**”模型。

以后如果再碰到这类题：

- 有一批对象，带有“门槛条件”
- 你当前状态变强后，会解锁更多对象
- 每一轮都要在“当前可选集合”里拿最优的那个

基本都可以往这个方向想：

1. 先按门槛排序
2. 再用堆维护当前可选项里的最优值

这类套路在题目里出现频率很高，记住比单独记这道题更有用。

### 完整代码

```java
import java.util.Arrays;
import java.util.PriorityQueue;

public class Main {

    public static int findMaximizedCapital(int k, int w, int[] profits, int[] capital) {
        int n = profits.length;

        int[][] projects = new int[n][2];
        for (int i = 0; i < n; i++) {
            projects[i][0] = capital[i];
            projects[i][1] = profits[i];
        }

        Arrays.sort(projects, (a, b) -> Integer.compare(a[0], b[0]));

        PriorityQueue<Integer> maxHeap = new PriorityQueue<>((a, b) -> b - a);

        int i = 0;
        for (int step = 0; step < k; step++) {
            while (i < n && projects[i][0] <= w) {
                maxHeap.offer(projects[i][1]);
                i++;
            }

            if (maxHeap.isEmpty()) {
                break;
            }

            w += maxHeap.poll();
        }

        return w;
    }

    public static void main(String[] args) {
        int k1 = 2;
        int w1 = 0;
        int[] profits1 = {1, 2, 3};
        int[] capital1 = {0, 1, 1};

        System.out.println("示例 1 输入: k = " + k1 + ", w = " + w1
                + ", profits = " + Arrays.toString(profits1)
                + ", capital = " + Arrays.toString(capital1));
        int result1 = findMaximizedCapital(k1, w1, profits1, capital1);
        System.out.println("示例 1 输出: " + result1);
        System.out.println("-----------------------------");

        int k2 = 3;
        int w2 = 0;
        int[] profits2 = {1, 2, 3};
        int[] capital2 = {0, 1, 2};

        System.out.println("示例 2 输入: k = " + k2 + ", w = " + w2
                + ", profits = " + Arrays.toString(profits2)
                + ", capital = " + Arrays.toString(capital2));
        int result2 = findMaximizedCapital(k2, w2, profits2, capital2);
        System.out.println("示例 2 输出: " + result2);
    }
}
```
