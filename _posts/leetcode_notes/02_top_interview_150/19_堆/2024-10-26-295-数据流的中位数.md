## 题目

**描述：**

**中位数**是有序整数列表中的中间值。如果列表的大小是偶数，则没有中间值，中位数是两个中间值的平均值。

- 例如 `arr = [2,3,4]` 的中位数是 `3`。
- 例如 `arr = [2,3]` 的中位数是 `(2 + 3) / 2 = 2.5`。

实现 `MedianFinder` 类：

- `MedianFinder()` 初始化 `MedianFinder` 对象。
- `void addNum(int num)` 将数据流中的整数 `num` 添加到数据结构中。
- `double findMedian()` 返回到目前为止所有元素的中位数。与实际答案相差 `10^-5` 以内的答案将被接受。

**示例 1:**

- **输入**

  `["MedianFinder", "addNum", "addNum", "findMedian", "addNum", "findMedian"]`

  `[[], [1], [2], [], [3], []]`

- **输出**

  `[null, null, null, 1.5, null, 2.0]`

- **解释**

  ```
  MedianFinder medianFinder = new MedianFinder();
  medianFinder.addNum(1);    // arr = [1]
  medianFinder.addNum(2);    // arr = [1, 2]
  medianFinder.findMedian(); // 返回 1.5 ((1 + 2) / 2)
  medianFinder.addNum(3);    // arr = [1, 2, 3]
  medianFinder.findMedian(); // 返回 2.0
  ```

**提示:**

- `-10^5 <= num <= 10^5`
- 在调用 `findMedian` 之前，数据结构中至少有一个元素
- 最多 `5 * 10^4` 次调用 `addNum` 和 `findMedian`

## 题解

### 这题难的不是求中位数，而是“动态维护”

如果数据是一次性给出来的，那中位数其实很好求：

- 先排序
- 再看中间位置
- 如果元素个数是偶数，就取中间两个数的平均值

但这题不是这样。它给的是**数据流**，也就是说，数字会一个一个不断加入，而且中途还会反复查询当前中位数。

这就意味着我们不能每来一个数都重新排序一遍。  
如果每次 `addNum` 后都把所有数据重新排好，代价会很高，显然不是这题想考的方向。

这题真正要解决的是：

**如何在不断插入新数字的同时，始终快速拿到中位数。**

### 先抓住中位数的本质

中位数这个东西，说到底就是把一组数分成左右两半。

假设排好序之后：

- 左边这一半是较小的数
- 右边这一半是较大的数

那么：

- 如果总数是奇数，中位数就是“中间那个数”
- 如果总数是偶数，中位数就是“左半部分最大值”和“右半部分最小值”的平均值

所以问题就转成了：

**能不能一边插入数据，一边维护这两半，并且随时知道左边最大的数、右边最小的数？**

这正是堆最擅长的事情。

### 用两个堆刚好合适

我们可以用两个优先队列来维护这两部分数据：

- 一个**大顶堆**，维护较小的一半元素
- 一个**小顶堆**，维护较大的一半元素

这样设计之后：

- 大顶堆的堆顶，就是左半部分的最大值
- 小顶堆的堆顶，就是右半部分的最小值

而这两个值，恰好就是求中位数最需要的信息。

### 两个堆要一直满足两个条件

这题真正的核心，不是“开两个堆”这么简单，而是始终维护下面两个条件。

#### 条件一：左边这堆的元素都要小于等于右边这堆

也就是说：

```text
左半部分 <= 右半部分
```

这样分出来的两堆，才真的对应“较小的一半”和“较大的一半”。

#### 条件二：两个堆的大小要尽量平衡

我们约定：

- 左边的大顶堆元素个数，要么等于右边的小顶堆
- 要么比右边多 1 个

为什么这样约定？  
因为这样处理中位数最方便：

- 如果总数是奇数，中位数就落在左边堆顶
- 如果总数是偶数，中位数就是两个堆顶的平均值

这个约定是整道题写顺的关键。

### 新数字加进来时，怎么维护这两个条件

很多人第一次做这题时，最容易卡在 `addNum`：  
新来的这个数，到底该放左边还是右边？

其实这里有一种很顺手的写法，不需要分很多情况判断。

做法分三步：

1. 先把新数字放进左边的大顶堆
2. 再把左边堆顶弹出来，放进右边的小顶堆
3. 如果右边元素个数超过左边，再把右边堆顶放回左边

这样处理以后，两个条件就都能自动维持住。

### 为什么这三步是对的

先看第一步。  
把新数字先丢进左边，其实只是为了让它先参与排序，不代表它最终一定属于左边。

第二步把左边最大的数移到右边，这一步很关键。  
因为左边维护的是较小的一半，所以一旦左边里出现了“偏大的数”，就应该把它送到右边去。这样能保证左边这堆整体不会比右边更大。

第三步做的是数量平衡。  
前两步之后，右边可能会比左边多一个元素。  
但我们前面约定过，应该让左边不少于右边，所以如果右边元素更多，就把右边最小的数再挪回左边。

整个过程看起来像是“来回倒”，但实际作用非常清楚：

- 第二步负责维护大小关系
- 第三步负责维护数量关系

### 用一个例子走一遍

假设数据流依次加入：

```text
1, 2, 3
```

我们来看看两个堆怎么变化。

#### 加入 1

先放进左边：

```text
左： [1]
右： []
```

然后把左边最大值移到右边：

```text
左： []
右： [1]
```

这时右边比左边多，于是把右边最小值移回左边：

```text
左： [1]
右： []
```

现在左边比右边多 1 个，符合要求。  
中位数就是左边堆顶 `1`。

#### 加入 2

先放进左边：

```text
左： [2,1]
右： []
```

再把左边最大值移到右边：

```text
左： [1]
右： [2]
```

此时两个堆大小相等，不需要再调。

所以当前中位数是：

```text
(1 + 2) / 2 = 1.5
```

#### 加入 3

先放进左边：

```text
左： [3,1]
右： [2]
```

再把左边最大值移到右边：

```text
左： [1]
右： [2,3]
```

这时右边比左边多，于是把右边最小值 `2` 放回左边：

```text
左： [2,1]
右： [3]
```

现在左边多一个元素，所以中位数就是左边堆顶 `2`。

这个过程和题目示例正好一致。

### `findMedian` 为什么能做到 `O(1)`

因为中位数只和两个堆顶有关。

- 如果左边元素更多，中位数就是左边堆顶
- 如果两边一样多，中位数就是两个堆顶的平均值

不需要遍历，也不需要额外调整，直接返回即可。

这也是双堆方案真正高效的地方：  
插入是对数级，查询直接常数级。

### 代码

```java
class MedianFinder {
    private PriorityQueue<Integer> smallHalf;
    private PriorityQueue<Integer> largeHalf;

    public MedianFinder() {
        smallHalf = new PriorityQueue<>((a, b) -> b - a);
        largeHalf = new PriorityQueue<>();
    }

    public void addNum(int num) {
        smallHalf.offer(num);
        largeHalf.offer(smallHalf.poll());

        if (smallHalf.size() < largeHalf.size()) {
            smallHalf.offer(largeHalf.poll());
        }
    }

    public double findMedian() {
        if (smallHalf.size() > largeHalf.size()) {
            return smallHalf.peek();
        }
        return (smallHalf.peek() + largeHalf.peek()) / 2.0;
    }
}
```

这段代码短，但背后要记住的是两个不变式：

- `smallHalf` 里的元素都不大于 `largeHalf`
- `smallHalf` 的元素个数要么等于 `largeHalf`，要么比它多 1

只要这两个条件一直成立，`findMedian` 就一定是对的。

### 复杂度分析

- `addNum` 的时间复杂度是 $O(\log n)$  
  因为插入和弹出堆顶都是对数级操作。

- `findMedian` 的时间复杂度是 $O(1)$  
  只需要看堆顶，不需要额外计算。

- 空间复杂度是 $O(n)$  
  因为所有加入过的数据最终都要存进两个堆里。

### 这题真正值得记住的地方

这题最值得记住的，不只是“双堆”这个结论，而是这种处理思路：

**当一个问题需要动态维护“中间位置”时，往往可以把数据拆成左右两半，再分别维护边界值。**

这里的边界值就是：

- 左半部分的最大值
- 右半部分的最小值

而堆，正好就是维护这种边界信息最合适的数据结构。

以后再碰到“动态第 k 大”“动态中位数”“不断插入后查询排名位置”这类题，也可以优先往这个方向想。

### 完整代码

```java
import java.util.PriorityQueue;

public class Main {

    static class MedianFinder {
        private PriorityQueue<Integer> smallHalf;
        private PriorityQueue<Integer> largeHalf;

        public MedianFinder() {
            smallHalf = new PriorityQueue<>((a, b) -> b - a);
            largeHalf = new PriorityQueue<>();
        }

        public void addNum(int num) {
            smallHalf.offer(num);
            largeHalf.offer(smallHalf.poll());

            if (smallHalf.size() < largeHalf.size()) {
                smallHalf.offer(largeHalf.poll());
            }
        }

        public double findMedian() {
            if (smallHalf.size() > largeHalf.size()) {
                return smallHalf.peek();
            }
            return (smallHalf.peek() + largeHalf.peek()) / 2.0;
        }
    }

    public static void main(String[] args) {
        MedianFinder medianFinder = new MedianFinder();

        medianFinder.addNum(1);
        System.out.println("addNum(1)");

        medianFinder.addNum(2);
        System.out.println("addNum(2)");

        System.out.println("findMedian() -> " + medianFinder.findMedian());

        medianFinder.addNum(3);
        System.out.println("addNum(3)");

        System.out.println("findMedian() -> " + medianFinder.findMedian());

        System.out.println("-----------------------------");

        medianFinder.addNum(4);
        System.out.println("addNum(4), findMedian() -> " + medianFinder.findMedian());

        medianFinder.addNum(5);
        System.out.println("addNum(5), findMedian() -> " + medianFinder.findMedian());
    }
}
```
