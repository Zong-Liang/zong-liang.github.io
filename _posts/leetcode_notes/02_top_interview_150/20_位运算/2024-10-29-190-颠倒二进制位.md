## 题目

**描述：**

颠倒给定的 32 位有符号整数的二进制位。

**示例 1:**

- **输入:** `n = 43261596`
- **输出:** `964176192`
- **解释:**

  | 整数      | 二进制                             |
  | :-------- | :--------------------------------- |
  | 43261596  | `00000010100101000001111010011100` |
  | 964176192 | `00111001011110000010100101000000` |

**示例 2:**

- **输入:** `n = 2147483644`
- **输出:** `1073741822`
- **解释:**

  | 整数       | 二进制                             |
  | :--------- | :--------------------------------- |
  | 2147483644 | `01111111111111111111111111111100` |
  | 1073741822 | `00111111111111111111111111111110` |

**提示:**

- `0 <= n <= 2^31 - 2`
- `n` 为偶数
- (注意：在某些语言（如 Java）中，没有无符号整数类型。在这种情况下，输入和输出都将被指定为有符号整数类型，并且不应影响您的实现，因为无论整数是有符号的还是无符号的，其内部的二进制表示形式都是相同的。)

**进阶:**

- 如果多次调用这个函数，你将如何优化你的算法？

## 题解

### 这题本质上不是“算数”，而是“搬位”

这道题如果只看题面，其实很容易把注意力放在“整数”上。  
但它真正要我们处理的，并不是数值大小，而是这个数的 **32 位二进制表示**。

所谓“颠倒二进制位”，说白了就是：

- 原来的最低位，搬到最高位
- 原来的次低位，搬到次高位
- ...
- 原来的最高位，搬到最低位

所以这题的核心不是做加减乘除，而是老老实实按位搬运。

一旦想清楚这一点，最直接的做法就出来了：  
**从低位开始，一位一位取出来，再按相反顺序拼回去。**

### 最直接的做法：循环 32 次，逐位翻转

因为题目固定就是 32 位整数，所以我们完全可以写一个固定循环 32 次的过程。

每一轮做三件事：

1. 先把结果 `res` 左移一位，给新来的这一位腾位置
2. 取出 `n` 当前的最低位，也就是 `n & 1`
3. 把这一位加到 `res` 的最低位上

然后再把 `n` 右移一位，去处理下一位。

整个过程和“把一个字符串倒着拼出来”有点像，只不过这里倒的不是字符，而是二进制位。

### 为什么这样做一定对

假设 `n` 的二进制从左到右是：

```text
b31 b30 b29 ... b2 b1 b0
```

我们第一轮取到的是最低位 `b0`。  
因为 `res` 一开始是 0，第一轮把 `b0` 放进去后，它会成为结果的最低位。

第二轮开始前，`res` 先左移一位，于是第一轮放进去的 `b0` 就被顶到更高一位上。  
然后我们再把 `b1` 放到当前最低位。

这样不断重复下去，最开始取出来的低位会被一层层往左推，最后正好跑到高位；而原来高位的那些比特因为最后才被取出，就会落到结果的低位。

整个顺序自然就反过来了。

所以这题表面上是在“翻转”，本质上其实是：

**不断取原数最低位，并把它们依次接到新数的右边，再通过左移把顺序倒过来。**

### Java 里这里为什么要用 `>>>`

这题虽然示例里给的数都是非负的，但题目本质是在操作 **32 位位模式**。  
在 Java 里，`int` 是有符号的，如果中间结果最高位变成了 `1`，这个数就可能表现成负数。

这时如果你继续用普通右移：

```java
n >> 1
```

Java 会做符号扩展，也就是最高位补符号位。  
而我们这里想要的并不是“保留正负号”，而是单纯把二进制整体往右挪一位。

所以更稳妥、也更符合题意的写法应该是：

```java
n >>> 1
```

也就是无符号右移。

这一步很容易被忽略，但从“按位处理”的角度看，`>>>` 才是更准确的写法。

### 用一个例子感受一下过程

假设输入是：

```text
n = 13
```

它的二进制是：

```text
00000000000000000000000000001101
```

我们从右往左依次取出最低位：

- 第 1 次取出 `1`
- 第 2 次取出 `0`
- 第 3 次取出 `1`
- 第 4 次取出 `1`
- 后面都是 `0`

每次都把 `res` 左移一位，再把当前最低位补进去。  
前几轮下来，你会发现结果的前几位刚好开始变成原来末尾那几位的倒序。

原本末尾是：

```text
1101
```

翻过去以后，就会出现在结果的高位部分。

虽然代码里我们没有显式地“交换两端”，但通过“取最低位 + 左移拼接”的方式，顺序自然就完成了翻转。

### 代码

```java
public int reverseBits(int n) {
    int res = 0;

    for (int i = 0; i < 32; i++) {
        res = (res << 1) | (n & 1);
        n >>>= 1;
    }

    return res;
}
```

### 复杂度分析

时间复杂度是 $O(1)$。  
虽然写了循环，但循环次数固定就是 32 次，和输入规模无关，所以本质上是常数时间。

空间复杂度是 $O(1)$。  
只用了几个额外变量。

### 如果函数会被反复调用，怎么优化

题目后面还有一个进阶问题：如果这个函数会被调用很多次，能不能再优化。

常规做法每次都要循环 32 次，已经不慢了，但如果调用次数非常多，还是有优化空间。

一个很常见的思路是：  
**把 32 位拆成 4 个字节，每个字节 8 位。**

因为一个字节的取值范围只有 `0 ~ 255`，总共就 256 种情况。  
我们可以提前把这 256 个数“翻转 8 位之后的结果”全部算好，存到缓存表里。

这样每次处理一个 32 位整数时，只需要：

1. 拆出 4 个字节
2. 分别查表得到每个字节翻转后的结果
3. 再把这 4 个字节按相反顺序拼回去

这个思路本质上就是典型的“空间换时间”。

一次调用时，普通做法和查表做法差距不大；但如果要高频调用，查表会更划算。

### 为什么拆成字节后还要换位置

这一点也很容易想岔。

假设原来的 32 位分成 4 段：

```text
A B C D
```

其中每一段是 8 位。

整体翻转之后，顺序应该变成：

```text
reverse(D) reverse(C) reverse(B) reverse(A)
```

也就是说，不只是每个字节内部要翻转，字节之间的顺序也要颠倒。

所以查表之后的拼接位置必须注意：

- 原最低字节，翻转后要放到最高字节位置
- 原最高字节，翻转后要放到最低字节位置

这一步如果只翻字节内容、不换字节位置，最后结果还是错的。

### 这题真正值得记住的地方

这题最值得记住的，不是“反转二进制位”本身，而是这种位运算处理方式：

**当题目要求按位重排时，最直接的做法通常就是“取最低位 + 移位拼接”。**

本题是反转位顺序，所以我们不断取最低位，再左移结果去接它。  
以后如果碰到按位提取、按位构造、掩码拼接这类题，其实也都是这一套基本动作：

- 与运算取位
- 左移/右移挪位
- 或运算拼位

这类题看着抽象，真正写起来其实都离不开这些最基础的操作。

### 完整代码

```java
public class Main {

    public static int reverseBits(int n) {
        int res = 0;
        for (int i = 0; i < 32; i++) {
            res = (res << 1) | (n & 1);
            n >>>= 1;
        }
        return res;
    }

    private static final int[] cache = new int[256];

    static {
        for (int i = 0; i < 256; i++) {
            cache[i] = reverseByteHelper(i);
        }
    }

    private static int reverseByteHelper(int b) {
        int res = 0;
        for (int i = 0; i < 8; i++) {
            res = (res << 1) | (b & 1);
            b >>= 1;
        }
        return res;
    }

    public static int reverseBitsOptimized(int n) {
        return (cache[n & 0xff] << 24) |
               (cache[(n >>> 8) & 0xff] << 16) |
               (cache[(n >>> 16) & 0xff] << 8) |
               cache[(n >>> 24) & 0xff];
    }

    public static void main(String[] args) {
        int n1 = 43261596;
        System.out.println("示例 1 输入: " + n1);
        System.out.println("输入二进制: " + String.format("%32s", Integer.toBinaryString(n1)).replace(' ', '0'));
        int res1 = reverseBits(n1);
        System.out.println("示例 1 输出: " + res1);
        System.out.println("输出二进制: " + String.format("%32s", Integer.toBinaryString(res1)).replace(' ', '0'));
        System.out.println("-----------------------------");

        int n2 = 2147483644;
        System.out.println("示例 2 输入: " + n2);
        System.out.println("输入二进制: " + String.format("%32s", Integer.toBinaryString(n2)).replace(' ', '0'));
        int res2 = reverseBitsOptimized(n2);
        System.out.println("示例 2 输出: " + res2);
        System.out.println("输出二进制: " + String.format("%32s", Integer.toBinaryString(res2)).replace(' ', '0'));

        assert reverseBits(n2) == reverseBitsOptimized(n2);
    }
}
```
