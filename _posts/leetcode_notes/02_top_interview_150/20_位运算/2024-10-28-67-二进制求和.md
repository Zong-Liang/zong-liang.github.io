## 题目

**描述：**

给你两个二进制字符串 `a` 和 `b`，以二进制字符串的形式返回它们的和。

**示例 1:**

- **输入:** `a = "11"`, `b = "1"`
- **输出:** `"100"`

**示例 2:**

- **输入:** `a = "1010"`, `b = "1011"`
- **输出:** `"10101"`

**提示:**

- `1 <= a.length, b.length <= 10^4`
- `a` 和 `b` 仅由字符 `'0'` 或 `'1'` 组成
- 字符串如果不是 `"0"`，就不含前导零

## 题解

### 这题本质上就是二进制版竖式加法

这道题本身没有太多弯子，核心思路其实很直接：  
**把它当成二进制下的竖式加法来做。**

平时做十进制加法时，我们就是从最低位开始一位一位往前算：

- 当前位相加
- 如果超过 `9` 就进位
- 再继续算下一位

这题完全一样，只不过这里不是“逢十进一”，而是“逢二进一”。

所以只要把这个过程模拟出来，题目就解决了。

### 为什么不能直接转成数字来算

很多人第一反应会是：  
把二进制字符串先转成整数，算完再转回二进制字符串不就行了？

这个思路在数据范围小的时候没问题，但这题不行。

因为题目给的字符串长度可以达到 `10^4`，远远超过了 `int` 或 `long` 能表示的范围。  
也就是说，这题从一开始就已经在提醒你：

**不要想着真的把它变成数值类型去算，而是应该老老实实按字符串逐位处理。**

这也是为什么最自然的做法就是模拟手算过程。

### 为什么一定要从后往前算

字符串最左边是高位，最右边是低位。  
而加法的计算顺序一定是从低位往高位走，因为当前位算完之后，进位会影响前一位。

所以这题最自然的遍历方式就是：

- 一个指针 `i` 指向 `a` 的末尾
- 一个指针 `j` 指向 `b` 的末尾
- 再用一个变量 `carry` 表示当前进位

每次取出当前这两位，再加上 `carry`，就能得到当前位的结果和新的进位。

这一步其实和我们手算加法完全一样，只不过换成了代码。

### 二进制加法这一位到底怎么算

假设当前这一位取到的两个数字分别是 `x` 和 `y`，再加上进位 `carry`，那么：

```text
sum = x + y + carry
```

因为这里每一位只可能是 `0` 或 `1`，而进位也只可能是 `0` 或 `1`，所以 `sum` 的取值只会是：

- `0`
- `1`
- `2`
- `3`

然后：

- 当前位结果就是 `sum % 2`
- 新的进位就是 `sum / 2`

比如：

- `1 + 0 + 0 = 1`，当前位写 `1`，进位是 `0`
- `1 + 1 + 0 = 2`，当前位写 `0`，进位是 `1`
- `1 + 1 + 1 = 3`，当前位写 `1`，进位是 `1`

所以整道题其实没有任何额外技巧，真的就是把这个规则不断重复下去。

### 为什么最后还要反转

因为我们是从字符串末尾开始算的，也就是从低位算到高位。

但 `StringBuilder` 的追加操作是往后加，所以我们得到的结果顺序其实是反的。

比如正常答案应该是：

```text
10101
```

但如果你边算边 `append`，实际得到的会是：

```text
10101` 的倒序
```

所以最后要做一次 `reverse()`，把顺序翻回来。

这一步不是技巧，只是因为我们的计算顺序和字符串正常书写顺序相反。

### 用例子走一遍

来看示例 2：

```text
a = "1010"
b = "1011"
```

初始时：

- `i` 指向 `a` 最后一位
- `j` 指向 `b` 最后一位
- `carry = 0`

#### 第 1 轮

当前位是：

- `a[i] = 0`
- `b[j] = 1`

所以：

```text
sum = 0 + 1 + 0 = 1
```

当前位结果是 `1`，进位还是 `0`。

#### 第 2 轮

当前位是：

- `1`
- `1`

所以：

```text
sum = 1 + 1 + 0 = 2
```

当前位结果是 `0`，进位变成 `1`。

#### 第 3 轮

当前位是：

- `0`
- `0`

再加上进位 `1`：

```text
sum = 0 + 0 + 1 = 1
```

当前位结果是 `1`，进位变回 `0`。

#### 第 4 轮

当前位是：

- `1`
- `1`

所以：

```text
sum = 1 + 1 + 0 = 2
```

当前位结果是 `0`，进位变成 `1`。

这时两个字符串都已经遍历完了，但还有进位 `1`，所以还要继续补一位。

最后得到的逆序结果是：

```text
10101
```

反转之后仍然是：

```text
10101
```

这就是答案。

### 代码

```java
public String addBinary(String a, String b) {
    StringBuilder sb = new StringBuilder();
    int i = a.length() - 1;
    int j = b.length() - 1;
    int carry = 0;

    while (i >= 0 || j >= 0 || carry != 0) {
        int sum = carry;

        if (i >= 0) {
            sum += a.charAt(i) - '0';
            i--;
        }

        if (j >= 0) {
            sum += b.charAt(j) - '0';
            j--;
        }

        sb.append(sum % 2);
        carry = sum / 2;
    }

    return sb.reverse().toString();
}
```

### 复杂度分析

时间复杂度是 $O(n)$，其中 `n` 是两个字符串中的较长长度。  
因为我们只需要从后往前遍历一遍。

空间复杂度是 $O(n)$。  
结果字符串最长可能比原来多一位，所以需要额外的 `StringBuilder` 来保存答案。

### 这题真正值得记住的地方

这题最值得记住的，不是“二进制求和”这四个字本身，而是这种处理方式：

**当数字长度大到不能直接用数值类型表示时，就不要想着转数字再运算，而应该直接在字符串层面模拟运算过程。**

本题是加法，所以模拟的是竖式加法。  
以后如果碰到大整数乘法、字符串相加这类题，其实也是同样的思路：按位处理，自己维护进位。

所以这题表面上是二进制，真正该记住的其实是“字符串模拟运算”的做法。

### 完整代码

```java
public class Main {

    public static String addBinary(String a, String b) {
        StringBuilder sb = new StringBuilder();
        int i = a.length() - 1;
        int j = b.length() - 1;
        int carry = 0;

        while (i >= 0 || j >= 0 || carry != 0) {
            int sum = carry;

            if (i >= 0) {
                sum += a.charAt(i) - '0';
                i--;
            }

            if (j >= 0) {
                sum += b.charAt(j) - '0';
                j--;
            }

            sb.append(sum % 2);
            carry = sum / 2;
        }

        return sb.reverse().toString();
    }

    public static void main(String[] args) {
        String a1 = "11";
        String b1 = "1";
        System.out.println("示例 1 输入: a = \"" + a1 + "\", b = \"" + b1 + "\"");
        System.out.println("示例 1 输出: \"" + addBinary(a1, b1) + "\"");
        System.out.println("-----------------------------");

        String a2 = "1010";
        String b2 = "1011";
        System.out.println("示例 2 输入: a = \"" + a2 + "\", b = \"" + b2 + "\"");
        System.out.println("示例 2 输出: \"" + addBinary(a2, b2) + "\"");
        System.out.println("-----------------------------");

        String a3 = "111";
        String b3 = "1";
        System.out.println("边界测试 输入: a = \"" + a3 + "\", b = \"" + b3 + "\"");
        System.out.println("边界测试 输出: \"" + addBinary(a3, b3) + "\"");
    }
}
```
