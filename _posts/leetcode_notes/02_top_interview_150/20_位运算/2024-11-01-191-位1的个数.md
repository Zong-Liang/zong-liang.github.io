## 题目

**描述：**

给定一个正整数 `n`，编写一个函数，获取一个正整数的二进制形式并返回其二进制表达式中 **设置位** 的个数（也被称为**汉明重量**）。

**示例 1:**

- **输入:** `n = 11`
- **输出:** 3
- **解释:** 输入的二进制串 `1011` 中，共有 3 个设置位。

**示例 2:**

- **输入:** `n = 128`
- **输出:** 1
- **解释:** 输入的二进制串 `10000000` 中，共有 1 个设置位。

**示例 3:**

- **输入:** `n = 2147483645` (即 `2^31 - 3`)
- **输出:** 30
- **解释:** 输入的二进制串 `11111111111111111111111111111101` 中，共有 30 个设置位。

**提示:**

- `1 <= n <= 2^31 - 1`

**进阶:**

- 如果多次调用这个函数，你将如何优化你的算法？

## 题解

### 这题表面上是在数 1，实际上是在想办法更快地消掉 1

这道题如果按最直觉的方式去做，其实并不难。

最直接的想法就是：  
把这个数的二进制从低位到高位一位一位看过去，遇到 `1` 就计数加一。

这个思路当然能做，但它的本质是“逐位检查”。  
而这题真正更值得学的地方在于：**能不能不一位一位看，而是每次直接去掉一个 1。**

这就是经典的 Brian Kernighan 算法。

### 核心结论：`n & (n - 1)` 会把最低位的 1 抹掉

这题最关键的性质就是：

```text
n & (n - 1)
```

会把 `n` 的二进制表示中**最低位的那个 1 变成 0**，其余更高位保持不变。

这个结论不是硬记的，拆开看其实很好理解。

假设某个数的二进制长这样：

```text
xxxxx1000...000
```

也就是从右往左数，最低位的那个 `1` 后面全是 `0`。

那么 `n - 1` 会变成：

```text
xxxxx0111...111
```

也就是说：

- 原来最低位那个 `1` 会变成 `0`
- 它后面的所有 `0` 都会变成 `1`

这时候再和原数做按位与：

```text
n       = xxxxx1000...000
n - 1   = xxxxx0111...111
n&(n-1) = xxxxx0000...000
```

你会发现，最低位那个 `1` 正好被清掉了，而更高位的部分不受影响。

这就是这个技巧的本质。

### 为什么这个性质刚好适合这道题

既然 `n & (n - 1)` 每做一次，就能消掉一个 `1`，那这题就很好办了。

我们只需要：

1. 准备一个计数器 `count`
2. 只要 `n` 还不等于 `0`，就执行一次 `n &= (n - 1)`
3. 每执行一次，说明刚刚消掉了一个 `1`，于是 `count++`

当 `n` 终于变成 `0` 时，说明所有的 `1` 都已经被消干净了。  
而执行了多少次，就说明原来有多少个 `1`。

所以这题其实不是“一个一个统计”，而是“一个一个删除”。  
删掉几个 `1` 才能变成 `0`，答案就是几。

### 用例子走一遍最容易明白

以示例 1 为例：

```text
n = 11
```

它的二进制是：

```text
1011
```

现在开始不断做 `n & (n - 1)`。

#### 第 1 次

```text
1011
1010
----
1010
```

结果变成 `1010`，也就是 `10`。  
最低位的那个 `1` 被抹掉了。

#### 第 2 次

```text
1010
1001
----
1000
```

结果变成 `1000`，也就是 `8`。  
又抹掉了一个 `1`。

#### 第 3 次

```text
1000
0111
----
0000
```

结果变成 `0`。  
第三个 `1` 也被抹掉了。

总共执行了 3 次，所以答案就是 `3`。

这比“从右往左一位一位数”更巧，也更能体现位运算的味道。

### 代码

```java
public int hammingWeight(int n) {
    int count = 0;
    while (n != 0) {
        n &= (n - 1);
        count++;
    }
    return count;
}
```

### 为什么这个写法比逐位检查更好

如果用最普通的方法去做，通常会循环固定的位数，比如 32 次，每次检查最低位是不是 1。

但这里这种写法不一样，它的循环次数并不取决于总位数，而只取决于 **1 的个数**。

也就是说：

- 如果一个数里只有 1 个 `1`，那只循环 1 次
- 如果有 3 个 `1`，就循环 3 次
- 只有在最坏情况下，才会接近逐位检查的次数

所以这个方法的重点不只是“能做出来”，而是它真正利用了题目的位结构。

### 如果这个函数会被频繁调用，怎么优化

题目后面还有一个进阶问题：如果这个函数会被调用很多次，能不能再快一点。

这种情况下，一个常见思路就是查表。

因为一个 32 位整数可以拆成 4 个字节，而一个字节只有 8 位，范围就是 `0 ~ 255`。  
这意味着，8 位二进制中“1 的个数”总共只有 256 种可能。

那我们完全可以提前把这 256 个数各自对应的答案算好，存到数组里。  
以后每次传来一个 32 位整数时，只要：

1. 把它拆成 4 个字节
2. 分别查表
3. 把 4 次结果加起来

就行了。

这个思路本质上是用空间换时间。  
一次调用时差别不大，但如果要大量重复调用，查表法会更划算。

### 查表法为什么成立

比如一个 32 位整数可以拆成：

```text
[高8位][次高8位][次低8位][低8位]
```

每一段里有多少个 `1`，都可以独立统计。  
而整个整数里 `1` 的总数，就是这 4 段的和。

所以查表时只需要这样做：

- `n & 0xff` 取最低 8 位
- `(n >>> 8) & 0xff` 取次低 8 位
- `(n >>> 16) & 0xff` 取次高 8 位
- `(n >>> 24) & 0xff` 取最高 8 位

然后分别查表相加即可。

这里用 `>>>` 是为了做无符号右移，避免高位补符号位带来干扰。

### 复杂度分析

Brian Kernighan 算法的时间复杂度是 $O(k)$，其中 `k` 是二进制中 `1` 的个数。  
空间复杂度是 $O(1)$。

查表法的时间复杂度是 $O(1)$，因为固定只做 4 次查表和加法。  
空间复杂度也是 $O(1)$，虽然用了一个长度为 256 的数组，但它的大小是固定的。

### 这题真正值得记住的地方

这题最值得记住的，不是“统计位 1 个数”本身，而是这个位运算技巧：

**`n & (n - 1)` 可以直接消掉最低位的那个 1。**

这个性质非常常用。

以后遇到下面这些问题时，它都很有用：

- 判断一个数是不是 2 的幂
- 统计二进制里 1 的个数
- 枚举状态压缩中的子集
- 快速删除某一位特征

所以这题虽然简单，但它背后的技巧非常值得记住。  
会了这个，后面很多位运算题都会顺手很多。

### 完整代码

```java
public class Main {

    public static int hammingWeight(int n) {
        int count = 0;
        while (n != 0) {
            n &= (n - 1);
            count++;
        }
        return count;
    }

    private static final int[] CACHE = new int[256];

    static {
        for (int i = 0; i < 256; i++) {
            int count = 0;
            int temp = i;
            while (temp != 0) {
                temp &= (temp - 1);
                count++;
            }
            CACHE[i] = count;
        }
    }

    public static int hammingWeightOptimized(int n) {
        return CACHE[n & 0xff] +
               CACHE[(n >>> 8) & 0xff] +
               CACHE[(n >>> 16) & 0xff] +
               CACHE[(n >>> 24) & 0xff];
    }

    public static void main(String[] args) {
        int n1 = 11;
        System.out.println("示例 1 输入: " + n1 + " (二进制: " + Integer.toBinaryString(n1) + ")");
        System.out.println("示例 1 输出: " + hammingWeight(n1));
        System.out.println("-----------------------------");

        int n2 = 128;
        System.out.println("示例 2 输入: " + n2 + " (二进制: " + Integer.toBinaryString(n2) + ")");
        System.out.println("示例 2 输出: " + hammingWeight(n2));
        System.out.println("-----------------------------");

        int n3 = 2147483645;
        System.out.println("示例 3 输入: " + n3 + " (二进制: " + Integer.toBinaryString(n3) + ")");
        System.out.println("示例 3 输出 (查表法): " + hammingWeightOptimized(n3));

        assert hammingWeight(n3) == hammingWeightOptimized(n3);
    }
}
```
