## 题目

**描述：**

给你两个整数 `left` 和 `right`，表示区间 `[left, right]`，返回此区间内所有数字 **按位与** 的结果（包含 `left`、`right` 端点）。

**示例 1:**

- **输入:** `left = 5`, `right = 7`
- **输出:** 4
- **解释:**
  - 5 (二进制: 101)
  - 6 (二进制: 110)
  - 7 (二进制: 111)
  - 101 & 110 & 111 = 100 (十进制: 4)

**示例 2:**

- **输入:** `left = 0`, `right = 0`
- **输出:** 0

**示例 3:**

- **输入:** `left = 1`, `right = 2147483647`
- **输出:** 0

**提示:**

- `0 <= left <= right <= 2^31 - 1`

## 题解

### 这题关键不在“把区间里的数都与一遍”，而在“看哪些位根本保不住”

如果直接按照题意去做，从 `left` 一路枚举到 `right`，然后把所有数按位与起来，当然也能得到答案。

但这种做法太慢了。  
题目给出的区间可能非常大，显然不是想让我们真的一个一个去算。

这题真正该抓住的是按位与的性质：

**只要某一位上出现过一个 0，那么最终结果在这一位上就一定是 0。**

而区间 `[left, right]` 里是一串连续整数。  
连续整数最大的特点就是：越靠右的低位，变化越频繁。只要区间跨度覆盖到了这些变化，这些位就不可能全部保持为 1，最终一定会被按位与清零。

所以这题真正要找的，不是“区间里所有数都算完之后剩什么”，而是：

**从二进制角度看，`left` 和 `right` 还有哪些高位前缀是始终不变的。**

### 结果其实就是 `left` 和 `right` 的公共前缀

先看示例 1：

```text
left = 5  -> 101
right = 7 -> 111
```

区间里包含：

```text
101
110
111
```

把它们按位与之后得到：

```text
100
```

为什么会这样？

因为从最高位看，这三个数的最高位一直都是 `1`，所以这一位能保留下来。  
但后面两位在区间变化过程中已经出现了不同情况，有的数这一位是 `0`，有的数这一位是 `1`，那按位与之后就一定变成 `0`。

所以最后留下来的，其实就是：

- 前面公共不变的那部分
- 后面全部补 0

这就是“公共前缀”这个结论的来源。

### 为什么低位一定会被清零

这题最容易想明白的地方就在这里。

假设 `left` 和 `right` 在某一位开始不一样了。  
那说明从这一位往后的低位，在区间递增的过程中一定发生过翻转。

而一旦翻转，就意味着这一区间里一定同时出现过：

- 某些数这一位是 `0`
- 某些数这一位是 `1`

那按位与的结果在这一位就不可能还是 `1`，只能是 `0`。

所以：

- 只有高位里那些从头到尾完全没变过的部分，才可能留下来
- 从第一个不同位开始，后面所有位最终都得归零

这就是为什么答案其实就是 `left` 和 `right` 的公共前缀。

### 最自然的做法：一起右移，直到它们相等

既然我们要找公共前缀，那最直接的方法就很自然了：

- 只要 `left` 和 `right` 还不相等，就同时右移一位
- 每右移一次，说明我们去掉了一位不稳定的低位
- 当它们终于相等时，剩下的部分就是公共前缀
- 最后再把它左移回去，把被删掉的低位补成 0

比如还是 `5` 和 `7`：

```text
5  = 101
7  = 111
```

第一次右移：

```text
10
11
```

还不相等，继续右移。

第二次右移：

```text
1
1
```

这时相等了，说明公共前缀就是 `1`。  
我们一共右移了 2 次，所以最后把它再左移 2 位：

```text
1 << 2 = 100
```

答案就是 `4`。

这个过程其实就是在不断丢掉“肯定保不住的低位”，直到只剩下稳定的公共前缀。

### 为什么 `while (left < right)` 就够了

这里有个很自然的小问题：为什么只要 `left < right` 就继续右移？

因为只要它们还不相等，就说明当前仍然存在不稳定的位。  
换句话说，公共前缀还没完全露出来。

而当它们终于相等时，说明所有会变化的低位都已经被移掉了，剩下来的部分就是两者共同拥有的那段高位前缀。

所以循环停止时：

- `left` 和 `right` 相等
- 这个值就是公共前缀
- 再左移回原位置，就是最终结果

### 用另一个例子再看一遍

比如：

```text
left = 9  -> 1001
right = 12 -> 1100
```

一起右移：

#### 第 1 次

```text
100
110
```

还不相等。

#### 第 2 次

```text
10
11
```

还不相等。

#### 第 3 次

```text
1
1
```

相等了。

说明公共前缀是 `1`，一共右移了 3 次，所以最后结果是：

```text
1 << 3 = 1000
```

也就是 `8`。

这和真正把区间 `9,10,11,12` 全部按位与出来的结果是一致的。

### 代码

```java
public int rangeBitwiseAnd(int left, int right) {
    int shift = 0;

    while (left < right) {
        left >>= 1;
        right >>= 1;
        shift++;
    }

    return left << shift;
}
```

### 另一种写法：不断消掉 `right` 最右边的 1

这题还有一种很常见的写法，基于 Brian Kernighan 算法。

我们知道：

```text
n & (n - 1)
```

会把 `n` 最右边的那个 `1` 清掉。

在这题里，只要 `right` 还大于 `left`，就说明 `right` 里面还有一些不属于公共前缀的低位信息。  
那我们就不断把 `right` 最右边的 `1` 消掉，直到 `right <= left` 为止。

这时留下来的部分，本质上也就是公共前缀补零后的结果。

这个写法更像技巧题写法，但本质和上面的位移法是一样的：  
**都是在抹掉不稳定的低位。**

```java
public int rangeBitwiseAnd(int left, int right) {
    while (left < right) {
        right = right & (right - 1);
    }
    return right;
}
```

### 复杂度分析

位移法的时间复杂度是 $O(\log n)$。  
因为最多只会右移整数位数那么多次，本题里也就是不超过 31 次。

空间复杂度是 $O(1)$。  
只用了几个额外变量。

Brian Kernighan 写法的时间复杂度同样可以看作 $O(\log n)$，空间复杂度也是 $O(1)$。

### 这题真正值得记住的地方

这题最值得记住的，不是“区间按位与”这个题型本身，而是这个判断方式：

**对于一段连续整数，只要某一位在区间内发生过变化，这一位在按位与结果里就一定保不住。**

所以最后能留下来的，只可能是高位中那段始终不变的公共前缀。

这其实是很多位运算题都会用到的观察角度：

- 不要真的去枚举整个区间
- 先想连续变化会对二进制哪些位造成影响
- 再看哪些位还有可能保留下来

一旦从这个角度去看，这题就会很顺。

### 完整代码

```java
public class Main {

    public static int rangeBitwiseAndShift(int left, int right) {
        int shift = 0;

        while (left < right) {
            left >>= 1;
            right >>= 1;
            shift++;
        }

        return left << shift;
    }

    public static int rangeBitwiseAndBK(int left, int right) {
        while (left < right) {
            right = right & (right - 1);
        }
        return right;
    }

    public static void main(String[] args) {
        int left1 = 5, right1 = 7;
        System.out.println("示例 1 输入: left = " + left1 + ", right = " + right1);
        System.out.println("示例 1 输出（位移法）: " + rangeBitwiseAndShift(left1, right1));
        System.out.println("示例 1 输出（BK 写法）: " + rangeBitwiseAndBK(left1, right1));
        System.out.println("-----------------------------");

        int left2 = 0, right2 = 0;
        System.out.println("示例 2 输入: left = " + left2 + ", right = " + right2);
        System.out.println("示例 2 输出: " + rangeBitwiseAndShift(left2, right2));
        System.out.println("-----------------------------");

        int left3 = 1, right3 = 2147483647;
        System.out.println("示例 3 输入: left = " + left3 + ", right = " + right3);
        System.out.println("示例 3 输出: " + rangeBitwiseAndShift(left3, right3));
        System.out.println("-----------------------------");

        int left4 = 9, right4 = 12;
        System.out.println("额外示例 输入: left = " + left4 + ", right = " + right4);
        System.out.println("额外示例 输出: " + rangeBitwiseAndShift(left4, right4));
    }
}
```
