## 题目

**描述：**

给你一个 **非空** 整数数组 `nums`，除了某个元素只出现一次以外，其余每个元素均出现两次。找出那个只出现了一次的元素。

你必须设计并实现线性时间复杂度的算法来解决此问题，且该算法只使用常数量额外空间。

**示例 1:**

- **输入:** `nums = [2,2,1]`
- **输出:** 1

**示例 2:**

- **输入:** `nums = [4,1,2,1,2]`
- **输出:** 4

**示例 3:**

- **输入:** `nums = [1]`
- **输出:** 1

**提示:**

- `1 <= nums.length <= 3 * 10^4`
- `-3 * 10^4 <= nums[i] <= 3 * 10^4`
- 除了某个元素只出现一次以外，其余每个元素均出现两次。

## 题解

### 这题关键不在“找”，而在“消掉成对元素”

这道题如果不看复杂度要求，其实很好想。

比如可以用哈希表统计每个数字出现了几次，最后找出出现一次的那个；  
或者先排序，再去找那个没有成对出现的元素。

但题目把条件卡得很死：

- 时间复杂度要是 $O(n)$
- 额外空间只能是 $O(1)$

这两个条件放在一起，其实就在逼着我们不要去“记录”，而是去想：

**能不能在遍历过程中，把那些成对出现的数直接消掉。**

而能做到这件事的，正好就是异或运算。

### 为什么异或这么适合这道题

异或 `^` 有几个非常关键的性质：

1. 一个数和自己异或，结果是 `0`

```text
a ^ a = 0
```

2. 一个数和 `0` 异或，结果还是它自己

```text
a ^ 0 = a
```

3. 异或满足交换律和结合律

也就是说：

```text
a ^ b ^ a = a ^ a ^ b = 0 ^ b = b
```

这几个性质放到这道题里，效果就非常直接了。

因为数组里除了一个数只出现一次之外，其余所有数都刚好出现两次。  
那如果我们把整个数组从头到尾全部异或一遍，成对出现的数字就会两两抵消，最后剩下的自然就是那个只出现一次的数。

所以这题并不是“把唯一的数找出来”，而是：

**把所有能抵消的都抵消掉，最后剩下的那个就是答案。**

### 为什么顺序完全无所谓

很多人第一次看到这里，会下意识担心一个问题：

“数组里的重复数字又不是挨在一起的，真的能抵消吗？”

可以，因为异或满足交换律和结合律。

也就是说，不管顺序怎么变，这一串异或最终的结果都不会变。  
所以像这样：

```text
4 ^ 1 ^ 2 ^ 1 ^ 2
```

我们完全可以在逻辑上把它改写成：

```text
4 ^ (1 ^ 1) ^ (2 ^ 2)
```

然后就变成：

```text
4 ^ 0 ^ 0 = 4
```

所以这题的关键不是位置，而是“出现次数”。

只要某个数出现两次，它就一定会被消掉；只有那个落单的数，最后才会留下来。

### 用例子走一遍最直观

看示例 2：

```text
nums = [4,1,2,1,2]
```

我们从左到右做异或。

初始时：

```text
result = 0
```

先异或 `4`：

```text
result = 0 ^ 4 = 4
```

再异或 `1`：

```text
result = 4 ^ 1
```

再异或 `2`：

```text
result = 4 ^ 1 ^ 2
```

再异或第二个 `1`：

```text
result = 4 ^ 1 ^ 2 ^ 1
```

最后异或第二个 `2`：

```text
result = 4 ^ 1 ^ 2 ^ 1 ^ 2
```

因为异或顺序无所谓，所以它等价于：

```text
4 ^ (1 ^ 1) ^ (2 ^ 2)
= 4 ^ 0 ^ 0
= 4
```

最后答案就是 `4`。

你会发现，代码里我们并没有真的“配对”，但异或本身已经帮我们把这件事做掉了。

### 代码

```java
public int singleNumber(int[] nums) {
    int result = 0;
    for (int num : nums) {
        result ^= num;
    }
    return result;
}
```

### 复杂度分析

时间复杂度是 $O(n)$。  
因为只需要遍历数组一次。

空间复杂度是 $O(1)$。  
只用了一个变量保存异或结果。

### 这题真正值得记住的地方

这题最值得记住的，不是“只出现一次的数字”这个题型本身，而是这个位运算思路：

**当题目里出现“其余元素都成对出现，只有一个例外”这种结构时，异或通常就是最自然的解法。**

因为异或天生就带有“相同元素互相抵消”的效果。  
本题刚好和这个性质完全吻合，所以写出来会非常干净。

以后如果再遇到下面这种问题，也可以优先想到异或：

- 成对出现，找唯一一个落单的
- 两个数组里找不同元素
- 用位运算做状态消除

这类题的共同点都在于：  
不是去“记住谁出现过”，而是去“让能抵消的先抵消”。

### 完整代码

```java
import java.util.Arrays;

public class Main {

    public static int singleNumber(int[] nums) {
        int result = 0;
        for (int num : nums) {
            result ^= num;
        }
        return result;
    }

    public static void main(String[] args) {
        int[] nums1 = {2, 2, 1};
        System.out.println("示例 1 输入: " + Arrays.toString(nums1));
        System.out.println("示例 1 输出: " + singleNumber(nums1));
        System.out.println("-----------------------------");

        int[] nums2 = {4, 1, 2, 1, 2};
        System.out.println("示例 2 输入: " + Arrays.toString(nums2));
        System.out.println("示例 2 输出: " + singleNumber(nums2));
        System.out.println("-----------------------------");

        int[] nums3 = {1};
        System.out.println("示例 3 输入: " + Arrays.toString(nums3));
        System.out.println("示例 3 输出: " + singleNumber(nums3));
        System.out.println("-----------------------------");
    }
}
```
