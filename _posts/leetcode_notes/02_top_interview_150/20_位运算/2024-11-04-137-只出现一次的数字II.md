## 题目

**描述：**

给你一个整数数组 `nums`，除某个元素仅出现 **一次** 外，其余每个元素都恰好出现 **三次**。请你找出并返回那个只出现了一次的元素。

你必须设计并实现线性时间复杂度的算法且使用常数级空间来解决此问题。

**示例 1:**

- **输入:** `nums = [2,2,3,2]`
- **输出:** 3

**示例 2:**

- **输入:** `nums = [0,1,0,1,0,1,99]`
- **输出:** 99

**提示:**

- `1 <= nums.length <= 3 * 10^4`
- `-2^31 <= nums[i] <= 2^31 - 1`
- `nums` 中，除某个元素仅出现 **一次** 外，其余每个元素都恰好出现 **三次**

## 题解

### 这一题和上一题很像，但异或已经不够用了

上一题“只出现一次的数字”里，除了一个数之外，其他数都出现两次。  
那种情况下，异或特别好用，因为：

```text
a ^ a = 0
```

成对出现的数会被直接抵消掉，最后剩下的就是答案。

但这一题不一样。  
这里其他数字都出现了 **三次**，而异或只能处理“成对抵消”，不能处理“三个一组归零”。

比如：

```text
2 ^ 2 ^ 2 = 2
```

它并不会变成 `0`。

所以这一题虽然题型看着像上一题，但核心思路已经不能照搬了。  
真正该想的是：

**既然三个相同的数不能整体异或掉，那能不能从二进制的每一位单独统计，把出现三次的影响消掉。**

这就是这题最自然的入口。

### 最容易理解的做法：一位一位统计

先把问题拆开来看。

如果数组里除了一个数只出现一次，其他所有数都出现三次，那么对于二进制的某一位来说，会发生什么？

假设我们只盯着第 `i` 位。

- 那些出现三次的数字，如果这一位是 `1`，它们对总和的贡献一定是 `3`
- 如果这一位是 `0`，贡献就是 `0`

也就是说，**所有出现三次的数字，在这一位上的贡献加起来，一定是 3 的倍数。**

那最后如果这一位统计出来的总和不是 3 的倍数，说明多出来的那一点，必然来自那个只出现一次的数字。

所以我们可以这样做：

1. 遍历 `int` 的 32 个二进制位
2. 对每一位统计数组里有多少个数在这一位上是 `1`
3. 把这个统计结果对 `3` 取模
4. 如果余数不是 `0`，说明答案在这一位上是 `1`

最后把这 32 位重新拼起来，就得到了答案。

### 为什么这样做一定对

这题最关键的地方，其实就一句话：

**出现三次的数，在任意一位上的贡献，最终都会被 3 整除。**

所以如果某一位最后统计出来：

```text
count % 3 == 0
```

说明那个只出现一次的数字在这一位上是 `0`。

如果：

```text
count % 3 == 1
```

说明那个只出现一次的数字在这一位上是 `1`。

由于题目保证除了一个数之外，其余数都恰好出现三次，所以不会出现别的干扰情况。  
这也是为什么按位统计后再模 3，能把答案完整还原出来。

### 用例子走一遍就很清楚了

看示例 1：

```text
nums = [2,2,3,2]
```

先写成二进制：

```text
2 = 10
2 = 10
3 = 11
2 = 10
```

我们按位统计。

#### 看最低位

最低位分别是：

```text
0, 0, 1, 0
```

总共有 `1` 个 `1`，所以：

```text
1 % 3 = 1
```

说明答案的最低位是 `1`。

#### 看次低位

次低位分别是：

```text
1, 1, 1, 1
```

总共有 `4` 个 `1`，所以：

```text
4 % 3 = 1
```

说明答案的这一位也是 `1`。

于是答案的二进制就是：

```text
11
```

也就是 `3`。

这个过程非常直观：  
出现三次的那些 `2`，在每一位上的贡献都会被 `3` 消掉，最后剩下来的就是 `3` 自己那部分信息。

### 代码

```java
public int singleNumber(int[] nums) {
    int ans = 0;

    for (int i = 0; i < 32; i++) {
        int count = 0;

        for (int num : nums) {
            count += (num >> i) & 1;
        }

        if (count % 3 != 0) {
            ans |= (1 << i);
        }
    }

    return ans;
}
```

### 这段代码为什么连负数也能处理

这题的数据范围里是允许负数的，所以这里有一个容易担心的点：  
按位统计之后，负数还能不能正确还原出来？

可以。

因为 Java 的 `int` 本来就是固定 32 位，负数也同样是按补码存储的。  
我们这里统计的是完整的 32 位，包括符号位在内，所以只要每一位都正确恢复，最终拼出来的结果自然也会是正确的那个负数。

换句话说，这个做法不是在处理“数值意义上的正负”，而是在完整地恢复它的 32 位二进制表示。  
位模式对了，结果就对了。

### 还有一种更巧的写法：用两个变量模拟“出现次数模 3”

上面的逐位统计已经足够好理解，也完全满足题目要求。

不过这题还有一种更精炼的做法，会用两个变量 `ones` 和 `twos` 来记录每一位当前出现次数对 3 取模后的状态。

你可以把每一位出现 `1` 的次数想成一个三状态循环：

- 出现 0 次
- 出现 1 次
- 出现 2 次
- 再来一次就回到 0 次

于是可以用：

- `ones` 记录“这一位目前出现 1 次”的状态
- `twos` 记录“这一位目前出现 2 次”的状态

每读入一个新数字，就让所有位按照这个规则一起更新。

写出来是这样：

```java
ones = (ones ^ num) & ~twos;
twos = (twos ^ num) & ~ones;
```

最后，那个只出现一次的数字会留在 `ones` 里。

这个写法很漂亮，也确实更像位运算题的标准答案。  
但如果只是从“让读者学会”这个角度出发，前面的逐位统计更容易第一次就想明白。

### 为什么 `ones / twos` 这个写法能成立

可以只看某一位来理解。

假设这一位不断读到 `1`，状态会这样循环：

```text
00 -> 01 -> 10 -> 00
```

其中：

- `00` 表示这一位出现了 0 次
- `01` 表示这一位出现了 1 次
- `10` 表示这一位出现了 2 次

第三次再来，就重新归零。

而 `ones` 和 `twos` 这两个变量，其实就是把 32 个二进制位的这种三状态变化同时打包处理了。  
所以它看起来像是一个小技巧，本质上其实是在做“每一位的计数器，只不过模 3”。

### 复杂度分析

逐位统计法的时间复杂度是 $O(n)$。  
虽然代码里是两层循环，但外层固定只有 32 次，所以整体仍然是线性复杂度。

空间复杂度是 $O(1)$。  
只用了常数个额外变量。

`ones / twos` 的写法时间复杂度也是 $O(n)$，空间复杂度同样是 $O(1)$。

### 这题真正值得记住的地方

这题最值得记住的，不是“出现三次找一次”这个题型本身，而是这个思路转换：

**当“成对抵消”失效时，就不要再执着于整体操作，而要把问题拆到每一个二进制位上单独看。**

上一题之所以能直接异或，是因为“出现两次”刚好和异或的抵消性质吻合。  
这一题变成“出现三次”，就说明要把“次数”这件事显式地考虑进去，而最自然的办法就是：

- 按位统计
- 对 3 取模
- 再把答案还原出来

这个思路非常重要。  
以后如果再遇到“除了一个数之外，其余元素都出现 k 次”这类题，通常也都可以从“按位统计 + 对 k 取模”这个方向去想。

### 完整代码

```java
import java.util.Arrays;

public class Main {

    public static int singleNumber(int[] nums) {
        int ans = 0;

        for (int i = 0; i < 32; i++) {
            int count = 0;
            for (int num : nums) {
                count += (num >> i) & 1;
            }
            if (count % 3 != 0) {
                ans |= (1 << i);
            }
        }

        return ans;
    }

    public static int singleNumberOptimized(int[] nums) {
        int ones = 0, twos = 0;

        for (int num : nums) {
            ones = (ones ^ num) & ~twos;
            twos = (twos ^ num) & ~ones;
        }

        return ones;
    }

    public static void main(String[] args) {
        int[] nums1 = {2, 2, 3, 2};
        System.out.println("示例 1 输入: " + Arrays.toString(nums1));
        System.out.println("示例 1 输出: " + singleNumber(nums1));
        System.out.println("-----------------------------");

        int[] nums2 = {0, 1, 0, 1, 0, 1, 99};
        System.out.println("示例 2 输入: " + Arrays.toString(nums2));
        System.out.println("示例 2 输出: " + singleNumber(nums2));
        System.out.println("-----------------------------");

        int[] nums3 = {-2, -2, -2, 1, 1, 1, -3, -3, -3, -4};
        System.out.println("示例 3 输入: " + Arrays.toString(nums3));
        System.out.println("示例 3 输出（逐位统计）: " + singleNumber(nums3));
        System.out.println("示例 3 输出（状态机写法）: " + singleNumberOptimized(nums3));
    }
}
```
