## 题目

**描述：**

给定一个只包括 `'('`，`')'`，`'{'`，`'}'`，`'['`，`']'` 的字符串 `s` ，判断字符串是否有效。

有效字符串需满足：

1.  左括号必须用相同类型的右括号闭合。
2.  左括号必须以正确的顺序闭合。
3.  每个右括号都有一个对应的相同类型的左括号。

**示例 1:**

- **输入:** `s = "()"`
- **输出:** `true`

**示例 2:**

- **输入:** `s = "()[]{}"`
- **输出:** `true`

**示例 3:**

- **输入:** `s = "(]"`
- **输出:** `false`

**示例 4:**

- **输入:** `s = "([)]"`
- **输出:** `false`

**示例 5:**

- **输入:** `s = "{[]}"`
- **输出:** `true`

**提示:**

- `1 <= s.length <= 10^4`
- `s` 仅由括号 `'()[]{}'` 组成

## 题解

### 这题看起来是在配对括号，本质上是在检查“关闭顺序”对不对

如果题目只是要求左括号和右括号数量相同，那事情会简单很多。  
但这题真正要求的不只是“数量能对上”，而是两件更严格的事：

- 类型要对应
- 顺序也要对应

比如：

```text
"([)]"
```

这里每种括号数量都没问题，但它依然是错的。  
因为 `(` 打开之后，按顺序应该最后再关；可中间先遇到了 `)`，把层级打乱了。

所以这题真正要判断的，不是“有没有成对出现”，而是：

**最后打开的那个左括号，是不是最先被正确关掉。**

这句话一出来，数据结构其实就很明确了。

### 为什么这题天然适合用栈

栈最核心的特点就是：**后进先出**。

而括号匹配刚好也是这个逻辑。

比如字符串：

```text
"{[]}"
```

顺着读进去的左括号顺序是：

```text
{  [
```

那它们被关闭的顺序就必须反过来：

```text
]  }
```

也就是说，最后进来的左括号，要最先被处理掉。  
这和栈的行为完全一致，所以这题用栈不是技巧，而是问题结构本身就在要求“后进先出”。

### 一个最容易想到的写法：遇到左括号就入栈

最直接的思路是：

- 遇到左括号，就把它压进栈里
- 遇到右括号，就拿它和栈顶的左括号匹配
- 如果能配上，就弹栈
- 如果配不上，直接返回 `false`

这个思路完全没问题，也是很多人第一次写时最自然的方案。

但这题还有一个更顺手的小技巧：  
**栈里不存左括号，直接存“它未来应该遇到的右括号”。**

### 为什么存“期望的右括号”会更顺

比如遇到：

- `(`，就把 `)` 压栈
- `[`，就把 `]` 压栈
- `{`，就把 `}` 压栈

这样后面真的遇到右括号时，就不需要再做一层映射判断了，直接看：

- 栈是不是空的
- 栈顶字符是不是刚好等于当前字符

如果不是，说明匹配失败。

这种写法的好处是逻辑很直：

**栈顶永远表示“我现在最期待看到哪个右括号”。**

当前字符一来，只要和这个期待值不一致，就知道错了。

### 奇数长度为什么可以直接判错

这题有个很简单但很实用的剪枝：

如果字符串长度是奇数，那一定不可能完全匹配。

原因很直接。  
括号必须成对出现，而成对就意味着总长度一定是偶数。

所以一开始就可以写：

```java
if (s.length() % 2 != 0) {
    return false;
}
```

这个判断不是必须的，因为即使不写，后面正常用栈也能判断出来；  
但它能提前排掉一批明显不可能的情况，让代码更干净一点。

### 用示例走一遍

看最简单的例子：

```text
s = "()"
```

先读到 `(`，把期望的 `)` 压栈。  
再读到 `)`，发现它正好等于栈顶，说明匹配成功，弹栈。

最后栈空了，返回 `true`。

再看：

```text
s = "(]"
```

先读到 `(`，压进去的是 `)`。  
接着读到 `]`，但栈顶期待的是 `)`，两者不一样，直接返回 `false`。

再看最典型的错例：

```text
s = "([)]"
```

读到 `(`，压 `)`。  
读到 `[`，压 `]`。

这时栈顶期待的是 `]`，可下一个字符却是 `)`。  
说明关闭顺序错了，当场判错，不需要继续往后看。

这就是这题为什么不能只数数量，而必须按顺序处理。

### 两种错误，其实都能被栈自然处理掉

这题常见的无效情况其实就两类。

#### 第一类：右括号来得太早

比如：

```text
")("
```

第一个字符就是右括号，但这时候栈还是空的。  
说明它根本没有对应的左括号，直接返回 `false`。

#### 第二类：左括号最后没关完

比如：

```text
"(()"
```

遍历结束时，栈里还剩一个期待的 `)` 没被满足。  
这说明还有左括号没闭合，仍然是无效字符串。

所以最终除了中途匹配失败之外，最后还要再检查一件事：

**栈必须为空。**

只有栈空了，才说明所有打开的括号都被正确关闭了。

### 代码

```java
public boolean isValid(String s) {
    if (s.length() % 2 != 0) {
        return false;
    }

    Deque<Character> stack = new ArrayDeque<>();

    for (char c : s.toCharArray()) {
        if (c == '(') {
            stack.push(')');
        } else if (c == '[') {
            stack.push(']');
        } else if (c == '{') {
            stack.push('}');
        } else {
            if (stack.isEmpty() || stack.pop() != c) {
                return false;
            }
        }
    }

    return stack.isEmpty();
}
```

时间复杂度是 $O(n)$，因为整个字符串只遍历一遍。  
空间复杂度是 $O(n)$，最坏情况下所有字符都是左括号，栈里会压入 `n` 个元素。

### 这题真正值得记住的地方

这题最值得记住的，不只是“括号匹配用栈”这个结论，而是更一般的一种判断：

**只要一个问题要求“最后开始的事情必须最先结束”，就应该优先想到栈。**

括号匹配是这个模式里最经典的例子。  
因为每遇到一个左括号，本质上就是“开启了一个尚未完成的任务”；  
而当右括号到来时，必须先去结束最近开启、但还没结束的那个任务。

这就是栈的工作方式。

以后再遇到类似场景，比如：

- 表达式匹配
- 嵌套结构解析
- 路径回退
- 成对消除

只要本质上带着“后进先出”的味道，都可以优先往栈上想。

### 完整代码

```java
import java.util.ArrayDeque;
import java.util.Deque;

public class Main {

    public static boolean isValid(String s) {
        if (s.length() % 2 != 0) {
            return false;
        }

        Deque<Character> stack = new ArrayDeque<>();

        for (char c : s.toCharArray()) {
            if (c == '(') {
                stack.push(')');
            } else if (c == '[') {
                stack.push(']');
            } else if (c == '{') {
                stack.push('}');
            } else {
                if (stack.isEmpty() || stack.pop() != c) {
                    return false;
                }
            }
        }

        return stack.isEmpty();
    }

    public static void main(String[] args) {
        String s1 = "()";
        System.out.println("示例 1 输入: " + s1);
        boolean res1 = isValid(s1);
        System.out.println("示例 1 输出: " + res1);
        System.out.println("-----------------------------");

        String s2 = "()[]{}";
        System.out.println("示例 2 输入: " + s2);
        boolean res2 = isValid(s2);
        System.out.println("示例 2 输出: " + res2);
        System.out.println("-----------------------------");

        String s3 = "(]";
        System.out.println("示例 3 输入: " + s3);
        boolean res3 = isValid(s3);
        System.out.println("示例 3 输出: " + res3);
        System.out.println("-----------------------------");

        String s4 = "([)]";
        System.out.println("示例 4 输入: " + s4);
        boolean res4 = isValid(s4);
        System.out.println("示例 4 输出: " + res4);
        System.out.println("-----------------------------");

        String s5 = "{[]}";
        System.out.println("示例 5 输入: " + s5);
        boolean res5 = isValid(s5);
        System.out.println("示例 5 输出: " + res5);
    }
}
```
