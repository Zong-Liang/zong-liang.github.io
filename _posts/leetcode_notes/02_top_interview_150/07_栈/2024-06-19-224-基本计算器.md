## 题目

**描述：**

给你一个字符串表达式 `s` ，请你实现一个基本计算器来计算并返回它的值。

注意: 不允许使用任何将字符串作为数学表达式计算的内置函数，比如 `eval()` 。

**示例 1:**

- **输入:** `s = "1 + 1"`
- **输出:** 2

**示例 2:**

- **输入:** `s = " 2-1 + 2 "`
- **输出:** 3

**示例 3:**

- **输入:** `s = "(1+(4+5+2)-3)+(6+8)"`
- **输出:** 23

**提示:**

- `1 <= s.length <= 3 * 10^5`
- `s` 由数字、`'+'`、`'-'`、`'('`、`')'`、和 `' '` 组成
- `s` 表示一个有效的表达式
- `'+'` 不能用作一元运算 (例如， `"+1"` 和 `"+(2 + 3)"` 无效)
- `'-'` 可以用作一元运算 (即 `"-1"` 和 `"-(2 + 3)"` 是有效的)
- 输入中不存在两个连续的操作符
- 每个数字和运行的计算将适合于一个有符号的 32 位整数

## 题解

### 这题难的不是加减法，而是括号会改变当前这层的计算环境

如果表达式里只有数字、加号、减号，没有括号，那其实很好处理。  
顺着扫过去，遇到数字就按当前符号加到结果里就行。

真正让这题变复杂的，是括号。

因为一旦进入括号，等于开始了一个新的子表达式。  
这个子表达式要先独立算完，算出来之后，再带着括号前面的符号回到外层。

所以这题真正要解决的，不是“怎么做加减法”，而是：

**进入括号之前，怎么把当前这一层的状态保存起来；括号结束之后，又怎么把里面的结果接回外面。**

### 先把没有括号时的计算想清楚

如果表达式像这样：

```text
1 + 2 - 3 + 4
```

其实只需要维护两个东西：

- `res`：当前已经算出来的结果
- `sign`：接下来读到的数字，前面应该带什么符号

比如：

- 遇到 `+`，就把 `sign` 设成 `1`
- 遇到 `-`，就把 `sign` 设成 `-1`
- 遇到数字，就做 `res += sign * num`

因为题目里只有加减，没有乘除，所以不需要考虑优先级问题。  
正常情况下，表达式就是这样一路累加下去的。

### 真正的关键，是遇到左括号时要把“现场”保存下来

假设你正在算：

```text
1 - (2 + 3)
```

当你扫到左括号 `(` 时，外层已经有一部分结果了：

- 外层当前结果是 `1`
- 而括号前面的符号是 `-`

接下来括号里的 `2 + 3` 应该单独算，不能直接和外层混在一起。  
所以进入括号之前，必须先把这两个信息存起来：

- 当前层的 `res`
- 当前括号前的 `sign`

然后把当前环境重置，开始专心计算括号内部：

- `res = 0`
- `sign = 1`

这一步其实就是在说：

**我先把外层暂停，进去把括号里的这部分单独算完。**

### 为什么这里适合用栈

括号是可以嵌套的。  
也就是说，你可能刚进了一个括号，还没算完，里面又遇到了新的括号。

这种场景下，最近进入的那一层括号，显然要最先结束。  
这就是非常标准的后进先出结构，所以适合用栈保存状态。

每次遇到 `(`：

- 把当前层的 `res` 压栈
- 再把当前层的 `sign` 压栈

每次遇到 `)`：

- 先弹出括号前的符号
- 再弹出括号前的结果
- 然后把括号内结果接回去

所以这里栈里存的，不是数字本身，而是：

**进入当前括号之前，外层那一层的计算状态。**

### 遇到右括号时，为什么要这样合并

假设现在已经把括号里面算完了，当前的 `res` 表示的是括号内部的值。

比如：

```text
1 - (2 + 3)
```

当括号内的 `2 + 3` 算完之后，当前 `res = 5`。

这时候从栈里取出外层状态：

- `prevSign = -1`
- `prevRes = 1`

那么整个表达式就应该恢复成：

```text
prevRes + prevSign * 当前括号结果
```

也就是：

```text
1 + (-1) * 5 = -4
```

所以右括号时的核心合并公式就是：

```java
res = prevRes + prevSign * res;
```

这一步就是把“括号内部独立算出的结果”，重新接回到外层表达式里。

### 这题为什么不需要真的建表达式树

很多人第一次看“计算器”题，会担心是不是要处理复杂的表达式解析。

但这题其实比那类题简单很多，因为它只有：

- 数字
- `+`
- `-`
- 括号
- 空格

没有乘除，也就没有更高优先级。  
所以整个表达式始终可以看成“一串带符号的数字和子表达式的累加”。

也正因为如此，我们不需要操作符栈，不需要优先级比较，更不需要构造语法树。  
只要维护当前结果、当前符号，再用栈处理括号层级，就够了。

### 多位数字为什么要连续读

题目里的数字不一定只有一位。  
所以一旦遇到数字字符，不能只读一个字符就结束，而是要把后面连续的数字一起读完。

比如读到：

```text
"123"
```

就要把它解析成整数 `123`，再按当前符号加入结果。

这也是为什么代码里通常会有一个内层 `while`，专门负责把完整数字读出来。

### 用示例 3 走一遍会更有感觉

看这个表达式：

```text
(1+(4+5+2)-3)+(6+8)
```

最外层先遇到 `(`，说明进入第一层括号，先把外层状态压栈。

然后往里算：

- 先有 `1`
- 再遇到 `+`
- 再遇到一个新的 `(`，说明又进入更深一层

在这个更深的括号里：

```text
4+5+2
```

可以直接算出 `11`。

遇到右括号之后，把这层结果接回上一层，于是外层这部分变成：

```text
1 + 11 - 3
```

算出来是 `9`。

后面再处理：

```text
+(6+8)
```

括号里算出 `14`，接回去之后就是：

```text
9 + 14 = 23
```

整个过程并不是在“展开表达式”，而是在不同括号层级之间不断切换和回退。

### 代码

```java
public int calculate(String s) {
    Deque<Integer> stack = new ArrayDeque<>();
    int sign = 1;
    int res = 0;
    int n = s.length();

    for (int i = 0; i < n; i++) {
        char ch = s.charAt(i);

        if (Character.isDigit(ch)) {
            int num = 0;
            while (i < n && Character.isDigit(s.charAt(i))) {
                num = num * 10 + (s.charAt(i) - '0');
                i++;
            }
            res += sign * num;
            i--;
        } else if (ch == '+') {
            sign = 1;
        } else if (ch == '-') {
            sign = -1;
        } else if (ch == '(') {
            stack.push(res);
            stack.push(sign);
            res = 0;
            sign = 1;
        } else if (ch == ')') {
            int prevSign = stack.pop();
            int prevRes = stack.pop();
            res = prevRes + prevSign * res;
        }
    }

    return res;
}
```

时间复杂度是 $O(n)$，因为整个字符串只会被顺着扫描一遍。  
空间复杂度是 $O(n)$，最坏情况下如果括号嵌套很深，栈里需要保存很多层状态。

### 这题真正值得记住的地方

这题最值得记住的，不只是“计算器题用栈”这个结论，而是更一般的一种处理方式：

**当一个表达式问题可以分层计算，而某一层结束后还要回到上一层继续时，就应该考虑把当前层的状态压栈保存。**

在这题里，所谓“状态”就是：

- 当前已经算出的结果 `res`
- 当前这层接下来要使用的符号 `sign`

括号一开，就保存外层状态；  
括号一关，就恢复外层状态并把内层结果接回去。

把这个思路想通之后，这题就不再像“解释器”那么吓人了，本质上只是一个带状态回退的线性扫描。

### 完整代码

```java
import java.util.ArrayDeque;
import java.util.Deque;

public class Main {

    public static int calculate(String s) {
        Deque<Integer> stack = new ArrayDeque<>();

        int sign = 1;
        int res = 0;
        int length = s.length();

        for (int i = 0; i < length; i++) {
            char ch = s.charAt(i);

            if (Character.isDigit(ch)) {
                int num = 0;
                while (i < length && Character.isDigit(s.charAt(i))) {
                    num = num * 10 + (s.charAt(i) - '0');
                    i++;
                }
                res += sign * num;
                i--;
            } else if (ch == '+') {
                sign = 1;
            } else if (ch == '-') {
                sign = -1;
            } else if (ch == '(') {
                stack.push(res);
                stack.push(sign);
                res = 0;
                sign = 1;
            } else if (ch == ')') {
                int prevSign = stack.pop();
                int prevRes = stack.pop();
                res = prevRes + prevSign * res;
            }
        }

        return res;
    }

    public static void main(String[] args) {
        String s1 = "1 + 1";
        System.out.println("示例 1 输入: \"" + s1 + "\"");
        int res1 = calculate(s1);
        System.out.println("示例 1 输出: " + res1);
        System.out.println("-----------------------------");

        String s2 = " 2-1 + 2 ";
        System.out.println("示例 2 输入: \"" + s2 + "\"");
        int res2 = calculate(s2);
        System.out.println("示例 2 输出: " + res2);
        System.out.println("-----------------------------");

        String s3 = "(1+(4+5+2)-3)+(6+8)";
        System.out.println("示例 3 输入: \"" + s3 + "\"");
        int res3 = calculate(s3);
        System.out.println("示例 3 输出: " + res3);
        System.out.println("-----------------------------");

        String s4 = "- (3 + (4 - 5))";
        System.out.println("示例 4 输入: \"" + s4 + "\"");
        int res4 = calculate(s4);
        System.out.println("示例 4 输出: " + res4);
    }
}
```
