## 题目

**描述：**

设计一个支持 `push` ，`pop` ，`top` 操作，并能在常数时间内检索到最小元素的栈。

实现 `MinStack` 类:

- `MinStack()` 初始化堆栈对象。
- `void push(int val)` 将元素 val 推入堆栈。
- `void pop()` 删除堆栈顶部的元素。
- `int top()` 获取堆栈顶部的元素。
- `int getMin()` 获取堆栈中的最小元素。

**示例 1:**

- **输入:**

  `["MinStack","push","push","push","getMin","pop","top","getMin"]`

  `[[],[-2],[0],[-3],[],[],[],[]]`

- **输出:**

  `[null,null,null,null,-3,null,0,-2]`

- **解释:**

  `MinStack minStack = new MinStack();`

  `minStack.push(-2);`

  `minStack.push(0);`

  `minStack.push(-3);`

  `minStack.getMin();` `--> 返回 -3.`

  `minStack.pop();`

  `minStack.top();` `--> 返回 0.`

  `minStack.getMin();` `--> 返回 -2.`

**提示:**

- `-2^31 <= val <= 2^31 - 1`
- `pop`、`top` 和 `getMin` 操作总是在 **非空栈** 上调用
- `push`, `pop`, `top`, and `getMin` 最多被调用 `3 * 10^4` 次

## 题解

这题难点不在“栈”本身，而在于最小值也必须跟着历史一起维护。普通栈只需要支持压栈、弹栈和取栈顶，但这里额外要求 `getMin()` 也得是 `O(1)`。这意味着最小值不能等查询时再现算，因为一旦每次都重新遍历整个栈，时间复杂度就会退化成 `O(n)`。

只用一个普通栈显然不够。假设栈里当前是 `[-2, 0, -3]`，这时最小值是 `-3`。如果执行一次 `pop()` 把 `-3` 弹掉，新的最小值应该变成 `-2`。问题在于，如果你没有提前把“过去某一时刻的最小值”记下来，`-3` 一旦出栈，你就只能重新扫一遍剩余元素才能知道答案。这正是题目不允许发生的事。

所以更自然的做法，是准备两个栈：一个普通数据栈 `stack`，照常存所有元素；另一个辅助栈 `minStack`，专门记录“到当前为止的最小值历史”。这样 `top()` 还是看 `stack` 的栈顶，而 `getMin()` 直接看 `minStack` 的栈顶，就能一直保持常数时间。

关键在于这两个栈要怎么同步。`push(val)` 时，数据栈一定要压入 `val`；辅助栈则只有在 `minStack` 为空，或者 `val <= minStack.peek()` 时才压入。这里的 `<=` 很重要，不能只写成 `<`。因为如果最小值重复出现，而你没有把重复的那个最小值也记进去，后面一旦弹出其中一个，辅助栈里的历史就会提前断掉。比如依次压入 `[-2, 0, -2]`，如果第二个 `-2` 没有同步进入 `minStack`，那么它一出栈，`minStack` 就会错误地以为当前最小值已经不再是 `-2` 了。

`pop()` 的同步规则反而很直接：如果数据栈弹出的值恰好等于当前最小值，那么 `minStack` 也要一起弹掉；如果弹出的不是当前最小值，辅助栈就不用动。这样做的本质就是把“最小值历史”跟着真实栈状态一起回退。

题目给的操作序列可以很好地说明这个过程：先 `push(-2)`，两个栈都变成 `[-2]`；再 `push(0)`，数据栈是 `[0, -2]`，而 `minStack` 仍然只有 `[-2]`；接着 `push(-3)`，因为 `-3` 更小，所以两个栈分别变成 `[-3, 0, -2]` 和 `[-3, -2]`。这时 `getMin()` 直接返回 `-3`。然后执行 `pop()` 把 `-3` 弹掉，因为它正好是当前最小值，所以 `minStack` 也同步弹出 `-3`，此时新的最小值自然就回到了 `-2`。

代码如下：

```java
class MinStack {
    private Deque<Integer> stack;
    private Deque<Integer> minStack;

    public MinStack() {
        stack = new ArrayDeque<>();
        minStack = new ArrayDeque<>();
    }

    public void push(int val) {
        stack.push(val);
        if (minStack.isEmpty() || val <= minStack.peek()) {
            minStack.push(val);
        }
    }

    public void pop() {
        int val = stack.pop();
        if (val == minStack.peek()) {
            minStack.pop();
        }
    }

    public int top() {
        return stack.peek();
    }

    public int getMin() {
        return minStack.peek();
    }
}
```

四个操作的时间复杂度都是 `O(1)`；空间复杂度是 `O(n)`，因为最坏情况下如果元素单调递减，辅助栈会和数据栈一样大。

这题更值得带走的，不只是“双栈做最小栈”这个答案，而是一种更通用的设计习惯：当题目要求某个全局信息必须实时、常数时间可查时，往往就不能等查询发生时再现算，而要在每次更新操作里顺手把这个信息维护出来。本题里被维护的那个全局信息，就是当前最小值。

完整代码如下：

```java
import java.util.ArrayDeque;
import java.util.Deque;
public class Main {
    static class MinStack {
        private Deque<Integer> stack;
        private Deque<Integer> minStack;
        public MinStack() {
            stack = new ArrayDeque<>();
            minStack = new ArrayDeque<>();
        }
        public void push(int val) {
            stack.push(val);
            if (minStack.isEmpty() || val <= minStack.peek()) {
                minStack.push(val);
            }
        }
        public void pop() {
            int popVal = stack.pop();
            if (popVal == minStack.peek()) {
                minStack.pop();
            }
        }
        public int top() {
            return stack.peek();
        }
        public int getMin() {
            return minStack.peek();
        }
    }
    public static void main(String[] args) {
        System.out.println("示例 1 开始测试:");
        MinStack minStack = new MinStack();
        System.out.println("push(-2)");
        minStack.push(-2);
        System.out.println("push(0)");
        minStack.push(0);
        System.out.println("push(-3)");
        minStack.push(-3);
        System.out.println("getMin() --> " + minStack.getMin());
        System.out.println("pop()");
        minStack.pop();
        System.out.println("top() --> " + minStack.top());
        System.out.println("getMin() --> " + minStack.getMin());
    }
}
```
