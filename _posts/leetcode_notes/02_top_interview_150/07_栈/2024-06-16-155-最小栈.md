## 题目

**描述：**

设计一个支持 `push` ，`pop` ，`top` 操作，并能在常数时间内检索到最小元素的栈。

实现 `MinStack` 类:

- `MinStack()` 初始化堆栈对象。
- `void push(int val)` 将元素 val 推入堆栈。
- `void pop()` 删除堆栈顶部的元素。
- `int top()` 获取堆栈顶部的元素。
- `int getMin()` 获取堆栈中的最小元素。

**示例 1:**

- **输入:**

  `["MinStack","push","push","push","getMin","pop","top","getMin"]`

  `[[],[-2],[0],[-3],[],[],[],[]]`

- **输出:**

  `[null,null,null,null,-3,null,0,-2]`

- **解释:**

  `MinStack minStack = new MinStack();`

  `minStack.push(-2);`

  `minStack.push(0);`

  `minStack.push(-3);`

  `minStack.getMin();` `--> 返回 -3.`

  `minStack.pop();`

  `minStack.top();` `--> 返回 0.`

  `minStack.getMin();` `--> 返回 -2.`

**提示:**

- `-2^31 <= val <= 2^31 - 1`
- `pop`、`top` 和 `getMin` 操作总是在 **非空栈** 上调用
- `push`, `pop`, `top`, and `getMin` 最多被调用 `3 * 10^4` 次

## 题解

### 这题真正难的地方，不是栈，而是“最小值也要跟着历史一起维护”

普通栈只需要解决一件事：  
把元素压进去、弹出来，并且能拿到栈顶。

但这题额外要求：

**在任意时刻，都要用 `O(1)` 时间拿到当前栈里的最小值。**

这就意味着，`getMin()` 不能临时去遍历整个栈。  
因为一旦每次都重新扫一遍，时间复杂度就会退化成 $O(n)$，题目要求就没法满足。

所以这题真正的关键不是“栈怎么写”，而是：

**当元素不断进出栈时，怎么把“当前最小值”也同步维护好。**

### 为什么只用一个普通栈不够

假设栈里现在是：

```text
[-2, 0, -3]
```

这时候最小值显然是 `-3`。

如果执行一次 `pop()`，`-3` 被弹掉了，新的最小值应该变成 `-2`。  
问题在于，如果你没有提前把“过去某一时刻的最小值”记下来，那 `-3` 一弹掉，你就不知道新的最小值是谁了，只能重新扫一遍剩余元素。

而题目偏偏不允许你这么做。

所以这题的核心思路其实很直接：

**最小值不能事后现算，而要在每次 `push` 和 `pop` 的时候同步维护。**

### 最自然的做法：一个数据栈 + 一个最小栈

最稳妥的写法，就是用两个栈。

- 一个普通栈 `stack`，正常存所有元素
- 一个辅助栈 `minStack`，专门存“到当前为止的最小值信息”

你可以把 `minStack` 理解成一个“最小值历史记录栈”。

它的栈顶永远表示：  
**当前 `stack` 里所有元素中的最小值。**

这样一来：

- `top()` 还是看普通栈栈顶
- `getMin()` 直接看辅助栈栈顶

两者都能保持 `O(1)`。

### Push 时为什么要把最小值也一起处理

当新元素 `val` 入栈时，有两种情况：

#### 第一种：`val` 比当前最小值还小

那很显然，新的最小值就变成了 `val`，所以它也必须压进 `minStack`。

#### 第二种：`val` 没有更小

这时候普通栈照常压入 `val`，但辅助栈不一定要动。  
因为当前最小值并没有发生变化。

所以 `push` 的判断条件就是：

- 如果 `minStack` 为空，直接压
- 或者 `val <= minStack.peek()`，也压进去

这里有个细节很重要：  
必须是 **小于等于**，不能只写成小于。

### 为什么一定要写 `<=`，而不是 `<`

这个点很容易被忽略，但其实很关键。

假设依次压入：

```text
[-2, 0, -2]
```

如果你只有在 `val < minStack.peek()` 时才把值压进 `minStack`，那么第二个 `-2` 就不会被记录。

接下来如果弹出栈顶这个 `-2`，你会把普通栈里的最小值弹掉，但辅助栈里只保存过一个 `-2`，一弹就没了。  
可实际上栈里还剩下一个 `-2`，当前最小值仍然应该是 `-2`。

所以只要新值 **等于** 当前最小值，也必须同步压入辅助栈。  
这样重复最小值才能被正确处理。

### Pop 时为什么辅助栈有时候也要一起弹

当普通栈弹出一个元素时，如果这个元素正好等于当前最小值，那么说明最小值历史也应该回退一层。

所以这时候要把 `minStack` 的栈顶也一起弹掉。

如果弹出的元素不是当前最小值，那辅助栈就不用动。

也就是说，`pop()` 的关键判断就是：

```java
if (popVal == minStack.peek()) {
    minStack.pop();
}
```

这一点对应的其实是一个很直观的事实：

**只有当你把当前最小值本身弹走时，最小值才需要更新。**

### 用示例走一遍

题目给的操作是：

```text
push(-2)
push(0)
push(-3)
getMin()
pop()
top()
getMin()
```

我们顺着看。

先 `push(-2)`：

- `stack = [-2]`
- `minStack = [-2]`

再 `push(0)`：

- `stack = [0, -2]`
- `minStack = [-2]`

因为 `0` 不是更小的值，所以辅助栈不变。

再 `push(-3)`：

- `stack = [-3, 0, -2]`
- `minStack = [-3, -2]`

因为 `-3 <= -2`，它要被记进辅助栈。

这时 `getMin()`，直接看 `minStack` 栈顶，就是 `-3`。

接着执行 `pop()`，弹掉的是 `-3`：

- `stack = [0, -2]`
- `minStack = [-2]`

因为弹掉的正好是当前最小值，所以辅助栈也一起弹。

这时：

- `top()` 返回 `0`
- `getMin()` 返回 `-2`

整个过程都只是在栈顶操作，所以时间复杂度始终是常数级。

### 代码

```java
class MinStack {
    private Deque<Integer> stack;
    private Deque<Integer> minStack;

    public MinStack() {
        stack = new ArrayDeque<>();
        minStack = new ArrayDeque<>();
    }

    public void push(int val) {
        stack.push(val);
        if (minStack.isEmpty() || val <= minStack.peek()) {
            minStack.push(val);
        }
    }

    public void pop() {
        int val = stack.pop();
        if (val == minStack.peek()) {
            minStack.pop();
        }
    }

    public int top() {
        return stack.peek();
    }

    public int getMin() {
        return minStack.peek();
    }
}
```

四个操作的时间复杂度都是 $O(1)$。  
空间复杂度是 $O(n)$，因为最坏情况下如果元素单调递减，辅助栈会和数据栈一样大。

### 进阶写法为什么不如双栈直观

这题还有一种常见写法，是不再单独开辅助栈，而是在一个栈里存“当前值和最小值的差”。

这种方法也能做，但理解门槛更高，而且还要小心整数溢出，通常需要用 `long` 来存。

如果是为了做题通过，它当然没问题；  
但如果是为了把思路讲清楚，双栈法仍然是更自然、也更稳定的版本。

因为双栈法几乎没有拐弯：

- 一个栈管数据
- 一个栈管最小值历史

逻辑很直，代码也不绕。

### 这题真正值得记住的地方

这题最值得记住的，不只是“最小栈用双栈”这个结论，而是背后的思路：

**当题目要求某个全局信息能被实时、常数时间地查询时，往往就不能等到查询时再现算，而要在更新操作发生的同时，把这个信息维护出来。**

在这题里，这个全局信息就是“当前最小值”。

所以真正重要的不是栈本身，而是这种设计习惯：

- 查询快，往往意味着更新时要多做一点事
- 想要 `O(1)` 查询，就得提前把答案维护好

这个思路在很多数据结构设计题里都会反复出现。

### 完整代码

```java
import java.util.ArrayDeque;
import java.util.Deque;

public class Main {

    static class MinStack {
        private Deque<Integer> stack;
        private Deque<Integer> minStack;

        public MinStack() {
            stack = new ArrayDeque<>();
            minStack = new ArrayDeque<>();
        }

        public void push(int val) {
            stack.push(val);
            if (minStack.isEmpty() || val <= minStack.peek()) {
                minStack.push(val);
            }
        }

        public void pop() {
            int popVal = stack.pop();
            if (popVal == minStack.peek()) {
                minStack.pop();
            }
        }

        public int top() {
            return stack.peek();
        }

        public int getMin() {
            return minStack.peek();
        }
    }

    public static void main(String[] args) {
        System.out.println("示例 1 开始测试:");

        MinStack minStack = new MinStack();

        System.out.println("push(-2)");
        minStack.push(-2);

        System.out.println("push(0)");
        minStack.push(0);

        System.out.println("push(-3)");
        minStack.push(-3);

        System.out.println("getMin() --> " + minStack.getMin());

        System.out.println("pop()");
        minStack.pop();

        System.out.println("top() --> " + minStack.top());

        System.out.println("getMin() --> " + minStack.getMin());
    }
}
```
