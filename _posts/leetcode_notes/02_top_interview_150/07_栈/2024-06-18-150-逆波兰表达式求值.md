## 题目

**描述：**

给你一个字符串数组 `tokens` ，表示一个根据 [逆波兰表示法](https://baike.baidu.com/item/%E9%80%86%E6%B3%A2%E5%85%B0%E5%BC%8F/128437) (Reverse Polish Notation) 表示的算术表达式。

请你计算该表达式。返回一个表示表达式值的整数。

**注意:**

- 有效的算符为 `'+'`、`'-'`、`'*'` 和 `'/'` 。
- 每个操作数（运算对象）都可能是一个整数，或者另一个表达式。
- 两个整数之间的除法总是 **向零截断** 。
- 表达式中不含除零运算。
- 输入是一个根据逆波兰表示法表示的算术表达式。
- 答案及所有中间计算结果可以用 **32 位** 整数表示。

**示例 1:**

- **输入:** `tokens = ["2","1","+","3","*"]`
- **输出:** 9
- **解释:** 该算式转化为常见的中缀算术表达式为：`((2 + 1) * 3) = 9`

**示例 2:**

- **输入:** `tokens = ["4","13","5","/","+"]`
- **输出:** 6
- **解释:** 该算式转化为常见的中缀算术表达式为：`(4 + (13 / 5)) = 6`

**示例 3:**

- **输入:** `tokens = ["10","6","9","3","+","-11","*","/","*","17","+","5","+"]`
- **输出:** 22

**提示:**

- `1 <= tokens.length <= 10^4`
- `tokens[i]` 是一个算符（`"+"、"-"、"*" 或 "/"`），或是在范围 `[-200, 200]` 内的一个整数

## 题解

### 这题表面上是在算表达式，本质上是在处理“操作数什么时候凑齐”

如果是普通中缀表达式，比如：

```text
(2 + 1) * 3
```

我们往往会先关心运算优先级、括号、先算哪一部分。  
但逆波兰表达式不一样，它已经把这些顺序都提前安排好了。

比如：

```text
["2","1","+","3","*"]
```

它的意思不是让你自己再去判断优先级，而是明确告诉你：

- 先看到两个数字 `2` 和 `1`
- 然后看到 `+`，说明这两个数现在可以算了
- 算完得到结果之后，再和后面的 `3` 继续参与下一步运算

所以这题真正要解决的，不是“怎么解析括号”，而是：

**每当遇到一个运算符时，如何准确拿到它对应的两个操作数。**

### 为什么逆波兰表达式天然适合用栈

逆波兰表达式的特点是：

- 数字先出现
- 运算符后出现
- 一个运算符一出现，就要立刻和它前面最近的两个结果结合

这其实就是一种很典型的“最近得到的结果，最先被继续使用”的结构。

而这种“最近的先拿出来”的模式，正好就是栈。

你可以这样理解：

- 遇到数字，就先压进栈里，表示“这个值已经准备好了”
- 遇到运算符，就从栈里取出最近准备好的两个值来计算
- 计算结果再压回栈里，作为新的“准备好”的值

所以这题用栈，不是因为“表达式题常用栈”，而是因为逆波兰表达式本身就是按这种后进先出的方式在组织中间结果。

### 这题最关键的细节，是弹栈顺序不能反

一看到运算符，确实是要从栈里弹两个数出来。  
但这里有一个特别容易写错的地方：

**先弹出来的，是右操作数；后弹出来的，才是左操作数。**

比如表达式：

```text
["4","13","5","/","+"]
```

当你处理 `"/"` 的时候，栈里顶部两个元素是：

```text
13, 5
```

先弹出来的是 `5`，它应该是右边那个数；  
再弹出来的是 `13`，它才是左边那个数。

所以真正的计算应该是：

```text
13 / 5
```

而不是：

```text
5 / 13
```

加法和乘法因为有交换律，顺序写反了有时候还看不出来；  
但减法和除法一旦顺序反了，结果会直接错掉。

所以这题最核心的一句其实是：

**弹栈时，第一个是右操作数，第二个才是左操作数。**

### 整个过程其实很机械

遍历 `tokens` 数组时，只需要分两类处理。

#### 第一类：当前是数字

直接转成整数，压进栈里。

因为这说明它已经是一个可以被后续运算使用的操作数了。

#### 第二类：当前是运算符

从栈里弹出两个数字：

- 第一个弹出的是右操作数 `num2`
- 第二个弹出的是左操作数 `num1`

然后根据运算符做计算：

- `+` 就算 `num1 + num2`
- `-` 就算 `num1 - num2`
- `*` 就算 `num1 * num2`
- `/` 就算 `num1 / num2`

最后把结果再压回栈里。

等整个数组遍历完之后，栈里只会剩下一个值，那就是最终答案。

### 用示例 1 走一遍

看最简单的例子：

```text
tokens = ["2","1","+","3","*"]
```

先读到 `"2"`，压栈：

```text
[2]
```

再读到 `"1"`，压栈：

```text
[2, 1]
```

然后读到 `"+"`，弹出两个数：

- 右操作数是 `1`
- 左操作数是 `2`

计算得到：

```text
2 + 1 = 3
```

把 `3` 再压回去：

```text
[3]
```

接着读到 `"3"`，压栈：

```text
[3, 3]
```

最后读到 `"*"`，再弹两个数：

- 右操作数是 `3`
- 左操作数是 `3`

计算：

```text
3 * 3 = 9
```

压回去之后，栈里只剩：

```text
[9]
```

所以结果是 `9`。

### 再看示例 2，顺便体会除法顺序

```text
tokens = ["4","13","5","/","+"]
```

先压入 `4`、`13`、`5`。

遇到 `"/"` 时：

- 先弹出 `5`
- 再弹出 `13`

所以算的是：

```text
13 / 5 = 2
```

注意这里是整数除法，并且题目要求向零截断。  
Java 的整数除法本身就符合这个要求，所以不用额外处理。

然后把 `2` 压回去，此时栈里是：

```text
[4, 2]
```

接着遇到 `"+"`，计算：

```text
4 + 2 = 6
```

最终答案就是 `6`。

### 为什么最后栈里一定只剩一个数

因为一个合法的逆波兰表达式，本质上是在不断把两个操作数和一个运算符“折叠”成一个新结果。

每做一次运算：

- 栈里弹出两个数
- 再压回一个结果

也就是说，栈中元素总数会减少 1。

整个表达式处理完之后，所有局部结果都会一步步被合并，最后自然只剩下一个总结果。

这也是为什么最终直接 `pop()` 一次就能拿到答案。

### 代码

```java
public int evalRPN(String[] tokens) {
    Deque<Integer> stack = new ArrayDeque<>();

    for (String token : tokens) {
        if ("+".equals(token) || "-".equals(token) || "*".equals(token) || "/".equals(token)) {
            int num2 = stack.pop();
            int num1 = stack.pop();

            int res = 0;
            switch (token) {
                case "+":
                    res = num1 + num2;
                    break;
                case "-":
                    res = num1 - num2;
                    break;
                case "*":
                    res = num1 * num2;
                    break;
                case "/":
                    res = num1 / num2;
                    break;
            }

            stack.push(res);
        } else {
            stack.push(Integer.parseInt(token));
        }
    }

    return stack.pop();
}
```

时间复杂度是 $O(n)$，因为每个 token 只会被处理一次。  
空间复杂度是 $O(n)$，最坏情况下栈里会暂时存下较多中间结果。

### 这题真正值得记住的地方

这题最值得记住的，不只是“逆波兰表达式用栈”这个结论，而是它背后的判断：

**只要一个问题是在“按顺序读取输入，并且每次都需要把最近形成的中间结果拿出来继续计算”，就应该优先想到栈。**

这题就是这个模式的标准例子：

- 数字入栈，表示中间结果已经就绪
- 运算符出现，表示要把最近的两个结果取出来组合
- 新结果再压回去，等待后续继续参与运算

把这个结构看清楚之后，这题其实不难，真正容易错的只有一个点：  
**左右操作数的顺序不能反。**

### 完整代码

```java
import java.util.ArrayDeque;
import java.util.Deque;

public class Main {

    public static int evalRPN(String[] tokens) {
        Deque<Integer> stack = new ArrayDeque<>();

        for (String token : tokens) {
            if ("+".equals(token) || "-".equals(token) || "*".equals(token) || "/".equals(token)) {
                int num2 = stack.pop();
                int num1 = stack.pop();

                int res = 0;
                switch (token) {
                    case "+":
                        res = num1 + num2;
                        break;
                    case "-":
                        res = num1 - num2;
                        break;
                    case "*":
                        res = num1 * num2;
                        break;
                    case "/":
                        res = num1 / num2;
                        break;
                }

                stack.push(res);
            } else {
                stack.push(Integer.parseInt(token));
            }
        }

        return stack.pop();
    }

    public static void main(String[] args) {
        String[] tokens1 = {"2", "1", "+", "3", "*"};
        System.out.println("示例 1 输入: [\"2\",\"1\",\"+\",\"3\",\"*\"]");
        int res1 = evalRPN(tokens1);
        System.out.println("示例 1 输出: " + res1);
        System.out.println("-----------------------------");

        String[] tokens2 = {"4", "13", "5", "/", "+"};
        System.out.println("示例 2 输入: [\"4\",\"13\",\"5\",\"/\",\"+\"]");
        int res2 = evalRPN(tokens2);
        System.out.println("示例 2 输出: " + res2);
        System.out.println("-----------------------------");

        String[] tokens3 = {"10", "6", "9", "3", "+", "-11", "*", "/", "*", "17", "+", "5", "+"};
        System.out.println("示例 3 输入: [\"10\",\"6\",\"9\",\"3\",\"+\",\"-11\",\"*\",\"/\",\"*\",\"17\",\"+\",\"5\",\"+\"]");
        int res3 = evalRPN(tokens3);
        System.out.println("示例 3 输出: " + res3);
    }
}
```
