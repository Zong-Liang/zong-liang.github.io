## 题目

**描述：**

给你一个字符串数组 `tokens` ，表示一个根据 [逆波兰表示法](https://baike.baidu.com/item/%E9%80%86%E6%B3%A2%E5%85%B0%E5%BC%8F/128437) (Reverse Polish Notation) 表示的算术表达式。

请你计算该表达式。返回一个表示表达式值的整数。

**注意:**

- 有效的算符为 `'+'`、`'-'`、`'*'` 和 `'/'` 。
- 每个操作数（运算对象）都可能是一个整数，或者另一个表达式。
- 两个整数之间的除法总是 **向零截断** 。
- 表达式中不含除零运算。
- 输入是一个根据逆波兰表示法表示的算术表达式。
- 答案及所有中间计算结果可以用 **32 位** 整数表示。

**示例 1:**

- **输入:** `tokens = ["2","1","+","3","*"]`
- **输出:** 9
- **解释:** 该算式转化为常见的中缀算术表达式为：`((2 + 1) * 3) = 9`

**示例 2:**

- **输入:** `tokens = ["4","13","5","/","+"]`
- **输出:** 6
- **解释:** 该算式转化为常见的中缀算术表达式为：`(4 + (13 / 5)) = 6`

**示例 3:**

- **输入:** `tokens = ["10","6","9","3","+","-11","*","/","*","17","+","5","+"]`
- **输出:** 22

**提示:**

- `1 <= tokens.length <= 10^4`
- `tokens[i]` 是一个算符（`"+"、"-"、"*" 或 "/"`），或是在范围 `[-200, 200]` 内的一个整数

## 题解

这题表面是在算表达式，真正要解决的却不是优先级，而是“运算符一出现时，怎么拿到它要用的两个操作数”。如果是普通中缀表达式，我们经常要先想括号和优先级谁先算；但逆波兰表达式把这个顺序已经提前安排好了。数字先出现，运算符后出现，运算符一出现，就表示它前面最近准备好的两个结果现在可以合并了。

这也是它天然适合用栈的原因。遇到数字时，先把它压进栈里，表示“这个值已经准备好，后面随时可以拿来参与运算”；遇到运算符时，就从栈里弹出最近准备好的两个值来计算，算完以后再把结果压回栈里，作为新的中间结果。整个过程本质上就是不断地把最近形成的结果拿出来继续折叠。

这里最容易写错的地方，是弹栈顺序。先弹出来的那个数，其实是右操作数；后弹出来的那个，才是左操作数。这个细节在加法和乘法里不太显眼，因为交换顺序结果也一样；但一到减法和除法，顺序一反就会直接算错。比如 `tokens = ["4","13","5","/","+"]`，处理 `"/"` 时，先弹出的是 `5`，再弹出的是 `13`，真正该算的是 `13 / 5`，而不是 `5 / 13`。

拿 `tokens = ["2","1","+","3","*"]` 走一遍会更直观。先读到 `2` 和 `1`，都压进栈里；读到 `+` 时，弹出 `1` 和 `2`，算出 `3`，再把 `3` 压回去；接着读到 `3`，继续压栈；最后读到 `*`，再弹出两个 `3` 相乘，得到 `9`。等整个数组处理完以后，栈里只剩一个值，它就是最终答案。

所以整道题其实很机械：遍历 `tokens`，数字就入栈，运算符就弹两个数出来计算，再把结果压回去。逆波兰表达式已经把运算顺序编码在输入里了，我们只需要老老实实按这个顺序执行即可。

代码如下：

```java
public int evalRPN(String[] tokens) {
    Deque<Integer> stack = new ArrayDeque<>();

    for (String token : tokens) {
        if ("+".equals(token) || "-".equals(token) || "*".equals(token) || "/".equals(token)) {
            int num2 = stack.pop();
            int num1 = stack.pop();
            int res = 0;

            switch (token) {
                case "+":
                    res = num1 + num2;
                    break;
                case "-":
                    res = num1 - num2;
                    break;
                case "*":
                    res = num1 * num2;
                    break;
                case "/":
                    res = num1 / num2;
                    break;
            }

            stack.push(res);
        } else {
            stack.push(Integer.parseInt(token));
        }
    }

    return stack.pop();
}
```

时间复杂度是 `O(n)`，因为每个 token 只会被处理一次；空间复杂度是 `O(n)`，最坏情况下栈里会暂时存下较多中间结果。

这题比较值得记的，不只是“逆波兰表达式用栈”，而是背后的结构判断：只要一个问题是在按顺序读取输入，而且每一步都需要把最近形成的中间结果拿出来继续计算，就很适合用栈来接这些中间值。这里唯一真正需要刻意记住的细节，就是减法和除法时左右操作数的顺序不能反。

完整代码如下：

```java
import java.util.ArrayDeque;
import java.util.Deque;
public class Main {
    public static int evalRPN(String[] tokens) {
        Deque<Integer> stack = new ArrayDeque<>();
        for (String token : tokens) {
            if ("+".equals(token) || "-".equals(token) || "*".equals(token) || "/".equals(token)) {
                int num2 = stack.pop();
                int num1 = stack.pop();
                int res = 0;
                switch (token) {
                    case "+":
                        res = num1 + num2;
                        break;
                    case "-":
                        res = num1 - num2;
                        break;
                    case "*":
                        res = num1 * num2;
                        break;
                    case "/":
                        res = num1 / num2;
                        break;
                }
                stack.push(res);
            } else {
                stack.push(Integer.parseInt(token));
            }
        }
        return stack.pop();
    }
    public static void main(String[] args) {
        String[] tokens1 = {"2", "1", "+", "3", "*"};
        System.out.println("示例 1 输入: [\"2\",\"1\",\"+\",\"3\",\"*\"]");
        int res1 = evalRPN(tokens1);
        System.out.println("示例 1 输出: " + res1);
        System.out.println("-----------------------------");
        String[] tokens2 = {"4", "13", "5", "/", "+"};
        System.out.println("示例 2 输入: [\"4\",\"13\",\"5\",\"/\",\"+\"]");
        int res2 = evalRPN(tokens2);
        System.out.println("示例 2 输出: " + res2);
        System.out.println("-----------------------------");
        String[] tokens3 = {"10", "6", "9", "3", "+", "-11", "*", "/", "*", "17", "+", "5", "+"};
        System.out.println("示例 3 输入: [\"10\",\"6\",\"9\",\"3\",\"+\",\"-11\",\"*\",\"/\",\"*\",\"17\",\"+\",\"5\",\"+\"]");
        int res3 = evalRPN(tokens3);
        System.out.println("示例 3 输出: " + res3);
    }
}
```
