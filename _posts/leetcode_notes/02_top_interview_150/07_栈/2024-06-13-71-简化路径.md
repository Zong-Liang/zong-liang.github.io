## 题目

**描述：**

给你一个字符串 `path` ，表示指向某一文件或目录的 Unix 风格 **绝对路径** （以 `'/'` 开头），请你将其转化为更加简洁的 **规范路径** 。

在 Unix 风格的文件系统中，一个点 `.` 表示当前目录本身；此外，两个点 `..` 表示将目录切换到上一级（指向父目录）；两者都可以是复杂相对路径的组成部分。任意多个连续的斜杠（即，`'//'` 或 `'///'` 等）都被视为单个斜杠 `'/'` 。对于此问题，任何其他格式的点（例如，`'...'`）均被视为有效的文件/目录名称。

返回的 **简化路径** 必须遵循如下格式：

- 始终以斜杠 `'/'` 开头。
- 两个目录名之间必须只有一个斜杠 `'/'` 。
- 最后一个目录名（如果存在）**不能** 以 `'/'` 结尾。
- 此外，路径仅包含从根目录到目标文件或目录的路径上的目录（即，不含 `.` 或 `..` ）。

返回简化后得到的 **规范路径** 。

**示例 1:**

- **输入:** `path = "/home/"`
- **输出:** `"/home"`
- **解释:** 应删除尾随斜杠。

**示例 2:**

- **输入:** `path = "/home//foo/"`
- **输出:** `"/home/foo"`
- **解释:** 多个连续的斜杠被单个斜杠替换。

**示例 3:**

- **输入:** `path = "/home/user/Documents/../Pictures"`
- **输出:** `"/home/user/Pictures"`
- **解释:** 两个点 `..` 表示上一级目录（父目录）。

**示例 4:**

- **输入:** `path = "/../"`
- **输出:** `"/"`
- **解释:** 不可能从根目录上升一级目录。

**示例 5:**

- **输入:** `path = "/.../a/../b/c/../d/./"`
- **输出:** `"/.../b/d"`
- **解释:** `...` 在这个问题中是一个合法的目录名。

**提示:**

- `1 <= path.length <= 3000`
- `path` 由英文字母，数字，`.`，`/` 或 `_` 组成。
- `path` 是一个有效的 Unix 风格绝对路径。

## 题解

这题表面看是在清洗字符串，真正处理的却是目录层级。路径里那些看起来像普通字符片段的东西，其实都在表达一种“往哪一层走”的动作：`.` 表示留在当前目录，`..` 表示回到上一级，普通目录名表示进入下一层。所以这题如果只从字符串替换的角度去想，很容易越改越乱；更自然的做法，是把它当成一次目录行走过程来模拟。

一旦换成这个视角，栈就很合适了。原因也不复杂：路径回退本质上就是把最近进入的目录拿掉，这恰好对应后进先出。比如当前路径已经走到 `/home/user/Documents`，接下来遇到 `..`，要退回去的显然不是最早进入的 `home`，而是最近进入的 `Documents`。所以栈里保存的，其实就是“当前这条规范路径上还保留下来的目录层级”。

具体做法是先按 `/` 把路径切开，然后逐段处理。切完以后会出现不少空字符串，这是正常现象，因为开头的 `/`、结尾的 `/`，以及中间连续的 `//`，都会在分割后留下空片段。遍历这些片段时，规则只有三种：空字符串和 `.` 直接跳过；`..` 表示回退一层，如果栈不空就弹出栈顶；剩下的都视为普通目录名，压进栈里。这里有个容易忽略的小细节，像 `"..."` 这样的片段并不是 `..`，它只是一个普通目录名，也应该正常入栈。

拿 `"/home/user/Documents/../Pictures"` 走一遍会更直观。处理到 `home`、`user`、`Documents` 时，栈里依次变成 `['home', 'user', 'Documents']`；读到 `..`，说明要回到上一级，于是把 `Documents` 弹掉；再读到 `Pictures`，就把它压进去。最后栈里剩下的是 `['home', 'user', 'Pictures']`，重新拼起来就是 `/home/user/Pictures`。

还有一个边界情况特别值得单独记住：如果当前已经在根目录，再遇到 `..`，也不能真的退到根目录之上。像 `"/../"` 这种输入，最后结果仍然是 `/`。所以当栈已经空了，再遇到 `..` 时，直接忽略就行。

遍历结束后，栈里剩下的就是简化后的有效目录层级。最后只需要按从栈底到栈顶的顺序重新拼接，每个目录前面补一个 `/`；如果栈本来就是空的，说明最终路径就是根目录 `/`。

代码如下：

```java
public String simplifyPath(String path) {
    String[] parts = path.split("/");
    Deque<String> stack = new ArrayDeque<>();

    for (String part : parts) {
        if ("..".equals(part)) {
            if (!stack.isEmpty()) {
                stack.pollLast();
            }
        } else if (part.length() > 0 && !".".equals(part)) {
            stack.offerLast(part);
        }
    }

    if (stack.isEmpty()) {
        return "/";
    }

    StringBuilder result = new StringBuilder();
    for (String dir : stack) {
        result.append("/").append(dir);
    }

    return result.toString();
}
```

时间复杂度是 `O(n)`，因为路径只会被顺着处理一遍；空间复杂度是 `O(n)`，最坏情况下路径里全是有效目录名，栈里需要把它们都存下来。

这题比较值得形成习惯的是：当一个字符串题真正维护的是“层级结构”，而且中途还需要撤销最近一步时，就不要陷在字符串细节里反复替换，优先把它转成栈上的压入和弹出。这样思路会清楚很多。

完整代码如下：

```java
import java.util.ArrayDeque;
import java.util.Deque;
public class Main {
    public static String simplifyPath(String path) {
        String[] parts = path.split("/");
        Deque<String> stack = new ArrayDeque<>();
        for (String part : parts) {
            if ("..".equals(part)) {
                if (!stack.isEmpty()) {
                    stack.pollLast();
                }
            } else if (part.length() > 0 && !".".equals(part)) {
                stack.offerLast(part);
            }
        }
        if (stack.isEmpty()) {
            return "/";
        }
        StringBuilder result = new StringBuilder();
        for (String dir : stack) {
            result.append("/");
            result.append(dir);
        }
        return result.toString();
    }
    public static void main(String[] args) {
        String path1 = "/home/";
        System.out.println("示例 1 输入: " + path1);
        String res1 = simplifyPath(path1);
        System.out.println("示例 1 输出: " + res1);
        System.out.println("-----------------------------");
        String path2 = "/home//foo/";
        System.out.println("示例 2 输入: " + path2);
        String res2 = simplifyPath(path2);
        System.out.println("示例 2 输出: " + res2);
        System.out.println("-----------------------------");
        String path3 = "/home/user/Documents/../Pictures";
        System.out.println("示例 3 输入: " + path3);
        String res3 = simplifyPath(path3);
        System.out.println("示例 3 输出: " + res3);
        System.out.println("-----------------------------");
        String path4 = "/../";
        System.out.println("示例 4 输入: " + path4);
        String res4 = simplifyPath(path4);
        System.out.println("示例 4 输出: " + res4);
        System.out.println("-----------------------------");
        String path5 = "/.../a/../b/c/../d/./";
        System.out.println("示例 5 输入: " + path5);
        String res5 = simplifyPath(path5);
        System.out.println("示例 5 输出: " + res5);
    }
}
```
