## 题目

**描述：**

给你一个字符串 `path` ，表示指向某一文件或目录的 Unix 风格 **绝对路径** （以 `'/'` 开头），请你将其转化为更加简洁的 **规范路径** 。

在 Unix 风格的文件系统中，一个点 `.` 表示当前目录本身；此外，两个点 `..` 表示将目录切换到上一级（指向父目录）；两者都可以是复杂相对路径的组成部分。任意多个连续的斜杠（即，`'//'` 或 `'///'` 等）都被视为单个斜杠 `'/'` 。对于此问题，任何其他格式的点（例如，`'...'`）均被视为有效的文件/目录名称。

返回的 **简化路径** 必须遵循如下格式：

- 始终以斜杠 `'/'` 开头。
- 两个目录名之间必须只有一个斜杠 `'/'` 。
- 最后一个目录名（如果存在）**不能** 以 `'/'` 结尾。
- 此外，路径仅包含从根目录到目标文件或目录的路径上的目录（即，不含 `.` 或 `..` ）。

返回简化后得到的 **规范路径** 。

**示例 1:**

- **输入:** `path = "/home/"`
- **输出:** `"/home"`
- **解释:** 应删除尾随斜杠。

**示例 2:**

- **输入:** `path = "/home//foo/"`
- **输出:** `"/home/foo"`
- **解释:** 多个连续的斜杠被单个斜杠替换。

**示例 3:**

- **输入:** `path = "/home/user/Documents/../Pictures"`
- **输出:** `"/home/user/Pictures"`
- **解释:** 两个点 `..` 表示上一级目录（父目录）。

**示例 4:**

- **输入:** `path = "/../"`
- **输出:** `"/"`
- **解释:** 不可能从根目录上升一级目录。

**示例 5:**

- **输入:** `path = "/.../a/../b/c/../d/./"`
- **输出:** `"/.../b/d"`
- **解释:** `...` 在这个问题中是一个合法的目录名。

**提示:**

- `1 <= path.length <= 3000`
- `path` 由英文字母，数字，`.`，`/` 或 `_` 组成。
- `path` 是一个有效的 Unix 风格绝对路径。

## 题解

### 这题看起来是在处理字符串，其实是在处理目录层级

如果只看表面，这题像是在做字符串清洗：

- 去掉多余的 `/`
- 去掉 `.`
- 处理 `..`

但真正麻烦的地方并不在这些符号本身，而在于它们背后代表的**目录层级变化**。

比如：

- `.` 表示留在当前目录，不动
- `..` 表示回到上一级目录
- 普通目录名表示进入下一层

所以这题本质上不是单纯的字符串替换，而是在模拟一条路径从根目录一步一步往下走、再往上退的过程。

一旦这么想，合适的数据结构其实就很自然了。

### 为什么这题适合用栈

路径里的目录层级，天然符合一种“后进先出”的关系。

比如路径走到：

```text
/home/user/Documents
```

如果接下来遇到 `..`，要退回的显然不是最早进入的 `home`，而是最近进入的 `Documents`。

也就是说，当前路径里最后进入的目录，应该最先被弹掉。  
这正是栈的行为。

所以这题用栈，不是因为“看到路径就用栈”，而是因为：

**`..` 本质上就是把最近压进去的那个目录拿掉。**

### 先把路径拆开，再逐段处理

因为路径是用 `/` 分隔的，所以最直接的做法就是先按 `/` 切开。

比如：

```text
"/a//b/../../c/"
```

分割之后大概会得到这些片段：

```text
["", "a", "", "b", "..", "..", "c", ""]
```

这里会出现很多空字符串，这是正常的。  
因为像开头的 `/`、结尾的 `/`，以及中间连续的 `//`，在分割后都会产生空片段。

所以分割只是第一步，真正关键的是：  
**怎么解释每一个片段代表什么。**

### 处理规则其实很简单，只有三种情况

遍历这些分割出来的片段时，分类处理就行。

#### 第一种：空字符串和 `.`

这两种都不用动。

- 空字符串只是分割产生的噪声
- `.` 表示当前目录，本来就不需要改路径

所以直接跳过。

#### 第二种：`..`

这表示回到上一级目录。

如果栈里还有目录，就弹出一个；  
如果栈已经空了，说明当前已经在根目录，再往上也退不了，这时候直接忽略。

这一点很重要，因为 Unix 的绝对路径不允许退到根目录之上。

#### 第三种：普通目录名

只要既不是空串，也不是 `.`，也不是 `..`，那它就是一个合法目录名，直接压进栈里。

这里要特别注意一个细节：

```text
"..."
```

并不是 `..`，它只是一个普通目录名。  
题目已经明确说了，只有单独的 `.` 和 `..` 有特殊含义，其他点的组合都应该当普通名字处理。

### 用示例走一遍会更清楚

看这个例子：

```text
"/home/user/Documents/../Pictures"
```

先按 `/` 分割，然后从左到右处理：

- `home`：压栈
- `user`：压栈
- `Documents`：压栈
- `..`：弹出栈顶，也就是把 `Documents` 去掉
- `Pictures`：再压栈

最后栈里剩下的是：

```text
["home", "user", "Pictures"]
```

重新拼起来就是：

```text
"/home/user/Pictures"
```

这就是简化后的规范路径。

### 为什么 `"/../"` 最后还是 `"/"`

看示例 4：

```text
"/../"
```

分割之后核心片段其实就是 `..`。

但这时候栈是空的，说明当前已经在根目录。  
根目录没有上一级，所以这个 `..` 不能真的再往上退，只能忽略。

最终栈还是空的，而栈空就表示路径停在根目录，所以答案是：

```text
"/"
```

这个边界情况很常见，也是这题判断逻辑里必须有的一部分。

### 最后一步不是弹栈，而是把栈里的目录重新拼回去

遍历结束后，栈里剩下的就是简化后的有效目录层级。

接下来只要把它们按顺序重新拼接起来即可：

- 每个目录前面补一个 `/`
- 如果栈为空，说明结果就是根目录 `/`

这里有个小点要注意：  
我们需要的是从根目录往下的顺序，所以拼接时应该按“栈底到栈顶”的顺序遍历，而不是反过来弹出来。

如果用 `Deque` 来存，这一点处理起来会比较自然。

### 代码

```java
public String simplifyPath(String path) {
    String[] parts = path.split("/");
    Deque<String> stack = new ArrayDeque<>();

    for (String part : parts) {
        if ("..".equals(part)) {
            if (!stack.isEmpty()) {
                stack.pollLast();
            }
        } else if (part.length() > 0 && !".".equals(part)) {
            stack.offerLast(part);
        }
    }

    if (stack.isEmpty()) {
        return "/";
    }

    StringBuilder result = new StringBuilder();
    for (String dir : stack) {
        result.append("/").append(dir);
    }

    return result.toString();
}
```

时间复杂度是 $O(n)$，因为路径只会被顺着处理一遍。  
空间复杂度是 $O(n)$，最坏情况下路径里全是有效目录名，栈里需要把它们都存下来。

### 这题真正值得记住的地方

这题最值得记住的，不只是“路径题用栈”，而是它背后的判断：

**只要一个字符串问题的本质是在维护层级、并且需要支持“撤销最近一步”，就应该优先想到栈。**

在这题里：

- 进入目录，就是压栈
- 返回上一级，就是弹栈
- 当前目录和多余斜杠，本质上都不改变层级

一旦把问题从“字符串长什么样”切换成“层级怎么变化”，思路就会顺很多。

以后再碰到类似的路径规范化、嵌套结构解析、操作回退之类的题，都可以优先往这个方向想。

### 完整代码

```java
import java.util.ArrayDeque;
import java.util.Deque;

public class Main {

    public static String simplifyPath(String path) {
        String[] parts = path.split("/");
        Deque<String> stack = new ArrayDeque<>();

        for (String part : parts) {
            if ("..".equals(part)) {
                if (!stack.isEmpty()) {
                    stack.pollLast();
                }
            } else if (part.length() > 0 && !".".equals(part)) {
                stack.offerLast(part);
            }
        }

        if (stack.isEmpty()) {
            return "/";
        }

        StringBuilder result = new StringBuilder();
        for (String dir : stack) {
            result.append("/");
            result.append(dir);
        }

        return result.toString();
    }

    public static void main(String[] args) {
        String path1 = "/home/";
        System.out.println("示例 1 输入: " + path1);
        String res1 = simplifyPath(path1);
        System.out.println("示例 1 输出: " + res1);
        System.out.println("-----------------------------");

        String path2 = "/home//foo/";
        System.out.println("示例 2 输入: " + path2);
        String res2 = simplifyPath(path2);
        System.out.println("示例 2 输出: " + res2);
        System.out.println("-----------------------------");

        String path3 = "/home/user/Documents/../Pictures";
        System.out.println("示例 3 输入: " + path3);
        String res3 = simplifyPath(path3);
        System.out.println("示例 3 输出: " + res3);
        System.out.println("-----------------------------");

        String path4 = "/../";
        System.out.println("示例 4 输入: " + path4);
        String res4 = simplifyPath(path4);
        System.out.println("示例 4 输出: " + res4);
        System.out.println("-----------------------------");

        String path5 = "/.../a/../b/c/../d/./";
        System.out.println("示例 5 输入: " + path5);
        String res5 = simplifyPath(path5);
        System.out.println("示例 5 输出: " + res5);
    }
}
```
