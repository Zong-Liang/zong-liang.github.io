## 题目

**描述：**

基因序列可以表示为一条由 8 个字符组成的字符串，其中每个字符都是 `'A'`、`'C'`、`'G'` 和 `'T'` 之一。

假设我们需要调查从基因序列 `start` 变为 `end` 所发生的基因变化。一次基因变化就意味着这个基因序列中的一个字符发生了变化。

- 例如，`"AACCGGTT" --> "AACCGGTA"` 就是一次基因变化。

另有一个基因库 `bank` 记录了所有有效的基因变化，只有基因库中的基因才是有效的基因序列。（变化后的基因必须位于基因库 `bank` 中）

给你两个基因序列 `start` 和 `end`，以及一个基因库 `bank`，请你找出并返回能够使 `start` 变化为 `end` 所需的 **最少变化次数**。如果无法完成此基因变化，返回 `-1`。

**注意：** 起始基因序列 `start` 默认是有效的，但是它并不一定会出现 在基因库中。

**示例 1:**

- **输入:** `start = "AACCGGTT"`, `end = "AACCGGTA"`, `bank = ["AACCGGTA"]`
- **输出:** 1

**示例 2:**

- **输入:** `start = "AACCGGTT"`, `end = "AAACGGTA"`, `bank = ["AACCGGTA", "AACCGCTA", "AAACGGTA"]`
- **输出:** 2

**示例 3:**

- **输入:** `start = "AAAAACCC"`, `end = "AACCCCCC"`, `bank = ["AAAACCCC", "AAACCCCC", "AACCCCCC"]`
- **输出:** 3

**提示:**

- `start.length == 8`
- `end.length == 8`
- `0 <= bank.length <= 10`
- `bank[i].length == 8`
- `start`、`end` 和 `bank[i]` 仅由字符 `['A', 'C', 'G', 'T']` 组成

## 题解

### 先别被“基因”这两个字带偏，这题本质上是在找最短变化路径

题目看起来像字符串模拟题，因为给了 `start`、`end` 和一个基因库 `bank`。

但只要把问题抽象一下，就会发现它其实是一个非常典型的最短路问题。

我们可以把每一个合法基因序列看成图中的一个节点。  
如果两个基因串只有一个字符不同，并且变化后的串出现在 `bank` 中，那它们之间就可以连一条边。

这样一来，题目问的“最少变化次数”，其实就是：

**从 `start` 到 `end`，最少经过多少条边。**

而每次变化的代价都一样，都是 1，所以这就是无权图最短路径问题。

### 为什么这题应该用 BFS

一旦把它看成图，思路就很自然了。

- 当前基因串是一个节点
- 一次合法变化，可以走到下一个节点
- 每次变化的代价都相同
- 题目要求最少变化次数

这种题最适合用广度优先搜索，也就是 BFS。

因为 BFS 天然就是一层一层扩展：

- 第 0 层表示变化 0 次能到的状态
- 第 1 层表示变化 1 次能到的状态
- 第 2 层表示变化 2 次能到的状态

所以当我们第一次找到 `end` 时，那一定就是最少变化次数。

### 这题真正的关键，不是遍历 bank，而是怎么生成“下一步”

很多人第一次写这题时，会先想到这样一种做法：

- 对当前字符串，去和 `bank` 里的每个字符串比较
- 看它们是不是只差一个字符
- 如果是，就当成下一步

这当然也能做，但这题其实有个更顺的角度：

**直接从当前基因串出发，主动生成所有可能的一步变化。**

因为每个基因串长度固定是 8，而且每一位字符只能是：

```text
A、C、G、T
```

所以对于当前串，我们完全可以：

- 枚举 8 个位置
- 每个位置尝试替换成另外 3 种字符
- 得到一个新串
- 再检查这个新串是不是在 `bank` 里

这样就能把“所有一步可达的状态”找出来。

这个角度比“拿当前串去和整个 bank 挨个比”更像 BFS 扩展邻居，逻辑也更统一。

### 为什么要先把 `bank` 转成哈希集合

因为我们在 BFS 过程中会频繁问一个问题：

**某个新生成的基因串是不是合法的？**

如果每次都去数组里线性查找，那效率就比较差。  
所以更自然的做法是先把 `bank` 放进 `HashSet` 里，这样判断一个字符串是否存在就是 `O(1)` 级别。

这一步虽然不难，但很关键，因为它决定了后面“生成新状态 -> 判断是否合法”这套流程能不能顺畅跑起来。

### 为什么 `end` 不在 `bank` 里时可以直接返回 `-1`

题目已经明确说了：

**变化后的基因必须出现在基因库 `bank` 中。**

这意味着什么？

意味着终点 `end` 如果根本不在 `bank` 里，那你无论怎么变，都不可能合法地变到它。

所以开头先做这一步判断：

```java
if (!bankSet.contains(end)) return -1;
```

可以直接剪掉一大类不可能的情况。

### BFS 具体怎么做

先把 `start` 放进队列里，表示我们从它开始做变化。

同时还要准备一个 `visited` 集合，记录已经访问过的基因串，防止重复搜索和来回绕圈。

然后开始 BFS：

1. 取出当前层的所有基因串
2. 对于每个基因串，枚举它的 8 个位置
3. 每个位置尝试替换成 `A、C、G、T`
4. 如果生成的新串：
   - 在 `bank` 中
   - 并且之前没有访问过
     那就把它加入队列
5. 一层处理完，步数加 1

如果过程中第一次遇到 `end`，直接返回当前步数即可。

### 为什么一定要做 `visited`

这个题如果不做去重，会有两个问题。

第一个是效率差。  
同一个合法基因串可能会从不同路径被反复生成，如果每次都重新入队，搜索会非常乱。

第二个是可能来回绕。  
比如某个基因串 A 可以变成 B，B 也可以再变回 A。如果不记录访问状态，BFS 会不断在这些状态之间反复走。

所以 `visited` 的作用很明确：

**每个合法状态只处理一次。**

这其实也是大多数图搜索题的基本操作。

### 用示例理解一下

比如示例 2：

```text
start = "AACCGGTT"
end   = "AAACGGTA"
bank  = ["AACCGGTA", "AACCGCTA", "AAACGGTA"]
```

从 `"AACCGGTT"` 出发，一次变化能到的合法状态里，有一个是：

```text
"AACCGGTA"
```

然后再从 `"AACCGGTA"` 出发，一次变化能到：

```text
"AAACGGTA"
```

所以最短路径就是两步，答案为 `2`。

你会发现，这题的本质和“单词接龙”那类题非常像，都是：

- 当前状态是一个字符串
- 一次操作只能改一个位置
- 只有满足条件的新字符串才是合法下一步
- 求最少操作次数

所以把它当成图的 BFS 去理解，会比把它当成字符串硬模拟更顺。

### 代码

```java
public int minMutation(String start, String end, String[] bank) {
    Set<String> bankSet = new HashSet<>(Arrays.asList(bank));
    if (!bankSet.contains(end)) return -1;

    char[] genes = {'A', 'C', 'G', 'T'};
    Queue<String> queue = new LinkedList<>();
    queue.offer(start);

    Set<String> visited = new HashSet<>();
    visited.add(start);

    int steps = 0;

    while (!queue.isEmpty()) {
        int size = queue.size();

        for (int i = 0; i < size; i++) {
            String curr = queue.poll();
            if (curr.equals(end)) return steps;

            char[] arr = curr.toCharArray();

            for (int j = 0; j < 8; j++) {
                char oldChar = arr[j];

                for (char gene : genes) {
                    if (gene == oldChar) continue;

                    arr[j] = gene;
                    String next = new String(arr);

                    if (bankSet.contains(next) && !visited.contains(next)) {
                        visited.add(next);
                        queue.offer(next);
                    }
                }

                arr[j] = oldChar;
            }
        }

        steps++;
    }

    return -1;
}
```

### 这段代码最关键的地方

关键其实就两层意思。

第一层，是这段“生成邻居”的逻辑：

```java
for (int j = 0; j < 8; j++) {
    for (char gene : genes) {
        ...
    }
}
```

它表示：  
对当前字符串的每一个位置，都去尝试一次单点替换，从而生成所有可能的一步变化。

第二层，是这句判断：

```java
if (bankSet.contains(next) && !visited.contains(next))
```

它表示：  
不是所有变化都能走，只有同时满足“合法”和“没访问过”的状态，才值得继续搜索。

所以这题真正的核心不是“会不会写 BFS 模板”，而是：

**你有没有把“合法的一步变化”定义清楚。**

### 为什么修改完之后要把字符恢复回来

这也是一个很容易忽略的小细节。

我们用的是同一个字符数组 `arr` 来反复尝试修改。  
当某一位尝试完之后，如果不把它恢复成原来的字符，那后面再改别的位置时，基础字符串就已经变了，生成出来的结果就不再是“只改一个字符”的合法下一步了。

所以每一轮结束后，都要做这一步：

```java
arr[j] = oldChar;
```

这不是多余操作，而是在保证每次生成的 `next` 都是从原始当前状态 `curr` 出发，只变化一个位置。

### 复杂度分析

设 `bank` 的大小为 `N`。

每次从队列里取出一个基因串时，我们最多会：

- 枚举 8 个位置
- 每个位置尝试 4 种字符

所以单个状态的扩展代价是常数级的，严格写出来是 `O(8 × 4)`。

因此总时间复杂度可以写成：

- 时间复杂度：$O(N \times 8 \times 4)$

因为 8 和 4 都是常数，实际也可以直接看作 `O(N)` 级别。

空间复杂度主要来自：

- `bankSet`
- `visited`
- `queue`

所以是：

- 空间复杂度：$O(N)$

### 这题真正值得记住的地方

这题最值得记住的，不是“基因变化也能 BFS”这么表面的结论。

真正值得记住的是这个判断：

**只要题目是在问“从一个状态变到另一个状态的最少操作次数”，而且每次操作代价相同，就要优先想 BFS。**

至于状态长什么样，不一定非得是数字或者坐标。  
它也可以是：

- 字符串
- 棋盘状态
- 单词
- 基因序列

很多图搜索题的难点都不在 BFS 本身，而在于：

**你能不能把题目的状态、转移和合法条件建模清楚。**

这题就是一个很典型的例子。

### 完整代码

```java
import java.util.Arrays;
import java.util.HashSet;
import java.util.LinkedList;
import java.util.Queue;
import java.util.Set;

public class Main {

    public static int minMutation(String start, String end, String[] bank) {
        Set<String> bankSet = new HashSet<>();
        for (String s : bank) {
            bankSet.add(s);
        }

        if (!bankSet.contains(end)) {
            return -1;
        }

        char[] geneChars = {'A', 'C', 'G', 'T'};
        Queue<String> queue = new LinkedList<>();
        queue.offer(start);

        Set<String> visited = new HashSet<>();
        visited.add(start);

        int steps = 0;

        while (!queue.isEmpty()) {
            int size = queue.size();

            for (int i = 0; i < size; i++) {
                String curr = queue.poll();

                if (curr.equals(end)) {
                    return steps;
                }

                char[] currArr = curr.toCharArray();

                for (int j = 0; j < 8; j++) {
                    char originalChar = currArr[j];

                    for (char g : geneChars) {
                        if (g == originalChar) {
                            continue;
                        }

                        currArr[j] = g;
                        String nextGene = new String(currArr);

                        if (bankSet.contains(nextGene) && !visited.contains(nextGene)) {
                            visited.add(nextGene);
                            queue.offer(nextGene);
                        }
                    }

                    currArr[j] = originalChar;
                }
            }

            steps++;
        }

        return -1;
    }

    public static void main(String[] args) {
        String start1 = "AACCGGTT";
        String end1 = "AACCGGTA";
        String[] bank1 = {"AACCGGTA"};

        System.out.println("示例 1 输入:");
        System.out.println("start: " + start1);
        System.out.println("end:   " + end1);
        System.out.println("bank:  " + Arrays.toString(bank1));
        System.out.println("示例 1 输出: " + minMutation(start1, end1, bank1));

        String start2 = "AACCGGTT";
        String end2 = "AAACGGTA";
        String[] bank2 = {"AACCGGTA", "AACCGCTA", "AAACGGTA"};

        System.out.println("示例 2 输入:");
        System.out.println("start: " + start2);
        System.out.println("end:   " + end2);
        System.out.println("bank:  " + Arrays.toString(bank2));
        System.out.println("示例 2 输出: " + minMutation(start2, end2, bank2));

        String start3 = "AAAAACCC";
        String end3 = "AACCCCCC";
        String[] bank3 = {"AAAACCCC", "AAACCCCC", "AACCCCCC"};

        System.out.println("示例 3 输入:");
        System.out.println("start: " + start3);
        System.out.println("end:   " + end3);
        System.out.println("bank:  " + Arrays.toString(bank3));
        System.out.println("示例 3 输出: " + minMutation(start3, end3, bank3));
    }
}
```
