## 题目

**描述：**

字典 `wordList` 中从单词 `beginWord` 到 `endWord` 的 **转换序列** 是一个按如下规格形成的序列 `beginWord -> s1 -> s2 -> ... -> sk`：

- 每一对相邻的单词只差一个字母。
- 对于 `1 <= i <= k` 时，每个 `si` 都在 `wordList` 中。注意，`beginWord` 不需要 nécessairement 在 `wordList` 中。
- `sk == endWord`

给你两个单词 `beginWord` 和 `endWord` 和一个字典 `wordList`，返回从 `beginWord` 到 `endWord` 的 **最短转换序列** 中的 **单词数目**。如果不存在这样的转换序列，返回 `0`。

**示例 1:**

- **输入:** `beginWord = "hit"`, `endWord = "cog"`, `wordList = ["hot","dot","dog","lot","log","cog"]`
- **输出:** 5
- **解释:** 一个最短转换序列是 `"hit" -> "hot" -> "dot" -> "dog" -> "cog"`, 返回它的长度 5。

**示例 2:**

- **输入:** `beginWord = "hit"`, `endWord = "cog"`, `wordList = ["hot","dot","dog","lot","log"]`
- **输出:** 0
- **解释:** `endWord` `"cog"` 不在字典中，所以无法进行转换。

**提示:**

- `1 <= beginWord.length <= 10`
- `endWord.length == beginWord.length`
- `1 <= wordList.length <= 5000`
- `wordList[i].length == beginWord.length`
- `beginWord`、`endWord` 和 `wordList[i]` 由小写英文字母组成
- `beginWord != endWord`
- `wordList` 中的所有字符串 **互不相同**

## 题解

### 先把题目翻译成更直接的话

这题表面上是在做单词变换，实际上是在问：

**从 `beginWord` 出发，每次只能改一个字母，并且变完之后的新单词必须在 `wordList` 里，最少几步能变到 `endWord`。**

一旦把题意拆成这句话，就会发现它本质上不是字符串题，而是一个图上的最短路问题。

因为每个单词都可以看成一个节点。  
如果两个单词只差一个字母，而且变化后的单词在字典里合法，那它们之间就可以连一条边。

题目问最短转换序列长度，其实就是在问：

**从起点单词到终点单词，最短要走多少步。**

### 为什么这题应该用 BFS

一旦把每个单词看成节点，这题就很典型了。

- 当前单词是一个状态
- 改一个字母，相当于走到相邻状态
- 每次变化的代价都一样，都是 1
- 题目要求最短路径

这种“每条边权重都相同”的最短路径问题，最适合用广度优先搜索，也就是 BFS。

因为 BFS 天然是按层扩展的：

- 第 1 层表示只变 1 次能到的单词
- 第 2 层表示只变 2 次能到的单词
- 第 3 层表示只变 3 次能到的单词

所以当我们第一次遇到 `endWord` 时，那一定就是最短转换序列。

### 这题真正的关键，不是“比较两个单词差几个字母”

很多人第一次做这题时，会下意识想到一种写法：

- 从当前单词出发
- 去遍历 `wordList` 里的所有单词
- 一个一个判断是不是只差一个字母
- 如果是，就当作下一步

这样当然能做，但不是最顺的讲法。

更自然的角度是：

**不要拿当前单词去和整个字典硬比，而是主动生成它所有可能的一步变化。**

因为题目规定一次只能改一个字母，那我们完全可以：

- 枚举当前单词的每一位
- 把这一位尝试改成 `'a' ~ 'z'`
- 生成一个新单词
- 再检查这个新单词是不是在字典里

这样做更像 BFS 里“从当前节点扩展邻居”，逻辑上也更统一。

### 为什么要先把 `wordList` 变成 `HashSet`

因为我们在 BFS 过程中会频繁问一个问题：

**某个新生成的单词是不是合法单词？**

如果每次都去 `List` 里线性查找，效率会差很多。  
所以第一步很自然就是把 `wordList` 转成 `HashSet`，这样判断某个单词是否存在可以做到接近 `O(1)`。

这一步是后面整套搜索能不能顺利跑起来的基础。

### 为什么 `endWord` 不在字典里时可以直接返回 `0`

题目说得很明确：

**转换过程中的每个中间单词都必须在 `wordList` 中。**

终点单词当然也不例外。

所以如果 `endWord` 根本不在字典里，那说明无论怎么变，最后都不可能合法地落到它身上。

这种情况可以一开始就直接剪掉：

```java
if (!wordSet.contains(endWord)) return 0;
```

这样能省掉很多没必要的搜索。

### BFS 具体怎么做

先把 `beginWord` 放进队列里，表示从它开始扩展。

然后按层做 BFS：

1. 每次取出当前层所有单词
2. 对每个单词，枚举每一个位置
3. 每个位置尝试替换成 `'a' ~ 'z'`
4. 得到新单词 `nextWord`
5. 如果 `nextWord` 就是 `endWord`，直接返回答案
6. 如果 `nextWord` 在字典里，就把它加入队列，作为下一层继续扩展

这就是这题的主线。

### 为什么这里可以直接从 `wordSet` 里删除已访问单词

这题有一个很好用的小技巧：

**不用额外开 `visited` 集合，直接把访问过的单词从 `wordSet` 里删掉。**

原因很简单。

只要某个单词已经入队，说明它已经会在后续某一层被处理。  
那它就没必要再被其他路径重复加入队列了。

所以：

```java
queue.offer(nextWord);
wordSet.remove(nextWord);
```

这两步放在一起，效果就等同于“加入队列并标记已访问”。

这样写的好处是代码更紧凑，也更不容易漏掉去重逻辑。

### 用示例看一遍

比如示例 1：

```text
beginWord = "hit"
endWord = "cog"
wordList = ["hot","dot","dog","lot","log","cog"]
```

从 `"hit"` 出发，一步合法变化里能找到 `"hot"`。

再从 `"hot"` 出发，可以到 `"dot"` 和 `"lot"`。

继续扩展：

- `"dot"` 可以到 `"dog"`
- `"lot"` 可以到 `"log"`

然后 `"dog"` 可以到 `"cog"`。

所以最短路径之一就是：

```text
"hit" -> "hot" -> "dot" -> "dog" -> "cog"
```

总共 5 个单词，所以答案是 5。

注意这里返回的是**序列中的单词数**，不是“变化次数”。  
变化了 4 次，但路径长度是 5，这个细节不能混。

### 代码

```java
public int ladderLength(String beginWord, String endWord, List<String> wordList) {
    Set<String> wordSet = new HashSet<>(wordList);
    if (!wordSet.contains(endWord)) return 0;

    Queue<String> queue = new LinkedList<>();
    queue.offer(beginWord);

    int step = 1;

    while (!queue.isEmpty()) {
        int size = queue.size();

        for (int i = 0; i < size; i++) {
            String currWord = queue.poll();
            char[] chars = currWord.toCharArray();

            for (int j = 0; j < chars.length; j++) {
                char originalChar = chars[j];

                for (char k = 'a'; k <= 'z'; k++) {
                    if (k == originalChar) continue;

                    chars[j] = k;
                    String nextWord = new String(chars);

                    if (nextWord.equals(endWord)) {
                        return step + 1;
                    }

                    if (wordSet.contains(nextWord)) {
                        queue.offer(nextWord);
                        wordSet.remove(nextWord);
                    }
                }

                chars[j] = originalChar;
            }
        }

        step++;
    }

    return 0;
}
```

### 这段代码最关键的地方

真正关键的其实有两处。

第一处，是“生成邻居”的逻辑：

```java
for (int j = 0; j < chars.length; j++) {
    for (char k = 'a'; k <= 'z'; k++) {
        ...
    }
}
```

它表示：  
对于当前单词的每一位，都尝试替换成别的字母，从而生成所有一步可达的新状态。

第二处，是这句：

```java
if (wordSet.contains(nextWord)) {
    queue.offer(nextWord);
    wordSet.remove(nextWord);
}
```

它表示：  
只有字典里存在的单词，才是合法邻居；而一旦它准备入队，就立刻删掉，避免后续重复访问。

所以这题的核心，不是背 BFS 模板，而是要清楚：

**“相邻状态”到底怎么定义，怎么生成。**

### 为什么修改完一位字符后一定要恢复

这个细节也很重要。

我们是用同一个字符数组 `chars` 在不断尝试替换。  
如果某一位替换完之后不恢复原字符，那后面再改下一位时，得到的就不是“只改一个字母”的单词了，而是建立在已经改过一次的基础上继续改，逻辑就错了。

所以每一轮结束后都要写：

```java
chars[j] = originalChar;
```

它的作用是保证每次生成的新单词，都只和当前单词差一个字母。

### 双向 BFS 为什么会更快

这题还有一个很经典的优化思路，就是双向 BFS。

普通 BFS 是从 `beginWord` 一端往外搜。  
但如果路径比较长，搜索层数一深，状态数会膨胀得很快。

双向 BFS 的思路是：

- 一边从 `beginWord` 开始搜
- 一边从 `endWord` 开始搜
- 当两边搜索到同一个单词时，说明路径接上了

这样做的好处是，通常能明显减少搜索空间。

而且每一轮都可以选择扩展“当前规模更小”的那一边，让两端尽量保持平衡，效率会更好。

不过这属于优化思路。  
如果只是正常写题，单向 BFS 已经完全够用了，也更适合第一次理解这道题。

### 这题该怎么选写法

如果你的目标是写一个清晰、稳定、面试里好解释的版本，那我更建议直接写单向 BFS。

原因很简单：

- 逻辑直接
- 不容易写错
- 足够通过这题
- 能把“最短路径 + 状态生成”这条主线讲清楚

双向 BFS 很好，但更适合在你已经完全理解普通 BFS 后，再作为进一步优化来看。

### 复杂度分析

设：

- `N` 是 `wordList` 的大小
- `L` 是每个单词的长度

BFS 中每个合法单词最多只会入队一次。  
处理一个单词时，我们会：

- 枚举 `L` 个位置
- 每个位置尝试 26 个字母
- 生成新字符串并做哈希判断

所以时间复杂度通常写作：

- 时间复杂度：$O(N \times L \times 26)$

如果把字符串构造成本也算进去，也常写成：

- 时间复杂度：$O(N \times L^2)$

因为这里 `L` 最大只有 10，这个复杂度完全可以接受。

空间复杂度主要来自：

- `wordSet`
- 队列

所以是：

- 空间复杂度：$O(N \times L)$

### 这题真正值得记住的地方

这题最值得记住的，不只是“单词接龙用 BFS”。

更重要的是这个判断：

**只要题目是在问“状态之间最少变换几次”，并且每次变换代价相同，就优先想 BFS。**

而且状态不一定非得是数字或者坐标，也可以是字符串。  
像这题里：

- 一个单词就是一个状态
- 改一个字母就是一次状态转移
- 字典决定了哪些状态合法

很多图搜索题的关键，往往都不是 BFS 本身，而是：

**你能不能把“状态”和“相邻状态”建模清楚。**

这题就是非常典型的一道。

### 完整代码

```java
import java.util.ArrayList;
import java.util.Arrays;
import java.util.HashSet;
import java.util.LinkedList;
import java.util.List;
import java.util.Queue;
import java.util.Set;

public class Main {

    public static int ladderLength(String beginWord, String endWord, List<String> wordList) {
        Set<String> wordSet = new HashSet<>(wordList);

        if (!wordSet.contains(endWord)) {
            return 0;
        }

        Queue<String> queue = new LinkedList<>();
        queue.offer(beginWord);

        int step = 1;

        while (!queue.isEmpty()) {
            int size = queue.size();

            for (int i = 0; i < size; i++) {
                String currWord = queue.poll();
                char[] chars = currWord.toCharArray();

                for (int j = 0; j < chars.length; j++) {
                    char originalChar = chars[j];

                    for (char ch = 'a'; ch <= 'z'; ch++) {
                        if (ch == originalChar) {
                            continue;
                        }

                        chars[j] = ch;
                        String nextWord = new String(chars);

                        if (nextWord.equals(endWord)) {
                            return step + 1;
                        }

                        if (wordSet.contains(nextWord)) {
                            queue.offer(nextWord);
                            wordSet.remove(nextWord);
                        }
                    }

                    chars[j] = originalChar;
                }
            }

            step++;
        }

        return 0;
    }

    public static void main(String[] args) {
        String beginWord1 = "hit";
        String endWord1 = "cog";
        List<String> wordList1 = new ArrayList<>(Arrays.asList("hot", "dot", "dog", "lot", "log", "cog"));

        System.out.println("示例 1 输入:");
        System.out.println("beginWord: " + beginWord1);
        System.out.println("endWord: " + endWord1);
        System.out.println("wordList: " + wordList1);
        System.out.println("示例 1 输出: " + ladderLength(beginWord1, endWord1, wordList1));

        String beginWord2 = "hit";
        String endWord2 = "cog";
        List<String> wordList2 = new ArrayList<>(Arrays.asList("hot", "dot", "dog", "lot", "log"));

        System.out.println("示例 2 输入:");
        System.out.println("beginWord: " + beginWord2);
        System.out.println("endWord: " + endWord2);
        System.out.println("wordList: " + wordList2);
        System.out.println("示例 2 输出: " + ladderLength(beginWord2, endWord2, wordList2));

        String beginWord3 = "a";
        String endWord3 = "c";
        List<String> wordList3 = new ArrayList<>(Arrays.asList("a", "b", "c"));

        System.out.println("示例 3 输入:");
        System.out.println("beginWord: " + beginWord3);
        System.out.println("endWord: " + endWord3);
        System.out.println("wordList: " + wordList3);
        System.out.println("示例 3 输出: " + ladderLength(beginWord3, endWord3, wordList3));
    }
}
```
