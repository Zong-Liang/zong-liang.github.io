## 题目

**描述：**

**峰值元素**是指其值严格大于左右相邻值的元素。

给你一个整数数组 `nums`，找到峰值元素并返回其索引。数组可能包含多个峰值，在这种情况下，返回 **任何一个峰值** 所在位置即可。

你可以假设 `nums[-1] = nums[n] = -∞`。

你必须实现时间复杂度为 `O(log n)` 的算法来解决此问题。

**示例 1:**

- **输入:** `nums = [1,2,3,1]`
- **输出:** 2
- **解释:** 3 是峰值元素，你的函数应该返回其索引 2。

**示例 2:**

- **输入:** `nums = [1,2,1,3,5,6,4]`
- **输出:** 1 或 5
- **解释:** 你的函数可以返回索引 1，其峰值元素为 2；或者返回索引 5，其峰值元素为 6。

**提示:**

- `1 <= nums.length <= 1000`
- `-2^31 <= nums[i] <= 2^31 - 1`
- 对于所有有效的 `i` 都有 `nums[i] != nums[i + 1]`

## 题解

### 这题难点不在“找峰值”，而在“为什么能二分”

这题第一眼看上去其实不太像标准二分题。

因为数组并不是整体有序的，我们既不能像普通二分那样根据大小直接判断“目标一定在左边”或者“目标一定在右边”，也没有一个明确的单调区间可以直接利用。

但题目给了两个非常关键的信息：

- 相邻元素一定不相等
- 可以把两端看成负无穷，也就是 `nums[-1] = nums[n] = -∞`

这两个条件一加进来，问题就变了。

它意味着数组虽然不整体有序，但一定会呈现出“上坡”或者“下坡”的局部趋势。而峰值，恰好就藏在这些趋势变化的地方。  
所以这题的突破口不是“有序”，而是：**通过当前位置的坡度，判断哪一边一定存在峰值。**

### 为什么看一眼坡度就够了

我们在某个位置 `mid`，只需要比较 `nums[mid]` 和 `nums[mid + 1]`。

如果：

```text
nums[mid] < nums[mid + 1]
```

说明当前位置处在一段上升区间里，也就是还在“上坡”。

既然右边比当前位置更大，那峰值就不可能出现在 `mid` 左边被我们错过了。继续往右走，要么某个位置开始下降，那下降前的那个点就是峰值；要么一路升到数组末尾，那末尾因为右边是负无穷，本身也是峰值。

所以只要看到上坡，就可以放心去右边找：

```java
left = mid + 1;
```

反过来，如果：

```text
nums[mid] > nums[mid + 1]
```

说明当前位置已经开始“下坡”了。

这时候峰值一定在左侧，或者 `mid` 自己就是峰值。因为你既然已经站在一个更高的位置往下走了，那左边这一段里面一定存在一个局部最高点。

所以这时应该收缩右边界：

```java
right = mid;
```

这里要注意，不是 `right = mid - 1`，因为 `mid` 本身就有可能是峰值，不能直接丢掉。

### 这样做为什么一定不会错

这题能二分，核心就在于每次比较 `mid` 和 `mid + 1` 后，我们都能确定：

- 上坡时，右边一定有峰值
- 下坡时，左边一定有峰值

也就是说，虽然我们不知道峰值具体在哪，但我们每次都能排掉一半不必再看的区间。

这就已经满足二分的使用条件了。

很多人第一次做这题会卡在这里：  
“数组又没排好序，为什么还能二分？”

原因就在于，二分并不一定只能用于“查有序数组里的某个值”。  
只要你每次都能依据某种性质，稳定地判断答案一定落在哪一半，就可以二分。

这题用的就是这个思路。

### 用例子走一遍

比如：

```text
nums = [1,2,1,3,5,6,4]
```

初始时：

- `left = 0`
- `right = 6`

先取中点：

- `mid = 3`
- `nums[mid] = 3`
- `nums[mid + 1] = 5`

因为 `3 < 5`，说明这里在上坡，峰值一定在右边，所以更新：

- `left = 4`

现在区间变成 `[4, 6]`。

再取中点：

- `mid = 5`
- `nums[mid] = 6`
- `nums[mid + 1] = 4`

因为 `6 > 4`，说明这里开始下坡，峰值就在左边或者就是 `mid`，所以更新：

- `right = 5`

现在区间变成 `[4, 5]`。

再取中点：

- `mid = 4`
- `nums[mid] = 5`
- `nums[mid + 1] = 6`

因为 `5 < 6`，还是上坡，所以更新：

- `left = 5`

这时 `left == right == 5`，循环结束，返回下标 `5`。  
对应的值是 `6`，确实是一个峰值。

你会发现，整个过程根本不需要知道“全局最大值在哪”，只要根据局部坡度不断缩小范围就行。

### 代码

```java
public int findPeakElement(int[] nums) {
    int left = 0;
    int right = nums.length - 1;

    while (left < right) {
        int mid = left + (right - left) / 2;

        if (nums[mid] < nums[mid + 1]) {
            left = mid + 1;
        } else {
            right = mid;
        }
    }

    return left;
}
```

### 复杂度分析

时间复杂度是 $O(\log n)$。  
因为每次比较后，搜索区间都会缩小一半。

空间复杂度是 $O(1)$。  
只用了几个额外变量。

### 这题真正该记住的是什么

这题最值得记住的，不是“寻找峰值”这四个字，而是这种二分思路：

**当数组不整体有序，但局部性质足够强，能够帮助你每次稳定排除一半区间时，仍然可以使用二分。**

本题里，这个局部性质就是“坡度”。

- 如果还在上坡，峰值一定在右边
- 如果已经下坡，峰值一定在左边或当前位置

顺着这个逻辑，答案就能一步步逼出来。

所以以后再遇到“数组不完全有序，但能通过局部比较判断方向”的题，可以优先想一想，能不能也用这种方式做二分。

### 完整代码

```java
import java.util.Arrays;

public class Main {

    public static int findPeakElement(int[] nums) {
        if (nums == null || nums.length == 0) {
            return -1;
        }

        int left = 0;
        int right = nums.length - 1;

        while (left < right) {
            int mid = left + (right - left) / 2;

            if (nums[mid] < nums[mid + 1]) {
                left = mid + 1;
            } else {
                right = mid;
            }
        }

        return left;
    }

    public static void main(String[] args) {
        int[] nums1 = {1, 2, 3, 1};
        System.out.println("示例 1 输入: " + Arrays.toString(nums1));
        System.out.println("示例 1 输出: " + findPeakElement(nums1));
        System.out.println("-----------------------------");

        int[] nums2 = {1, 2, 1, 3, 5, 6, 4};
        System.out.println("示例 2 输入: " + Arrays.toString(nums2));
        System.out.println("示例 2 输出: " + findPeakElement(nums2));
        System.out.println("-----------------------------");

        int[] nums3 = {1, 2, 3, 4, 5};
        System.out.println("示例 3 输入: " + Arrays.toString(nums3));
        System.out.println("示例 3 输出: " + findPeakElement(nums3));
        System.out.println("-----------------------------");

        int[] nums4 = {5, 4, 3, 2, 1};
        System.out.println("示例 4 输入: " + Arrays.toString(nums4));
        System.out.println("示例 4 输出: " + findPeakElement(nums4));
    }
}
```
