## 题目

**描述：**

给你一个按照非递减顺序排列的整数数组 `nums`，和一个目标值 `target`。请你找出给定目标值在数组中的开始位置和结束位置。

如果数组中不存在目标值 `target`，返回 `[-1, -1]`。

你必须设计并实现时间复杂度为 `O(log n)` 的算法解决此问题。

**示例 1:**

- **输入:** `nums = [5,7,7,8,8,10]`, `target = 8`
- **输出:** `[3,4]`

**示例 2:**

- **输入:** `nums = [5,7,7,8,8,10]`, `target = 6`
- **输出:** `[-1,-1]`

**示例 3:**

- **输入:** `nums = []`, `target = 0`
- **输出:** `[-1,-1]`

**提示:**

- `0 <= nums.length <= 10^5`
- `-10^9 <= nums[i] <= 10^9`
- `nums` 是一个非递减数组
- `-10^9 <= target <= 10^9`

## 题解

### 这题表面上是在找值，实际上是在找边界

如果题目只是让我们判断 `target` 在不在数组里，普通二分查找就够了。

但这题不是只找“有没有”，而是要找出：

- 第一个等于 `target` 的位置
- 最后一个等于 `target` 的位置

也就是说，题目真正要我们找的，不是某一个 `target`，而是 `target` 在数组中的左右边界。

这也是这题和普通二分最大的区别。

普通二分一旦找到 `target`，就可以直接返回；但这题不行。因为你找到的那个位置，只能说明数组里确实有 `target`，却不能保证它就是最左边那个，或者最右边那个。

所以这题虽然还是二分，但思路得往前多走一步：  
**找到 target 还不够，还得继续逼近边界。**

### 为什么要做两次二分

很多人一开始会想，能不能一次二分直接把左右边界一起找出来。

真写的时候就会发现很别扭，因为“找最左边”和“找最右边”虽然都用二分，但收缩区间的方向并不一样。

当 `nums[mid] == target` 时：

- 如果你想找左边界，就不能停，而是应该继续往左缩
- 如果你想找右边界，也不能停，而是应该继续往右缩

这两个目标本身就是两套不同的收缩逻辑，所以最自然的写法，就是分成两次二分：

- 第一次找第一个出现的位置
- 第二次找最后一个出现的位置

这样思路最清楚，代码也最稳。

### 找左边界时，关键在“找到后别停”

先说找左边界。

二分过程中，如果：

- `nums[mid] < target`，说明目标一定在右边
- `nums[mid] > target`，说明目标一定在左边

这些和普通二分没有区别。

真正关键的是 `nums[mid] == target` 的时候。

这时候不能直接返回，因为你只能确定 `mid` 是一个合法位置，但左边说不定还有更早出现的 `target`。

所以应该先把当前答案记下来，然后继续往左边找：

```java
res = mid;
right = mid - 1;
```

这样做的意思是：

“我已经知道这里有一个 `target` 了，但我怀疑左边还有更靠前的，所以继续缩右边界。”

最后循环结束时，记录下来的那个位置，就是最左边界。

### 找右边界时，思路完全对称

找右边界时，逻辑和上面几乎一样，只是方向反过来了。

当 `nums[mid] == target` 时：

- 先把当前位置记下来
- 然后继续往右边找

也就是：

```java
res = mid;
left = mid + 1;
```

因为你现在不是想确认“有没有”，而是想确认“后面还有没有更靠右的”。

所以整道题最核心的地方其实就一句话：

**遇到 target 时不要停，而是把它当成一个暂时答案，再继续朝边界方向逼近。**

### 用例子走一遍

来看这个例子：

```text
nums = [5,7,7,8,8,10], target = 8
```

先找左边界。

初始时：

- `left = 0`
- `right = 5`

第一次取中点：

- `mid = 2`
- `nums[mid] = 7`

因为 `7 < 8`，说明目标在右边，于是更新：

- `left = 3`

现在区间变成 `[3, 5]`。

再取中点：

- `mid = 4`
- `nums[mid] = 8`

找到了目标，但不能停。先记下：

- `res = 4`

因为现在找的是左边界，所以继续往左缩：

- `right = 3`

现在区间变成 `[3, 3]`。

再取中点：

- `mid = 3`
- `nums[mid] = 8`

还是等于 `target`，继续更新：

- `res = 3`
- `right = 2`

这时循环结束，最终左边界就是 `3`。

右边界同理，只不过找到之后继续往右逼近，最后会得到 `4`。

所以答案就是：

```text
[3, 4]
```

### 代码

```java
public int[] searchRange(int[] nums, int target) {
    int first = findBound(nums, target, true);
    int last = findBound(nums, target, false);
    return new int[]{first, last};
}

private int findBound(int[] nums, int target, boolean isFirst) {
    int left = 0;
    int right = nums.length - 1;
    int res = -1;

    while (left <= right) {
        int mid = left + (right - left) / 2;

        if (nums[mid] == target) {
            res = mid;
            if (isFirst) {
                right = mid - 1;
            } else {
                left = mid + 1;
            }
        } else if (nums[mid] < target) {
            left = mid + 1;
        } else {
            right = mid - 1;
        }
    }

    return res;
}
```

### 复杂度分析

时间复杂度是 $O(\log n)$。  
虽然做了两次二分，但每次都是对数级别，所以总体仍然是 $O(\log n)$。

空间复杂度是 $O(1)$。  
只用了几个额外变量。

### 这题真正要记住的地方

这题最值得记住的，不是“两次二分”这四个字本身，而是这个思考方式：

**当题目要求的不是“某个值在不在”，而是“某类值的最左位置”或者“最右位置”时，普通二分找到答案后往往还不能停。**

你要做的不是立刻返回，而是：

- 先把当前答案记下来
- 再继续往边界方向缩

本题找的是 `target` 的起点和终点，背后其实就是“找左边界”和“找右边界”这两个经典问题。

以后再遇到有序数组里找边界范围、找第一个满足条件的位置、找最后一个满足条件的位置这类题，都可以往这个方向想。

### 完整代码

```java
import java.util.Arrays;

public class Main {

    public static int[] searchRange(int[] nums, int target) {
        int first = findBound(nums, target, true);
        int last = findBound(nums, target, false);
        return new int[]{first, last};
    }

    private static int findBound(int[] nums, int target, boolean isFirst) {
        int left = 0;
        int right = nums.length - 1;
        int res = -1;

        while (left <= right) {
            int mid = left + (right - left) / 2;

            if (nums[mid] == target) {
                res = mid;
                if (isFirst) {
                    right = mid - 1;
                } else {
                    left = mid + 1;
                }
            } else if (nums[mid] < target) {
                left = mid + 1;
            } else {
                right = mid - 1;
            }
        }

        return res;
    }

    public static void main(String[] args) {
        int[] nums1 = {5, 7, 7, 8, 8, 10};
        int target1 = 8;
        System.out.println("示例 1 输入: nums = " + Arrays.toString(nums1) + ", target = " + target1);
        System.out.println("示例 1 输出: " + Arrays.toString(searchRange(nums1, target1)));
        System.out.println("-----------------------------");

        int[] nums2 = {5, 7, 7, 8, 8, 10};
        int target2 = 6;
        System.out.println("示例 2 输入: nums = " + Arrays.toString(nums2) + ", target = " + target2);
        System.out.println("示例 2 输出: " + Arrays.toString(searchRange(nums2, target2)));
        System.out.println("-----------------------------");

        int[] nums3 = {};
        int target3 = 0;
        System.out.println("示例 3 输入: nums = " + Arrays.toString(nums3) + ", target = " + target3);
        System.out.println("示例 3 输出: " + Arrays.toString(searchRange(nums3, target3)));
        System.out.println("-----------------------------");

        int[] nums4 = {1};
        int target4 = 1;
        System.out.println("额外示例 输入: nums = " + Arrays.toString(nums4) + ", target = " + target4);
        System.out.println("额外示例 输出: " + Arrays.toString(searchRange(nums4, target4)));
    }
}
```
