## 题目

**描述：**

给你一个满足下列两条属性的 `m x n` 整数矩阵：

- 每行中的整数从左到右按非严格递增顺序排列。
- 每行的第一个整数大于前一行的最后一个整数。

给你一个整数 `target`，如果 `target` 在矩阵中，返回 `true`；否则，返回 `false`。

**示例 1:**

![示例1图片](https://assets.leetcode.com/uploads/2020/10/05/mat.jpg)

- **输入:** `matrix = [[1,3,5,7],[10,11,16,20],[23,30,34,60]]`, `target = 3`
- **输出:** `true`

**示例 2:**

![示例2图片](https://assets.leetcode-cn.com/aliyun-lc-upload/uploads/2020/11/25/mat2.jpg)

- **输入:** `matrix = [[1,3,5,7],[10,11,16,20],[23,30,34,60]]`, `target = 13`
- **输出:** `false`

**提示:**

- `m == matrix.length`
- `n == matrix[i].length`
- `1 <= m, n <= 100`
- `-10^4 <= matrix[i][j], target <= 10^4`

## 题解

### 先看这道题为什么能用二分

二维矩阵一上来很容易让人想到“按行查”或者“先确定行，再在这一行里查”。这些思路不算错，但这题其实还有一个更直接的切入点。

题目给了两个很关键的条件：

- 每一行从左到右递增
- 下一行的第一个数大于上一行的最后一个数

这两个条件放在一起，意味着整个矩阵虽然看起来是二维的，但如果按行依次拼起来，它本质上就是一个整体有序的数组。

比如：

```text
[
  [1, 3, 5, 7],
  [10, 11, 16, 20],
  [23, 30, 34, 60]
]
```

实际上就可以看成：

```text
[1, 3, 5, 7, 10, 11, 16, 20, 23, 30, 34, 60]
```

一旦想到这里，这题就已经从“二维矩阵查找”变成了“有序数组查找”，后面自然就是二分查找。

### 关键不在展开数组，而在映射下标

虽然可以把它理解成一个一维有序数组，但代码里并不需要真的把矩阵拍平成新数组。

我们只需要在逻辑上把它当成长度为 `m * n` 的有序数组，然后在二分过程中，把一维下标映射回二维坐标就行了。

假设矩阵有 `m` 行、`n` 列，那么：

- 一维下标 `idx` 对应的行号是 `idx / n`
- 一维下标 `idx` 对应的列号是 `idx % n`

也就是说，如果二分过程中算出了中点 `mid`，那么它在矩阵中的值就是：

```java
matrix[mid / n][mid % n]
```

这个映射关系就是这题的核心。

### 思路顺下来其实很简单

做法和普通二分几乎一样。

先把整个矩阵看成下标范围是 `0` 到 `m * n - 1` 的有序数组：

- `left = 0`
- `right = m * n - 1`

每次取中点 `mid`，再通过映射拿到矩阵中的实际值：

- `row = mid / n`
- `col = mid % n`
- `midVal = matrix[row][col]`

然后和 `target` 比较：

- 如果相等，直接返回 `true`
- 如果 `midVal < target`，说明目标值只可能在右半边，更新 `left = mid + 1`
- 如果 `midVal > target`，说明目标值只可能在左半边，更新 `right = mid - 1`

最后如果区间缩完都没找到，就返回 `false`。

### 用示例感受一下

还是拿示例 1 来看：

```text
matrix = [[1,3,5,7],[10,11,16,20],[23,30,34,60]], target = 3
```

这个矩阵有 `3` 行、`4` 列，所以我们逻辑上把它视为一个长度为 `12` 的有序数组，下标范围是 `0` 到 `11`。

初始时：

- `left = 0`
- `right = 11`

第一次取中点：

- `mid = 5`
- 对应坐标是 `row = 5 / 4 = 1`，`col = 5 % 4 = 1`
- 所以取到的值是 `matrix[1][1] = 11`

因为 `11 > 3`，说明目标值只可能在左半边，于是更新：

- `right = 4`

再取中点：

- `mid = 2`
- 对应坐标是 `row = 2 / 4 = 0`，`col = 2 % 4 = 2`
- 取到的值是 `matrix[0][2] = 5`

因为 `5 > 3`，继续往左缩：

- `right = 1`

再取中点：

- `mid = 0`
- 对应值是 `matrix[0][0] = 1`

因为 `1 < 3`，往右找：

- `left = 1`

再取中点：

- `mid = 1`
- 对应值是 `matrix[0][1] = 3`

正好找到，返回 `true`。

这样走一遍之后，这种“把二维坐标映射回去”的写法就不会觉得绕了。

### 代码

```java
public boolean searchMatrix(int[][] matrix, int target) {
    int m = matrix.length;
    int n = matrix[0].length;

    int left = 0;
    int right = m * n - 1;

    while (left <= right) {
        int mid = left + (right - left) / 2;
        int midVal = matrix[mid / n][mid % n];

        if (midVal == target) {
            return true;
        } else if (midVal < target) {
            left = mid + 1;
        } else {
            right = mid - 1;
        }
    }

    return false;
}
```

### 复杂度分析

时间复杂度是 $O(\log(mn))$。  
因为本质上是在一个长度为 `m * n` 的有序数组上做二分。

空间复杂度是 $O(1)$。  
没有额外开数组，只做了下标映射。

### 这题真正值得记住的点

这题最值得记住的，不是“二维矩阵也能二分”这句话本身，而是判断依据。

不是所有矩阵都能这样做，关键要看它能不能在逻辑上被看成一个整体有序的一维数组。

本题之所以可以，是因为它不仅每行有序，而且下一行的开头还严格大于上一行的结尾。这个条件保证了“按行展开后仍然整体有序”。

所以以后再遇到二维查找问题，不要急着被“二维”这两个字带跑，先看它有没有可能被转成你更熟悉的一维问题。很多时候，真正的突破口就在这里。

### 完整代码

```java
import java.util.Arrays;

public class Main {

    public static boolean searchMatrix(int[][] matrix, int target) {
        if (matrix == null || matrix.length == 0) {
            return false;
        }

        int m = matrix.length;
        int n = matrix[0].length;

        int left = 0;
        int right = m * n - 1;

        while (left <= right) {
            int mid = left + (right - left) / 2;

            int row = mid / n;
            int col = mid % n;
            int midVal = matrix[row][col];

            if (midVal == target) {
                return true;
            } else if (midVal < target) {
                left = mid + 1;
            } else {
                right = mid - 1;
            }
        }

        return false;
    }

    public static void main(String[] args) {
        int[][] matrix1 = {
            {1, 3, 5, 7},
            {10, 11, 16, 20},
            {23, 30, 34, 60}
        };
        int target1 = 3;

        System.out.println("示例 1 输入:");
        System.out.println("matrix = " + Arrays.deepToString(matrix1));
        System.out.println("target = " + target1);
        System.out.println("示例 1 输出: " + searchMatrix(matrix1, target1));
        System.out.println("-----------------------------");

        int[][] matrix2 = {
            {1, 3, 5, 7},
            {10, 11, 16, 20},
            {23, 30, 34, 60}
        };
        int target2 = 13;

        System.out.println("示例 2 输入:");
        System.out.println("matrix = " + Arrays.deepToString(matrix2));
        System.out.println("target = " + target2);
        System.out.println("示例 2 输出: " + searchMatrix(matrix2, target2));
    }
}
```
