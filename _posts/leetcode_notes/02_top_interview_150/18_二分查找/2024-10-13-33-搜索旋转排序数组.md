## 题目

**描述：**

整数数组 `nums` 按升序排列，数组中的值 **互不相同**。

在传递给函数之前，`nums` 在预先未知的某个下标 `k`（`0 <= k < nums.length`）上进行了 **向左旋转**，使数组变为 `[nums[k], nums[k+1], ..., nums[n-1], nums[0], nums[1], ..., nums[k-1]]`（下标 **从 0 开始** 计数）。例如，`[0,1,2,4,5,6,7]` 下标 `3` 上向左旋转后可能变为 `[4,5,6,7,0,1,2]`。

给你 **旋转后** 的数组 `nums` 和一个整数 `target`，如果 `nums` 中存在这个目标值 `target`，则返回它的下标，否则返回 `-1`。

你必须设计一个时间复杂度为 `O(log n)` 的算法解决此问题。

**示例 1:**

- **输入:** `nums = [4,5,6,7,0,1,2]`, `target = 0`
- **输出:** 4

**示例 2:**

- **输入:** `nums = [4,5,6,7,0,1,2]`, `target = 3`
- **输出:** -1

**示例 3:**

- **输入:** `nums = [1]`, `target = 0`
- **输出:** -1

**提示:**

- `1 <= nums.length <= 5000`
- `-10^4 <= nums[i] <= 10^4`
- `nums` 中的每个值都 **独一无二**
- 题目数据保证 `nums` 在预先未知的某个下标上进行了旋转
- `-10^4 <= target <= 10^4`

## 题解

### 这题难的不是“旋转”，而是“旋转以后怎么继续二分”

如果数组是普通升序数组，那这题就没有任何悬念，直接二分查找就行。

麻烦在于，这个数组先升序，再被旋转过一次。旋转之后，它已经不是整体有序了，所以我们不能再像普通二分那样，单纯根据 `nums[mid]` 和 `target` 的大小关系来判断往左还是往右。

但题目又明确要求时间复杂度是 `O(log n)`，这就说明虽然它不再“整体有序”，但一定还保留了某种适合二分的结构。

这道题最关键的地方就在这里：

**虽然整个数组不再有序，但每次从中间切开，至少有一半一定还是有序的。**

只要把这一点想明白，后面的思路就顺了。

### 为什么每次至少有一半是有序的

先看一个例子：

```text
[4,5,6,7,0,1,2]
```

这个数组是把原来的升序数组旋转得到的。  
你从中间切一刀，不管切在哪里，总会出现一种情况：

- 左半边有序
- 或者右半边有序

比如这里如果取中间位置是 `7`，那么：

- 左边 `[4,5,6,7]` 是有序的
- 右边 `[7,0,1,2]` 不是完整有序段

换个位置取中点，也可能变成右半边有序。  
总之，不会两边都乱掉，因为它本质上只是把一个升序数组从某个位置断开再拼起来而已。

所以这题虽然表面上是“旋转数组”，但二分时真正要做的不是直接找目标，而是先判断：

**当前这一次，哪一半是有序的？**

只要知道哪边有序，就能继续判断目标值应不应该落在这一边。

### 先判断哪半边有序，再判断目标在不在里面

我们在区间 `[left, right]` 内取中点 `mid`。

如果 `nums[mid] == target`，那当然直接返回。

如果不相等，就看下面这个判断：

```java
if (nums[left] <= nums[mid])
```

如果这个条件成立，说明左半边 `[left, mid]` 是有序的。

既然左半边有序，那就可以继续判断 `target` 有没有落在这个有序区间里：

- 如果 `nums[left] <= target < nums[mid]`，说明目标就在左边，缩右边界
- 否则，目标只能在右边，缩左边界

反过来，如果 `nums[left] > nums[mid]`，说明左边这一段跨过了旋转点，不是完整有序的，那右半边 `[mid, right]` 就一定是有序的。

这时再判断：

- 如果 `nums[mid] < target <= nums[right]`，说明目标在右边
- 否则，目标在左边

整道题其实就是不断重复这个过程。

### 用例子走一遍

还是看这个例子：

```text
nums = [4,5,6,7,0,1,2], target = 0
```

初始时：

- `left = 0`
- `right = 6`

先取中点：

- `mid = 3`
- `nums[mid] = 7`

显然 `7 != 0`，继续判断哪边有序。

此时：

```text
nums[left] = 4
nums[mid] = 7
```

因为 `4 <= 7`，说明左半边 `[4,5,6,7]` 是有序的。

再看 `target = 0` 是否落在这个区间内。  
很明显不在，因为 `0` 不满足 `4 <= 0 < 7`。

那就说明目标只能去右边找，于是更新：

- `left = mid + 1 = 4`

现在区间变成 `[4, 6]`，也就是：

```text
[0,1,2]
```

再取中点：

- `mid = 5`
- `nums[mid] = 1`

还是不等于 `0`。

继续判断：

```text
nums[left] = 0
nums[mid] = 1
```

因为 `0 <= 1`，所以左半边 `[0,1]` 有序。

再看 `target = 0` 是否在这段范围内。  
满足 `0 <= 0 < 1`，所以目标在左边，于是更新：

- `right = mid - 1 = 4`

这时 `left == right == 4`，再比较一次就能找到答案，返回 `4`。

整个过程的关键不是“旋转数组怎么恢复”，而是每一步都先找出那一半有序区间，再利用这个区间做判断。

### 代码

```java
public int search(int[] nums, int target) {
    int left = 0;
    int right = nums.length - 1;

    while (left <= right) {
        int mid = left + (right - left) / 2;

        if (nums[mid] == target) {
            return mid;
        }

        if (nums[left] <= nums[mid]) {
            if (nums[left] <= target && target < nums[mid]) {
                right = mid - 1;
            } else {
                left = mid + 1;
            }
        } else {
            if (nums[mid] < target && target <= nums[right]) {
                left = mid + 1;
            } else {
                right = mid - 1;
            }
        }
    }

    return -1;
}
```

### 复杂度分析

时间复杂度是 $O(\log n)$。  
因为每次都能排除掉一半区间。

空间复杂度是 $O(1)$。  
只用了几个额外变量。

### 这题真正要记住的地方

这题最值得记住的，不是“旋转数组怎么查找”这句话本身，而是这种判断方式：

**当数组整体不再有序时，先不要急着放弃二分，先看能不能在当前区间里找出一半稳定有序的部分。**

本题就是典型例子。

虽然整个数组被旋转了，但它并没有完全失去结构。每一次取中点后，至少都有一半是有序的。只要抓住这半边，再判断目标值是否落在这个有序区间里，就能继续像二分一样缩小范围。

以后再遇到这种“整体乱了一点，但局部还有规则”的题，也可以往这个方向去想。

### 完整代码

```java
import java.util.Arrays;

public class Main {

    public static int search(int[] nums, int target) {
        if (nums == null || nums.length == 0) {
            return -1;
        }

        int left = 0;
        int right = nums.length - 1;

        while (left <= right) {
            int mid = left + (right - left) / 2;

            if (nums[mid] == target) {
                return mid;
            }

            if (nums[left] <= nums[mid]) {
                if (nums[left] <= target && target < nums[mid]) {
                    right = mid - 1;
                } else {
                    left = mid + 1;
                }
            } else {
                if (nums[mid] < target && target <= nums[right]) {
                    left = mid + 1;
                } else {
                    right = mid - 1;
                }
            }
        }

        return -1;
    }

    public static void main(String[] args) {
        int[] nums1 = {4, 5, 6, 7, 0, 1, 2};
        int target1 = 0;
        System.out.println("示例 1 输入: nums = " + Arrays.toString(nums1) + ", target = " + target1);
        System.out.println("示例 1 输出: " + search(nums1, target1));
        System.out.println("-----------------------------");

        int[] nums2 = {4, 5, 6, 7, 0, 1, 2};
        int target2 = 3;
        System.out.println("示例 2 输入: nums = " + Arrays.toString(nums2) + ", target = " + target2);
        System.out.println("示例 2 输出: " + search(nums2, target2));
        System.out.println("-----------------------------");

        int[] nums3 = {1};
        int target3 = 0;
        System.out.println("示例 3 输入: nums = " + Arrays.toString(nums3) + ", target = " + target3);
        System.out.println("示例 3 输出: " + search(nums3, target3));
        System.out.println("-----------------------------");

        int[] nums4 = {3, 1};
        int target4 = 1;
        System.out.println("额外示例 输入: nums = " + Arrays.toString(nums4) + ", target = " + target4);
        System.out.println("额外示例 输出: " + search(nums4, target4));
    }
}
```
