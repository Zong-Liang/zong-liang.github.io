## 题目

**描述：**

给定两个大小分别为 `m` 和 `n` 的正序（从小到大）数组 `nums1` 和 `nums2`。请你找出并返回这两个正序数组的 **中位数**。

算法的时间复杂度应该为 `O(log (m+n))`。

**示例 1:**

- **输入:** `nums1 = [1,3]`, `nums2 = [2]`
- **输出:** `2.00000`
- **解释:** 合并数组 = `[1,2,3]`，中位数 2

**示例 2:**

- **输入:** `nums1 = [1,2]`, `nums2 = [3,4]`
- **输出:** `2.50000`
- **解释:** 合并数组 = `[1,2,3,4]`，中位数 `(2 + 3) / 2 = 2.5`

**提示:**

- `nums1.length == m`
- `nums2.length == n`
- `0 <= m <= 1000`
- `0 <= n <= 1000`
- `1 <= m + n <= 2000`
- `-10^6 <= nums1[i], nums2[i] <= 10^6`

## 题解

### 这题最难的不是求中位数，而是别去真的合并

看到“两个有序数组的中位数”，最自然的想法通常是先把两个数组合并，再直接取中位数。

这个思路当然能做出来，但时间复杂度是 $O(m+n)$，而题目明确要求的是 $O(\log(m+n))$。  
这就说明这题真正考的，不是怎么把两个数组拼起来，而是：

**能不能在不合并数组的前提下，直接判断中位数落在哪。**

这也是这题和普通“双指针合并有序数组”最大的区别。

### 中位数问题，本质上是在找一条合适的分割线

先别急着想“中位数具体是多少”，先想一个更关键的问题：

如果把两个有序数组一起看，什么情况下，我们就能立刻确定中位数？

答案是：  
**当我们能把两个数组分别切一刀，使得左边的所有数都不大于右边的所有数，并且左边总数和右边总数平衡时，中位数就出来了。**

举个例子。

假设两个数组是：

```text
nums1 = [1,3]
nums2 = [2]
```

如果我们把它们切成这样：

```text
nums1: [1 | 3]
nums2: [2 | ]
```

那么左边总共有两个数：`1, 2`  
右边总共有一个数：`3`

这时：

- 左边元素总数正好满足要求
- 左边最大值不大于右边最小值

那么中位数其实就是左边最大值，也就是 `2`。

所以这题真正要找的，不是某个具体位置的数，而是：

**找到一个合法切分，让左半部分和右半部分刚好满足中位数定义。**

### 为什么只在较短的数组上二分

设 `nums1` 长度为 `m`，`nums2` 长度为 `n`。

我们假设在 `nums1` 里切在位置 `i`，那么为了保证左边总元素数量正确，在 `nums2` 里切的位置 `j` 就自动确定了：

```text
i + j = (m + n + 1) / 2
```

也就是说，一旦 `i` 定了，`j` 根本不用自己再找。

所以整道题其实只需要决定一件事：  
**`nums1` 应该切在哪。**

既然只需要决定一个数组的切分位置，那自然应该在较短的数组上做二分，这样更高效，也更容易控制边界。

这也是为什么代码一开始通常会先保证：

```java
nums1.length <= nums2.length
```

这一步不是细节，而是整个做法成立的前提之一。

### 一刀切完之后，只需要看四个数

假设：

- `nums1` 切在 `i`
- `nums2` 切在 `j`

那么切分线两边最关键的，只有四个数：

- `nums1LeftMax = nums1[i - 1]`
- `nums1RightMin = nums1[i]`
- `nums2LeftMax = nums2[j - 1]`
- `nums2RightMin = nums2[j]`

我们真正关心的，其实不是左右所有元素，而只是这四个边界值。

因为只要满足：

```text
nums1LeftMax <= nums2RightMin
nums2LeftMax <= nums1RightMin
```

就说明左边所有元素都不大于右边所有元素，这个切分就是合法的。

这两个条件缺一个都不行：

- 如果 `nums1LeftMax > nums2RightMin`，说明 `nums1` 左边切得太多了，左边混进了过大的数，切分线应该左移
- 如果 `nums2LeftMax > nums1RightMin`，说明 `nums1` 切得还不够，切分线应该右移

这就是整道题的二分依据。

### 为什么边界用正负无穷会很顺

这题如果硬写各种边界判断，会特别碎。

因为有可能：

- `i = 0`，说明 `nums1` 左边没有元素
- `i = m`，说明 `nums1` 右边没有元素
- `j = 0`
- `j = n`

这些情况如果全都手写分支，代码会很乱。

更自然的处理方式是直接把越界值看成无穷：

- 左边没有元素，就把左侧最大值看成负无穷
- 右边没有元素，就把右侧最小值看成正无穷

这样上面的判断公式就完全不用改，逻辑会统一很多。

### 用例子走一遍

来看示例 2：

```text
nums1 = [1,2]
nums2 = [3,4]
```

总长度是 `4`，所以左半部分需要有：

```text
(2 + 2 + 1) / 2 = 2
```

个元素。

现在我们在 `nums1` 上二分。

#### 第一次尝试

令：

- `i = 1`
- `j = 2 - 1 = 1`

此时四个边界值分别是：

- `nums1LeftMax = 1`
- `nums1RightMin = 2`
- `nums2LeftMax = 3`
- `nums2RightMin = 4`

检查合法性：

```text
nums1LeftMax <= nums2RightMin   -> 1 <= 4，成立
nums2LeftMax <= nums1RightMin   -> 3 <= 2，不成立
```

说明 `nums1` 切得太靠左了，右边太小，撑不住 `nums2` 左边的 `3`。  
所以要把 `i` 往右移。

#### 第二次尝试

令：

- `i = 2`
- `j = 0`

这时四个边界值变成：

- `nums1LeftMax = 2`
- `nums1RightMin = +∞`
- `nums2LeftMax = -∞`
- `nums2RightMin = 3`

再次检查：

```text
2 <= 3，成立
-∞ <= +∞，成立
```

说明这次切分合法。

总长度是偶数，所以中位数就是：

```text
(左半部分最大值 + 右半部分最小值) / 2
= (max(2, -∞) + min(+∞, 3)) / 2
= (2 + 3) / 2
= 2.5
```

答案就出来了。

### 代码

```java
public double findMedianSortedArrays(int[] nums1, int[] nums2) {
    if (nums1.length > nums2.length) {
        return findMedianSortedArrays(nums2, nums1);
    }

    int m = nums1.length;
    int n = nums2.length;
    int totalLeft = (m + n + 1) / 2;

    int left = 0;
    int right = m;

    while (left <= right) {
        int i = left + (right - left) / 2;
        int j = totalLeft - i;

        int nums1LeftMax = (i == 0) ? Integer.MIN_VALUE : nums1[i - 1];
        int nums1RightMin = (i == m) ? Integer.MAX_VALUE : nums1[i];
        int nums2LeftMax = (j == 0) ? Integer.MIN_VALUE : nums2[j - 1];
        int nums2RightMin = (j == n) ? Integer.MAX_VALUE : nums2[j];

        if (nums1LeftMax <= nums2RightMin && nums2LeftMax <= nums1RightMin) {
            if ((m + n) % 2 == 1) {
                return Math.max(nums1LeftMax, nums2LeftMax);
            } else {
                return (Math.max(nums1LeftMax, nums2LeftMax) + Math.min(nums1RightMin, nums2RightMin)) / 2.0;
            }
        } else if (nums1LeftMax > nums2RightMin) {
            right = i - 1;
        } else {
            left = i + 1;
        }
    }

    return 0.0;
}
```

### 复杂度分析

时间复杂度是 $O(\log(\min(m,n)))$。  
因为我们只在较短的数组上做二分。

空间复杂度是 $O(1)$。  
只用了常数级额外变量。

### 这题真正要记住的地方

这题最值得记住的，不是“两个数组中位数”这个题型本身，而是这种转换方式：

**原问题看起来是在求值，实际上可以先转成“找一个合法切分”。**

一旦切分合法：

- 奇数长度时，中位数就是左半部分最大值
- 偶数长度时，中位数就是左半部分最大值和右半部分最小值的平均值

所以整道题最核心的不是“怎么算中位数”，而是：

1. 固定左半部分元素个数
2. 在较短数组上二分切分位置
3. 用四个边界值判断切分是否合法

这个思路比较绕，但一旦顺下来，这题其实就是一个非常标准的“二分答案位置”问题。

public class Main {

    public static double findMedianSortedArrays(int[] nums1, int[] nums2) {
        if (nums1.length > nums2.length) {
            return findMedianSortedArrays(nums2, nums1);

        int m = nums1.length;
        int n = nums2.length;

        int left = 0;
        int right = m;

        while (left <= right) {
            int i = left + (right - left) / 2;
            int j = totalLeft - i;

            int nums1LeftMax = (i == 0) ? Integer.MIN_VALUE : nums1[i - 1];
            int nums1RightMin = (i == m) ? Integer.MAX_VALUE : nums1[i];
            int nums2LeftMax = (j == 0) ? Integer.MIN_VALUE : nums2[j - 1];
            int nums2RightMin = (j == n) ? Integer.MAX_VALUE : nums2[j];

            if (nums1LeftMax <= nums2RightMin && nums2LeftMax <= nums1RightMin) {

            } else {

            }
        }

    }

        int[] nums1_ex2 = {1, 2};
        int[] nums2_ex2 = {3, 4};
