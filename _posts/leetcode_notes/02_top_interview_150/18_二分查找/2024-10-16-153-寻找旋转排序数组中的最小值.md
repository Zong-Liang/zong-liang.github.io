## 题目

**描述：**

已知一个长度为 `n` 的数组，预先按照升序排列，经由 `1` 到 `n` 次 **旋转** 后，得到输入数组。例如，原数组 `nums = [0,1,2,4,5,6,7]` 在变化后可能得到：

- 若旋转 `4` 次，则可以得到 `[4,5,6,7,0,1,2]`
- 若旋转 `7` 次，则可以得到 `[0,1,2,4,5,6,7]`

**注意**，数组 `[a[0], a[1], a[2], ..., a[n-1]]` **旋转一次** 的结果为数组 `[a[n-1], a[0], a[1], a[2], ..., a[n-2]]`。

给你一个元素值 **互不相同** 的数组 `nums`，它原来是一个升序排列的数组，并按上述情形进行了多次旋转。请你找出并返回数组中的 **最小元素**。

你必须设计一个时间复杂度为 `O(log n)` 的算法解决此问题。

**示例 1:**

- **输入:** `nums = [3,4,5,1,2]`
- **输出:** 1
- **解释:** 原数组为 `[1,2,3,4,5]` ，旋转 `3` 次得到输入数组。

**示例 2:**

- **输入:** `nums = [4,5,6,7,0,1,2]`
- **输出:** 0
- **解释:** 原数组为 `[0,1,2,4,5,6,7]` ，旋转 `4` 次得到输入数组。

**示例 3:**

- **输入:** `nums = [11,13,15,17]`
- **输出:** 11
- **解释:** 原数组为 `[11,13,15,17]` ，旋转 `4` 次得到输入数组。

**提示:**

- `n == nums.length`
- `1 <= n <= 5000`
- `-5000 <= nums[i] <= 5000`
- `nums` 中的所有整数 **互不相同**
- `nums` 原来是一个升序排序的数组，并进行了 `1` 至 `n` 次旋转

## 题解

### 这题真正要找的，其实是旋转断点

如果数组没有旋转，那最小值显然就是第一个元素。  
但一旦旋转之后，数组就会从某个位置断开，变成两段分别有序的部分。

比如：

```text
[4,5,6,7,0,1,2]
```

你可以把它看成：

- 前半段 `[4,5,6,7]`
- 后半段 `[0,1,2]`

真正的最小值，其实就是这两段交界处后面的第一个元素，也就是旋转断点的位置。

所以这题表面上是在“找最小值”，实际上是在找：

**那条从大数突然掉到小数的位置，最小值就在它后面。**

题目又要求时间复杂度必须是 `O(log n)`，那就说明不能顺着扫，只能想办法继续用二分。

### 虽然整体不有序，但可以靠二分不断缩小范围

旋转之后，数组不再是整体升序的，所以我们不能像普通二分那样直接判断“目标值在左边还是右边”。

但这题和“搜索旋转排序数组”不一样，它不需要找某个具体值，只需要找最小值。  
这就简单了很多，因为我们只需要判断：

**最小值到底在 `mid` 左边，还是在 `mid` 右边。**

而判断的方法，就是拿 `nums[mid]` 和 `nums[right]` 比较。

### 为什么要和 `nums[right]` 比

这是这题最关键的地方。

我们每次取中点 `mid`，然后比较：

```java
nums[mid] 和 nums[right]
```

如果：

```text
nums[mid] > nums[right]
```

说明 `mid` 落在左半段，也就是那一段比较大的有序区间里。

比如：

```text
[4,5,6,7,0,1,2]
         ^
        mid
             ^
           right
```

此时 `nums[mid]` 明显比 `nums[right]` 大，说明最小值肯定还没到 `mid`，它一定在右边那段较小的区间里。

所以这时可以直接更新：

```java
left = mid + 1;
```

反过来，如果：

```text
nums[mid] < nums[right]
```

说明从 `mid` 到 `right` 这一段是正常升序的，而且 `mid` 已经落在右半段那段较小的区间里了。

这时最小值要么就是 `mid`，要么在 `mid` 左边，所以应该更新：

```java
right = mid;
```

这里一定不能写成 `right = mid - 1`，因为 `mid` 本身就有可能是最小值，直接减一就把答案丢掉了。

### 为什么这样判断一定对

你可以把旋转数组想成两段：

- 左边一段，值整体偏大
- 右边一段，值整体偏小
- 最小值就在右边那段的开头

而 `right` 永远落在当前区间的最右边。  
只要当前区间里还包含最小值，那么 `nums[right]` 就能起到一个很稳定的参照作用。

于是：

- `nums[mid] > nums[right]`，说明 `mid` 在左边大数那段，最小值在右边
- `nums[mid] < nums[right]`，说明 `mid` 已经在右边小数那段，最小值在左边或就是 `mid`

整个二分就是靠这个关系不断缩小范围。

### 用例子走一遍

来看示例 1：

```text
nums = [3,4,5,1,2]
```

初始时：

- `left = 0`
- `right = 4`

第一次取中点：

- `mid = 2`
- `nums[mid] = 5`
- `nums[right] = 2`

因为 `5 > 2`，说明 `mid` 在左边那段较大的区间里，最小值一定在右边，所以更新：

- `left = mid + 1 = 3`

现在区间变成 `[3, 4]`，也就是：

```text
[1,2]
```

再取中点：

- `mid = 3`
- `nums[mid] = 1`
- `nums[right] = 2`

因为 `1 < 2`，说明 `mid` 已经落在右边那段较小区间里了，最小值可能就是它，所以更新：

- `right = mid = 3`

这时 `left == right == 3`，循环结束。  
答案就是 `nums[3] = 1`。

整个过程里，我们从来没有真的去“找断点”，而是通过和 `right` 比较，一步步把断点夹出来了。

### 代码

```java
public int findMin(int[] nums) {
    int left = 0;
    int right = nums.length - 1;

    while (left < right) {
        int mid = left + (right - left) / 2;

        if (nums[mid] > nums[right]) {
            left = mid + 1;
        } else {
            right = mid;
        }
    }

    return nums[left];
}
```

### 复杂度分析

时间复杂度是 $O(\log n)$。  
因为每次比较后，搜索区间都会缩小一半。

空间复杂度是 $O(1)$。  
只用了几个额外变量。

### 这题真正值得记住的点

这题最值得记住的，不是“旋转数组找最小值”这个题型本身，而是这种判断方式：

**当数组被旋转后，虽然整体不再有序，但仍然可以借助边界值，判断中点落在哪一段。**

本题里，`right` 就是那个非常关键的参照物。

- 比 `right` 大，说明在左边大数区间
- 比 `right` 小，说明在右边小数区间

一旦这个关系想顺，这题其实就是一个很干净的二分模板题。

以后再遇到旋转数组相关的问题，也可以先想：  
有没有哪个边界值，可以作为判断中点所在区间的参照。

### 完整代码

```java
import java.util.Arrays;

public class Main {

    public static int findMin(int[] nums) {
        if (nums == null || nums.length == 0) {
            throw new IllegalArgumentException("Array must not be empty");
        }

        int left = 0;
        int right = nums.length - 1;

        while (left < right) {
            int mid = left + (right - left) / 2;

            if (nums[mid] > nums[right]) {
                left = mid + 1;
            } else {
                right = mid;
            }
        }

        return nums[left];
    }

    public static void main(String[] args) {
        int[] nums1 = {3, 4, 5, 1, 2};
        System.out.println("示例 1 输入: " + Arrays.toString(nums1));
        System.out.println("示例 1 输出: " + findMin(nums1));
        System.out.println("-----------------------------");

        int[] nums2 = {4, 5, 6, 7, 0, 1, 2};
        System.out.println("示例 2 输入: " + Arrays.toString(nums2));
        System.out.println("示例 2 输出: " + findMin(nums2));
        System.out.println("-----------------------------");

        int[] nums3 = {11, 13, 15, 17};
        System.out.println("示例 3 输入: " + Arrays.toString(nums3));
        System.out.println("示例 3 输出: " + findMin(nums3));
        System.out.println("-----------------------------");

        int[] nums4 = {2, 1};
        System.out.println("额外示例 输入: " + Arrays.toString(nums4));
        System.out.println("额外示例 输出: " + findMin(nums4));
    }
}
```
