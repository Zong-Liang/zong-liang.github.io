## 题目

**描述：**

假设你正在爬楼梯。需要 `n` 阶你才能到达楼顶。

每次你可以爬 `1` 或 `2` 个台阶。你有多少种不同的方法可以爬到楼顶呢？

**示例 1:**

- **输入:** `n = 2`
- **输出:** 2
- **解释:** 有两种方法可以爬到楼顶。
  1.  1 阶 + 1 阶
  2.  2 阶

**示例 2:**

- **输入:** `n = 3`
- **输出:** 3
- **解释:** 有三种方法可以爬到楼顶。
  1.  1 阶 + 1 阶 + 1 阶
  2.  1 阶 + 2 阶
  3.  2 阶 + 1 阶

**提示:**

- `1 <= n <= 45`

## 题解

### 这题关键不是枚举走法，而是先想“最后一步从哪来”

这道题如果一开始就去枚举所有走法，很容易越想越乱。

因为每一阶都可以走 `1` 步或者 `2` 步，往前展开时分支会越来越多。  
这种题如果硬枚举，虽然小数据还能列出来，但稍微大一点就会发现根本不适合这么做。

真正更自然的想法是反过来问：

**如果我要走到第 `n` 阶，最后一步会是什么？**

答案其实只有两种可能：

- 最后一步走了 `1` 阶，那么前面一定已经走到了第 `n-1` 阶
- 最后一步走了 `2` 阶，那么前面一定已经走到了第 `n-2` 阶

也就是说，所有到达第 `n` 阶的方法，最后都会落在这两类里。  
所以第 `n` 阶的方法数，其实就是：

```text
到达第 n-1 阶的方法数 + 到达第 n-2 阶的方法数
```

这就是这题最核心的状态转移。

### 为什么状态转移一定是 `f(n) = f(n-1) + f(n-2)`

设 `f(n)` 表示爬到第 `n` 阶的方法数。

刚才已经分析过，到达第 `n` 阶，最后一步只有两种来源：

1. 从第 `n-1` 阶走 `1` 步上来
2. 从第 `n-2` 阶走 `2` 步上来

而这两类方法不会重复，因为最后一步走法不同。  
所以它们可以直接相加。

于是就得到：

```text
f(n) = f(n-1) + f(n-2)
```

这就是这题的本质。

很多人会在这里马上想到斐波那契数列。这个联想没问题，但更重要的是先把这个递推关系自己想顺。  
因为这题真正有价值的地方，不是认出“像斐波那契”，而是学会这种动态规划里非常常见的思路：

**先看最后一步怎么来的。**

### 边界为什么是 `f(1)=1`，`f(2)=2`

状态转移有了以后，还差边界。

这个边界其实也不复杂：

- `n = 1` 时，只有一种走法：走 `1` 步
- `n = 2` 时，有两种走法：
  - `1 + 1`
  - `2`

所以：

```text
f(1) = 1
f(2) = 2
```

有了这两个起点，后面的值就都能顺着递推出来。

比如：

```text
f(3) = f(2) + f(1) = 2 + 1 = 3
f(4) = f(3) + f(2) = 3 + 2 = 5
f(5) = f(4) + f(3) = 5 + 3 = 8
```

这也是为什么这题看起来会很像斐波那契数列。

### 用例子走一遍就很容易明白

以 `n = 5` 为例。

如果你想走到第 5 阶，那么最后一步只可能是：

- 从第 4 阶走 1 步上来
- 从第 3 阶走 2 步上来

所以：

```text
f(5) = f(4) + f(3)
```

而：

```text
f(4) = 5
f(3) = 3
```

于是：

```text
f(5) = 5 + 3 = 8
```

这里其实不用真的把 8 种方法一条一条写出来，也能把结果推出来。  
这就是动态规划和暴力枚举最不一样的地方：

**它不关心每条路径具体长什么样，而是关心问题之间的递推关系。**

### 为什么不一定要开数组

如果按最常规的动态规划写法，我们当然可以开一个数组 `dp`，让：

```text
dp[i] = dp[i-1] + dp[i-2]
```

这样写没问题，也很好理解。

但这题还有一个特点：  
在计算当前状态时，我们其实只会用到前两个状态。

也就是说，算 `f(i)` 的时候，只需要：

- `f(i-1)`
- `f(i-2)`

更早的值已经用不到了。

那就说明没必要把整个数组都存下来，只要保留前两个变量，边算边滚动更新就够了。  
这就是所谓的“滚动数组优化”。

所以这题虽然本质是动态规划，但空间上其实可以直接压到 `O(1)`。

### 代码

```java
public int climbStairs(int n) {
    if (n <= 2) {
        return n;
    }

    int first = 1;
    int second = 2;

    for (int i = 3; i <= n; i++) {
        int third = first + second;
        first = second;
        second = third;
    }

    return second;
}
```

### 复杂度分析

时间复杂度是 $O(n)$。  
因为从 `3` 到 `n` 只需要顺着计算一遍。

空间复杂度是 $O(1)$。  
只用了几个变量保存前两个状态。

### 这题真正值得记住的地方

这题最值得记住的，不是“爬楼梯”这个题型本身，也不是简单记住它像斐波那契。

真正该记住的是这种动态规划的思考方式：

**不要一上来就枚举所有方案，而是先看当前状态能由哪些更小的状态转移过来。**

本题里，这个转移非常自然：

- 走到第 `n` 阶
- 最后一步只可能来自 `n-1` 或 `n-2`

所以状态转移就顺手出来了。

以后再遇到这种“求总方案数”“每一步有有限选择”的题，通常都可以先从“最后一步怎么来”这个角度切进去。很多一维动态规划题，核心都是这个套路。

### 完整代码

```java
public class Main {

    public static int climbStairs(int n) {
        if (n <= 1) return 1;
        if (n == 2) return 2;

        int first = 1;
        int second = 2;

        for (int i = 3; i <= n; i++) {
            int third = first + second;
            first = second;
            second = third;
        }

        return second;
    }

    public static void main(String[] args) {
        int n1 = 2;
        System.out.println("示例 1 输入: n = " + n1);
        System.out.println("示例 1 输出: " + climbStairs(n1));
        System.out.println("-----------------------------");

        int n2 = 3;
        System.out.println("示例 2 输入: n = " + n2);
        System.out.println("示例 2 输出: " + climbStairs(n2));
        System.out.println("-----------------------------");

        int n3 = 5;
        System.out.println("额外示例 输入: n = " + n3);
        System.out.println("额外示例 输出: " + climbStairs(n3));
    }
}
```
