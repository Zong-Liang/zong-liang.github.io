## 题目

**描述：**

给你一个字符串 `s` 和一个字符串列表 `wordDict` 作为字典。如果可以利用字典中出现的一个或多个单词拼接出 `s` 则返回 `true`。

**注意：** 不要求字典中出现的单词全部都使用，并且字典中的单词可以重复使用。

**示例 1:**

- **输入:** `s = "leetcode", wordDict = ["leet", "code"]`
- **输出:** `true`
- **解释:** 返回 `true` 因为 `"leetcode"` 可以由 `"leet"` 和 `"code"` 拼接而成。

**示例 2:**

- **输入:** `s = "applepenapple", wordDict = ["apple", "pen"]`
- **输出:** `true`
- **解释:** 返回 `true` 因为 `"applepenapple"` 可以由 `"apple" "pen" "apple"` 拼接而成。注意，你可以重复使用字典中的单词。

**示例 3:**

- **输入:** `s = "catsandog", wordDict = ["cats", "dog", "sand", "and", "cat"]`
- **输出:** `false`

**提示:**

- `1 <= s.length <= 300`
- `1 <= wordDict.length <= 1000`
- `1 <= wordDict[i].length <= 20`
- `s` 和 `wordDict[i]` 仅由小写英文字母组成
- `wordDict` 中的所有字符串 **互不相同**

## 题解

### 这题不要一上来想“整串能不能拆”，先想“前缀能不能成立”

这道题表面上是在问：整个字符串 `s` 能不能由字典里的单词拼出来。

但如果直接盯着整串去想，通常会比较乱。  
因为你会不断遇到这样的问题：

- 第一刀切哪里？
- 如果这一刀切错了怎么办？
- 后面还能不能继续拼？

这种题如果一直围着“整串怎么切”打转，很容易陷进反复试切分的位置里。  
更自然的思路是把问题换一下：

**如果我只看 `s` 的前 i 个字符，它能不能被成功拆分？**

这就是这题动态规划真正的切入点。

### 为什么“前缀是否可拆”是个好状态

设 `dp[i]` 表示：

**字符串 `s` 的前 `i` 个字符，能不能被字典中的单词拼出来。**

这个定义有个好处：  
它把“整个字符串怎么拆”变成了很多个更小的前缀问题。

比如我们想知道前 `i` 个字符能不能拆开，只需要去找一个切分点 `j`：

- 前 `j` 个字符已经可以拆开，也就是 `dp[j] == true`
- 剩下这一段 `s[j...i-1]` 本身正好是字典里的一个单词

只要存在这样一个 `j`，那 `dp[i]` 就成立。

所以这题的核心根本不是“怎么从头拼到尾”，而是：

**能不能找到一个最后切分点，让前半部分已经成立，后半部分刚好是一个单词。**

### 状态转移为什么是这样

既然 `dp[i]` 表示前 `i` 个字符能否拆分，那么对于每个 `i`，我们都可以尝试枚举最后一刀切在哪里。

假设最后一刀切在 `j`，那么前缀 `s[0...i-1]` 就被分成了两段：

- `s[0...j-1]`
- `s[j...i-1]`

这时 `dp[i]` 成立需要同时满足两个条件：

1. 前半段已经能拆出来，也就是 `dp[j] == true`
2. 后半段 `s[j...i-1]` 在字典里

只要这两个条件同时成立，就说明前 `i` 个字符可以拆分成功。

所以状态转移就是：

```text
dp[i] = 是否存在某个 j，使得 dp[j] 为 true，且 s[j...i-1] 在字典中
```

这就是整道题的核心逻辑。

### 为什么 `dp[0]` 要设成 `true`

这是这题里一个很关键、也很容易被忽略的初始化。

我们规定：

```text
dp[0] = true
```

也就是空字符串默认可以被成功拆分。

这看起来有点抽象，但其实非常必要。  
因为当某个前缀本身就是一个完整单词时，比如 `"leet"`，我们希望它能直接成立。

假设 `"leet"` 正好对应前 4 个字符，那么只要：

- 前面空串成立，也就是 `dp[0] == true`
- `"leet"` 在字典里

就能推出 `dp[4] = true`。

如果你不把 `dp[0]` 设成 `true`，这种最基础的情况反而没法顺利转移。

所以这里不是技巧，而是动态规划的基准。

### 用例子走一遍最容易明白

看示例 1：

```text
s = "leetcode"
wordDict = ["leet", "code"]
```

我们来看看 `dp` 是怎么推进的。

#### 初始状态

```text
dp[0] = true
```

表示空串可以成立。

#### 看前 1 到前 3 个字符

无论是 `"l"`、`"le"` 还是 `"lee"`，都不在字典里，所以这些位置都还是 `false`。

#### 看前 4 个字符 `"leet"`

此时我们枚举切分点。

如果切在 `j = 0`，那么：

- `dp[0] == true`
- `s[0...3] = "leet"` 在字典中

所以：

```text
dp[4] = true
```

#### 再看前 8 个字符 `"leetcode"`

继续枚举切分点。

当切在 `j = 4` 时：

- `dp[4] == true`
- `s[4...7] = "code"` 在字典中

所以：

```text
dp[8] = true
```

最后 `dp[8]` 成立，也就说明整个字符串可以拆出来。

你会发现，这个过程里我们根本不需要真的把所有拆法列出来。  
只要能判断某个前缀是否能成立，后面的状态就能顺着推出来。

### 为什么要先把字典放进 `HashSet`

这题在转移时会频繁判断：

```java
s.substring(j, i)
```

是不是在字典里。

如果字典还是 `List`，那每次都要线性查找，开销会比较大。  
而如果先把 `wordDict` 转成 `HashSet`，判断某个单词是否存在就会快很多。

所以这里把字典转成集合，并不是代码习惯问题，而是为了让状态转移里的“查词”这一步更高效。

### 代码

```java
public boolean wordBreak(String s, List<String> wordDict) {
    Set<String> wordSet = new HashSet<>(wordDict);
    int n = s.length();
    boolean[] dp = new boolean[n + 1];
    dp[0] = true;

    for (int i = 1; i <= n; i++) {
        for (int j = 0; j < i; j++) {
            if (dp[j] && wordSet.contains(s.substring(j, i))) {
                dp[i] = true;
                break;
            }
        }
    }

    return dp[n];
}
```

### 为什么找到一个可行切分点就可以 `break`

因为 `dp[i]` 只是一个布尔值，我们关心的是“能不能”，不是“有多少种”。

所以对固定的 `i` 来说，只要找到一个 `j` 满足条件：

- `dp[j] == true`
- `s[j...i-1]` 在字典中

那 `dp[i]` 就已经可以确定为 `true` 了，后面再继续枚举别的 `j` 已经没有意义。  
这时候直接 `break` 就行。

### 复杂度分析

时间复杂度通常记作 $O(n^2)$，其中 `n` 是字符串长度。  
因为外层枚举前缀长度，内层枚举切分点。

空间复杂度是 $O(n)$。  
主要来自长度为 `n + 1` 的 `dp` 数组。  
另外字典集合也会占用额外空间，但这部分通常不算在动态规划数组本身的额外状态里。

### 这题真正值得记住的地方

这题最值得记住的，不是“单词拆分”这个题型本身，而是这种非常典型的动态规划转换方式：

**当题目问“整个字符串能不能成立”时，可以先把它改写成“每个前缀能不能成立”。**

一旦变成前缀问题，很多本来不好下手的字符串题就会顺很多。  
因为你总能尝试去找最后一个切分点，然后把问题拆成：

- 前半段是否已经成立
- 后半段是否满足某个条件

本题里，后半段的条件就是“它是不是字典里的单词”。

以后再遇到字符串拆分、前缀匹配、分段合法性判断这类题，都可以优先往这个方向想。

### 完整代码

```java
import java.util.Arrays;
import java.util.HashSet;
import java.util.List;
import java.util.Set;

public class Main {

    public static boolean wordBreak(String s, List<String> wordDict) {
        Set<String> wordSet = new HashSet<>(wordDict);

        int n = s.length();
        boolean[] dp = new boolean[n + 1];
        dp[0] = true;

        for (int i = 1; i <= n; i++) {
            for (int j = 0; j < i; j++) {
                if (dp[j] && wordSet.contains(s.substring(j, i))) {
                    dp[i] = true;
                    break;
                }
            }
        }

        return dp[n];
    }

    public static void main(String[] args) {
        String s1 = "leetcode";
        List<String> wordDict1 = Arrays.asList("leet", "code");
        System.out.println("示例 1 输入: s = \"" + s1 + "\", wordDict = " + wordDict1);
        System.out.println("示例 1 输出: " + wordBreak(s1, wordDict1));
        System.out.println("-----------------------------");

        String s2 = "applepenapple";
        List<String> wordDict2 = Arrays.asList("apple", "pen");
        System.out.println("示例 2 输入: s = \"" + s2 + "\", wordDict = " + wordDict2);
        System.out.println("示例 2 输出: " + wordBreak(s2, wordDict2));
        System.out.println("-----------------------------");

        String s3 = "catsandog";
        List<String> wordDict3 = Arrays.asList("cats", "dog", "sand", "and", "cat");
        System.out.println("示例 3 输入: s = \"" + s3 + "\", wordDict = " + wordDict3);
        System.out.println("示例 3 输出: " + wordBreak(s3, wordDict3));
    }
}
```
