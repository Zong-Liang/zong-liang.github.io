## 题目

**描述：**

给你一个整数数组 `nums`，找到其中最长严格递增子序列的长度。

**子序列** 是由数组派生而来的序列，删除（或不删除）数组中的元素而不改变其余元素的顺序。例如，`[3,6,2,7]` 是数组 `[0,3,1,6,2,2,7]` 的子序列。

**示例 1:**

- **输入:** `nums = [10,9,2,5,3,7,101,18]`
- **输出:** 4
- **解释:** 最长递增子序列是 `[2,3,7,101]`，因此长度为 4。

**示例 2:**

- **输入:** `nums = [0,1,0,3,2,3]`
- **输出:** 4

**示例 3:**

- **输入:** `nums = [7,7,7,7,7,7,7]`
- **输出:** 1

**提示:**

- `1 <= nums.length <= 2500`
- `-10^4 <= nums[i] <= 10^4`

**进阶:**

- 你能将算法的时间复杂度降低到 O(n log(n)) 吗？

## 题解

### 这题最容易想错的地方，是把“子序列”当成“子数组”

这道题表面上是在找一个“递增序列”，但真正容易让人卡住的点，不是递增，而是“子序列”。

子序列和子数组不一样。  
它不要求元素连续，只要求相对顺序不能变。

比如：

```text
[10,9,2,5,3,7,101,18]
```

其中最长递增子序列可以是：

```text
[2,3,7,101]
```

虽然这些数在原数组里不是挨在一起的，但顺序没变，所以它是合法的子序列。

这也是为什么这题不能用“连续区间”的思路去做，而要换个角度想：

**对于每个位置，它能不能接在前面某个更小的数后面。**

### 最基础的想法：枚举“以每个位置结尾”的答案

先从最容易理解的做法开始。

设 `dp[i]` 表示：

**以 `nums[i]` 结尾的最长严格递增子序列长度。**

这个定义很关键，因为它把原问题拆成了很多个小问题。

比如，如果我们想知道以 `nums[i]` 结尾的最长递增子序列有多长，那就只需要去看它前面所有位置 `j`：

- 如果 `nums[j] < nums[i]`，说明 `nums[i]` 可以接在 `nums[j]` 后面
- 那么以 `nums[i]` 结尾的长度，就有机会从 `dp[j] + 1` 转移过来

所以转移方程就是：

```text
dp[i] = max(dp[j] + 1)，其中 0 <= j < i 且 nums[j] < nums[i]
```

如果前面没有任何数比它小，那它自己单独就能构成一个长度为 1 的递增子序列，所以初始值都是 `1`。

### 为什么这个定义很自然

这题如果直接设“前 i 个数里的最长递增子序列长度”，虽然也能做，但写转移时会比较别扭。  
因为你很难知道当前这个数到底有没有被选进去。

而设成“以 `nums[i]` 结尾”，就会很清楚：

- 当前这个数一定被选了
- 那么它前面能接谁，只需要看那些比它小的位置

这样状态转移就非常自然。

动态规划里很常见的一类思路就是这样：  
**不是直接求整体答案，而是先求“以某个位置结尾”的局部最优。**

### 用例子走一遍

来看示例 1：

```text
nums = [10,9,2,5,3,7,101,18]
```

我们来顺着看几个关键位置。

#### `10`

前面没有数，所以：

```text
dp[0] = 1
```

#### `9`

前面虽然有 `10`，但 `10 > 9`，不能接，所以：

```text
dp[1] = 1
```

#### `2`

前面没有比 `2` 更小的数，所以：

```text
dp[2] = 1
```

#### `5`

前面比 `5` 小的有 `2`，所以它可以接在 `2` 后面：

```text
dp[3] = dp[2] + 1 = 2
```

#### `3`

前面比 `3` 小的也只有 `2`，所以：

```text
dp[4] = dp[2] + 1 = 2
```

#### `7`

前面比 `7` 小的有很多：`2`、`5`、`3`。  
其中最好的接法是接在长度更长的后面，比如接在 `5` 或 `3` 后面：

```text
dp[5] = 3
```

#### `101`

它几乎可以接在所有前面更小的数后面，而最优的是接在长度为 3 的序列后面，所以：

```text
dp[6] = 4
```

#### `18`

它不能接在 `101` 后面，但可以接在 `7` 后面，所以：

```text
dp[7] = 4
```

最后整个数组里的最大值就是答案，也就是 `4`。

所以这题的动态规划本质上就是一句话：

**每个位置都去看看，自己能接在谁后面，并把最好的那条链延长一位。**

### 动态规划代码

```java
public int lengthOfLIS_DP(int[] nums) {
    int n = nums.length;
    int[] dp = new int[n];
    Arrays.fill(dp, 1);

    int ans = 1;

    for (int i = 1; i < n; i++) {
        for (int j = 0; j < i; j++) {
            if (nums[i] > nums[j]) {
                dp[i] = Math.max(dp[i], dp[j] + 1);
            }
        }
        ans = Math.max(ans, dp[i]);
    }

    return ans;
}
```

### 这个做法为什么是 `O(n^2)`

因为对每个位置 `i`，我们都要回头看前面的所有位置 `j`。  
所以是双层循环，时间复杂度自然是：

```text
O(n^2)
```

在 `n <= 2500` 的数据范围下，这个做法是能通过的，而且也最容易讲清楚。

但题目后面还问了一个进阶问题：能不能做到 `O(n log n)`。  
这就需要换一种思路。

### 进阶的关键，不是维护真实序列，而是维护“同长度下尽量小的结尾”

这里是这题最难、也最容易讲乱的地方。

我们先明确一点：  
在求最长递增子序列长度时，我们并不一定要把那条真实的序列完整维护出来。

更重要的是维护这样一个信息：

**对于某个长度的递增子序列，它的结尾元素越小越好。**

为什么？

因为结尾越小，后面就越容易接上更多更大的数，未来继续变长的机会就越大。

比如长度为 2 的递增子序列，如果你手里有这两种结尾：

```text
[1, 5]
[2, 3]
```

虽然它们长度都一样，但第二个结尾是 `3`，明显比 `5` 更有潜力。  
因为以后如果再来一个 `4`，它能接在 `[2,3]` 后面，却接不到 `[1,5]` 后面。

所以对于相同长度的递增子序列，我们只关心那个**结尾最小**的版本。

### `tails` 数组到底表示什么

我们维护一个数组 `tails`。

其中：

```text
tails[k]
```

表示：

**长度为 `k + 1` 的递增子序列，所有可能结尾中的最小值。**

这个定义有两个非常重要的含义：

第一，`tails` 的下标表示“长度”。  
第二，`tails` 存的是“这个长度下最优的结尾”。

比如如果当前：

```text
tails = [2, 3, 7, 18]
```

它并不一定表示真实存在一条子序列 `[2,3,7,18]`。  
它真正表达的是：

- 长度为 1 的递增子序列，最小结尾是 `2`
- 长度为 2 的递增子序列，最小结尾是 `3`
- 长度为 3 的递增子序列，最小结尾是 `7`
- 长度为 4 的递增子序列，最小结尾是 `18`

而 `tails` 的长度，本身就是当前已知的最长递增子序列长度。

### 新元素来了以后，为什么要二分找到第一个大于等于它的位置

现在我们遍历数组中的每个 `num`。

有两种情况。

#### 第一种：`num` 比当前所有结尾都大

比如当前：

```text
tails = [2, 3, 7]
```

新来一个 `18`。

因为 `18 > 7`，说明它可以直接接在当前最长序列后面，于是长度增加一位：

```text
tails = [2, 3, 7, 18]
```

#### 第二种：`num` 不能让长度增加，但可以让结尾变得更小

比如当前还是：

```text
tails = [2, 3, 7, 18]
```

新来一个 `5`。

它不能接在 `18` 后面，所以没法把长度从 4 变成 5。  
但它可以把长度为 3 的序列结尾从 `7` 改成 `5`：

```text
tails = [2, 3, 5, 18]
```

这样长度没变，但序列“潜力”更好了。

所以每次我们要在 `tails` 里找到**第一个大于等于 `num` 的位置**，然后用 `num` 去替换它。

之所以是“第一个大于等于”，是因为我们希望：

- 不破坏更短长度的最优结尾
- 同时尽量早地把当前值放进去，让对应长度的结尾更小

而 `tails` 本身是递增的，所以这个位置可以用二分查找。

### 为什么 `tails` 一定是递增的

这一点如果想不顺，后面的二分就会显得像硬套模板。

其实很好理解。

如果长度为 1 的递增子序列最小结尾是 `tails[0]`，  
长度为 2 的递增子序列最小结尾是 `tails[1]`，  
那想要长度从 1 变成 2，后一个数一定要严格大于前一个数。

所以必然有：

```text
tails[0] < tails[1] < tails[2] < ...
```

正因为 `tails` 保持递增，我们才能在上面做二分。

### 用示例感受一下 `tails` 的变化

还是看：

```text
[10,9,2,5,3,7,101,18]
```

遍历过程大致是这样：

#### 读到 `10`

```text
tails = [10]
```

#### 读到 `9`

替换第一个大于等于 `9` 的位置：

```text
tails = [9]
```

#### 读到 `2`

继续替换：

```text
tails = [2]
```

#### 读到 `5`

`5` 大于当前最后一个元素 `2`，可以直接接上：

```text
tails = [2,5]
```

#### 读到 `3`

替换第一个大于等于 `3` 的位置，也就是 `5`：

```text
tails = [2,3]
```

#### 读到 `7`

比最后一个元素大，接上：

```text
tails = [2,3,7]
```

#### 读到 `101`

接上：

```text
tails = [2,3,7,101]
```

#### 读到 `18`

替换第一个大于等于 `18` 的位置，也就是 `101`：

```text
tails = [2,3,7,18]
```

最后 `tails` 长度是 `4`，所以答案就是 `4`。

注意，这里的 `tails` 不一定就是某条真实的最长递增子序列。  
但它的长度一定是正确的。

### 二分优化代码

```java
public int lengthOfLIS_BS(int[] nums) {
    int[] tails = new int[nums.length];
    int len = 0;

    for (int num : nums) {
        int left = 0, right = len;

        while (left < right) {
            int mid = left + (right - left) / 2;
            if (tails[mid] < num) {
                left = mid + 1;
            } else {
                right = mid;
            }
        }

        tails[left] = num;
        if (left == len) {
            len++;
        }
    }

    return len;
}
```

### 复杂度分析

动态规划做法的时间复杂度是 $O(n^2)$，空间复杂度是 $O(n)$。

二分优化做法的时间复杂度是 $O(n \log n)$，因为每个元素都要进行一次二分查找。  
空间复杂度仍然是 $O(n)$，因为需要维护 `tails` 数组。

### 这题真正值得记住的地方

这题最值得记住的，不是单纯背下“LIS 可以用 tails + 二分”。

真正该记住的是两层思路：

第一层，动态规划的角度：

**以每个位置结尾，去看它能接在谁后面。**

第二层，进阶优化的角度：

**对于同样长度的递增子序列，结尾越小越好。**

正是因为这个“结尾越小越有潜力”的观察，才让 `O(n log n)` 的做法成立。

所以以后再遇到这种“最长长度 + 可替换更优结尾”的题，不要只记模板，先问自己一句：

**我维护的这个状态，是否在同长度下越小越好？**

如果答案是肯定的，就很可能有机会继续往二分优化的方向走。

### 完整代码

```java
import java.util.Arrays;

public class Main {

    public static int lengthOfLIS_DP(int[] nums) {
        if (nums.length == 0) return 0;

        int[] dp = new int[nums.length];
        Arrays.fill(dp, 1);
        int ans = 1;

        for (int i = 1; i < nums.length; i++) {
            for (int j = 0; j < i; j++) {
                if (nums[i] > nums[j]) {
                    dp[i] = Math.max(dp[i], dp[j] + 1);
                }
            }
            ans = Math.max(ans, dp[i]);
        }

        return ans;
    }

    public static int lengthOfLIS_BS(int[] nums) {
        if (nums.length == 0) return 0;

        int[] tails = new int[nums.length];
        int len = 0;

        for (int num : nums) {
            int left = 0, right = len;

            while (left < right) {
                int mid = left + (right - left) / 2;
                if (tails[mid] < num) {
                    left = mid + 1;
                } else {
                    right = mid;
                }
            }

            tails[left] = num;
            if (left == len) {
                len++;
            }
        }

        return len;
    }

    public static void main(String[] args) {
        int[] nums1 = {10, 9, 2, 5, 3, 7, 101, 18};
        System.out.println("示例 1 输入: " + Arrays.toString(nums1));
        System.out.println("DP 解法输出: " + lengthOfLIS_DP(nums1));
        System.out.println("二分优化解法输出: " + lengthOfLIS_BS(nums1));
        System.out.println("-----------------------------");

        int[] nums2 = {0, 1, 0, 3, 2, 3};
        System.out.println("示例 2 输入: " + Arrays.toString(nums2));
        System.out.println("DP 解法输出: " + lengthOfLIS_DP(nums2));
        System.out.println("二分优化解法输出: " + lengthOfLIS_BS(nums2));
        System.out.println("-----------------------------");

        int[] nums3 = {7, 7, 7, 7, 7, 7, 7};
        System.out.println("示例 3 输入: " + Arrays.toString(nums3));
        System.out.println("DP 解法输出: " + lengthOfLIS_DP(nums3));
        System.out.println("二分优化解法输出: " + lengthOfLIS_BS(nums3));
    }
}
```
