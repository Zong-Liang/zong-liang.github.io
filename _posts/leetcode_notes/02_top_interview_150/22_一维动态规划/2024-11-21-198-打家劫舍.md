## 题目

**描述：**

你是一个专业的小偷，计划偷窃沿街的房屋。每间房内都藏有一定的现金，影响你偷窃的唯一制约因素就是相邻的房屋装有相互连通的防盗系统，**如果两间相邻的房屋在同一晚上被小偷闯入，系统会自动报警**。

给定一个代表每个房屋存放金额的非负整数数组，计算你 **不触动警报装置** 的情况下，一夜之内能够偷窃到的最高金额。

**示例 1:**

- **输入:** `[1,2,3,1]`
- **输出:** 4
- **解释:** 偷窃 1 号房屋 (金额 = 1)，然后偷窃 3 号房屋 (金额 = 3)。

  偷窃到的最高金额 = 1 + 3 = 4。

**示例 2:**

- **输入:** `[2,7,9,3,1]`
- **输出:** 12
- **解释:** 偷窃 1 号房屋 (金额 = 2)，偷窃 3 号房屋 (金额 = 9)，接着偷窃 5 号房屋 (金额 = 1)。

  偷窃到的最高金额 = 2 + 9 + 1 = 12。

**提示:**

- `1 <= nums.length <= 100`
- `0 <= nums[i] <= 400`

## 题解

### 这题关键不在“怎么偷”，而在“每一间房只有偷或不偷两种选择”

这道题看着像故事题，但把题目条件抽掉以后，核心其实很简单：

- 每间房都有一个价值
- 不能同时偷相邻两间
- 要求拿到的总金额最大

很多人一开始会试着去想具体该偷哪几家，但这条路很容易越想越乱。  
因为一旦开始枚举组合，分支会越来越多，很快就不适合继续手推了。

真正更自然的切入点是：

**走到第 `i` 间房时，这一间其实只有两种选择：偷，或者不偷。**

这就是整道题的核心。

### 为什么状态转移是 `max(不偷, 偷)`

设 `dp[i]` 表示：  
**从第 0 间房到第 i 间房，能够偷到的最大金额。**

那么来到第 `i` 间房时，有两种情况。

#### 第一种：不偷第 `i` 间

如果这一间不偷，那答案就和前一间的最优解一样，因为你什么都没多拿：

```text
dp[i-1]
```

#### 第二种：偷第 `i` 间

如果这一间要偷，那第 `i-1` 间就一定不能偷。  
所以你能接上的，只有前 `i-2` 间房的最优解，再加上当前这间的钱：

```text
dp[i-2] + nums[i]
```

这两种情况里，哪个更大，就取哪个。

所以状态转移方程就是：

```text
dp[i] = max(dp[i-1], dp[i-2] + nums[i])
```

这就是这题最核心的递推关系。

### 这条转移为什么很自然

你可以把它理解成一个很朴素的决策过程：

来到当前这间房时，你要么放弃它，保留之前已经拿到的最好结果；  
要么选择拿它，但代价是必须放弃上一间。

所以“偷当前这家”这件事，不是单独决定的，它会直接影响你上一间能不能选。  
这也是为什么这题本质上是动态规划，而不是简单贪心。

因为你不能只看当前这间值不值钱，而要同时看：

- 不偷它，之前最优是多少
- 偷它，能接上的历史最优是多少

比较完这两个值，当前状态才算定下来。

### 用例子走一遍

看示例 2：

```text
nums = [2,7,9,3,1]
```

我们从前往后推。

#### 第 0 间

只有一间房，那最优解就是偷它：

```text
2
```

#### 第 1 间

两间房不能一起偷，所以只能在 `2` 和 `7` 里选大的：

```text
7
```

#### 第 2 间

现在来到金额为 `9` 的房子。

有两种选择：

- 不偷它：保持前一间的最优解 `7`
- 偷它：前一间不能偷，只能接上第 0 间，所以是 `2 + 9 = 11`

取更大值：

```text
11
```

#### 第 3 间

当前金额是 `3`。

- 不偷它：保持 `11`
- 偷它：接上前两间最优 `7 + 3 = 10`

取更大值：

```text
11
```

#### 第 4 间

当前金额是 `1`。

- 不偷它：保持 `11`
- 偷它：接上前两间最优 `11 + 1 = 12`

取更大值：

```text
12
```

所以最终答案就是 `12`。

整个过程里，你会发现我们从头到尾都没有真的“列出所有偷法”，而是在每一步都只比较当前这间偷还是不偷。  
这就是动态规划真正省事的地方。

### 为什么可以不用整个数组

如果按最标准的写法，我们可以开一个 `dp` 数组，把每一步最优解都存下来。

但这题还有一个很明显的特点：  
在计算 `dp[i]` 时，我们真正会用到的只有两个值：

- `dp[i-1]`
- `dp[i-2]`

更早以前的状态，当前这一步其实已经不关心了。

那就说明没有必要把整个 `dp` 数组都保留下来，只要用两个变量滚动维护前两个状态就够了。

这也是为什么这题可以把空间复杂度压到 `O(1)`。

### 代码

```java
public int rob(int[] nums) {
    int prev = 0;
    int curr = 0;

    for (int num : nums) {
        int next = Math.max(curr, prev + num);
        prev = curr;
        curr = next;
    }

    return curr;
}
```

### 这段代码里两个变量分别表示什么

这个写法看起来很短，但第一次看时容易有点抽象。

可以这样理解：

- `prev` 表示前两间房范围内的最优解，也就是 `dp[i-2]`
- `curr` 表示前一间房范围内的最优解，也就是 `dp[i-1]`

每遍历到一个新的 `num`，我们就计算当前这一间结束时的最优值：

```java
next = Math.max(curr, prev + num);
```

含义正好就是：

- 不偷当前这家：`curr`
- 偷当前这家：`prev + num`

然后再把状态往前滚动：

- 原来的 `curr` 变成新的 `prev`
- 原来的 `next` 变成新的 `curr`

这样一路推到最后，`curr` 就是答案。

### 复杂度分析

时间复杂度是 $O(n)$。  
因为只需要把数组遍历一遍。

空间复杂度是 $O(1)$。  
只用了常数个变量保存状态。

### 这题真正值得记住的地方

这题最值得记住的，不是“打家劫舍”这个题面本身，而是这种非常典型的一维动态规划思路：

**对于当前位置，答案通常来自“选”或者“不选”两种决策。**

本题里：

- 不偷当前房子，就继承前一个状态
- 偷当前房子，就接上前两个状态

这种“当前位置选不选”的模型，其实在很多动态规划题里都会反复出现。  
以后再遇到“相邻不能同时选”“当前选择会影响前后状态”这类题时，都可以优先往这个方向想。

### 完整代码

```java
import java.util.Arrays;

public class Main {

    public static int rob(int[] nums) {
        if (nums == null || nums.length == 0) {
            return 0;
        }

        int prev = 0;
        int curr = 0;

        for (int num : nums) {
            int next = Math.max(curr, prev + num);
            prev = curr;
            curr = next;
        }

        return curr;
    }

    public static void main(String[] args) {
        int[] nums1 = {1, 2, 3, 1};
        System.out.println("示例 1 输入: " + Arrays.toString(nums1));
        System.out.println("示例 1 输出: " + rob(nums1));
        System.out.println("-----------------------------");

        int[] nums2 = {2, 7, 9, 3, 1};
        System.out.println("示例 2 输入: " + Arrays.toString(nums2));
        System.out.println("示例 2 输出: " + rob(nums2));
        System.out.println("-----------------------------");

        int[] nums3 = {0};
        System.out.println("示例 3 输入: " + Arrays.toString(nums3));
        System.out.println("示例 3 输出: " + rob(nums3));
        System.out.println("-----------------------------");

        int[] nums4 = {100};
        System.out.println("示例 4 输入: " + Arrays.toString(nums4));
        System.out.println("示例 4 输出: " + rob(nums4));
    }
}
```
