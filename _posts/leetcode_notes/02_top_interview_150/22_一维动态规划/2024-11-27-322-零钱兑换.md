## 题目

**描述：**

给你一个整数数组 `coins`，表示不同面额的硬币；以及一个整数 `amount`，表示总金额。

计算并返回可以凑成总金额所需的 **最少的硬币个数**。如果没有任何一种硬币组合能组成总金额，返回 `-1`。

你可以认为每种硬币的数量是无限的。

**示例 1:**

- **输入:** `coins = [1, 2, 5]`, `amount = 11`
- **输出:** 3
- **解释:** `11 = 5 + 5 + 1`

**示例 2:**

- **输入:** `coins = [2]`, `amount = 3`
- **输出:** -1

**示例 3:**

- **输入:** `coins = [1]`, `amount = 0`
- **输出:** 0

**提示:**

- `1 <= coins.length <= 12`
- `1 <= coins[i] <= 2^31 - 1`
- `0 <= amount <= 10^4`

## 题解

### 这题不要先想“怎么组合”，先想“最后放进来的是哪枚硬币”

这道题如果直接去想“11 可以怎么凑”“3 可以怎么拼”，很快就会进入各种组合枚举里。

比如金额是 `11`，你会开始想：

- `5 + 5 + 1`
- `5 + 2 + 2 + 2`
- `2 + 2 + 2 + 2 + 2 + 1`
- ...

这种想法当然不算错，但不适合写成高效算法。  
因为题目要的不是“列出所有组合”，而是“最少用多少枚硬币”。

更自然的切入点其实是反过来想：

**如果我要凑出金额 `i`，那最后放进去的那枚硬币会是谁？**

只要这个问题想顺了，这题的动态规划就出来了。

### 为什么状态转移是 `dp[i] = min(dp[i - coin] + 1)`

设 `dp[i]` 表示：

**凑出金额 `i` 所需要的最少硬币数。**

现在来考虑金额 `i`。

假设最后一枚放进去的硬币面额是 `coin`，那在放这枚硬币之前，我们其实已经先凑出了金额：

```text
i - coin
```

如果凑出 `i - coin` 的最优解已经知道是 `dp[i - coin]`，那再加上当前这一枚硬币，凑出 `i` 的方案就变成了：

```text
dp[i - coin] + 1
```

因为最后一枚硬币可能是任意一种面额，所以我们要把所有可行的 `coin` 都试一遍，取最小值。

于是状态转移就自然变成：

```text
dp[i] = min(dp[i], dp[i - coin] + 1)
```

前提当然是这枚硬币面额不能超过当前金额，也就是：

```text
coin <= i
```

这就是整道题最核心的递推关系。

### 为什么 `dp[0] = 0` 很重要

动态规划里最关键的，不只是转移式，初始状态也同样重要。

这里我们设：

```text
dp[0] = 0
```

意思是：凑出金额 `0`，不需要任何硬币。

这个定义看起来简单，但它是整个递推的起点。

比如金额是 `1`，如果硬币里刚好有 `1`，那么：

```text
dp[1] = dp[0] + 1 = 1
```

如果没有这个起点，后面的很多状态都没法顺利推出来。

所以 `dp[0] = 0` 不是顺手一写，而是整个问题最基本的基准。

### 为什么其他位置要先初始化成“不可能的大值”

这题还有一个很容易忽略的细节：  
在开始转移之前，我们并不知道某个金额是不是能凑出来。

所以对 `dp[1]` 到 `dp[amount]`，一开始不能随便设成 `0`，因为 `0` 会让后面的最小值比较失真。  
更合理的做法是先把它们设成一个“不可能的很大值”，表示当前还不可达。

最常见的写法就是：

```java
int max = amount + 1;
Arrays.fill(dp, max);
```

为什么用 `amount + 1`？

因为即使硬币里有面额 `1`，凑出金额 `amount` 最多也只需要 `amount` 枚硬币。  
所以 `amount + 1` 一定比任何真实答案都大，可以安全地拿来当“无穷大”。

这样后面做 `min` 更新时，才能正确地把它替换成真实答案。

### 用例子走一遍最容易明白

看示例 1：

```text
coins = [1,2,5], amount = 11
```

我们来顺着推几个关键状态。

#### 金额 1

只能用 `1`：

```text
dp[1] = dp[0] + 1 = 1
```

#### 金额 2

可以选择：

- 用一个 `2`：`dp[0] + 1 = 1`
- 或者两个 `1`：`dp[1] + 1 = 2`

取最小值：

```text
dp[2] = 1
```

#### 金额 5

可以选择：

- `dp[4] + 1`
- `dp[3] + 1`
- `dp[0] + 1`

显然最后一种最好，因为直接用一枚 `5`：

```text
dp[5] = 1
```

#### 金额 10

最优方案其实是两个 `5`：

```text
dp[10] = dp[5] + 1 = 2
```

#### 金额 11

可以从：

- `dp[10] + 1`，最后放一个 `1`
- `dp[9] + 1`，最后放一个 `2`
- `dp[6] + 1`，最后放一个 `5`

其中最优的是：

```text
dp[10] + 1 = 2 + 1 = 3
```

所以：

```text
dp[11] = 3
```

答案就是 `3`，对应方案是：

```text
5 + 5 + 1
```

你会发现，整个过程里我们并没有显式枚举所有组合，而是在每个金额上都只问一句：

**如果最后放一枚硬币，最优解会不会更好？**

这就是这题动态规划真正省力的地方。

### 代码

```java
public int coinChange(int[] coins, int amount) {
    int max = amount + 1;
    int[] dp = new int[amount + 1];
    Arrays.fill(dp, max);
    dp[0] = 0;

    for (int i = 1; i <= amount; i++) {
        for (int coin : coins) {
            if (coin <= i) {
                dp[i] = Math.min(dp[i], dp[i - coin] + 1);
            }
        }
    }

    return dp[amount] > amount ? -1 : dp[amount];
}
```

### 为什么最后要判断是否仍然“大于 amount”

因为有些金额根本凑不出来。

比如：

```text
coins = [2], amount = 3
```

无论怎么试，都没法把 `3` 拼出来。  
这种情况下，`dp[3]` 就永远不会被更新，仍然保持初始值 `amount + 1`。

所以最后如果发现：

```text
dp[amount] > amount
```

就说明这个金额不可达，应该返回 `-1`。

否则返回真正的最小硬币数即可。

### 复杂度分析

时间复杂度是 $O(amount \times n)$，其中 `n` 是硬币种类数。  
因为每个金额都要尝试所有硬币。

空间复杂度是 $O(amount)$。  
主要来自一维 `dp` 数组。

### 这题真正值得记住的地方

这题最值得记住的，不是“零钱兑换”这个题面本身，也不是简单记住它叫“完全背包”。

真正该记住的是这种动态规划视角：

**当题目要求“最少数量”时，通常可以去想：当前状态的最优解，最后一步是怎么来的。**

本题里，“最后一步”就是最后放进去的那枚硬币。  
只要这个角度想顺，转移式就会非常自然：

- 先看去掉这枚硬币之前的金额
- 再加上这 1 枚硬币
- 在所有可能的硬币里取最小值

以后再遇到“最少步数”“最少次数”“最少物品数”这类题，也都可以优先往这个方向想。

### 完整代码

```java
import java.util.Arrays;

public class Main {

    public static int coinChange(int[] coins, int amount) {
        int max = amount + 1;
        int[] dp = new int[amount + 1];

        Arrays.fill(dp, max);
        dp[0] = 0;

        for (int i = 1; i <= amount; i++) {
            for (int coin : coins) {
                if (coin <= i) {
                    dp[i] = Math.min(dp[i], dp[i - coin] + 1);
                }
            }
        }

        return dp[amount] > amount ? -1 : dp[amount];
    }

    public static void main(String[] args) {
        int[] coins1 = {1, 2, 5};
        int amount1 = 11;
        System.out.println("示例 1 输入: coins = " + Arrays.toString(coins1) + ", amount = " + amount1);
        System.out.println("示例 1 输出: " + coinChange(coins1, amount1));
        System.out.println("-----------------------------");

        int[] coins2 = {2};
        int amount2 = 3;
        System.out.println("示例 2 输入: coins = " + Arrays.toString(coins2) + ", amount = " + amount2);
        System.out.println("示例 2 输出: " + coinChange(coins2, amount2));
        System.out.println("-----------------------------");

        int[] coins3 = {1};
        int amount3 = 0;
        System.out.println("示例 3 输入: coins = " + Arrays.toString(coins3) + ", amount = " + amount3);
        System.out.println("示例 3 输出: " + coinChange(coins3, amount3));
        System.out.println("-----------------------------");

        int[] coins4 = {186, 419, 83, 408};
        int amount4 = 6249;
        System.out.println("额外示例 输入: coins = " + Arrays.toString(coins4) + ", amount = " + amount4);
        System.out.println("额外示例 输出: " + coinChange(coins4, amount4));
    }
}
```
