## 题目

**描述：**

给你一个整数数组 `nums`，其中元素已经按 **升序** 排列，请你将其转换为一棵 **高度平衡** 二叉搜索树。

**高度平衡** 二叉树是一棵满足「每个节点的左右两个子树的高度差的绝对值不超过 1」的二叉树。

**示例 1:**

![示例1图片](https://assets.leetcode.com/uploads/2021/02/18/btree1.jpg)

- **输入:** `nums = [-10,-3,0,5,9]`
- **输出:** `[0,-3,9,-10,null,5]`
- **解释:** `[0,-10,5,null,-3,null,9]` 也将被视为正确答案。

**示例 2:**

![示例2图片](https://assets.leetcode.com/uploads/2021/02/18/btree2.jpg)

- **输入:** `nums = [1,3]`
- **输出:** `[3,1]`
- **解释:** `[1,null,3]` 和 `[3,1]` 都是高度平衡二叉搜索树。

**提示:**

- `1 <= nums.length <= 10^4`
- `-10^4 <= nums[i] <= 10^4`
- `nums` 按 **严格递增** 顺序排列

## 题解

### 这题真正要同时满足两个条件

题目给了一个已经排好序的数组，让我们把它变成一棵二叉搜索树。

这里其实有两个要求要同时满足：

- 它得是一棵 **二叉搜索树**
- 它还得是 **高度平衡** 的

第一个要求决定了节点值之间的相对位置不能乱。  
第二个要求则决定了树不能长得太偏，左右子树要尽量均衡。

所以这题的关键不是“怎么建树”，而是：

**怎么选根节点，才能既保持 BST 性质，又尽量平衡。**

### 为什么中间元素最适合当根

既然数组本身已经是升序的，那么如果我们选了某个位置作为根节点：

- 它左边的所有元素都比它小
- 它右边的所有元素都比它大

这正好天然符合二叉搜索树的定义。

那接下来问题就只剩一个：

**选哪个位置做根，才能让树尽量平衡？**

答案就是中间位置。

因为中间元素一旦作为根：

- 左边一半去构建左子树
- 右边一半去构建右子树

这样左右两边的元素数量最接近，构建出来的树自然就更平衡。

所以这题最核心的一步，不是递归，而是先想明白：

**要想让树平衡，就应该优先选中间元素当根。**

### 为什么这其实就是分治

一旦根节点确定了，剩下的问题就和原问题完全一样了。

比如数组是：

```text
[-10, -3, 0, 5, 9]
```

如果选中间元素 `0` 作为根，那么：

- 左半部分 `[-10, -3]` 要构建成左子树
- 右半部分 `[5, 9]` 要构建成右子树

你会发现，左右两部分其实还是同样的问题：

> 给一段有序数组，把它变成一棵高度平衡 BST

这就是非常标准的分治结构。

也就是说，这题并不是先想“树怎么递归”，而是先看出：

**整个问题可以被拆成两个规模更小、结构完全相同的子问题。**

### 递归到底在做什么

所以递归函数本质上只需要描述一件事：

**把数组的某个区间 `nums[left...right]` 转成一棵平衡 BST。**

那这个函数里要做的事情也很固定：

1. 如果区间已经空了，也就是 `left > right`，说明这里没有节点，返回 `null`
2. 取中间位置 `mid`
3. 用 `nums[mid]` 创建当前根节点
4. 递归构建左半段，挂到 `root.left`
5. 递归构建右半段，挂到 `root.right`
6. 返回当前根节点

整个过程非常自然，因为它完全是顺着“中间做根、左右分治”这条思路展开的。

### 用示例走一遍

还是看示例 1：

```text
nums = [-10, -3, 0, 5, 9]
```

先取中间元素 `0` 作为根：

```text
        0
```

左边部分 `[-10, -3]` 去构建左子树。  
如果这里取中间偏左的 `-10` 做根，那左子树可以长成：

```text
      -10
        \
        -3
```

右边部分 `[5, 9]` 去构建右子树。  
同理，可以长成：

```text
        5
         \
         9
```

最后整棵树就是：

```text
        0
      /   \
   -10     5
      \     \
      -3     9
```

当然，这不是唯一答案。  
比如左半段选 `-3` 做根、右半段选 `9` 做根，也同样是合法的。

这也是题目里为什么会说：  
**只要结果是高度平衡 BST，就都算对。**

### 为什么答案不唯一也没关系

很多人第一次做这题时会有个疑问：

如果数组长度是偶数，那中间其实有两个位置，到底该选哪一个？

比如：

```text
[1, 3]
```

你可以选左边的 `1` 做根，结果是：

```text
1
 \
  3
```

也可以选右边的 `3` 做根，结果是：

```text
  3
 /
1
```

这两棵树都满足：

- BST 性质
- 高度平衡

所以这题并不要求你构造出某一棵唯一的树，只要结果合法即可。

这也是为什么代码里常见的写法会直接固定成：

```java
int mid = left + (right - left) / 2;
```

它选的是中间偏左的位置。  
你改成偏右也一样能过。

### 代码

```java
public TreeNode sortedArrayToBST(int[] nums) {
    return build(nums, 0, nums.length - 1);
}

private TreeNode build(int[] nums, int left, int right) {
    if (left > right) {
        return null;
    }

    int mid = left + (right - left) / 2;
    TreeNode root = new TreeNode(nums[mid]);

    root.left = build(nums, left, mid - 1);
    root.right = build(nums, mid + 1, right);

    return root;
}
```

### 这段代码最关键的地方

真正关键的其实就一句：

```java
int mid = left + (right - left) / 2;
```

因为它决定了当前这棵子树的根是谁。

而一旦根选成中间元素，后面的左右递归其实就是顺水推舟：

- 左边区间自然都比根小
- 右边区间自然都比根大
- 左右规模尽量均衡，所以树也尽量平衡

所以这题本质上并不是“背一个递归模板”，而是：

**先用中间元素保证平衡，再用有序数组保证 BST。**

这两点结合起来，答案就自然出来了。

### 为什么这题不需要额外排序或调整结构

因为题目已经把最关键的条件提前给你了：

```text
nums 按严格递增顺序排列
```

这意味着数组本身已经具备了 BST 中序遍历结果该有的顺序。

所以我们不用再去排序，也不用在建树后额外调整。  
只要每次都把当前区间的中间元素拿出来做根，顺序和结构就都自动满足了。

换句话说，这题真正省事的地方就在于：

**有序数组已经帮你把值的大小关系准备好了。**

### 复杂度分析

数组中的每个元素都会恰好被拿来创建一次树节点，所以时间复杂度是：

- 时间复杂度：$O(n)$

空间复杂度主要来自递归调用栈。  
因为每次都取中间元素，构建出来的树是平衡的，所以递归深度大约是树的高度：

- 空间复杂度：$O(\log n)$

这也是这题用分治递归非常合适的原因之一。

### 这题真正值得记住的地方

这题最值得记住的，不是“有序数组转 BST”这个题面本身，而是这个思路：

**当题目既要求“有序关系”，又要求“结构尽量平衡”时，优先考虑中间切分。**

因为中间切分往往同时满足两件事：

- 顺序不会乱
- 左右规模最均衡

这其实是分治题里一个很常见的切入口。  
这题只是把它用在了 BST 构造上。

### 完整代码

```java
import java.util.ArrayList;
import java.util.Arrays;
import java.util.LinkedList;
import java.util.List;
import java.util.Queue;

class TreeNode {
    int val;
    TreeNode left;
    TreeNode right;

    TreeNode(int val) {
        this.val = val;
    }
}

public class Main {

    public static TreeNode sortedArrayToBST(int[] nums) {
        return build(nums, 0, nums.length - 1);
    }

    private static TreeNode build(int[] nums, int left, int right) {
        if (left > right) {
            return null;
        }

        int mid = left + (right - left) / 2;
        TreeNode root = new TreeNode(nums[mid]);

        root.left = build(nums, left, mid - 1);
        root.right = build(nums, mid + 1, right);

        return root;
    }

    public static String printTree(TreeNode root) {
        if (root == null) return "[]";

        List<String> output = new ArrayList<>();
        Queue<TreeNode> queue = new LinkedList<>();
        queue.offer(root);

        while (!queue.isEmpty()) {
            TreeNode node = queue.poll();

            if (node == null) {
                output.add("null");
            } else {
                output.add(String.valueOf(node.val));
                queue.offer(node.left);
                queue.offer(node.right);
            }
        }

        int i = output.size() - 1;
        while (i >= 0 && output.get(i).equals("null")) {
            output.remove(i);
            i--;
        }

        return output.toString();
    }

    public static void main(String[] args) {
        int[] nums1 = {-10, -3, 0, 5, 9};
        System.out.println("示例 1 输入: nums = " + Arrays.toString(nums1));
        TreeNode root1 = sortedArrayToBST(nums1);
        System.out.println("示例 1 输出: " + printTree(root1));

        int[] nums2 = {1, 3};
        System.out.println("示例 2 输入: nums = " + Arrays.toString(nums2));
        TreeNode root2 = sortedArrayToBST(nums2);
        System.out.println("示例 2 输出: " + printTree(root2));
    }
}
```
