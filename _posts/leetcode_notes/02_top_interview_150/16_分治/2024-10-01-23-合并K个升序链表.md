## 题目

**描述：**

给你一个链表数组，每个链表都已经按升序排列。

请你将所有链表合并到一个升序链表中，返回合并后的链表。

**示例 1:**

- **输入:** `lists = [[1,4,5],[1,3,4],[2,6]]`
- **输出:** `[1,1,2,3,4,4,5,6]`
- **解释:** 链表数组如下：

  ```
  [
    1->4->5,
    1->3->4,
    2->6
  ]
  ```

  将它们合并到一个有序链表中得到。

  `1->1->2->3->4->4->5->6`

**示例 2:**

- **输入:** `lists = []`
- **输出:** `[]`

**示例 3:**

- **输入:** `lists = [[]]`
- **输出:** `[]`

**提示:**

- `k == lists.length`
- `0 <= k <= 10^4`
- `0 <= lists[i].length <= 500`
- `-10^4 <= lists[i][j] <= 10^4`
- `lists[i]` 按 **升序** 排列
- `lists[i].length` 的总和不超过 `10^4`

## 题解

### 这题真正难的地方，不是“合并”，而是“怎么选”

如果题目只给两个升序链表，那这题其实就是最基础的“合并两个有序链表”。

难点在于现在不是 2 个，而是 `k` 个。

这时候问题就变了。  
真正要解决的，不再是“两个链表怎么接”，而是：

**在这 `k` 个链表当前的头节点里，怎么快速找到最小的那个。**

因为最终结果一定是从小到大依次接起来的。  
而每一步我们真正需要做的事情，其实只有一件：

**从所有还没处理完的链表里，挑出当前最小的节点。**

只要这件事能高效完成，后面的拼接反而很直接。

### 为什么最容易想到的是小顶堆

假设现在每个链表都还有若干节点没处理完。

对于每个链表来说，当前能参与竞争的，只有它的头节点。  
因为链表本身已经有序，头节点后面的元素只会更大，不可能先于头节点进入答案。

所以任意时刻，我们真正关心的只是：

- 链表 1 当前头节点
- 链表 2 当前头节点
- ...
- 链表 k 当前头节点

然后从这些节点里选最小的。

这个需求和小顶堆非常吻合，因为小顶堆最擅长做的事情就是：

**维护一批元素，并且随时快速取出其中最小的那个。**

所以这题最自然的主解法就是优先队列，也就是小顶堆。

### 小顶堆解法到底在做什么

思路其实很顺：

1. 先把所有非空链表的头节点放进小顶堆
2. 此时堆顶就是当前所有候选节点里最小的那个
3. 取出堆顶，把它接到结果链表后面
4. 如果这个节点后面还有下一个节点，就把它的下一个节点重新放回堆里
5. 重复这个过程，直到堆空为止

你会发现，这个过程很像“多路归并”。

因为每条链表都像一条已经排好序的小路，而堆负责帮我们在这些小路的当前入口里，永远选出最小的那个。

### 为什么每次只需要把“弹出节点的下一个”放进堆

这个点很关键。

假设某个链表当前头节点被弹出了，说明它已经进入答案了。  
那这个链表里，下一个有资格参与比较的，自然就是它的 `next` 节点。

至于这个链表更后面的节点，现在根本不用管。  
因为它们一定不会比当前这个 `next` 更早进入最终答案。

所以对每条链表来说，堆里始终只需要维护“当前还没处理的最前面那个节点”。

这也是为什么堆的大小最多只会到 `k`，不会把所有节点一股脑都塞进去。

### 用示例看一遍

比如：

```text
lists = [
  1->4->5,
  1->3->4,
  2->6
]
```

初始化时，把三个头节点放进堆里：

```text
1, 1, 2
```

堆顶最小是 `1`，弹出来，接到答案后面。  
这个 `1` 来自第一个链表，所以把它的下一个节点 `4` 再放回堆里。

此时堆里变成：

```text
1, 2, 4
```

再弹出最小的 `1`，接到答案后面。  
它来自第二个链表，所以再把 `3` 放进堆。

堆里变成：

```text
2, 3, 4
```

继续这样做，最终结果就是：

```text
1 -> 1 -> 2 -> 3 -> 4 -> 4 -> 5 -> 6
```

整个过程其实非常机械，但逻辑很清楚：  
**每一步都只做“取最小 + 补下一个”这两件事。**

### 代码

```java
public ListNode mergeKLists(ListNode[] lists) {
    if (lists == null || lists.length == 0) {
        return null;
    }

    PriorityQueue<ListNode> pq = new PriorityQueue<>((a, b) -> a.val - b.val);

    for (ListNode head : lists) {
        if (head != null) {
            pq.offer(head);
        }
    }

    ListNode dummy = new ListNode(0);
    ListNode tail = dummy;

    while (!pq.isEmpty()) {
        ListNode minNode = pq.poll();
        tail.next = minNode;
        tail = tail.next;

        if (minNode.next != null) {
            pq.offer(minNode.next);
        }
    }

    return dummy.next;
}
```

### 这段代码最关键的地方

真正关键的其实就两步。

第一步是初始化：

```java
for (ListNode head : lists) {
    if (head != null) {
        pq.offer(head);
    }
}
```

这表示：  
每个链表先只把当前头节点放进去，作为第一轮候选。

第二步是循环里的这段：

```java
ListNode minNode = pq.poll();
...
if (minNode.next != null) {
    pq.offer(minNode.next);
}
```

它表示：  
谁最小，就先把谁接到答案里；然后让这个链表的下一位顶上来，继续参加比较。

所以这题的本质不是“把 k 个链表一次性合在一起”，而是：

**维护 k 个当前候选节点，反复选最小。**

### 为什么时间复杂度是 `O(N log k)`

设所有链表节点总数是 `N`，链表个数是 `k`。

每个节点都会：

- 入堆一次
- 出堆一次

而堆的大小最多是 `k`，所以每次入堆、出堆的代价都是 `O(log k)`。

因此总时间复杂度就是：

- 时间复杂度：$O(N \log k)$

这比“每次都在线性扫描 k 个头节点里找最小”要好得多。  
因为如果线性扫，每取一个节点都要花 `O(k)`，总复杂度会变成 `O(Nk)`，明显更差。

### 分治法为什么也值得会

除了小顶堆，这题还有一种非常经典的思路，就是分治。

它的核心想法不是“同时维护 k 个候选头节点”，而是：

**把 k 个链表两两合并，逐步减少链表数量。**

比如原来有 8 个链表：

- 第一轮合并成 4 个
- 第二轮合并成 2 个
- 第三轮合并成 1 个

这和归并排序的结构很像，所以整体复杂度也能做到：

- 时间复杂度：$O(N \log k)$

为什么它比“从前往后一个一个并”更好？

因为如果你总是拿当前结果去和下一个链表合并，那么随着结果链表越来越长，后面会反复遍历很长的链表，效率不够均衡。

而分治的好处就在于：  
每一轮都是规模差不多的链表在互相合并，整体更均匀。

### 分治法的代码逻辑

```java
public ListNode mergeKLists(ListNode[] lists) {
    if (lists == null || lists.length == 0) return null;
    return merge(lists, 0, lists.length - 1);
}

private ListNode merge(ListNode[] lists, int left, int right) {
    if (left == right) return lists[left];

    int mid = left + (right - left) / 2;
    ListNode l1 = merge(lists, left, mid);
    ListNode l2 = merge(lists, mid + 1, right);

    return mergeTwoLists(l1, l2);
}

private ListNode mergeTwoLists(ListNode l1, ListNode l2) {
    ListNode dummy = new ListNode(0);
    ListNode curr = dummy;

    while (l1 != null && l2 != null) {
        if (l1.val <= l2.val) {
            curr.next = l1;
            l1 = l1.next;
        } else {
            curr.next = l2;
            l2 = l2.next;
        }
        curr = curr.next;
    }

    if (l1 != null) curr.next = l1;
    if (l2 != null) curr.next = l2;

    return dummy.next;
}
```

### 这题该怎么选写法

如果是正常做题或者面试，我会更推荐先写小顶堆版本。

原因很简单：

- 思路直接
- 代码不长
- 很符合“每次选最小”的直觉
- 复杂度也已经很好

分治法当然也很好，而且很优雅。  
但它更适合在你已经熟悉“合并两个有序链表”之后，再当成“多链表归并”的另一种思路去理解。

换句话说：

- **小顶堆** 更像是“动态维护最小值”
- **分治** 更像是“批量两两归并”

两者都值得会，但第一反应更容易想到的通常还是堆。

### 复杂度分析

小顶堆解法：

- 时间复杂度：$O(N \log k)$
- 空间复杂度：$O(k)$，因为堆里最多放 `k` 个节点

分治解法：

- 时间复杂度：$O(N \log k)$
- 空间复杂度：递归写法是 $O(\log k)$，如果写成迭代版可以更低

所以两者在时间上是一个量级，主要区别在实现方式和常数开销上。

### 这题真正值得记住的地方

这题最值得记住的，不是“合并 K 个链表背哪份模板”。

真正值得记住的是这个判断：

**当题目要求从多个有序序列里反复选当前最小值时，优先考虑小顶堆。**

而如果题目更适合批量配对合并，也可以考虑分治。

所以这题其实是在教你两种常见模式：

- **堆**：适合动态选当前最优元素
- **分治**：适合把多个同类子问题两两合并

这两个模式后面还会反复遇到。

### 完整代码

```java
import java.util.PriorityQueue;

class ListNode {
    int val;
    ListNode next;

    ListNode() {}

    ListNode(int val) {
        this.val = val;
    }

    ListNode(int val, ListNode next) {
        this.val = val;
        this.next = next;
    }
}

public class Main {

    public static ListNode mergeKLists(ListNode[] lists) {
        if (lists == null || lists.length == 0) {
            return null;
        }

        PriorityQueue<ListNode> pq = new PriorityQueue<>((a, b) -> a.val - b.val);

        for (ListNode node : lists) {
            if (node != null) {
                pq.offer(node);
            }
        }

        ListNode dummy = new ListNode(-1);
        ListNode curr = dummy;

        while (!pq.isEmpty()) {
            ListNode minNode = pq.poll();
            curr.next = minNode;
            curr = curr.next;

            if (minNode.next != null) {
                pq.offer(minNode.next);
            }
        }

        return dummy.next;
    }

    public static void printList(ListNode head) {
        while (head != null) {
            System.out.print(head.val);
            if (head.next != null) {
                System.out.print(" -> ");
            }
            head = head.next;
        }
        System.out.println();
    }

    public static ListNode buildList(int[] nums) {
        ListNode dummy = new ListNode(0);
        ListNode curr = dummy;

        for (int num : nums) {
            curr.next = new ListNode(num);
            curr = curr.next;
        }

        return dummy.next;
    }

    public static void main(String[] args) {
        ListNode l1 = buildList(new int[]{1, 4, 5});
        ListNode l2 = buildList(new int[]{1, 3, 4});
        ListNode l3 = buildList(new int[]{2, 6});
        ListNode[] lists1 = new ListNode[]{l1, l2, l3};

        System.out.println("示例 1 输入: [[1,4,5],[1,3,4],[2,6]]");
        ListNode result1 = mergeKLists(lists1);
        System.out.print("示例 1 输出: ");
        printList(result1);

        ListNode[] lists2 = new ListNode[]{};
        System.out.println("示例 2 输入: []");
        ListNode result2 = mergeKLists(lists2);
        System.out.print("示例 2 输出: ");
        printList(result2);

        ListNode[] lists3 = new ListNode[]{null};
        System.out.println("示例 3 输入: [[]]");
        ListNode result3 = mergeKLists(lists3);
        System.out.print("示例 3 输出: ");
        printList(result3);
    }
}
```
