## 题目

**描述：**

给你链表的头结点 `head`，请将其按 **升序** 排列并返回 **排序后的链表**。

**示例 1:**

![示例1图片](https://assets.leetcode.com/uploads/2020/09/14/sort_list_1.jpg)

- **输入:** `head = [4,2,1,3]`
- **输出:** `[1,2,3,4]`

**示例 2:**

![示例2图片](https://assets.leetcode.com/uploads/2020/09/14/sort_list_2.jpg)

- **输入:** `head = [-1,5,3,4,0]`
- **输出:** `[-1,0,3,4,5]`

**示例 3:**

- **输入:** `head = []`
- **输出:** `[]`

**提示:**

- 链表中节点的数目在范围 `[0, 5 * 10^4]` 内
- `-10^5 <= Node.val <= 10^5`

**进阶：** 你可以在 `O(n log n)` 时间复杂度和常数级空间复杂度下，对链表进行排序吗？

## 题解

### 先想清楚：为什么这题最适合归并排序

数组排序时，很多人会先想到快速排序。  
但这题给的是链表，不是数组。

链表和数组最大的区别在于：

- 数组支持随机访问，可以很方便地按下标交换元素
- 链表不支持随机访问，想访问中间位置必须一路往后走
- 但链表在“拆成两段”和“合并两段有序链表”这件事上反而很自然

所以这题虽然表面上只是“排序”，但一旦数据结构换成链表，最合适的选择其实就不是快排，而是归并排序。

归并排序正好非常契合链表的特点：

- 可以用快慢指针找中点，把链表一分为二
- 可以直接通过改 `next` 指针，把两个有序链表合并起来
- 整体时间复杂度稳定是 $O(n \log n)$

这也是为什么这题最标准的思路就是归并。

### 先说最容易理解的版本：递归归并排序

这题最好先从最直观的版本理解，也就是自顶向下的归并排序。

整个过程其实就三步：

1. 找到链表中点，把链表拆成左右两半
2. 递归地把左半部分和右半部分分别排好序
3. 把两个有序链表合并起来

你会发现，这和数组版归并排序的思想完全一样，只是“切分”和“合并”的方式换成了链表版本。

### 链表怎么找中点

这里用快慢指针是最自然的。

- 慢指针一次走一步
- 快指针一次走两步

当快指针走到末尾时，慢指针就走到了中间附近。

然后把慢指针后面的部分断开，就能把链表拆成两段。

比如：

```text
4 -> 2 -> 1 -> 3
```

拆开后会变成：

```text
4 -> 2
1 -> 3
```

接着再分别递归排序这两段。

### 为什么拆完之后就能递归

因为左半段和右半段本质上还是同一个问题：

**给你一个链表头节点，把这段链表排好序。**

这就是标准的分治结构。  
原问题被拆成了两个规模更小、但形式完全一样的子问题，所以递归就很自然。

### 合并这一步为什么在链表里特别顺手

归并排序最核心的操作就是合并两个有序序列。

对于数组来说，合并时往往需要额外空间。  
但对于链表，合并两个有序链表本来就是一个非常经典、非常顺手的操作：

- 用一个虚拟头节点 `dummy`
- 每次比较两个链表当前节点的值
- 谁小就把谁接到结果链表后面
- 最后把剩余部分直接接上

因为整个过程只是在改指针，所以非常适合链表。

### 递归版代码

```java
public ListNode sortList(ListNode head) {
    if (head == null || head.next == null) {
        return head;
    }

    ListNode slow = head;
    ListNode fast = head.next;

    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;
    }

    ListNode rightHead = slow.next;
    slow.next = null;

    ListNode left = sortList(head);
    ListNode right = sortList(rightHead);

    return merge(left, right);
}

private ListNode merge(ListNode l1, ListNode l2) {
    ListNode dummy = new ListNode(0);
    ListNode curr = dummy;

    while (l1 != null && l2 != null) {
        if (l1.val <= l2.val) {
            curr.next = l1;
            l1 = l1.next;
        } else {
            curr.next = l2;
            l2 = l2.next;
        }
        curr = curr.next;
    }

    if (l1 != null) curr.next = l1;
    if (l2 != null) curr.next = l2;

    return dummy.next;
}
```

### 这个版本已经很好了，但还差最后一步

上面的递归归并排序，时间复杂度已经达到了题目要求的 $O(n \log n)$。

但它还有一个问题：  
递归调用会占用栈空间，所以空间复杂度不是常数级，而是 $O(\log n)$。

这也是题目进阶要求真正卡你的地方：

> 你能不能在 `O(n log n)` 时间复杂度和常数级空间复杂度下完成排序？

如果要尽量贴近这个要求，就得把递归去掉，改成迭代版归并排序。

### 进阶要求对应的其实是：自底向上归并排序

递归版是“先不断拆，再往回合并”。  
那如果不想递归，我们就可以反过来想：

**既然最终一定是若干有序段不断合并变大，那能不能直接从最小段开始，迭代着往上合并？**

这就是自底向上的归并排序。

思路是这样的：

- 先把每个长度为 1 的链表看成已经有序
- 两两合并，得到长度为 2 的有序段
- 再把长度为 2 的有序段两两合并，得到长度为 4 的有序段
- 再变成 8、16……

直到整个链表合并成一个整体

也就是说，子链表长度 `subLength` 会这样增长：

```text
1 -> 2 -> 4 -> 8 -> ...
```

### 自底向上到底在做什么

假设链表是：

```text
4 -> 2 -> 1 -> 3
```

第一轮，`subLength = 1`。  
把链表看成：

```text
[4] [2] [1] [3]
```

两两合并后变成：

```text
[2 -> 4] [1 -> 3]
```

第二轮，`subLength = 2`。  
再把这两段合并：

```text
[1 -> 2 -> 3 -> 4]
```

排序完成。

你会发现，这个过程本质上还是归并排序，只不过以前是递归拆出来的有序段，现在是手动按长度一轮轮组织出来的。

### 这个版本最难的地方在哪里

真正难的不是“合并”，而是：

**怎么把链表按指定长度一段一段地切开，再把合并后的结果重新接回去。**

因为数组可以按下标直接分段，链表不行。  
所以你必须手动维护几个指针：

- 当前这一轮处理到哪里了
- 第一段从哪里开始
- 第二段从哪里开始
- 下一轮该从哪里继续
- 合并后的结果该接到哪里去

这也是为什么很多人第一次写自底向上归并排序时，会觉得代码明显比递归版绕一些。

### 为什么要先统计链表长度

因为我们要知道 `subLength` 什么时候停止翻倍。

如果链表长度是 `n`，那么当 `subLength >= n` 时，说明整条链表已经可以视为一个整体，排序自然就完成了。

所以在进入主循环之前，先扫一遍链表统计总长度，这是很正常的一步。

### 这题完整代码为什么更适合放进阶版

如果只是为了做对题，递归版已经够用了，而且更容易讲清楚。

但既然题目专门给了进阶要求，那完整代码更适合展示自底向上的版本。  
这样一来，读者既能先从递归版理解思路，也能顺着看到真正满足进阶要求的写法。

这比一上来就直接扔迭代版代码，更容易读懂。

### 复杂度分析

先看递归版：

- 时间复杂度：$O(n \log n)$
- 空间复杂度：$O(\log n)$，主要来自递归栈

再看自底向上版：

- 时间复杂度：$O(n \log n)$
- 空间复杂度：$O(1)$，不考虑结果本身占用的空间，只用了有限几个指针变量

所以两者时间一样，真正差别在空间上。

### 这题真正值得记住的地方

这题最值得记住的，不是“链表排序背哪份模板”。

真正值得记住的是下面这个判断：

**当题目要求对链表做排序，而且还希望达到 `O(n log n)`，优先考虑归并排序。**

因为链表非常适合做两件事：

- 用快慢指针找中点
- 合并两个有序链表

这两件事刚好就是归并排序最需要的。

而如果题目还进一步要求常数级额外空间，那就要意识到：

**递归版归并还不够，真正对应的是自底向上的迭代归并。**

### 完整代码

```java
public class Main {

    public static class ListNode {
        int val;
        ListNode next;

        ListNode() {}

        ListNode(int val) {
            this.val = val;
        }

        ListNode(int val, ListNode next) {
            this.val = val;
            this.next = next;
        }
    }

    public static ListNode sortList(ListNode head) {
        if (head == null) {
            return null;
        }

        int length = 0;
        ListNode node = head;
        while (node != null) {
            length++;
            node = node.next;
        }

        ListNode dummy = new ListNode(0, head);

        for (int subLength = 1; subLength < length; subLength <<= 1) {
            ListNode prev = dummy;
            ListNode curr = dummy.next;

            while (curr != null) {
                ListNode head1 = curr;
                for (int i = 1; i < subLength && curr.next != null; i++) {
                    curr = curr.next;
                }

                ListNode head2 = curr.next;
                curr.next = null;
                curr = head2;

                ListNode nextHead = null;
                if (curr != null) {
                    for (int i = 1; i < subLength && curr.next != null; i++) {
                        curr = curr.next;
                    }
                    nextHead = curr.next;
                    curr.next = null;
                }

                ListNode merged = merge(head1, head2);
                prev.next = merged;

                while (prev.next != null) {
                    prev = prev.next;
                }

                curr = nextHead;
            }
        }

        return dummy.next;
    }

    private static ListNode merge(ListNode l1, ListNode l2) {
        ListNode dummy = new ListNode(0);
        ListNode curr = dummy;

        while (l1 != null && l2 != null) {
            if (l1.val <= l2.val) {
                curr.next = l1;
                l1 = l1.next;
            } else {
                curr.next = l2;
                l2 = l2.next;
            }
            curr = curr.next;
        }

        if (l1 != null) {
            curr.next = l1;
        }
        if (l2 != null) {
            curr.next = l2;
        }

        return dummy.next;
    }

    public static void printList(ListNode head) {
        ListNode curr = head;
        while (curr != null) {
            System.out.print(curr.val);
            if (curr.next != null) {
                System.out.print(" -> ");
            }
            curr = curr.next;
        }
        System.out.println();
    }

    public static void main(String[] args) {
        ListNode head1 = new ListNode(4, new ListNode(2, new ListNode(1, new ListNode(3))));
        System.out.println("示例 1 输入:");
        printList(head1);
        ListNode sorted1 = sortList(head1);
        System.out.print("示例 1 输出: ");
        printList(sorted1);

        ListNode head2 = new ListNode(-1, new ListNode(5, new ListNode(3, new ListNode(4, new ListNode(0)))));
        System.out.println("示例 2 输入:");
        printList(head2);
        ListNode sorted2 = sortList(head2);
        System.out.print("示例 2 输出: ");
        printList(sorted2);
    }
}
```
