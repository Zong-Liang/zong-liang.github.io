## 题目

**描述：**

将一个给定字符串 `s` 根据给定的行数 `numRows` ，以从上往下、从左到右进行 Z 字形排列。

比如输入字符串为 `"PAYPALISHIRING"` 行数为 3 时，排列如下：

```
P   A   H   N
A P L S I I G
Y   I   R
```

之后，你的输出需要从左往右逐行读取，产生出一个新的字符串，比如：`"PAHNAPLSIIGYIR"`。

请你实现这个将字符串进行指定行数变换的函数：
`string convert(string s, int numRows);`

**示例 1:**

- **输入:** `s = "PAYPALISHIRING"`, `numRows = 3`
- **输出:** `"PAHNAPLSIIGYIR"`

**示例 2:**

- **输入:** `s = "PAYPALISHIRING"`, `numRows = 4`
- **输出:** `"PINALSIGYAHRPIR"`
- **解释:**
  ```
  P     I    N
  A   L S  I G
  Y A   H R
  P     I
  ```

**示例 3:**

- **输入:** `s = "A"`, `numRows = 1`
- **输出:** `"A"`

**提示:**

- `1 <= s.length <= 1000`
- `s` 由英文字母（小写和大写）、`,` 和 `.` 组成
- `1 <= numRows <= 1000`

## 题解

### 这题最容易想偏的地方，是一上来就去算坐标

很多人看到 Z 字形，会本能地去想：

- 每个字符最后落在第几行第几列
- 要不要先构造一个二维矩阵
- 读结果时是不是还得再按行扫描一遍

这样当然也能做，但没必要把题做得这么重。

因为题目真正要的，不是把这个图形完整画出来，而是：

**按照 Z 字形的书写顺序，把字符分配到不同的行里，最后再按行拼起来。**

换句话说，这题的关键不是“坐标”，而是“字符当前应该落到哪一行”。

### 先把这个过程想成“在几行之间来回走”

如果 `numRows = 3`，写字符的轨迹其实是这样的：

```text
第 0 行 -> 第 1 行 -> 第 2 行 -> 第 1 行 -> 第 0 行 -> ...
```

如果 `numRows = 4`，轨迹就是：

```text
第 0 行 -> 第 1 行 -> 第 2 行 -> 第 3 行 -> 第 2 行 -> 第 1 行 -> 第 0 行 -> ...
```

也就是说，行号并不是乱跳的，而是在：

- 到达最上面时开始往下走
- 到达最下面时开始往上走

这就是整个 Z 字形的本质。

所以这题最自然的做法不是建矩阵，而是：

**准备每一行的容器，然后按“向下 / 向上”的轨迹，把字符一个个放进去。**

### 为什么按行模拟就够了

因为最后答案本来就是“逐行读取”。

既然最终只关心每一行里有哪些字符，那我们完全可以在处理时直接维护：

- 第 0 行目前收到了哪些字符
- 第 1 行目前收到了哪些字符
- ...
- 第 `numRows - 1` 行目前收到了哪些字符

等所有字符都分配完，再把这些行从上到下拼起来，就是答案。

这一步其实省掉了很多没必要的信息。  
你不需要知道字符具体在第几列，也不需要真的画出空格，只需要知道它属于哪一行。

### 用示例 1 走一遍

看这个例子：

```text
s = "PAYPALISHIRING", numRows = 3
```

我们准备 3 行：

```text
row0
row1
row2
```

然后从第 0 行开始，方向先往下。

依次放字符：

- `P` 放到 `row0`
- `A` 放到 `row1`
- `Y` 放到 `row2`

走到最下面了，方向改成往上：

- `P` 放到 `row1`
- `A` 放到 `row0`

走到最上面了，再改成往下：

- `L` 放到 `row1`
- `I` 放到 `row2`
- `S` 放到 `row1`
- `H` 放到 `row0`
- ...

最后三行会变成：

```text
row0: PAHN
row1: APLSIIG
row2: YIR
```

按行拼接后得到：

```text
PAHNAPLSIIGYIR
```

这正是答案。

### 再看 `numRows = 4` 时为什么也一样处理

如果是：

```text
s = "PAYPALISHIRING", numRows = 4
```

轨迹会变成：

```text
0 -> 1 -> 2 -> 3 -> 2 -> 1 -> 0 -> ...
```

整个变化只是“来回摆动的范围”从 3 行变成了 4 行，处理方式完全没变。

最后四行分别会收集到：

```text
row0: PIN
row1: ALSIG
row2: YAHR
row3: PI
```

拼起来就是：

```text
PINALSIGYAHRPIR
```

所以这题真正稳定的部分，不是具体例子里的形状，而是这个“行号上下折返”的过程。

### 为什么 `numRows = 1` 要单独处理

这是这题最容易忽略的边界。

如果只有 1 行，所谓 Z 字形其实根本不存在折返，字符串就只能原样排开：

```text
A -> A
HELLO -> HELLO
```

这时候如果还按“向下 / 向上”去模拟，方向逻辑就没有意义了，甚至容易把下标写乱。

所以只要：

```java
numRows == 1
```

或者更宽一点写成：

```java
numRows < 2
```

直接返回原字符串就行。

### 实现时只需要两个状态

整题实现上，核心只需要维护两个东西：

- `currentRow`：当前字符应该放到哪一行
- `direction`：当前是往下走，还是往上走

每处理一个字符：

1. 先把它放进 `currentRow` 对应的行
2. 如果当前已经在第一行或最后一行，就反转方向
3. 按新的方向移动到下一行

所以整题其实就是一个非常标准的“状态模拟”。

### 代码

```java
public String convert(String s, int numRows) {
    if (numRows < 2) {
        return s;
    }

    List<StringBuilder> rows = new ArrayList<>();
    for (int i = 0; i < numRows; i++) {
        rows.add(new StringBuilder());
    }

    int currentRow = 0;
    int direction = -1;

    for (char c : s.toCharArray()) {
        rows.get(currentRow).append(c);

        if (currentRow == 0 || currentRow == numRows - 1) {
            direction = -direction;
        }

        currentRow += direction;
    }

    StringBuilder res = new StringBuilder();
    for (StringBuilder row : rows) {
        res.append(row);
    }

    return res.toString();
}
```

时间复杂度是 $O(n)$，因为每个字符只处理一次，最后拼接各行也是线性的。  
空间复杂度是 $O(n)$，因为所有字符都会被存到这些行容器里。

### 这题真正值得记住的地方

这题最值得记住的，不是“Z 字形怎么画”，而是这种处理方式：

**当题目表面上像是在构造一个二维图形，但最终只关心某种读取顺序时，先想一想能不能直接按读取目标去维护结果，而不是把整个图形真的建出来。**

这题里：

- 最终目标是按行读取
- 所以处理中直接维护每一行的内容就够了
- Z 字形本身，只体现在“当前行号如何上下折返”这件事上

以后再遇到这种“过程像二维，结果只看一维输出”的题，也可以优先往这个方向想。

### 完整代码

```java
import java.util.ArrayList;
import java.util.List;

public class Main {

    public static String convert(String s, int numRows) {
        if (numRows < 2) {
            return s;
        }

        List<StringBuilder> rows = new ArrayList<>();
        for (int i = 0; i < numRows; i++) {
            rows.add(new StringBuilder());
        }

        int currentRow = 0;
        int direction = -1;

        for (char c : s.toCharArray()) {
            rows.get(currentRow).append(c);

            if (currentRow == 0 || currentRow == numRows - 1) {
                direction = -direction;
            }

            currentRow += direction;
        }

        StringBuilder res = new StringBuilder();
        for (StringBuilder row : rows) {
            res.append(row);
        }

        return res.toString();
    }

    public static void main(String[] args) {
        String s1 = "PAYPALISHIRING";
        int numRows1 = 3;

        System.out.println("示例 1 输入: s = \"" + s1 + "\", numRows = " + numRows1);
        System.out.println("示例 1 输出: \"" + convert(s1, numRows1) + "\"");

        String s2 = "PAYPALISHIRING";
        int numRows2 = 4;

        System.out.println("示例 2 输入: s = \"" + s2 + "\", numRows = " + numRows2);
        System.out.println("示例 2 输出: \"" + convert(s2, numRows2) + "\"");

        String s3 = "A";
        int numRows3 = 1;

        System.out.println("示例 3 输入: s = \"" + s3 + "\", numRows = " + numRows3);
        System.out.println("示例 3 输出: \"" + convert(s3, numRows3) + "\"");
    }
}
```
