## 题目

**描述：**

将一个给定字符串 `s` 根据给定的行数 `numRows` ，以从上往下、从左到右进行 Z 字形排列。

比如输入字符串为 `"PAYPALISHIRING"` 行数为 3 时，排列如下：

```
P   A   H   N
A P L S I I G
Y   I   R
```

之后，你的输出需要从左往右逐行读取，产生出一个新的字符串，比如：`"PAHNAPLSIIGYIR"`。

请你实现这个将字符串进行指定行数变换的函数：
`string convert(string s, int numRows);`

**示例 1:**

- **输入:** `s = "PAYPALISHIRING"`, `numRows = 3`
- **输出:** `"PAHNAPLSIIGYIR"`

**示例 2:**

- **输入:** `s = "PAYPALISHIRING"`, `numRows = 4`
- **输出:** `"PINALSIGYAHRPIR"`
- **解释:**
  ```
  P     I    N
  A   L S  I G
  Y A   H R
  P     I
  ```

**示例 3:**

- **输入:** `s = "A"`, `numRows = 1`
- **输出:** `"A"`

**提示:**

- `1 <= s.length <= 1000`
- `s` 由英文字母（小写和大写）、`,` 和 `.` 组成
- `1 <= numRows <= 1000`

## 题解

这题第一眼很像“把字符摆到一个二维坐标里，再按行读出来”，真写起来其实没必要这么重。题目最后只关心按行拼接出来的结果，并不关心每个字符具体落在第几列，所以更自然的想法是：别去画整张矩阵，直接维护每一行收到了哪些字符。

关键在于先把行号的变化看清楚。以 `numRows = 4` 为例，字符落行的顺序其实很规律：`0 -> 1 -> 2 -> 3 -> 2 -> 1 -> 0 -> ...`。它本质上就是在最上面和最下面之间来回折返。于是遍历字符串时，我们只要记住两件事：当前字符该放到哪一行，以及下一步是往下走还是往上走。

拿 `s = "PAYPALISHIRING"`、`numRows = 3` 走一遍会很直观。开始时字符依次落在第 `0`、`1`、`2` 行；到达最下面以后，方向掉头，接下来的字符就落在第 `1`、`0` 行；到顶后再掉头，继续往下。这样一路分配完，三行里分别会积累出 `PAHN`、`APLSIIG`、`YIR`，最后把三行从上到下拼起来，就是 `PAHNAPLSIIGYIR`。

这样做之所以够用，是因为题目最终要的就是“逐行读取”的结果。既然如此，处理中直接让每一行自己把该收的字符收好，最后再统一拼接就行。字符具体处在第几列、斜着那一段中间隔了多少空位，其实都不需要真的存下来。

还有一个边界最好早点拎出来：如果 `numRows < 2`，Z 字形根本不会发生折返，字符串只能原样排开，这时直接返回原字符串即可。把这个特判写在最前面，后面的方向切换就会干净很多。

代码如下：

```java
public String convert(String s, int numRows) {
    if (numRows < 2) {
        return s;
    }

    List<StringBuilder> rows = new ArrayList<>();
    for (int i = 0; i < numRows; i++) {
        rows.add(new StringBuilder());
    }

    int currentRow = 0;
    boolean goingDown = false;

    for (char c : s.toCharArray()) {
        rows.get(currentRow).append(c);

        if (currentRow == 0 || currentRow == numRows - 1) {
            goingDown = !goingDown;
        }

        currentRow += goingDown ? 1 : -1;
    }

    StringBuilder res = new StringBuilder();
    for (StringBuilder row : rows) {
        res.append(row);
    }

    return res.toString();
}
```

时间复杂度是 `O(n)`，因为每个字符只会被处理一次；空间复杂度是 `O(n)`，因为所有字符最终都会被存进这些行容器里。

这题比较值得带走的，不一定是 Z 字形本身，而是这个处理角度：当题目表面上像是在构造二维图形，但最后只关心某种分组后的输出时，可以先想想能不能直接维护那组结果，而不是把整张图老老实实画出来。

完整代码如下：

```java
import java.util.ArrayList;
import java.util.List;

public class Main {

    public static String convert(String s, int numRows) {
        if (numRows < 2) {
            return s;
        }

        List<StringBuilder> rows = new ArrayList<>();
        for (int i = 0; i < numRows; i++) {
            rows.add(new StringBuilder());
        }

        int currentRow = 0;
        boolean goingDown = false;

        for (char c : s.toCharArray()) {
            rows.get(currentRow).append(c);

            if (currentRow == 0 || currentRow == numRows - 1) {
                goingDown = !goingDown;
            }

            currentRow += goingDown ? 1 : -1;
        }

        StringBuilder res = new StringBuilder();
        for (StringBuilder row : rows) {
            res.append(row);
        }

        return res.toString();
    }

    public static void main(String[] args) {
        String s1 = "PAYPALISHIRING";
        int numRows1 = 3;

        System.out.println("示例 1 输入: s = \"" + s1 + "\", numRows = " + numRows1);
        System.out.println("示例 1 输出: \"" + convert(s1, numRows1) + "\"");

        String s2 = "PAYPALISHIRING";
        int numRows2 = 4;

        System.out.println("示例 2 输入: s = \"" + s2 + "\", numRows = " + numRows2);
        System.out.println("示例 2 输出: \"" + convert(s2, numRows2) + "\"");

        String s3 = "A";
        int numRows3 = 1;

        System.out.println("示例 3 输入: s = \"" + s3 + "\", numRows = " + numRows3);
        System.out.println("示例 3 输出: \"" + convert(s3, numRows3) + "\"");
    }
}
```
