## 题目

**描述：**

给定一个整数数组 `nums`，将数组中的元素向右轮转 `k` 个位置，其中 `k` 是非负数。

**示例 1:**

- **输入:** `nums = [1,2,3,4,5,6,7], k = 3`
- **输出:** `[5,6,7,1,2,3,4]`
- **解释:**

  向右轮转 1 步: `[7,1,2,3,4,5,6]`

  向右轮转 2 步: `[6,7,1,2,3,4,5]`

  向右轮转 3 步: `[5,6,7,1,2,3,4]`

**示例 2:**

- **输入:** `nums = [-1,-100,3,99], k = 2`
- **输出:** `[3,99,-1,-100]`
- **解释:**

  向右轮转 1 步: `[99,-1,-100,3]`

  向右轮转 2 步: `[3,99,-1,-100]`

**提示:**

- `1 <= nums.length <= 10^5`
- `-2^31 <= nums[i] <= 2^31 - 1`
- `0 <= k <= 10^5`

**进阶:**

- 尽可能想出更多的解决方案，至少有 **三种** 不同的方法可以解决这个问题。
- 你可以使用空间复杂度为 `O(1)` 的 **原地** 算法解决这个问题吗？

## 题解

这题如果按题面去理解，很容易把自己带到“向右挪 `k` 次”这个方向里。其实更自然的理解是：把数组最后 `k` 个元素搬到前面，再把前面的 `n-k` 个元素接到后面。拿 `nums = [1,2,3,4,5,6,7]`、`k = 3` 来看，目标其实就是把 `[5,6,7]` 放到前面，再把 `[1,2,3,4]` 接到后面，结果变成 `[5,6,7,1,2,3,4]`。

最直观的做法是开一个新数组，把每个元素直接放到轮转后该去的位置。如果原来 `nums[i]` 在位置 `i`，那轮转后的新位置就是 `(i + k) % n`。这里 `% n` 很关键，因为数组是环状的，越过末尾之后要重新回到开头。

```java
public void rotate(int[] nums, int k) {
    int n = nums.length;
    k %= n;

    int[] newArr = new int[n];
    for (int i = 0; i < n; i++) {
        newArr[(i + k) % n] = nums[i];
    }

    System.arraycopy(newArr, 0, nums, 0, n);
}
```

这个做法不难写，也很直观，但它多用了一个长度为 `n` 的数组，空间复杂度是 $O(n)$。题目既然追问能不能用 $O(1)$ 的额外空间原地完成，就说明还得往下想一步。

原地做法里，环状替换当然可以，但第一次写时比较容易把自己绕进去。对这题来说，更顺手也更好记的办法其实是三次翻转。思路不复杂：既然最终结果是“后半段跑到前面，前半段跑到后面”，那就先把整个数组翻过来，再分别把前后两段各自翻回来。

还是用 `nums = [1,2,3,4,5,6,7]`、`k = 3` 这个例子。先翻转整个数组，得到 `[7,6,5,4,3,2,1]`；再翻转前 `3` 个元素，得到 `[5,6,7,4,3,2,1]`；最后翻转后面的 `4` 个元素，结果就是 `[5,6,7,1,2,3,4]`。之所以成立，是因为整体翻转之后，原来最后那一段确实已经跑到了前面，只是内部顺序反了；原来前面的那一段也跑到了后面，内部顺序同样反了。再把这两段各自翻转一次，顺序就恢复正常了。

这题还有一个很容易漏掉的小点：`k` 可能比数组长度大。比如数组长度是 `7`，`k = 10`，其实和轮转 `3` 位是一样的，因为转满一圈之后数组会回到原样。所以一开始一定要先做 `k %= n`。

代码如下：

```java
public void rotate(int[] nums, int k) {
    int n = nums.length;
    k %= n;

    reverse(nums, 0, n - 1);
    reverse(nums, 0, k - 1);
    reverse(nums, k, n - 1);
}

private void reverse(int[] nums, int left, int right) {
    while (left < right) {
        int temp = nums[left];
        nums[left] = nums[right];
        nums[right] = temp;
        left++;
        right--;
    }
}
```

时间复杂度是 $O(n)$，因为三次翻转加起来仍然只是线性级别；空间复杂度是 $O(1)$，整个过程都在原数组上完成。

这题比较适合顺手记住的是这个转换视角：有些看起来像“循环移动”的题，直接去挪每个元素会很绕，但如果换个角度把它看成“几段连续区间的重组”，写法往往会干净很多。

完整代码如下：

```java
import java.util.Arrays;

public class Main {

    public static void rotate(int[] nums, int k) {
        int n = nums.length;
        k %= n;

        reverse(nums, 0, n - 1);
        reverse(nums, 0, k - 1);
        reverse(nums, k, n - 1);
    }

    private static void reverse(int[] nums, int left, int right) {
        while (left < right) {
            int temp = nums[left];
            nums[left] = nums[right];
            nums[right] = temp;
            left++;
            right--;
        }
    }

    public static void main(String[] args) {
        int[] nums1 = new int[]{1, 2, 3, 4, 5, 6, 7};
        int k1 = 3;

        System.out.println("示例 1 输入:");
        System.out.println("nums = " + Arrays.toString(nums1) + ", k = " + k1);

        rotate(nums1, k1);

        System.out.println("示例 1 输出: " + Arrays.toString(nums1));

        int[] nums2 = new int[]{-1, -100, 3, 99};
        int k2 = 2;

        System.out.println("示例 2 输入:");
        System.out.println("nums = " + Arrays.toString(nums2) + ", k = " + k2);

        rotate(nums2, k2);

        System.out.println("示例 2 输出: " + Arrays.toString(nums2));
    }
}
```
