## 题目

**描述：**

给定一个整数数组 `nums`，将数组中的元素向右轮转 `k` 个位置，其中 `k` 是非负数。

**示例 1:**

- **输入:** `nums = [1,2,3,4,5,6,7], k = 3`
- **输出:** `[5,6,7,1,2,3,4]`
- **解释:**

  向右轮转 1 步: `[7,1,2,3,4,5,6]`

  向右轮转 2 步: `[6,7,1,2,3,4,5]`

  向右轮转 3 步: `[5,6,7,1,2,3,4]`

**示例 2:**

- **输入:** `nums = [-1,-100,3,99], k = 2`
- **输出:** `[3,99,-1,-100]`
- **解释:**

  向右轮转 1 步: `[99,-1,-100,3]`

  向右轮转 2 步: `[3,99,-1,-100]`

**提示:**

- `1 <= nums.length <= 10^5`
- `-2^31 <= nums[i] <= 2^31 - 1`
- `0 <= k <= 10^5`

**进阶:**

- 尽可能想出更多的解决方案，至少有 **三种** 不同的方法可以解决这个问题。
- 你可以使用空间复杂度为 `O(1)` 的 **原地** 算法解决这个问题吗？

## 题解

### 先把“轮转”这件事想清楚

这题说的是把数组向右轮转 `k` 个位置。

如果拿 `nums = [1,2,3,4,5,6,7]`、`k = 3` 来看，其实就是把最后 `3` 个元素：

```text
[5,6,7]
```

挪到最前面，再把前面的部分：

```text
[1,2,3,4]
```

接到后面，结果变成：

```text
[5,6,7,1,2,3,4]
```

所以这题本质上不是“一个一个往右挪 `k` 次”，而是：

**把数组切成两段，再把后半段搬到前面。**

一旦这么看，后面的几种做法就都好理解了。

### 先说最直观的做法

最容易想到的办法，就是开一个新数组，把每个元素直接放到轮转后应该去的位置。

如果原来 `nums[i]` 在下标 `i`，向右轮转 `k` 位之后，它的新位置就是：

```java
(i + k) % n
```

这里 `% n` 很关键，因为数组是环状的，超过末尾之后要重新回到开头。

所以做法就是：

- 遍历原数组
- 把 `nums[i]` 放到 `newArr[(i + k) % n]`
- 最后再把新数组拷回原数组

代码很好写：

```java
public void rotate(int[] nums, int k) {
    int n = nums.length;
    k %= n;

    int[] newArr = new int[n];
    for (int i = 0; i < n; i++) {
        newArr[(i + k) % n] = nums[i];
    }

    System.arraycopy(newArr, 0, nums, 0, n);
}
```

这个方法的优点是直观，不容易写错。  
缺点也很明显：多用了一个长度为 `n` 的数组，空间复杂度是 $O(n)$。

如果只是要求做出来，这个方法已经够用了；但题目后面专门问了能不能用 $O(1)$ 额外空间原地完成，所以还得继续往下看。

### 原地做，最先想到的通常是“挪位置”

既然不能开新数组，那很自然就会想到：能不能直接在原数组里，把元素一个一个挪到它该去的位置？

答案是可以，这就是环状替换。

比如某个元素当前位置是 `i`，轮转后它应该去 `(i + k) % n`。  
那我们就把它放过去，同时把目标位置原来的值先存起来，再继续处理下一个位置。

这样会形成一个“环”：

- 当前元素去下一个位置
- 下一个位置原来的元素再去它自己的目标位置
- 一直绕，直到回到起点

听起来是能做的，但这题麻烦就麻烦在这里：  
**并不是从 0 出发一次就一定能覆盖整个数组。**

有时候转一圈只是覆盖了一部分位置，还会留下别的环没走到，所以需要从新的起点继续补。

这也是这个写法为什么不算难想，但写起来容易绕。

代码如下：

```java
public void rotate(int[] nums, int k) {
    int n = nums.length;
    k %= n;
    int count = 0;

    for (int start = 0; count < n; start++) {
        int current = start;
        int prev = nums[start];

        do {
            int next = (current + k) % n;
            int temp = nums[next];
            nums[next] = prev;
            prev = temp;
            current = next;
            count++;
        } while (current != start);
    }
}
```

这个方法时间复杂度是 $O(n)$，空间复杂度是 $O(1)$`。  
它是原地算法，但不算最适合第一次接触这题时记忆的写法。

### 这题真正值得记的是三次翻转

这题最常用、也最值得记住的解法，其实是三次翻转。

它看起来有点巧，但一旦看懂，之后非常顺手。

还是拿这个例子：

```text
nums = [1,2,3,4,5,6,7], k = 3
```

我们想得到的结果是：

```text
[5,6,7,1,2,3,4]
```

注意这个结果其实可以看成两段：

- 前面是原数组最后 `k` 个元素
- 后面是原数组前 `n-k` 个元素

那怎么原地把这两段换过来？

办法是：

1. 先翻转整个数组
2. 再翻转前 `k` 个元素
3. 最后翻转后 `n-k` 个元素

我们实际走一遍：

原数组：

```text
[1,2,3,4,5,6,7]
```

整体翻转之后：

```text
[7,6,5,4,3,2,1]
```

翻转前 `k = 3` 个元素：

```text
[5,6,7,4,3,2,1]
```

再翻转后面的部分：

```text
[5,6,7,1,2,3,4]
```

结果就出来了。

### 为什么三次翻转能成立

关键就在于：  
整体翻转之后，原来最后的那一段已经跑到了前面，只不过它内部顺序反了；原来前面的那一段跑到了后面，内部顺序也反了。

所以再分别把这两段各自翻转一次，它们内部顺序就恢复正常了，而两段的前后位置已经交换完成。

这个思路一旦想通，代码其实非常短。

### 别忘了先处理 `k`

这题还有一个很容易忽略的小点：`k` 可能比数组长度还大。

比如长度是 `7`，如果 `k = 10`，其实就等价于向右轮转 `3` 位。  
因为转满一圈之后，数组会回到原样。

所以一开始要先做：

```java
k %= n;
```

这一步不是细节，而是必须有。

### 代码

```java
public void rotate(int[] nums, int k) {
    int n = nums.length;
    k %= n;

    reverse(nums, 0, n - 1);
    reverse(nums, 0, k - 1);
    reverse(nums, k, n - 1);
}

private void reverse(int[] nums, int left, int right) {
    while (left < right) {
        int temp = nums[left];
        nums[left] = nums[right];
        nums[right] = temp;
        left++;
        right--;
    }
}
```

时间复杂度是 $O(n)$，因为三次翻转加起来仍然只是线性级别。  
空间复杂度是 $O(1)$，整个过程都在原数组上完成。

### 这题真正值得记住的地方

这题值得记住的，不只是“三次翻转”这个结论，而是这种转换思路：

**有些看起来像“循环移动”的题，直接模拟移动会很绕，但如果换个角度把它看成“区间重组”，问题就会简单很多。**

这题里：

- 向右轮转，本质上是在交换两段连续区间的位置
- 三次翻转，本质上是在原地完成这次区间重组

以后再遇到“数组旋转”“区间交换”“循环位移”这类题，都可以先想一想：  
这件事到底是在“移动元素”，还是其实在“重组区间”。

### 完整代码

```java
import java.util.Arrays;

public class Main {

    public static void rotate(int[] nums, int k) {
        int n = nums.length;
        k %= n;

        reverse(nums, 0, n - 1);
        reverse(nums, 0, k - 1);
        reverse(nums, k, n - 1);
    }

    private static void reverse(int[] nums, int left, int right) {
        while (left < right) {
            int temp = nums[left];
            nums[left] = nums[right];
            nums[right] = temp;
            left++;
            right--;
        }
    }

    public static void main(String[] args) {
        int[] nums1 = new int[]{1, 2, 3, 4, 5, 6, 7};
        int k1 = 3;

        System.out.println("示例 1 输入:");
        System.out.println("nums = " + Arrays.toString(nums1) + ", k = " + k1);

        rotate(nums1, k1);

        System.out.println("示例 1 输出: " + Arrays.toString(nums1));

        int[] nums2 = new int[]{-1, -100, 3, 99};
        int k2 = 2;

        System.out.println("示例 2 输入:");
        System.out.println("nums = " + Arrays.toString(nums2) + ", k = " + k2);

        rotate(nums2, k2);

        System.out.println("示例 2 输出: " + Arrays.toString(nums2));
    }
}
```
