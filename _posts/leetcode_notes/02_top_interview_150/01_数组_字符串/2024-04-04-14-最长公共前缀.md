## 题目

**描述：**

编写一个函数来查找字符串数组中的最长公共前缀。

如果不存在公共前缀，返回空字符串 `""`。

**示例 1:**

- **输入:** `strs = ["flower","flow","flight"]`
- **输出:** `"fl"`

**示例 2:**

- **输入:** `strs = ["dog","racecar","car"]`
- **输出:** `""`
- **解释:** 输入不存在公共前缀。

**提示:**

- `1 <= strs.length <= 200`
- `0 <= strs[i].length <= 200`
- `strs[i]` 如果非空，则仅由小写英文字母组成

## 题解

### 这题真正要想清楚的，不是“怎么找”，而是“公共前缀会怎么变短”

这题看起来像是在一堆字符串里找共同部分，但如果直接把注意力放在“找”上，反而容易绕。

更自然的想法其实是：

**公共前缀只会越来越短，不会越来越长。**

因为你每多比较一个字符串，原来的公共前缀要么还能保留，要么就得继续缩短，不可能反过来变得更长。

这个判断一旦立住，这题最顺手的写法就出来了。

### 最直接的做法：拿第一个字符串当初始答案

先假设第一个字符串就是公共前缀：

```text
prefix = strs[0]
```

然后从第二个字符串开始，一个一个去比。

如果当前字符串是以 `prefix` 开头的，说明目前这个前缀还能成立，那就继续看下一个字符串。  
如果不是，就把 `prefix` 砍短一点，再试一次。

一直缩，直到：

- 当前字符串确实以这个前缀开头
- 或者前缀已经缩成空串

这就是这题最核心的过程。

### 为什么这样做是对的

因为“最长公共前缀”本质上就是一个不断收缩的过程。

假设前两个字符串的公共前缀已经确定成了 `prefix`。  
那当你再拿这个 `prefix` 去和第三个字符串比较时，新的公共前缀只可能是：

- 还是原来的 `prefix`
- 或者是 `prefix` 的某个更短前缀

绝不可能出现一个比 `prefix` 更长的新答案。

所以整题其实不需要重新搜索，只需要不断把候选答案缩短到合法为止。

### 用示例 1 走一遍

看这个例子：

```text
strs = ["flower", "flow", "flight"]
```

先把：

```text
prefix = "flower"
```

拿去和 `"flow"` 比。

显然 `"flow"` 不是以 `"flower"` 开头的，所以前缀太长了，得缩短：

- `"flower"` -> `"flowe"`
- `"flowe"` 还是不行
- `"flow"` 可以

所以这一步之后：

```text
prefix = "flow"
```

接着拿 `"flow"` 去和 `"flight"` 比。

还是不匹配，再继续缩：

- `"flow"` -> `"flo"`
- `"flo"` 还是不行
- `"fl"` 可以

于是最后得到：

```text
"fl"
```

这就是答案。

### 示例 2 为什么会变成空串

再看：

```text
strs = ["dog", "racecar", "car"]
```

一开始：

```text
prefix = "dog"
```

拿去和 `"racecar"` 比，显然不匹配。  
于是一路缩：

- `"dog"` -> `"do"`
- `"do"` -> `"d"`
- `"d"` -> `""`

缩到空串之后，就说明根本不存在公共前缀了，直接返回空字符串。

这一点也正好说明了这题的本质：  
不是去“拼出”答案，而是不断排除不可能的部分。

### 为什么代码里会不断截短前缀

这题里最常见的一句代码是：

```java
prefix = prefix.substring(0, prefix.length() - 1);
```

它的含义很直接：

**既然当前前缀太长，导致和当前字符串对不上，那就把最后一个字符砍掉，再试。**

为什么每次只砍一个字符就够？

因为我们要找的是“最长”公共前缀。  
如果一下子砍太多，虽然也可能最终找到答案，但会跳过中间那些本来可能成立的更长前缀。

所以最稳的方式就是一点点缩，直到刚好能匹配。

### 这题还有一种常见思路，但不是最适合记的那种

有些题解会用排序法：

- 先把整个字符串数组排序
- 然后只比较排序后的第一个字符串和最后一个字符串

这个思路也对，因为排序后最不相似的两个字符串，往往正好卡住了公共前缀的上界。  
如果首尾两个字符串都有某个公共前缀，那中间的字符串也一定不会比它们更差。

但这题更适合记住的，还是“横向扫描”这套思路。  
因为它更贴近题目本身：前缀随着比较对象增加而不断收缩。

### 代码

```java
public String longestCommonPrefix(String[] strs) {
    if (strs == null || strs.length == 0) {
        return "";
    }

    String prefix = strs[0];

    for (int i = 1; i < strs.length; i++) {
        while (strs[i].indexOf(prefix) != 0) {
            prefix = prefix.substring(0, prefix.length() - 1);

            if (prefix.isEmpty()) {
                return "";
            }
        }
    }

    return prefix;
}
```

时间复杂度可以记作 $O(S)$，其中 $S$ 是所有字符串字符总数的量级。  
空间复杂度是 $O(1)$，不需要额外的复杂数据结构。

### 这题真正值得记住的地方

这题最值得记住的，不是“最长公共前缀用横向扫描”这句话，而是这种处理方式：

**当答案随着比较对象增加只会单调收缩时，就可以先假设一个最大候选，再不断把它缩短到合法。**

这题里：

- 初始候选是第一个字符串
- 每遇到一个新字符串，就把候选前缀缩到还能匹配为止
- 缩到最后剩下的，就是答案

以后再遇到这种“候选答案只会越来越短”的字符串题，也可以优先往这个方向想。

### 完整代码

```java
import java.util.Arrays;

public class Main {

    public static String longestCommonPrefix(String[] strs) {
        if (strs == null || strs.length == 0) {
            return "";
        }

        String prefix = strs[0];

        for (int i = 1; i < strs.length; i++) {
            while (strs[i].indexOf(prefix) != 0) {
                prefix = prefix.substring(0, prefix.length() - 1);

                if (prefix.isEmpty()) {
                    return "";
                }
            }
        }

        return prefix;
    }

    public static void main(String[] args) {
        String[] strs1 = new String[]{"flower", "flow", "flight"};
        System.out.println("示例 1 输入: " + Arrays.toString(strs1));
        System.out.println("示例 1 输出: \"" + longestCommonPrefix(strs1) + "\"");

        String[] strs2 = new String[]{"dog", "racecar", "car"};
        System.out.println("示例 2 输入: " + Arrays.toString(strs2));
        System.out.println("示例 2 输出: \"" + longestCommonPrefix(strs2) + "\"");

        String[] strs3 = new String[]{"ab", "a"};
        System.out.println("额外测试 输入: " + Arrays.toString(strs3));
        System.out.println("额外测试 输出: \"" + longestCommonPrefix(strs3) + "\"");
    }
}
```
