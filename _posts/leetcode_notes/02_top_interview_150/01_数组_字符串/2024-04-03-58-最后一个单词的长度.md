## 题目

**描述：**

给你一个字符串 `s`，由若干单词组成，单词前后用一些空格字符隔开。返回字符串中 **最后一个** 单词的长度。

**单词** 是指仅由字母组成、不包含任何空格字符的最大子字符串。

**示例 1:**

- **输入:** `s = "Hello World"`
- **输出:** 5
- **解释:** 最后一个单词是“World”，长度为 5。

**示例 2:**

- **输入:** `s = "   fly me   to   the moon  "`
- **输出:** 4
- **解释:** 最后一个单词是“moon”，长度为 4。

**示例 3:**

- **输入:** `s = "luffy is still joyboy"`
- **输出:** 6
- **解释:** 最后一个单词是长度为 6 的“joyboy”。

**提示:**

- `1 <= s.length <= 10^4`
- `s` 仅有英文字母和空格 `' '` 组成
- `s` 中至少存在一个单词

## 题解

### 这题其实不难，难的是别把它做重了

看到“最后一个单词的长度”，很多人第一反应会去想：

- 先 `trim()` 去掉首尾空格
- 再 `split(" ")` 切成若干段
- 最后取最后一个单词的长度

这个做法当然能做，但有点绕，也做重了。

因为题目真正只问一件事：

**最后一个单词有多长。**

既然只关心最后一个单词，那根本没必要把前面的所有单词都拆出来。  
最直接的办法，其实就是从字符串末尾往前找。

### 为什么从后往前最顺

题目里有个很容易忽略的小点：

**字符串末尾可能有空格。**

比如：

```text
"   fly me   to   the moon  "
```

最后一个单词明明是 `moon`，但字符串真正的结尾其实是空格。

所以如果从前往后处理，你还得考虑怎么定位“最后一个非空单词”。  
但如果从后往前看，事情就非常自然：

1.  先跳过结尾所有空格
2.  找到最后一个单词的最后一个字符
3.  继续往前数，直到遇到空格为止

数出来的长度，就是答案。

这道题最适合的思路，其实不是“切分字符串”，而是：

**直接从尾部往前扫描。**

### 先跳过末尾空格，再数单词长度

整个过程可以拆成两步。

第一步，先把结尾那些没用的空格跳过去。

比如：

```text
"Hello World   "
```

从最后一个字符开始看，会先遇到几个空格。  
这些空格不属于最后一个单词，所以直接一路往前跳。

跳完之后，指针会停在 `d` 上，也就是最后一个单词 `World` 的末尾。

第二步，从这个位置继续往前数：

- 只要当前不是空格，就说明还在最后一个单词里
- 每走一步，长度加一
- 一旦遇到空格，就说明这个单词结束了

这样就结束了。

### 用示例走一遍

先看：

```text
s = "Hello World"
```

从后往前：

- 最后一个字符是 `d`，不是空格，不需要跳
- 开始计数：`d -> l -> r -> o -> W`
- 再往前是空格，停止

长度是 `5`。

再看：

```text
s = "   fly me   to   the moon  "
```

从后往前：

- 先跳过最后两个空格
- 停在 `n` 上
- 开始计数：`n -> o -> o -> m`
- 再往前是空格，停止

长度是 `4`。

再看：

```text
s = "luffy is still joyboy"
```

末尾本来就没有空格，所以直接从最后一个字符开始数：

```text
joyboy
```

长度是 `6`。

### 这题为什么不需要额外处理别的情况

因为题目已经保证了：

- 字符串里至少有一个单词
- 只包含字母和空格

所以我们不用担心什么空字符串、全是空格之类的情况。  
在这个前提下，从后往前扫一定能找到最后一个单词。

也正因为条件已经给得很明确，所以最简单的线性扫描就是最合适的写法。

### 代码

```java
public int lengthOfLastWord(String s) {
    int index = s.length() - 1;

    while (index >= 0 && s.charAt(index) == ' ') {
        index--;
    }

    int length = 0;
    while (index >= 0 && s.charAt(index) != ' ') {
        length++;
        index--;
    }

    return length;
}
```

时间复杂度是 $O(n)$，最坏情况下要从后往前扫完整个字符串。  
空间复杂度是 $O(1)$，因为只用了常数个变量。

### 这题真正值得记住的地方

这题最值得记住的，不是“最后一个单词长度怎么写”，而是这种判断方式：

**如果题目只关心字符串末尾附近的信息，优先考虑从后往前扫。**

因为这样往往可以直接跳过很多无关内容，不需要额外切分，也不需要构造新字符串。

这题里：

- 末尾的空格可以直接跳过
- 最后一个单词可以直接原地计数

以后再遇到这种“只关心最后一段内容”的字符串题，也可以优先往这个方向想。

### 完整代码

```java
public class Main {

    public static int lengthOfLastWord(String s) {
        int index = s.length() - 1;

        while (index >= 0 && s.charAt(index) == ' ') {
            index--;
        }

        int length = 0;
        while (index >= 0 && s.charAt(index) != ' ') {
            length++;
            index--;
        }

        return length;
    }

    public static void main(String[] args) {
        String s1 = "Hello World";
        System.out.println("示例 1 输入: s = \"" + s1 + "\"");
        System.out.println("示例 1 输出: " + lengthOfLastWord(s1));

        String s2 = "   fly me   to   the moon  ";
        System.out.println("示例 2 输入: s = \"" + s2 + "\"");
        System.out.println("示例 2 输出: " + lengthOfLastWord(s2));

        String s3 = "luffy is still joyboy";
        System.out.println("示例 3 输入: s = \"" + s3 + "\"");
        System.out.println("示例 3 输出: " + lengthOfLastWord(s3));
    }
}
```
