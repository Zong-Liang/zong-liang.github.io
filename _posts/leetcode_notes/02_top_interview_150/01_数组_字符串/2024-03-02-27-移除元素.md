## 题目

**描述：**

给你一个数组 `nums` 和一个值 `val`，你需要 **原地** 移除所有数值等于 `val` 的元素。元素的顺序可能发生改变。然后返回 `nums` 中与 `val` 不同的元素的数量。

假设 `nums` 中不等于 `val` 的元素数量为 `k`，要通过此题，您需要执行以下操作：

- 更改 `nums` 数组，使 `nums` 的前 `k` 个元素包含不等于 `val` 的元素。
- 返回 `k`。

**用户评测:**

评测机将使用以下代码测试您的解决方案：

```java
int[] nums = [...]; // 输入数组
int val = ...; // 要移除的值
int[] expectedNums = [...]; // 长度正确的预期答案

int k = removeElement(nums, val); // 调用你的实现

assert k == expectedNums.length;
sort(nums, 0, k); // 排序 nums 的前 k 个元素
for (int i = 0; i < k; i++) {
    assert nums[i] == expectedNums[i];
}
```

**示例 1:**

- **输入:** `nums = [3,2,2,3]`, `val = 3`
- **输出:** 2, `nums = [2,2,_,_]`
- **解释:** 你的函数函数应该返回 `k = 2`, 并且 `nums` 中的前两个元素均为 2。

  你在返回的 k 个元素之外留下了什么并不重要（因此它们并不计入评测）。

**示例 2:**

- **输入:** `nums = [0,1,2,2,3,0,4,2]`, `val = 2`
- **输出:** 5, `nums = [0,1,4,0,3,_,_,_]`
- **解释:** 你的函数应该返回 `k = 5`，并且 `nums` 中的前五个元素为 `0,0,1,3,4`。

  注意这五个元素可以任意顺序返回。

  你在返回的 k 个元素之外留下了什么并不重要（因此它们并不计入评测）。

**提示:**

- `0 <= nums.length <= 100`
- `0 <= nums[i] <= 50`
- `0 <= val <= 100`

## 题解

这题表面上说的是“移除元素”，但题目并不要求你真的把数组删短。评测只会检查前 `k` 个位置里是不是都放着不等于 `val` 的元素，至于后面的内容是什么，其实并不重要。所以换个角度看，这题更像是在做一件事：把所有应该保留下来的元素原地压到数组前面，再返回这段有效区间的长度。

顺着这个目标去想，双指针就很自然了。`fast` 从左到右扫描整个数组，负责判断当前元素要不要留下；`slow` 指向下一个可以写入保留元素的位置。每当 `nums[fast] != val`，就把它写到 `nums[slow]`，然后让 `slow` 往后走一位；如果 `nums[fast] == val`，直接跳过即可。这样一趟扫下来，数组前 `slow` 个位置就是所有保留下来的元素。

这里真正需要想清楚的是 `slow` 的含义。它始终指向“下一个可写入的位置”，所以它前面的那一段天然就是已经整理好的有效区间。也正因为如此，遍历结束后直接返回 `slow` 就行，不需要再额外统计一次数量。

拿示例 1 看一下这个过程。`nums = [3,2,2,3]`，`val = 3`。一开始 `slow = 0`。`fast` 先看到第一个 `3`，因为它等于 `val`，所以直接跳过；接着看到 `2`，就把它写到 `nums[0]`，然后 `slow` 变成 `1`；再看到第二个 `2`，继续写到 `nums[1]`，`slow` 变成 `2`；最后一个 `3` 再次跳过。遍历结束时，数组前两位已经是 `[2,2]`，因此返回 `2` 即可。

代码如下：

```java
public int removeElement(int[] nums, int val) {
    int slow = 0;

    for (int fast = 0; fast < nums.length; fast++) {
        if (nums[fast] != val) {
            nums[slow] = nums[fast];
            slow++;
        }
    }

    return slow;
}
```

时间复杂度是 $O(n)$，因为整个数组只遍历了一遍；空间复杂度是 $O(1)$，因为额外用到的只有两个指针变量。

做这类题时，可以先问自己一句：题目到底是在“删除”，还是只是在“维护前面那段有效部分”？如果后半段不重要，往往就可以用一个指针扫描、另一个指针维护写入位置。

完整代码如下：

```java
import java.util.Arrays;

public class Main {

    public static int removeElement(int[] nums, int val) {
        int slow = 0;
        for (int fast = 0; fast < nums.length; fast++) {
            if (nums[fast] != val) {
                nums[slow] = nums[fast];
                slow++;
            }
        }
        return slow;
    }

    public static void main(String[] args) {
        int[] nums1 = new int[]{3, 2, 2, 3};
        int val1 = 3;

        System.out.println("示例 1 输入:");
        System.out.println("nums = " + Arrays.toString(nums1) + ", val = " + val1);

        int k1 = removeElement(nums1, val1);
        int[] result1 = Arrays.copyOfRange(nums1, 0, k1);

        System.out.println("示例 1 输出: " + k1 + ", nums = " + Arrays.toString(result1));
        System.out.println("-----------------------------");

        int[] nums2 = new int[]{0, 1, 2, 2, 3, 0, 4, 2};
        int val2 = 2;

        System.out.println("示例 2 输入:");
        System.out.println("nums = " + Arrays.toString(nums2) + ", val = " + val2);

        int k2 = removeElement(nums2, val2);
        int[] result2 = Arrays.copyOfRange(nums2, 0, k2);

        System.out.println("示例 2 输出: " + k2 + ", nums = " + Arrays.toString(result2));
    }
}
```
