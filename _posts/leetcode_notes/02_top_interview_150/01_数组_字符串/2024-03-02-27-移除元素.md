## 题目

**描述：**

给你一个数组 `nums` 和一个值 `val`，你需要 **原地** 移除所有数值等于 `val` 的元素。元素的顺序可能发生改变。然后返回 `nums` 中与 `val` 不同的元素的数量。

假设 `nums` 中不等于 `val` 的元素数量为 `k`，要通过此题，您需要执行以下操作：

- 更改 `nums` 数组，使 `nums` 的前 `k` 个元素包含不等于 `val` 的元素。
- 返回 `k`。

**用户评测:**

评测机将使用以下代码测试您的解决方案：

```java
int[] nums = [...]; // 输入数组
int val = ...; // 要移除的值
int[] expectedNums = [...]; // 长度正确的预期答案

int k = removeElement(nums, val); // 调用你的实现

assert k == expectedNums.length;
sort(nums, 0, k); // 排序 nums 的前 k 个元素
for (int i = 0; i < k; i++) {
    assert nums[i] == expectedNums[i];
}
```

**示例 1:**

- **输入:** `nums = [3,2,2,3]`, `val = 3`
- **输出:** 2, `nums = [2,2,_,_]`
- **解释:** 你的函数函数应该返回 `k = 2`, 并且 `nums` 中的前两个元素均为 2。

  你在返回的 k 个元素之外留下了什么并不重要（因此它们并不计入评测）。

**示例 2:**

- **输入:** `nums = [0,1,2,2,3,0,4,2]`, `val = 2`
- **输出:** 5, `nums = [0,1,4,0,3,_,_,_]`
- **解释:** 你的函数应该返回 `k = 5`，并且 `nums` 中的前五个元素为 `0,0,1,3,4`。

  注意这五个元素可以任意顺序返回。

  你在返回的 k 个元素之外留下了什么并不重要（因此它们并不计入评测）。

**提示:**

- `0 <= nums.length <= 100`
- `0 <= nums[i] <= 50`
- `0 <= val <= 100`

## 题解

### 先别急着想“删除”

这题看上去是在让你“移除元素”，但如果直接顺着“删除”这个词去想，其实很容易把自己绕进去。

题目真正关心的并不是数组后面变成什么样，而是：

- 把所有 **不等于** `val` 的元素放到数组前面
- 返回这些元素的个数 `k`

也就是说，这题并不要求你真的把数组长度变短，也不要求你把后面的内容处理得多干净。评测时只看前 `k` 个位置对不对，后面是什么并不重要。

所以换个角度看，这题本质上不是“删除”，而是：

**把所有要保留的元素，原地挪到前面去。**

### 想明白这一点，做法就很直接了

既然目标是把“不等于 `val` 的元素”收集到数组前面，那最自然的做法就是一边遍历，一边把需要保留的数往前放。

这里可以用两个指针：

- `fast` 负责从头到尾扫描整个数组
- `slow` 负责指向下一个应该放入保留元素的位置

遍历过程中：

- 如果 `nums[fast] == val`，说明这个元素要移除，直接跳过
- 如果 `nums[fast] != val`，说明这个元素要保留，就把它放到 `nums[slow]`，然后 `slow` 往后走一位

遍历结束后，`slow` 走了多少步，就说明前面一共留下了多少个有效元素。

### 两个指针到底在做什么

可以把它理解成这样：

- `fast` 像是在检查每个元素要不要留下
- `slow` 像是在维护“保留下来的部分”目前写到了哪里

只要遇到一个不该删的元素，就把它接到前面那段“有效区域”的末尾。

这样扫完整个数组之后，前 `slow` 个位置自然就是所有需要保留的元素。

这其实就是快慢指针最常见的一种用法：  
**一个负责找，一个负责放。**

### 用示例走一遍

以示例 1 为例：

```text
nums = [3,2,2,3], val = 3
```

初始时，`slow = 0`。

`fast = 0` 时，`nums[fast] = 3`，等于 `val`，跳过。

`fast = 1` 时，`nums[fast] = 2`，不等于 `val`，把它放到 `nums[slow]`：

```text
nums[0] = 2
```

然后 `slow++`，此时 `slow = 1`。

`fast = 2` 时，`nums[fast] = 2`，同样要保留，再放到 `nums[1]`：

```text
nums[1] = 2
```

然后 `slow++`，此时 `slow = 2`。

`fast = 3` 时，`nums[fast] = 3`，等于 `val`，跳过。

遍历结束后，`slow = 2`，说明保留下来的元素有 2 个，数组前两位就是结果：

```text
[2,2,_,_]
```

后面是什么不重要。

### 为什么返回 `slow`

因为 `slow` 始终指向“下一个可以写入的位置”。

换句话说，它前面已经放好的元素个数，恰好就是 `slow` 的值。

所以遍历结束后，直接返回 `slow` 就可以了，不需要再额外统计一次。

### 代码

```java
public int removeElement(int[] nums, int val) {
    int slow = 0;

    for (int fast = 0; fast < nums.length; fast++) {
        if (nums[fast] != val) {
            nums[slow] = nums[fast];
            slow++;
        }
    }

    return slow;
}
```

时间复杂度是 $O(n)$，因为整个数组只遍历了一遍。  
空间复杂度是 $O(1)$，因为只用了两个额外变量。

### 这题真正值得记住的地方

这题最值得记住的，不是“移除元素”这四个字，而是这种处理方式：

**当题目要求原地修改数组，又只关心某一段有效结果时，可以考虑用一个指针遍历、另一个指针维护有效区间。**

这类题的核心通常都不是“删”，而是“筛”和“压缩”。

以后如果再遇到“保留满足条件的元素到前面”“原地去重”“原地过滤”这种题，基本都可以往这个思路上靠。

### 完整代码

```java
import java.util.Arrays;

public class Main {

    public static int removeElement(int[] nums, int val) {
        int slow = 0;
        for (int fast = 0; fast < nums.length; fast++) {
            if (nums[fast] != val) {
                nums[slow] = nums[fast];
                slow++;
            }
        }
        return slow;
    }

    public static void main(String[] args) {
        int[] nums1 = new int[]{3, 2, 2, 3};
        int val1 = 3;

        System.out.println("示例 1 输入:");
        System.out.println("nums = " + Arrays.toString(nums1) + ", val = " + val1);

        int k1 = removeElement(nums1, val1);
        int[] result1 = Arrays.copyOfRange(nums1, 0, k1);

        System.out.println("示例 1 输出: " + k1 + ", nums = " + Arrays.toString(result1));
        System.out.println("-----------------------------");

        int[] nums2 = new int[]{0, 1, 2, 2, 3, 0, 4, 2};
        int val2 = 2;

        System.out.println("示例 2 输入:");
        System.out.println("nums = " + Arrays.toString(nums2) + ", val = " + val2);

        int k2 = removeElement(nums2, val2);
        int[] result2 = Arrays.copyOfRange(nums2, 0, k2);

        System.out.println("示例 2 输出: " + k2 + ", nums = " + Arrays.toString(result2));
    }
}
```
