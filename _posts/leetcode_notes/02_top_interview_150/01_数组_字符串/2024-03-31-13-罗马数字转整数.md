## 题目

**描述：**

罗马数字包含以下七种字符: `I`, `V`, `X`, `L`, `C`, `D` 和 `M`。

| 字符 | 数值 |
| :--- | :--- |
| I    | 1    |
| V    | 5    |
| X    | 10   |
| L    | 50   |
| C    | 100  |
| D    | 500  |
| M    | 1000 |

通常情况下，罗马数字中小的数字在大的数字的右边。但也存在特例，例如 4 不写做 `IIII`，而是 `IV`。这个特殊的规则只适用于以下六种情况：

- `I` 可以放在 `V` (5) 和 `X` (10) 的左边，来表示 4 和 9。
- `X` 可以放在 `L` (50) 和 `C` (100) 的左边，来表示 40 和 90。
- `C` 可以放在 `D` (500) 和 `M` (1000) 的左边，来表示 400 和 900。

给定一个罗马数字，将其转换成整数。

**示例 1:**

- **输入:** `s = "III"`
- **输出:** 3

**示例 2:**

- **输入:** `s = "IV"`
- **输出:** 4

**示例 3:**

- **输入:** `s = "IX"`
- **输出:** 9

**示例 4:**

- **输入:** `s = "LVIII"`
- **输出:** 58

**示例 5:**

- **输入:** `s = "MCMXCIV"`
- **输出:** 1994

**提示:**

- `1 <= s.length <= 15`
- `s` 仅含字符 `('I', 'V', 'X', 'L', 'C', 'D', 'M')`
- 题目数据保证 `s` 是一个有效的罗马数字，且表示整数在范围 `[1, 3999]` 内
- 题目所给测试用例皆符合罗马数字书写规则，不会出现跨位等情况。

## 题解

### 这题表面是在背规则，真正要抓的是“什么时候减”

这题如果只是把七个字符和对应数值硬背下来，其实还不够。  
真正会让人卡住的地方，不是 `I=1`、`V=5` 这些映射，而是：

**为什么有时候要加，有时候要减。**

大多数情况下，罗马数字都是“大数在前，小数在后”，所以一路加起来就行。  
比如：

```text
VIII = 5 + 1 + 1 + 1 = 8
```

但这题里有一类特殊写法：

- `IV = 4`
- `IX = 9`
- `XL = 40`
- `XC = 90`
- `CD = 400`
- `CM = 900`

这些地方不是“前一个数加上后一个数”，而是“前一个小数在给后一个大数做减法”。

所以这题真正要想明白的是：

**当前字符，到底应该加进去，还是减出去。**

### 什么时候加，什么时候减

判断其实很简单，只看当前字符和下一个字符的大小关系。

设当前字符的值是 `value`。

- 如果当前值 **小于** 下一个值，说明它处在一个减法组合里，这时候应该减掉它
- 否则，就正常加上它

比如：

```text
IV
```

当前看到 `I = 1`，下一个是 `V = 5`。  
因为 `1 < 5`，说明这个 `I` 不是单独的 1，而是在表示“少 1”，所以应该减掉：

```text
-1 + 5 = 4
```

再比如：

```text
VI
```

当前看到 `V = 5`，下一个是 `I = 1`。  
因为 `5 > 1`，说明这里没有减法规则，就是正常加：

```text
5 + 1 = 6
```

所以整题其实只是在重复做一个判断：

**当前字符比后一个小吗？小就减，不小就加。**

### 为什么这样做刚好能覆盖所有情况

因为题目已经保证输入一定是合法的罗马数字。

这点很重要。

如果输入不合法，那你可能还得额外判断各种奇怪组合。  
但现在题目已经保证：

- 不会乱写
- 不会跨位
- 只会出现合法的减法形式

那我们就不用去枚举 `IV`、`IX`、`XL`、`XC`、`CD`、`CM` 这些组合本身，直接用“当前值和下一个值比较”的规则统一处理就够了。

也就是说，这题真正厉害的地方就在于：

**不用死记六种特殊组合，只需要抓住它们共同的结构：前小后大。**

### 用几个例子走一遍

先看最简单的：

```text
III
```

从左到右看：

- 第一个 `I`，后面还是 `I`，不比后面小，加 `1`
- 第二个 `I`，后面还是 `I`，加 `1`
- 第三个 `I`，后面没有了，也加 `1`

结果就是：

```text
1 + 1 + 1 = 3
```

再看：

```text
IV
```

- `I = 1`，后面是 `V = 5`，`1 < 5`，减 `1`
- `V = 5`，加 `5`

结果：

```text
-1 + 5 = 4
```

再看：

```text
LVIII
```

逐个处理：

- `L = 50`，后面是 `V = 5`，加 `50`
- `V = 5`，后面是 `I = 1`，加 `5`
- `I = 1`，后面是 `I = 1`，加 `1`
- `I = 1`，后面是 `I = 1`，加 `1`
- 最后一个 `I = 1`，加 `1`

结果：

```text
50 + 5 + 1 + 1 + 1 = 58
```

最后看稍微复杂一点的：

```text
MCMXCIV
```

从左到右：

- `M = 1000`，后面 `C = 100`，加 `1000`
- `C = 100`，后面 `M = 1000`，减 `100`
- `M = 1000`，后面 `X = 10`，加 `1000`
- `X = 10`，后面 `C = 100`，减 `10`
- `C = 100`，后面 `I = 1`，加 `100`
- `I = 1`，后面 `V = 5`，减 `1`
- `V = 5`，加 `5`

最后是：

```text
1000 - 100 + 1000 - 10 + 100 - 1 + 5 = 1994
```

这几个例子看完，规律其实已经非常固定了。

### 所以实现上只需要两件事

第一件事：能把罗马字符映射成整数。  
第二件事：遍历字符串时，看当前字符和下一个字符的大小关系。

映射可以用 `switch`，也可以用哈希表。  
这里字符种类固定，直接 `switch` 就够了。

### 代码

```java
public int romanToInt(String s) {
    int sum = 0;

    for (int i = 0; i < s.length(); i++) {
        int value = getValue(s.charAt(i));

        if (i < s.length() - 1 && value < getValue(s.charAt(i + 1))) {
            sum -= value;
        } else {
            sum += value;
        }
    }

    return sum;
}

private int getValue(char ch) {
    switch (ch) {
        case 'I': return 1;
        case 'V': return 5;
        case 'X': return 10;
        case 'L': return 50;
        case 'C': return 100;
        case 'D': return 500;
        case 'M': return 1000;
        default: return 0;
    }
}
```

时间复杂度是 $O(n)$，因为只遍历了一遍字符串。  
空间复杂度是 $O(1)$，因为只用了常数个变量。

### 这题真正值得记住的地方

这题最值得记住的，不是“罗马数字有六种特殊情况”这句话，而是这种更统一的判断方式：

**当某个符号的含义会受到后一个符号影响时，可以考虑在遍历时顺手看一眼下一个字符。**

这题里：

- 当前值小于后一个值，就说明它在做减法
- 否则就是正常加法

所以与其背六种特例，不如直接记住这一个规则。  
这样以后哪怕细节忘了，看到题也还能自己推回来。

### 完整代码

```java
public class Main {

    private static int getValue(char ch) {
        switch (ch) {
            case 'I': return 1;
            case 'V': return 5;
            case 'X': return 10;
            case 'L': return 50;
            case 'C': return 100;
            case 'D': return 500;
            case 'M': return 1000;
            default: return 0;
        }
    }

    public static int romanToInt(String s) {
        int sum = 0;

        for (int i = 0; i < s.length(); i++) {
            int value = getValue(s.charAt(i));

            if (i < s.length() - 1 && value < getValue(s.charAt(i + 1))) {
                sum -= value;
            } else {
                sum += value;
            }
        }

        return sum;
    }

    public static void main(String[] args) {
        String s1 = "III";
        System.out.println("示例 1 输入: s = \"" + s1 + "\"");
        System.out.println("示例 1 输出: " + romanToInt(s1));

        String s2 = "IV";
        System.out.println("示例 2 输入: s = \"" + s2 + "\"");
        System.out.println("示例 2 输出: " + romanToInt(s2));

        String s3 = "IX";
        System.out.println("示例 3 输入: s = \"" + s3 + "\"");
        System.out.println("示例 3 输出: " + romanToInt(s3));

        String s4 = "LVIII";
        System.out.println("示例 4 输入: s = \"" + s4 + "\"");
        System.out.println("示例 4 输出: " + romanToInt(s4));

        String s5 = "MCMXCIV";
        System.out.println("示例 5 输入: s = \"" + s5 + "\"");
        System.out.println("示例 5 输出: " + romanToInt(s5));
    }
}
```
