## 题目

**描述：**

罗马数字包含以下七种字符: `I`, `V`, `X`, `L`, `C`, `D` 和 `M`。

| 字符 | 数值 |
| :--- | :--- |
| I    | 1    |
| V    | 5    |
| X    | 10   |
| L    | 50   |
| C    | 100  |
| D    | 500  |
| M    | 1000 |

通常情况下，罗马数字中小的数字在大的数字的右边。但也存在特例，例如 4 不写做 `IIII`，而是 `IV`。这个特殊的规则只适用于以下六种情况：

- `I` 可以放在 `V` (5) 和 `X` (10) 的左边，来表示 4 和 9。
- `X` 可以放在 `L` (50) 和 `C` (100) 的左边，来表示 40 和 90。
- `C` 可以放在 `D` (500) 和 `M` (1000) 的左边，来表示 400 和 900。

给定一个罗马数字，将其转换成整数。

**示例 1:**

- **输入:** `s = "III"`
- **输出:** 3

**示例 2:**

- **输入:** `s = "IV"`
- **输出:** 4

**示例 3:**

- **输入:** `s = "IX"`
- **输出:** 9

**示例 4:**

- **输入:** `s = "LVIII"`
- **输出:** 58

**示例 5:**

- **输入:** `s = "MCMXCIV"`
- **输出:** 1994

**提示:**

- `1 <= s.length <= 15`
- `s` 仅含字符 `('I', 'V', 'X', 'L', 'C', 'D', 'M')`
- 题目数据保证 `s` 是一个有效的罗马数字，且表示整数在范围 `[1, 3999]` 内
- 题目所给测试用例皆符合罗马数字书写规则，不会出现跨位等情况。

## 题解

这题看起来像在考罗马数字表，代码里最关键的其实不是背下 `I / V / X / L / C / D / M` 这七个映射，而是弄清楚：当前这个字符到底该加，还是该减。大多数情况下，罗马数字都是把较大的值写在前面，所以一路累加就行；麻烦只出在 `IV`、`IX`、`XL` 这类“前小后大”的组合上。

与其死记那六种特殊写法，不如直接抓住它们共同的结构。扫描到位置 `i` 时，先拿到当前字符的数值 `value`。如果它后面还有字符，而且 `value < nextValue`，说明这个字符不是单独在贡献数值，而是在给后面的更大数做减法，这时候就把它减掉；否则就正常加上去。题目已经保证输入一定是合法的罗马数字，所以这个判断足够覆盖所有情况。

拿 `MCMXCIV` 走一遍会很清楚。第一个 `M` 后面是 `C`，`1000` 不小于 `100`，所以加上 `1000`；接着 `C` 后面是 `M`，`100 < 1000`，说明这里是在表示 `900` 里的那一部分，于是减掉 `100`；后面的 `X` 和 `I` 也是同样的道理。一路算下来就是 `1000 - 100 + 1000 - 10 + 100 - 1 + 5 = 1994`。整道题其实一直都在重复同一个动作：看当前值和下一位的大小关系。

因为字符种类固定，映射部分直接写成 `switch` 就够了。遍历字符串时，如果当前值小于下一位，就做减法；否则就做加法，最后得到的和就是答案。

代码如下：

```java
public int romanToInt(String s) {
    int sum = 0;

    for (int i = 0; i < s.length(); i++) {
        int value = getValue(s.charAt(i));

        if (i < s.length() - 1 && value < getValue(s.charAt(i + 1))) {
            sum -= value;
        } else {
            sum += value;
        }
    }

    return sum;
}

private int getValue(char ch) {
    switch (ch) {
        case 'I': return 1;
        case 'V': return 5;
        case 'X': return 10;
        case 'L': return 50;
        case 'C': return 100;
        case 'D': return 500;
        case 'M': return 1000;
        default: return 0;
    }
}
```

时间复杂度是 `O(n)`，因为只遍历了一遍字符串；空间复杂度是 `O(1)`，因为只用了常数个变量。

这题做完以后更适合留下来的，不一定是哪六种特殊组合，而是这个判断口径：当一个符号的含义会受到后一位影响时，遍历时顺手看一眼下一位，往往就能把看似分散的特例统一起来。

完整代码如下：

```java
public class Main {

    private static int getValue(char ch) {
        switch (ch) {
            case 'I': return 1;
            case 'V': return 5;
            case 'X': return 10;
            case 'L': return 50;
            case 'C': return 100;
            case 'D': return 500;
            case 'M': return 1000;
            default: return 0;
        }
    }

    public static int romanToInt(String s) {
        int sum = 0;

        for (int i = 0; i < s.length(); i++) {
            int value = getValue(s.charAt(i));

            if (i < s.length() - 1 && value < getValue(s.charAt(i + 1))) {
                sum -= value;
            } else {
                sum += value;
            }
        }

        return sum;
    }

    public static void main(String[] args) {
        String s1 = "III";
        System.out.println("示例 1 输入: s = \"" + s1 + "\"");
        System.out.println("示例 1 输出: " + romanToInt(s1));

        String s2 = "IV";
        System.out.println("示例 2 输入: s = \"" + s2 + "\"");
        System.out.println("示例 2 输出: " + romanToInt(s2));

        String s3 = "IX";
        System.out.println("示例 3 输入: s = \"" + s3 + "\"");
        System.out.println("示例 3 输出: " + romanToInt(s3));

        String s4 = "LVIII";
        System.out.println("示例 4 输入: s = \"" + s4 + "\"");
        System.out.println("示例 4 输出: " + romanToInt(s4));

        String s5 = "MCMXCIV";
        System.out.println("示例 5 输入: s = \"" + s5 + "\"");
        System.out.println("示例 5 输出: " + romanToInt(s5));
    }
}
```
