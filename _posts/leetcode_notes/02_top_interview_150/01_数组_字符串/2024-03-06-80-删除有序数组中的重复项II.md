## 题目

**描述：**

给你一个有序数组 `nums` ，请你 **原地** 删除重复出现的元素，使得出现次数超过两次的元素**只出现两次** ，返回删除后数组的新长度。

不要使用额外的数组空间，你必须在 **原地 修改输入数组** 并在使用 O(1) 额外空间的条件下完成。

**说明：**

为什么返回数值是整数，但输出的答案是数组呢？

请注意，输入数组是以 **「引用」** 方式传递的，这意味着在函数里修改输入数组对于调用者是可见的。

你可以想象内部操作如下:

```java
// nums 是以“引用”方式传递的。也就是说，不对实参做任何拷贝
int len = removeDuplicates(nums);

// 在函数里修改输入数组对于调用者是可见的。
// 根据你的函数返回的长度, 它会打印出数组中 该长度范围内 的所有元素。
for (int i = 0; i < len; i++) {
    print(nums[i]);
}
```

**示例 1:**

- **输入:** `nums = [1,1,1,2,2,3]`
- **输出:** `5`, `nums = [1,1,2,2,3,_]`
- **解释:** 函数应返回新长度 `length = 5`, 并且原数组的前五个元素被修改为 `1, 1, 2, 2, 3` 。 不需要考虑数组中超出新长度后面的元素。

**示例 2:**

- **输入:** `nums = [0,0,1,1,1,1,2,3,3]`
- **输出:** `7`, `nums = [0,0,1,1,2,3,3,_,_]`
- **解释:** 函数应返回新长度 length = 7, 并且原数组的前七个元素被修改为 0, 0, 1, 1, 2, 3, 3。不需要考虑数组中超出新长度后面的元素。

**提示:**

- `1 <= nums.length <= 3 * 10^4`
- `-10^4 <= nums[i] <= 10^4`
- `nums` 已按升序排列

## 题解

### 这题比上一题多拐了一下

上一题是“每个元素只保留一个”，这一题改成了“每个元素最多保留两个”。

看起来只是把条件改了一点，但思路上确实要多想一步：  
我们不再是简单地判断“这个数是不是重复了”，而是要判断“这个数放进去之后，会不会让某个元素出现超过两次”。

题目真正要求的其实是：

**在有序数组里原地保留每个元素的前两个副本，把多出来的丢掉。**

数组后面的内容不重要，只看前面整理好的那一段。

### 有序这个条件还是关键

这题能用双指针，根本原因还是数组有序。

一旦数组有序，相同元素就一定挨在一起。这样一来，我们在遍历当前元素时，不需要回头找它之前出现过多少次，只需要看它放进结果区之后，会不会形成三个相同的数连在一起。

也就是说，这题虽然表面上是在“删除重复项”，本质上还是在做一件事：

**维护一段合法的有效区间。**

这个有效区间里的每个元素，出现次数都不能超过两次。

### 真正要判断的是什么

这里最容易卡住的地方，不是双指针本身，而是这一句判断到底怎么来的：

```java
nums[fast] != nums[slow - 2]
```

先别急着背这个式子，先想它在判断什么。

假设当前已经整理好的结果区间是 `nums[0...slow-1]`，现在准备看 `nums[fast]` 能不能放进去。

因为题目允许每个元素最多出现两次，所以只要它不会让结果区间里出现三个相同元素，就可以保留。

那怎么知道会不会出现三个相同元素？

就看它和结果区间里“倒数第二个元素”是不是一样。

- 如果一样，说明再放进去就会凑成三个相同元素，不能放
- 如果不一样，说明放进去也不会超过两次，可以放

这就是为什么要比较 `nums[fast]` 和 `nums[slow - 2]`。

### 为什么前两个元素可以直接保留

因为题目允许每个元素最多出现两次。

那不管前两个元素是什么，它们都不可能仅凭自己就违反规则。所以当有效区间长度还不到 2 的时候，当前元素可以直接放进去，不需要判断。

这也是代码里这部分的来源：

```java
if (slow < 2 || nums[fast] != nums[slow - 2]) {
    nums[slow] = nums[fast];
    slow++;
}
```

这里的 `slow < 2`，本质上就是在说：

**前两个位置先别管，先放进去再说。**

### 用示例走一遍

以示例 1 为例：

```text
nums = [1,1,1,2,2,3]
```

初始时，`slow = 0`。

`fast = 0` 时，`slow < 2`，直接保留：

```text
[1,1,1,2,2,3]
 ^
slow = 1
```

`fast = 1` 时，还是 `slow < 2`，继续保留：

```text
[1,1,1,2,2,3]
   ^
slow = 2
```

`fast = 2` 时，当前元素还是 `1`。  
这时不能再无脑放了，要比较 `nums[fast]` 和 `nums[slow - 2]`。

现在：

- `nums[fast] = 1`
- `nums[slow - 2] = nums[0] = 1`

两者相等，说明如果再放一个 `1` 进去，结果区间里就会出现三个 `1`，所以跳过。

`fast = 3` 时，当前元素是 `2`。  
比较后发现：

- `nums[fast] = 2`
- `nums[slow - 2] = nums[0] = 1`

不相等，说明可以放进去：

```text
[1,1,2,2,2,3]
     ^
slow = 3
```

接着 `fast = 4`，当前元素还是 `2`。  
比较：

- `nums[fast] = 2`
- `nums[slow - 2] = nums[1] = 1`

仍然不相等，所以第二个 `2` 也可以保留：

```text
[1,1,2,2,2,3]
       ^
slow = 4
```

再往后处理 `3`，最终前面的有效区间会变成：

```text
[1,1,2,2,3,_]
```

返回长度 `5`。

### 这题里 `slow` 表示什么

这题里的 `slow` 和上一题不太一样。

这里的 `slow` 始终指向**下一个可以写入的位置**，所以它本身就等于当前有效区间的长度。

每成功保留一个元素，就执行：

```java
nums[slow] = nums[fast];
slow++;
```

所以遍历结束后，`slow` 走了多少步，就说明一共保留了多少个元素。最后直接返回 `slow` 就行，不需要再加一。

### 代码

```java
public int removeDuplicates(int[] nums) {
    int slow = 0;

    for (int fast = 0; fast < nums.length; fast++) {
        if (slow < 2 || nums[fast] != nums[slow - 2]) {
            nums[slow] = nums[fast];
            slow++;
        }
    }

    return slow;
}
```

时间复杂度是 $O(n)$，因为整个数组只遍历了一遍。  
空间复杂度是 $O(1)$，因为只用了常数个额外变量。

### 这题可以顺手再记一层

这题其实是上一题的进一步推广。

上一题是“每个元素最多保留 1 个”，这一题是“每个元素最多保留 2 个”。  
如果把这个思路再往前推，其实可以得到一个更通用的结论：

**如果题目要求每个元素最多保留 `k` 个，那么就去比较 `nums[fast]` 和 `nums[slow - k]`。**

只要当前元素不等于结果区间里倒数第 `k` 个元素，它就还能继续放进去。

所以这题真正值得记住的，不只是这一个写法，而是这类题背后的判断方式：

**在有序数组里做“最多保留 k 个重复项”时，可以用双指针维护结果区间，再用 `slow - k` 这个位置判断当前元素还能不能进。**

### 完整代码

```java
import java.util.Arrays;

public class Main {

    public static int removeDuplicates(int[] nums) {
        int slow = 0;

        for (int fast = 0; fast < nums.length; fast++) {
            if (slow < 2 || nums[fast] != nums[slow - 2]) {
                nums[slow] = nums[fast];
                slow++;
            }
        }

        return slow;
    }

    public static void main(String[] args) {
        int[] nums1 = new int[]{1, 1, 1, 2, 2, 3};

        System.out.println("示例 1 输入:");
        System.out.println("nums = " + Arrays.toString(nums1));

        int k1 = removeDuplicates(nums1);
        int[] result1 = Arrays.copyOfRange(nums1, 0, k1);

        System.out.println("示例 1 输出: " + k1 + ", nums = " + Arrays.toString(result1));

        int[] nums2 = new int[]{0, 0, 1, 1, 1, 1, 2, 3, 3};

        System.out.println("示例 2 输入:");
        System.out.println("nums = " + Arrays.toString(nums2));

        int k2 = removeDuplicates(nums2);
        int[] result2 = Arrays.copyOfRange(nums2, 0, k2);

        System.out.println("示例 2 输出: " + k2 + ", nums = " + Arrays.toString(result2));
    }
}
```
