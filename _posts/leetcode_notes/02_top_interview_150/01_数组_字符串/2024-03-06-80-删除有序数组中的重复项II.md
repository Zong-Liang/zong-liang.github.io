## 题目

**描述：**

给你一个有序数组 `nums` ，请你 **原地** 删除重复出现的元素，使得出现次数超过两次的元素**只出现两次** ，返回删除后数组的新长度。

不要使用额外的数组空间，你必须在 **原地 修改输入数组** 并在使用 O(1) 额外空间的条件下完成。

**说明：**

为什么返回数值是整数，但输出的答案是数组呢？

请注意，输入数组是以 **「引用」** 方式传递的，这意味着在函数里修改输入数组对于调用者是可见的。

你可以想象内部操作如下:

```java
// nums 是以“引用”方式传递的。也就是说，不对实参做任何拷贝
int len = removeDuplicates(nums);

// 在函数里修改输入数组对于调用者是可见的。
// 根据你的函数返回的长度, 它会打印出数组中 该长度范围内 的所有元素。
for (int i = 0; i < len; i++) {
    print(nums[i]);
}
```

**示例 1:**

- **输入:** `nums = [1,1,1,2,2,3]`
- **输出:** `5`, `nums = [1,1,2,2,3,_]`
- **解释:** 函数应返回新长度 `length = 5`, 并且原数组的前五个元素被修改为 `1, 1, 2, 2, 3` 。 不需要考虑数组中超出新长度后面的元素。

**示例 2:**

- **输入:** `nums = [0,0,1,1,1,1,2,3,3]`
- **输出:** `7`, `nums = [0,0,1,1,2,3,3,_,_]`
- **解释:** 函数应返回新长度 length = 7, 并且原数组的前七个元素被修改为 0, 0, 1, 1, 2, 3, 3。不需要考虑数组中超出新长度后面的元素。

**提示:**

- `1 <= nums.length <= 3 * 10^4`
- `-10^4 <= nums[i] <= 10^4`
- `nums` 已按升序排列

## 题解

这一题和上一题的骨架几乎一样，只是规则从“每个元素只保留一个”变成了“每个元素最多保留两个”。看起来只多了一个条件，判断方式却要跟着变一下：我们不再是简单地问“这个数是不是重复了”，而是要看“把它放进结果区之后，会不会让某个元素出现三次”。

数组有序这一点依然是关键。因为有序，相同元素一定挨在一起出现，所以我们不需要回头去统计某个数已经出现了多少次，只需要盯住当前维护的那段有效区间，判断再放一个当前元素进去会不会超标。

这里最重要的一句判断可以直接写成这样。

```java
nums[fast] != nums[slow - 2]
```

这句式子其实不难理解。假设当前已经整理好的有效区间是 `nums[0...slow-1]`，而 `slow` 指向下一个可写入位置。如果题目允许每个元素最多出现两次，那么当前元素只要不等于有效区间里的倒数第二个元素，就还能放进去；一旦两者相等，就说明如果再写入一次，就会凑成三个相同的值。

前两个元素之所以可以直接保留，也是同一个道理。只要当前有效区间长度还不到 `2`，无论看到什么元素，都不可能仅凭这一步就违反“最多出现两次”的限制。所以代码里会先判断 `slow < 2`，也就是先把前两个位置填起来，再从第三个位置开始做真正的去重判断。

拿示例 1 走一遍。`nums = [1,1,1,2,2,3]`。一开始 `slow = 0`，第一个 `1` 因为 `slow < 2`，直接保留；第二个 `1` 仍然因为 `slow < 2`，也直接保留。等 `fast` 走到第三个 `1` 时，`slow` 已经是 `2`，这时候就要比较 `nums[fast]` 和 `nums[slow - 2]`。两者都是 `1`，说明再放进去就会出现三个 `1`，所以这一位跳过。再往后遇到 `2` 时，因为 `2 != nums[slow - 2]`，说明它还能进结果区；第二个 `2` 同样可以留下。最后数组前面整理出来的就是 `[1,1,2,2,3]`。

这题里 `slow` 的含义也和上一题稍有不同。这里它始终指向的是“下一个可写入位置”，所以它本身就等于当前有效区间的长度。每成功保留一个元素，就把它写到 `nums[slow]`，然后 `slow++`。因此遍历结束后，直接返回 `slow` 就行，不需要再加一。

代码如下：

```java
public int removeDuplicates(int[] nums) {
    int slow = 0;

    for (int fast = 0; fast < nums.length; fast++) {
        if (slow < 2 || nums[fast] != nums[slow - 2]) {
            nums[slow] = nums[fast];
            slow++;
        }
    }

    return slow;
}
```

时间复杂度是 $O(n)$，因为整个数组只遍历了一遍；空间复杂度是 $O(1)$，因为只用了常数个额外变量。

这题其实就是上一题的进一步推广。上一题相当于“每个元素最多保留 1 个”，这一题变成“每个元素最多保留 2 个”。如果把这个思路再往前推，就会得到一个很实用的结论：当题目要求每个元素最多保留 `k` 个时，通常可以去比较 `nums[fast]` 和 `nums[slow - k]`。

完整代码如下：

```java
import java.util.Arrays;

public class Main {

    public static int removeDuplicates(int[] nums) {
        int slow = 0;

        for (int fast = 0; fast < nums.length; fast++) {
            if (slow < 2 || nums[fast] != nums[slow - 2]) {
                nums[slow] = nums[fast];
                slow++;
            }
        }

        return slow;
    }

    public static void main(String[] args) {
        int[] nums1 = new int[]{1, 1, 1, 2, 2, 3};

        System.out.println("示例 1 输入:");
        System.out.println("nums = " + Arrays.toString(nums1));

        int k1 = removeDuplicates(nums1);
        int[] result1 = Arrays.copyOfRange(nums1, 0, k1);

        System.out.println("示例 1 输出: " + k1 + ", nums = " + Arrays.toString(result1));

        int[] nums2 = new int[]{0, 0, 1, 1, 1, 1, 2, 3, 3};

        System.out.println("示例 2 输入:");
        System.out.println("nums = " + Arrays.toString(nums2));

        int k2 = removeDuplicates(nums2);
        int[] result2 = Arrays.copyOfRange(nums2, 0, k2);

        System.out.println("示例 2 输出: " + k2 + ", nums = " + Arrays.toString(result2));
    }
}
```
