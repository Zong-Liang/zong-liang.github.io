## 题目

**描述：**

`n` 个孩子站成一排。给你一个整数数组 `ratings` 表示每个孩子的评分。

你需要按照以下要求，给这些孩子分发糖果：

- 每个孩子至少分配到 1 个糖果。
- 相邻两个孩子中，评分更高的那个会获得更多的糖果。

请你给每个孩子分发糖果，计算并返回需要准备的 **最少糖果数目** 。

**示例 1:**

- **输入:** `ratings = [1,0,2]`
- **输出:** 5
- **解释:** 你可以分别给第一个、第二个、第三个孩子分发 2、1、2 颗糖果。

**示例 2:**

- **输入:** `ratings = [1,2,2]`
- **输出:** 4
- **解释:** 你可以分别给第一个、第二个、第三个孩子分发 1、2、1 颗糖果。第三个孩子只得到 1 颗糖果，这满足题面中的两个条件。

**提示:**

- `n == ratings.length`
- `1 <= n <= 2 * 10^4`
- `0 <= ratings[i] <= 2 * 10^4`

## 题解

这题看起来像是在按评分分糖果，麻烦的地方并不是“最少”两个字，而是每个位置同时受左右两边约束。一个孩子拿多少，可能取决于左边，也可能取决于右边。也正因为如此，这题不像很多相邻关系题那样，顺着一个方向扫一遍就能结束。

如果只从左往右处理，很容易顾此失彼。比如 `ratings = [1, 0, 2]`，从左往右时，先给每个人一颗糖，会得到 `[1, 1, 1]`；继续扫下去，只会因为最后一个孩子比左边评分高，把它改成 `2`，结果变成 `[1, 1, 2]`。但这个分法其实不合法，因为第一个孩子评分比第二个高，糖果却没有更多。这就说明：左边规则和右边规则不能混在一遍里硬做。

比较自然的拆法，是先让所有孩子都至少拿到 `1` 颗，然后把两侧约束分开处理。第一遍从左往右，只管“如果当前评分比左边高，那当前糖果就比左边多一颗”。这样扫完以后，所有“右边评分更高”的位置都已经满足要求了。接着再从右往左，专门处理另一条规则：如果当前评分比右边高，那当前糖果至少要比右边多一颗。

这里有个细节特别值得说清楚：第二遍不能直接写成 `candies[i] = candies[i + 1] + 1`，而要写成 `Math.max(candies[i], candies[i + 1] + 1)`。因为左往右那一遍可能已经给当前位置分了更多糖果，右往左这次只能补够，不能把前面已经满足的关系改坏。像 `ratings = [1, 3, 4, 5, 2]`，第一遍扫完后糖果是 `[1, 2, 3, 4, 1]`。这时下标 `3` 的孩子评分比右边高，按右侧规则它至少得有 `2` 颗，但它原来已经拿到 `4` 颗了；如果直接覆盖成 `2`，前面那条递增链就会被破坏掉。

再回到示例 `ratings = [1, 0, 2]`。初始化后是 `[1, 1, 1]`。第一遍从左往右，只会把最后一个位置改成 `2`，得到 `[1, 1, 2]`。第二遍从右往左时，看到第一个孩子评分比右边高，于是把它补成 `2`，最后就得到 `[2, 1, 2]`，总数正好是 `5`。这也能看出来，两遍扫描并不是在重复做同一件事，而是在分别修正左右两边的关系。

代码如下：

```java
public int candy(int[] ratings) {
    int n = ratings.length;
    int[] candies = new int[n];
    Arrays.fill(candies, 1);

    for (int i = 1; i < n; i++) {
        if (ratings[i] > ratings[i - 1]) {
            candies[i] = candies[i - 1] + 1;
        }
    }

    for (int i = n - 2; i >= 0; i--) {
        if (ratings[i] > ratings[i + 1]) {
            candies[i] = Math.max(candies[i], candies[i + 1] + 1);
        }
    }

    int sum = 0;
    for (int candy : candies) {
        sum += candy;
    }

    return sum;
}
```

时间复杂度是 `O(n)`，因为总共做了两次线性扫描和一次求和；空间复杂度是 `O(n)`，因为额外开了一个 `candies` 数组。

这题比较适合记住的，不一定只是“两次遍历”这四个字，而是遇到双向约束时的处理方式：单方向总是顾不全，就把左右两边的要求拆开，各自满足，再用 `max` 这样的方式把结果合在一起。

完整代码如下：

```java
import java.util.Arrays;

public class Main {

    public static int candy(int[] ratings) {
        int n = ratings.length;
        int[] candies = new int[n];

        Arrays.fill(candies, 1);

        for (int i = 1; i < n; i++) {
            if (ratings[i] > ratings[i - 1]) {
                candies[i] = candies[i - 1] + 1;
            }
        }

        for (int i = n - 2; i >= 0; i--) {
            if (ratings[i] > ratings[i + 1]) {
                candies[i] = Math.max(candies[i], candies[i + 1] + 1);
            }
        }

        int totalCandies = 0;
        for (int count : candies) {
            totalCandies += count;
        }

        return totalCandies;
    }

    public static void main(String[] args) {
        int[] ratings1 = new int[]{1, 0, 2};

        System.out.println("示例 1 输入:");
        System.out.println("ratings = " + Arrays.toString(ratings1));

        int result1 = candy(ratings1);

        System.out.println("示例 1 输出: " + result1);

        int[] ratings2 = new int[]{1, 2, 2};

        System.out.println("示例 2 输入:");
        System.out.println("ratings = " + Arrays.toString(ratings2));

        int result2 = candy(ratings2);

        System.out.println("示例 2 输出: " + result2);
    }
}
```
