## 题目

**描述：**

`n` 个孩子站成一排。给你一个整数数组 `ratings` 表示每个孩子的评分。

你需要按照以下要求，给这些孩子分发糖果：

- 每个孩子至少分配到 1 个糖果。
- 相邻两个孩子中，评分更高的那个会获得更多的糖果。

请你给每个孩子分发糖果，计算并返回需要准备的 **最少糖果数目** 。

**示例 1:**

- **输入:** `ratings = [1,0,2]`
- **输出:** 5
- **解释:** 你可以分别给第一个、第二个、第三个孩子分发 2、1、2 颗糖果。

**示例 2:**

- **输入:** `ratings = [1,2,2]`
- **输出:** 4
- **解释:** 你可以分别给第一个、第二个、第三个孩子分发 1、2、1 颗糖果。第三个孩子只得到 1 颗糖果，这满足题面中的两个条件。

**提示:**

- `n == ratings.length`
- `1 <= n <= 2 * 10^4`
- `0 <= ratings[i] <= 2 * 10^4`

## 题解

### 这题难就难在，约束是双向的

这题看起来像是在按评分分糖果，但真正麻烦的地方不在“分”，而在约束不是单边的。

题目要求的是：

- 每个人至少 1 颗
- 如果一个孩子评分比左边高，那他的糖果要更多
- 如果一个孩子评分比右边高，那他的糖果也要更多

也就是说，一个位置上的糖果数，可能同时受到左边和右边的影响。

这就是为什么这题很难一遍直接做对。  
因为你刚满足了左边，可能又把右边破坏掉；刚满足了右边，可能又把左边改坏了。

所以这题真正的关键不是“怎么分最少”，而是先意识到：

**左侧约束和右侧约束，最好拆开处理。**

### 为什么不能只从一个方向扫一遍

先看一个很简单的例子：

```text
ratings = [1,0,2]
```

如果你只从左往右看：

- 第一个孩子先给 1
- 第二个评分更低，也给 1
- 第三个评分比第二个高，所以给 2

这样会得到：

```text
[1,1,2]
```

但这其实不合法。  
因为第一个孩子评分比第二个高，糖果却没有更多。

这说明：  
**只看左边不够。**

反过来，如果你只从右往左扫，也会遇到同样的问题。  
因为你能保证“评分比右边高的人糖果更多”，但未必能保证左边那条规则。

所以这题没法靠单方向一次遍历解决，必须把两边都照顾到。

### 最自然的拆法：先管左边，再管右边

既然一个方向顾不全，那就分两遍来做。

先让每个人都拿到最少的一颗糖。  
这样至少满足了“每个人至少 1 颗”这条底线。

然后：

- 第一遍从左到右，专门处理“比左边评分高”的情况
- 第二遍从右到左，专门处理“比右边评分高”的情况

这样每一遍都只解决一侧的约束，思路就会清楚很多。

### 第一遍在做什么

从左往右时，只关心这一条：

**如果 `ratings[i] > ratings[i - 1]`，那 `candies[i]` 必须比 `candies[i - 1]` 多。**

所以写法就是：

```java
if (ratings[i] > ratings[i - 1]) {
    candies[i] = candies[i - 1] + 1;
}
```

如果当前评分没有比左边高，那这一遍就不用动，保持 1 就行。

这一遍结束后，数组已经保证了：

**每个评分比左邻居高的孩子，糖果一定更多。**

但这时候还没有保证右边关系。

### 第二遍为什么还要取 `max`

接着从右往左扫，处理另一条约束：

**如果 `ratings[i] > ratings[i + 1]`，那 `candies[i]` 必须比右边多。**

这时候很多人第一次会直接写：

```java
candies[i] = candies[i + 1] + 1;
```

但这样有风险。  
因为这个位置 `i` 在第一遍里，可能已经因为左边规则拿到了更多糖果。

比如：

```text
ratings = [1,3,4,5,2]
```

第一遍从左往右后，糖果可能是：

```text
[1,2,3,4,1]
```

现在从右往左看，位置 `3` 的评分 `5` 比右边 `2` 高，所以它至少应该比右边多一颗，也就是至少要有 `2` 颗。

但它原来已经有 `4` 颗了。  
如果你直接改成 `2`，反而把左边已经满足的关系破坏掉了。

所以这里不能直接覆盖，只能这样写：

```java
candies[i] = Math.max(candies[i], candies[i + 1] + 1);
```

意思很简单：

- 先看右边规则要求我至少有多少
- 再和我现在已经有的糖果数比一下
- 取更大的那个

这样才能同时满足左右两边的约束。

### 用示例 1 走一遍

看这个例子：

```text
ratings = [1,0,2]
```

先初始化：

```text
candies = [1,1,1]
```

第一遍从左往右：

- `ratings[1] = 0` 不大于 `ratings[0] = 1`，不动
- `ratings[2] = 2` 大于 `ratings[1] = 0`，所以 `candies[2] = candies[1] + 1 = 2`

这时变成：

```text
[1,1,2]
```

第二遍从右往左：

- 看位置 `1`，`ratings[1] = 0` 不大于右边 `2`，不动
- 看位置 `0`，`ratings[0] = 1` 大于右边 `0`，所以它至少要比右边多一颗

于是：

```text
candies[0] = max(1, 1 + 1) = 2
```

最后得到：

```text
[2,1,2]
```

总数就是 `5`。

### 再看示例 2 为什么不是 5

再看：

```text
ratings = [1,2,2]
```

初始化：

```text
[1,1,1]
```

第一遍从左往右：

- `2 > 1`，所以第二个孩子拿 `2`
- 第三个孩子评分和第二个相等，不需要更多

结果是：

```text
[1,2,1]
```

第二遍从右往左：

- 第一个和右边比，不需要改
- 第二个和第三个比，虽然评分更高，但它本来就已经满足“比右边多”

所以最后还是：

```text
[1,2,1]
```

总数是 `4`。

这个例子也说明了一点：  
题目要求的是“评分更高的那个拿更多”，不是“评分相同也要一样多”或者“评分不低就要更多”。

### 代码

```java
public int candy(int[] ratings) {
    int n = ratings.length;
    int[] candies = new int[n];
    Arrays.fill(candies, 1);

    for (int i = 1; i < n; i++) {
        if (ratings[i] > ratings[i - 1]) {
            candies[i] = candies[i - 1] + 1;
        }
    }

    for (int i = n - 2; i >= 0; i--) {
        if (ratings[i] > ratings[i + 1]) {
            candies[i] = Math.max(candies[i], candies[i + 1] + 1);
        }
    }

    int sum = 0;
    for (int c : candies) {
        sum += c;
    }

    return sum;
}
```

时间复杂度是 $O(n)$，因为总共做了两次线性扫描，再加一次求和。  
空间复杂度是 $O(n)$，因为额外开了一个 `candies` 数组。

### 这题真正值得记住的地方

这题最值得记住的，不只是“两次遍历”这个写法，而是这种拆问题的方式：

**如果一个位置同时受到左边和右边的约束，单方向不好处理时，往往可以把两个方向的要求拆开，分别满足，再合并结果。**

这题里：

- 第一遍负责满足左规则
- 第二遍负责满足右规则
- `max` 负责把两边的要求合到一起

以后再遇到这种“相邻关系同时约束左右两边”的题，也可以优先往这个方向想。

### 完整代码

```java
import java.util.Arrays;

public class Main {

    public static int candy(int[] ratings) {
        int n = ratings.length;
        int[] candies = new int[n];

        Arrays.fill(candies, 1);

        for (int i = 1; i < n; i++) {
            if (ratings[i] > ratings[i - 1]) {
                candies[i] = candies[i - 1] + 1;
            }
        }

        for (int i = n - 2; i >= 0; i--) {
            if (ratings[i] > ratings[i + 1]) {
                candies[i] = Math.max(candies[i], candies[i + 1] + 1);
            }
        }

        int totalCandies = 0;
        for (int count : candies) {
            totalCandies += count;
        }

        return totalCandies;
    }

    public static void main(String[] args) {
        int[] ratings1 = new int[]{1, 0, 2};

        System.out.println("示例 1 输入:");
        System.out.println("ratings = " + Arrays.toString(ratings1));

        int result1 = candy(ratings1);

        System.out.println("示例 1 输出: " + result1);

        int[] ratings2 = new int[]{1, 2, 2};

        System.out.println("示例 2 输入:");
        System.out.println("ratings = " + Arrays.toString(ratings2));

        int result2 = candy(ratings2);

        System.out.println("示例 2 输出: " + result2);
    }
}
```
