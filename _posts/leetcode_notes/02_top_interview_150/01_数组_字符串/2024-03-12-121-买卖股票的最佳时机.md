## 题目

**描述：**

给定一个数组 `prices` ，它的第 `i` 个元素 `prices[i]` 表示一支给定股票第 `i` 天的价格。

你只能选择 **某一天** 买入这只股票，并选择在 **未来的某一个不同的日子** 卖出该股票。设计一个算法来计算你所能获取的最大利润。

返回你可以从这笔交易中获取的最大利润。如果你不能获取任何利润，返回 `0` 。

**示例 1:**

- **输入:** `[7,1,5,3,6,4]`
- **输出:** 5
- **解释:** 在第 2 天（股票价格 = 1）的时候买入，在第 5 天（股票价格 = 6）的时候卖出，最大利润 = 6-1 = 5 。注意利润不能是 7-1 = 6, 因为卖出价格需要大于买入价格；同时，你不能在买入前卖出股票。

**示例 2:**

- **输入:** `prices = [7,6,4,3,1]`
- **输出:** 0
- **解释:** 在这种情况下, 没有交易完成, 所以最大利润为 0。

**提示:**

- `1 <= prices.length <= 10^5`
- `0 <= prices[i] <= 10^4`

## 题解

### 这题别一上来就想“什么时候卖最赚”

很多人第一次看到这题，会先盯着“卖出”去想：  
我要在哪一天卖，利润才最大？

但这样想很容易把问题想复杂。因为卖出赚不赚钱，不只取决于卖出的价格高不高，还取决于你之前有没有在更低的价格买到。

所以这题真正要想清楚的，不是“哪天卖最值”，而是：

**走到当天时，如果今天卖出，我之前能买到的最低价格是多少？**

一旦这个问题想明白了，这题基本就已经做完一半了。

### 先说最直接的思路

最朴素的做法，就是把所有可能的买卖组合都试一遍。

- 枚举买入日
- 再枚举它后面的卖出日
- 计算每一组利润
- 取最大值

代码当然能写，而且逻辑也很直白：

```java
public int maxProfit(int[] prices) {
    int maxProfit = 0;

    for (int i = 0; i < prices.length - 1; i++) {
        for (int j = i + 1; j < prices.length; j++) {
            int profit = prices[j] - prices[i];
            if (profit > maxProfit) {
                maxProfit = profit;
            }
        }
    }

    return maxProfit;
}
```

这个做法的问题不在对不对，而在太慢。

如果数组长度是 `n`，那就要比较将近 `n^2` 组买卖方案。题目给的数据规模是 `10^5`，这种写法肯定扛不住。

所以这题真正需要解决的，不是“如何列出所有可能”，而是：

**能不能在遍历到当天时，顺手就知道以今天卖出能赚多少。**

### 这题其实只需要记两件事

往后扫数组时，每一天你真正关心的只有两件事：

- 到今天为止，出现过的最低价格是多少
- 如果今天卖出，最多能赚多少

于是我们就可以一直维护两个变量：

- `minPrice`：到当前位置为止的历史最低价
- `maxProfit`：到当前位置为止能拿到的最大利润

每看到一个新价格 `price`，只做两步判断：

- 如果它比历史最低价还低，就更新 `minPrice`
- 否则就看看 `price - minPrice` 能不能刷新最大利润

整个过程只扫一遍数组就够了。

### 为什么这个思路不会漏答案

因为你只能先买后卖。

这句话看起来像废话，但它其实正好说明了为什么“维护历史最低价”就够了。

假设今天是卖出日，那你唯一合法的买入位置，只能在今天之前。  
而在今天之前想让利润最大，买入价当然应该尽可能低。

所以对每一天来说，最优选择其实都很明确：

**拿今天的价格，减去之前出现过的最低价格。**

把每一天都这样算一遍，取最大值，就是全局最优答案。

这也是为什么不需要回头枚举所有买卖组合。  
因为“过去最低价”已经把过去所有可能的买入点压缩成了一个最有价值的信息。

### 用示例走一遍

以示例 1 为例：

```text
prices = [7,1,5,3,6,4]
```

开始时：

- `minPrice = +∞`
- `maxProfit = 0`

先看到 `7`：

- 当前最低价更新为 `7`
- 还谈不上赚钱

再看到 `1`：

- 当前最低价更新为 `1`

再看到 `5`：

- 如果今天卖出，利润是 `5 - 1 = 4`
- 所以 `maxProfit = 4`

再看到 `3`：

- 利润是 `3 - 1 = 2`
- 不如当前最大利润，跳过

再看到 `6`：

- 利润是 `6 - 1 = 5`
- 刷新最大利润，`maxProfit = 5`

最后看到 `4`：

- 利润是 `4 - 1 = 3`
- 还是不如 `5`

所以最终答案就是 `5`。

整个过程里，你并没有显式枚举“第几天买、第几天卖”的所有组合，但答案已经自然出来了。

### 为什么一直下跌时答案是 0

像示例 2 这种：

```text
prices = [7,6,4,3,1]
```

数组一路下降，意味着后面的价格永远不会高于前面的最低价。

那每次计算出来的利润都小于等于 `0`，自然就不可能有正利润。

题目要求的是“如果不能获利，返回 0”，所以 `maxProfit` 一开始就设成 `0`，这样即使全程没有赚钱机会，最后返回的也正好符合题意。

### 代码

```java
public int maxProfit(int[] prices) {
    int minPrice = Integer.MAX_VALUE;
    int maxProfit = 0;

    for (int price : prices) {
        if (price < minPrice) {
            minPrice = price;
        } else if (price - minPrice > maxProfit) {
            maxProfit = price - minPrice;
        }
    }

    return maxProfit;
}
```

时间复杂度是 $O(n)$，因为整个数组只遍历了一遍。  
空间复杂度是 $O(1)$，因为只用了两个额外变量。

### 这题真正值得记住的地方

这题最值得记住的，不是“股票题要用贪心”这种标签，而是这类题的处理方式：

**当某个位置的最优答案，只依赖于它前面的一部分最值信息时，往往可以一边遍历、一边维护这个最值。**

这题里维护的是“历史最低买入价”。  
以后如果遇到类似“当前位置的收益取决于前面最小值 / 最大值”的题，也可以优先往这个方向想。

说到底，这题不是在找一对买卖日期，而是在维护一个过程：

- 前面最便宜的是谁
- 现在卖掉能赚多少
- 到目前为止最赚的是多少

### 完整代码

```java
import java.util.Arrays;

public class Main {

    public static int maxProfit(int[] prices) {
        int minPrice = Integer.MAX_VALUE;
        int maxProfit = 0;

        for (int price : prices) {
            if (price < minPrice) {
                minPrice = price;
            } else if (price - minPrice > maxProfit) {
                maxProfit = price - minPrice;
            }
        }

        return maxProfit;
    }

    public static void main(String[] args) {
        int[] prices1 = new int[]{7, 1, 5, 3, 6, 4};

        System.out.println("示例 1 输入:");
        System.out.println("prices = " + Arrays.toString(prices1));

        int result1 = maxProfit(prices1);

        System.out.println("示例 1 输出: " + result1);

        int[] prices2 = new int[]{7, 6, 4, 3, 1};

        System.out.println("示例 2 输入:");
        System.out.println("prices = " + Arrays.toString(prices2));

        int result2 = maxProfit(prices2);

        System.out.println("示例 2 输出: " + result2);
    }
}
```
