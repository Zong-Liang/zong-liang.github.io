## 题目

**描述：**

给你一个 **非严格递增排列** 的数组 `nums` ，请你 **原地** 删除重复出现的元素，使每个元素 **只出现一次** ，返回删除后数组的新长度。元素的 **相对顺序** 应该保持 **一致** 。然后返回 `nums` 中唯一元素的个数。

考虑 `nums` 的唯一元素的数量为 `k`，去重后，返回 `k`。`nums` 的前 `k` 个元素应包含排序后的唯一数字。下标 `k-1` 之后的剩余元素可以忽略。

**判题标准：**

系统会用下面的代码来测试你的题解:

```java
int[] nums = [...]; // 输入数组
int[] expectedNums = [...]; // 长度正确的期望答案

int k = removeDuplicates(nums); // 调用

assert k == expectedNums.length;
for (int i = 0; i < k; i++) {
    assert nums[i] == expectedNums[i];
}
```

**示例 1:**

- **输入:** `nums = [1,1,2]`
- **输出:** `2`, `nums = [1,2,_]`
- **解释:** 函数应该返回新的长度 `2` ，并且原数组 `nums` 的前两个元素被修改为 `1, 2` 。不需要考虑数组中超出新长度后面的元素。

**示例 2:**

- **输入:** `nums = [0,0,1,1,1,2,2,3,3,4]`
- **输出:** `5`, `nums = [0,1,2,3,4,_,_,_,_,_]`
- **解释:** 函数应该返回新的长度 `5` ， 并且原数组 `nums` 的前五个元素被修改为 `0, 1, 2, 3, 4` 。

**提示:**

- `1 <= nums.length <= 3 * 10^4`
- `-100 <= nums[i] <= 100`
- `nums` 已按 **非递减** 顺序排列。

## 题解

### 这题和上一题不一样的地方在哪

如果只是看“删除重复项”这几个字，很容易把它理解成一道普通的原地删除题。

但这题和“移除元素”那题不太一样，关键多了两个条件：

- 数组本身是有序的
- 去重后还要保持原来的相对顺序

这两个条件放在一起，其实已经把做法提示得很明显了。

因为数组有序，所以重复元素一定是挨在一起出现的。也就是说，我们不需要真的去“删除”中间某些位置的数，只需要把不重复的元素依次放到数组前面就行。

题目真正要的，其实是：

**把有序数组中的每个不同元素保留一个，按原顺序写回前面，然后返回新的长度。**

### 有序这个条件是解题关键

如果数组不是有序的，那“重复元素”可能散落在各个位置，处理起来就会麻烦很多。

但这里数组已经是非递减排列，相同的数一定连续出现。这样一来，我们只需要判断当前元素和前一个保留下来的元素是不是一样，就知道它该不该留下。

也正因为这个条件成立，这题才能用双指针在线性时间内做完。

### 用两个指针就够了

这里还是双指针，但它的含义和上一题稍微有点不同。

- `fast` 负责从左到右扫描数组
- `slow` 负责指向当前去重后序列的最后一个位置

可以这样理解：

- `fast` 一直往前看，负责找“新元素”
- `slow` 留在已经整理好的那段有效区间末尾

如果 `nums[fast]` 和 `nums[slow]` 相同，说明它是重复元素，不需要处理，继续往后看。

如果 `nums[fast]` 和 `nums[slow]` 不同，说明找到了一个新的、不重复的元素。这时候先让 `slow` 往后走一位，再把 `nums[fast]` 放到 `nums[slow]` 上。

这样处理完后，数组前面这一段就始终保持着“已经去重好的结果”。

### 为什么 `slow` 从 0 开始，`fast` 从 1 开始

因为数组是有序的，而且题目保证长度至少为 1，所以第一个元素一定要保留。

既然第一个元素肯定属于答案，那它自然就是去重后结果的起点。  
所以：

- `slow = 0`，表示当前有效区间先从第一个元素开始
- `fast = 1`，从第二个元素开始继续往后检查

这也是这题和“移除元素”那题不太一样的地方。上一题里，保不保留某个元素取决于它是否等于 `val`；而这题里，第一个元素本身就已经确定要留下。

### 用示例走一遍

以示例 2 为例：

```text
nums = [0,0,1,1,1,2,2,3,3,4]
```

初始时：

- `slow = 0`
- `fast = 1`

先看 `nums[1] = 0`，它和 `nums[slow] = 0` 一样，说明重复了，跳过。

接着 `fast` 走到 `2`，此时 `nums[2] = 1`，和 `nums[slow] = 0` 不一样，说明找到了一个新元素。

这时候先让 `slow++`，变成 `1`，再把 `nums[2]` 放到 `nums[1]`：

```text
[0,1,1,1,1,2,2,3,3,4]
```

继续往后扫：

- 遇到重复的 `1`，跳过
- 遇到 `2`，放到下一位
- 遇到 `3`，放到下一位
- 遇到 `4`，放到下一位

最后数组前面就会变成：

```text
[0,1,2,3,4,_,_,_,_,_]
```

此时 `slow = 4`，所以答案长度就是 `slow + 1 = 5`。

### 为什么最后返回的是 `slow + 1`

因为 `slow` 表示的是“去重后最后一个元素的下标”，不是元素个数。

下标从 `0` 开始，所以如果最后一个有效元素在下标 `4` 的位置，那有效元素个数其实是 `5` 个。

这也是这题里最容易顺手写错的地方之一：  
**`slow` 是位置，返回值是长度，两者差 1。**

### 代码

```java
public int removeDuplicates(int[] nums) {
    int slow = 0;

    for (int fast = 1; fast < nums.length; fast++) {
        if (nums[fast] != nums[slow]) {
            slow++;
            nums[slow] = nums[fast];
        }
    }

    return slow + 1;
}
```

时间复杂度是 $O(n)$，因为整个数组只扫描了一遍。  
空间复杂度是 $O(1)$，只用了两个额外变量。

### 这题真正值得记住的地方

这题值得记住的，不只是“有序数组去重”这一个结论，而是这类题常见的判断方式：

**一旦数组有序，相同元素会聚在一起，很多原地去重问题都可以转化成“保留每一段重复元素中的第一个”。**

这时双指针通常就会很好用：

- 一个指针负责遍历
- 一个指针负责维护整理后的有效区间

以后再遇到“有序数组去重”“有序数组保留唯一值”“原地压缩有序序列”这类题，基本都可以从这个思路出发。

### 完整代码

```java
import java.util.Arrays;

public class Main {

    public static int removeDuplicates(int[] nums) {
        int slow = 0;

        for (int fast = 1; fast < nums.length; fast++) {
            if (nums[fast] != nums[slow]) {
                slow++;
                nums[slow] = nums[fast];
            }
        }

        return slow + 1;
    }

    public static void main(String[] args) {
        int[] nums1 = new int[]{1, 1, 2};

        System.out.println("示例 1 输入:");
        System.out.println("nums = " + Arrays.toString(nums1));

        int k1 = removeDuplicates(nums1);
        int[] result1 = Arrays.copyOfRange(nums1, 0, k1);

        System.out.println("示例 1 输出: " + k1 + ", nums = " + Arrays.toString(result1));
        System.out.println("-----------------------------");

        int[] nums2 = new int[]{0, 0, 1, 1, 1, 2, 2, 3, 3, 4};

        System.out.println("示例 2 输入:");
        System.out.println("nums = " + Arrays.toString(nums2));

        int k2 = removeDuplicates(nums2);
        int[] result2 = Arrays.copyOfRange(nums2, 0, k2);

        System.out.println("示例 2 输出: " + k2 + ", nums = " + Arrays.toString(result2));
    }
}
```
