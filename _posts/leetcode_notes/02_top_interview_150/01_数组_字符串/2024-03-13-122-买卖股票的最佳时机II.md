## 题目

**描述：**

给你一个整数数组 `prices` ，其中 `prices[i]` 表示某支股票第 `i` 天的价格。

在每一天，你可以决定是否购买和/或出售股票。你在任何时候 **最多** 只能持有 **一股** 股票。你也可以在 **同一天** 多次买卖该股票，但要确保你持有的股票不超过一股。

返回你能获得的 **最大利润** 。

**示例 1:**

- **输入:** `prices = [7,1,5,3,6,4]`
- **输出:** 7
- **解释:** 在第 2 天（股票价格 = 1）的时候买入，在第 3 天（股票价格 = 5）的时候卖出, 这笔交易所能获得利润 = 5-1 = 4 。随后，在第 4 天（股票价格 = 3）的时候买入，在第 5 天（股票价格 = 6）的时候卖出, 这笔交易所能获得利润 = 6-3 = 3 。最大总利润为 4 + 3 = 7 。

**示例 2:**

- **输入:** `prices = [1,2,3,4,5]`
- **输出:** 4
- **解释:** 在第 1 天（股票价格 = 1）的时候买入，在第 5 天 （股票价格 = 5）的时候卖出, 这笔交易所能获得利润 = 5-1 = 4 。最大总利润为 4 。

**示例 3:**

- **输入:** `prices = [7,6,4,3,1]`
- **输出:** 0
- **解释:** 在这种情况下, 交易无法获得正利润，所以不参与交易可以获得最大利润，最大利润为 0 。

**提示:**

- `1 <= prices.length <= 3 * 10^4`
- `0 <= prices[i] <= 10^4`

## 题解

### 这题和上一题最根本的区别

上一题只能交易一次，所以核心是找一组“最低买入价 + 最高卖出价”。

这一题不一样。  
题目允许你进行多次交易，只要任何时候手里最多持有一股就行。

这意味着，问题已经不再是“找一次最好的买卖”，而是：

**只要有利润可赚，就应该尽量把它都拿到。**

这就是这题和上一题思路完全不同的地方。

### 别把它想成“挑几段最赚钱的区间”

很多人第一次看到“可以多次交易”，会下意识去想：

我要不要找几个上涨区间，再从里面挑最划算的组合？

其实没必要把问题想得这么重。

因为这题里最关键的一点是：  
**一段连续上涨带来的总利润，等于这段里每天上涨利润的总和。**

比如价格是：

```text
1 -> 2 -> 3 -> 5
```

如果你从 `1` 买入、`5` 卖出，利润是：

```text
5 - 1 = 4
```

但这个 `4`，其实也完全等于：

```text
(2 - 1) + (3 - 2) + (5 - 3) = 1 + 1 + 2 = 4
```

这说明一件很重要的事：

**只要今天比昨天贵，那么这一天的上涨就一定可以被我们赚到。**

一旦这个结论成立，题目就已经可以直接贪心做了。

### 为什么把所有正利润加起来就是最优答案

这题最容易让人犹豫的地方，是：  
这样一天一天地加，会不会把交易拆得太碎，反而影响总利润？

答案是不会。

假设一段价格是持续上涨的：

```text
1 -> 3 -> 5
```

你可以理解成：

- 第一天买，第三天卖，利润是 `5 - 1 = 4`

也可以理解成：

- 第一天买，第二天卖，赚 `2`
- 第二天再买，第三天再卖，赚 `2`

总利润还是 `4`。

也就是说，在连续上涨的区间里：

- 一次性持有到最后卖出，可以
- 每天都做一次短交易，也可以

这两种做法的总收益完全一样。

既然一样，那最简单的做法当然就是：  
**把所有上涨的那一小段利润全部累加起来。**

### 什么时候加，什么时候跳过

所以遍历数组时，其实只需要看相邻两天的价格关系：

- 如果 `prices[i] > prices[i - 1]`，说明今天比昨天贵，这一段利润应该拿走
- 如果 `prices[i] <= prices[i - 1]`，说明今天没涨，那就不用操作

说得再直白一点：

**只赚上涨，不碰下跌。**

因为下跌的部分不会给你带来正收益，硬参与进去只会拉低答案。

### 用示例走一遍

以示例 1 为例：

```text
prices = [7,1,5,3,6,4]
```

从第二天开始看：

- `1 - 7 < 0`，不赚
- `5 - 1 > 0`，赚 `4`
- `3 - 5 < 0`，不赚
- `6 - 3 > 0`，赚 `3`
- `4 - 6 < 0`，不赚

最后总利润就是：

```text
4 + 3 = 7
```

这和题目给出的最优方案一致。

再看示例 2：

```text
prices = [1,2,3,4,5]
```

每天都在涨，那就把每天的差价全加起来：

```text
(2-1) + (3-2) + (4-3) + (5-4) = 4
```

这和从 `1` 一直拿到 `5` 再卖出的利润也是一样的。

### 为什么下跌时什么都不做

像示例 3：

```text
prices = [7,6,4,3,1]
```

每天都比前一天低，没有任何一天的差价是正数。

那就说明这段过程中根本没有利润可以拿。  
这时候最优策略当然就是什么都不做，答案自然是 `0`。

### 代码

```java
public int maxProfit(int[] prices) {
    int totalProfit = 0;

    for (int i = 1; i < prices.length; i++) {
        if (prices[i] > prices[i - 1]) {
            totalProfit += prices[i] - prices[i - 1];
        }
    }

    return totalProfit;
}
```

时间复杂度是 $O(n)$，因为整个数组只遍历了一遍。  
空间复杂度是 $O(1)$，因为只用了一个额外变量。

### 这题真正值得记住的地方

这题最值得记住的，不只是“股票 II 用贪心”，而是背后的拆分方式：

**如果一段总收益可以被拆成若干段彼此独立的正收益之和，那么最优策略往往就是把每一段局部正收益都收下。**

在这题里，这种拆分恰好成立，所以我们根本不需要纠结“这一段要不要持有到最后”，因为连续上涨区间的总利润，拆不拆都一样。

真正该抓住的是这个判断：

- 只要今天比昨天高，这部分差价就不会亏
- 所有这样的正差价加起来，就是最大利润

### 完整代码

```java
import java.util.Arrays;

public class Main {

    public static int maxProfit(int[] prices) {
        int totalProfit = 0;

        for (int i = 1; i < prices.length; i++) {
            if (prices[i] > prices[i - 1]) {
                totalProfit += prices[i] - prices[i - 1];
            }
        }

        return totalProfit;
    }

    public static void main(String[] args) {
        int[] prices1 = new int[]{7, 1, 5, 3, 6, 4};

        System.out.println("示例 1 输入:");
        System.out.println("prices = " + Arrays.toString(prices1));

        int result1 = maxProfit(prices1);

        System.out.println("示例 1 输出: " + result1);

        int[] prices2 = new int[]{1, 2, 3, 4, 5};

        System.out.println("示例 2 输入:");
        System.out.println("prices = " + Arrays.toString(prices2));

        int result2 = maxProfit(prices2);

        System.out.println("示例 2 输出: " + result2);

        int[] prices3 = new int[]{7, 6, 4, 3, 1};

        System.out.println("示例 3 输入:");
        System.out.println("prices = " + Arrays.toString(prices3));

        int result3 = maxProfit(prices3);

        System.out.println("示例 3 输出: " + result3);
    }
}
```
