## 题目

**描述：**

给你一个整数数组 `prices` ，其中 `prices[i]` 表示某支股票第 `i` 天的价格。

在每一天，你可以决定是否购买和/或出售股票。你在任何时候 **最多** 只能持有 **一股** 股票。你也可以在 **同一天** 多次买卖该股票，但要确保你持有的股票不超过一股。

返回你能获得的 **最大利润** 。

**示例 1:**

- **输入:** `prices = [7,1,5,3,6,4]`
- **输出:** 7
- **解释:** 在第 2 天（股票价格 = 1）的时候买入，在第 3 天（股票价格 = 5）的时候卖出, 这笔交易所能获得利润 = 5-1 = 4 。随后，在第 4 天（股票价格 = 3）的时候买入，在第 5 天（股票价格 = 6）的时候卖出, 这笔交易所能获得利润 = 6-3 = 3 。最大总利润为 4 + 3 = 7 。

**示例 2:**

- **输入:** `prices = [1,2,3,4,5]`
- **输出:** 4
- **解释:** 在第 1 天（股票价格 = 1）的时候买入，在第 5 天 （股票价格 = 5）的时候卖出, 这笔交易所能获得利润 = 5-1 = 4 。最大总利润为 4 。

**示例 3:**

- **输入:** `prices = [7,6,4,3,1]`
- **输出:** 0
- **解释:** 在这种情况下, 交易无法获得正利润，所以不参与交易可以获得最大利润，最大利润为 0 。

**提示:**

- `1 <= prices.length <= 3 * 10^4`
- `0 <= prices[i] <= 10^4`

## 题解

这一题和上一题最根本的区别，是交易次数不再只限一次。上一题要找的是“一次买入和一次卖出之间的最大差值”，这一题则允许你做很多次交易，只要同一时刻手里别持有超过一股就行。这样一来，问题就不再是“挑哪一笔最赚”，而是“所有能拿到的正利润，能不能都收下”。

很多人第一次看到“可以多次交易”时，会下意识想去找几段上涨区间，再从里面挑最好的组合。其实没必要把问题想得这么重，因为连续上涨区间的总利润，本来就等于这段里每天上涨利润的总和。比如价格从 `1 -> 2 -> 3 -> 5`，如果从 `1` 买到 `5`，利润是 `4`；如果拆成 `(2 - 1) + (3 - 2) + (5 - 3)`，总和也还是 `4`。

这就说明，只要今天比昨天贵，那么这一天的上涨本身就一定值得拿走。也正因为如此，这题最自然的贪心策略就是：把所有正的相邻差价都加起来。遍历数组时，只要 `prices[i] > prices[i - 1]`，就把这部分差价记进答案；如果今天不比昨天贵，就什么都不做。

拿示例 `prices = [7,1,5,3,6,4]` 来看，从第二天开始逐个比较：`1 - 7` 是负数，不赚；`5 - 1` 是正数，赚 `4`；`3 - 5` 是负数，跳过；`6 - 3` 是正数，再赚 `3`；`4 - 6` 是负数，跳过。最后总利润就是 `4 + 3 = 7`。这和题目给出的最优方案完全一致。

这个写法之所以不会漏答案，关键就在于“连续上涨区间拆开以后，总利润不变”。既然一整段涨势无论你是持有到最后还是拆成多次短交易，得到的收益都一样，那最简单的写法当然就是把每天上涨的那一点点利润全都加起来。相反，遇到下跌时就别碰，因为那部分差价不可能给你带来正收益。

像示例 `prices = [7,6,4,3,1]` 这种一路下跌的情况，也就很好理解了：没有任何一天的差价是正数，所以什么都不做就是最优，答案自然是 `0`。

代码如下：

```java
public int maxProfit(int[] prices) {
    int totalProfit = 0;

    for (int i = 1; i < prices.length; i++) {
        if (prices[i] > prices[i - 1]) {
            totalProfit += prices[i] - prices[i - 1];
        }
    }

    return totalProfit;
}
```

时间复杂度是 $O(n)$，因为整个数组只遍历了一遍；空间复杂度是 $O(1)$，因为只用了一个额外变量。

这题比较适合记住的是这种拆分方式：如果一段总收益本来就能写成若干段彼此独立的正收益之和，那最优策略往往就是把这些局部正收益直接全部收下，不必再为“要不要持有到最后”纠结。

完整代码如下：

```java
import java.util.Arrays;

public class Main {

    public static int maxProfit(int[] prices) {
        int totalProfit = 0;

        for (int i = 1; i < prices.length; i++) {
            if (prices[i] > prices[i - 1]) {
                totalProfit += prices[i] - prices[i - 1];
            }
        }

        return totalProfit;
    }

    public static void main(String[] args) {
        int[] prices1 = new int[]{7, 1, 5, 3, 6, 4};

        System.out.println("示例 1 输入:");
        System.out.println("prices = " + Arrays.toString(prices1));

        int result1 = maxProfit(prices1);

        System.out.println("示例 1 输出: " + result1);

        int[] prices2 = new int[]{1, 2, 3, 4, 5};

        System.out.println("示例 2 输入:");
        System.out.println("prices = " + Arrays.toString(prices2));

        int result2 = maxProfit(prices2);

        System.out.println("示例 2 输出: " + result2);

        int[] prices3 = new int[]{7, 6, 4, 3, 1};

        System.out.println("示例 3 输入:");
        System.out.println("prices = " + Arrays.toString(prices3));

        int result3 = maxProfit(prices3);

        System.out.println("示例 3 输出: " + result3);
    }
}
```
