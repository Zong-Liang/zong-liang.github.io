## 题目

**描述：**

给你一个整数数组 `citations` ，其中 `citations[i]` 表示研究者的第 `i` 篇论文被引用的次数。计算并返回该研究者的 **`h` 指数**。

根据维基百科上 h 指数的定义：`h` 代表“高引用次数”，一名科研人员的 `h` **指数** 是指他（她）至少发表了 `h` 篇论文，并且 **至少** 有 `h` 篇论文被引用次数大于等于 `h` 。如果 `h` 有多种可能的值，**`h` 指数** 是其中最大的那个。

**示例 1:**

- **输入:** `citations = [3,0,6,1,5]`
- **输出:** 3
- **解释:** 给定数组表示研究者总共有 5 篇论文，每篇论文相应的被引用了 3, 0, 6, 1, 5 次。由于研究者有 3 篇论文每篇 **至少** 被引用了 3 次，其余两篇论文每篇被引用 **不多于** 3 次，所以她的 `h` 指数是 3。

**示例 2:**

- **输入:** `citations = [1,3,1]`
- **输出:** 1

**提示:**

- `n == citations.length`
- `1 <= n <= 5000`
- `0 <= citations[i] <= 1000`

## 题解

### 先别急着套定义，先把这题在问什么想清楚

这题表面上是在算一个指标，但它本质上只问一件事：

**有没有一个整数 `h`，能让你至少有 `h` 篇论文的引用次数都不少于 `h`。**

而且这个 `h` 不是随便找一个满足条件的值，而是要尽可能大。

所以这题真正的难点，不在公式本身，而在于怎么快速判断：

- 某个 `h` 到底可不可以
- 最大的那个 `h` 到底是多少

只要这个判断思路清楚了，后面的写法就顺了。

### 先看排序之后为什么会变简单

如果数组是乱的，你很难一眼看出“至少有多少篇论文引用次数不低于某个值”。

但一旦排好序，事情就会变得很直接。

比如把引用次数从小到大排好之后：

```text
[0,1,3,5,6]
```

你站在某个位置 `i` 上看，后面一共还剩下 `n - i` 篇论文。  
因为数组已经有序，所以这 `n - i` 篇论文的引用次数，都会 **大于等于** `citations[i]`。

这时候如果：

```text
citations[i] >= n - i
```

就说明后面这 `n - i` 篇论文，每一篇的引用次数都至少是 `n - i`。  
那 `h = n - i` 就成立。

而且由于我们是从左往右找，第一个满足条件的位置，对应的就是最大的 `h`。

### 为什么排序法是成立的

这题里有个很容易绕的点：  
为什么找到 `citations[i] >= n - i`，就可以直接返回 `n - i`？

原因就在于“后面有多少篇”这件事本身。

假设数组长度是 `n`。  
排好序后，从位置 `i` 到最后，一共有 `n - i` 篇论文。

如果当前这一篇的引用次数都已经不少于 `n - i`，那后面那些更大的数自然也都不少于 `n - i`。  
于是你就得到了：

- 至少有 `n - i` 篇论文
- 它们的引用次数都至少是 `n - i`

这正好就是 H 指数的定义。

所以排序法的核心不是“排序本身”，而是：

**排序之后，后缀区间的长度，恰好可以和当前引用次数直接比较。**

### 用示例走一遍排序法

以示例 1 为例：

```text
citations = [3,0,6,1,5]
```

先排序：

```text
[0,1,3,5,6]
```

数组长度 `n = 5`。

从左往右看：

- `i = 0` 时，`h = 5`，但 `citations[0] = 0 < 5`，不行
- `i = 1` 时，`h = 4`，但 `citations[1] = 1 < 4`，不行
- `i = 2` 时，`h = 3`，而 `citations[2] = 3 >= 3`，成立

所以答案就是 `3`。

这一步一旦找到，就不用继续往后看了。因为再往后，`h` 只会更小，不可能得到更大的 H 指数。

### 排序法不难，但这题还能再快一点

排序法已经很自然了，时间复杂度是 $O(n \log n)$，对这题其实也完全够用。

但这题还有一个很适合利用的性质：

**H 指数最大不会超过论文总数 `n`。**

这个结论非常关键。

比如你有 5 篇论文，那 H 指数最多就是 5，不可能是 6。  
因为 H 指数要求“至少有 `h` 篇论文”，你总共都没这么多论文，当然不可能超过 `n`。

这就意味着：

- 引用次数是 `20` 也好
- 引用次数是 `200` 也好
- 对 H 指数来说，只要它大于等于 `n`，作用其实都一样

因为它们都只能被算进“足够大”的那一档。

### 这样就可以不用真的排序了

既然 H 指数只关心 `0` 到 `n` 这个范围，那我们完全可以统计：

- 引用次数等于 `0` 的有多少篇
- 等于 `1` 的有多少篇
- ...
- 大于等于 `n` 的统一记到 `n` 这一档

也就是说，可以开一个长度为 `n + 1` 的计数数组 `counter`：

- `counter[i]` 表示引用次数恰好为 `i` 的论文有多少篇
- `counter[n]` 表示引用次数大于等于 `n` 的论文有多少篇

然后从大到小累加：

- 从 `n` 开始往下看
- 统计“引用次数至少为 `i` 的论文一共有多少篇”
- 一旦这个数量 `total` 满足 `total >= i`，那当前的 `i` 就是答案

因为我们是从大往小试的，所以第一个满足条件的就是最大的 H 指数。

### 为什么从后往前累加就能找到答案

这个过程本质上是在问：

**有多少篇论文的引用次数至少是 `i`？**

如果从大到小累加 `counter`，那累计和 `total` 表示的，恰好就是这个量。

比如你现在扫到 `i = 3`，那已经加进来的就是：

- 引用 3 次的
- 引用 4 次的
- ...
- 引用至少 `n` 次的

所以这时候的 `total`，就是“引用次数至少为 3 的论文数量”。

如果这时 `total >= 3`，那就说明至少有 3 篇论文被引用至少 3 次，H 指数至少可以取 3。  
而由于你是从高往低找，第一个满足的自然就是最大值。

### 用示例 1 走一遍计数法

还是看：

```text
citations = [3,0,6,1,5]
```

总共有 `n = 5` 篇论文，所以只需要统计 `0` 到 `5`。

注意 `6` 已经大于 `5`，所以它也记到 `5` 这一档里。

统计结果可以理解成：

```text
counter[0] = 1
counter[1] = 1
counter[2] = 0
counter[3] = 1
counter[4] = 0
counter[5] = 2
```

然后从后往前累加：

- `i = 5`，`total = 2`，不够
- `i = 4`，`total = 2`，不够
- `i = 3`，`total = 3`，够了

所以答案就是 `3`。

### 代码

```java
public int hIndex(int[] citations) {
    int n = citations.length;
    int[] counter = new int[n + 1];

    for (int c : citations) {
        if (c >= n) {
            counter[n]++;
        } else {
            counter[c]++;
        }
    }

    int total = 0;
    for (int i = n; i >= 0; i--) {
        total += counter[i];
        if (total >= i) {
            return i;
        }
    }

    return 0;
}
```

时间复杂度是 $O(n)$，因为只做了两次线性遍历。  
空间复杂度是 $O(n)$，因为额外开了一个长度为 `n + 1` 的计数数组。

### 这题真正值得记住的地方

这题最值得记住的，不只是“H 指数怎么写”，而是这种处理方式：

**当题目要找的是某个满足条件的最大值，而且这个值本身有明确上界时，可以考虑直接围绕这个上界做计数，而不是老老实实排序。**

这题里，引用次数本身可以很大，但 H 指数最多只会到 `n`。  
一旦抓住这个上界，问题就从“处理原始数据”变成了“统计每个等级上有多少篇论文”。

这也是为什么计数法能把时间复杂度压到线性。

### 完整代码

```java
import java.util.Arrays;

public class Main {

    public static int hIndex(int[] citations) {
        int n = citations.length;
        int[] counter = new int[n + 1];

        for (int c : citations) {
            if (c >= n) {
                counter[n]++;
            } else {
                counter[c]++;
            }
        }

        int total = 0;
        for (int i = n; i >= 0; i--) {
            total += counter[i];
            if (total >= i) {
                return i;
            }
        }

        return 0;
    }

    public static int hIndexSorted(int[] citations) {
        Arrays.sort(citations);
        int n = citations.length;

        for (int i = 0; i < n; i++) {
            int h = n - i;
            if (citations[i] >= h) {
                return h;
            }
        }

        return 0;
    }

    public static void main(String[] args) {
        int[] citations1 = new int[]{3, 0, 6, 1, 5};

        System.out.println("示例 1 输入:");
        System.out.println("citations = " + Arrays.toString(citations1));

        int result1 = hIndex(citations1);

        System.out.println("示例 1 输出: " + result1);

        int[] citations2 = new int[]{1, 3, 1};

        System.out.println("示例 2 输入:");
        System.out.println("citations = " + Arrays.toString(citations2));

        int result2 = hIndex(citations2);

        System.out.println("示例 2 输出: " + result2);
    }
}
```
