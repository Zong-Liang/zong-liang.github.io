## 题目

**描述：**

给你一个整数数组 `citations` ，其中 `citations[i]` 表示研究者的第 `i` 篇论文被引用的次数。计算并返回该研究者的 **`h` 指数**。

根据维基百科上 h 指数的定义：`h` 代表“高引用次数”，一名科研人员的 `h` **指数** 是指他（她）至少发表了 `h` 篇论文，并且 **至少** 有 `h` 篇论文被引用次数大于等于 `h` 。如果 `h` 有多种可能的值，**`h` 指数** 是其中最大的那个。

**示例 1:**

- **输入:** `citations = [3,0,6,1,5]`
- **输出:** 3
- **解释:** 给定数组表示研究者总共有 5 篇论文，每篇论文相应的被引用了 3, 0, 6, 1, 5 次。由于研究者有 3 篇论文每篇 **至少** 被引用了 3 次，其余两篇论文每篇被引用 **不多于** 3 次，所以她的 `h` 指数是 3。

**示例 2:**

- **输入:** `citations = [1,3,1]`
- **输出:** 1

**提示:**

- `n == citations.length`
- `1 <= n <= 5000`
- `0 <= citations[i] <= 1000`

## 题解

这题表面上是在算一个指标，但本质上只是在问：有没有一个整数 `h`，能让你至少有 `h` 篇论文的引用次数都不少于 `h`，并且这个 `h` 还要尽可能大。只要把这个判断条件看清楚，后面的写法就不难了。

最自然的想法是先排序。因为数组一旦从小到大排好，站在某个位置 `i` 上往后看，后面一共还有 `n - i` 篇论文，而这些论文的引用次数都会大于等于 `citations[i]`。这时候如果 `citations[i] >= n - i`，就说明至少有 `n - i` 篇论文的引用次数都不少于 `n - i`，也就是 `h = n - i` 成立了。

拿示例 `citations = [3,0,6,1,5]` 来看，排序后得到 `[0,1,3,5,6]`。数组长度是 `5`。从左往右检查：当 `i = 0` 时，后面共有 `5` 篇论文，但 `0 < 5`；当 `i = 1` 时，后面共有 `4` 篇论文，但 `1 < 4`；当 `i = 2` 时，后面共有 `3` 篇论文，而 `citations[2] = 3`，这时条件刚好成立，所以答案就是 `3`。排序法的关键不在排序本身，而在于排序以后，“后缀长度”这件事可以直接拿来和当前值比较。

不过这题还能再快一点，因为 H 指数本身有一个很重要的上界：它不可能超过论文总数 `n`。如果你总共只有 `5` 篇论文，那 H 指数就不可能是 `6`。这意味着对于 H 指数来说，引用次数是 `20` 还是 `200` 其实没有本质区别，只要它大于等于 `n`，都可以归到同一档里。

一旦抓住这个上界，问题就能从排序转成计数。我们只需要统计引用次数落在 `0` 到 `n` 之间各有多少篇，其中所有大于等于 `n` 的论文统一记到 `counter[n]` 这一档。接下来从大到小累加这个计数数组：当扫到 `i` 时，累计和 `total` 表示的就是“引用次数至少为 `i` 的论文有多少篇”。一旦 `total >= i`，当前的 `i` 就是答案，而且由于我们是从大往小试的，第一个满足条件的也一定就是最大的 H 指数。

还是看 `citations = [3,0,6,1,5]`。因为 `n = 5`，所以只需要关心 `0` 到 `5` 这几个档位，其中 `6` 也会被记到 `5` 这一档。统计后，从后往前累加：扫到 `5` 时，至少被引用 `5` 次的论文有 `2` 篇，不够；扫到 `4` 时，仍然只有 `2` 篇，不够；扫到 `3` 时，累计变成 `3` 篇，这时条件成立，所以答案就是 `3`。

代码如下：

```java
public int hIndex(int[] citations) {
    int n = citations.length;
    int[] counter = new int[n + 1];

    for (int c : citations) {
        if (c >= n) {
            counter[n]++;
        } else {
            counter[c]++;
        }
    }

    int total = 0;
    for (int i = n; i >= 0; i--) {
        total += counter[i];
        if (total >= i) {
            return i;
        }
    }

    return 0;
}
```

时间复杂度是 $O(n)$，因为只做了两次线性遍历；空间复杂度是 $O(n)$，因为额外开了一个长度为 `n + 1` 的计数数组。

这题比较适合记住的是这种处理方式：如果题目要找的是某个满足条件的最大值，而且这个值本身有明确上界，就可以试着围绕这个上界做计数，而不是先把所有原始数据老老实实排好序。

完整代码如下：

```java
import java.util.Arrays;

public class Main {

    public static int hIndex(int[] citations) {
        int n = citations.length;
        int[] counter = new int[n + 1];

        for (int c : citations) {
            if (c >= n) {
                counter[n]++;
            } else {
                counter[c]++;
            }
        }

        int total = 0;
        for (int i = n; i >= 0; i--) {
            total += counter[i];
            if (total >= i) {
                return i;
            }
        }

        return 0;
    }

    public static int hIndexSorted(int[] citations) {
        Arrays.sort(citations);
        int n = citations.length;

        for (int i = 0; i < n; i++) {
            int h = n - i;
            if (citations[i] >= h) {
                return h;
            }
        }

        return 0;
    }

    public static void main(String[] args) {
        int[] citations1 = new int[]{3, 0, 6, 1, 5};

        System.out.println("示例 1 输入:");
        System.out.println("citations = " + Arrays.toString(citations1));

        int result1 = hIndex(citations1);

        System.out.println("示例 1 输出: " + result1);

        int[] citations2 = new int[]{1, 3, 1};

        System.out.println("示例 2 输入:");
        System.out.println("citations = " + Arrays.toString(citations2));

        int result2 = hIndex(citations2);

        System.out.println("示例 2 输出: " + result2);
    }
}
```
