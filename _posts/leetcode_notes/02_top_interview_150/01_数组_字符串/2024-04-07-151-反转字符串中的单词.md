## 题目

**描述：**

给你一个字符串 `s` ，请你反转字符串中 **单词** 的顺序。

**单词** 是由非空格字符组成的字符串。`s` 中使用至少一个空格将字符串中的 **单词** 分隔开。

返回 **单词** 顺序颠倒且 **单词** 之间用单个空格连接的结果字符串。

**注意：** 输入字符串 `s` 中可能会存在前导空格、尾随空格或者单词间的多个空格。返回的结果字符串中，单词间应当仅用单个空格分隔，且不包含任何额外的空格。

**示例 1:**

- **输入:** `s = "the sky is blue"`
- **输出:** `"blue is sky the"`

**示例 2:**

- **输入:** `s = "  hello world  "`
- **输出:** `"world hello"`
- **解释:** 反转后的字符串中不能存在前导空格和尾随空格。

**示例 3:**

- **输入:** `s = "a good   example"`
- **输出:** `"example good a"`
- **解释:** 如果两个单词间有多余的空格，反转后的字符串需要将单词间的空格减少到仅有一个。

**提示:**

- `1 <= s.length <= 10^4`
- `s` 包含英文大小写字母、数字和空格 `' '`
- `s` 中 **至少存在一个** 单词

**进阶：** 如果字符串在你使用的编程语言中是一种可变数据类型，请尝试使用 `O(1)` 额外空间复杂度的 **原地** 解法。

## 题解

### 这题真正麻烦的，不是反转，而是空格

如果题目只是让你把单词顺序倒过来，其实并不复杂。  
真正麻烦的是，题目还额外要求了三件事：

- 前导空格不能保留
- 尾随空格不能保留
- 单词之间如果有多个空格，结果里也只能剩一个

所以这题不能简单理解成“把字符串倒过来”。  
它真正要做的是：

**先把字符串整理成规范的单词序列，再把这些单词的顺序反过来。**

### 最直观的做法其实很简单

如果站在“把题做出来”的角度，最自然的思路就是：

1. 先去掉首尾空格
2. 再按一个或多个空格把字符串切成单词
3. 最后倒序拼接，并且单词之间只放一个空格

比如：

```text
"  hello   world  "
```

处理后先得到：

```text
["hello", "world"]
```

再倒过来拼接，就是：

```text
"world hello"
```

这个思路本身没有问题，而且很符合题意。  
因为题目最后要的，本来就不是原字符串的某种“原地反转结果”，而是一个新的、规范化后的字符串。

### 为什么 `trim + split` 能顺手把空格问题一起解决

这里最关键的是：

```java
s.trim().split("\\s+")
```

这两步分别在做两件事：

- `trim()`：去掉首尾空格
- `split("\\s+")`：按一个或多个空白字符切分

这样一来：

- 首尾多余空格没了
- 中间连续的多个空格也不会留下空字符串
- 数组里只会剩下真正的单词

所以这题里最容易写乱的“空格处理”，其实已经被这一步顺手解决掉了。

### 用示例走一遍

先看：

```text
s = "the sky is blue"
```

切分后得到：

```text
["the", "sky", "is", "blue"]
```

倒序拼接后变成：

```text
"blue is sky the"
```

再看：

```text
s = "  hello world  "
```

先 `trim()` 掉两边空格，再切分：

```text
["hello", "world"]
```

倒过来就是：

```text
"world hello"
```

再看：

```text
s = "a good   example"
```

中间虽然有多个空格，但 `split("\\s+")` 只会把它当成一个分隔符，所以切分结果仍然是：

```text
["a", "good", "example"]
```

倒序后得到：

```text
"example good a"
```

可以看到，这种写法不需要你手动一个个处理空格，逻辑会干净很多。

### 代码

```java
public String reverseWords(String s) {
    String[] words = s.trim().split("\\s+");
    StringBuilder sb = new StringBuilder();

    for (int i = words.length - 1; i >= 0; i--) {
        sb.append(words[i]);
        if (i > 0) {
            sb.append(" ");
        }
    }

    return sb.toString();
}
```

时间复杂度是 $O(n)$，因为 `trim`、`split` 和最后的拼接本质上都只是在做线性处理。  
空间复杂度是 $O(n)$，因为需要存储拆分后的单词数组和结果字符串。

### 这题如果继续往下想，核心其实是“三步走”

题目里有个进阶，问能不能做到原地处理。

如果不借助现成的 `split`，这题背后最经典的思路其实是：

1. 先去掉多余空格
2. 再把整个字符串反转
3. 最后把每个单词内部再反转回来

为什么这样可行？

因为整体反转之后，单词顺序已经倒过来了，只是每个单词内部的字符也被倒过来了。  
所以再把每个单词单独反转一次，单词本身就恢复正常了。

比如：

```text
"the sky is blue"
```

整体反转后是：

```text
"eulb si yks eht"
```

再把每个单词分别反转，就会得到：

```text
"blue is sky the"
```

这个思路更适合拿来理解题目的结构。  
但在 Java 里，`String` 本身不可变，所以真要写的时候，用 `trim + split + 倒序拼接` 通常会更自然。

### 这题真正值得记住的地方

这题最值得记住的，不是“反转单词顺序怎么写”，而是这个判断：

**只要题目对空格格式有额外要求，就不要急着做主逻辑，先想清楚输入该怎么规范化。**

这题里：

- 单词顺序反转只是目标
- 去掉首尾空格、压缩中间空格，才是前置条件

很多字符串题其实都一样。  
主逻辑不一定复杂，真正容易出错的，往往是这些边角格式。

### 完整代码

```java
public class Main {

    public static String reverseWords(String s) {
        String[] words = s.trim().split("\\s+");
        StringBuilder sb = new StringBuilder();

        for (int i = words.length - 1; i >= 0; i--) {
            sb.append(words[i]);
            if (i > 0) {
                sb.append(" ");
            }
        }

        return sb.toString();
    }

    public static void main(String[] args) {
        String s1 = "the sky is blue";
        System.out.println("示例 1 输入: \"" + s1 + "\"");
        System.out.println("示例 1 输出: \"" + reverseWords(s1) + "\"");

        String s2 = "  hello world  ";
        System.out.println("示例 2 输入: \"" + s2 + "\"");
        System.out.println("示例 2 输出: \"" + reverseWords(s2) + "\"");

        String s3 = "a good   example";
        System.out.println("示例 3 输入: \"" + s3 + "\"");
        System.out.println("示例 3 输出: \"" + reverseWords(s3) + "\"");
    }
}
```
