## 题目

**描述：**

给定一个大小为 `n` 的数组 `nums` ，返回其中的 **多数元素** 。多数元素是指在数组中出现次数 **大于** `⌊ n/2 ⌋` 的元素。

你可以假设数组是非空的，并且给定的数组总是存在多数元素。

**示例 1:**

- **输入:** `nums = [3,2,3]`
- **输出:** 3

**示例 2:**

- **输入:** `nums = [2,2,1,1,1,2,2]`
- **输出:** 2

**提示:**

- `n == nums.length`
- `1 <= n <= 5 * 10^4`
- `-10^9 <= nums[i] <= 10^9`
- 输入保证数组中一定有一个多数元素。

**进阶：** 尝试设计时间复杂度为 O(n)、空间复杂度为 O(1) 的算法解决此问题。

## 题解

这题如果顺着题面去想，最直接的办法当然是统计次数：用 `HashMap` 记录每个数出现了多少次，再找出那个出现次数大于 `n/2` 的元素。这个做法没有问题，但题目后面专门问了进阶，要求时间复杂度 $O(n)$、额外空间复杂度 $O(1)$，说明它真正想考的不是“会不会计数”，而是能不能把“多数元素一定存在”这个条件用起来。

题目说，多数元素的出现次数严格大于 `⌊n/2⌋`。这句话其实已经很强了，它等价于说：这个元素的数量比数组里其他所有元素加起来还多。只要这个条件成立，就会有一个很关键的结论：如果把多数元素和非多数元素一对一配对抵消，最后剩下来的仍然只能是多数元素。

摩尔投票法做的就是这件事。整个过程只维护两个变量：`candidate` 表示当前候选人，`count` 表示这个候选人在已经扫描过的部分里还剩多少“净优势”。遍历数组时，如果 `count == 0`，说明前面这一段已经抵消干净了，就把当前元素设成新的候选人；如果当前元素等于 `candidate`，`count++`；如果不等于 `candidate`，`count--`。相同元素在给候选人加票，不同元素在把这张票抵消掉。

很多人第一次看这题时，会对“票数归零就换候选人”这一步不太放心。其实 `count == 0` 并不表示前面的候选人一定不是答案，它只表示在当前已经扫描过的这一段里，这个候选人和其他元素已经互相抵消完了。既然这部分已经没有净贡献，后面能决定结果的就只剩尚未遍历到的元素，所以从当前位置重新选候选人是安全的。

拿示例 2 走一遍会更清楚。`nums = [2,2,1,1,1,2,2]`。一开始 `count = 0`，看到第一个 `2` 时，把它设成候选人，`count = 1`；再看到一个 `2`，因为和候选人相同，`count = 2`；接着看到两个 `1`，它们会把这两票慢慢抵消掉，于是 `count` 重新回到 `0`。这表示前面这一段已经抵消干净了，所以下一个元素 `1` 可以暂时成为新候选人。但再往后看到 `2` 时，它又会继续把 `1` 的优势抵消掉，最后剩下来的候选人仍然是 `2`。中间候选人确实可能变化，但只要多数元素的条件成立，最后留下来的那个就不会错。

代码如下：

```java
public int majorityElement(int[] nums) {
    int count = 0;
    int candidate = 0;

    for (int num : nums) {
        if (count == 0) {
            candidate = num;
        }

        if (num == candidate) {
            count++;
        } else {
            count--;
        }
    }

    return candidate;
}
```

时间复杂度是 $O(n)$，因为整个数组只遍历一遍；空间复杂度是 $O(1)$，因为只用了两个额外变量。

这题还有一个经常会被顺手问到的细节：为什么最后不用再验证一次候选人是不是多数元素？原因就在题目条件里。它已经明确保证数组中一定存在多数元素，所以遍历结束后直接返回 `candidate` 就可以了；如果没有这个前提，摩尔投票法只能给出一个候选人，通常还要再统计一遍确认。

这题更适合记住的不是算法名字，而是那个判断本身：只要某个元素的数量严格多于其他所有元素的总和，就可以试着从“成对抵消”这个角度去想。

完整代码如下：

```java
import java.util.Arrays;

public class Main {

    public static int majorityElement(int[] nums) {
        int count = 0;
        int candidate = 0;

        for (int num : nums) {
            if (count == 0) {
                candidate = num;
            }

            if (num == candidate) {
                count++;
            } else {
                count--;
            }
        }

        return candidate;
    }

    public static void main(String[] args) {
        int[] nums1 = new int[]{3, 2, 3};

        System.out.println("示例 1 输入:");
        System.out.println("nums = " + Arrays.toString(nums1));

        int result1 = majorityElement(nums1);

        System.out.println("示例 1 输出: " + result1);

        int[] nums2 = new int[]{2, 2, 1, 1, 1, 2, 2};

        System.out.println("示例 2 输入:");
        System.out.println("nums = " + Arrays.toString(nums2));

        int result2 = majorityElement(nums2);

        System.out.println("示例 2 输出: " + result2);
    }
}
```
