## 题目

**描述：**

给定一个大小为 `n` 的数组 `nums` ，返回其中的 **多数元素** 。多数元素是指在数组中出现次数 **大于** `⌊ n/2 ⌋` 的元素。

你可以假设数组是非空的，并且给定的数组总是存在多数元素。

**示例 1:**

- **输入:** `nums = [3,2,3]`
- **输出:** 3

**示例 2:**

- **输入:** `nums = [2,2,1,1,1,2,2]`
- **输出:** 2

**提示:**

- `n == nums.length`
- `1 <= n <= 5 * 10^4`
- `-10^9 <= nums[i] <= 10^9`
- 输入保证数组中一定有一个多数元素。

**进阶：** 尝试设计时间复杂度为 O(n)、空间复杂度为 O(1) 的算法解决此问题。

## 题解

### 这题真正难的不是“找次数”，而是怎么在线性时间里做完

这题如果顺着题面去想，最容易冒出来的思路其实是统计次数：用 `HashMap` 记录每个数出现了多少次，然后找出次数最多、并且大于 `n/2` 的那个。

这个做法当然能做出来，但题目后面专门问了进阶：  
能不能做到时间复杂度 $O(n)$、空间复杂度 $O(1)$？

这就说明，题目真正想考的不是“会不会计数”，而是你能不能利用“多数元素一定存在”这个条件，把问题继续压缩。

### 先看一个很好理解的结论

题目说，多数元素的出现次数 **大于** `⌊n/2⌋`。

这句话其实很强，它意味着：

**多数元素的数量，比数组里所有其他元素加起来还要多。**

这不是“最多”，也不是“至少一半”，而是严格地超过一半。

只要这个条件成立，就会带来一个非常关键的推论：

**把多数元素和非多数元素一对一抵消，最后剩下来的，仍然会是多数元素。**

而摩尔投票法，核心就是在做这件事。

### 为什么“抵消”这件事是成立的

可以把数组里的元素分成两类：

- 一类是多数元素
- 另一类是所有其他元素

因为多数元素的个数严格大于一半，所以它的数量一定多于“其他所有元素的总数”。

那如果每次拿一个多数元素，去和一个非多数元素配对抵消，最后会发生什么？

非多数元素会先被抵消完，而多数元素还会有剩余。  
所以不管中间顺序怎么乱，最后能活下来的那个阵营，一定是多数元素。

这就是这题最核心的直觉。后面的代码，其实只是把这个过程用常数空间写出来。

### 摩尔投票法到底在维护什么

整个算法只维护两个变量：

- `candidate`：当前候选人
- `count`：当前候选人的“净票数”

这里的“净票数”不要理解成真实票数，它更像是一个抵消后的余额。

遍历数组时：

- 如果 `count == 0`，说明前面的候选人已经被抵消光了，那就把当前元素设成新的候选人
- 如果当前元素等于 `candidate`，说明它和候选人是同一阵营，`count++`
- 如果当前元素不等于 `candidate`，说明它在和候选人对冲，`count--`

你可以把它理解成：

**相同元素给候选人加一票，不同元素拿走一票。**

一旦票数归零，就说明这一段里谁也没占到便宜，需要重新换一个候选人继续往后看。

### 为什么票数归零就可以换人

这是很多人第一次看摩尔投票法时最不放心的地方。

为什么 `count` 变成 0，就能直接把前面的候选人扔掉？

因为 `count == 0` 的意思不是“这个数一定不是多数元素”，而是：

**在当前已经扫描过的这一段里，这个候选人和其他元素已经互相抵消干净了。**

既然这部分已经两两抵消完，那它对最终结果就不再提供额外信息了。  
接下来真正可能影响结果的，只剩后面的元素，所以可以直接从当前位置重新选候选人。

你不需要担心把真正的多数元素“误删”掉。  
如果它真的是多数元素，那么即便前面被抵消了一部分，后面也仍然会把它重新推回来。

### 用示例走一遍

以示例 2 为例：

```text
nums = [2,2,1,1,1,2,2]
```

初始时：

- `candidate` 还没定
- `count = 0`

看到第一个 `2` 时，`count == 0`，所以把 `2` 设为候选人。  
当前：

- `candidate = 2`
- `count = 1`

再看到一个 `2`，和候选人相同，`count++`：

- `candidate = 2`
- `count = 2`

再看到 `1`，和候选人不同，抵消一次，`count--`：

- `candidate = 2`
- `count = 1`

再看到一个 `1`，继续抵消：

- `candidate = 2`
- `count = 0`

到这里的意思是：前面这一段已经互相抵消干净了。

接下来再看到 `1`，因为 `count == 0`，所以把当前这个 `1` 临时设成新候选人：

- `candidate = 1`
- `count = 1`

再往后看到 `2`，抵消一次：

- `candidate = 1`
- `count = 0`

最后再看到一个 `2`，重新设候选人：

- `candidate = 2`
- `count = 1`

遍历结束，最终候选人是 `2`，也就是答案。

中间候选人确实可能变化，但最后留下来的那个，一定是多数元素。

### 代码

```java
public int majorityElement(int[] nums) {
    int count = 0;
    int candidate = 0;

    for (int num : nums) {
        if (count == 0) {
            candidate = num;
        }

        if (num == candidate) {
            count++;
        } else {
            count--;
        }
    }

    return candidate;
}
```

时间复杂度是 $O(n)$，因为整个数组只遍历一遍。  
空间复杂度是 $O(1)$，因为只用了两个额外变量。

### 为什么最后不用再验证一次

在有些题里，摩尔投票法跑完之后，还需要再遍历一遍，确认候选人是不是真的出现超过一半。

但这题不用。

因为题目已经明确保证：**数组中一定存在多数元素。**

这个前提非常重要。正是因为它已经被题目保证了，所以最后返回 `candidate` 就可以，不需要额外做验证。

如果没有这个前提，单靠摩尔投票法只能得到一个“候选人”，还不能百分之百保证它就是答案。

### 这题真正值得记住的地方

这题最值得记住的，不是“有个算法叫摩尔投票法”，而是它背后的判断方式：

**只要某个元素的数量严格多于其他所有元素总和，就可以考虑用“配对抵消”的思路处理。**

多数元素问题本质上不是计数，而是阵营对抗。  
谁能在不断抵消之后还留下来，谁就是答案。

所以这题真正应该记住的是这个直觉：

- 相同的元素累积优势
- 不同的元素相互抵消
- 优势大到超过总量一半的那个，最后一定还在

### 完整代码

```java
import java.util.Arrays;

public class Main {

    public static int majorityElement(int[] nums) {
        int count = 0;
        int candidate = 0;

        for (int num : nums) {
            if (count == 0) {
                candidate = num;
            }

            if (num == candidate) {
                count++;
            } else {
                count--;
            }
        }

        return candidate;
    }

    public static void main(String[] args) {
        int[] nums1 = new int[]{3, 2, 3};

        System.out.println("示例 1 输入:");
        System.out.println("nums = " + Arrays.toString(nums1));

        int result1 = majorityElement(nums1);

        System.out.println("示例 1 输出: " + result1);

        int[] nums2 = new int[]{2, 2, 1, 1, 1, 2, 2};

        System.out.println("示例 2 输入:");
        System.out.println("nums = " + Arrays.toString(nums2));

        int result2 = majorityElement(nums2);

        System.out.println("示例 2 输出: " + result2);
    }
}
```
