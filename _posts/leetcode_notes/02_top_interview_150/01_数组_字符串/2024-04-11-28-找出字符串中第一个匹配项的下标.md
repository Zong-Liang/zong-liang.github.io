## 题目

**描述：**

给你两个字符串 `haystack` 和 `needle` ，请你在 `haystack` 字符串中找出 `needle` 字符串的第一个匹配项的下标（下标从 0 开始）。如果 `needle` 不是 `haystack` 的一部分，则返回 `-1` 。

**示例 1:**

- **输入:** `haystack = "sadbutsad"`, `needle = "sad"`
- **输出:** 0
- **解释:** `"sad"` 在下标 0 和 6 处匹配。第一个匹配项的下标是 0 ，所以返回 0 。

**示例 2:**

- **输入:** `haystack = "leetcode"`, `needle = "leeto"`
- **输出:** -1
- **解释:** `"leeto"` 没有在 `"leetcode"` 中出现，所以返回 -1 。

**提示:**

- `1 <= haystack.length, needle.length <= 10^4`
- `haystack` 和 `needle` 仅由小写英文字符组成

## 题解

### 这题先别急着上 KMP，先把问题本身看清楚

题目要做的事其实很直接：

给你一个主串 `haystack` 和一个模式串 `needle`，问 `needle` 第一次出现在 `haystack` 的哪个位置。

所以最自然的想法就是：

**把 `needle` 放到 `haystack` 的每个可能起点上试一遍。**

这个思路本身没有任何问题，而且它其实已经很接近正确答案了。  
KMP 只是对这个过程做优化，不是另起炉灶的一套完全不同的东西。

### 最直观的做法，就是一个起点一个起点试

假设当前枚举到 `haystack` 的第 `i` 个位置，就把它当成一次匹配的起点。

接下来做的事情很简单：

- 用另一个指针 `j` 从 `0` 开始，去和 `needle[j]` 比较
- 如果 `haystack[i + j] == needle[j]`，就继续往后比
- 只要中间有一个字符不一样，这次尝试就失败
- 如果一路比到 `j == needle.length()`，说明整个 `needle` 都匹配上了，直接返回 `i`

这就是最朴素的匹配过程。

### 为什么暴力法虽然简单，但会有很多重复比较

问题出在“失败之后怎么办”。

比如：

```text
haystack = "aaab"
needle   = "aab"
```

先从下标 `0` 开始比：

- `a == a`
- `a == a`
- 第三个字符不一样，失败

这时候如果用暴力法，只能让起点往后挪一位，再从头开始比较。  
也就是说，前面那两个已经匹配过的 `a`，下一轮又会重新比一次。

这就是暴力法的低效之处：

**一旦失配，它不会利用已经匹配过的信息，只会老老实实从头再来。**

### 所以 KMP 真正解决的问题，其实不是“怎么匹配”，而是“失配后怎么少走回头路”

KMP 最核心的价值就在这里：

**匹配失败时，不让模式串从头重来，而是尽量利用已经匹配成功的那一段。**

这里最关键的问题是：

如果 `needle` 前面一部分已经匹配上了，现在某个位置失配了，那模式串能不能不用完全退回起点，而是直接跳到一个“还有希望”的位置继续比？

KMP 的答案是可以。  
而支撑这个跳转的，就是 `next` 数组。

### `next` 数组到底在记录什么

这个地方如果只背定义，很容易越看越糊。  
最稳的理解方式是：

**`next[i]` 记录的是：在 `needle[0...i]` 这一段里，最长的“相同前缀和后缀”的长度。**

这里的“前缀”和“后缀”都指真前缀、真后缀，也就是不能把整个字符串本身算进去。

比如字符串：

```text
"abab"
```

它的前缀有：

```text
"a", "ab", "aba"
```

后缀有：

```text
"b", "ab", "bab"
```

其中最长的相同前后缀是：

```text
"ab"
```

长度是 `2`。

所以如果某一段已经匹配到了 `"abab"`，结果下一个字符失配了，那就说明：

前面这段里，开头的 `"ab"` 和结尾的 `"ab"` 是一样的。  
既然这样，模式串其实没必要完全回退到头，而是可以直接把这段“还能接得上”的部分留下来，继续往后比。

这就是 `next` 数组存在的意义。

### KMP 在匹配时到底怎么用 `next`

匹配时我们用两个指针：

- `i` 指向 `haystack`
- `j` 指向 `needle`

如果：

```text
haystack[i] == needle[j]
```

那就继续往后走，`i++`，`j++`。

如果失配了：

- 暴力法会让 `i` 回退，然后 `j` 从 0 重新开始
- KMP 不回退 `i`
- 它只会根据 `next[j - 1]` 来调整 `j`

也就是说，主串已经走过的位置不再重看，模式串自己根据之前的匹配结构跳到一个更合理的位置。

所以 KMP 的本质可以概括成一句话：

**主串不回头，模式串按 `next` 数组跳。**

### 为什么 `j` 要跳到 `next[j - 1]`

假设当前已经匹配到了 `needle[j - 1]`，也就是前面 `j` 个字符都对上了，现在第 `j` 个字符失配。

这时候前面那一段已经匹配成功的内容，并不是全部作废。  
因为在这段成功匹配的部分里，可能存在一段“既是前缀又是后缀”的结构。

那模式串就可以把这段公共部分保留下来，直接从它后面继续尝试，而不是退回最开始。

所以：

```java
j = next[j - 1];
```

这句的含义其实就是：

**把模式串移动到“还能复用之前匹配结果”的位置。**

### 用示例 1 先从结果上理解

看这个例子：

```text
haystack = "sadbutsad"
needle = "sad"
```

从下标 `0` 开始：

- `s == s`
- `a == a`
- `d == d`

整个模式串都匹配完了，所以第一次匹配的起点就是 `0`。

这题里 KMP 当然能做出来，但其实连暴力法都已经足够顺。

所以这题真正值得学 KMP 的地方，不在这个例子，而在那些有大量重复前缀的情况里。

### 再看一个更能体现 KMP 价值的例子

比如：

```text
haystack = "aaaaab"
needle = "aaab"
```

暴力法会在多个起点上反复比较前面的 `a`。  
而 KMP 会发现：

模式串里 `"aaa"` 本身就带有重复前后缀结构，所以失配时不需要从头重新开始，可以直接跳过一部分重复工作。

这就是它能把复杂度压到线性的原因。

### 先把暴力法放在前面，其实更容易接受 KMP

如果上来就讲 KMP，很多人会觉得它像是凭空冒出来的技巧。  
但如果先把暴力法的过程想明白，再问一句：

**到底浪费在哪里？**

那 KMP 的作用就会变得非常自然。

它不是为了“写得高级”，而是专门解决这一类重复比较。

所以这题最好的理解顺序其实是：

1. 先知道暴力法在做什么
2. 再知道它为什么会重复比较
3. 最后再看 KMP 怎么避免这些重复

### 代码

```java
public int strStr(String haystack, String needle) {
    int[] next = new int[needle.length()];
    buildNext(next, needle);

    int j = 0;
    for (int i = 0; i < haystack.length(); i++) {
        while (j > 0 && haystack.charAt(i) != needle.charAt(j)) {
            j = next[j - 1];
        }

        if (haystack.charAt(i) == needle.charAt(j)) {
            j++;
        }

        if (j == needle.length()) {
            return i - needle.length() + 1;
        }
    }

    return -1;
}

private void buildNext(int[] next, String s) {
    int j = 0;
    next[0] = 0;

    for (int i = 1; i < s.length(); i++) {
        while (j > 0 && s.charAt(i) != s.charAt(j)) {
            j = next[j - 1];
        }

        if (s.charAt(i) == s.charAt(j)) {
            j++;
        }

        next[i] = j;
    }
}
```

时间复杂度是 $O(n + m)$，其中 `n` 是 `haystack` 的长度，`m` 是 `needle` 的长度。  
构造 `next` 数组要一遍，真正匹配再一遍。  
空间复杂度是 $O(m)$，因为需要一个长度为 `m` 的 `next` 数组。

### 这题真正值得记住的地方

这题最值得记住的，不是“KMP 公式怎么背”，而是这个更本质的判断：

**如果一个匹配过程里，失败后会反复重看已经比较过的内容，那就要想办法把这些已知信息存下来复用。**

这题里，这份“已知信息”就是：

- 模式串前面已经匹配成功的部分
- 其中还能继续复用的最长相同前后缀

KMP 本质上做的事情其实很朴素：

**尽量不让已经做过的比较白做。**

### 完整代码

```java
public class Main {

    public static int strStr(String haystack, String needle) {
        int[] next = new int[needle.length()];
        buildNext(next, needle);

        int j = 0;
        for (int i = 0; i < haystack.length(); i++) {
            while (j > 0 && haystack.charAt(i) != needle.charAt(j)) {
                j = next[j - 1];
            }

            if (haystack.charAt(i) == needle.charAt(j)) {
                j++;
            }

            if (j == needle.length()) {
                return i - needle.length() + 1;
            }
        }

        return -1;
    }

    private static void buildNext(int[] next, String s) {
        int j = 0;
        next[0] = 0;

        for (int i = 1; i < s.length(); i++) {
            while (j > 0 && s.charAt(i) != s.charAt(j)) {
                j = next[j - 1];
            }

            if (s.charAt(i) == s.charAt(j)) {
                j++;
            }

            next[i] = j;
        }
    }

    public static void main(String[] args) {
        String haystack1 = "sadbutsad";
        String needle1 = "sad";

        System.out.println("示例 1 输入: haystack = \"" + haystack1 + "\", needle = \"" + needle1 + "\"");
        System.out.println("示例 1 输出: " + strStr(haystack1, needle1));

        String haystack2 = "leetcode";
        String needle2 = "leeto";

        System.out.println("示例 2 输入: haystack = \"" + haystack2 + "\", needle = \"" + needle2 + "\"");
        System.out.println("示例 2 输出: " + strStr(haystack2, needle2));

        String haystack3 = "hello";
        String needle3 = "ll";

        System.out.println("额外测试 输入: haystack = \"" + haystack3 + "\", needle = \"" + needle3 + "\"");
        System.out.println("额外测试 输出: " + strStr(haystack3, needle3));
    }
}
```
