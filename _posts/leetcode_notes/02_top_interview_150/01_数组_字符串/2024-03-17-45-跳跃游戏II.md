## 题目

**描述：**

给定一个长度为 `n` 的 **0 索引** 整数数组 `nums`。初始位置在下标 0。

每个元素 `nums[i]` 表示从索引 `i` 向后跳转的最大长度。换句话说，如果你在索引 `i` 处，你可以跳转到任意 `(i + j)` 处:

- `0 <= j <= nums[i]` 且
- `i + j < n`

返回到达 `n - 1` 的最小跳跃次数。测试用例保证可以到达 `n - 1`。

**示例 1:**

- **输入:** `nums = [2,3,1,1,4]`
- **输出:** 2
- **解释:** 跳到最后一个位置的最小跳跃数是 2。从下标 0 跳到下标 1 的位置，跳 1 步，然后跳 3 步到达数组的最后一个位置。

**示例 2:**

- **输入:** `nums = [2,3,0,1,4]`
- **输出:** 2

**提示:**

- `1 <= nums.length <= 10^4`
- `0 <= nums[i] <= 1000`
- 题目保证可以到达 `n - 1`

## 题解

### 这题和上一题只差一个问题，但思路会更进一步

上一题问的是：**能不能到终点**。  
这一题问的是：**最少跳几次才能到终点**。

看起来只是把 `true / false` 换成了一个次数，但问题其实变了。

上一题里，我们只需要维护“当前最远能到哪”；  
这一题里，除了要看最远能到哪，还要知道：

**什么时候必须跳下一步。**

这就是这题真正的关键。

### 先别急着想“每一步跳到哪里最好”

很多人看到“最少跳几次”，第一反应会是：

- 这一跳落在哪个位置最优
- 要不要把所有可能的落点都比较一下
- 是不是要用 BFS 一层一层找最短路

这种想法不算错，但会把题做重。

因为这题虽然在问“最少步数”，但它其实仍然有很强的贪心结构。  
你不需要真的决定每一步具体落在哪个点上，你只需要关心一件事：

**在当前这一步能覆盖的范围内，下一步最远能扩到哪里。**

### 可以把它理解成按“层”往前推进

这题最自然的理解方式，其实很像按层扩展。

假设你当前跳了 `steps` 步，那么这 `steps` 步所能到达的位置，其实会形成一个区间。  
比如当前这一层能覆盖到：

```text
[0 ... end]
```

那你就把这个区间里的每个位置都看一遍，看看从这些位置再跳一步，最远能到哪里。这个“下一层”的最远边界，记成 `maxPosition`。

等你把当前区间扫完了，就说明：

- 当前这一步的能力已经用尽了
- 如果还想继续往前，就必须多跳一次
- 新的一层边界就是刚才统计出来的 `maxPosition`

所以这题要维护的其实是两个边界：

- `end`：当前步数下，最远能到哪里
- `maxPosition`：在当前范围内再跳一步，最远能到哪里

而 `steps` 记录的，就是你一共扩了多少层。

### 为什么这样做一定是最少步数

因为每次加一步，都是在“当前这一步能覆盖到的所有位置”里，把下一步的最远范围一次性算出来。

也就是说，你不是从某个单独位置贪心，而是在当前整层可达区间里，统一选出下一层最远边界。

这和 BFS 的“按层找最短路”本质上是一样的：

- 当前层里的所有点，都是用同样的步数到达的
- 只有把这一层全部看完，你才能知道下一层最远能扩到哪里
- 所以每次 `steps++`，都代表确实多跳了一次，而且这次增加是必要的

这就是为什么它虽然写法像贪心，但本质上是“用贪心方式实现按层推进”。

### 用示例 1 走一遍

看这个例子：

```text
nums = [2,3,1,1,4]
```

开始时：

- `end = 0`
- `maxPosition = 0`
- `steps = 0`

此时可以理解为：  
你还没有真正起跳，但当前这一层只覆盖下标 `0`。

#### 扫描下标 0

从下标 `0` 出发，最远能到：

```text
0 + nums[0] = 2
```

所以：

```text
maxPosition = 2
```

而现在 `i == end`，说明当前这一层已经扫完了。  
这意味着你必须跳一次，进入下一层：

- `steps = 1`
- `end = 2`

现在表示：  
用 1 步，最远可以覆盖到下标 `2`。

#### 扫描下标 1 和 2

接着继续看当前层 `[1, 2]` 里的位置。

先看下标 `1`：

```text
1 + nums[1] = 4
```

更新后：

```text
maxPosition = 4
```

再看下标 `2`：

```text
2 + nums[2] = 3
```

不如当前的 `4`，不用更新。

当扫到 `i == end == 2` 时，说明第二层也扫完了。  
于是再跳一次：

- `steps = 2`
- `end = 4`

而 `4` 已经是最后一个下标，所以答案就是 `2`。

### 为什么遍历时不用看最后一个元素

代码里有个很容易让人疑惑的细节：

```java
for (int i = 0; i < nums.length - 1; i++)
```

为什么不是遍历到最后，而是只到 `nums.length - 2`？

因为题目求的是“到达最后一个位置所需的最小跳跃次数”。

一旦你已经进入了包含最后一个下标的那一层，步数其实就已经确定了，不需要再从最后一个位置继续往外扩。

所以最后一个元素不用参与“更新下一层边界”的过程。

### 代码

```java
public int jump(int[] nums) {
    int end = 0;
    int maxPosition = 0;
    int steps = 0;

    for (int i = 0; i < nums.length - 1; i++) {
        maxPosition = Math.max(maxPosition, i + nums[i]);

        if (i == end) {
            end = maxPosition;
            steps++;
        }
    }

    return steps;
}
```

时间复杂度是 $O(n)$，因为整个数组只遍历了一遍。  
空间复杂度是 $O(1)$，因为只用了几个额外变量。

### 这题真正值得记住的地方

这题最值得记住的，不是“跳跃游戏 II 用贪心”这句话，而是这种思考方式：

**当题目要求最少步数，而每一步又能覆盖一整段范围时，可以把问题理解成按层推进。**

这题里：

- `end` 表示当前这一层的右边界
- `maxPosition` 表示下一层最远能扩到哪里
- 每次扫完整一层，步数就加一

所以它本质上不是在做“我下一跳该落在哪”，而是在做：

**我用当前这一步，能把下一步的活动范围推进到哪里。**

以后如果再遇到这种“最少次数 + 区间扩展”的题，也可以优先往这个方向想。

### 完整代码

```java
import java.util.Arrays;

public class Main {

    public static int jump(int[] nums) {
        int end = 0;
        int maxPosition = 0;
        int steps = 0;

        for (int i = 0; i < nums.length - 1; i++) {
            maxPosition = Math.max(maxPosition, i + nums[i]);

            if (i == end) {
                end = maxPosition;
                steps++;
            }
        }

        return steps;
    }

    public static void main(String[] args) {
        int[] nums1 = new int[]{2, 3, 1, 1, 4};

        System.out.println("示例 1 输入:");
        System.out.println("nums = " + Arrays.toString(nums1));

        int result1 = jump(nums1);

        System.out.println("示例 1 输出: " + result1);

        int[] nums2 = new int[]{2, 3, 0, 1, 4};

        System.out.println("示例 2 输入:");
        System.out.println("nums = " + Arrays.toString(nums2));

        int result2 = jump(nums2);

        System.out.println("示例 2 输出: " + result2);
    }
}
```
