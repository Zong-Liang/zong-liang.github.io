## 题目

**描述：**

给定一个长度为 `n` 的 **0 索引** 整数数组 `nums`。初始位置在下标 0。

每个元素 `nums[i]` 表示从索引 `i` 向后跳转的最大长度。换句话说，如果你在索引 `i` 处，你可以跳转到任意 `(i + j)` 处:

- `0 <= j <= nums[i]` 且
- `i + j < n`

返回到达 `n - 1` 的最小跳跃次数。测试用例保证可以到达 `n - 1`。

**示例 1:**

- **输入:** `nums = [2,3,1,1,4]`
- **输出:** 2
- **解释:** 跳到最后一个位置的最小跳跃数是 2。从下标 0 跳到下标 1 的位置，跳 1 步，然后跳 3 步到达数组的最后一个位置。

**示例 2:**

- **输入:** `nums = [2,3,0,1,4]`
- **输出:** 2

**提示:**

- `1 <= nums.length <= 10^4`
- `0 <= nums[i] <= 1000`
- 题目保证可以到达 `n - 1`

## 题解

上一题问的是能不能到终点，这一题问的是最少跳几次才能到终点。看起来只是把 `true / false` 换成了一个次数，但思路其实要再往前走一步：除了知道当前最远能到哪里，还要知道什么时候必须多跳一次。

很多人一看到“最少步数”，就会开始想这一跳到底落在哪个位置最优，甚至把它往 BFS 或最短路上靠。这样想不算错，但会把题做重。更顺手的理解方式是把它看成按层往前推进：假设你当前已经跳了 `steps` 步，那么这 `steps` 步能到达的位置其实会形成一个区间。你要做的不是在区间里选一个最优落点，而是把这个区间里的所有位置都看一遍，看看再多跳一步，最远能扩到哪里。

于是就有了三个变量：`end` 表示当前这一步能覆盖到的右边界，`maxPosition` 表示在当前范围里再跳一步最远能到哪里，`steps` 记录目前一共跳了几次。遍历数组时，不断用 `i + nums[i]` 去更新 `maxPosition`；而一旦扫到了当前层的右边界 `end`，就说明这一层已经看完了，如果还要继续往前，就必须多跳一次，这时令 `steps++`，并把 `end` 更新成 `maxPosition`。

这样做之所以是最少步数，本质上和 BFS 按层找最短路是一样的。当前层里的所有位置，都是用同样的跳数到达的；只有把这一层全部扫完，才知道下一层最远能扩到哪里。所以每次 `steps++` 都不是随便加的，而是真的跨进了下一层。

拿 `nums = [2,3,1,1,4]` 走一遍。开始时 `end = 0`，`maxPosition = 0`，`steps = 0`，可以理解成还没起跳，当前层只覆盖下标 `0`。扫描下标 `0` 时，最远能扩到 `2`，所以 `maxPosition = 2`。此时 `i == end`，说明第一层已经扫完，必须跳一次，于是 `steps = 1`，`end = 2`。接着继续扫描下标 `1` 和 `2`，其中下标 `1` 能把 `maxPosition` 更新到 `4`。等扫到 `i == end == 2` 时，第二层也结束了，于是再跳一次，`steps = 2`，`end = 4`。而 `4` 已经是最后一个下标，所以答案就是 `2`。

代码里还有一个容易忽略的小细节。

```java
for (int i = 0; i < nums.length - 1; i++)
```

遍历时并不需要走到最后一个元素。因为题目求的是“到达最后一个位置所需的最小跳跃次数”，只要最后一个下标已经被纳入当前层或下一层的覆盖范围，步数就已经确定了，没必要再从最后一个位置继续往外扩展。

代码如下：

```java
public int jump(int[] nums) {
    int end = 0;
    int maxPosition = 0;
    int steps = 0;

    for (int i = 0; i < nums.length - 1; i++) {
        maxPosition = Math.max(maxPosition, i + nums[i]);

        if (i == end) {
            end = maxPosition;
            steps++;
        }
    }

    return steps;
}
```

时间复杂度是 $O(n)$，因为整个数组只遍历了一遍；空间复杂度是 $O(1)$，因为只用了几个额外变量。

这题比较值得记的是这个转换方式：当题目要求最少步数，而每一步又天然能覆盖一整段范围时，可以先别急着想“下一跳具体落在哪”，而是把它理解成按层推进。

完整代码如下：

```java
import java.util.Arrays;

public class Main {

    public static int jump(int[] nums) {
        int end = 0;
        int maxPosition = 0;
        int steps = 0;

        for (int i = 0; i < nums.length - 1; i++) {
            maxPosition = Math.max(maxPosition, i + nums[i]);

            if (i == end) {
                end = maxPosition;
                steps++;
            }
        }

        return steps;
    }

    public static void main(String[] args) {
        int[] nums1 = new int[]{2, 3, 1, 1, 4};

        System.out.println("示例 1 输入:");
        System.out.println("nums = " + Arrays.toString(nums1));

        int result1 = jump(nums1);

        System.out.println("示例 1 输出: " + result1);

        int[] nums2 = new int[]{2, 3, 0, 1, 4};

        System.out.println("示例 2 输入:");
        System.out.println("nums = " + Arrays.toString(nums2));

        int result2 = jump(nums2);

        System.out.println("示例 2 输出: " + result2);
    }
}
```
