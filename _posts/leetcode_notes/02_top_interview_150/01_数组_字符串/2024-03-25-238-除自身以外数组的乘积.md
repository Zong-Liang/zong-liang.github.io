## 题目

**描述：**

给你一个整数数组 `nums`，返回 数组 `answer` ，其中 `answer[i]` 等于 `nums` 中除 `nums[i]` 之外其余各元素的乘积。

题目数据 **保证** 数组 `nums`之中任意元素的全部前缀元素和后缀的乘积都在 **32 位** 整数范围内。

请 **不要使用除法**，且在 `O(n)` 时间复杂度内完成此题。

**示例 1:**

- **输入:** `nums = [1,2,3,4]`
- **输出:** `[24,12,8,6]`

**示例 2:**

- **输入:** `nums = [-1,1,0,-3,3]`
- **输出:** `[0,0,9,0,0]`

**提示:**

- `2 <= nums.length <= 10^5`
- `-30 <= nums[i] <= 30`
- 输入 **保证** 数组 `answer[i]` 在 **32 位** 整数范围内

**进阶：** 你可以在 `O(1)` 的额外空间复杂度内完成这个题目吗？（出于对空间复杂度分析的目的，输出数组 **不被视为** 额外空间。）

## 题解

### 这题真正卡人的地方，不是乘法，而是“不能除以自己”

如果没有这句限制，这题其实很简单。

先把整个数组所有元素乘起来，再除以 `nums[i]`，就能得到答案。  
但题目明确说了两件事：

- 不能用除法
- 要在线性时间内完成

这就逼着我们换个角度去想。

对于每个位置 `i` 来说，`answer[i]` 本质上就是两部分的乘积：

- `i` 左边所有元素的乘积
- `i` 右边所有元素的乘积

也就是说：

**当前位置不要自己参与，那就把“左边的贡献”和“右边的贡献”拆开算。**

这才是这题的核心。

### 先把问题拆成左右两部分

比如：

```text
nums = [1,2,3,4]
```

那么每个位置的答案其实可以这么看：

- 位置 `0`：右边是 `2 * 3 * 4 = 24`
- 位置 `1`：左边是 `1`，右边是 `3 * 4 = 12`
- 位置 `2`：左边是 `1 * 2 = 2`，右边是 `4`，结果 `8`
- 位置 `3`：左边是 `1 * 2 * 3 = 6`

所以这题真正要求的，其实不是“整体乘积减掉自己”，而是：

**对每个位置，分别拿到它左边乘积和右边乘积，再把这两个值乘起来。**

一旦这样拆开，做法就清楚了。

### 最直观的写法：分别维护左乘积和右乘积

最容易想到的办法，是先准备两个数组：

- `L[i]`：位置 `i` 左边所有元素的乘积
- `R[i]`：位置 `i` 右边所有元素的乘积

这样最后答案就是：

```java
answer[i] = L[i] * R[i]
```

怎么求这两个数组也不难。

对于左乘积：

- `L[0] = 1`，因为最左边没有元素
- 从左往右递推：

```java
L[i] = L[i - 1] * nums[i - 1]
```

对于右乘积：

- `R[n - 1] = 1`，因为最右边没有元素
- 从右往左递推：

```java
R[i] = R[i + 1] * nums[i + 1]
```

代码是这样：

```java
public int[] productExceptSelf(int[] nums) {
    int n = nums.length;
    int[] L = new int[n];
    int[] R = new int[n];
    int[] answer = new int[n];

    L[0] = 1;
    for (int i = 1; i < n; i++) {
        L[i] = L[i - 1] * nums[i - 1];
    }

    R[n - 1] = 1;
    for (int i = n - 2; i >= 0; i--) {
        R[i] = R[i + 1] * nums[i + 1];
    }

    for (int i = 0; i < n; i++) {
        answer[i] = L[i] * R[i];
    }

    return answer;
}
```

这个写法很好理解，时间复杂度是 $O(n)$，但空间复杂度是 $O(n)$，因为多开了两个辅助数组。

### 这题还能再往下压一层空间

题目后面还有个进阶要求：  
能不能做到 `O(1)` 额外空间。

这里的关键在于，题目明确说了：

**返回的输出数组不算额外空间。**

这句话等于在提醒你：  
既然结果数组本来就必须返回，那能不能直接拿它来顺手存一些中间结果？

答案是可以。

### 先把左边乘积直接存进答案数组

我们可以先让 `answer[i]` 暂时表示“位置 `i` 左边所有元素的乘积”。

也就是：

- `answer[0] = 1`
- 从左往右递推：

```java
answer[i] = answer[i - 1] * nums[i - 1]
```

还是用 `nums = [1,2,3,4]` 举例。

第一遍处理完后，`answer` 会变成：

```text
[1,1,2,6]
```

它表示的含义分别是：

- 位置 `0` 左边没有元素，所以是 `1`
- 位置 `1` 左边只有 `1`
- 位置 `2` 左边是 `1 * 2 = 2`
- 位置 `3` 左边是 `1 * 2 * 3 = 6`

到这里为止，我们其实已经拿到了答案的一半。

### 再从右往左补上右边乘积

接下来只差“右边所有元素的乘积”。

这时候没必要再开一个 `R` 数组。  
只需要用一个变量 `rightProduct`，在从右往左遍历时动态维护它就够了。

开始时：

```java
rightProduct = 1
```

因为最右边那个位置右侧没有元素。

然后从右往左扫：

- 当前 `answer[i]` 里本来已经存的是左边乘积
- 再乘上当前的 `rightProduct`
- 就变成完整答案了
- 接着再更新 `rightProduct *= nums[i]`，给前一个位置使用

还是 `[1,2,3,4]` 这个例子。

第一遍后：

```text
answer = [1,1,2,6]
```

第二遍从右往左：

- 到位置 `3` 时，右边乘积是 `1`，所以 `answer[3] = 6 * 1 = 6`
- 更新 `rightProduct = 4`
- 到位置 `2` 时，`answer[2] = 2 * 4 = 8`
- 更新 `rightProduct = 12`
- 到位置 `1` 时，`answer[1] = 1 * 12 = 12`
- 更新 `rightProduct = 24`
- 到位置 `0` 时，`answer[0] = 1 * 24 = 24`

最终得到：

```text
[24,12,8,6]
```

### 为什么这种写法正好能做到 `O(1)` 额外空间

因为除了返回用的 `answer` 数组之外，我们只多用了一个变量 `rightProduct`。

而题目已经说了，输出数组不算额外空间，所以真正新增的空间只有常数级。

这也是这题最巧的地方：

**不是不用中间结果，而是把中间结果直接复用到输出数组里。**

### 用第二个示例再看一眼

看这个例子：

```text
nums = [-1,1,0,-3,3]
```

第一遍先算左乘积：

```text
answer = [1,-1,-1,0,0]
```

然后从右往左补右乘积，最后会得到：

```text
[0,0,9,0,0]
```

这里也能看出，这种做法根本不需要讨论“0 怎么办”“负数怎么办”。  
因为它完全没有用除法，只是在老老实实地累乘左边和右边的值，所以这些情况都会自然处理好。

这也是它比“总乘积再除自己”更稳的地方。

### 代码

```java
public int[] productExceptSelf(int[] nums) {
    int n = nums.length;
    int[] answer = new int[n];

    answer[0] = 1;
    for (int i = 1; i < n; i++) {
        answer[i] = answer[i - 1] * nums[i - 1];
    }

    int rightProduct = 1;
    for (int i = n - 1; i >= 0; i--) {
        answer[i] *= rightProduct;
        rightProduct *= nums[i];
    }

    return answer;
}
```

时间复杂度是 $O(n)$，因为数组只遍历了两遍。  
额外空间复杂度是 $O(1)$，因为除了返回数组外，只用了一个变量。

### 这题真正值得记住的地方

这题最值得记住的，不是“前缀积 + 后缀积”这几个字，而是这种拆法：

**当一个位置的答案依赖于它左边一整段和右边一整段时，通常可以把左右贡献拆开，分别预处理，再合并。**

这题里：

- 第一遍负责把左边贡献存下来
- 第二遍负责把右边贡献补上去

而且由于输出数组本来就要返回，所以还可以顺手复用它来省空间。

以后再遇到这种“当前位置答案 = 左边整体信息 × 右边整体信息”的题，都可以优先往这个方向想。

### 完整代码

```java
import java.util.Arrays;

public class Main {

    public static int[] productExceptSelf(int[] nums) {
        int n = nums.length;
        int[] answer = new int[n];

        answer[0] = 1;
        for (int i = 1; i < n; i++) {
            answer[i] = answer[i - 1] * nums[i - 1];
        }

        int rightProduct = 1;
        for (int i = n - 1; i >= 0; i--) {
            answer[i] *= rightProduct;
            rightProduct *= nums[i];
        }

        return answer;
    }

    public static void main(String[] args) {
        int[] nums1 = new int[]{1, 2, 3, 4};

        System.out.println("示例 1 输入:");
        System.out.println("nums = " + Arrays.toString(nums1));

        int[] result1 = productExceptSelf(nums1);

        System.out.println("示例 1 输出: " + Arrays.toString(result1));

        int[] nums2 = new int[]{-1, 1, 0, -3, 3};

        System.out.println("示例 2 输入:");
        System.out.println("nums = " + Arrays.toString(nums2));

        int[] result2 = productExceptSelf(nums2);

        System.out.println("示例 2 输出: " + Arrays.toString(result2));
    }
}
```
