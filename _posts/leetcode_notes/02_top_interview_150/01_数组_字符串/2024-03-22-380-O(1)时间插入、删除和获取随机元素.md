## 题目

**描述：**

实现 `RandomizedSet` 类：

- `RandomizedSet()` 初始化 `RandomizedSet` 对象
- `bool insert(int val)` 当元素 `val` 不存在时，向集合中插入该项，并返回 `true` ；否则，返回 `false` 。
- `bool remove(int val)` 当元素 `val` 存在时，从集合中移除该项，并返回 `true` ；否则，返回 `false` 。
- `int getRandom()` 随机返回现有集合中的一项（测试用例保证调用此方法时集合中至少存在一个元素）。每个元素应该有 **相同的概率** 被返回。

你必须实现类的所有函数，并满足每个函数的 **平均** 时间复杂度为 `O(1)` 。

**示例:**

- **输入**
  `["RandomizedSet", "insert", "remove", "insert", "getRandom", "remove", "insert", "getRandom"]`
  `[[], [1], [2], [2], [], [1], [2], []]`
- **输出**
  `[null, true, false, true, 2, true, false, 2]`
  _(getRandom 的输出可能是 1 或 2)_

**提示:**

- `-2^31 <= val <= 2^31 - 1`
- 最多调用 `insert`、`remove` 和 `getRandom` 函数 `2 * 10^5` 次
- 在调用 `getRandom` 方法时，数据结构中 **至少** 存在一个元素。

## 题解

这题表面上是在实现一个集合，麻烦的地方却是三个要求要同时成立：插入要快，删除要快，还要能等概率地随机返回一个元素。单靠一种常见数据结构很难把这三件事一起做好。比如 `HashSet` 插入和删除都很方便，但没法在 `O(1)` 时间里按“随机下标”取值；`ArrayList` 倒是适合随机访问，可一旦删除中间元素，后面的值就得整体前移，时间复杂度立刻变成 `O(n)`。

所以这题一开始就更适合往“两个结构配合”上想。比较顺手的拆法是：用动态数组保存当前集合里的所有元素，再用哈希表记录 `值 -> 它在数组中的下标`。这样一来，`insert` 很自然：先看哈希表里有没有这个值，没有的话就把它追加到数组末尾，同时把下标记进哈希表；`getRandom` 也直接，对数组长度随机出一个下标，再返回对应位置的元素即可。

最需要想透的是 `remove`。如果按照 `ArrayList` 的普通删除逻辑，删掉一个中间位置后，后面的元素都会左移，这一步是线性的。但这题实现的是集合，本来就不要求保持元素顺序，所以完全没必要老老实实搬移整段数组。更省事的做法是：要删哪个位置，就拿数组最后一个元素来补这个洞，然后把数组最后一位删掉。这样实际发生删除的地方始终都在尾部，代价就是常数级。

举个例子，假设现在 `nums = [10, 20, 30, 40]`，哈希表里记录着它们各自的下标。如果要删除 `20`，先通过哈希表知道它在下标 `1`，再把最后一个元素 `40` 放到下标 `1`，数组临时变成 `[10, 40, 30, 40]`，随后删掉最后一位，就得到 `[10, 40, 30]`。别忘了同时把哈希表里 `40` 的位置更新成 `1`，再把 `20` 的映射删掉。这一步就是整题最关键的技巧。

很多人第一次写的时候，会担心“如果要删的本来就是最后一个元素，会不会出错”。其实不用额外特判。因为这时“补洞”的还是它自己，接着删掉尾部，再把哈希表中的旧记录移除，最后结果仍然是对的。

代码如下：

```java
public boolean remove(int val) {
    if (!indices.containsKey(val)) {
        return false;
    }

    int index = indices.get(val);
    int lastElement = nums.get(nums.size() - 1);

    nums.set(index, lastElement);
    indices.put(lastElement, index);

    nums.remove(nums.size() - 1);
    indices.remove(val);

    return true;
}
```

三个操作的平均时间复杂度都是 `O(1)`，其中哈希表的查询、插入、删除都是平均常数时间；空间复杂度是 `O(n)`，因为数组和哈希表都需要保存集合中的元素信息。

这题可以顺手记住一种很常见的搭配思路：如果题目一边要求按值快速定位，一边又要求按下标快速访问，往往就得让哈希表和数组各管一头；如果顺序本身不重要，删除时再想想能不能用“尾元素补洞”把线性搬移省掉。

完整代码如下：

```java
import java.util.*;

public class Main {

    static class RandomizedSet {
        private final List<Integer> nums;
        private final Map<Integer, Integer> indices;
        private final Random random;

        public RandomizedSet() {
            nums = new ArrayList<>();
            indices = new HashMap<>();
            random = new Random();
        }

        public boolean insert(int val) {
            if (indices.containsKey(val)) {
                return false;
            }

            indices.put(val, nums.size());
            nums.add(val);
            return true;
        }

        public boolean remove(int val) {
            if (!indices.containsKey(val)) {
                return false;
            }

            int index = indices.get(val);
            int lastElement = nums.get(nums.size() - 1);

            nums.set(index, lastElement);
            indices.put(lastElement, index);

            nums.remove(nums.size() - 1);
            indices.remove(val);
            return true;
        }

        public int getRandom() {
            return nums.get(random.nextInt(nums.size()));
        }
    }

    public static void main(String[] args) {
        System.out.println("开始测试 RandomizedSet:");

        RandomizedSet set = new RandomizedSet();

        System.out.println("insert(1): " + set.insert(1));
        System.out.println("remove(2): " + set.remove(2));
        System.out.println("insert(2): " + set.insert(2));
        System.out.println("getRandom(): " + set.getRandom());
        System.out.println("remove(1): " + set.remove(1));
        System.out.println("insert(2): " + set.insert(2));
        System.out.println("getRandom(): " + set.getRandom());
    }
}
```
