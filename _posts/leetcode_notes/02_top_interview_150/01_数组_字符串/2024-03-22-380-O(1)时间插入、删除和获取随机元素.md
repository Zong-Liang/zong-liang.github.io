## 题目

**描述：**

实现 `RandomizedSet` 类：

- `RandomizedSet()` 初始化 `RandomizedSet` 对象
- `bool insert(int val)` 当元素 `val` 不存在时，向集合中插入该项，并返回 `true` ；否则，返回 `false` 。
- `bool remove(int val)` 当元素 `val` 存在时，从集合中移除该项，并返回 `true` ；否则，返回 `false` 。
- `int getRandom()` 随机返回现有集合中的一项（测试用例保证调用此方法时集合中至少存在一个元素）。每个元素应该有 **相同的概率** 被返回。

你必须实现类的所有函数，并满足每个函数的 **平均** 时间复杂度为 `O(1)` 。

**示例:**

- **输入**
  `["RandomizedSet", "insert", "remove", "insert", "getRandom", "remove", "insert", "getRandom"]`
  `[[], [1], [2], [2], [], [1], [2], []]`
- **输出**
  `[null, true, false, true, 2, true, false, 2]`
  _(getRandom 的输出可能是 1 或 2)_

**提示:**

- `-2^31 <= val <= 2^31 - 1`
- 最多调用 `insert`、`remove` 和 `getRandom` 函数 `2 * 10^5` 次
- 在调用 `getRandom` 方法时，数据结构中 **至少** 存在一个元素。

## 题解

### 这题真正难的地方，不是“集合”，而是同时满足三件事

如果只是让你实现一个集合，事情并不复杂。  
但这题要求的是：

- `insert` 要快
- `remove` 要快
- `getRandom` 也要快，而且每个元素被返回的概率还要一样

问题就出在这里：  
**这三个要求，很难靠单一数据结构同时满足。**

比如：

- 用 `HashSet`，插入和删除确实快，但没法在 `O(1)` 时间里随机取出一个元素
- 用数组或 `ArrayList`，随机取元素很方便，但删除中间元素会导致后面的元素整体前移，时间复杂度变成 `O(n)`

所以这题的关键不是“选哪一种结构”，而是要意识到：

**需要把两种数据结构拼起来用。**

### 为什么要同时用数组和哈希表

这题最适合的组合是：

- `ArrayList`：负责存元素
- `HashMap`：负责记录每个元素在数组里的位置

这样分工之后，每个操作都有着落：

- `ArrayList` 支持按下标随机访问，所以 `getRandom()` 可以在 `O(1)` 时间完成
- `HashMap` 支持按值查下标，所以可以在 `O(1)` 时间判断元素是否存在，以及定位它在数组中的位置

这两个结构放在一起，才有可能把题目要求的三个操作都压到平均 `O(1)`。

### 整体怎么设计

我们维护两个成员变量：

- `nums`：一个动态数组，里面存当前集合里的所有元素
- `indices`：一个哈希表，记录 `值 -> 该值在 nums 中的下标`

比如当前集合里是：

```text
nums = [10, 20, 30]
```

那哈希表就是：

```text
indices = {
    10 -> 0,
    20 -> 1,
    30 -> 2
}
```

只要这两个结构始终保持同步，整个数据结构就能正常工作。

### 插入为什么很简单

`insert(val)` 其实最好理解。

先看 `indices` 里有没有这个值：

- 如果已经有了，说明集合里本来就存在，直接返回 `false`
- 如果没有，就把它加到 `nums` 末尾，同时在 `indices` 里记录它的下标

因为往数组末尾添加元素是 `O(1)`，哈希表插入也是平均 `O(1)`，所以这个操作没问题。

```java
public boolean insert(int val) {
    if (indices.containsKey(val)) {
        return false;
    }

    indices.put(val, nums.size());
    nums.add(val);
    return true;
}
```

### 随机取元素为什么也不难

`getRandom()` 的核心只有一句话：

**既然所有元素都存在数组里，那就随机生成一个合法下标，再返回这个位置上的元素。**

比如当前 `nums.size() = 5`，那就随机生成 `0` 到 `4` 之间的一个整数，然后 `nums.get(index)` 即可。

因为数组支持按下标 `O(1)` 访问，所以这个操作天然满足要求。

```java
public int getRandom() {
    return nums.get(random.nextInt(nums.size()));
}
```

而且由于每个下标被选中的概率相同，所以每个元素被返回的概率也相同。

### 真正麻烦的是删除

难点全在 `remove(val)`。

如果只看逻辑，删除似乎也不复杂：

- 先通过哈希表找到 `val` 在数组中的位置
- 再把它从数组里删掉
- 再把哈希表里的记录删掉

问题是，`ArrayList` 删除中间元素不是 `O(1)`。  
因为一旦删掉中间某个位置，后面的元素都得往前挪一格。

比如：

```text
nums = [10, 20, 30, 40]
```

如果要删掉 `20`，删完之后会变成：

```text
[10, 30, 40]
```

这里 `30` 和 `40` 都得搬位置，这一步就是线性的。

而题目不允许这样。

### 这题最关键的技巧：拿最后一个元素补洞

既然不能真的把中间删掉，那就换个思路：

**把要删的位置，用最后一个元素顶上，然后再删掉数组最后一个元素。**

因为删除数组末尾是 `O(1)`，这样就能把整个删除操作压下来。

还是刚才那个例子：

```text
nums = [10, 20, 30, 40]
```

现在要删除 `20`，它在下标 `1`。  
我们可以这样做：

1. 先拿到最后一个元素 `40`
2. 把 `40` 放到下标 `1`
3. 这样数组临时变成：

```text
[10, 40, 30, 40]
```

4. 再删除最后一个位置
5. 最终得到：

```text
[10, 40, 30]
```

这样中间没有整体搬移，代价就是常数级。

这时候还必须记得一件事：  
**`40` 的位置变了，所以哈希表里它对应的下标也要一起改。**

### 为什么这种“交换删除”不会出问题

因为这题实现的是集合，不要求元素有序。

这点非常重要。

如果题目要求维护插入顺序，那你把最后一个元素硬塞到中间，顺序就乱了。  
但这里不关心顺序，只关心元素是否存在，以及能不能随机取，所以这样做完全合法。

换句话说，这题能用这个技巧，前提就是：

**集合只关心成员，不关心排列顺序。**

### 删除时具体做什么

`remove(val)` 的完整流程是：

1. 先检查 `val` 在不在
2. 如果不在，返回 `false`
3. 如果在，通过 `indices` 找到它在数组里的下标 `index`
4. 取出数组最后一个元素 `lastElement`
5. 用 `lastElement` 覆盖 `nums[index]`
6. 更新 `indices` 中 `lastElement` 的下标
7. 删除数组最后一个元素
8. 删除 `val` 在哈希表中的记录
9. 返回 `true`

代码是这样：

```java
public boolean remove(int val) {
    if (!indices.containsKey(val)) {
        return false;
    }

    int index = indices.get(val);
    int lastElement = nums.get(nums.size() - 1);

    nums.set(index, lastElement);
    indices.put(lastElement, index);

    nums.remove(nums.size() - 1);
    indices.remove(val);

    return true;
}
```

### 删除最后一个元素时也没问题

这里还有一个细节，很多人第一次会有点不放心：

如果要删除的元素本来就是最后一个元素，会不会出错？

其实不会。

比如：

```text
nums = [10, 20, 30]
```

要删的是 `30`。

那这时候：

- `index = 2`
- `lastElement = 30`

你还是照样执行：

- `nums.set(2, 30)`
- `indices.put(30, 2)`
- `nums.remove(2)`
- `indices.remove(30)`

虽然前两步看起来像是“自己覆盖自己”，但逻辑上没有问题，最后结果仍然是正确的。

所以这个写法不需要额外特判。

### 代码

```java
class RandomizedSet {
    List<Integer> nums;
    Map<Integer, Integer> indices;
    Random random;

    public RandomizedSet() {
        nums = new ArrayList<>();
        indices = new HashMap<>();
        random = new Random();
    }

    public boolean insert(int val) {
        if (indices.containsKey(val)) {
            return false;
        }

        indices.put(val, nums.size());
        nums.add(val);
        return true;
    }

    public boolean remove(int val) {
        if (!indices.containsKey(val)) {
            return false;
        }

        int index = indices.get(val);
        int lastElement = nums.get(nums.size() - 1);

        nums.set(index, lastElement);
        indices.put(lastElement, index);

        nums.remove(nums.size() - 1);
        indices.remove(val);
        return true;
    }

    public int getRandom() {
        return nums.get(random.nextInt(nums.size()));
    }
}
```

三个操作的平均时间复杂度都是 `O(1)`。  
空间复杂度是 `O(n)`，因为数组和哈希表都要存这 `n` 个元素的信息。

### 这题真正值得记住的地方

这题最值得记住的，不只是“哈希表 + 动态数组”这个组合，而是背后的思路：

**当一个操作需要按值快速定位，另一个操作又需要按下标快速访问时，往往要把“值”和“位置”拆开，分别交给不同的数据结构维护。**

这题里：

- 数组负责随机访问
- 哈希表负责值到位置的映射
- 删除靠“最后一个元素补洞”来避免线性搬移

以后如果再遇到“既要快速查找，又要快速随机访问，还要支持删除”的题，基本都可以往这个组合上想。

### 完整代码

```java
import java.util.*;

public class Main {

    static class RandomizedSet {
        private List<Integer> nums;
        private Map<Integer, Integer> indices;
        private Random random;

        public RandomizedSet() {
            nums = new ArrayList<>();
            indices = new HashMap<>();
            random = new Random();
        }

        public boolean insert(int val) {
            if (indices.containsKey(val)) {
                return false;
            }

            indices.put(val, nums.size());
            nums.add(val);
            return true;
        }

        public boolean remove(int val) {
            if (!indices.containsKey(val)) {
                return false;
            }

            int index = indices.get(val);
            int lastElement = nums.get(nums.size() - 1);

            nums.set(index, lastElement);
            indices.put(lastElement, index);

            nums.remove(nums.size() - 1);
            indices.remove(val);

            return true;
        }

        public int getRandom() {
            return nums.get(random.nextInt(nums.size()));
        }
    }

    public static void main(String[] args) {
        System.out.println("开始测试 RandomizedSet:");

        RandomizedSet randomizedSet = new RandomizedSet();

        boolean i1 = randomizedSet.insert(1);
        System.out.println("insert(1): " + i1);

        boolean r2 = randomizedSet.remove(2);
        System.out.println("remove(2): " + r2);

        boolean i2 = randomizedSet.insert(2);
        System.out.println("insert(2): " + i2);

        int rand1 = randomizedSet.getRandom();
        System.out.println("getRandom(): " + rand1);

        boolean r1 = randomizedSet.remove(1);
        System.out.println("remove(1): " + r1);

        boolean i2_again = randomizedSet.insert(2);
        System.out.println("insert(2): " + i2_again);

        int rand2 = randomizedSet.getRandom();
        System.out.println("getRandom(): " + rand2);
    }
}
```
