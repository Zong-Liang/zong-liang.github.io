## 题目

**描述：**

给定 `n` 个非负整数表示每个宽度为 1 的柱子的高度图，计算按此排列的柱子，下雨之后能接多少雨水。

**示例 1:**

- **输入:** `height = [0,1,0,2,1,0,1,3,2,1,2,1]`
- **输出:** 6
- **解释:** 上面是由数组 `[0,1,0,2,1,0,1,3,2,1,2,1]` 表示的高度图，在这种情况下，可以接 6 个单位的雨水（蓝色部分表示雨水）。

**示例 2:**

- **输入:** `height = [4,2,0,3,2,5]`
- **输出:** 9

**提示:**

- `n == height.length`
- `1 <= n <= 2 * 10^4`
- `0 <= height[i] <= 10^5`

## 题解

这题第一眼很容易盯着“凹槽”去看，觉得哪一格低就能存水，哪一格高就不能。但决定某一列能接多少水的，并不是它自己有多矮，而是它左边和右边最高挡板里较矮的那个。对位置 `i` 来说，能接的水量其实就是 `min(leftMax, rightMax) - height[i]`。这句话一旦想清楚，后面的做法无非是在考虑：怎么更高效地拿到每个位置的 `leftMax` 和 `rightMax`。

最直观的写法，是先预处理两张表：`leftMax[i]` 表示从左到右看到位置 `i` 为止的最高柱子，`rightMax[i]` 表示从右到左看到位置 `i` 为止的最高柱子。这样最后再扫一遍，把每一列的 `min(leftMax[i], rightMax[i]) - height[i]` 加起来就可以了。这个版本很好理解，也是很多人第一次把这题做顺时最自然的思路。

不过这题还能再往前走一步。既然对当前位置起作用的是左右最高值里的较小者，那我们其实没必要把整张表都存下来，只要在双指针移动的过程中维护当前左侧最高值 `leftMax` 和右侧最高值 `rightMax` 就够了。更难想通的地方在于：为什么每次都可以先处理较矮的那一边。

假设当前 `height[left] < height[right]`。这说明右边至少已经有一根柱子不低于 `height[right]`，也就一定不低于 `height[left]`。对 `left` 这个位置来说，右边这一侧此时已经“够高”了，限制它能接多少水的，只剩下左边最高值 `leftMax`。所以这时候 `left` 位置的答案已经定了，可以直接加上 `leftMax - height[left]`，然后把 `left` 往中间移动。反过来，如果当前右边更矮，那就说明右边这格的答案已经可以确定，于是先结算右边。

拿示例 `height = [4, 2, 0, 3, 2, 5]` 走一段会更直观。开始时 `left = 0`，`right = 5`，左侧最高是 `4`，右侧最高是 `5`。因为左边当前柱子更矮，所以先处理左边：下标 `0` 这一列接不到水；接着左指针来到下标 `1`，左侧最高仍是 `4`，这时它能接 `4 - 2 = 2`；再往右到下标 `2`，能接 `4 - 0 = 4`。继续按这个规则往中间收缩，最后总和就是 `9`。整个过程本质上仍然是在算同一个公式，只是把“预处理整张表”换成了“边走边结算”。

还有一个顺手能想明白的细节是，代码里不会算出负数。因为无论处理左边还是右边，都是先更新 `leftMax` / `rightMax`，再做减法，所以它们一定不会小于当前位置的柱子高度。

代码如下：

```java
public int trap(int[] height) {
    int left = 0;
    int right = height.length - 1;
    int leftMax = 0;
    int rightMax = 0;
    int ans = 0;

    while (left < right) {
        leftMax = Math.max(leftMax, height[left]);
        rightMax = Math.max(rightMax, height[right]);

        if (height[left] < height[right]) {
            ans += leftMax - height[left];
            left++;
        } else {
            ans += rightMax - height[right];
            right--;
        }
    }

    return ans;
}
```

时间复杂度是 `O(n)`，因为左右指针总共只会把整个数组走一遍；空间复杂度是 `O(1)`，因为只用了几个额外变量。

这题更值得留在脑子里的，不只是“接雨水可以双指针”，而是那个判断口径：当一个位置的答案由左右边界共同决定时，先问清楚到底是谁在充当短板。只要短板能被确定，往往就有机会一边扫描一边结算。

完整代码如下：

```java
import java.util.Arrays;

public class Main {

    public static int trap(int[] height) {
        if (height == null || height.length == 0) {
            return 0;
        }

        int left = 0;
        int right = height.length - 1;
        int leftMax = 0;
        int rightMax = 0;
        int ans = 0;

        while (left < right) {
            leftMax = Math.max(leftMax, height[left]);
            rightMax = Math.max(rightMax, height[right]);

            if (height[left] < height[right]) {
                ans += leftMax - height[left];
                left++;
            } else {
                ans += rightMax - height[right];
                right--;
            }
        }

        return ans;
    }

    public static void main(String[] args) {
        int[] height1 = new int[]{0, 1, 0, 2, 1, 0, 1, 3, 2, 1, 2, 1};

        System.out.println("示例 1 输入:");
        System.out.println("height = " + Arrays.toString(height1));

        int result1 = trap(height1);

        System.out.println("示例 1 输出: " + result1);

        int[] height2 = new int[]{4, 2, 0, 3, 2, 5};

        System.out.println("示例 2 输入:");
        System.out.println("height = " + Arrays.toString(height2));

        int result2 = trap(height2);

        System.out.println("示例 2 输出: " + result2);
    }
}
```
