## 题目

**描述：**

给定 `n` 个非负整数表示每个宽度为 1 的柱子的高度图，计算按此排列的柱子，下雨之后能接多少雨水。

**示例 1:**

- **输入:** `height = [0,1,0,2,1,0,1,3,2,1,2,1]`
- **输出:** 6
- **解释:** 上面是由数组 `[0,1,0,2,1,0,1,3,2,1,2,1]` 表示的高度图，在这种情况下，可以接 6 个单位的雨水（蓝色部分表示雨水）。

**示例 2:**

- **输入:** `height = [4,2,0,3,2,5]`
- **输出:** 9

**提示:**

- `n == height.length`
- `1 <= n <= 2 * 10^4`
- `0 <= height[i] <= 10^5`

## 题解

### 这题最容易卡住的地方，是不知道“水位”到底由谁决定

很多人第一次做这题，会盯着某个凹槽去看，觉得哪里低就能存水，哪里高就不能存水。

但如果只盯着当前位置本身，其实不够。  
因为一个位置能不能接水，不取决于它自己有多高，而取决于它两边有没有足够高的“挡板”。

对下标 `i` 来说，它最终能接多少水，只看三件事：

- 它左边最高的柱子有多高
- 它右边最高的柱子有多高
- 它自己本身有多高

真正的水位，其实是：

```text
min(左边最高, 右边最高)
```

因为水不可能高过短的那一边。  
所以当前位置能接的水量就是：

```text
min(leftMax, rightMax) - height[i]
```

这才是这题最核心的公式。

### 先把这个公式想透

为什么一定要取两边最高值里的较小者？

因为接水本质上是被两边夹住。  
左边再高，如果右边很低，水还是会从右边流走；右边再高，如果左边很低，水也会从左边流走。

所以真正能把水“封住”的高度，不是两边最高的那个，而是两边里更矮的那个。

比如某个位置：

- 左边最高是 `5`
- 右边最高是 `3`
- 当前柱子高度是 `1`

那它最多只能接到高度 `3`，不能接到 `5`。  
所以这一格能存的水就是：

```text
3 - 1 = 2
```

只要这个判断立住了，后面的做法其实都是围绕它展开。

### 最直观的做法：先把每个位置左右最高值都算出来

既然每个位置都要用到左边最高和右边最高，那最直接的想法就是：

- 先算出每个位置左边的最高柱子
- 再算出每个位置右边的最高柱子
- 最后套公式求和

这就是最常见的 `leftMax` / `rightMax` 做法。

定义：

- `leftMax[i]`：从左到右看到位置 `i` 为止的最高柱子
- `rightMax[i]`：从右到左看到位置 `i` 为止的最高柱子

那么：

```java
water[i] = Math.min(leftMax[i], rightMax[i]) - height[i]
```

代码是这样：

```java
public int trap(int[] height) {
    int n = height.length;
    int[] leftMax = new int[n];
    int[] rightMax = new int[n];

    leftMax[0] = height[0];
    for (int i = 1; i < n; i++) {
        leftMax[i] = Math.max(leftMax[i - 1], height[i]);
    }

    rightMax[n - 1] = height[n - 1];
    for (int i = n - 2; i >= 0; i--) {
        rightMax[i] = Math.max(rightMax[i + 1], height[i]);
    }

    int ans = 0;
    for (int i = 0; i < n; i++) {
        ans += Math.min(leftMax[i], rightMax[i]) - height[i];
    }

    return ans;
}
```

这个做法很好理解，时间复杂度是 $O(n)$，但额外空间也是 $O(n)$，因为开了两个辅助数组。

### 这题还能继续优化，因为你并不是真的需要整张表

上面这种写法已经够好了，但它还有个地方可以继续压：

对于某个位置来说，我们最终只关心：

```text
min(leftMax, rightMax)
```

也就是说，并不是非得把所有位置的左最高数组、右最高数组完整存下来，才有办法算答案。

只要在移动过程中，始终知道当前左边的最高值和右边的最高值，其实就够了。

这就是双指针写法的出发点。

### 双指针真正难想通的地方：为什么可以先算矮的一边

这题的双指针不难写，真正难的是第一次看时很容易不放心：

为什么比较 `height[left]` 和 `height[right]`，哪边矮就先算哪边？

这里必须把逻辑想清楚。

假设当前：

```text
height[left] < height[right]
```

那说明什么？

说明右边至少存在一根柱子，它的高度不低于 `height[right]`，而 `height[right]` 又比 `height[left]` 高。  
所以对 `left` 这个位置来说，右边这侧一定“够高”，不会成为短板。此时真正限制它能接多少水的，只可能是左边最高值 `leftMax`。

换句话说：

- 右边现在已经足够高了
- 当前这一格到底能存多少，只看左边挡板有多高

所以这时候可以直接结算左边这一格的水量：

```java
leftMax - height[left]
```

反过来，如果：

```text
height[left] >= height[right]
```

那就说明左边这侧已经不短了，当前右边这一格能存多少水，只看 `rightMax`。

这就是双指针的核心逻辑：

**哪边当前更矮，哪边的水量就已经可以确定。**

### 用示例 1 先按公式理解一下

看这个例子：

```text
height = [0,1,0,2,1,0,1,3,2,1,2,1]
```

比如中间下标 `2`，高度是 `0`。

- 它左边最高是 `1`
- 它右边最高是 `3`

所以它能接的水量是：

```text
min(1, 3) - 0 = 1
```

再比如下标 `5`，高度是 `0`。

- 左边最高是 `2`
- 右边最高是 `3`

所以它能接：

```text
min(2, 3) - 0 = 2
```

整题其实就是把每一列这样算一遍，加起来得到总量 `6`。

所以无论是动态规划还是双指针，本质都没有变，都是在算这件事。

### 双指针到底怎么走

双指针做法里，我们维护四个变量：

- `left`：左指针
- `right`：右指针
- `leftMax`：左侧扫描到目前为止的最高柱子
- `rightMax`：右侧扫描到目前为止的最高柱子

每次循环先更新：

- `leftMax = max(leftMax, height[left])`
- `rightMax = max(rightMax, height[right])`

然后判断：

- 如果 `height[left] < height[right]`，处理左边
- 否则处理右边

为什么不是比较 `leftMax` 和 `rightMax`，而是比较当前高度？  
因为当当前左柱子更矮时，可以确定右边存在足够高的边界，于是左边这格的答案已经定了；右边同理。

代码就是这样：

```java
public int trap(int[] height) {
    int left = 0, right = height.length - 1;
    int leftMax = 0, rightMax = 0;
    int ans = 0;

    while (left < right) {
        leftMax = Math.max(leftMax, height[left]);
        rightMax = Math.max(rightMax, height[right]);

        if (height[left] < height[right]) {
            ans += leftMax - height[left];
            left++;
        } else {
            ans += rightMax - height[right];
            right--;
        }
    }

    return ans;
}
```

### 为什么不会算出负数

这里还有个细节值得顺手想清楚。

无论处理左边还是右边，代码里都会做类似：

```java
leftMax - height[left]
```

或者：

```java
rightMax - height[right]
```

之所以不会出现负数，是因为 `leftMax` 和 `rightMax` 都是先更新再使用的。

也就是说：

- `leftMax` 一定大于等于当前的 `height[left]`
- `rightMax` 一定大于等于当前的 `height[right]`

所以减出来的值最小也是 `0`。

### 代码

```java
public int trap(int[] height) {
    int left = 0;
    int right = height.length - 1;
    int leftMax = 0;
    int rightMax = 0;
    int ans = 0;

    while (left < right) {
        leftMax = Math.max(leftMax, height[left]);
        rightMax = Math.max(rightMax, height[right]);

        if (height[left] < height[right]) {
            ans += leftMax - height[left];
            left++;
        } else {
            ans += rightMax - height[right];
            right--;
        }
    }

    return ans;
}
```

时间复杂度是 $O(n)$，因为左右指针总共只会走完整个数组一遍。  
空间复杂度是 $O(1)$，因为只用了常数个变量。

### 这题真正值得记住的地方

这题最值得记住的，不只是“接雨水用双指针”，而是这个判断方式：

**当一个位置的答案取决于左右两边的信息时，先想清楚它到底受谁限制。**

这题里，真正决定水位的不是左右最高值里更高的那个，而是更矮的那个。  
双指针之所以成立，也是因为它抓住了这件事：哪边短，哪边就先决定答案。

以后再遇到这种“当前位置的结果由左右边界共同决定”的题，也可以先问自己一句：

**限制它的，到底是哪一边的短板？**

### 完整代码

```java
import java.util.Arrays;

public class Main {

    public static int trap(int[] height) {
        if (height == null || height.length == 0) {
            return 0;
        }

        int left = 0;
        int right = height.length - 1;
        int leftMax = 0;
        int rightMax = 0;
        int ans = 0;

        while (left < right) {
            leftMax = Math.max(leftMax, height[left]);
            rightMax = Math.max(rightMax, height[right]);

            if (height[left] < height[right]) {
                ans += leftMax - height[left];
                left++;
            } else {
                ans += rightMax - height[right];
                right--;
            }
        }

        return ans;
    }

    public static void main(String[] args) {
        int[] height1 = new int[]{0, 1, 0, 2, 1, 0, 1, 3, 2, 1, 2, 1};

        System.out.println("示例 1 输入:");
        System.out.println("height = " + Arrays.toString(height1));

        int result1 = trap(height1);

        System.out.println("示例 1 输出: " + result1);

        int[] height2 = new int[]{4, 2, 0, 3, 2, 5};

        System.out.println("示例 2 输入:");
        System.out.println("height = " + Arrays.toString(height2));

        int result2 = trap(height2);

        System.out.println("示例 2 输出: " + result2);
    }
}
```
