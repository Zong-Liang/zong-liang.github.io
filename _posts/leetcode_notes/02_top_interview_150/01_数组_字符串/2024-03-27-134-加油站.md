## 题目

**描述：**

在一条环路上有 `n` 个加油站，其中第 `i` 个加油站有汽油 `gas[i]` 升。

你有一辆油箱容量无限的汽车，从第 `i` 个加油站开往第 `i+1` 个加油站需要消耗汽油 `cost[i]` 升。你从其中的一个加油站出发，开始时油箱为空。

给定两个整数数组 `gas` 和 `cost` ，如果你可以按顺序绕环路行驶一周，则返回出发时加油站的编号，否则返回 -1 。如果存在解，则 **保证** 它是 **唯一** 的。

**示例 1:**

- **输入:** `gas = [1,2,3,4,5]`, `cost = [3,4,5,1,2]`
- **输出:** 3
- **解释:**

  从 3 号加油站（索引为 3 处）出发，可获得 4 升汽油。此时油箱有 = 0 + 4 = 4 升汽油

  开往 4 号加油站，此时油箱有 4 - 1 + 5 = 8 升汽油

  开往 0 号加油站，此时油箱有 8 - 2 + 1 = 7 升汽油

  开往 1 号加油站，此时油箱有 7 - 3 + 2 = 6 升汽油

  开往 2 号加油站，此时油箱有 6 - 4 + 3 = 5 升汽油

  开往 3 号加油站，你需要消耗 5 升汽油，正好足够你返回到 3 号加油站。

  因此，3 可为起始索引。

**示例 2:**

- **输入:** `gas = [2,3,4]`, `cost = [3,4,3]`
- **输出:** -1

**提示:**

- `n == gas.length == cost.length`
- `1 <= n <= 10^5`
- `0 <= gas[i], cost[i] <= 10^4`
- 输入保证答案唯一。

## 题解

这题如果老老实实去试每一个起点，思路当然不难：从某个加油站出发，看看能不能把一圈开完，不行就换下一个。问题在于，这样最坏会把很多路段反复走很多次，复杂度很容易变成 `O(n^2)`。所以这题想考的，不是“会不会模拟”，而是当某个起点失败以后，能不能顺手排除掉一整段根本不必再试的位置。

先有一个很基础的判断：如果所有加油站的油量总和都小于总消耗，那不管从哪里出发都不可能绕完整个环，这时直接返回 `-1` 就够了。也就是说，`sum(gas) >= sum(cost)` 是存在解的必要条件。这个判断虽然简单，但很重要，因为它先告诉我们：问题到底是“起点没找对”，还是“根本不存在合法起点”。

贪心就发生在扫描过程中。我们维护 `currentSum` 表示从当前候选起点 `start` 出发，走到当前位置时手里还剩多少油。如果走到某个位置 `i` 时，`currentSum` 变成了负数，就说明从 `start` 出发连 `i + 1` 都到不了。关键不只是在于“`start` 不行”，而是还能更进一步推出：`start` 到 `i` 之间的所有位置，其实都不可能成为答案。

这个结论为什么成立？因为如果我们是从 `start` 一路走到 `i` 才第一次失败，说明在 `start` 到 `i - 1` 的过程中，累计剩余油量一直没有掉到负数，否则起点早就该在更前面被放弃了。于是对于区间里的任意一个位置 `k`，从 `start` 走到 `k - 1` 这一段的累计和一定是非负的。既然从 `start` 走到 `i` 的总和都已经小于 `0`，那把前面这段非负贡献拿掉以后，从 `k` 走到 `i` 的总和只会更小，不可能突然变成可行。这样一来，下一个还值得尝试的起点就只能是 `i + 1`。

拿示例 `gas = [1, 2, 3, 4, 5]`、`cost = [3, 4, 5, 1, 2]` 走一遍会更直观。每一站的净收益是 `[-2, -2, -2, 3, 3]`。从 `0` 出发，第一站就亏到 `-2`，所以起点直接跳到 `1`；从 `1` 出发又失败，于是再跳到 `2`；接着同样失败，再跳到 `3`。等起点来到 `3` 以后，后面的累计和变成 `3`、`6`，说明从这里出发可以把剩下这一段撑住。再结合全局总和恰好为 `0`，就能确定答案就是 `3`。

于是整个过程只需要维护三个量：`totalSum` 用来判断全局上有没有解，`currentSum` 用来判断当前候选起点会不会中途失败，`start` 记录当前候选起点。数组扫完后，如果 `totalSum < 0` 就返回 `-1`，否则返回最终留下来的 `start`。

代码如下：

```java
public int canCompleteCircuit(int[] gas, int[] cost) {
    int totalSum = 0;
    int currentSum = 0;
    int start = 0;

    for (int i = 0; i < gas.length; i++) {
        int diff = gas[i] - cost[i];
        totalSum += diff;
        currentSum += diff;

        if (currentSum < 0) {
            start = i + 1;
            currentSum = 0;
        }
    }

    return totalSum < 0 ? -1 : start;
}
```

时间复杂度是 `O(n)`，因为整个数组只扫描了一遍；空间复杂度是 `O(1)`，因为只用了几个额外变量。

这题挺适合带走的一点，不一定是“加油站用贪心”这个标签，而是那种“失败后整段排除”的感觉：一旦你能证明某个起点失败会连带否掉后面一整段候选点，原本看起来像枚举的问题，往往就能收缩成一次线性扫描。

完整代码如下：

```java
import java.util.Arrays;

public class Main {

    public static int canCompleteCircuit(int[] gas, int[] cost) {
        int totalSum = 0;
        int currentSum = 0;
        int start = 0;

        for (int i = 0; i < gas.length; i++) {
            int diff = gas[i] - cost[i];
            totalSum += diff;
            currentSum += diff;

            if (currentSum < 0) {
                start = i + 1;
                currentSum = 0;
            }
        }

        return totalSum < 0 ? -1 : start;
    }

    public static void main(String[] args) {
        int[] gas1 = new int[]{1, 2, 3, 4, 5};
        int[] cost1 = new int[]{3, 4, 5, 1, 2};

        System.out.println("示例 1 输入:");
        System.out.println("gas = " + Arrays.toString(gas1));
        System.out.println("cost = " + Arrays.toString(cost1));

        int result1 = canCompleteCircuit(gas1, cost1);

        System.out.println("示例 1 输出: " + result1);

        int[] gas2 = new int[]{2, 3, 4};
        int[] cost2 = new int[]{3, 4, 3};

        System.out.println("示例 2 输入:");
        System.out.println("gas = " + Arrays.toString(gas2));
        System.out.println("cost = " + Arrays.toString(cost2));

        int result2 = canCompleteCircuit(gas2, cost2);

        System.out.println("示例 2 输出: " + result2);
    }
}
```
