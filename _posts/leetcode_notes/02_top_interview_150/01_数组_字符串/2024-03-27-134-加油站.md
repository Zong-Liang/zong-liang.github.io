## 题目

**描述：**

在一条环路上有 `n` 个加油站，其中第 `i` 个加油站有汽油 `gas[i]` 升。

你有一辆油箱容量无限的汽车，从第 `i` 个加油站开往第 `i+1` 个加油站需要消耗汽油 `cost[i]` 升。你从其中的一个加油站出发，开始时油箱为空。

给定两个整数数组 `gas` 和 `cost` ，如果你可以按顺序绕环路行驶一周，则返回出发时加油站的编号，否则返回 -1 。如果存在解，则 **保证** 它是 **唯一** 的。

**示例 1:**

- **输入:** `gas = [1,2,3,4,5]`, `cost = [3,4,5,1,2]`
- **输出:** 3
- **解释:**

  从 3 号加油站（索引为 3 处）出发，可获得 4 升汽油。此时油箱有 = 0 + 4 = 4 升汽油

  开往 4 号加油站，此时油箱有 4 - 1 + 5 = 8 升汽油

  开往 0 号加油站，此时油箱有 8 - 2 + 1 = 7 升汽油

  开往 1 号加油站，此时油箱有 7 - 3 + 2 = 6 升汽油

  开往 2 号加油站，此时油箱有 6 - 4 + 3 = 5 升汽油

  开往 3 号加油站，你需要消耗 5 升汽油，正好足够你返回到 3 号加油站。

  因此，3 可为起始索引。

**示例 2:**

- **输入:** `gas = [2,3,4]`, `cost = [3,4,3]`
- **输出:** -1

**提示:**

- `n == gas.length == cost.length`
- `1 <= n <= 10^5`
- `0 <= gas[i], cost[i] <= 10^4`
- 输入保证答案唯一。

## 题解

### 这题最容易想复杂的地方

很多人第一次做这题，会先去想：

- 从哪个加油站开始试
- 如果这个起点不行，再换下一个
- 要不要把每个位置都模拟一圈

这种想法很自然，但也会直接把题做成 `O(n^2)`。

这题真正该先想清楚的，不是“怎么模拟一圈”，而是两个更本质的问题：

- 有没有可能存在解
- 如果当前起点走到一半失败了，这个起点前面的那些位置还有没有必要再试

一旦这两个问题想明白，整题就会一下子简化成贪心。

### 先看一个最基本的判断：总油量够不够

无论从哪里出发，想绕完整个环路，至少要满足一件事：

**所有加油站的总油量，不能小于跑完整圈的总消耗。**

也就是：

```text
sum(gas) >= sum(cost)
```

如果连这个条件都不满足，那就不是“起点选错了”，而是根本不可能跑完，直接返回 `-1` 就行。

所以这题第一层判断其实是全局可行性。

### 真正关键的是第二层：为什么失败后可以直接跳过一整段

这是这题的核心。

假设我们从某个起点 `start` 出发，一路往后开，过程中不断累加剩余油量：

```text
currentSum += gas[i] - cost[i]
```

如果走到某个位置 `i` 时，`currentSum < 0`，说明什么？

说明从 `start` 出发，车连 `i + 1` 都到不了。

到这里最重要的不是“这个起点不行”，而是更进一步：

**从 `start` 到 `i` 之间的任何一个位置出发，也都不可能到达 `i + 1`。**

这才是贪心成立的关键。

### 为什么中间那些点也不可能作为起点

这个地方如果只记结论，会很容易忘。最好直接把原因想透。

假设从 `start` 出发走到 `i` 时已经亏空了，也就是：

```text
从 start 到 i 的总剩余 < 0
```

那说明这一整段路，整体上是入不敷出的。

现在假设中间某个位置 `k`（`start < k <= i`）居然可以作为合法起点。  
那就意味着从 `k` 出发，至少能顺利开到 `i + 1`。

可问题在于：

- 从 `start` 走到 `k` 这一段，如果有盈余，那它只会让从 `start` 出发更有优势
- 如果连“带着前面这段积累”都撑不到 `i + 1`
- 那么从中途某个位置 `k` 空油箱重新起步，只会更难，不会更容易

所以一旦从 `start` 到 `i` 失败了，就可以直接下结论：

**`start` 到 `i` 这一整段，全都不可能是答案。**

于是下一个有希望的起点，只能是：

```text
i + 1
```

这一步才是这题真正的贪心。

### 所以整个过程其实只做两件事

遍历数组时，同时维护三个量：

- `totalSum`：全局总剩余油量，用来判断到底有没有解
- `currentSum`：从当前候选起点出发，到当前位置为止的剩余油量
- `start`：当前候选起点

每到一个位置 `i`，先计算：

```text
diff = gas[i] - cost[i]
```

然后：

- `totalSum += diff`
- `currentSum += diff`

如果此时 `currentSum < 0`，说明当前起点失败了，那么：

- 直接把起点更新为 `i + 1`
- 把 `currentSum` 重置为 `0`

因为前面那一整段已经都可以排除了。

遍历结束后：

- 如果 `totalSum < 0`，返回 `-1`
- 否则返回最终找到的 `start`

### 用示例 1 走一遍

看这个例子：

```text
gas  = [1,2,3,4,5]
cost = [3,4,5,1,2]
```

先看每一站的净收益：

```text
gas[i] - cost[i] = [-2,-2,-2,3,3]
```

开始时：

- `start = 0`
- `currentSum = 0`
- `totalSum = 0`

扫到下标 `0`：

- `currentSum = -2`
- 已经小于 `0`

说明从 `0` 出发走不下去，所以：

- `start = 1`
- `currentSum = 0`

扫到下标 `1`：

- `currentSum = -2`
- 还是失败

所以：

- `start = 2`
- `currentSum = 0`

扫到下标 `2`：

- `currentSum = -2`
- 又失败

所以：

- `start = 3`
- `currentSum = 0`

接着看下标 `3`：

- `currentSum = 3`

看下标 `4`：

- `currentSum = 6`

后面不会再失败了。

最后虽然前面连续失败了几次，但全局总和是：

```text
-2 + -2 + -2 + 3 + 3 = 0
```

说明整体是可行的，所以答案就是最后留下来的 `start = 3`。

### 示例 2 为什么一定无解

再看：

```text
gas  = [2,3,4]
cost = [3,4,3]
```

净收益是：

```text
[-1,-1,1]
```

总和为：

```text
-1 + -1 + 1 = -1
```

既然总油量小于总消耗，那不管从哪里出发，都不可能跑完整圈，直接返回 `-1`。

这里甚至不需要关心具体从哪个点开始试，因为全局上就已经不成立了。

### 代码

```java
public int canCompleteCircuit(int[] gas, int[] cost) {
    int totalSum = 0;
    int currentSum = 0;
    int start = 0;

    for (int i = 0; i < gas.length; i++) {
        int diff = gas[i] - cost[i];
        totalSum += diff;
        currentSum += diff;

        if (currentSum < 0) {
            start = i + 1;
            currentSum = 0;
        }
    }

    return totalSum < 0 ? -1 : start;
}
```

时间复杂度是 $O(n)$，因为只遍历了一遍数组。  
空间复杂度是 $O(1)$，因为只用了常数个变量。

### 这题真正值得记住的地方

这题最值得记住的，不只是“加油站用贪心”这个结论，而是这种排除思路：

**当从某个起点出发在中途失败时，如果能证明这段区间里的所有点都会一起失败，就可以一次性跳过整段，而不是一个一个重试。**

这题里：

- `totalSum` 负责判断有没有解
- `currentSum` 负责判断当前候选起点会不会中途失败
- 一旦失败，就整段丢弃，起点直接跳到下一位

以后再遇到这种“线性扫描 + 区间整体排除”的题，也可以优先往这个方向想。

### 完整代码

```java
import java.util.Arrays;

public class Main {

    public static int canCompleteCircuit(int[] gas, int[] cost) {
        int totalSum = 0;
        int currentSum = 0;
        int start = 0;

        for (int i = 0; i < gas.length; i++) {
            int diff = gas[i] - cost[i];
            totalSum += diff;
            currentSum += diff;

            if (currentSum < 0) {
                start = i + 1;
                currentSum = 0;
            }
        }

        return totalSum < 0 ? -1 : start;
    }

    public static void main(String[] args) {
        int[] gas1 = new int[]{1, 2, 3, 4, 5};
        int[] cost1 = new int[]{3, 4, 5, 1, 2};

        System.out.println("示例 1 输入:");
        System.out.println("gas = " + Arrays.toString(gas1));
        System.out.println("cost = " + Arrays.toString(cost1));

        int result1 = canCompleteCircuit(gas1, cost1);

        System.out.println("示例 1 输出: " + result1);

        int[] gas2 = new int[]{2, 3, 4};
        int[] cost2 = new int[]{3, 4, 3};

        System.out.println("示例 2 输入:");
        System.out.println("gas = " + Arrays.toString(gas2));
        System.out.println("cost = " + Arrays.toString(cost2));

        int result2 = canCompleteCircuit(gas2, cost2);

        System.out.println("示例 2 输出: " + result2);
    }
}
```
