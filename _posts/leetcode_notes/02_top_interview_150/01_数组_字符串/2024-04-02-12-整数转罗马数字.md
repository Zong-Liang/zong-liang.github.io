## 题目

**描述：**

七个不同的符号代表罗马数字，其值如下：

| 符号 | 值   |
| :--- | :--- |
| I    | 1    |
| V    | 5    |
| X    | 10   |
| L    | 50   |
| C    | 100  |
| D    | 500  |
| M    | 1000 |

罗马数字通常是按照从大到小的顺序书写的。但也存在一些特例，例如 4 不写做 `IIII`，而是 `IV`。数字 1 在 5 的左边，表示 5 减 1 等于 4。同样地，9 表示为 `IX`。这个特殊的规则只适用于以下六种情况：

- `I` 可以放在 `V` (5) 和 `X` (10) 的左边，来表示 4 和 9。
- `X` 可以放在 `L` (50) 和 `C` (100) 的左边，来表示 40 和 90。
- `C` 可以放在 `D` (500) 和 `M` (1000) 的左边，来表示 400 和 900。

给定一个整数，将其转换为罗马数字。

**示例 1:**

- **输入:** `num = 3749`
- **输出:** `"MMMDCCXLIX"`

**示例 2:**

- **输入:** `num = 58`
- **输出:** `"LVIII"`

**示例 3:**

- **输入:** `num = 1994`
- **输出:** `"MCMXCIV"`

**提示:**

- `1 <= num <= 3999`

## 题解

这题和“罗马数字转整数”正好反过来：重点已经不在识别，而在构造。构造时最容易走弯路的地方，是把减法规则当成一堆临时特判去写，比如碰到 `4`、`9`、`40` 时再额外分支。其实没必要。更省事的做法，是一开始就把这些特殊值和普通值放进同一张表里，然后按从大到小去取。

为什么这种写法是顺的？因为罗马数字本身就带着“尽量先用大的块”这个味道。比如 `58`，自然会先写 `50`，也就是 `L`，剩下 `8`；再写 `5`，也就是 `V`，最后补上 `III`。`1994` 也是一样：先拿 `1000 -> M`，剩下 `994`；再拿 `900 -> CM`，剩下 `94`；接着 `90 -> XC`，最后 `4 -> IV`。一路拼下来就是 `MCMXCIV`。

这里好用的地方在于：我们不需要单独去判断“现在是不是该用减法写法了”。只要把 `900 -> CM`、`400 -> CD`、`90 -> XC`、`40 -> XL`、`9 -> IX`、`4 -> IV` 这些特殊块，和 `1000 -> M`、`500 -> D`、`100 -> C` 这些普通块一起按从大到小排好，遍历过程就统一了：当前数字够大，就取这一块，减掉对应的值，继续往下走。

这样做的时候，候选表的顺序很重要。必须先放 `900` 再放 `500` 和 `400`，先放 `90` 再放 `50` 和 `40`，否则像 `9` 这样的值就会被错误地拆开。顺序一旦排对，整个过程其实就是标准的贪心：每一步都先取当前还能使用的最大合法块。

代码里通常会准备两个一一对应的数组，一个放数值，一个放字符串。遍历数值数组时，只要 `num >= values[i]`，就把对应的罗马符号拼进答案，同时让 `num -= values[i]`，直到 `num` 变成 `0` 为止。

代码如下：

```java
public String intToRoman(int num) {
    int[] values = {1000, 900, 500, 400, 100, 90, 50, 40, 10, 9, 5, 4, 1};
    String[] symbols = {"M", "CM", "D", "CD", "C", "XC", "L", "XL", "X", "IX", "V", "IV", "I"};

    StringBuilder sb = new StringBuilder();

    for (int i = 0; i < values.length; i++) {
        while (num >= values[i]) {
            num -= values[i];
            sb.append(symbols[i]);
        }
    }

    return sb.toString();
}
```

时间复杂度可以记作 `O(1)`，因为题目把 `num` 限定在 `1` 到 `3999`，候选表长度也是固定的；空间复杂度同样是 `O(1)`。

这题挺适合顺手记住一种整理规则的方法：当一个表示系统里既有普通规则，又有少量固定的特殊规则时，往往不如把它们全都收进一张表里统一处理。这样一来，后面的构造过程反而更干净。

完整代码如下：

```java
public class Main {

    public static String intToRoman(int num) {
        int[] values = {1000, 900, 500, 400, 100, 90, 50, 40, 10, 9, 5, 4, 1};
        String[] symbols = {"M", "CM", "D", "CD", "C", "XC", "L", "XL", "X", "IX", "V", "IV", "I"};

        StringBuilder sb = new StringBuilder();

        for (int i = 0; i < values.length; i++) {
            while (num >= values[i]) {
                num -= values[i];
                sb.append(symbols[i]);
            }
        }

        return sb.toString();
    }

    public static void main(String[] args) {
        int num1 = 3749;
        System.out.println("示例 1 输入: num = " + num1);
        System.out.println("示例 1 输出: " + intToRoman(num1));

        int num2 = 58;
        System.out.println("示例 2 输入: num = " + num2);
        System.out.println("示例 2 输出: " + intToRoman(num2));

        int num3 = 1994;
        System.out.println("示例 3 输入: num = " + num3);
        System.out.println("示例 3 输出: " + intToRoman(num3));
    }
}
```
