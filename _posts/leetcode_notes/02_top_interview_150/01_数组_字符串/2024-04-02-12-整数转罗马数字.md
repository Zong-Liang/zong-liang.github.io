## 题目

**描述：**

七个不同的符号代表罗马数字，其值如下：

| 符号 | 值   |
| :--- | :--- |
| I    | 1    |
| V    | 5    |
| X    | 10   |
| L    | 50   |
| C    | 100  |
| D    | 500  |
| M    | 1000 |

罗马数字通常是按照从大到小的顺序书写的。但也存在一些特例，例如 4 不写做 `IIII`，而是 `IV`。数字 1 在 5 的左边，表示 5 减 1 等于 4。同样地，9 表示为 `IX`。这个特殊的规则只适用于以下六种情况：

- `I` 可以放在 `V` (5) 和 `X` (10) 的左边，来表示 4 和 9。
- `X` 可以放在 `L` (50) 和 `C` (100) 的左边，来表示 40 和 90。
- `C` 可以放在 `D` (500) 和 `M` (1000) 的左边，来表示 400 和 900。

给定一个整数，将其转换为罗马数字。

**示例 1:**

- **输入:** `num = 3749`
- **输出:** `"MMMDCCXLIX"`

**示例 2:**

- **输入:** `num = 58`
- **输出:** `"LVIII"`

**示例 3:**

- **输入:** `num = 1994`
- **输出:** `"MCMXCIV"`

**提示:**

- `1 <= num <= 3999`

## 题解

### 这题别先去背规则，先想“怎么拼”

如果是“罗马数字转整数”，重点在判断什么时候加、什么时候减。  
但这题反过来，重点已经不是“识别”，而是“构造”。

给你一个整数，要把它写成合法的罗马数字。  
这时候最自然的问题其实是：

**每一步应该优先拿哪个符号去拼？**

这才是这题真正的核心。

### 为什么这题本质上是贪心

罗马数字的写法，本身就带着一种“尽量先用大的”味道。

比如 `58`，你不会先想怎么凑 `8`，而是会先看到：

- 先拿 `50`，也就是 `L`
- 剩下 `8`
- 再拿 `5`，也就是 `V`
- 剩下 `3`
- 最后补上 `III`

于是得到：

```text
LVIII
```

这其实就是贪心：  
**每一步都先选当前能用的、最大的那个合法罗马数字。**

只要这个思路成立，整题就很好写了。

### 真正要注意的不是大数，而是那六个特殊值

如果没有减法规则，那事情会更简单，直接按 `1000, 500, 100, 50, 10, 5, 1` 往下减就行。

但罗马数字里有六个特殊写法：

- `4 -> IV`
- `9 -> IX`
- `40 -> XL`
- `90 -> XC`
- `400 -> CD`
- `900 -> CM`

这些值不能拆开写。

比如 `4` 不能写成 `IIII`，`9` 不能写成 `VIIII`。  
所以这题真正稳妥的写法，不是临时判断“是不是该减法了”，而是直接把这些特殊值也当成正常候选项，一起放进表里。

也就是说，我们不只维护：

- `1000 -> M`
- `500 -> D`
- `100 -> C`
- ...

还要把下面这些也一起放进去：

- `900 -> CM`
- `400 -> CD`
- `90 -> XC`
- `40 -> XL`
- `9 -> IX`
- `4 -> IV`

一旦把它们也视作普通的“可选块”，整题就会非常顺。

### 为什么把特殊值放进表里，贪心就能直接做

这里最关键的一点是：

**只要候选表本身已经包含所有合法的基本块，并且按从大到小排好，那每次优先取最大的那个，就是正确的。**

比如 `1994`：

- 先取 `1000 -> M`，剩下 `994`
- 再取 `900 -> CM`，剩下 `94`
- 再取 `90 -> XC`，剩下 `4`
- 再取 `4 -> IV`

最后就是：

```text
MCMXCIV
```

整个过程里，你根本不需要额外思考“这里是不是要减法”，因为 `900`、`90`、`4` 已经都作为完整块存在了。

所以这题真正的技巧不是在代码里判断特殊情况，而是：

**提前把特殊情况收进映射表，让遍历过程保持统一。**

### 用示例走一遍

先看：

```text
num = 58
```

按从大到小试：

- `1000` 不行
- `900` 不行
- ...
- `50` 可以，先放一个 `L`，剩下 `8`
- `5` 可以，放一个 `V`，剩下 `3`
- `1` 可以放三次，补成 `III`

最后得到：

```text
LVIII
```

再看：

```text
num = 1994
```

从大到小减：

- `1000` 可以，放 `M`，剩 `994`
- `900` 可以，放 `CM`，剩 `94`
- `90` 可以，放 `XC`，剩 `4`
- `4` 可以，放 `IV`

结果就是：

```text
MCMXCIV
```

再看：

```text
num = 3749
```

一步一步拆：

- `1000` 可以放三次，得到 `MMM`，剩 `749`
- `500` 可以放一次，得到 `MMMD`，剩 `249`
- `100` 可以放两次，得到 `MMMDC C`，剩 `49`
- `40` 可以放一次，得到 `MMMDCCXL`，剩 `9`
- `9` 可以放一次，得到 `MMMDCCXLIX`

也就是：

```text
MMMDCCXLIX
```

这样走几遍之后，这题的结构其实已经很固定了。

### 所以实现上只需要一张表

最简单的实现方式，就是准备两组一一对应的数据：

- 一个整数数组 `values`
- 一个字符串数组 `symbols`

内容按从大到小排列：

```text
1000, 900, 500, 400, 100, 90, 50, 40, 10, 9, 5, 4, 1
M,    CM,  D,   CD,  C,   XC, L,  XL, X,  IX, V, IV, I
```

然后从头往后扫：

- 只要当前 `num >= values[i]`
- 就把对应的 `symbols[i]` 拼到答案里
- 同时 `num -= values[i]`

一直做到 `num == 0` 为止。

### 代码

```java
public String intToRoman(int num) {
    int[] values = {1000, 900, 500, 400, 100, 90, 50, 40, 10, 9, 5, 4, 1};
    String[] symbols = {"M", "CM", "D", "CD", "C", "XC", "L", "XL", "X", "IX", "V", "IV", "I"};

    StringBuilder sb = new StringBuilder();

    for (int i = 0; i < values.length; i++) {
        while (num >= values[i]) {
            num -= values[i];
            sb.append(symbols[i]);
        }
    }

    return sb.toString();
}
```

时间复杂度可以记作 $O(1)$。  
因为题目范围固定在 `1 <= num <= 3999`，候选表长度也是固定的，循环次数有明确上界。  
空间复杂度也是 $O(1)$。

### 这题真正值得记住的地方

这题最值得记住的，不是“罗马数字怎么写”本身，而是这种处理方式：

**如果一个表示系统里既有普通规则，又有少量固定的特殊规则，最稳的办法往往不是边写边判断，而是直接把特殊情况也当成普通映射项，一起处理。**

这题里：

- 普通项有 `1000, 500, 100, 50, 10, 5, 1`
- 特殊项有 `900, 400, 90, 40, 9, 4`

把它们统一放进表里之后，整个构造过程就只剩一句话：

**从大到小，能拿多少拿多少。**

### 完整代码

```java
public class Main {

    public static String intToRoman(int num) {
        int[] values = {1000, 900, 500, 400, 100, 90, 50, 40, 10, 9, 5, 4, 1};
        String[] symbols = {"M", "CM", "D", "CD", "C", "XC", "L", "XL", "X", "IX", "V", "IV", "I"};

        StringBuilder sb = new StringBuilder();

        for (int i = 0; i < values.length; i++) {
            while (num >= values[i]) {
                num -= values[i];
                sb.append(symbols[i]);
            }
        }

        return sb.toString();
    }

    public static void main(String[] args) {
        int num1 = 3749;
        System.out.println("示例 1 输入: num = " + num1);
        System.out.println("示例 1 输出: " + intToRoman(num1));

        int num2 = 58;
        System.out.println("示例 2 输入: num = " + num2);
        System.out.println("示例 2 输出: " + intToRoman(num2));

        int num3 = 1994;
        System.out.println("示例 3 输入: num = " + num3);
        System.out.println("示例 3 输出: " + intToRoman(num3));
    }
}
```
