## 题目

**描述：**

给你一个非负整数数组 `nums` ，你最初位于数组的 **第一个下标** 。数组中的每个元素代表你在该位置可以跳跃的最大长度。

判断你是否能够到达最后一个下标，如果可以，返回 `true` ；否则，返回 `false` 。

**示例 1:**

- **输入:** `nums = [2,3,1,1,4]`
- **输出:** `true`
- **解释:** 可以先跳 1 步，从下标 0 到达下标 1, 然后再从下标 1 跳 3 步到达最后一个下标。

**示例 2:**

- **输入:** `nums = [3,2,1,0,4]`
- **输出:** `false`
- **解释:** 无论怎样，总会到达下标为 3 的位置。但该下标的最大跳跃长度是 0 ，所以永远不可能到达最后一个下标。

**提示:**

- `1 <= nums.length <= 10^4`
- `0 <= nums[i] <= 10^5`

## 题解

### 这题容易想偏的地方

很多人看到“每个位置可以跳多远”，第一反应会去想：

- 我这一步到底该跳几格
- 是不是要枚举所有跳法
- 要不要用搜索把所有路径试一遍

其实这题不需要真的把每一步跳出来。  
题目问的不是“怎么跳到终点”，而是**能不能到终点**。

这两个问题差很多。

一旦只关心“能不能到”，真正重要的就不再是某一步具体落在哪，而是：

**到当前为止，最远能覆盖到哪个位置。**

### 这题真正该维护的，是“最远可达位置”

假设我们从左往右扫数组。

当我们来到位置 `i` 时，真正应该关心的是：

- 这个位置能不能到
- 如果能到，从这里出发最远又能扩展到哪里

所以只需要维护一个变量：

- `rightmost`：当前能到达的最远下标

初始时我们站在下标 `0`，所以：

```java
rightmost = 0
```

接下来往后遍历数组时，每到一个位置 `i`：

- 如果 `i > rightmost`，说明这个位置根本到不了，那后面自然也不用看了，直接返回 `false`
- 如果 `i <= rightmost`，说明这个位置可达，那就用它来继续更新最远范围：

```java
rightmost = Math.max(rightmost, i + nums[i])
```

整个过程其实就是在不断维护一段“当前能覆盖到的区间”。

### 为什么不用真的决定“跳哪一步”

这是这题最关键的一层。

比如你现在站在某个位置，可以跳 1 步、2 步、3 步。  
直觉上好像要选一个最优跳法，但实际上完全没必要。

因为如果一个位置 `i` 可达，那它带来的价值只有一个：  
它能把最远可达范围扩展到 `i + nums[i]`。

至于你到底是怎么走到 `i` 的，并不重要。  
这题只关心覆盖范围能不能一路推到终点，而不是具体路径长什么样。

所以本质上，这题不是“路径选择题”，而是“区间覆盖题”。

### 用示例 1 走一遍

看这个例子：

```text
nums = [2,3,1,1,4]
```

开始时：

- `rightmost = 0`

来到下标 `0`：

- `0 <= rightmost`，说明当前位置可达
- 从这里最远能跳到 `0 + 2 = 2`
- 所以更新：

```text
rightmost = 2
```

来到下标 `1`：

- `1 <= 2`，可达
- 从这里最远能跳到 `1 + 3 = 4`
- 更新后：

```text
rightmost = 4
```

而最后一个下标正好是 `4`，这时候已经覆盖到终点了，所以可以直接返回 `true`。

注意这里我们并没有真的去模拟“先跳 1 步还是跳 2 步”，只是看最远覆盖范围有没有推过去。

### 示例 2 为什么不行

再看：

```text
nums = [3,2,1,0,4]
```

开始时：

- `rightmost = 0`

下标 `0` 可达，最远更新到 `3`。  
下标 `1` 可达，最远还是 `3`。  
下标 `2` 可达，最远还是 `3`。  
下标 `3` 也可达，但这里的值是 `0`，所以最远仍然只能停在 `3`。

接下来准备看下标 `4` 时，会发现：

```text
4 > rightmost
```

这说明下标 `4` 根本到不了，终点自然也到不了，只能返回 `false`。

这题的失败，不是因为“某一步选错了”，而是因为可达范围在某个位置彻底断掉了。

### 代码

```java
public boolean canJump(int[] nums) {
    int rightmost = 0;

    for (int i = 0; i < nums.length; i++) {
        if (i > rightmost) {
            return false;
        }

        rightmost = Math.max(rightmost, i + nums[i]);

        if (rightmost >= nums.length - 1) {
            return true;
        }
    }

    return true;
}
```

时间复杂度是 $O(n)$，因为整个数组只遍历了一遍。  
空间复杂度是 $O(1)$，因为只用了一个额外变量。

### 这题真正值得记住的地方

这题最值得记住的，不是“跳跃游戏要用贪心”这句话，而是这种判断方式：

**当题目只问“能不能到达”，而不要求具体路径时，很多时候不需要真的模拟路径，只需要维护当前能扩展到的最远边界。**

这题里：

- 每个可达位置都会给你一个新的覆盖范围
- 你要做的只是不断合并这些覆盖范围
- 一旦终点被覆盖到，答案就是 `true`
- 一旦出现当前位置已经超出覆盖范围，答案就是 `false`

以后再遇到类似“区间能否连续推进”“最远可达位置”“能否覆盖到终点”这类题，都可以优先往这个思路上想。

### 完整代码

```java
import java.util.Arrays;

public class Main {

    public static boolean canJump(int[] nums) {
        int rightmost = 0;

        for (int i = 0; i < nums.length; i++) {
            if (i > rightmost) {
                return false;
            }

            rightmost = Math.max(rightmost, i + nums[i]);

            if (rightmost >= nums.length - 1) {
                return true;
            }
        }

        return true;
    }

    public static void main(String[] args) {
        int[] nums1 = new int[]{2, 3, 1, 1, 4};

        System.out.println("示例 1 输入:");
        System.out.println("nums = " + Arrays.toString(nums1));

        boolean result1 = canJump(nums1);

        System.out.println("示例 1 输出: " + result1);

        int[] nums2 = new int[]{3, 2, 1, 0, 4};

        System.out.println("示例 2 输入:");
        System.out.println("nums = " + Arrays.toString(nums2));

        boolean result2 = canJump(nums2);

        System.out.println("示例 2 输出: " + result2);
    }
}
```
