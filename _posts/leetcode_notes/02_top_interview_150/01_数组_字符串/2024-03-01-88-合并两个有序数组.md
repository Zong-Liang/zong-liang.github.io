## 题目

**描述：**

给你两个按 **非递减顺序** 排列的整数数组 `nums1` 和 `nums2`，另有两个整数 `m` 和 `n` ，分别表示 `nums1` 和 `nums2` 中的元素数目。

请你 **合并** `nums2` 到 `nums1` 中，使合并后的数组同样按 **非递减顺序** 排列。

**注意：** 最终，合并后数组不应由函数返回，而是存储在数组 `nums1` 中。为了应对这种情况，`nums1` 的初始长度为 `m + n`，其中前 `m` 个元素表示应合并的元素，后 `n` 个元素为 `0` ，应忽略。`nums2` 的长度为 `n` 。

**示例 1:**

- **输入:** `nums1 = [1,2,3,0,0,0]`, `m = 3`, `nums2 = [2,5,6]`, `n = 3`
- **输出:** `[1,2,2,3,5,6]`
- **解释:** 需要合并 `[1,2,3]` 和 `[2,5,6]` 。

  合并结果是 `[1,2,2,3,5,6]`。

**示例 2:**

- **输入:** `nums1 = [1]`, `m = 1`, `nums2 = []`, `n = 0`
- **输出:** `[1]`
- **解释:** 需要合并 `[1]` 和 `[]` 。

  合并结果是 `[1]`。

**示例 3:**

- **输入:** `nums1 = [0]`, `m = 0`, `nums2 = [1]`, `n = 1`
- **输出:** `[1]`
- **解释:** 需要合并 `[]` 和 `[1]` 。

  合并结果是 `[1]`。

  注意，因为 `m = 0` ，所以 `nums1` 中没有元素。`nums1` 中仅存的 `0` 仅仅是为了确保合并结果可以顺利存放到 `nums1` 中。

**提示:**

- `nums1.length == m + n`
- `nums2.length == n`
- `0 <= m, n <= 200`
- `1 <= m + n <= 200`
- `-10^9 <= nums1[i], nums2[j] <= 10^9`

**进阶：** 你可以设计实现一个时间复杂度为 `O(m + n)` 的算法解决此问题吗？

## 题解

这题表面上只是“合并两个有序数组”，难点却不在“合并”本身，而在于答案必须直接写回 `nums1`。如果这是普通的归并问题，我们完全可以新开一个数组，从前往后比较、从前往后填值，最后再拷回去；但这题偏偏要求原地修改，而 `nums1` 的前 `m` 个位置里还放着尚未处理完的数据。做这题时，先要想清楚一件事：**在不覆盖旧数据的前提下，结果到底该从哪里开始放。**

最直观的做法当然是先把 `nums2` 接到 `nums1` 后面，再对整个数组排序：

```java
public void merge(int[] nums1, int m, int[] nums2, int n) {
    for (int i = 0; i < n; i++) {
        nums1[m + i] = nums2[i];
    }
    Arrays.sort(nums1);
}
```

这个方法能过题，也适合在考场上兜底，但它并没有真正利用“两个输入数组本身已经有序”这个条件。既然有序，合并本来就应该在线性时间里完成；如果最后还要整体再排一次序，复杂度就退化成 $O((m+n)\log(m+n))$，多少有点把现成信息浪费掉了。

很多人第一次做这题，会下意识沿用归并排序里的写法，从前往后把较小值写回 `nums1`。问题在于，`nums1` 的前 `m` 个位置一边是“旧数据”，一边又是“新结果应该落下的位置”，这两个角色是冲突的。以 `nums1 = [1,2,3,0,0,0]`、`nums2 = [2,5,6]` 为例，如果从前往后写，前面的空位还没完全腾出来，`nums1` 里尚未比较的 `2`、`3` 就可能被提前覆盖掉。

既然前面不安全，那就换个方向，从后面填。两个数组都是非递减排列，所以当前最大的元素只可能出现在 `nums1` 有效区的末尾或者 `nums2` 的末尾；而 `nums1` 的尾部又恰好预留了 `n` 个空位。这样一来，每次只要比较两个末尾元素，把较大的那个放到 `nums1[tail]`，就既不会破坏前面还没处理的数据，也能保证放进去的值已经处在最终位置。

于是三个指针就很自然了：`p1` 指向 `nums1` 有效部分的最后一个元素，`p2` 指向 `nums2` 的最后一个元素，`tail` 指向 `nums1` 的最后一个位置。循环里比较 `nums1[p1]` 和 `nums2[p2]`，谁大就把谁写到 `nums1[tail]`，然后移动对应的指针和 `tail`。整个过程中，每个元素最多只会被处理一次，所以时间复杂度可以控制在线性级别。

拿示例 1 走一遍会更直观。初始时 `p1 = 2`，指向 `3`；`p2 = 2`，指向 `6`；`tail = 5`。先比较 `3` 和 `6`，把 `6` 放到末尾，数组变成 `[1,2,3,0,0,6]`。接着比较 `3` 和 `5`，把 `5` 放到 `tail = 4`，得到 `[1,2,3,0,5,6]`。然后 `3` 比 `2` 大，于是 `3` 落到 `tail = 3`，数组变成 `[1,2,3,3,5,6]`。最后比较两个 `2`，放谁都可以，这里把 `nums2[p2]` 写进去，结果就是 `[1,2,2,3,5,6]`。这时 `nums2` 已经处理完，合并结束。

这里还有一个很容易忽略的收尾细节：循环结束后，不需要再专门处理 `nums1` 的剩余部分，只需要补 `nums2`。原因是如果 `nums2` 先耗尽，`nums1` 里那些还没动到的元素本来就已经在正确位置；只有在 `nums1` 先耗尽时，`nums2` 剩下的元素才需要继续往前补。

时间复杂度是 $O(m+n)$，因为两个指针总共最多移动 `m+n` 次；额外空间复杂度是 $O(1)$，整个过程只用了几个变量。

这题比较值得留下来的，是这个处理习惯：如果题目要求原地修改，而从前往后处理又会覆盖还没用到的数据，就可以想想能不能倒着做。本题里尾部刚好预留了空间，所以从后往前填会比从前往后写顺手得多。

完整代码如下：


```java
import java.util.Arrays;

public class Main {

    public static void merge(int[] nums1, int m, int[] nums2, int n) {
        int p1 = m - 1;
        int p2 = n - 1;
        int tail = m + n - 1;

        while (p1 >= 0 && p2 >= 0) {
            if (nums1[p1] > nums2[p2]) {
                nums1[tail] = nums1[p1];
                p1--;
            } else {
                nums1[tail] = nums2[p2];
                p2--;
            }
            tail--;
        }

        while (p2 >= 0) {
            nums1[tail] = nums2[p2];
            p2--;
            tail--;
        }
    }

    public static void main(String[] args) {
        int[] nums1_ex1 = new int[]{1, 2, 3, 0, 0, 0};
        int m1 = 3;
        int[] nums2_ex1 = new int[]{2, 5, 6};
        int n1 = 3;

        System.out.println("示例 1 输入:");
        System.out.println("nums1 = " + Arrays.toString(nums1_ex1) + ", m = " + m1);
        System.out.println("nums2 = " + Arrays.toString(nums2_ex1) + ", n = " + n1);

        merge(nums1_ex1, m1, nums2_ex1, n1);

        System.out.println("示例 1 输出: " + Arrays.toString(nums1_ex1));
        System.out.println("-----------------------------");

        int[] nums1_ex2 = new int[]{1};
        int m2 = 1;
        int[] nums2_ex2 = new int[]{};
        int n2 = 0;

        System.out.println("示例 2 输入:");
        System.out.println("nums1 = " + Arrays.toString(nums1_ex2) + ", m = " + m2);
        System.out.println("nums2 = " + Arrays.toString(nums2_ex2) + ", n = " + n2);

        merge(nums1_ex2, m2, nums2_ex2, n2);

        System.out.println("示例 2 输出: " + Arrays.toString(nums1_ex2));
        System.out.println("-----------------------------");

        int[] nums1_ex3 = new int[]{0};
        int m3 = 0;
        int[] nums2_ex3 = new int[]{1};
        int n3 = 1;

        System.out.println("示例 3 输入:");
        System.out.println("nums1 = " + Arrays.toString(nums1_ex3) + ", m = " + m3);
        System.out.println("nums2 = " + Arrays.toString(nums2_ex3) + ", n = " + n3);

        merge(nums1_ex3, m3, nums2_ex3, n3);

        System.out.println("示例 3 输出: " + Arrays.toString(nums1_ex3));
    }
}
```
