## 题目

**描述：**

给你两个按 **非递减顺序** 排列的整数数组 `nums1` 和 `nums2`，另有两个整数 `m` 和 `n` ，分别表示 `nums1` 和 `nums2` 中的元素数目。

请你 **合并** `nums2` 到 `nums1` 中，使合并后的数组同样按 **非递减顺序** 排列。

**注意：** 最终，合并后数组不应由函数返回，而是存储在数组 `nums1` 中。为了应对这种情况，`nums1` 的初始长度为 `m + n`，其中前 `m` 个元素表示应合并的元素，后 `n` 个元素为 `0` ，应忽略。`nums2` 的长度为 `n` 。

**示例 1:**

- **输入:** `nums1 = [1,2,3,0,0,0]`, `m = 3`, `nums2 = [2,5,6]`, `n = 3`
- **输出:** `[1,2,2,3,5,6]`
- **解释:** 需要合并 `[1,2,3]` 和 `[2,5,6]` 。

  合并结果是 `[1,2,2,3,5,6]`。

**示例 2:**

- **输入:** `nums1 = [1]`, `m = 1`, `nums2 = []`, `n = 0`
- **输出:** `[1]`
- **解释:** 需要合并 `[1]` 和 `[]` 。

  合并结果是 `[1]`。

**示例 3:**

- **输入:** `nums1 = [0]`, `m = 0`, `nums2 = [1]`, `n = 1`
- **输出:** `[1]`
- **解释:** 需要合并 `[]` 和 `[1]` 。

  合并结果是 `[1]`。

  注意，因为 `m = 0` ，所以 `nums1` 中没有元素。`nums1` 中仅存的 `0` 仅仅是为了确保合并结果可以顺利存放到 `nums1` 中。

**提示:**

- `nums1.length == m + n`
- `nums2.length == n`
- `0 <= m, n <= 200`
- `1 <= m + n <= 200`
- `-10^9 <= nums1[i], nums2[j] <= 10^9`

**进阶：** 你可以设计实现一个时间复杂度为 `O(m + n)` 的算法解决此问题吗？

## 题解

### 先想清楚这题难在哪里

这题表面上是在考“合并两个有序数组”，但真正麻烦的地方其实不是“合并”，而是**结果必须直接写回 `nums1`**。

如果这是普通的归并问题，我们完全可以新开一个数组，从前往后比较、从前往后填入，最后再拷贝回去。但这题不行，因为 `nums1` 的前 `m` 个位置本来就存着有效元素，处理时稍不注意，就会把还没比较的数据提前覆盖掉。

所以这题的关键不是“会不会双指针”，而是先想明白一件事：

**在原地修改 `nums1` 的前提下，结果应该从哪里开始放？**

### 先说一个最直接的做法

最直观的思路，是先把 `nums2` 的元素放到 `nums1` 后面，再对整个 `nums1` 排序。

```java
public void merge(int[] nums1, int m, int[] nums2, int n) {
    for (int i = 0; i < n; i++) {
        nums1[m + i] = nums2[i];
    }
    Arrays.sort(nums1);
}
```

这个方法当然能过，写起来也简单，但它其实没有真正利用题目给出的“两个数组本身已经有序”这个条件。

既然已经有序，那么合并本身就应该可以在线性时间内完成；如果最后还要整体排序，其实等于又多做了一遍没必要的工作。

时间复杂度是 $O((m+n)\log(m+n))$，空间复杂度和排序实现有关，一般记作 $O(1)$ 或 $O(\log(m+n))$。

这个解法适合拿来兜底，但不是这题真正想考的东西。

### 问题就出在不能从前往后写

很多人第一次做这题时，都会本能地想从前往后合并。毕竟两个数组都有序，看起来和归并排序里的合并过程很像。

但这里有个很容易忽略的问题：如果你从前往后把结果写回 `nums1`，那 `nums1` 前面那些还没来得及参与比较的元素就可能被覆盖掉。

例如：

```text
nums1 = [1,2,3,0,0,0], m = 3
nums2 = [2,5,6], n = 3
```

`nums1` 的前半部分既是“已有数据”，又是“未来要写结果的位置”。这两个角色冲突了，所以从前往后放并不安全。

既然前面的位置不安全，那就只能反过来想：**从后往前放会不会更合适？**

### 从后往前放，问题就顺了

这两个数组都是非递减顺序排列的，所以当前最大的元素，只可能出现在：

- `nums1` 有效部分的末尾
- `nums2` 的末尾

而 `nums1` 的最后面，刚好预留了足够的空位。

这就意味着我们每次只要比较这两个末尾元素，把较大的那个放到 `nums1` 的最后一个可用位置上，就可以了。这样既不会破坏前面还没处理的数据，也能保证放进去的数已经处在最终位置。

思路一旦转到这里，整题就很自然了。

### 三个指针怎么设

我们定义三个指针：

- `p1` 指向 `nums1` 有效部分的最后一个元素，也就是 `m - 1`
- `p2` 指向 `nums2` 的最后一个元素，也就是 `n - 1`
- `tail` 指向 `nums1` 的最后一个位置，也就是 `m + n - 1`

接下来不断比较 `nums1[p1]` 和 `nums2[p2]`：

- 谁大，就把谁放到 `nums1[tail]`
- 对应的指针左移
- `tail` 也跟着左移

一直做到某个数组先处理完为止。

### 用示例走一遍

以示例 1 为例：

```text
nums1 = [1,2,3,0,0,0], m = 3
nums2 = [2,5,6], n = 3
```

初始时：

- `p1 = 2`，指向 `3`
- `p2 = 2`，指向 `6`
- `tail = 5`

先比较 `3` 和 `6`，较大的是 `6`，放到 `nums1[5]`：

```text
[1,2,3,0,0,6]
```

然后 `p2--`，`tail--`。

再比较 `3` 和 `5`，较大的是 `5`，放到 `nums1[4]`：

```text
[1,2,3,0,5,6]
```

然后 `p2--`，`tail--`。

接着比较 `3` 和 `2`，较大的是 `3`，放到 `nums1[3]`：

```text
[1,2,3,3,5,6]
```

然后 `p1--`，`tail--`。

再比较 `2` 和 `2`，放谁都可以。这里放 `nums2[p2]`，结果变成：

```text
[1,2,2,3,5,6]
```

到这里 `nums2` 已经处理完，合并结束。

### 为什么最后只需要处理 `nums2`

循环结束后，通常有两种情况。

第一种是 `nums2` 先处理完了。  
这时候 `nums1` 里剩下的那些元素，本来就在正确位置上，不需要再动。

第二种是 `nums1` 先处理完了。  
这说明 `nums2` 还有剩余元素没放进去，这部分就必须继续补到 `nums1` 前面。

所以最后只需要补 `nums2` 的剩余部分：

```java
while (p2 >= 0) {
    nums1[tail] = nums2[p2];
    p2--;
    tail--;
}
```

而不需要再专门处理 `p1`。

### 代码

```java
public void merge(int[] nums1, int m, int[] nums2, int n) {
    int p1 = m - 1;
    int p2 = n - 1;
    int tail = m + n - 1;

    while (p1 >= 0 && p2 >= 0) {
        if (nums1[p1] > nums2[p2]) {
            nums1[tail] = nums1[p1];
            p1--;
        } else {
            nums1[tail] = nums2[p2];
            p2--;
        }
        tail--;
    }

    while (p2 >= 0) {
        nums1[tail] = nums2[p2];
        p2--;
        tail--;
    }
}
```

时间复杂度是 $O(m+n)$，因为两个指针总共最多移动 `m+n` 次；空间复杂度是 $O(1)$，整个过程只用了几个额外变量。

### 这题真正值得记住的地方

这题真正值得记住的，其实不是“合并两个有序数组”这句话本身，而是这种处理方式：

**当题目要求原地修改，而从前往后处理又可能覆盖未处理数据时，就可以想一想，能不能倒着做。**

本题里，当前最大值很容易确定，`nums1` 尾部又刚好留出了空间，所以从后往前填就是最顺手、也最稳妥的办法。

以后再遇到“原地覆盖”“尾部预留空间”“有序合并”这类题，也可以优先往这个方向想。

### 完整代码

```java
import java.util.Arrays;

public class Main {

    public static void merge(int[] nums1, int m, int[] nums2, int n) {
        int p1 = m - 1;
        int p2 = n - 1;
        int tail = m + n - 1;

        while (p1 >= 0 && p2 >= 0) {
            if (nums1[p1] > nums2[p2]) {
                nums1[tail] = nums1[p1];
                p1--;
            } else {
                nums1[tail] = nums2[p2];
                p2--;
            }
            tail--;
        }

        while (p2 >= 0) {
            nums1[tail] = nums2[p2];
            p2--;
            tail--;
        }
    }

    public static void main(String[] args) {
        int[] nums1_ex1 = new int[]{1, 2, 3, 0, 0, 0};
        int m1 = 3;
        int[] nums2_ex1 = new int[]{2, 5, 6};
        int n1 = 3;

        System.out.println("示例 1 输入:");
        System.out.println("nums1 = " + Arrays.toString(nums1_ex1) + ", m = " + m1);
        System.out.println("nums2 = " + Arrays.toString(nums2_ex1) + ", n = " + n1);

        merge(nums1_ex1, m1, nums2_ex1, n1);

        System.out.println("示例 1 输出: " + Arrays.toString(nums1_ex1));
        System.out.println("-----------------------------");

        int[] nums1_ex2 = new int[]{1};
        int m2 = 1;
        int[] nums2_ex2 = new int[]{};
        int n2 = 0;

        System.out.println("示例 2 输入:");
        System.out.println("nums1 = " + Arrays.toString(nums1_ex2) + ", m = " + m2);
        System.out.println("nums2 = " + Arrays.toString(nums2_ex2) + ", n = " + n2);

        merge(nums1_ex2, m2, nums2_ex2, n2);

        System.out.println("示例 2 输出: " + Arrays.toString(nums1_ex2));
        System.out.println("-----------------------------");

        int[] nums1_ex3 = new int[]{0};
        int m3 = 0;
        int[] nums2_ex3 = new int[]{1};
        int n3 = 1;

        System.out.println("示例 3 输入:");
        System.out.println("nums1 = " + Arrays.toString(nums1_ex3) + ", m = " + m3);
        System.out.println("nums2 = " + Arrays.toString(nums2_ex3) + ", n = " + n3);

        merge(nums1_ex3, m3, nums2_ex3, n3);

        System.out.println("示例 3 输出: " + Arrays.toString(nums1_ex3));
    }
}
```
