## 题目

**描述：**

给你一个长度为 `n` 的链表，每个节点包含一个额外增加的随机指针 `random`，该指针可以指向链表中的任何节点或空节点。

构造这个链表的 **深拷贝**。深拷贝应该正好由 `n` 个 **全新** 节点组成，其中每个新节点的值都设为其对应的原节点的值。新节点的 `next` 指针和 `random` 指针也都应指向复制链表中的新节点，并使原链表和复制链表中的这些指针能够表示相同的链表状态。**复制链表中的指针都不应指向原链表中的节点**。

返回复制链表的头节点。

**示例 1:**

![示例1图片](https://assets.leetcode.com/uploads/2019/12/18/e1.png)

- **输入:** `head = [[7,null],[13,0],[11,4],[10,2],[1,0]]`
- **输出:** `[[7,null],[13,0],[11,4],[10,2],[1,0]]`

**示例 2:**

- **输入:** `head = [[1,1],[2,1]]`
- **输出:** `[[1,1],[2,1]]`

**提示:**

- `0 <= n <= 1000`
- `-10^4 <= Node.val <= 10^4`
- `Node.random` 为 `null` 或指向链表中的节点。

## 题解

### 先看这题到底难在哪

这题如果只有 `next` 指针，其实并不难，顺着链表一个个复制节点就行。真正让它变复杂的，是每个节点还多了一个 `random` 指针，而且这个指针可以：

- 指向任意一个节点
- 也可能是 `null`

这就意味着，复制时不能只顾着把节点值抄过去，还必须把原链表里的“指向关系”也完整搬过去。

而且题目要求的是**深拷贝**。这句话的意思一定要先想清楚：

- 新链表里的每个节点都必须是新建出来的
- 新链表里的 `next` 和 `random` 只能指向新链表中的节点
- 绝不能让新链表里的指针还指回原链表

所以这题真正要解决的问题，其实不是“怎么复制值”，而是：

**如何建立“原节点”和“新节点”之间的一一对应关系。**

### 最容易想到的方法：用哈希表保存映射

这题最直观的解法，就是用一个哈希表把映射关系记下来：

```text
原节点  ->  对应的新节点
```

只要这个映射建好了，后面事情就简单了。

比如原链表里有一个节点 `A`，我们知道它对应的新节点是 `A'`。  
如果 `A.next` 指向 `B`，那么 `A'` 的 `next` 就应该指向 `B'`；  
如果 `A.random` 指向 `C`，那么 `A'` 的 `random` 就应该指向 `C'`。

而 `B'`、`C'` 到底是谁，正好可以通过哈希表查出来。

所以这题用哈希表做，本质上就是先把“对应关系”固定住，再去补指针。

### 为什么要分成两轮遍历

很多人第一次看到这题时，会想一边遍历一边把 `next` 和 `random` 全都设好。问题在于，`random` 指向的节点不一定已经复制出来了。

也就是说，当你刚走到某个节点时：

- 它自己的新节点也许刚建好
- 但它的 `random` 目标节点，可能还在链表后面，暂时还没有副本

所以一步到位并不稳。

更自然的做法是分两轮：

第一轮，只做一件事：  
**把所有新节点都创建出来，并存进哈希表。**

第二轮，再做另一件事：  
**根据哈希表，把每个新节点的 `next` 和 `random` 补完整。**

这样思路就很顺，不会遇到“目标节点还没创建”的问题。

### 哈希表解法怎么理解

第一轮遍历时，我们只复制节点值：

```java
map.put(curr, new Node(curr.val));
```

这里的意思是：  
原链表里的 `curr`，在新链表里对应一个值相同的新节点。

注意这一步只是“把人找齐”，还没有把它们之间的关系连起来。

第二轮遍历时，再去设置指针：

```java
map.get(curr).next = map.get(curr.next);
map.get(curr).random = map.get(curr.random);
```

这两句很关键。

- `map.get(curr)`：拿到当前原节点对应的新节点
- `map.get(curr.next)`：拿到“原节点的下一个节点”对应的新节点
- `map.get(curr.random)`：拿到“原节点的随机指向节点”对应的新节点

这样一来，原链表里的连接关系就被原样复制到新链表里了。

这里还有一个很顺手的地方：  
如果 `curr.next` 或 `curr.random` 本来就是 `null`，那么 `map.get(null)` 也会得到 `null`，刚好符合预期，不需要额外写很多判断。

### 哈希表解法

```java
public Node copyRandomList(Node head) {
    if (head == null) {
        return null;
    }

    Map<Node, Node> map = new HashMap<>();
    Node curr = head;

    while (curr != null) {
        map.put(curr, new Node(curr.val));
        curr = curr.next;
    }

    curr = head;
    while (curr != null) {
        map.get(curr).next = map.get(curr.next);
        map.get(curr).random = map.get(curr.random);
        curr = curr.next;
    }

    return map.get(head);
}
```

### 这个方法为什么好用

这个解法最大的优点，就是思路特别稳定。

它没有试图在复制过程中“顺手猜出”某个 `random` 应该连到哪里，而是先把所有副本节点统一建好，再通过映射关系一次性把连接补齐。

所以你可以把它理解成两步：

- 先复制所有节点
- 再复制所有边

一旦这么理解，这题就不会乱。

时间复杂度是 $O(n)$，因为链表一共遍历两次。  
空间复杂度是 $O(n)$，因为哈希表里要存每个原节点和新节点的映射关系。

### 还有一种更巧的做法：把副本节点插进原链表

如果继续追求空间优化，可以不用哈希表，而是直接利用原链表本身来临时保存映射关系。

这个做法的核心很巧：

假设原链表是：

```text
A -> B -> C
```

我们复制每个节点后，不是把副本单独放到别处，而是直接插到原节点后面：

```text
A -> A' -> B -> B' -> C -> C'
```

这样一来，原节点和副本节点的对应关系就不用哈希表存了，因为：

**每个原节点的下一个节点，就是它自己的副本。**

这时候如果原节点 `A` 的 `random` 指向 `C`，那么 `A'` 的 `random` 就应该指向 `C'`。  
而 `C'` 恰好就是 `C.next`。

所以有：

```text
A'.random = A.random.next
```

这就是这个解法最关键的一步。

### 拼接再拆分的过程

整个过程可以分成三步。

第一步，复制每个节点，并把副本插到原节点后面。

```text
A -> B -> C
变成
A -> A' -> B -> B' -> C -> C'
```

第二步，设置副本节点的 `random`。

因为每个副本节点都紧跟在原节点后面，所以：

- 原节点是 `curr`
- 副本节点就是 `curr.next`
- 原节点的 `random` 指向谁，它的副本就是谁的 `next`

所以可以写成：

```java
curr.next.random = curr.random.next;
```

当然前提是 `curr.random != null`。

第三步，把交织在一起的链表拆开：

- 一条恢复成原链表
- 另一条提取成复制后的新链表

这样就能在不额外开哈希表的情况下完成深拷贝。

### 空间优化写法

```java
public Node copyRandomList(Node head) {
    if (head == null) {
        return null;
    }

    Node curr = head;
    while (curr != null) {
        Node newNode = new Node(curr.val);
        newNode.next = curr.next;
        curr.next = newNode;
        curr = newNode.next;
    }

    curr = head;
    while (curr != null) {
        if (curr.random != null) {
            curr.next.random = curr.random.next;
        }
        curr = curr.next.next;
    }

    Node newHead = head.next;
    curr = head;

    while (curr != null) {
        Node copy = curr.next;
        curr.next = copy.next;
        if (copy.next != null) {
            copy.next = copy.next.next;
        }
        curr = curr.next;
    }

    return newHead;
}
```

这个方法时间复杂度同样是 $O(n)$，但空间复杂度可以做到 $O(1)$`。  
不过它对链表结构做了临时修改，理解门槛会比哈希表解法高一些。

### 这题更推荐先掌握哪一种

如果是第一次做这题，建议先把哈希表解法彻底吃透。

原因很简单：

- 它更符合人的直觉
- 思路清楚
- 出错点少
- 面试时也容易讲明白

等哈希表解法完全想通之后，再去看“拼接 + 拆分”的做法，会更容易明白它到底巧在哪里。

学习顺序上，通常应该是：

- 先会写哈希表版本
- 再理解空间优化版本

不要一上来就硬记 `curr.next.random = curr.random.next` 这种结论，不然很容易写出来但自己其实没真正懂。

### 这题真正值得记住的地方

这题真正值得记住的，不只是“随机链表怎么复制”，而是这种处理思路：

**当一个结构里的节点之间存在复杂引用关系时，核心往往不是复制节点本身，而是先想办法建立原对象和新对象之间的映射。**

哈希表做的是显式映射；  
拼接法做的是隐式映射。

两种方法表面不一样，但本质上都在解决同一个问题：  
**原节点对应的新节点，到底是谁。**

只要这件事想明白了，这题就不会再显得杂乱。

### 完整代码

```java
import java.util.HashMap;
import java.util.Map;

public class Main {

    static class Node {
        int val;
        Node next;
        Node random;

        public Node(int val) {
            this.val = val;
        }
    }

    public static Node copyRandomList(Node head) {
        if (head == null) {
            return null;
        }

        Map<Node, Node> map = new HashMap<>();

        Node curr = head;
        while (curr != null) {
            map.put(curr, new Node(curr.val));
            curr = curr.next;
        }

        curr = head;
        while (curr != null) {
            Node newNode = map.get(curr);
            newNode.next = map.get(curr.next);
            newNode.random = map.get(curr.random);
            curr = curr.next;
        }

        return map.get(head);
    }

    public static void printList(Node head) {
        Node curr = head;
        StringBuilder sb = new StringBuilder();
        sb.append("[");
        while (curr != null) {
            String randomVal = (curr.random != null) ? String.valueOf(curr.random.val) : "null";
            sb.append("[").append(curr.val).append(", ").append(randomVal).append("]");
            if (curr.next != null) {
                sb.append(", ");
            }
            curr = curr.next;
        }
        sb.append("]");
        System.out.println(sb.toString());
    }

    public static void main(String[] args) {
        Node n0 = new Node(7);
        Node n1 = new Node(13);
        Node n2 = new Node(11);
        Node n3 = new Node(10);
        Node n4 = new Node(1);

        n0.next = n1;
        n1.next = n2;
        n2.next = n3;
        n3.next = n4;

        n0.random = null;
        n1.random = n0;
        n2.random = n4;
        n3.random = n2;
        n4.random = n0;

        System.out.println("示例 1 原链表:");
        printList(n0);

        Node copiedList1 = copyRandomList(n0);

        System.out.println("示例 1 新链表:");
        printList(copiedList1);

        System.out.println("验证深拷贝: 修改原链表头节点值为 99");
        n0.val = 99;
        System.out.print("原链表: ");
        printList(n0);
        System.out.print("新链表: ");
        printList(copiedList1);
        System.out.println("-----------------------------");

        Node a0 = new Node(1);
        Node a1 = new Node(2);
        a0.next = a1;
        a0.random = a1;
        a1.random = a1;

        System.out.println("示例 2 原链表:");
        printList(a0);
        Node copiedList2 = copyRandomList(a0);
        System.out.println("示例 2 新链表:");
        printList(copiedList2);
    }
}
```
