## 题目

**描述：**

给你单链表的头指针 `head` 和两个整数 `left` 和 `right`，其中 `left <= right`。请你反转从位置 `left` 到位置 `right` 的链表节点，返回 **反转后的链表**。

**示例 1:**

![示例1图片](https://assets.leetcode.com/uploads/2021/02/19/rev2ex2.jpg)

- **输入:** `head = [1,2,3,4,5]`, `left = 2`, `right = 4`
- **输出:** `[1,4,3,2,5]`

**示例 2:**

- **输入:** `head = [5]`, `left = 1`, `right = 1`
- **输出:** `[5]`

**提示:**

- 链表中节点数目为 `n`
- `1 <= n <= 500`
- `-500 <= Node.val <= 500`
- `1 <= left <= right <= n`

**进阶：** 你可以使用一趟扫描完成反转吗？

## 题解

### 先看这题和普通反转链表有什么不同

如果是整条链表反转，思路比较直接，一路改指针就行。  
但这题不是反转整条链表，而是只反转第 `left` 个到第 `right` 个节点之间这一段，其他部分必须保持原样。

所以这题真正的难点，不是“会不会反转链表”，而是：

**如何只改中间这一段的指针，同时把前后两部分重新接好。**

把这个问题想清楚之后，这题就不会显得乱。

### 最容易想到的办法：先切出来，再反转，再接回去

先从整体上看，这题完全可以拆成三步：

1. 找到需要反转的那一段
2. 把这一段单独反转
3. 再把反转后的结果接回原链表

比如：

```text
1 -> 2 -> 3 -> 4 -> 5
left = 2, right = 4
```

真正要反转的部分其实就是：

```text
2 -> 3 -> 4
```

如果把它单独拿出来，反转后变成：

```text
4 -> 3 -> 2
```

然后再接回去，整条链表就变成：

```text
1 -> 4 -> 3 -> 2 -> 5
```

这个思路很自然，因为它本质上是在做一件很朴素的事：

**把局部问题先独立出来，再按普通反转链表去处理。**

### 为什么要先找到三个关键位置

如果想把中间这一段切出来再接回去，至少要先找到这几个位置：

- `left` 前一个节点
- `left` 位置的节点
- `right` 位置的节点
- `right` 后一个节点

其中最关键的是两头的连接点。

还是刚才这个例子：

```text
1 -> 2 -> 3 -> 4 -> 5
```

如果反转区间是 `[2, 4]`，那：

- `1` 是反转区间前面的节点
- `2` 是反转区间原来的头
- `4` 是反转区间原来的尾
- `5` 是反转区间后面的节点

反转完成后：

- 前面的 `1` 要接到新的区间头 `4`
- 原来的区间头 `2` 会变成新的区间尾，它要接回 `5`

也就是说，这题虽然表面上在反转链表，真正决定能不能做对的，其实是**前后连接关系有没有理顺**。

### 截断再反转的写法

这种写法思路最直，适合先理解题目。

```java
public ListNode reverseBetween(ListNode head, int left, int right) {
    ListNode dummy = new ListNode(-1);
    dummy.next = head;

    ListNode pre = dummy;
    for (int i = 0; i < left - 1; i++) {
        pre = pre.next;
    }

    ListNode rightNode = pre;
    for (int i = 0; i < right - left + 1; i++) {
        rightNode = rightNode.next;
    }

    ListNode leftNode = pre.next;
    ListNode succ = rightNode.next;

    pre.next = null;
    rightNode.next = null;

    reverseLinkedList(leftNode);

    pre.next = rightNode;
    leftNode.next = succ;

    return dummy.next;
}

private void reverseLinkedList(ListNode head) {
    ListNode pre = null;
    ListNode cur = head;
    while (cur != null) {
        ListNode next = cur.next;
        cur.next = pre;
        pre = cur;
        cur = next;
    }
}
```

这套做法的核心在于：  
先把要反转的区间单独切出来，反转完后再拼回去。

它不算最巧，但很适合理解整道题的结构。

### 不过这题更常用的，是头插法

如果继续往下想，其实这题没必要真的“切出来再反转”。  
因为我们只想反转局部，而局部反转的过程，本身就可以在原链表上直接完成。

这里最常见的技巧，就是**头插法**。

它的思路不是把整段拿出来翻转，而是：

**把区间后面的节点，一个一个摘出来，依次插到区间最前面。**

听起来有点抽象，但一走过程就很清楚。

### 头插法到底在做什么

还是这个例子：

```text
1 -> 2 -> 3 -> 4 -> 5
left = 2, right = 4
```

先让 `pre` 指向反转区间前一个节点，也就是 `1`；  
再让 `cur` 指向反转区间第一个节点，也就是 `2`。

这时候链表还是：

```text
1 -> 2 -> 3 -> 4 -> 5
```

注意，`cur` 一开始指向 `2`，后面它会始终留在反转区间的尾部位置附近不动。真正被不断摘出来往前插的是它后面的节点。

#### 第 1 轮

把 `3` 从 `2` 后面摘出来，插到 `1` 后面：

原来是：

```text
1 -> 2 -> 3 -> 4 -> 5
```

变成：

```text
1 -> 3 -> 2 -> 4 -> 5
```

#### 第 2 轮

再把 `4` 从 `2` 后面摘出来，插到 `1` 后面：

原来是：

```text
1 -> 3 -> 2 -> 4 -> 5
```

变成：

```text
1 -> 4 -> 3 -> 2 -> 5
```

到这里，区间 `[2,4]` 就已经反转完成了。

所以头插法的本质，不是“整段翻转”，而是：

**固定住区间前驱和区间尾巴，把中间节点不断往前搬。**

一旦这么理解，这个方法就很好记。

### 为什么这种写法只需要扫描一遍

因为从找到 `left - 1` 位置开始，后面的操作都是在原地完成的。

我们没有再去额外找一遍区间尾，也没有把链表切开后再单独反转一遍，而是边走边改指针，直接在局部把顺序调整好。

这也就是题目里“能不能一趟扫描完成”的核心意思。

### 头插法代码

这题最推荐掌握的，还是下面这个版本：

```java
public ListNode reverseBetween(ListNode head, int left, int right) {
    ListNode dummy = new ListNode(0);
    dummy.next = head;

    ListNode pre = dummy;
    for (int i = 0; i < left - 1; i++) {
        pre = pre.next;
    }

    ListNode cur = pre.next;

    for (int i = 0; i < right - left; i++) {
        ListNode next = cur.next;
        cur.next = next.next;
        next.next = pre.next;
        pre.next = next;
    }

    return dummy.next;
}
```

### 这几句代码到底该怎么理解

这段代码最容易卡人的地方，就是循环里的三次指针修改：

```java
ListNode next = cur.next;
cur.next = next.next;
next.next = pre.next;
pre.next = next;
```

可以把它拆成三步看。

第一步，先把 `next` 暂存下来。  
这里的 `next` 是 `cur` 后面的那个节点，也就是这一轮准备被“提到前面去”的节点。

第二步，把 `next` 从原位置摘出来：

```java
cur.next = next.next;
```

这样 `cur` 就直接跳过了 `next`。

第三步，把 `next` 插到 `pre` 后面：

```java
next.next = pre.next;
pre.next = next;
```

这样 `next` 就被放到了当前反转区间的最前面。

整个循环每执行一次，就把一个节点搬到前面去。执行 `right - left` 次之后，整个区间就完成反转了。

### 为什么要用 dummy 节点

这题里 `dummy` 不是装饰，它是很有必要的。

因为当 `left = 1` 时，反转区间从头节点开始。  
如果没有 `dummy`，那“反转区间前一个节点”这个位置就不存在，代码里就得额外分情况处理。

加上 `dummy` 之后，无论 `left` 是多少，都可以统一认为：

- `pre` 总是指向反转区间前一个位置
- 最后统一返回 `dummy.next`

这样代码会稳定很多。

### 复杂度分析

时间复杂度是 $O(n)$。  
我们最多遍历链表一次，找到 `left` 前一个节点后，再在区间内做有限次指针调整。

空间复杂度是 $O(1)$。  
整个过程只用了几个指针变量，属于原地修改。

### 这题真正值得记住的地方

这题真正值得记住的，不只是“反转链表某一段”这个结论，而是这种处理局部链表问题的方式：

**当题目只要求修改链表中的某个区间时，重点往往不是整条链表怎么处理，而是先找到区间前驱，再围绕这个前驱做局部穿针引线。**

很多链表题看起来在考“反转”，其实真正难的是连接关系。  
谁是前驱，谁是尾巴，谁要摘出来，谁要接回去，这些关系一旦理顺，代码自然就顺了。

这题里的头插法，就是一个很典型的例子。

### 完整代码

```java
import java.util.*;

// 链表节点定义
class ListNode {
    int val;
    ListNode next;

    ListNode() {}

    ListNode(int val) {
        this.val = val;
    }

    ListNode(int val, ListNode next) {
        this.val = val;
        this.next = next;
    }
}

public class Main {

    public static ListNode reverseBetween(ListNode head, int left, int right) {
        ListNode dummy = new ListNode(0);
        dummy.next = head;

        ListNode pre = dummy;
        for (int i = 0; i < left - 1; i++) {
            pre = pre.next;
        }

        ListNode cur = pre.next;

        for (int i = 0; i < right - left; i++) {
            ListNode next = cur.next;
            cur.next = next.next;
            next.next = pre.next;
            pre.next = next;
        }

        return dummy.next;
    }

    public static void printList(ListNode head) {
        ListNode curr = head;
        System.out.print("[");
        while (curr != null) {
            System.out.print(curr.val);
            if (curr.next != null) {
                System.out.print(", ");
            }
            curr = curr.next;
        }
        System.out.println("]");
    }

    public static ListNode buildList(int[] vals) {
        ListNode dummy = new ListNode(0);
        ListNode cur = dummy;
        for (int val : vals) {
            cur.next = new ListNode(val);
            cur = cur.next;
        }
        return dummy.next;
    }

    public static void main(String[] args) {
        int[] input1 = {1, 2, 3, 4, 5};
        ListNode head1 = buildList(input1);
        int left1 = 2, right1 = 4;

        System.out.println("示例 1 输入: head = " + Arrays.toString(input1) + ", left = " + left1 + ", right = " + right1);
        ListNode res1 = reverseBetween(head1, left1, right1);
        System.out.print("示例 1 输出: ");
        printList(res1);
        System.out.println("-----------------------------");

        int[] input2 = {5};
        ListNode head2 = buildList(input2);
        int left2 = 1, right2 = 1;

        System.out.println("示例 2 输入: head = " + Arrays.toString(input2) + ", left = " + left2 + ", right = " + right2);
        ListNode res2 = reverseBetween(head2, left2, right2);
        System.out.print("示例 2 输出: ");
        printList(res2);
    }
}
```
