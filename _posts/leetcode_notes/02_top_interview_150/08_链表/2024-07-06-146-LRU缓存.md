## 题目

**描述：**

请你设计并实现一个满足 **LRU (最近最少使用)** 缓存约束的数据结构。

实现 `LRUCache` 类：

- `LRUCache(int capacity)` 以 **正整数** 作为容量 `capacity` 初始化 LRU 缓存
- `int get(int key)` 如果关键字 `key` 存在于缓存中，则返回关键字的值，否则返回 `-1`。
- `void put(int key, int value)` 如果关键字 `key` 已经存在，则变更其数据值 `value`；如果不存在，则向缓存中插入该组 `key-value`。如果插入操作导致关键字数量超过 `capacity`，则应该 **逐出** 最久未使用的关键字。

函数 `get` 和 `put` 都必须以 `O(1)` 的平均时间复杂度运行。

**示例：**

- **输入**
  `["LRUCache", "put", "put", "get", "put", "get", "put", "get", "get", "get"]`

  `[[2], [1, 1], [2, 2], [1], [3, 3], [2], [4, 4], [1], [3], [4]]`

- **输出**

  `[null, null, null, 1, null, -1, null, -1, 3, 4]`

- **解释**

  ```
  LRUCache lRUCache = new LRUCache(2);
  lRUCache.put(1, 1); // 缓存是 {1=1}
  lRUCache.put(2, 2); // 缓存是 {1=1, 2=2}
  lRUCache.get(1);    // 返回 1，{2=2, 1=1}
  lRUCache.put(3, 3); // 该操作会使得关键字 2 作废，缓存是 {1=1, 3=3}
  lRUCache.get(2);    // 返回 -1 (未找到)
  lRUCache.put(4, 4); // 该操作会使得关键字 1 作废，缓存是 {3=3, 4=4}
  lRUCache.get(1);    // 返回 -1 (未找到)
  lRUCache.get(3);    // 返回 3
  lRUCache.get(4);    // 返回 4
  ```

**提示:**

- `1 <= capacity <= 3000`
- `0 <= key <= 10000`
- `0 <= value <= 10^5`
- 最多调用 `2 * 10^5` 次 `get` 和 `put`

## 题解

### 先看这题真正难在哪

这题表面上是在实现一个缓存，但真正的难点不是“存数据”，而是要同时满足下面两件事：

1. 能通过 `key` 很快找到对应的值
2. 每次访问或写入之后，还要及时更新“最近使用”的顺序

更关键的是，题目明确要求：

- `get` 要做到平均 `O(1)`
- `put` 也要做到平均 `O(1)`

这就意味着，普通做法基本都不够用。

比如只用哈希表，虽然查找很快，但没法维护“谁最久没用”；  
只用链表，虽然能维护顺序，但查找一个 key 又会退化成 `O(n)`。

所以这题的核心不是某一个数据结构，而是：

**必须把“快速查找”和“快速调整顺序”这两件事拆开处理。**

### 为什么要同时用哈希表和双向链表

这题最标准的做法，就是：

- 用 **哈希表** 解决“按 key 快速定位”
- 用 **双向链表** 解决“按使用顺序快速移动和淘汰”

两者分别负责一件事。

#### 哈希表负责什么

哈希表里存的是：

```text
key -> 对应的链表节点
```

这样一来，只要给你一个 `key`，就能立刻找到这个节点，时间复杂度是 `O(1)`。

#### 双向链表负责什么

双向链表里真正保存的是缓存节点本身，也就是：

- `key`
- `value`

同时，它还维护这些节点的访问顺序。

我们约定：

- 越靠近链表头部，表示越最近使用
- 越靠近链表尾部，表示越久没使用

这样做之后：

- `get` 某个节点时，可以把它移动到头部
- `put` 一个新节点时，也可以把它插到头部
- 容量满了要淘汰时，直接删掉尾部前一个节点即可

这时候，“最近使用”和“最久未使用”就都很好处理了。

### 为什么必须是双向链表，单向链表不行

这一点也得先想清楚。

题目里有一个高频操作：  
**把某个已经存在的节点从原位置摘出来，再放到头部。**

如果是单向链表，你虽然知道这个节点是谁，但你不知道它前一个节点是谁，删除时还得重新遍历去找前驱，时间复杂度就不是 `O(1)` 了。

而双向链表因为节点自带：

- `prev`
- `next`

所以在已知某个节点引用的前提下，可以直接把它摘出来，整个过程是 `O(1)`。

这就是为什么这题不能只说“链表”，而要明确是**双向链表**。

### 先把整体结构想象出来

最常见的写法，是在双向链表两端各放一个哨兵节点：

- `head`：伪头节点
- `tail`：伪尾节点

真正的数据节点都放在它们中间。

结构大概是这样：

```text
head <-> 最近使用 <-> ... <-> 最久未使用 <-> tail
```

这样做的好处是，插入和删除时不用特判“当前是不是第一个节点”或者“当前是不是最后一个节点”，逻辑会统一很多。

所以这题里整个缓存的结构，其实可以理解成：

- 哈希表：负责定位
- 双向链表：负责排序
- 伪头伪尾：负责简化边界处理

### `get` 操作到底做了什么

很多人会觉得 `get` 很简单，不就是查一下然后返回值吗？

但在 LRU 里，`get` 不只是“读”，它还意味着：

**这个 key 刚刚被使用过了。**

既然它刚被使用过，就不能继续留在原来的位置，而应该被标记成“最近使用”。

所以 `get(key)` 的正确流程其实是：

1. 先去哈希表查这个 key 在不在
2. 不在，返回 `-1`
3. 在，就拿到对应节点
4. 把这个节点移动到链表头部
5. 再返回它的 value

也就是说，`get` 本质上是一个“查找 + 刷新优先级”的操作。

### `put` 操作为什么分两种情况

`put(key, value)` 也不能只理解成“写入”。

它有两种完全不同的场景。

#### 情况一：key 已经存在

如果 key 已经存在，那就不是新增，而是更新。

这时候要做两件事：

- 更新节点的 value
- 把这个节点移动到头部

因为一旦发生 `put`，说明这个 key 刚刚被使用过，也应该变成“最近使用”。

#### 情况二：key 不存在

如果 key 不存在，那就要新增一个节点。

这时候流程是：

1. 创建新节点
2. 加入哈希表
3. 插入链表头部
4. 检查是否超出容量
5. 如果超了，就删掉链表尾部那个最久未使用的节点
6. 同时把它从哈希表里也删掉

这里要特别注意：  
**淘汰时，链表和哈希表都要同步删除。**

删了链表不删哈希表，数据会脏；  
删了哈希表不删链表，结构会乱。

### 用示例走一遍

假设容量是 2。

```java
LRUCache cache = new LRUCache(2);
```

执行：

```java
put(1, 1)
```

缓存里只有一个节点，顺序是：

```text
1
```

再执行：

```java
put(2, 2)
```

因为新插入的是最近使用，所以顺序变成：

```text
2 -> 1
```

再执行：

```java
get(1)
```

查到了 `1`，返回 `1`。  
同时，因为 `1` 刚被访问过，所以它要移动到头部，顺序变成：

```text
1 -> 2
```

再执行：

```java
put(3, 3)
```

容量满了，新节点 `3` 插入头部后，需要淘汰最久未使用的那个，也就是尾部的 `2`。

所以结果变成：

```text
3 -> 1
```

再执行：

```java
get(2)
```

因为 `2` 已经被淘汰了，所以返回 `-1`。

整个过程说明，这题的关键并不是单纯“能不能查到值”，而是每次操作之后，链表顺序是不是始终对的。

### 手写版本为什么最值得掌握

如果是刷题或者面试，这题最推荐掌握的，还是“哈希表 + 双向链表”的手写版本。

因为它最能体现你是否真的理解了 LRU 的结构设计，而不是只会调用现成容器。

实现时，通常会把双向链表的几个基本操作单独拆成辅助函数：

- `addToHead(node)`：插到头部
- `removeNode(node)`：删除某个节点
- `moveToHead(node)`：移动到头部
- `removeTail()`：删除尾部节点

一旦这几个操作写对了，`get` 和 `put` 就会非常清楚。

### 代码

```java
class LRUCache {
    class Node {
        int key;
        int value;
        Node prev;
        Node next;

        Node(int key, int value) {
            this.key = key;
            this.value = value;
        }
    }

    private int capacity;
    private Map<Integer, Node> map;
    private Node head;
    private Node tail;

    public LRUCache(int capacity) {
        this.capacity = capacity;
        this.map = new HashMap<>();

        head = new Node(-1, -1);
        tail = new Node(-1, -1);
        head.next = tail;
        tail.prev = head;
    }

    public int get(int key) {
        Node node = map.get(key);
        if (node == null) {
            return -1;
        }

        moveToHead(node);
        return node.value;
    }

    public void put(int key, int value) {
        Node node = map.get(key);

        if (node != null) {
            node.value = value;
            moveToHead(node);
        } else {
            Node newNode = new Node(key, value);
            map.put(key, newNode);
            addToHead(newNode);

            if (map.size() > capacity) {
                Node removed = removeTail();
                map.remove(removed.key);
            }
        }
    }

    private void addToHead(Node node) {
        node.prev = head;
        node.next = head.next;
        head.next.prev = node;
        head.next = node;
    }

    private void removeNode(Node node) {
        node.prev.next = node.next;
        node.next.prev = node.prev;
    }

    private void moveToHead(Node node) {
        removeNode(node);
        addToHead(node);
    }

    private Node removeTail() {
        Node node = tail.prev;
        removeNode(node);
        return node;
    }
}
```

### 这段代码怎么理解

这段代码里，真正需要抓住的是两层关系。

第一层是哈希表：

```java
Map<Integer, Node> map
```

它负责用 key 快速拿到节点。

第二层是双向链表：

```java
head <-> ... <-> tail
```

它负责记录使用顺序。

#### `addToHead(node)`

把节点插到头部后面，也就是标记为最近使用。

#### `removeNode(node)`

把一个节点从原位置摘出来。

#### `moveToHead(node)`

先删再插，本质上就是“刷新最近使用状态”。

#### `removeTail()`

删掉尾部前一个节点，也就是当前最久未使用的节点。

所以整个 LRU 其实没有什么特别玄的地方，本质上就是：

- 哈希表找到节点
- 双向链表调整顺序

### `LinkedHashMap` 为什么也能做

如果是 Java 工程开发，这题其实还有一个很实用的解法，就是直接用 `LinkedHashMap`。

原因很简单：  
`LinkedHashMap` 底层本来就是“哈希表 + 双向链表”。

而且它支持按访问顺序维护元素，只要构造时把 `accessOrder` 设为 `true`，再重写 `removeEldestEntry`，就能直接做出 LRU 的效果。

代码如下：

```java
class LRUCache extends LinkedHashMap<Integer, Integer> {
    private int capacity;

    public LRUCache(int capacity) {
        super(capacity, 0.75F, true);
        this.capacity = capacity;
    }

    public int get(int key) {
        return super.getOrDefault(key, -1);
    }

    public void put(int key, int value) {
        super.put(key, value);
    }

    @Override
    protected boolean removeEldestEntry(Map.Entry<Integer, Integer> eldest) {
        return size() > capacity;
    }
}
```

这个版本在工程里很实用，但如果是面试，通常还是更建议先掌握手写版。因为手写版才能说明你真正理解了 LRU 为什么能做到 `O(1)`。

### 复杂度分析

无论是手写版，还是基于 `LinkedHashMap` 的版本，核心目标都是一样的：

- `get` 平均时间复杂度 `O(1)`
- `put` 平均时间复杂度 `O(1)`

哈希表负责常数级查找，双向链表负责常数级插入、删除和移动。

空间复杂度是 $O(capacity)$，因为最多只会存 `capacity` 个缓存节点。

### 这题真正值得记住的地方

这题真正值得记住的，不只是“LRU 要用哈希表加双向链表”，而是这种设计思路：

**当一个数据结构既要支持按 key 快速访问，又要支持按顺序快速调整时，往往需要把“定位”和“顺序维护”交给两个不同的结构分别处理。**

LRU 就是一个非常典型的组合题：

- 哈希表负责查
- 双向链表负责排

理解了这个分工，后面再看 LFU、缓存淘汰策略，或者很多“查找 + 顺序维护”类结构时，思路都会顺很多。

### 完整代码

```java
import java.util.HashMap;
import java.util.Map;

public class Main {

    static class LRUCache {
        class Node {
            int key;
            int value;
            Node prev;
            Node next;

            Node() {}

            Node(int key, int value) {
                this.key = key;
                this.value = value;
            }
        }

        private Map<Integer, Node> cache = new HashMap<>();
        private int capacity;
        private int size;
        private Node head;
        private Node tail;

        public LRUCache(int capacity) {
            this.capacity = capacity;
            this.size = 0;

            head = new Node();
            tail = new Node();
            head.next = tail;
            tail.prev = head;
        }

        public int get(int key) {
            Node node = cache.get(key);
            if (node == null) {
                return -1;
            }

            moveToHead(node);
            return node.value;
        }

        public void put(int key, int value) {
            Node node = cache.get(key);

            if (node == null) {
                Node newNode = new Node(key, value);
                cache.put(key, newNode);
                addToHead(newNode);
                size++;

                if (size > capacity) {
                    Node removed = removeTail();
                    cache.remove(removed.key);
                    size--;
                }
            } else {
                node.value = value;
                moveToHead(node);
            }
        }

        private void addToHead(Node node) {
            node.prev = head;
            node.next = head.next;
            head.next.prev = node;
            head.next = node;
        }

        private void removeNode(Node node) {
            node.prev.next = node.next;
            node.next.prev = node.prev;
        }

        private void moveToHead(Node node) {
            removeNode(node);
            addToHead(node);
        }

        private Node removeTail() {
            Node res = tail.prev;
            removeNode(res);
            return res;
        }
    }

    public static void main(String[] args) {
        System.out.println("开始测试 LRUCache:");

        LRUCache lRUCache = new LRUCache(2);

        System.out.println("put(1, 1)");
        lRUCache.put(1, 1);

        System.out.println("put(2, 2)");
        lRUCache.put(2, 2);

        System.out.println("get(1) -> " + lRUCache.get(1));

        System.out.println("put(3, 3)");
        lRUCache.put(3, 3);

        System.out.println("get(2) -> " + lRUCache.get(2));

        System.out.println("put(4, 4)");
        lRUCache.put(4, 4);

        System.out.println("get(1) -> " + lRUCache.get(1));
        System.out.println("get(3) -> " + lRUCache.get(3));
        System.out.println("get(4) -> " + lRUCache.get(4));

        System.out.println("-----------------------------");
        System.out.println("测试完成");
    }
}
```
