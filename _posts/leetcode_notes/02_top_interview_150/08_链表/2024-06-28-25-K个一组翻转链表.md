## 题目

**描述：**

给你链表的头节点 `head`，每 `k` 个节点一组进行翻转，请你返回修改后的链表。

- `k` 是一个正整数，它的值小于或等于链表的长度。
- 如果节点总数不是 `k` 的整数倍，那么请将最后剩余的节点保持原有顺序。

你不能只是单纯的改变节点内部的值，而是需要实际进行节点交换。

**示例 1:**

![示例1图片](https://assets.leetcode.com/uploads/2020/10/03/reverse_ex1.jpg)

- **输入:** `head = [1,2,3,4,5]`, `k = 2`
- **输出:** `[2,1,4,3,5]`

**示例 2:**

![示例2图片](https://assets.leetcode.com/uploads/2020/10/03/reverse_ex2.jpg)

- **输入:** `head = [1,2,3,4,5]`, `k = 3`
- **输出:** `[3,2,1,4,5]`

**提示:**

- 链表中的节点数目为 `n`
- `1 <= k <= n <= 5000`
- `0 <= Node.val <= 1000`

**进阶：** 你可以设计一个只用 `O(1)` 额外内存空间的算法解决此问题吗？

## 题解

### 先看这题比“局部反转”又多了什么

如果是反转一整条链表，做法比较固定；  
如果是反转指定区间，那重点是找到区间前驱和区间尾部；  
而这题再往前走了一步，它要求的是：

**每 `k` 个节点看成一组，整组反转；如果最后剩下的不够 `k` 个，就保持不动。**

所以这题真正麻烦的地方，不是“会不会反转链表”，而是：

- 怎么判断当前这一组够不够 `k` 个
- 怎么把这一组单独反转
- 反转之后怎么和前后重新接起来
- 然后继续处理下一组

把这四件事理顺，这题就不难了。

### 整体思路其实很朴素

这题最自然的想法就是：  
**一组一组处理。**

每次先看当前这一段能不能凑够 `k` 个节点：

- 如果凑不够，说明这是最后剩下的部分，直接保留原样
- 如果凑够了，就把这一组反转，然后接回去

也就是说，整道题本质上不是在“整条链表上做一种神秘操作”，而是在不断重复同一个动作：

1. 找到一组长度为 `k` 的子链表
2. 反转它
3. 接回原链表
4. 继续处理下一组

这么看，题目结构其实很清楚。

### 为什么不能一上来就直接反转

这题有个很关键的限制：  
**只有凑够 `k` 个节点，才能反转。**

比如：

```text
1 -> 2 -> 3 -> 4 -> 5
k = 3
```

前 3 个节点可以反转，得到：

```text
3 -> 2 -> 1
```

但后面只剩下 `4 -> 5`，不够 3 个，所以必须保持原样。

也就是说，在动手反转之前，必须先确认当前这一组到底有没有 `k` 个节点。  
这一点如果不先判断，很容易把最后那一小段也误反转了。

### 每一组反转时，真正要找的是哪几个位置

每次处理一组时，最关键的其实不是组内节点本身，而是它和外部的连接关系。

假设当前链表是：

```text
dummy -> a -> b -> c -> d -> e -> ...
```

如果现在要反转的是 `b -> c -> d` 这一组，那么我们最关心的是四个位置：

- `pre`：这一组前面的节点，也就是 `a`
- `start`：这一组的头，也就是 `b`
- `end`：这一组的尾，也就是 `d`
- `nextGroup`：下一组的头，也就是 `e`

反转之前是：

```text
a -> b -> c -> d -> e
```

反转之后应该变成：

```text
a -> d -> c -> b -> e
```

这里要特别注意：

- 原来的 `start`，反转后会变成这一组的尾巴
- 原来的 `end`，反转后会变成这一组的新头

所以整道题真正考的，还是链表题老问题：

**前驱是谁，尾巴是谁，断开在哪里，接回到哪里。**

### 递归写法为什么好理解

递归法很适合先建立整体感觉。

它的思路是：

- 先判断当前这一段够不够 `k` 个节点
- 如果不够，直接返回，不反转
- 如果够，就先把后面的部分递归处理好
- 然后再把当前这 `k` 个节点反转
- 当前组反转后的尾巴，再接上后面已经处理好的结果

这种思路很像“先把后面的事情交给递归，再回过头处理自己这一组”。

代码如下：

```java
public ListNode reverseKGroup(ListNode head, int k) {
    ListNode cursor = head;
    int count = 0;

    while (cursor != null && count < k) {
        cursor = cursor.next;
        count++;
    }

    if (count == k) {
        ListNode reversedRest = reverseKGroup(cursor, k);

        while (count > 0) {
            ListNode next = head.next;
            head.next = reversedRest;
            reversedRest = head;
            head = next;
            count--;
        }
        return reversedRest;
    }

    return head;
}
```

这个写法的优点是思路清楚。  
但它会占用递归栈空间，所以不是题目进阶要求里最理想的版本。

### 更推荐掌握的，是迭代写法

如果想做到 `O(1)` 额外空间，还是要用迭代。

迭代法的核心也不复杂，就是每次做下面几件事：

1. 用 `end` 往后走 `k` 步，确认这一组够不够 `k` 个
2. 如果不够，直接结束
3. 如果够，就把这一组临时断开
4. 调用标准反转链表逻辑，把这一组反转
5. 把反转后的这一组重新接回去
6. 更新指针，继续下一组

这个思路其实很稳，因为它把大问题拆成了很多个“小范围反转”。

### 用示例走一遍会更清楚

以示例 1 为例：

```text
head = [1,2,3,4,5], k = 2
```

先处理第一组 `1 -> 2`。

- 这一组够 2 个，可以反转
- 反转后变成 `2 -> 1`
- 再接上后面的 `3 -> 4 -> 5`

此时链表变成：

```text
2 -> 1 -> 3 -> 4 -> 5
```

接着处理第二组 `3 -> 4`。

- 这一组也够 2 个
- 反转后变成 `4 -> 3`

链表变成：

```text
2 -> 1 -> 4 -> 3 -> 5
```

最后只剩下 `5`。

- 不够 2 个
- 保持原样

最终结果就是：

```text
[2,1,4,3,5]
```

这个过程说明，这题并不是在做一次连续的大反转，而是在不断对固定长度的小段做局部处理。

### 迭代代码

这题更推荐掌握下面这个版本：

```java
public ListNode reverseKGroup(ListNode head, int k) {
    ListNode dummy = new ListNode(0);
    dummy.next = head;

    ListNode pre = dummy;
    ListNode end = dummy;

    while (end.next != null) {
        for (int i = 0; i < k && end != null; i++) {
            end = end.next;
        }

        if (end == null) {
            break;
        }

        ListNode start = pre.next;
        ListNode nextGroup = end.next;
        end.next = null;

        pre.next = reverse(start);
        start.next = nextGroup;

        pre = start;
        end = pre;
    }

    return dummy.next;
}
```

辅助的反转函数就是普通的链表反转：

```java
private ListNode reverse(ListNode head) {
    ListNode pre = null;
    ListNode curr = head;

    while (curr != null) {
        ListNode next = curr.next;
        curr.next = pre;
        pre = curr;
        curr = next;
    }

    return pre;
}
```

### 这段代码该怎么理解

可以把 `pre` 理解成“当前这组前面的那个节点”，  
把 `end` 理解成“用来找当前这组结尾的位置”。

每轮循环里最关键的几步是：

#### 第一步：确认够不够 `k` 个

```java
for (int i = 0; i < k && end != null; i++) {
    end = end.next;
}
```

这一步不是为了反转，而是为了先判断当前组是否合法。  
如果 `end` 走着走着变成了 `null`，说明剩余节点不足 `k` 个，直接结束。

#### 第二步：记住边界

```java
ListNode start = pre.next;
ListNode nextGroup = end.next;
```

这里：

- `start` 是当前组的头
- `nextGroup` 是下一组的头

后面反转和接回都要靠这两个点。

#### 第三步：断开当前组

```java
end.next = null;
```

这样当前这 `k` 个节点就变成了一条独立链表，可以直接拿去做标准反转。

#### 第四步：反转并接回去

```java
pre.next = reverse(start);
start.next = nextGroup;
```

这里特别关键。

- `reverse(start)` 返回的是反转后的新头
- 原来的 `start` 反转后会变成这一组的尾巴
- 所以要让 `start.next` 指回下一组的开头

#### 第五步：更新指针，准备下一组

```java
pre = start;
end = pre;
```

为什么 `pre = start`？

因为 `start` 现在已经是这一组反转后的尾节点了，它正好就是下一组前面的那个节点。

这一步一旦看懂，整段代码就完全顺了。

### 为什么一定要有 dummy 节点

这题里 `dummy` 很有必要。

因为第一组也可能会被反转，反转之后整条链表的头节点会发生变化。  
如果没有 `dummy`，你就得单独处理“第一组反转后谁是新头节点”这个问题。

加上 `dummy` 之后，所有分组都可以统一处理：

- `pre` 永远指向当前组前一个节点
- 最后统一返回 `dummy.next`

这会让代码稳定很多。

### 复杂度分析

时间复杂度是 $O(n)$。  
虽然看起来有“找一组 + 反转一组”两个动作，但每个节点被访问的次数都是常数级，总体仍然是线性时间。

空间复杂度是 $O(1)$。  
这里只用了若干个指针变量，没有额外开和链表长度相关的空间。

如果用递归写法，时间复杂度同样是 $O(n)$，但空间复杂度会因为递归栈变成 $O(n/k)$。

### 这题真正值得记住的地方

这题真正值得记住的，不只是“K 个一组翻转链表”这个结论，而是这种处理方式：

**当链表题要求按固定块做局部操作时，通常要先把每一块的边界找出来，再把这块当成一个独立问题处理。**

这一题里，每一组的处理流程其实很标准：

- 先确认长度是否足够
- 再定位前驱、头、尾、后继
- 然后断开、反转、接回

很多看起来复杂的链表题，最后都是在反复做这套边界管理。  
只要边界清楚，代码就不会乱。

### 完整代码

```java
import java.util.*;

// 链表节点定义
class ListNode {
    int val;
    ListNode next;

    ListNode() {}

    ListNode(int val) {
        this.val = val;
    }

    ListNode(int val, ListNode next) {
        this.val = val;
        this.next = next;
    }
}

public class Main {

    public static ListNode reverseKGroup(ListNode head, int k) {
        if (head == null || k == 1) {
            return head;
        }

        ListNode dummy = new ListNode(0);
        dummy.next = head;

        ListNode pre = dummy;
        ListNode end = dummy;

        while (end.next != null) {
            for (int i = 0; i < k && end != null; i++) {
                end = end.next;
            }

            if (end == null) {
                break;
            }

            ListNode start = pre.next;
            ListNode nextGroup = end.next;

            end.next = null;

            pre.next = reverse(start);
            start.next = nextGroup;

            pre = start;
            end = pre;
        }

        return dummy.next;
    }

    private static ListNode reverse(ListNode head) {
        ListNode pre = null;
        ListNode curr = head;

        while (curr != null) {
            ListNode next = curr.next;
            curr.next = pre;
            pre = curr;
            curr = next;
        }

        return pre;
    }

    public static void printList(ListNode head) {
        System.out.print("[");
        ListNode curr = head;
        while (curr != null) {
            System.out.print(curr.val);
            if (curr.next != null) {
                System.out.print(", ");
            }
            curr = curr.next;
        }
        System.out.println("]");
    }

    public static ListNode buildList(int[] vals) {
        ListNode dummy = new ListNode(0);
        ListNode curr = dummy;
        for (int val : vals) {
            curr.next = new ListNode(val);
            curr = curr.next;
        }
        return dummy.next;
    }

    public static void main(String[] args) {
        int[] vals1 = {1, 2, 3, 4, 5};
        int k1 = 2;
        ListNode head1 = buildList(vals1);

        System.out.println("示例 1 输入: head = " + Arrays.toString(vals1) + ", k = " + k1);
        ListNode res1 = reverseKGroup(head1, k1);
        System.out.print("示例 1 输出: ");
        printList(res1);
        System.out.println("-----------------------------");

        int[] vals2 = {1, 2, 3, 4, 5};
        int k2 = 3;
        ListNode head2 = buildList(vals2);

        System.out.println("示例 2 输入: head = " + Arrays.toString(vals2) + ", k = " + k2);
        ListNode res2 = reverseKGroup(head2, k2);
        System.out.print("示例 2 输出: ");
        printList(res2);
    }
}
```
