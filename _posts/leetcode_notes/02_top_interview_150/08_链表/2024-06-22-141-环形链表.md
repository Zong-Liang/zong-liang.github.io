## 题目

**描述：**

给你一个链表的头节点 `head`，判断链表中是否有环。

如果链表中存在某个节点，可以通过连续跟踪 `next` 指针再次到达，则链表中存在环。为了表示给定链表中的环，评测系统内部使用整数 `pos` 来表示链表尾连接到链表中的位置（索引从 0 开始）。**注意：`pos` 不作为参数进行传递**。仅仅是为了标识链表的实际情况。

如果链表中存在环，则返回 `true`。否则，返回 `false`。

**示例 1:**

![示例1图片](https://assets.leetcode-cn.com/aliyun-lc-upload/uploads/2018/12/07/circularlinkedlist.png)

- **输入:** `head = [3,2,0,-4]`, `pos = 1`
- **输出:** `true`
- **解释:** 链表中有一个环，其尾部连接到第二个节点。

**示例 2:**

![示例2图片](https://assets.leetcode-cn.com/aliyun-lc-upload/uploads/2018/12/07/circularlinkedlist_test2.png)

- **输入:** `head = [1,2]`, `pos = 0`
- **输出:** `true`
- **解释:** 链表中有一个环，其尾部连接到第一个节点。

**示例 3:**

![示例3图片](https://assets.leetcode-cn.com/aliyun-lc-upload/uploads/2018/12/07/circularlinkedlist_test3.png)

- **输入:** `head = [1]`, `pos = -1`
- **输出:** `false`
- **解释:** 链表中没有环。

**提示:**

- 链表中节点的数目范围是 `[0, 10^4]`
- `-10^5 <= Node.val <= 10^5`
- `pos` 为 `-1` 或者链表中的一个 **有效索引**。

**进阶：** 你能用 `O(1)`（即，常量）内存解决此问题吗？

## 题解

### 先抓住这题的关键

这题问的是：如何判断一个链表里有没有环。

链表和数组不一样，数组可以通过下标反复访问同一个位置，但链表只能顺着 `next` 一直往后走。所以判断有没有环，本质上就是看：

**在不断往后走的过程中，会不会再次回到之前到过的节点。**

只要能把这个问题想清楚，后面的两种解法就都顺了。

### 最直接的想法：记录访问过的节点

既然环的特点是“会再次回到某个节点”，那最容易想到的办法，就是把遍历过程中见过的节点都记下来。

具体做法很直接：

- 从头节点开始遍历链表
- 每到一个节点，就先看它是不是已经出现过
- 如果出现过，说明绕回来了，链表有环
- 如果没出现过，就把它加入集合，继续往后走
- 如果最后走到了 `null`，说明链表没有环

这个思路很好理解，因为它几乎就是在按题意做判断。

### 哈希表解法

这里可以用 `HashSet` 来保存已经访问过的节点。

```java
public boolean hasCycle(ListNode head) {
    Set<ListNode> seen = new HashSet<>();
    while (head != null) {
        if (!seen.add(head)) {
            return true;
        }
        head = head.next;
    }
    return false;
}
```

这段代码里，`seen.add(head)` 很关键。

- 如果当前节点之前没出现过，`add` 会返回 `true`
- 如果当前节点之前已经加过了，`add` 会返回 `false`

一旦返回 `false`，就说明我们又走回了同一个节点，也就说明链表里有环。

这个解法的优点是直观，几乎不会绕；缺点也很明显：需要额外开集合存节点。

时间复杂度是 $O(N)$，因为最多访问每个节点一次。  
空间复杂度也是 $O(N)$，因为最坏情况下要把所有节点都存进哈希表。

### 这题真正想考的，是不用额外空间怎么做

题目进阶要求里专门问了能不能用 `O(1)` 额外空间解决，这就说明哈希表不是它最想考的解法。

那不用集合，还怎么判断“是否回到之前的节点”？

这里就要换一个角度了。

我们不一定非得把走过的节点记下来，也可以通过“移动速度不同”来判断链表里是否有环。

### 快慢指针为什么能判断有环

我们定义两个指针：

- 慢指针 `slow` 每次走一步
- 快指针 `fast` 每次走两步

如果链表没有环，那么快指针只会越来越靠近链表尾部，最后一定会走到 `null`。

但如果链表里有环，情况就不一样了。

可以把它想成一条环形跑道：

- 慢指针像一个跑得慢的人
- 快指针像一个跑得快的人

只要两个人都进入了环，快指针每轮都会比慢指针多走一步。既然是在一个封闭的圈里跑，快指针迟早会从后面追上慢指针。只要两者相遇，就说明链表里一定有环。

这就是快慢指针能成立的根本原因。

### 用一个过程来理解

假设链表有环，结构大致是这样：

```text
1 -> 2 -> 3 -> 4 -> 5
          ^         |
          |_________|
```

开始时，`slow` 和 `fast` 都从头节点出发。

- 第 1 轮：`slow` 走 1 步，`fast` 走 2 步
- 第 2 轮：`slow` 再走 1 步，`fast` 再走 2 步
- 进入环之后，`fast` 每轮都会比 `slow` 多前进 1 个节点

也就是说，它们之间的距离会不断缩小。只要链表中有环，这个“追及”过程就一定会发生，最终两者会在环内相遇。

反过来，如果链表没有环，快指针根本不可能绕回来，它只会先走到末尾。

所以这题真正要看的，不是“某个节点值是不是重复”，而是**两个指针会不会相遇**。

### 快慢指针怎么写

写法上，最稳妥的一种方式是让 `slow` 和 `fast` 一开始都指向 `head`。

然后在循环里判断：

- 只要 `fast != null && fast.next != null`，就说明快指针还能继续往前跳两步
- 每轮让 `slow` 走一步，`fast` 走两步
- 如果某一轮之后 `slow == fast`，说明相遇，有环
- 如果循环结束，说明 `fast` 走到了链表末尾，没有环

代码如下：

```java
public boolean hasCycle(ListNode head) {
    ListNode slow = head;
    ListNode fast = head;

    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;

        if (slow == fast) {
            return true;
        }
    }

    return false;
}
```

这里判断相遇时，用的是 `slow == fast`，而不是比较节点值。因为题目要判断的是“是不是同一个节点”，不是“值是否相同”。

### 为什么这套写法更合适

有些写法会让 `slow = head`，`fast = head.next`，然后先判断 `slow != fast` 再进入循环。这样也能做出来，但理解和边界处理会稍微绕一点。

两者都从 `head` 出发的写法更自然：

- 初始化统一
- 循环条件更清晰
- 相遇判断也更直观

对博客读者来说，这种写法通常更容易一下子看明白。

### 代码

```java
public boolean hasCycle(ListNode head) {
    ListNode slow = head;
    ListNode fast = head;

    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;

        if (slow == fast) {
            return true;
        }
    }

    return false;
}
```

时间复杂度是 $O(N)$。  
如果没有环，快指针最多走完整条链表；如果有环，快慢指针也会在有限步数内相遇。  
空间复杂度是 $O(1)$，因为整个过程只用了两个指针变量。

### 这题真正值得记住的地方

这题真正值得记住的，不只是“快慢指针判环”这个结论，而是这种判断思路：

**当一个结构中存在“循环回到之前位置”的可能时，可以尝试用不同速度的指针去制造相遇。**

哈希表解法更像是“我把来过的地方都记下来”；  
快慢指针更像是“我不记路径，但我通过运动规律判断这里是不是个圈”。

前者更直观，后者更巧，也更符合题目进阶要求。

### 完整代码

```java
import java.util.*;

// 链表节点定义
class ListNode {
    int val;
    ListNode next;

    ListNode(int x) {
        val = x;
        next = null;
    }
}

public class Main {

    public static boolean hasCycle(ListNode head) {
        ListNode slow = head;
        ListNode fast = head;

        while (fast != null && fast.next != null) {
            slow = slow.next;
            fast = fast.next.next;

            if (slow == fast) {
                return true;
            }
        }

        return false;
    }

    public static void main(String[] args) {
        // 示例 1：3 -> 2 -> 0 -> -4 -> 回到节点 2
        ListNode n1 = new ListNode(3);
        ListNode n2 = new ListNode(2);
        ListNode n3 = new ListNode(0);
        ListNode n4 = new ListNode(-4);

        n1.next = n2;
        n2.next = n3;
        n3.next = n4;
        n4.next = n2;

        System.out.println("示例 1 输入: head = [3,2,0,-4], pos = 1");
        System.out.println("示例 1 输出: " + hasCycle(n1));
        System.out.println("-----------------------------");

        // 示例 2：1 -> 2 -> 回到节点 1
        ListNode m1 = new ListNode(1);
        ListNode m2 = new ListNode(2);

        m1.next = m2;
        m2.next = m1;

        System.out.println("示例 2 输入: head = [1,2], pos = 0");
        System.out.println("示例 2 输出: " + hasCycle(m1));
        System.out.println("-----------------------------");

        // 示例 3：1 -> null
        ListNode k1 = new ListNode(1);

        System.out.println("示例 3 输入: head = [1], pos = -1");
        System.out.println("示例 3 输出: " + hasCycle(k1));
    }
}
```
