## 题目

**描述：**

给你一个链表的头节点 `head`，判断链表中是否有环。

如果链表中存在某个节点，可以通过连续跟踪 `next` 指针再次到达，则链表中存在环。为了表示给定链表中的环，评测系统内部使用整数 `pos` 来表示链表尾连接到链表中的位置（索引从 0 开始）。**注意：`pos` 不作为参数进行传递**。仅仅是为了标识链表的实际情况。

如果链表中存在环，则返回 `true`。否则，返回 `false`。

**示例 1:**

![示例1图片](https://assets.leetcode-cn.com/aliyun-lc-upload/uploads/2018/12/07/circularlinkedlist.png)

- **输入:** `head = [3,2,0,-4]`, `pos = 1`
- **输出:** `true`
- **解释:** 链表中有一个环，其尾部连接到第二个节点。

**示例 2:**

![示例2图片](https://assets.leetcode-cn.com/aliyun-lc-upload/uploads/2018/12/07/circularlinkedlist_test2.png)

- **输入:** `head = [1,2]`, `pos = 0`
- **输出:** `true`
- **解释:** 链表中有一个环，其尾部连接到第一个节点。

**示例 3:**

![示例3图片](https://assets.leetcode-cn.com/aliyun-lc-upload/uploads/2018/12/07/circularlinkedlist_test3.png)

- **输入:** `head = [1]`, `pos = -1`
- **输出:** `false`
- **解释:** 链表中没有环。

**提示:**

- 链表中节点的数目范围是 `[0, 10^4]`
- `-10^5 <= Node.val <= 10^5`
- `pos` 为 `-1` 或者链表中的一个 **有效索引**。

**进阶：** 你能用 `O(1)`（即，常量）内存解决此问题吗？

## 题解

这题最容易写成“背结论”，但真正该讲清楚的是为什么能判断出环。直观做法是用 `HashSet` 记录访问过的节点，遍历时只要再次遇到同一个节点，就说明走回来了；这个方法很好理解，也很稳，不过要额外空间。题目的进阶要求是 `O(1)` 额外空间，所以更推荐快慢指针。

快慢指针的关键不在“两个指针”本身，而在“相对速度”。让 `slow` 每次走一步、`fast` 每次走两步：如果没有环，`fast` 最终会先到 `null`；如果有环，两者进入环后，`fast` 每轮都会比 `slow` 多走一步，距离会不断缩小，最后一定在环里相遇。这个推理比记代码更重要，因为它解释了为什么 `slow == fast` 可以直接返回 `true`。

实现时只要把边界守好就行：循环条件写成 `fast != null && fast.next != null`，每轮先移动再比较，循环正常退出就返回 `false`。这里比较的是节点引用而不是节点值，判断的是“是不是同一个节点”，不是“值是否重复”。

代码如下：

```java
public boolean hasCycle(ListNode head) {
    ListNode slow = head;
    ListNode fast = head;
    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;
        if (slow == fast) {
            return true;
        }
    }
    return false;
}
```

快慢指针方案的时间复杂度是 $O(N)$，空间复杂度是 $O(1)$。如果用哈希表记录节点，时间同样是 $O(N)$，但空间会变成 $O(N)$。

以后遇到“结构里可能回到旧位置”的题，不一定都要靠额外存储去记轨迹；很多时候可以用运动规律来判断，快慢指针就是这种思路最经典的例子。

完整代码如下：

```java
import java.util.*;
// 链表节点定义
class ListNode {
    int val;
    ListNode next;
    ListNode(int x) {
        val = x;
        next = null;
    }
}
public class Main {
    public static boolean hasCycle(ListNode head) {
        ListNode slow = head;
        ListNode fast = head;
        while (fast != null && fast.next != null) {
            slow = slow.next;
            fast = fast.next.next;
            if (slow == fast) {
                return true;
            }
        }
        return false;
    }
    public static void main(String[] args) {
        // 示例 1：3 -> 2 -> 0 -> -4 -> 回到节点 2
        ListNode n1 = new ListNode(3);
        ListNode n2 = new ListNode(2);
        ListNode n3 = new ListNode(0);
        ListNode n4 = new ListNode(-4);
        n1.next = n2;
        n2.next = n3;
        n3.next = n4;
        n4.next = n2;
        System.out.println("示例 1 输入: head = [3,2,0,-4], pos = 1");
        System.out.println("示例 1 输出: " + hasCycle(n1));
        System.out.println("-----------------------------");
        // 示例 2：1 -> 2 -> 回到节点 1
        ListNode m1 = new ListNode(1);
        ListNode m2 = new ListNode(2);
        m1.next = m2;
        m2.next = m1;
        System.out.println("示例 2 输入: head = [1,2], pos = 0");
        System.out.println("示例 2 输出: " + hasCycle(m1));
        System.out.println("-----------------------------");
        // 示例 3：1 -> null
        ListNode k1 = new ListNode(1);
        System.out.println("示例 3 输入: head = [1], pos = -1");
        System.out.println("示例 3 输出: " + hasCycle(k1));
    }
}
```
