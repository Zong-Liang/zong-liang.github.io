## 题目

**描述：**

请你设计一个数据结构，支持 **添加新单词** 和 **查找字符串是否与任何先前添加的字符串匹配**。

实现词典类 `WordDictionary`：

- `WordDictionary()` 初始化词典对象
- `void addWord(word)` 将 `word` 添加到数据结构中，之后可以对它进行匹配
- `bool search(word)` 如果数据结构中存在字符串与 `word` 匹配，则返回 `true`；否则，返回 `false`。`word` 中可能包含一些 `'.'`，每个 `.` 都可以表示任何一个字母。

**示例:**

- **输入**

  `["WordDictionary","addWord","addWord","addWord","search","search","search","search"]`

  `[[],["bad"],["dad"],["mad"],["pad"],["bad"],[".ad"],["b.."]]`

- **输出**

  `[null,null,null,null,false,true,true,true]`

- **解释**

  ```
  WordDictionary wordDictionary = new WordDictionary();
  wordDictionary.addWord("bad");
  wordDictionary.addWord("dad");
  wordDictionary.addWord("mad");
  wordDictionary.search("pad"); // 返回 False
  wordDictionary.search("bad"); // 返回 True
  wordDictionary.search(".ad"); // 返回 True
  wordDictionary.search("b.."); // 返回 True
  ```

**提示:**

- `1 <= word.length <= 25`
- `addWord` 中的 `word` 由小写英文字母组成
- `search` 中的 `word` 由 `'.'` 或小写英文字母组成
- 最多调用 `10^4` 次 `addWord` 和 `search`

## 题解

### 这题和普通 Trie 的区别，其实就差在一个 `.`

如果没有 `.`，这题本质上就是上一题的普通前缀树：

- `addWord` 把单词插进去
- `search` 顺着字符路径往下走
- 走完后看最后一个节点是不是单词结尾

但这题麻烦的地方在于，`search` 里允许出现 `.`，而 `.` 可以表示任意一个字母。

这意味着什么？

意味着查询时，有时候你不能只沿着一条确定的路径往下走。  
一旦碰到 `.`，当前节点下面的很多分支都有可能匹配成功。

所以这题真正的关键不是 Trie 本身，而是：

**普通字符时精确匹配，遇到 `.` 时要尝试所有可能分支。**

### 为什么这题还是要用 Trie

虽然查询变复杂了，但底层结构仍然很适合用 Trie。

因为 Trie 本来就是按字符一层一层往下存路径的。  
当你查找 `"bad"` 时，可以走：

```text
b -> a -> d
```

而当你查找 `".ad"` 时，意思其实是：

- 第 1 个字符不确定
- 但第 2 个字符必须是 `a`
- 第 3 个字符必须是 `d`

如果底层是 Trie，那遇到第一个 `.`` 时，我们只需要从当前节点往下试所有存在的子节点，然后继续匹配后面的 `"ad"` 就行。

也就是说，Trie 负责提供“按字符展开的结构”，而 `.` 带来的“不确定性”，交给搜索过程去处理。

### `addWord` 为什么和普通 Trie 完全一样

因为 `addWord` 不涉及通配符，插入时仍然是在做最基础的事情：

- 从根节点开始
- 依次处理单词的每个字符
- 如果路径不存在，就创建新节点
- 最后把终点节点标记成 `isEnd = true`

这一部分和普通前缀树没有任何区别。

所以这题真正有变化的地方，只在 `search`。

### 为什么 `search` 不能再像普通 Trie 那样一路走到底

普通 Trie 查找时，每个字符都是确定的。  
比如查 `"bad"`，那每一步都只有唯一选择：

- `b`
- `a`
- `d`

如果其中某一步对应子节点不存在，就可以直接返回 `false`。

但现在有了 `.`，情况就不一样了。

比如查 `".ad"` 时，第一个字符可以是：

- `b`
- `d`
- `m`
- 甚至别的任何字母

只要其中有一条路径最后能走成一个完整单词，就应该返回 `true`。

所以你不能再把查找写成“只有一条路的线性过程”，而要改成：

**遇到普通字符就沿唯一一条路走，遇到 `.` 就把所有可能的路都试一遍。**

这也是为什么这里很适合递归。

### 为什么这里自然会想到 DFS

因为 `.` 的本质就是“当前位置不确定，要尝试多个分支”。

而这种“从某个节点出发，依次尝试所有可能路径，只要有一条成功就返回 true”的问题，最典型的写法就是 DFS。

具体来说：

- 如果当前字符是普通字母，就继续往唯一对应的子节点递归
- 如果当前字符是 `.`，就枚举当前节点的所有非空子节点，挨个递归尝试
- 只要有一个分支返回 `true`，整体就返回 `true`
- 如果所有分支都失败，才返回 `false`

这就是这题的核心搜索逻辑。

### 递归函数到底在做什么

可以把递归函数理解成这样：

```text
dfs(word, index, node)
```

它表示：

**从 Trie 的 `node` 这个位置出发，能不能匹配 `word` 从 `index` 开始的剩余部分。**

这样定义之后，整个逻辑会非常清楚。

#### 终止条件

如果：

```java
index == word.length()
```

说明查询串已经匹配完了。

这时候能不能算成功，不取决于路径是否存在，而取决于当前节点是不是某个完整单词的结尾：

```java
return node.isEnd;
```

这个点很重要。  
因为路径能走到，不代表它一定是完整单词，可能只是某个更长单词的前缀。

### 普通字符怎么处理

如果当前字符不是 `.`，那就和普通 Trie 查找一样：

1. 计算这个字符对应的下标
2. 看对应子节点是否存在
3. 不存在就返回 `false`
4. 存在就继续递归下去

这一部分没有什么额外难点。

### `.` 为什么必须枚举所有子节点

因为 `.` 可以表示任意一个字母。

也就是说，当你遇到 `.` 时，你不能提前知道下一步到底该走哪条边。  
唯一正确的做法，就是把当前节点下面所有可能存在的子节点都试一遍。

如果其中任何一个子节点，后面的剩余字符串能匹配成功，那整个查询就应该返回 `true`。

所以这里的逻辑不是“找唯一答案”，而是“有没有至少一条路走得通”。

这也是 DFS 最适合的地方。

### 用题目的例子理解一下

假设我们先插入了：

```text
bad
dad
mad
```

那么 Trie 里大致会有三条路径：

```text
b -> a -> d
d -> a -> d
m -> a -> d
```

这时候：

- `search("bad")`  
  走唯一一条路径，能走到结尾，返回 `true`

- `search("pad")`  
  根节点下没有 `p` 这条路，直接返回 `false`

- `search(".ad")`  
  第一个字符是 `.`，那就去试根节点下所有存在的子节点  
  只要其中某一条能继续匹配 `"ad"`，就返回 `true`  
  这里显然可以，所以答案是 `true`

- `search("b..")`  
  先走 `b`，后面两个 `.` 分别继续尝试子节点  
  最终能匹配到 `"bad"`，所以返回 `true`

这个例子一旦走顺，整道题的核心也就明白了。

### 代码

```java
class WordDictionary {
    private class TrieNode {
        TrieNode[] children = new TrieNode[26];
        boolean isEnd = false;
    }

    private TrieNode root;

    public WordDictionary() {
        root = new TrieNode();
    }

    public void addWord(String word) {
        TrieNode node = root;
        for (char c : word.toCharArray()) {
            int idx = c - 'a';
            if (node.children[idx] == null) {
                node.children[idx] = new TrieNode();
            }
            node = node.children[idx];
        }
        node.isEnd = true;
    }

    public boolean search(String word) {
        return dfs(word, 0, root);
    }

    private boolean dfs(String word, int index, TrieNode node) {
        if (index == word.length()) {
            return node.isEnd;
        }

        char c = word.charAt(index);

        if (c == '.') {
            for (int i = 0; i < 26; i++) {
                if (node.children[i] != null) {
                    if (dfs(word, index + 1, node.children[i])) {
                        return true;
                    }
                }
            }
            return false;
        } else {
            int idx = c - 'a';
            if (node.children[idx] == null) {
                return false;
            }
            return dfs(word, index + 1, node.children[idx]);
        }
    }
}
```

### 这段代码最关键的地方

这题最关键的，不是 Trie 本身，而是要看出：

**查询过程已经不再是单一路径，而是“路径 + 分支搜索”的组合。**

也就是说：

- `addWord` 还是标准 Trie
- 真正的难点全部集中在 `search`
- 而 `search` 的本质，就是在 Trie 上做一遍带分支的 DFS

如果这一点想清楚，这题其实就是“上一题 Trie 的升级版”。

### 为什么这里不适合写成循环查找

如果没有 `.`，那 `search` 完全可以写成普通循环，因为每一步都只有唯一选择。

但一旦出现 `.`，当前位置就可能分裂成很多条路径。  
这种情况下，循环写法会变得很别扭，因为你要自己手动维护很多待尝试状态。

而递归天然适合描述这种“走一条路，失败就回退，再试另一条路”的过程。

所以这里用 DFS，不是为了炫技巧，而是因为问题本身就是这种结构。

### 复杂度怎么理解

设查询字符串长度为 `L`。

- `addWord(word)` 的时间复杂度是 $O(L)$  
  因为只是顺着路径插入一遍。

- `search(word)` 在最坏情况下会比较大。  
  如果查询串里有很多 `.`，每个 `.` 都可能分裂成最多 26 个分支，所以最坏复杂度可以写成指数级。

不过这里只需要知道一件事：

**最坏情况虽然大，但实际 Trie 通常远没有那么满，而且题目数据范围也足够让这种写法通过。**

空间复杂度主要来自 Trie 本身，和插入单词总字符数有关。

### 这题真正值得记住的地方

这题最值得记住的，不是“再背一个 Trie 变种”。

真正值得记住的是这个判断：

**当底层数据结构能提供路径，但查询条件里出现了“不确定字符”或“通配符”时，通常就要在结构上做 DFS / 回溯搜索。**

这类题的共同点都很像：

- 数据先按结构存好
- 查询时不一定只有唯一选择
- 一旦出现分支，就要试探多条路

Trie + DFS，本质上就是这么来的。

### 完整代码

```java
public class Main {

    static class WordDictionary {

        private class TrieNode {
            TrieNode[] children;
            boolean isEnd;

            public TrieNode() {
                children = new TrieNode[26];
                isEnd = false;
            }
        }

        private TrieNode root;

        public WordDictionary() {
            root = new TrieNode();
        }

        public void addWord(String word) {
            TrieNode node = root;
            for (int i = 0; i < word.length(); i++) {
                char c = word.charAt(i);
                int index = c - 'a';

                if (node.children[index] == null) {
                    node.children[index] = new TrieNode();
                }

                node = node.children[index];
            }
            node.isEnd = true;
        }

        public boolean search(String word) {
            return dfs(word, 0, root);
        }

        private boolean dfs(String word, int index, TrieNode node) {
            if (index == word.length()) {
                return node.isEnd;
            }

            char c = word.charAt(index);

            if (c == '.') {
                for (int i = 0; i < 26; i++) {
                    if (node.children[i] != null) {
                        if (dfs(word, index + 1, node.children[i])) {
                            return true;
                        }
                    }
                }
                return false;
            } else {
                int i = c - 'a';
                if (node.children[i] == null) {
                    return false;
                }
                return dfs(word, index + 1, node.children[i]);
            }
        }
    }

    public static void main(String[] args) {
        WordDictionary wordDictionary = new WordDictionary();

        System.out.println("addWord(\"bad\")");
        wordDictionary.addWord("bad");

        System.out.println("addWord(\"dad\")");
        wordDictionary.addWord("dad");

        System.out.println("addWord(\"mad\")");
        wordDictionary.addWord("mad");

        System.out.println("search(\"pad\") -> " + wordDictionary.search("pad"));
        System.out.println("search(\"bad\") -> " + wordDictionary.search("bad"));
        System.out.println("search(\".ad\") -> " + wordDictionary.search(".ad"));
        System.out.println("search(\"b..\") -> " + wordDictionary.search("b.."));

        System.out.println("addWord(\"apple\")");
        wordDictionary.addWord("apple");
        System.out.println("search(\"app\") -> " + wordDictionary.search("app"));
        System.out.println("search(\"apple\") -> " + wordDictionary.search("apple"));
        System.out.println("search(\".....\") -> " + wordDictionary.search("....."));
    }
}
```
