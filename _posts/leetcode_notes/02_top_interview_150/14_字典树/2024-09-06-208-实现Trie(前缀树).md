## 题目

**描述：**

**Trie**（发音类似 "try"）或者说 **前缀树** 是一种树形数据结构，用于高效地存储和检索字符串数据集中的键。这一数据结构有相当多的应用情景，例如自动补全和拼写检查。

请你实现 Trie 类：

- `Trie()` 初始化前缀树对象。
- `void insert(String word)` 向前缀树中插入字符串 `word`。
- `boolean search(String word)` 如果字符串 `word` 在前缀树中，返回 `true`（即，在检索之前已经插入）；否则，返回 `false`。
- `boolean startsWith(String prefix)` 如果之前已经插入的字符串 `word` 的前缀之一为 `prefix`，返回 `true`；否则，返回 `false`。

**示例：**

- **输入**

  `["Trie", "insert", "search", "search", "startsWith", "insert", "search"]`

  `[[], ["apple"], ["apple"], ["app"], ["app"], ["app"], ["app"]]`

- **输出**

  `[null, null, true, false, true, null, true]`

- **解释**

  ```
  Trie trie = new Trie();
  trie.insert("apple");
  trie.search("apple");   // 返回 True
  trie.search("app");     // 返回 False
  trie.startsWith("app"); // 返回 True
  trie.insert("app");
  trie.search("app");     // 返回 True
  ```

**提示:**

- `1 <= word.length, prefix.length <= 2000`
- `word` 和 `prefix` 仅由小写英文字母组成
- `insert`、`search` 和 `startsWith` 调用次数 **总计** 不超过 `3 * 10^4` 次

## 题解

### 先别急着看代码，先想前缀树到底是拿来干什么的

这题不是普通的字符串查找题，它真正想考的是一种专门处理“字符串前缀”的数据结构：**Trie**，也叫前缀树。

如果只是判断某个完整单词是否存在，用 `HashSet` 当然也能做。  
但 Trie 的优势不在“单词本身”，而在于它能把很多字符串的**公共前缀合并起来存**。

比如插入下面几个单词：

```text
apple
app
apply
```

它们前面的 `app` 是完全一样的。  
如果分别单独存储，会有很多重复信息；而 Trie 会把这段公共前缀共用掉，等走到分叉处再往下延伸。

所以 Trie 最适合的场景就是：

- 判断某个单词是否存在
- 判断某个前缀是否存在
- 做自动补全、搜索建议这类前缀相关操作

这道题本质上就是在让你实现这个最基础的版本。

### Trie 为什么长得像一棵树

因为它存储的不是“整个单词”，而是**从前到后的一条字符路径**。

比如插入 `"apple"` 时，实际做的事情不是把 `"apple"` 整体扔进某个容器里，而是：

- 从根节点出发
- 走 `'a'`
- 再走 `'p'`
- 再走 `'p'`
- 再走 `'l'`
- 再走 `'e'`

走完之后，在最后那个节点上打一个标记，表示：

**这里正好是一个完整单词的结尾。**

这样一来，如果以后再插入 `"app"`，前面的路径就不用重新建了，只需要走到第二个 `'p'` 后，把那个位置也标记成一个单词结尾即可。

这就是 Trie 的核心。

### 节点里到底要存什么

实现 Trie 时，每个节点通常要存两样东西。

第一样是子节点信息。  
因为题目说字符串只包含小写英文字母，所以一个节点最多只会往下连出 26 条边，对应 `'a' ~ 'z'`。

最常见的写法，就是准备一个长度为 26 的数组：

```java
TrieNode[] children = new TrieNode[26];
```

其中：

- `children[0]` 表示字符 `'a'`
- `children[1]` 表示字符 `'b'`
- ...
- `children[25]` 表示字符 `'z'`

第二样是一个结束标记 `isEnd`。  
它用来区分两种情况：

- 这条路径只是某个单词的前缀
- 这条路径本身就是一个完整单词

这个标记非常关键，因为它正好对应 `search` 和 `startsWith` 的区别。

### 为什么一定要有 `isEnd`

这一点是 Trie 最容易只会写、不会讲清的地方。

比如你插入了 `"apple"` 之后，再去查 `"app"`。

路径上当然是能走到的，因为 `"app"` 本来就是 `"apple"` 的前缀。  
但题目里 `search("app")` 要返回的是 `false`，因为 `"app"` 还没有被当作完整单词插入过。

所以光有路径还不够，还得知道：

**这个节点是不是某个单词真正结束的位置。**

这就是 `isEnd` 存在的意义。

- `search(word)`：路径存在还不够，最后一个节点必须 `isEnd == true`
- `startsWith(prefix)`：只要路径存在就行，不要求是单词结尾

这两个操作的区别，本质上就落在这里。

### `insert` 到底在做什么

插入一个单词时，过程其实很机械：

1. 从根节点开始
2. 依次读取单词的每个字符
3. 计算这个字符在数组中的位置：`c - 'a'`
4. 如果对应子节点不存在，就新建一个
5. 然后指针移动到这个子节点
6. 所有字符处理完后，把当前节点的 `isEnd` 设为 `true`

注意最后这一步不能漏。  
因为它表示“这条路径现在不仅存在，而且它本身就是一个完整单词”。

### `search` 和 `startsWith` 为什么看起来这么像

因为它们本质上都在做同一件事：

**沿着字符路径往下走。**

区别只在“走完之后怎么判断”。

- `search(word)`：要求路径存在，而且最后一个节点必须是完整单词结尾
- `startsWith(prefix)`：只要求路径存在，不关心这个位置是不是单词结尾

所以这两个函数最适合共用一个辅助方法。  
这个辅助方法只负责一件事：

**给我一个字符串，我沿着 Trie 走下去，最后返回对应的节点；如果中途断了，就返回 `null`。**

这样主逻辑就会很干净。

### 辅助函数为什么很值得单独抽出来

如果不抽辅助函数，那 `search` 和 `startsWith` 会各写一遍“从头顺着路径找”的代码。

这会有两个问题：

- 代码重复
- 读者看不出这两个操作本质上只差最后一步判断

而把这部分抽成 `searchPrefix` 之后，结构会非常清楚：

- `searchPrefix` 负责“路径能不能走通”
- `search` 负责“走通之后是不是完整单词”
- `startsWith` 负责“只要走通就算成功”

这样讲起来也顺很多。

### 代码

```java
class Trie {
    private class TrieNode {
        private TrieNode[] children;
        private boolean isEnd;

        public TrieNode() {
            children = new TrieNode[26];
            isEnd = false;
        }
    }

    private TrieNode root;

    public Trie() {
        root = new TrieNode();
    }

    public void insert(String word) {
        TrieNode node = root;
        for (char c : word.toCharArray()) {
            int index = c - 'a';
            if (node.children[index] == null) {
                node.children[index] = new TrieNode();
            }
            node = node.children[index];
        }
        node.isEnd = true;
    }

    public boolean search(String word) {
        TrieNode node = searchPrefix(word);
        return node != null && node.isEnd;
    }

    public boolean startsWith(String prefix) {
        TrieNode node = searchPrefix(prefix);
        return node != null;
    }

    private TrieNode searchPrefix(String prefix) {
        TrieNode node = root;
        for (char c : prefix.toCharArray()) {
            int index = c - 'a';
            if (node.children[index] == null) {
                return null;
            }
            node = node.children[index];
        }
        return node;
    }
}
```

### 这段代码最关键的地方

真正关键的其实不是数组，也不是语法，而是你要理解下面这件事：

**Trie 里每走一步，不是在“比较整个字符串”，而是在沿着前缀路径往下移动。**

所以它和普通字符串存储最大的区别就在于：

- 普通做法往往按“单词”为单位处理
- Trie 按“字符路径”为单位处理

这也是为什么它在做前缀查询时会特别自然。  
因为前缀本来就是路径的前半段。

### 用题目的例子理解一下

假设执行下面这些操作：

```text
insert("apple")
search("apple")
search("app")
startsWith("app")
insert("app")
search("app")
```

过程可以这样理解：

先插入 `"apple"`，Trie 里有了一条：

```text
a -> p -> p -> l -> e
```

而且只有最后的 `'e'` 节点被标记为单词结尾。

这时候：

- `search("apple")` 返回 `true`，因为路径存在，最后节点也是结尾
- `search("app")` 返回 `false`，因为虽然路径存在，但第二个 `'p'` 还不是单词结尾
- `startsWith("app")` 返回 `true`，因为它只关心路径在不在

然后再插入 `"app"`，这时候第二个 `'p'` 节点也会被标记成结尾。  
所以之后再 `search("app")`，就会返回 `true`。

这个过程一旦看顺了，Trie 的本质就基本明白了。

### 这题为什么适合用数组而不是 `HashMap`

因为题目条件已经很明确：

- 所有字符串都只由小写字母组成

这意味着字符集大小固定就是 26。  
在这种情况下，用数组存子节点会更直接：

- 下标映射简单
- 访问速度快
- 代码也更规整

如果字符集很大，或者不是固定的小写字母，比如可能包含大小写、数字、符号，那用 `HashMap<Character, TrieNode>` 会更灵活。

但这题里，数组版本就是最标准、最适合讲清原理的实现方式。

### 复杂度分析

设单词长度为 `L`。

对于 Trie 的三个操作：

- `insert(word)`：需要从头到尾走一遍单词，所以是 $O(L)$
- `search(word)`：同样要顺着路径走一遍，所以是 $O(L)$
- `startsWith(prefix)`：只要检查前缀路径是否存在，所以也是 $O(L)$

空间复杂度比较难写成一个特别精确的式子，但核心可以这样理解：

- 每插入一个新字符，最坏情况下就可能新建一个节点
- 每个节点内部有一个长度为 26 的子节点数组

所以总体空间开销和所有插入字符串的总字符数有关。  
如果很多单词共享前缀，空间会节省不少；如果几乎没有公共前缀，节点数就会更多。

### 这题真正值得记住的地方

这题最值得记住的，不是“把接口背下来”。

真正值得记住的是这两个认识：

第一，**Trie 适合处理前缀问题。**  
因为它本质上就是把字符串按前缀路径组织起来。

第二，**`search` 和 `startsWith` 的差别不在查找过程，而在最后是否要求 `isEnd == true`。**  
这个点想透了，Trie 的实现就不会再觉得碎。

很多人第一次学 Trie，会觉得它只是一个“特殊模板”。  
其实不是。它只是把“公共前缀”这件事结构化了而已。

### 完整代码

```java
public class Main {

    static class Trie {

        private class TrieNode {
            private TrieNode[] children;
            private boolean isEnd;

            public TrieNode() {
                children = new TrieNode[26];
                isEnd = false;
            }
        }

        private TrieNode root;

        public Trie() {
            root = new TrieNode();
        }

        public void insert(String word) {
            TrieNode node = root;
            for (int i = 0; i < word.length(); i++) {
                char c = word.charAt(i);
                int index = c - 'a';

                if (node.children[index] == null) {
                    node.children[index] = new TrieNode();
                }

                node = node.children[index];
            }
            node.isEnd = true;
        }

        public boolean search(String word) {
            TrieNode node = searchPrefix(word);
            return node != null && node.isEnd;
        }

        public boolean startsWith(String prefix) {
            TrieNode node = searchPrefix(prefix);
            return node != null;
        }

        private TrieNode searchPrefix(String prefix) {
            TrieNode node = root;
            for (int i = 0; i < prefix.length(); i++) {
                char c = prefix.charAt(i);
                int index = c - 'a';

                if (node.children[index] == null) {
                    return null;
                }

                node = node.children[index];
            }
            return node;
        }
    }

    public static void main(String[] args) {
        Trie trie = new Trie();

        System.out.println("insert(\"apple\")");
        trie.insert("apple");

        System.out.println("search(\"apple\") = " + trie.search("apple"));
        System.out.println("search(\"app\") = " + trie.search("app"));
        System.out.println("startsWith(\"app\") = " + trie.startsWith("app"));

        System.out.println("insert(\"app\")");
        trie.insert("app");

        System.out.println("search(\"app\") = " + trie.search("app"));

        System.out.println("search(\"banana\") = " + trie.search("banana"));
        trie.insert("banana");
        System.out.println("search(\"banana\") = " + trie.search("banana"));
        System.out.println("startsWith(\"ban\") = " + trie.startsWith("ban"));
    }
}
```
