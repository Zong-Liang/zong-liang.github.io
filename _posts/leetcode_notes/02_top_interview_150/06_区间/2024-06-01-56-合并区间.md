## 题目

**描述：**

以数组 `intervals` 表示若干个区间的集合，其中单个区间为 `intervals[i] = [start_i, end_i]` 。请你合并所有重叠的区间，并返回一个不重叠的区间数组，该数组需恰好覆盖输入中的所有区间。

**示例 1:**

- **输入:** `intervals = [[1,3],[2,6],[8,10],[15,18]]`
- **输出:** `[[1,6],[8,10],[15,18]]`
- **解释:** 区间 `[1,3]` 和 `[2,6]` 重叠, 将它们合并为 `[1,6]`。

**示例 2:**

- **输入:** `intervals = [[1,4],[4,5]]`
- **输出:** `[[1,5]]`
- **解释:** 区间 `[1,4]` 和 `[4,5]` 可被视为重叠区间。

**示例 3:**

- **输入:** `intervals = [[4,7],[1,4]]`
- **输出:** `[[1,7]]`
- **解释:** 区间 `[1,4]` 和 `[4,7]` 可被视为重叠区间。

**提示:**

- `1 <= intervals.length <= 10^4`
- `intervals[i].length == 2`
- `0 <= start_i <= end_i <= 10^4`

## 题解

这题如果拿着一堆乱序区间就直接开始合并，往往会越看越乱。因为当顺序没有理清时，你很难判断“当前这个区间到底应该跟谁合在一起”。所以这题真正应该先做的，不是合并，而是先把所有区间按左端点排好序。

一旦排完序，原本分散的重叠关系就会被拉成一条线。能合并的区间一定会挨在一起出现，这时我们只需要维护结果数组里的最后一个区间，再把后面的区间一个个接上去看能不能并。

假设结果里的最后一个区间是 `[lastStart, lastEnd]`，当前扫描到的区间是 `[currStart, currEnd]`。因为已经按左端点升序排列，所以只要判断 `currStart <= lastEnd` 是否成立就够了：成立，说明两个区间有重叠，就把右端点更新成 `max(lastEnd, currEnd)`；不成立，说明当前区间已经彻底落在后面，只能另开一段。

这里 `<=` 很重要。像 `[1,4]` 和 `[4,5]` 这种情况，端点 `4` 同时属于两个闭区间，题目认为它们是可以合并的。如果写成 `<`，这个边界就会判错。

拿 `[[1,3],[2,6],[8,10],[15,18]]` 走一遍：先把 `[1,3]` 放进结果；看到 `[2,6]` 时，因为 `2 <= 3`，它和前一段重叠，于是合成 `[1,6]`；再看 `[8,10]`，因为 `8 > 6`，说明这一段断开了，只能单独加入；最后 `[15,18]` 同理。结果就是 `[[1,6],[8,10],[15,18]]`。

所以整道题其实是在做两件事：先排序，把全局关系理顺；再线性扫描，只和“结果里的最后一段”打交道。这个视角一旦理顺，代码会非常直接。

代码如下：

```java
public int[][] merge(int[][] intervals) {
    if (intervals.length == 0) {
        return new int[0][2];
    }

    Arrays.sort(intervals, (a, b) -> Integer.compare(a[0], b[0]));

    List<int[]> merged = new ArrayList<>();
    merged.add(intervals[0]);

    for (int i = 1; i < intervals.length; i++) {
        int[] current = intervals[i];
        int[] last = merged.get(merged.size() - 1);

        if (current[0] <= last[1]) {
            last[1] = Math.max(last[1], current[1]);
        } else {
            merged.add(current);
        }
    }

    return merged.toArray(new int[merged.size()][]);
}
```

时间复杂度是 `O(n log n)`，主要花在排序上，后面的扫描和合并只需要 `O(n)`；空间复杂度如果不算返回结果本身，主要来自排序过程和结果列表。

做完这题后，可以顺手记住一个更普遍的判断：只要区间之间的关系强依赖相对位置，先排序通常就能把“我要和谁比较”这种全局问题压缩成局部问题。本题里被压缩出来的局部关系，就是“当前区间只需要和结果里的最后一个区间比较”。

完整代码如下：

```java
import java.util.ArrayList;
import java.util.Arrays;
import java.util.List;

public class Main {

    public static int[][] merge(int[][] intervals) {
        if (intervals.length == 0) {
            return new int[0][2];
        }

        Arrays.sort(intervals, (a, b) -> Integer.compare(a[0], b[0]));

        List<int[]> merged = new ArrayList<>();
        merged.add(intervals[0]);

        for (int i = 1; i < intervals.length; i++) {
            int[] currentInterval = intervals[i];
            int[] lastInterval = merged.get(merged.size() - 1);

            if (currentInterval[0] <= lastInterval[1]) {
                lastInterval[1] = Math.max(lastInterval[1], currentInterval[1]);
            } else {
                merged.add(currentInterval);
            }
        }

        return merged.toArray(new int[merged.size()][]);
    }

    public static void main(String[] args) {
        int[][] intervals1 = {{1, 3}, {2, 6}, {8, 10}, {15, 18}};
        System.out.println("示例 1 输入: " + Arrays.deepToString(intervals1));
        int[][] res1 = merge(intervals1);
        System.out.println("示例 1 输出: " + Arrays.deepToString(res1));
        System.out.println("-----------------------------");

        int[][] intervals2 = {{1, 4}, {4, 5}};
        System.out.println("示例 2 输入: " + Arrays.deepToString(intervals2));
        int[][] res2 = merge(intervals2);
        System.out.println("示例 2 输出: " + Arrays.deepToString(res2));
        System.out.println("-----------------------------");

        int[][] intervals3 = {{4, 7}, {1, 4}};
        System.out.println("示例 3 输入: " + Arrays.deepToString(intervals3));
        int[][] res3 = merge(intervals3);
        System.out.println("示例 3 输出: " + Arrays.deepToString(res3));
    }
}
```
