## 题目

**描述：**

以数组 `intervals` 表示若干个区间的集合，其中单个区间为 `intervals[i] = [start_i, end_i]` 。请你合并所有重叠的区间，并返回一个不重叠的区间数组，该数组需恰好覆盖输入中的所有区间。

**示例 1:**

- **输入:** `intervals = [[1,3],[2,6],[8,10],[15,18]]`
- **输出:** `[[1,6],[8,10],[15,18]]`
- **解释:** 区间 `[1,3]` 和 `[2,6]` 重叠, 将它们合并为 `[1,6]`。

**示例 2:**

- **输入:** `intervals = [[1,4],[4,5]]`
- **输出:** `[[1,5]]`
- **解释:** 区间 `[1,4]` 和 `[4,5]` 可被视为重叠区间。

**示例 3:**

- **输入:** `intervals = [[4,7],[1,4]]`
- **输出:** `[[1,7]]`
- **解释:** 区间 `[1,4]` 和 `[4,7]` 可被视为重叠区间。

**提示:**

- `1 <= intervals.length <= 10^4`
- `intervals[i].length == 2`
- `0 <= start_i <= end_i <= 10^4`

## 题解

### 这题真正的关键，不是“合并”，而是先把区间排成能合并的顺序

如果区间本来就是乱的，比如：

```text
[[8,10],[1,3],[2,6]]
```

你很难一眼判断当前区间到底该和谁合并。  
因为可能前面能接，后面也能接，顺序一乱，处理起来就很别扭。

所以这题最关键的一步，不是上来就想怎么合并，而是先做一件更基础的事：

**把所有区间按左端点从小到大排好。**

一旦排完序，事情就简单很多了。因为此时如果某些区间能合并，它们一定会挨在一起出现，不会隔着别的区间。

### 为什么排序之后，合并就变成了一次线性扫描

假设区间已经按左端点升序排列。

这时我们从前往后看，每次只需要关心两件事：

- 当前结果里的最后一个区间
- 正在遍历到的新区间

如果这两个区间有重叠，那就把它们合成一个更大的区间；  
如果没有重叠，那说明当前区间应该单独开一个新区间。

也就是说，排序之后，这题不再是“当前区间到底该和谁合”，而变成了更简单的判断：

**当前区间是接在前一个区间后面，还是另起一段。**

这就是为什么这题的标准做法一定是“排序 + 一次遍历”。

### 两个区间什么时候算重叠

设当前结果里的最后一个区间是：

```text
[lastStart, lastEnd]
```

当前遍历到的区间是：

```text
[currStart, currEnd]
```

因为已经按左端点排过序，所以一定有：

```text
currStart >= lastStart
```

这时候只要看：

```text
currStart <= lastEnd
```

是否成立。

- 如果成立，说明两个区间有交集，或者刚好挨在一起，应该合并
- 如果不成立，说明当前区间已经完全落在后面，只能单独开新区间

这里要特别注意一点：  
像 `[1,4]` 和 `[4,5]` 这种情况，题目也算重叠，所以判断条件必须写成：

```java
current[0] <= last[1]
```

而不是严格小于。

### 真正的合并动作其实很简单

如果确定当前区间和最后一个区间重叠，那新的左端点其实不用动，因为排序之后左端点更小的一定已经在前面了。

真正要更新的，只有右端点：

```text
新的右端点 = max(lastEnd, currEnd)
```

也就是说，合并的本质就是把当前结果区间往右尽量延长。

所以代码里常见的写法就是：

```java
last[1] = Math.max(last[1], current[1]);
```

这一步看起来普通，但它其实正对应着这道题的本质：  
**维护当前这一段合并后的最右边界。**

### 用示例走一遍

看示例 1：

```text
intervals = [[1,3],[2,6],[8,10],[15,18]]
```

它本来就已经按左端点排好了。

先把 `[1,3]` 放进结果里。  
接着看 `[2,6]`：

- 当前结果最后一个区间是 `[1,3]`
- `2 <= 3`，说明重叠

于是合并成：

```text
[1,6]
```

再看 `[8,10]`：

- 当前结果最后一个区间是 `[1,6]`
- `8 > 6`，说明不重叠

所以它不能接在前面后面，只能自己开一个新区间：

```text
[1,6], [8,10]
```

最后 `[15,18]` 同理，也是不重叠，于是结果变成：

```text
[[1,6],[8,10],[15,18]]
```

这就是最终答案。

### 再看一个最容易忽略的边界情况

看示例 2：

```text
[[1,4],[4,5]]
```

很多人第一次做时，会把它们看成“不重叠”，因为一个在 4 结束，一个从 4 开始。

但题目这里把它们视为可以合并。  
因为区间是闭区间，端点 `4` 同时属于两个区间，所以这两个区间确实有交集。

所以最后答案是：

```text
[[1,5]]
```

这也是为什么判断条件必须是 `<=`，不能写成 `<`。

### 代码

```java
public int[][] merge(int[][] intervals) {
    if (intervals.length == 0) {
        return new int[0][2];
    }

    Arrays.sort(intervals, (a, b) -> Integer.compare(a[0], b[0]));

    List<int[]> merged = new ArrayList<>();
    merged.add(intervals[0]);

    for (int i = 1; i < intervals.length; i++) {
        int[] current = intervals[i];
        int[] last = merged.get(merged.size() - 1);

        if (current[0] <= last[1]) {
            last[1] = Math.max(last[1], current[1]);
        } else {
            merged.add(current);
        }
    }

    return merged.toArray(new int[merged.size()][]);
}
```

时间复杂度是 $O(n \log n)$，主要花在排序上。  
后面的扫描和合并只需要 $O(n)$。

空间复杂度如果不算返回结果本身，通常主要来自排序过程和结果列表；整体思路上额外负担不大。

### 这题真正值得记住的地方

这题最值得记住的，不只是“区间题先排序”这句话，而是它背后的判断方式：

**当问题里的关系取决于相对位置时，先排序，往往能把原本分散的关系变成局部关系。**

在这题里，区间能不能合并，本来是一个全局问题；  
但一旦按左端点排好序，它就变成了：

**当前区间只需要和结果里的最后一个区间比较。**

这一下就把问题从“全局找能不能合”压成了“局部看能不能接”。

以后再遇到区间类问题，尤其是这种：

- 要合并
- 要找重叠
- 要按顺序输出结果

都应该先问自己一句：

**排个序之后，会不会简单很多？**

### 完整代码

```java
import java.util.ArrayList;
import java.util.Arrays;
import java.util.List;

public class Main {

    public static int[][] merge(int[][] intervals) {
        if (intervals.length == 0) {
            return new int[0][2];
        }

        Arrays.sort(intervals, (a, b) -> Integer.compare(a[0], b[0]));

        List<int[]> merged = new ArrayList<>();
        merged.add(intervals[0]);

        for (int i = 1; i < intervals.length; i++) {
            int[] currentInterval = intervals[i];
            int[] lastInterval = merged.get(merged.size() - 1);

            if (currentInterval[0] <= lastInterval[1]) {
                lastInterval[1] = Math.max(lastInterval[1], currentInterval[1]);
            } else {
                merged.add(currentInterval);
            }
        }

        return merged.toArray(new int[merged.size()][]);
    }

    public static void main(String[] args) {
        int[][] intervals1 = {{1, 3}, {2, 6}, {8, 10}, {15, 18}};
        System.out.println("示例 1 输入: " + Arrays.deepToString(intervals1));
        int[][] res1 = merge(intervals1);
        System.out.println("示例 1 输出: " + Arrays.deepToString(res1));
        System.out.println("-----------------------------");

        int[][] intervals2 = {{1, 4}, {4, 5}};
        System.out.println("示例 2 输入: " + Arrays.deepToString(intervals2));
        int[][] res2 = merge(intervals2);
        System.out.println("示例 2 输出: " + Arrays.deepToString(res2));
        System.out.println("-----------------------------");

        int[][] intervals3 = {{4, 7}, {1, 4}};
        System.out.println("示例 3 输入: " + Arrays.deepToString(intervals3));
        int[][] res3 = merge(intervals3);
        System.out.println("示例 3 输出: " + Arrays.deepToString(res3));
    }
}
```
