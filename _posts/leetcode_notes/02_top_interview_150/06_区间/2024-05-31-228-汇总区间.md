## 题目

**描述：**

给定一个 **无重复元素** 的 **有序** 整数数组 `nums` 。

区间 `[a,b]` 是从 `a` 到 `b` (包含) 的所有整数的集合。

返回 **恰好覆盖** 数组中所有数字的 **最小有序** 区间范围列表。也就是说，`nums` 的每个元素都恰好被某个区间范围所覆盖，并且不存在属于某个区间但不属于 `nums` 的数字 `x` 。

列表中的每个区间范围 `[a,b]` 应该按如下格式输出：

- `"a->b"` ，如果 `a != b`
- `"a"` ，如果 `a == b`

**示例 1:**

- **输入:** `nums = [0,1,2,4,5,7]`
- **输出:** `["0->2","4->5","7"]`
- **解释:** 区间范围是:
  - `[0,2] --> "0->2"`
  - `[4,5] --> "4->5"`
  - `[7,7] --> "7"`

**示例 2:**

- **输入:** `nums = [0,2,3,4,6,8,9]`
- **输出:** `["0","2->4","6","8->9"]`
- **解释:** 区间范围是:
  - `[0,0] --> "0"`
  - `[2,4] --> "2->4"`
  - `[6,6] --> "6"`
  - `[8,9] --> "8->9"`

**提示:**

- `0 <= nums.length <= 20`
- `-2^31 <= nums[i] <= 2^31 - 1`
- `nums` 中的所有值都 **互不相同**
- `nums` 按 **升序** 排列

## 题解

### 这题本质上是在把连续数字“打包”

题目给的是一个**有序**、**无重复**的数组，要求把它压成若干个区间。

比如：

```text
[0,1,2,4,5,7]
```

里面的 `0,1,2` 是连续的，就应该被打成一个区间 `"0->2"`；  
`4,5` 也是连续的，打成 `"4->5"`；  
`7` 单独一个数，就写成 `"7"`。

所以这题真正要做的事情，不是复杂的区间合并，而是很简单的一件事：

**顺着数组往后扫，把一段一段连续的数字找出来。**

### 这题能做得很直接，是因为题目条件已经给得很友好

这里有两个条件很重要：

- 数组已经升序排列
- 数组里没有重复元素

正因为有这两个条件，我们才可以放心地用“当前数是不是前一个数加一”来判断连续性。

如果数组无序，那你根本没法顺着扫。  
如果数组有重复，连续区间的定义也会变得麻烦。

但这题都帮你排干净了，所以整道题其实就是一次线性扫描。

### 一次遍历就够了

可以把整个过程理解成：  
每次找到一个区间的起点，然后一直往后走，直到这个区间断掉。

具体做法就是用一个指针 `i` 从头往后扫。

当 `i` 指向某个位置时，这个位置就是当前区间的起点。  
先把它记下来，然后继续往后看：

- 如果 `nums[i] == nums[i - 1] + 1`，说明还连续，继续往后走
- 一旦不满足，说明这个区间到前一个数为止结束了

这样一来，每次外层循环负责开一个新区间，内层循环负责把这个区间尽量往后扩。

### 为什么这里虽然像双层循环，复杂度还是 O(n)

很多人看到这种“外层 while + 内层 while”的写法，会下意识担心是不是 $O(n^2)$。

其实不是。

因为指针 `i` 在整个过程中只会一直往前走，不会回头。  
换句话说，每个元素最多只会被扫到一次。

所以虽然代码结构上看起来有两层循环，但总移动次数加起来还是 `n` 级别，时间复杂度仍然是 $O(n)$。

### 关键不是怎么找，而是怎么确定区间边界

这题真正要想清楚的是：  
当你找到一段连续区间后，怎么知道它的起点和终点分别是谁。

通常的写法是：

- 用 `low` 记录当前区间开始时的下标
- 让 `i` 一路往后走，直到不连续
- 这时区间终点就是 `i - 1`

比如数组是：

```text
[0,1,2,4,5,7]
```

一开始 `i = 0`，那么：

- `low = 0`
- 然后 `i` 往后走到 `3`，因为 `nums[3] = 4`，已经不连续了
- 所以当前区间的结束下标就是 `2`

也就是区间 `[0,2]`，对应输出 `"0->2"`。

### 输出格式其实只有两种

找到区间之后，格式化并不复杂，只有两种情况：

- 如果起点和终点是同一个数，就直接输出这个数，比如 `"7"`
- 如果起点和终点不同，就输出 `"a->b"`，比如 `"0->2"`

所以这题最后不是在返回区间对象，而是在返回格式化好的字符串列表。

### 用示例走一遍

看示例 1：

```text
nums = [0,1,2,4,5,7]
```

从头开始：

- `0,1,2` 连续，所以形成一个区间 `"0->2"`
- 接着 `4,5` 连续，所以形成一个区间 `"4->5"`
- 最后 `7` 单独一个数，所以就是 `"7"`

最终结果：

```text
["0->2","4->5","7"]
```

再看示例 2：

```text
nums = [0,2,3,4,6,8,9]
```

这里：

- `0` 单独一个区间，输出 `"0"`
- `2,3,4` 连续，输出 `"2->4"`
- `6` 单独输出 `"6"`
- `8,9` 连续，输出 `"8->9"`

所以结果是：

```text
["0","2->4","6","8->9"]
```

### 代码

```java
public List<String> summaryRanges(int[] nums) {
    List<String> result = new ArrayList<>();
    int n = nums.length;
    int i = 0;

    while (i < n) {
        int low = i;
        i++;

        while (i < n && nums[i] == nums[i - 1] + 1) {
            i++;
        }

        int high = i - 1;

        StringBuilder sb = new StringBuilder();
        sb.append(nums[low]);
        if (low < high) {
            sb.append("->");
            sb.append(nums[high]);
        }

        result.add(sb.toString());
    }

    return result;
}
```

时间复杂度是 $O(n)$，因为整个数组只会被顺着扫一遍。  
空间复杂度如果不算结果列表本身，就是 $O(1)$；额外只用了几个变量。

### 这题真正值得记住的地方

这题本身不难，但很适合作为一种基础模板去记：

**当数组已经有序时，很多“分段”“分组”“压缩区间”的问题，都可以用“找到起点，再一路扩到不能扩”为止”的方式处理。**

这类题通常有几个共同特征：

- 输入是有序的
- 你要找的是一段一段的连续结构
- 每一段内部满足某种简单关系

像这题里，这个关系就是：

```text
nums[i] == nums[i - 1] + 1
```

一旦这个关系断掉，就说明上一段结束，下一段开始。

把这种“按段扫描”的感觉练熟，后面很多区间题、分组题都会顺很多。

### 完整代码

```java
import java.util.ArrayList;
import java.util.Arrays;
import java.util.List;

public class Main {

    public static List<String> summaryRanges(int[] nums) {
        List<String> result = new ArrayList<>();
        int n = nums.length;
        int i = 0;

        while (i < n) {
            int low = i;
            i++;

            while (i < n && nums[i] == nums[i - 1] + 1) {
                i++;
            }

            int high = i - 1;

            StringBuilder sb = new StringBuilder();
            sb.append(nums[low]);
            if (low < high) {
                sb.append("->");
                sb.append(nums[high]);
            }

            result.add(sb.toString());
        }

        return result;
    }

    public static void main(String[] args) {
        int[] nums1 = {0, 1, 2, 4, 5, 7};
        System.out.println("示例 1 输入: " + Arrays.toString(nums1));
        List<String> res1 = summaryRanges(nums1);
        System.out.println("示例 1 输出: " + res1);
        System.out.println("-----------------------------");

        int[] nums2 = {0, 2, 3, 4, 6, 8, 9};
        System.out.println("示例 2 输入: " + Arrays.toString(nums2));
        List<String> res2 = summaryRanges(nums2);
        System.out.println("示例 2 输出: " + res2);
        System.out.println("-----------------------------");

        int[] nums3 = {};
        System.out.println("示例 3 输入: " + Arrays.toString(nums3));
        List<String> res3 = summaryRanges(nums3);
        System.out.println("示例 3 输出: " + res3);
        System.out.println("-----------------------------");

        int[] nums4 = {-1};
        System.out.println("示例 4 输入: " + Arrays.toString(nums4));
        List<String> res4 = summaryRanges(nums4);
        System.out.println("示例 4 输出: " + res4);
    }
}
```
