## 题目

**描述：**

给定一个 **无重复元素** 的 **有序** 整数数组 `nums` 。

区间 `[a,b]` 是从 `a` 到 `b` (包含) 的所有整数的集合。

返回 **恰好覆盖** 数组中所有数字的 **最小有序** 区间范围列表。也就是说，`nums` 的每个元素都恰好被某个区间范围所覆盖，并且不存在属于某个区间但不属于 `nums` 的数字 `x` 。

列表中的每个区间范围 `[a,b]` 应该按如下格式输出：

- `"a->b"` ，如果 `a != b`
- `"a"` ，如果 `a == b`

**示例 1:**

- **输入:** `nums = [0,1,2,4,5,7]`
- **输出:** `["0->2","4->5","7"]`
- **解释:** 区间范围是:
  - `[0,2] --> "0->2"`
  - `[4,5] --> "4->5"`
  - `[7,7] --> "7"`

**示例 2:**

- **输入:** `nums = [0,2,3,4,6,8,9]`
- **输出:** `["0","2->4","6","8->9"]`
- **解释:** 区间范围是:
  - `[0,0] --> "0"`
  - `[2,4] --> "2->4"`
  - `[6,6] --> "6"`
  - `[8,9] --> "8->9"`

**提示:**

- `0 <= nums.length <= 20`
- `-2^31 <= nums[i] <= 2^31 - 1`
- `nums` 中的所有值都 **互不相同**
- `nums` 按 **升序** 排列

## 题解

这题看着像区间题，其实更像一道“分段扫描”题。

题目已经把条件给得很干净了：数组本身有序，而且没有重复元素。这样一来，连续数字一定会挨在一起出现，我们要做的无非是顺着数组往后走，把每一段连续区间的起点和终点找出来，再按题目要求拼成字符串。

如果输入是无序的，或者里面允许重复，这题就不会这么顺手，因为你没法只靠相邻元素来判断一段区间有没有结束。正因为题目提前把顺序理好了，才让我们可以放心地用一趟扫描解决问题。

具体做法并不复杂。用一个指针 `i` 从左往右扫，先把当前位置记成当前区间的起点，然后继续往后扩：只要 `nums[i] == nums[i - 1] + 1`，说明这一段还没断，就继续前进；一旦不满足，当前这段连续区间就结束了，终点就是前一个位置。

这样处理时，虽然代码里会出现“外层找起点，内层往后扩”的写法，但复杂度并不会变成 `O(n^2)`。原因很简单：指针 `i` 整个过程中只会单调往前走，每个元素最多被扫过一次，所以总成本还是线性的。

拿示例 `nums = [0,1,2,4,5,7]` 来看会更直观。先从 `0` 出发，往后能一路扩到 `2`，于是得到第一段 `"0->2"`；接着从 `4` 出发，扩到 `5`，得到 `"4->5"`；最后只剩下一个 `7`，它自己就是一段，所以直接记成 `"7"`。这也正好对应题目要求的两种输出格式：如果一段里只有一个数，就直接写这个数；如果一段里有多个连续数字，就写成 `"start->end"`。

代码里通常会用 `low` 记录当前区间的起点下标，等指针停下来以后，再用 `i - 1` 当作终点下标。这样既容易写，也不容易把边界绕乱。真正需要注意的只是格式化输出那一步：`low == high` 时输出单个数字，否则输出箭头区间。

代码如下：

```java
public List<String> summaryRanges(int[] nums) {
    List<String> result = new ArrayList<>();
    int n = nums.length;
    int i = 0;

    while (i < n) {
        int low = i;
        i++;

        while (i < n && nums[i] == nums[i - 1] + 1) {
            i++;
        }

        int high = i - 1;

        StringBuilder sb = new StringBuilder();
        sb.append(nums[low]);
        if (low < high) {
            sb.append("->");
            sb.append(nums[high]);
        }

        result.add(sb.toString());
    }

    return result;
}
```

时间复杂度是 `O(n)`，因为整个数组只会被顺着扫一遍；空间复杂度如果不算结果列表本身，就是 `O(1)`。

这题可以顺手记一个很常见的处理动作：当输入已经有序，而你要找的是一段一段满足相邻关系的连续块时，优先考虑“找到起点，再一路扩到不能扩为止”的写法。后面遇到按段压缩、分组统计、合并连续片段之类的题，这个思路都会很好用。

完整代码如下：

```java
import java.util.ArrayList;
import java.util.Arrays;
import java.util.List;

public class Main {

    public static List<String> summaryRanges(int[] nums) {
        List<String> result = new ArrayList<>();
        int n = nums.length;
        int i = 0;

        while (i < n) {
            int low = i;
            i++;

            while (i < n && nums[i] == nums[i - 1] + 1) {
                i++;
            }

            int high = i - 1;

            StringBuilder sb = new StringBuilder();
            sb.append(nums[low]);
            if (low < high) {
                sb.append("->");
                sb.append(nums[high]);
            }

            result.add(sb.toString());
        }

        return result;
    }

    public static void main(String[] args) {
        int[] nums1 = {0, 1, 2, 4, 5, 7};
        System.out.println("示例 1 输入: " + Arrays.toString(nums1));
        List<String> res1 = summaryRanges(nums1);
        System.out.println("示例 1 输出: " + res1);
        System.out.println("-----------------------------");

        int[] nums2 = {0, 2, 3, 4, 6, 8, 9};
        System.out.println("示例 2 输入: " + Arrays.toString(nums2));
        List<String> res2 = summaryRanges(nums2);
        System.out.println("示例 2 输出: " + res2);
        System.out.println("-----------------------------");

        int[] nums3 = {};
        System.out.println("示例 3 输入: " + Arrays.toString(nums3));
        List<String> res3 = summaryRanges(nums3);
        System.out.println("示例 3 输出: " + res3);
        System.out.println("-----------------------------");

        int[] nums4 = {-1};
        System.out.println("示例 4 输入: " + Arrays.toString(nums4));
        List<String> res4 = summaryRanges(nums4);
        System.out.println("示例 4 输出: " + res4);
    }
}
```
