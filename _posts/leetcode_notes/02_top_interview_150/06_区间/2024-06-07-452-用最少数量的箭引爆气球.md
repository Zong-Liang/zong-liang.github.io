## 题目

**描述：**

有一些球形气球贴在一堵用 XY 平面表示的墙面上。墙面上的气球记录在整数数组 `points` ，其中`points[i] = [x_start, x_end]` 表示水平直径在 `x_start` 和 `x_end` 之间的气球。你不知道气球的精确 y 坐标。

一支弓箭可以沿着 x 轴从不同点 **完全垂直** 地射出。在坐标 `x` 处射出一支箭，若有一个气球的直径的开始和结束坐标为 `x_start`，`x_end`， 且满足 `x_start ≤ x ≤ x_end`，则该气球会被引爆。可以射出的弓箭的数量 **没有限制** 。弓箭一旦被射出之后，可以无限地前进。

给你一个数组 `points` ，返回引爆所有气球所必须射出的 **最小** 弓箭数。

**示例 1:**

- **输入:** `points = [[10,16],[2,8],[1,6],[7,12]]`
- **输出:** 2
- **解释:** 气球可以用 2 支箭来爆破:
  - 在 x = 6 处射出箭，击破气球 和。
  - 在 x = 11 处发射箭，击破气球 和。

**示例 2:**

- **输入:** `points = [[1,2],[3,4],[5,6],[7,8]]`
- **输出:** 4
- **解释:** 每个气球需要射出一支箭，总共需要 4 支箭。

**示例 3:**

- **输入:** `points = [[1,2],[2,3],[3,4],[4,5]]`
- **输出:** 2
- **解释:** 气球可以用 2 支箭来爆破:
  - 在 x = 2 处发射箭，击破气球 和。
  - 在 x = 4 处射出箭，击破气球 和。

**提示:**

- `1 <= points.length <= 10^5`
- `points[i].length == 2`
- `-2^31 <= x_start < x_end <= 2^31 - 1`

## 题解

这题换个说法会清楚很多：每个气球都是一个区间 `[x_start, x_end]`，一支箭如果射在某个位置 `x`，只要 `x` 落在这个区间里，这个气球就会爆。于是问题就变成了：用尽可能少的点，去覆盖所有区间。

一旦这么看，它就是很典型的区间贪心题。关键不是“箭该不该多试几个位置”，而是每次已经决定要补一支箭时，应该把它放在哪里最划算。这里最稳的做法，是始终把箭压在当前最早结束的那个区间的右端点上。

所以第一步要先按右端点排序。原因也很自然：对于结束得最早的那个区间，你已经没有继续往右拖的空间了；如果箭落到它右边，这个区间就直接漏掉。既然如此，那不如干脆把箭射在它的右边界上。这样既能保证这个区间被打中，又把位置尽量往右留，方便顺手覆盖后面更多区间。

排完序以后，扫描过程就很直白了。先用第一支箭打第一个区间，位置放在它的右边界。之后依次看后面的区间：如果当前区间的左边界 `<=` 这支箭的位置，说明它也被这支箭覆盖到了，不用做任何事；如果当前区间的左边界已经 `>` 这支箭的位置，说明前一支箭够不到它了，只能新补一支箭，而新箭的位置仍然放在当前区间的右边界。

拿 `[[10,16],[2,8],[1,6],[7,12]]` 走一遍。按右端点排序后得到 `[[1,6],[2,8],[7,12],[10,16]]`。第一支箭放在 `6`，它能同时打掉 `[1,6]` 和 `[2,8]`；到 `[7,12]` 时，因为 `7 > 6`，第一支箭已经不够了，于是补第二支箭，位置放在 `12`；最后 `[10,16]` 也会被第二支箭顺手打掉，所以答案是 `2`。

这道题的贪心之所以成立，本质上是因为“结束最早的区间”约束最强。想兼顾它和后面的区间，你的选择空间只能在它的右边界之前；既然如此，把箭放到这个最右的位置只会更有利，不会更吃亏。

代码如下：

```java
public int findMinArrowShots(int[][] points) {
    if (points.length == 0) {
        return 0;
    }

    Arrays.sort(points, (a, b) -> Integer.compare(a[1], b[1]));

    int arrows = 1;
    int currentArrowPos = points[0][1];

    for (int i = 1; i < points.length; i++) {
        if (points[i][0] > currentArrowPos) {
            arrows++;
            currentArrowPos = points[i][1];
        }
    }

    return arrows;
}
```

实现时还有个细节值得顺手留意：排序比较器不要直接写成 `a[1] - b[1]`，因为这题的坐标范围很大，直接相减有溢出的风险，写成 `Integer.compare(a[1], b[1])` 更稳。

时间复杂度是 `O(n log n)`，主要花在排序上，后面的扫描只有 `O(n)`；空间复杂度主要取决于排序实现本身。

这题可以顺手记住一个很常见的区间贪心套路：当目标是用尽量少的选择去覆盖尽量多的区间时，优先按结束位置排序，再把当前决策压在最早结束区间的终点上，往往就是正确方向。像活动安排、区间覆盖这类题里，这个判断会反复出现。

完整代码如下：

```java
import java.util.Arrays;

public class Main {

    public static int findMinArrowShots(int[][] points) {
        if (points.length == 0) {
            return 0;
        }

        Arrays.sort(points, (a, b) -> Integer.compare(a[1], b[1]));

        int arrows = 1;
        int currentArrowPos = points[0][1];

        for (int i = 1; i < points.length; i++) {
            if (points[i][0] > currentArrowPos) {
                arrows++;
                currentArrowPos = points[i][1];
            }
        }

        return arrows;
    }

    public static void main(String[] args) {
        int[][] points1 = {{10, 16}, {2, 8}, {1, 6}, {7, 12}};
        System.out.println("示例 1 输入: " + Arrays.deepToString(points1));
        int res1 = findMinArrowShots(points1);
        System.out.println("示例 1 输出: " + res1);
        System.out.println("-----------------------------");

        int[][] points2 = {{1, 2}, {3, 4}, {5, 6}, {7, 8}};
        System.out.println("示例 2 输入: " + Arrays.deepToString(points2));
        int res2 = findMinArrowShots(points2);
        System.out.println("示例 2 输出: " + res2);
        System.out.println("-----------------------------");

        int[][] points3 = {{1, 2}, {2, 3}, {3, 4}, {4, 5}};
        System.out.println("示例 3 输入: " + Arrays.deepToString(points3));
        int res3 = findMinArrowShots(points3);
        System.out.println("示例 3 输出: " + res3);
        System.out.println("-----------------------------");

        int[][] points4 = {{-2147483646, -2147483645}, {2147483646, 2147483647}};
        System.out.println("示例 4 输入: " + Arrays.deepToString(points4));
        int res4 = findMinArrowShots(points4);
        System.out.println("示例 4 输出: " + res4);
    }
}
```
