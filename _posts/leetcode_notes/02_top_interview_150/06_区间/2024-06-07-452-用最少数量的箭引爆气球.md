## 题目

**描述：**

有一些球形气球贴在一堵用 XY 平面表示的墙面上。墙面上的气球记录在整数数组 `points` ，其中`points[i] = [x_start, x_end]` 表示水平直径在 `x_start` 和 `x_end` 之间的气球。你不知道气球的精确 y 坐标。

一支弓箭可以沿着 x 轴从不同点 **完全垂直** 地射出。在坐标 `x` 处射出一支箭，若有一个气球的直径的开始和结束坐标为 `x_start`，`x_end`， 且满足 `x_start ≤ x ≤ x_end`，则该气球会被引爆。可以射出的弓箭的数量 **没有限制** 。弓箭一旦被射出之后，可以无限地前进。

给你一个数组 `points` ，返回引爆所有气球所必须射出的 **最小** 弓箭数。

**示例 1:**

- **输入:** `points = [[10,16],[2,8],[1,6],[7,12]]`
- **输出:** 2
- **解释:** 气球可以用 2 支箭来爆破:
  - 在 x = 6 处射出箭，击破气球 和。
  - 在 x = 11 处发射箭，击破气球 和。

**示例 2:**

- **输入:** `points = [[1,2],[3,4],[5,6],[7,8]]`
- **输出:** 4
- **解释:** 每个气球需要射出一支箭，总共需要 4 支箭。

**示例 3:**

- **输入:** `points = [[1,2],[2,3],[3,4],[4,5]]`
- **输出:** 2
- **解释:** 气球可以用 2 支箭来爆破:
  - 在 x = 2 处发射箭，击破气球 和。
  - 在 x = 4 处射出箭，击破气球 和。

**提示:**

- `1 <= points.length <= 10^5`
- `points[i].length == 2`
- `-2^31 <= x_start < x_end <= 2^31 - 1`

## 题解

### 这题表面上是在射箭，本质上还是区间问题

每个气球都可以看成一个区间 `[x_start, x_end]`。  
如果一支箭射在位置 `x`，并且满足：

```text
x_start <= x <= x_end
```

那这个气球就会被引爆。

所以这题翻译一下，其实就是：

**用尽可能少的点，去覆盖所有区间。**

一旦换成这个视角，这题就不是模拟题了，而是一个很典型的区间贪心问题。

### 真正要想清楚的是：一支箭应该尽量射在哪里

假设已经决定要射出一支箭，那这支箭当然希望能顺便打掉尽可能多的气球。

问题是，射在哪最划算？

答案不是射在区间中间，也不是随便找个位置，而是：

**尽量射在当前能处理的最靠右的位置。**

因为只要这支箭还没超过某个气球的右边界，它就仍然能打中这个气球；而位置越靠右，它就越有机会顺便落进后面更多区间里。

这就是这题贪心策略的出发点。

### 为什么一定要按右边界排序

这题最关键的一步，是先把所有气球按**右边界从小到大排序**。

为什么不是按左边界？

因为对于“结束得最早”的那个气球，你已经没有太多选择了。  
如果想打中它，箭的位置就必须落在它的区间里；而为了不浪费这支箭，最好的办法就是把箭射在它的右边界上。

这样做有两个好处：

- 这支箭一定能打中当前这个最早结束的气球
- 同时它尽量靠右，更有机会覆盖后面那些也包含这个位置的气球

所以排序按右端点，不是习惯问题，而是因为贪心选择就是围绕“最早结束的区间”展开的。

### 排完序之后，整个过程就很简单了

先把区间按右边界升序排序。  
然后默认先射出第一支箭，位置就放在第一个气球的右边界上。

接下来从前往后扫：

- 如果当前气球的左边界 `<=` 这支箭的位置，说明它也能被这支箭引爆，不需要额外操作
- 如果当前气球的左边界 `>` 这支箭的位置，说明这支箭已经够不到它了，必须新射一支箭

而新箭最合理的位置，仍然是当前这个气球的右边界。

整个过程其实就是在不断做这个判断：

**当前气球还能不能被上一支箭顺手打掉。**

### 用示例 1 走一遍

看示例 1：

```text
points = [[10,16],[2,8],[1,6],[7,12]]
```

先按右边界排序，得到：

```text
[[1,6],[2,8],[7,12],[10,16]]
```

先处理第一个区间 `[1,6]`。  
为了打中它，同时尽量兼顾后面更多气球，第一支箭射在 `x = 6`。

再看 `[2,8]`：

- 它的左边界是 `2`
- `2 <= 6`，说明这支箭还能打中它

所以不用新箭。

再看 `[7,12]`：

- 它的左边界是 `7`
- `7 > 6`，说明前一支箭已经够不到它了

这时候必须补一支新箭，最合理的位置是这个区间的右边界 `12`。

最后看 `[10,16]`：

- `10 <= 12`，说明第二支箭还能打中它

所以最终只需要两支箭。

### 为什么这样不会漏掉更优解

这题贪心成立的关键，在于：  
当你面对当前右边界最小的区间时，这支箭无论如何都不能射得比它的右边界更靠右，否则这个区间就漏了。

既然如此，那把箭射在它的右边界，已经是能兼顾后面区间的最优选择。

你如果射得更靠左，虽然当前区间照样能打中，但只会让这支箭覆盖的范围变小，不会更优。  
所以每一步“把箭放在当前最早结束区间的右边界”都是合理的，而且不会妨碍整体最优。

这其实和很多经典区间贪心题是一个思路：  
**优先处理结束最早的那个，并把选择压到它的结束位置。**

### 代码

```java
public int findMinArrowShots(int[][] points) {
    if (points.length == 0) {
        return 0;
    }

    Arrays.sort(points, (a, b) -> Integer.compare(a[1], b[1]));

    int arrows = 1;
    int currentArrowPos = points[0][1];

    for (int i = 1; i < points.length; i++) {
        if (points[i][0] > currentArrowPos) {
            arrows++;
            currentArrowPos = points[i][1];
        }
    }

    return arrows;
}
```

时间复杂度是 $O(n \log n)$，主要花在排序上；后面的扫描只有 $O(n)$。  
空间复杂度主要取决于排序实现本身，算法思路上额外只用了几个变量。

### 这题有个容易忽略的小细节

排序时不要直接写：

```java
a[1] - b[1]
```

因为这题里的坐标范围很大，可能接近 `int` 的上下界，直接相减有溢出的风险。

更稳妥的写法是：

```java
Integer.compare(a[1], b[1])
```

这一点在区间题里很常见，尤其是坐标范围开到 `10^9` 甚至 `2^31 - 1` 附近时，要养成这个习惯。

### 这题真正值得记住的地方

这题最值得记住的，不只是“按右边界排序”这个结论，而是背后的贪心判断：

**当你想用最少的选择去覆盖最多的区间时，往往应该优先考虑结束最早的那个区间，并把当前选择压在它的结束位置上。**

这个思路在很多区间贪心题里都会反复出现。  
形式上可能是：

- 射箭打气球
- 选点覆盖区间
- 选尽量少的会议室/资源
- 安排最多的不冲突活动

但底层判断常常是一样的：  
**先看结束最早的，再尽量把当前决策卡在它的结束点。**

把这个感觉练出来，后面很多区间贪心题会顺很多。

### 完整代码

```java
import java.util.Arrays;

public class Main {

    public static int findMinArrowShots(int[][] points) {
        if (points.length == 0) {
            return 0;
        }

        Arrays.sort(points, (a, b) -> Integer.compare(a[1], b[1]));

        int arrows = 1;
        int currentArrowPos = points[0][1];

        for (int i = 1; i < points.length; i++) {
            if (points[i][0] > currentArrowPos) {
                arrows++;
                currentArrowPos = points[i][1];
            }
        }

        return arrows;
    }

    public static void main(String[] args) {
        int[][] points1 = {{10, 16}, {2, 8}, {1, 6}, {7, 12}};
        System.out.println("示例 1 输入: " + Arrays.deepToString(points1));
        int res1 = findMinArrowShots(points1);
        System.out.println("示例 1 输出: " + res1);
        System.out.println("-----------------------------");

        int[][] points2 = {{1, 2}, {3, 4}, {5, 6}, {7, 8}};
        System.out.println("示例 2 输入: " + Arrays.deepToString(points2));
        int res2 = findMinArrowShots(points2);
        System.out.println("示例 2 输出: " + res2);
        System.out.println("-----------------------------");

        int[][] points3 = {{1, 2}, {2, 3}, {3, 4}, {4, 5}};
        System.out.println("示例 3 输入: " + Arrays.deepToString(points3));
        int res3 = findMinArrowShots(points3);
        System.out.println("示例 3 输出: " + res3);
        System.out.println("-----------------------------");

        int[][] points4 = {{-2147483646, -2147483645}, {2147483646, 2147483647}};
        System.out.println("示例 4 输入: " + Arrays.deepToString(points4));
        int res4 = findMinArrowShots(points4);
        System.out.println("示例 4 输出: " + res4);
    }
}
```
