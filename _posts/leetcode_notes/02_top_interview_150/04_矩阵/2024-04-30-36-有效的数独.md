## 题目

**描述：**

请你判断一个 `9 x 9` 的数独是否有效。只需要 **根据以下规则** ，验证已经填入的数字是否有效即可。

1.  数字 `1-9` 在每一行只能出现一次。
2.  数字 `1-9` 在每一列只能出现一次。
3.  数字 `1-9` 在每一个以粗实线分隔的 `3x3` 宫内只能出现一次。(请参考示例图)

**注意：**

- 一个有效的数独（部分已被填充）不一定是可解的。
- 只需要根据以上规则，验证已经填入的数字是否有效即可。
- 空白格用 `'.'` 表示。

**示例 1:**

- **输入:** `board` (一个有效的 9x9 数独板)
- **输出:** `true`

**示例 2:**

- **输入:** `board` (一个无效的 9x9 数独板，左上角 3x3 宫内有两个 8)
- **输出:** `false`

**提示：**

- `board.length == 9`
- `board[i].length == 9`
- `board[i][j]` 是一位数字（`1-9`）或者 `.`

## 题解

### 这题不是解数独，只是检查当前棋盘有没有冲突

很多人一看到“数独”，会下意识想到回溯、搜索、填格子。  
但这题其实没让你去解数独，它只要求判断：**当前这个 9×9 棋盘，已经填进去的数字是否合法。**

也就是说，我们根本不需要考虑：

- 这个数独最后能不能解出来
- 空位该填什么
- 要不要尝试回溯

题目只关心一件事：

**已经出现的数字，有没有在行、列、或者 3×3 宫里重复。**

把这一点想清楚之后，这题就简单很多了。

### 合法性的判断，其实只有三条

对于棋盘上的每一个已填数字，只需要检查它是否违反下面三条规则：

- 同一行里不能重复
- 同一列里不能重复
- 同一个 3×3 宫里不能重复

只要有任意一条被破坏，这个数独就是无效的。

所以这题本质上不是“推理题”，而是一个很直接的**状态记录题**。

### 最自然的做法，就是一边遍历，一边记录

既然我们只是想知道某个数字之前有没有出现过，那最直接的方法，就是准备三个记录表：

- `rows[i][num]`：表示第 `i` 行里，数字 `num` 是否出现过
- `cols[j][num]`：表示第 `j` 列里，数字 `num` 是否出现过
- `boxes[k][num]`：表示第 `k` 个 3×3 宫里，数字 `num` 是否出现过

然后从左到右、从上到下把整个棋盘扫一遍。

遇到 `'.'`，说明这个位置是空的，直接跳过。  
遇到数字，就去查这三个记录表：

- 如果行里已经出现过，返回 `false`
- 如果列里已经出现过，返回 `false`
- 如果宫里已经出现过，返回 `false`

如果三个地方都没出现过，就把它标记进去，继续往下扫。

整个过程其实很朴素：  
**看到一个数，就检查它会不会和之前的数撞车。**

### 这题最容易卡住的地方，是 3×3 宫怎么编号

行和列都好理解，真正让人停一下的是：  
怎么知道当前位置 `(i, j)` 属于哪个 3×3 宫？

这里可以直接记公式：

```text
boxIndex = (i / 3) * 3 + (j / 3)
```

为什么这样算？

因为：

- `i / 3` 决定它在第几排宫
- `j / 3` 决定它在第几列宫

整个棋盘一共 9 个宫，可以按下面这样编号：

```text
0 1 2
3 4 5
6 7 8
```

比如：

- `(0, 0)` 在左上角宫，对应 `0`
- `(1, 5)` 在第一排第三个宫，对应 `1`
- `(4, 7)` 在中间偏右那个宫，对应 `5`

这个公式本质上就是把二维位置映射成一个 0 到 8 的编号。

### 用一个格子走一遍

假设当前扫到位置 `(0, 1)`，上面的字符是 `'3'`。

那我们会先把它转换成数字下标。  
因为数组下标从 0 开始，所以通常把：

- `'1'` 映射成 `0`
- `'2'` 映射成 `1`
- ...
- `'9'` 映射成 `8`

也就是：

```java
int num = board[i][j] - '1';
```

然后计算它所在的宫：

```java
int boxIndex = (i / 3) * 3 + j / 3;
```

接下来检查：

- `rows[i][num]`
- `cols[j][num]`
- `boxes[boxIndex][num]`

只要其中有一个已经是 `true`，说明重复了，直接返回 `false`。  
否则就把这三个位置都标成 `true`，表示这个数字已经在对应的行、列、宫里出现过。

### 为什么一次遍历就够了

因为我们并不需要统计次数，也不需要把冲突的具体位置全部找出来。  
题目只问有效还是无效，所以一旦发现重复，立刻返回 `false` 就行。

如果从头到尾都没撞上任何重复，最后返回 `true`。

这也是为什么这题写出来会很短：  
它没有什么搜索过程，本质就是固定规则下的扫描和判重。

### 代码

```java
public boolean isValidSudoku(char[][] board) {
    boolean[][] rows = new boolean[9][9];
    boolean[][] cols = new boolean[9][9];
    boolean[][] boxes = new boolean[9][9];

    for (int i = 0; i < 9; i++) {
        for (int j = 0; j < 9; j++) {
            if (board[i][j] == '.') {
                continue;
            }

            int num = board[i][j] - '1';
            int boxIndex = (i / 3) * 3 + j / 3;

            if (rows[i][num] || cols[j][num] || boxes[boxIndex][num]) {
                return false;
            }

            rows[i][num] = true;
            cols[j][num] = true;
            boxes[boxIndex][num] = true;
        }
    }

    return true;
}
```

如果把数独大小视为固定的 9×9，那么时间复杂度就是 $O(1)$，因为总共只会检查 81 个格子。  
如果把它抽象成一般的 `N × N` 棋盘，那时间复杂度就是 $O(N^2)$。

空间复杂度同样是常数级。因为这里用到的三个辅助数组大小固定，不会随着输入继续增长。

### 这题真正值得记住的地方

这题本身不难，关键是别一上来把它想复杂了。

它不是回溯题，也不是搜索题，更不是让你去“求解一个数独”。  
它只是一个很典型的**边遍历、边判重**的问题。

以后遇到这种题，只要规则本质上是在检查“某个元素在几个维度上是否重复出现”，就可以优先考虑这种写法：

- 为每个维度准备一个记录表
- 扫描输入
- 发现重复就立刻返回

把问题从“怎么推出来”改成“怎么记录状态”，题就会简单很多。

### 完整代码

```java
public class Main {

    public static boolean isValidSudoku(char[][] board) {
        boolean[][] rows = new boolean[9][9];
        boolean[][] cols = new boolean[9][9];
        boolean[][] boxes = new boolean[9][9];

        for (int i = 0; i < 9; i++) {
            for (int j = 0; j < 9; j++) {
                char c = board[i][j];

                if (c == '.') {
                    continue;
                }

                int num = c - '1';
                int boxIndex = (i / 3) * 3 + j / 3;

                if (rows[i][num] || cols[j][num] || boxes[boxIndex][num]) {
                    return false;
                }

                rows[i][num] = true;
                cols[j][num] = true;
                boxes[boxIndex][num] = true;
            }
        }

        return true;
    }

    public static void main(String[] args) {
        char[][] board1 = {
            {'5','3','.','.','7','.','.','.','.'},
            {'6','.','.','1','9','5','.','.','.'},
            {'.','9','8','.','.','.','.','6','.'},
            {'8','.','.','.','6','.','.','.','3'},
            {'4','.','.','8','.','3','.','.','1'},
            {'7','.','.','.','2','.','.','.','6'},
            {'.','6','.','.','.','.','2','8','.'},
            {'.','.','.','4','1','9','.','.','5'},
            {'.','.','.','.','8','.','.','7','9'}
        };

        System.out.println("示例 1 输出: " + isValidSudoku(board1));
        System.out.println("-----------------------------");

        char[][] board2 = {
            {'5','3','.','.','7','.','.','.','5'},
            {'6','.','.','1','9','5','.','.','.'},
            {'.','9','8','.','.','.','.','6','.'},
            {'8','.','.','.','6','.','.','.','3'},

            {'.','.','.','4','1','9','.','.','5'},
            {'.','.','.','.','8','.','.','7','9'}
        };

        System.out.println("示例 2 输出: " + isValidSudoku(board2));
    }
}
```
