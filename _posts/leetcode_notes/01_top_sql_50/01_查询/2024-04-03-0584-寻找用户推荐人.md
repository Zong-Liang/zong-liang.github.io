## 题目：0584. 寻找用户推荐人

**表:** `Customer`

| Column Name | Type    |
| ----------- | ------- |
| id          | int     |
| name        | varchar |
| referee_id  | int     |

- 在 SQL 中, `id` 是该表的主键列。
- 该表的每一行表示一个客户的 `id`、姓名以及推荐他们的客户的 `id`。

**问题描述:**

找出以下客户的姓名：

1. 被任何 `id != 2` 的用户推荐。
2. 没有被任何用户推荐。

以 **任意顺序** 返回结果表。

**示例 1:**

**输入:**

`Customer` 表:

| id  | name | referee_id |
| --- | ---- | ---------- |
| 1   | Will | null       |
| 2   | Jane | null       |
| 3   | Alex | 2          |
| 4   | Bill | null       |
| 5   | Zack | 1          |
| 6   | Mark | 2          |

**输出:**

| name |
| ---- |
| Will |
| Jane |
| Bill |
| Zack |

## 题解

这题的意思其实很直接：找出**不是被 `id = 2` 的用户推荐**的人，同时**没有推荐人的用户也要保留**。

换成筛选条件来看，就是只排除 `referee_id = 2` 的记录，其余都要留下。

这里有一个很容易忽略的点：`referee_id` 这一列可能是 `NULL`。  
在 SQL 里，`NULL` 不能直接当作普通值去比较，所以如果只写 `referee_id != 2`，那么那些 `referee_id` 为 `NULL` 的用户不会被查出来。

因此，这题要写成下面这样：

```sql
SELECT name
FROM Customer
WHERE referee_id != 2 OR referee_id IS NULL;
```

这条语句可以分成两部分理解：

- `referee_id != 2`：筛出推荐人不是 `2` 的用户；
- `referee_id IS NULL`：筛出没有推荐人的用户。

两部分用 `OR` 连起来，刚好就是题目要求的结果。

结合示例也很好理解：

- `Will` 的 `referee_id` 是 `NULL`，要保留；
- `Jane` 的 `referee_id` 是 `NULL`，要保留；
- `Alex` 的 `referee_id` 是 `2`，要排除；
- `Zack` 的 `referee_id` 是 `1`，要保留。

所以最终返回的就是：

- `Will`
- `Jane`
- `Bill`
- `Zack`

这题本身不难，关键就在于 **`NULL` 的处理**。  
以后遇到“某个字段不等于某个值，但这个字段又可能为 `NULL`”这类题时，要多留意一下：**不能只写 `!=`，往往还要补上 `IS NULL`。**
