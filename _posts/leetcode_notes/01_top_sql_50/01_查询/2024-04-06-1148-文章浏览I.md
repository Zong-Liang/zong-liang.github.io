## 题目：1148. 文章浏览 I

**表:** `Views`

| Column Name | Type |
| ----------- | ---- |
| article_id  | int  |
| author_id   | int  |
| viewer_id   | int  |
| view_date   | date |

- 此表可能会存在重复行。(换句话说，在 SQL 中这个表没有主键)
- 此表的每一行都表示某人在某天浏览了某位作者的某篇文章。
- 请注意，同一人的 `author_id` 和 `viewer_id` 是相同的。

**问题描述:**

请查询出所有浏览过自己文章的作者。

结果按照作者的 `id` **升序排列**。

**示例 1:**

**输入:**

`Views` 表:

| article_id | author_id | viewer_id | view_date  |
| ---------- | --------- | --------- | ---------- |
| 1          | 3         | 5         | 2019-08-01 |
| 1          | 3         | 6         | 2019-08-02 |
| 2          | 7         | 7         | 2019-08-01 |
| 2          | 7         | 6         | 2019-08-02 |
| 4          | 7         | 1         | 2019-07-22 |
| 3          | 4         | 4         | 2019-07-21 |
| 3          | 4         | 4         | 2019-07-21 |

**输出:**

| id  |
| --- |
| 4   |
| 7   |

## 题解

这题先抓住一句话：**浏览过自己文章的作者**。

放到表里来看，就是一条浏览记录里：

- `author_id` 表示文章作者；
- `viewer_id` 表示浏览者。

如果一个作者看了自己的文章，那么这一行里就一定满足：

```sql
author_id = viewer_id
```

所以这题的第一步，就是把这些记录筛出来。

不过题目最后要的不是浏览记录本身，而是**作者的 id 列表**。  
再结合题目给的信息，这张表**可能有重复行**，也就是说同一个作者浏览自己文章的记录可能会出现多次，因此结果里要去重，这里就要用 `DISTINCT`。

完整 SQL 如下：

```sql
SELECT DISTINCT author_id AS id
FROM Views
WHERE author_id = viewer_id
ORDER BY id ASC;
```

这条语句可以这样理解：

- `WHERE author_id = viewer_id`：先找出作者看了自己文章的记录；
- `SELECT DISTINCT author_id AS id`：只保留作者编号，并且去重；
- `ORDER BY id ASC`：按照题目要求，按作者 id 升序排列。

结合示例数据也能很快看出来：

- `(2, 7, 7, 2019-08-01)` 这一行里，`author_id` 和 `viewer_id` 都是 `7`，说明作者 `7` 看过自己的文章；
- `(3, 4, 4, 2019-07-21)` 这一行里，`author_id` 和 `viewer_id` 都是 `4`，说明作者 `4` 也看过自己的文章；
- 而且作者 `4` 这一行在表里出现了两次，所以如果不写 `DISTINCT`，结果里就会重复。

因此最后返回的就是：

- `4`
- `7`

这题本质上考的有两个点：

- 会不会把题意准确翻译成条件 `author_id = viewer_id`；
- 看到“表中可能有重复行”时，能不能意识到结果可能需要去重。
