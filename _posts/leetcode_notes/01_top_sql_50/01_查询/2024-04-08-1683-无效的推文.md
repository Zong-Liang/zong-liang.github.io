## 题目：1683. 无效的推文

**表:** `Tweets`

| Column Name | Type    |
| ----------- | ------- |
| tweet_id    | int     |
| content     | varchar |

- 在 SQL 中, `tweet_id` 是这个表的主键。
- `content` 只包含字母数字字符, `' '`, `'!'`, `','` , 不包含其它特殊字符。
- 这个表包含某社交媒体 App 中所有的推文。

**问题描述:**

查询所有无效推文的编号 (ID)。当推文内容中的字符数 **严格大于** 15 时，该推文是无效的。

以 **任意顺序** 返回结果表。

**示例 1:**

**输入:**

`Tweets` 表:

| tweet_id | content                          |
| -------- | -------------------------------- |
| 1        | Vote for Biden                   |
| 2        | Let us make America great again! |

**输出:**

| tweet_id |
| -------- |
| 2        |

**解释:**

- 推文 1 的长度 length = 14。该推文是有效的。
- 推文 2 的长度 length = 32。该推文是无效的。

## 题解

这题非常直接，题目已经把“无效推文”的判定标准说清楚了：**内容长度严格大于 `15`**。

所以这题本质上就是做一件事：  
先计算每条推文 `content` 的长度，再把长度大于 `15` 的记录筛出来。

对应的 SQL 写法如下：

```sql
SELECT tweet_id
FROM Tweets
WHERE LENGTH(content) > 15;
```

这条语句可以这样理解：

- `LENGTH(content)`：计算推文内容的长度；
- `> 15`：只保留长度严格大于 `15` 的推文；
- `SELECT tweet_id`：题目只要求返回无效推文的编号。

结合示例来看：

- `"Vote for Biden"` 的长度是 `14`，没有超过 `15`，所以是有效推文；
- `"Let us make America great again!"` 的长度是 `32`，超过了 `15`，所以是无效推文。

因此最终返回它的 `tweet_id` 即可。

这题考点比较单一，重点就是看到“按字符串长度筛选”时，能够立刻想到用字符串长度函数。  
在这题里直接用 `LENGTH(content)` 就可以解决。
