## 题目：0595. 大的国家

**表:** `World`

| Column Name | Type    |
| ----------- | ------- |
| name        | varchar |
| continent   | varchar |
| area        | int     |
| population  | int     |
| gdp         | bigint  |

- `name` 是该表的主键（具有唯一值的列）。
- 这张表的每一行提供：国家名称、所属大陆、面积、人口和 GDP 值。

**问题描述:**

如果一个国家满足下述两个条件之一，则认为该国是 **大国** ：

- 面积至少为 300 万平方公里（即，3000000 km²），或者
- 人口至少为 2500 万（即 25000000）

编写解决方案找出 **大国** 的国家名称、人口和面积。

按 **任意顺序** 返回结果表。

**示例 1:**

**输入:**

`World` 表:

| name        | continent | area    | population | gdp          |
| ----------- | --------- | ------- | ---------- | ------------ |
| Afghanistan | Asia      | 652230  | 25500100   | 20343000000  |
| Albania     | Europe    | 28748   | 2831741    | 12960000000  |
| Algeria     | Africa    | 2381741 | 37100000   | 188681000000 |
| Andorra     | Europe    | 468     | 78115      | 3712000000   |
| Angola      | Africa    | 1246700 | 20609294   | 100990000000 |

**输出:**

| name        | population | area    |
| ----------- | ---------- | ------- |
| Afghanistan | 25500100   | 652230  |
| Algeria     | 37100000   | 2381741 |

## 题解

这题本身比较直接，关键是先把“大国”的定义翻译成查询条件。

题目说，一个国家只要满足下面两个条件中的任意一个，就算大国：

- 面积不少于 `3000000`；
- 或者人口不少于 `25000000`。

也就是说，这题不是要求两个条件同时满足，而是**满足任意一个就可以**，所以这里要用 `OR` 来连接。

对应的 SQL 写法如下：

```sql
SELECT name,
       population,
       area
FROM World
WHERE area >= 3000000 OR population >= 25000000;
```

这条语句可以直接这样理解：

- `SELECT name, population, area`：题目只要求返回国家名称、人口和面积这三列；
- `FROM World`：数据来自 `World` 表；
- `WHERE area >= 3000000 OR population >= 25000000`：筛选出面积达到标准，或者人口达到标准的国家。

结合示例数据来看：

- `Afghanistan` 的面积没有达到 `3000000`，但人口超过了 `25000000`，所以要保留；
- `Algeria` 的面积虽然没有达到 `3000000`，但人口超过了 `25000000`，也要保留；
- `Albania` 的面积和人口都没达到要求，所以不保留。

这题考的其实就是对条件的理解。  
看到题目里出现“满足以下条件之一”这种表述时，基本就要想到用 `OR`；如果题目写的是“同时满足”，那才是用 `AND`。
