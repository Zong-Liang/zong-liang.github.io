## 题目：1731. 每位经理的下属员工数量

**表:** `Employees`

| Column Name | Type    |
| ----------- | ------- |
| employee_id | int     |
| name        | varchar |
| reports_to  | int     |
| age         | int     |

- `employee_id` 是这个表中具有不同值的列。
- 该表包含员工以及需要听取他们汇报的上级经理的 ID 的信息。有些员工不需要向任何人汇报 (`reports_to` 为空)。

**问题描述:**

对于此问题，我们将至少有一个其他员工需要向他汇报的员工，视为一个经理。

编写一个解决方案来返回需要听取汇报的所有经理的 ID、名称、直接向该经理汇报的员工人数，以及这些员工的平均年龄，其中该平均年龄需要四舍五入到最接近的整数。

返回的结果集需要按照 `employee_id` 进行排序。

**示例 1:**

**输入:**

`Employees` 表:

| employee_id | name    | reports_to | age |
| ----------- | ------- | ---------- | --- |
| 9           | Hercy   | null       | 43  |
| 6           | Alice   | 9          | 41  |
| 4           | Bob     | 9          | 36  |
| 2           | Winston | null       | 37  |

**输出:**

| employee_id | name  | reports_count | average_age |
| ----------- | ----- | ------------- | ----------- |
| 9           | Hercy | 2             | 39          |

**解释:**

Hercy 有两个需要向他汇报的员工，他们是 Alice and Bob。他们的平均年龄是 (41+36)/2 = 38.5，四舍五入的结果是 39。

**示例 2:**

**输入:**

`Employees` 表:

| employee_id | name    | reports_to | age |
| ----------- | ------- | ---------- | --- |
| 1           | Michael | null       | 45  |
| 2           | Alice   | 1          | 38  |
| 3           | Bob     | 1          | 42  |
| 4           | Charlie | 2          | 34  |
| 5           | David   | 2          | 40  |
| 6           | Eve     | 3          | 37  |
| 7           | Frank   | null       | 50  |
| 8           | Grace   | null       | 48  |

**输出:**

| employee_id | name    | reports_count | average_age |
| ----------- | ------- | ------------- | ----------- |
| 1           | Michael | 2             | 40          |
| 2           | Alice   | 2             | 37          |
| 3           | Bob     | 1             | 37          |

## 题解

这题表面上是在统计经理的信息，实际上真正要做的是：

**先把“员工”和“他的直属经理”对应起来，再按经理分组统计。**

题目要求返回的是每位经理的：

- `employee_id`
- `name`
- 直属下属人数
- 直属下属的平均年龄（四舍五入）

这里最关键的一点是要看明白 `reports_to` 这列的含义。  
对于某个员工来说，`reports_to` 记录的是他直属经理的 `employee_id`。  
所以如果要找到“谁是谁的下属”，最自然的办法就是把 `Employees` 表和自己做一次自连接：

- 一份表示员工；
- 另一份表示经理。

当员工表里的 `reports_to` 等于经理表里的 `employee_id` 时，这两个人就对应上了。

对应的 SQL 写法如下：

```sql
SELECT m.employee_id,
       m.name,
       COUNT(e.employee_id) AS reports_count,
       ROUND(AVG(e.age)) AS average_age
FROM Employees e
JOIN Employees m
  ON e.reports_to = m.employee_id
GROUP BY m.employee_id, m.name
ORDER BY m.employee_id;
```

这段查询可以顺着思路分成三步理解。

第一步，先把员工和经理对应起来：

```sql
FROM Employees e
JOIN Employees m
  ON e.reports_to = m.employee_id
```

这里虽然只用了一张表，但它在查询里扮演了两个角色：

- `e` 表示普通员工；
- `m` 表示经理。

连接条件 `e.reports_to = m.employee_id` 的意思是：  
把每个员工和他的直属经理配对起来。

比如如果某个员工的 `reports_to = 9`，那么他就会和 `employee_id = 9` 的那一行连在一起。

第二步，按经理分组：

```sql
GROUP BY m.employee_id, m.name
```

因为我们最终要的是“每位经理”的统计结果，所以要以经理为单位分组。  
这样同一个经理手下的所有员工，就会被放到同一组里，后面才能做人数统计和平均值计算。

第三步，分别算出每位经理的下属人数和平均年龄：

```sql
COUNT(e.employee_id) AS reports_count,
ROUND(AVG(e.age)) AS average_age
```

这里的意思也很直接：

- `COUNT(e.employee_id)`：统计这个经理手下有多少个直属下属；
- `AVG(e.age)`：计算这些直属下属的平均年龄；
- `ROUND(...)`：按照题目要求，把平均年龄四舍五入成整数。

结合示例 1 来看：

- Hercy 的 `employee_id` 是 `9`；
- Alice 和 Bob 的 `reports_to` 都是 `9`；

所以在连接之后，Alice 和 Bob 都会和 Hercy 对应起来。  
这样按 Hercy 分组时：

- 下属人数是 `2`；
- 平均年龄是 `(41 + 36) / 2 = 38.5`；
- 四舍五入后得到 `39`。

所以结果就是：

- `9, Hercy, 2, 39`

这题本质上考的是**自连接 + 分组聚合**。  
只要题目里出现“表中一部分行和同一张表里的另一部分行存在上下级、父子、推荐、关联”这种关系，通常都可以先考虑自连接。连起来之后，再配合 `GROUP BY` 和聚合函数去做统计，思路就会比较顺。
