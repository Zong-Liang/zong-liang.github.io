## 题目：1204. 最后一个能进入巴士的人

**表:** `Queue`

| Column Name | Type    |
| ----------- | ------- |
| person_id   | int     |
| person_name | varchar |
| weight      | int     |
| turn        | int     |

- `person_id` 是这个表具有唯一值的列。
- 该表展示了所有候车乘客的信息。
- 表中 `person_id` 和 `turn` 列将包含从 1 到 n 的所有数字，其中 n 是表中的行数。
- `turn` 决定了候车乘客上巴士的顺序，其中 `turn=1` 表示第一个上巴士，`turn=n` 表示最后一个上巴士。
- `weight` 表示候车乘客的体重，以千克为单位。

**问题描述:**

有一队乘客在等着上巴士。然而，巴士有 **1000 千克** 的重量限制，所以其中一部分乘客可能无法上巴士。

编写解决方案找出 **最后一个** 上巴士且不超過重量限制的乘客，并报告 `person_name`。题目测试用例确保顺位第一的人可以上巴士且巴士不会超重。

**示例 1:**

**输入:**

`Queue` 表:

| person_id | person_name | weight | turn |
| --------- | ----------- | ------ | ---- |
| 5         | Alice       | 250    | 1    |
| 3         | Alex        | 350    | 2    |
| 6         | John Cena   | 400    | 3    |
| 2         | Marie       | 200    | 4    |
| 1         | Winston     | 500    | 6    |
| 4         | Bob         | 175    | 5    |

**输出:**

| person_name |
| ----------- |
| John Cena   |

**解释:**

按 `turn` 排序后的上车顺序和累计重量如下：

- 1. Alice: 250 (累计: 250)
- 2. Alex: 350 (累计: 600)
- 3. John Cena: 400 (累计: 1000)
- 4. Marie: 200 (累计: 1200 - 超重)

因此，John Cena 是最后一个能上车的人。

## 题解

这题的关键不是找“谁最后上车”，而是先想清楚：

**每个人上车之后，车上的总重量是多少。**

因为乘客必须按照 `turn` 的顺序依次上车，所以只要我们能算出每个人上车时的**累计重量**，问题就变得很直接了：

- 如果某个人上车后的累计重量 `<= 1000`，说明他还能上车；
- 如果某个人上车后的累计重量 `> 1000`，说明从他开始就超重了，不能再上车。

这样一来，答案其实就是：

**所有累计重量不超过 `1000` 的乘客里，`turn` 最大的那个人。**

对应的 SQL 写法如下：

```sql
SELECT q.person_name
FROM (
    SELECT person_name,
           turn,
           SUM(weight) OVER (ORDER BY turn) AS cumulative_weight
    FROM Queue
) AS q
WHERE q.cumulative_weight <= 1000
ORDER BY q.turn DESC
LIMIT 1;
```

这段查询可以分成两步理解。

第一步，先算出每个人上车时的累计重量：

```sql
SELECT person_name,
       turn,
       SUM(weight) OVER (ORDER BY turn) AS cumulative_weight
FROM Queue
```

这里的关键是窗口函数：

```sql
SUM(weight) OVER (ORDER BY turn)
```

它会按照 `turn` 的顺序，把前面所有已经上车的乘客体重加起来，得到当前这位乘客上车后的总重量。

也就是说：

- 第 1 个人的累计重量，就是他自己的体重；
- 第 2 个人的累计重量，就是前 2 个人体重之和；
- 第 3 个人的累计重量，就是前 3 个人体重之和；
- 依次类推。

第二步，再从这些人里找出最后一个没有超重的人：

```sql
WHERE q.cumulative_weight <= 1000
ORDER BY q.turn DESC
LIMIT 1
```

这部分的意思是：

- 先筛出所有累计重量不超过 `1000` 的乘客，也就是所有真正能上车的人；
- 再按 `turn` 从大到小排序；
- 最后取第一条记录，也就是最后一个能上车的人。

结合示例来看：

- Alice 上车后累计重量是 `250`；
- Alex 上车后累计重量是 `600`；
- John Cena 上车后累计重量是 `1000`；
- Marie 上车后累计重量是 `1200`，已经超重了。

所以真正能上车的人只有前 3 个，其中 `turn` 最大的是 John Cena，因此答案就是他。

这题本质上考的是**累计和**。  
只要题目里出现“按顺序处理”“到当前位置为止的总和”“前面所有记录累加起来”这种描述，就要想到窗口函数里的 `SUM() OVER (ORDER BY ...)`。这种写法在 SQL 里非常常见，而且比自己手动做自连接或者子查询更直接。
