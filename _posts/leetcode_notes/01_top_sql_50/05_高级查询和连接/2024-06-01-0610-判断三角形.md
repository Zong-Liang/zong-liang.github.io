## 题目：0610. 判断三角形

**表:** `Triangle`

| Column Name | Type |
| ----------- | ---- |
| x           | int  |
| y           | int  |
| z           | int  |

- 在 SQL 中, `(x, y, z)` 是该表的主键列。
- 该表的每一行包含三个线段的长度。

**问题描述:**

对每三个线段报告它们是否可以形成一个三角形。

以 **任意顺序** 返回结果表。

**示例 1:**

**输入:**

`Triangle` 表:

| x   | y   | z   |
| --- | --- | --- |
| 13  | 15  | 30  |
| 10  | 20  | 15  |

**输出:**

| x   | y   | z   | triangle |
| --- | --- | --- | -------- |
| 13  | 15  | 30  | No       |
| 10  | 20  | 15  | Yes      |

## 题解

这题本质上是在问：表中的每一组 `x`、`y`、`z`，到底能不能组成三角形。

判断方法其实是固定的，不需要分类讨论，直接用三角形的判定条件就行：

**任意两边之和都要大于第三边。**

也就是说，下面三个条件必须同时成立：

- `x + y > z`
- `x + z > y`
- `y + z > x`

只要有一个条件不满足，这三条边就不能组成三角形。

题目要求的是对表中的每一行都给出判断结果，所以这里最自然的写法就是用 `CASE WHEN`：条件成立时返回 `'Yes'`，否则返回 `'No'`。

对应的 SQL 如下：

```sql
SELECT x,
       y,
       z,
       CASE
           WHEN x + y > z AND x + z > y AND y + z > x THEN 'Yes'
           ELSE 'No'
       END AS triangle
FROM Triangle;
```

这段代码的思路很直接：

先把原表中的 `x`、`y`、`z` 查出来，然后额外生成一列 `triangle`，专门用来表示这一组边长能不能构成三角形。

其中最关键的就是这一句：

```sql
WHEN x + y > z AND x + z > y AND y + z > x THEN 'Yes'
```

因为三角形判定要求三个条件同时满足，所以这里要用 `AND` 连接，而不是只判断其中一个。

结合示例来看：

- `13, 15, 30` 这一组里，`13 + 15 = 28`，并没有大于 `30`，所以不能构成三角形，结果是 `'No'`。
- `10, 20, 15` 这一组里，任意两边之和都大于第三边，所以可以构成三角形，结果是 `'Yes'`。

所以这题其实没有什么 SQL 技巧，重点就是把数学判断准确翻译成查询条件。

以后遇到这种“对每一行做条件判断，并返回不同结果”的题，基本都可以优先想到 `CASE WHEN`。这类写法在 SQL 里很常见，适合把规则直接变成结果列。
