## 题目：1789. 员工的直属部门

**表:** `Employee`

| Column Name   | Type    |
| ------------- | ------- |
| employee_id   | int     |
| department_id | int     |
| primary_flag  | varchar |

- 这张表的主键为 `employee_id, department_id`。
- `employee_id` 是员工的 ID。
- `department_id` 是部门的 ID，表示员工与该部门有关系。
- `primary_flag` 是一个枚举类型，值分别为 ('Y', 'N')。如果值为'Y'，表示该部门是员工的直属部门。如果值是'N'，则否。

**问题描述:**

一个员工可以属于多个部门。当一个员工加入 **超过一个部门** 的时候，他需要决定哪个部门是他的直属部门。请注意，当员工只加入一个部门的时候，那个这个部门将默认为他的直属部门，虽然表记录的值为 'N'。

请编写解决方案，查出员工所属的直属部门。

返回结果 **没有顺序要求**。

**示例 1:**

**输入:**

`Employee` table:

| employee_id | department_id | primary_flag |
| ----------- | ------------- | ------------ |
| 1           | 1             | N            |
| 2           | 1             | Y            |
| 2           | 2             | N            |
| 3           | 3             | N            |
| 4           | 2             | N            |
| 4           | 3             | Y            |
| 4           | 4             | N            |

**输出:**

| employee_id | department_id |
| ----------- | ------------- |
| 1           | 1             |
| 2           | 1             |
| 3           | 3             |
| 4           | 3             |

**解释:**

- 员工 1 的直属部门是 1 (因为他只在一个部门)。
- 员工 2 的直属部门是 1 (因为 `primary_flag` = 'Y')。
- 员工 3 的直属部门是 3 (因为他只在一个部门)。
- 员工 4 的直属部门是 3 (因为 `primary_flag` = 'Y')。

## 题解

这题看起来是在找“直属部门”，实际上关键是先分清楚题目里的两种情况：

- **员工属于多个部门**：这时真正的直属部门就是 `primary_flag = 'Y'` 的那个部门；
- **员工只属于一个部门**：这时就算 `primary_flag = 'N'`，这个唯一的部门也要当作直属部门。

所以这题并不是靠一个条件就能一次性筛出来，而是要把这两种情况分别处理，再把结果合并起来。

对应的 SQL 写法如下：

```sql
SELECT employee_id,
       department_id
FROM Employee
WHERE primary_flag = 'Y'

UNION

SELECT employee_id,
       department_id
FROM Employee
GROUP BY employee_id
HAVING COUNT(department_id) = 1;
```

这段查询的思路其实很直接。

先看第一部分：

```sql
SELECT employee_id,
       department_id
FROM Employee
WHERE primary_flag = 'Y'
```

它负责处理“员工属于多个部门”的情况。  
因为题目已经明确说了：如果员工加入了多个部门，就会从中选一个直属部门，并把这一行的 `primary_flag` 标记成 `'Y'`。

所以这里直接把 `primary_flag = 'Y'` 的记录找出来，就能得到这部分员工的直属部门。

再看第二部分：

```sql
SELECT employee_id,
       department_id
FROM Employee
GROUP BY employee_id
HAVING COUNT(department_id) = 1;
```

它负责处理“员工只属于一个部门”的情况。

这里要注意题目的一个隐藏点：  
有些员工虽然只有一个部门，但表里的 `primary_flag` 不一定会写成 `'Y'`，可能仍然是 `'N'`。  
所以这类员工不能只靠 `WHERE primary_flag = 'Y'` 找出来，必须单独判断。

`GROUP BY employee_id` 是按员工分组，  
`HAVING COUNT(department_id) = 1` 则表示：这个员工只出现在一个部门里。

既然他只属于一个部门，那么这个部门自然就是直属部门。

最后用 `UNION` 把这两部分结果合并起来：

```sql
UNION
```

这样就能同时覆盖：

- 多部门员工中被标记为 `'Y'` 的直属部门；
- 单部门员工默认的直属部门。

结合示例来看：

- 员工 `1` 只在部门 `1`，所以直属部门就是 `1`；
- 员工 `2` 在多个部门里，`primary_flag = 'Y'` 的是部门 `1`，所以直属部门是 `1`；
- 员工 `3` 只在部门 `3`，所以直属部门是 `3`；
- 员工 `4` 在多个部门里，`primary_flag = 'Y'` 的是部门 `3`，所以直属部门是 `3`。

所以最后结果就是：

- `1 -> 1`
- `2 -> 1`
- `3 -> 3`
- `4 -> 3`

这题本质上考的是**分类讨论**。  
当题目里出现“满足条件 A 的按一种规则处理，不满足 A 的按另一种规则处理”这种结构时，SQL 往往就可以拆成两部分分别查询，再用 `UNION` 合并。这样写通常比硬凑成一个复杂条件更清晰。
