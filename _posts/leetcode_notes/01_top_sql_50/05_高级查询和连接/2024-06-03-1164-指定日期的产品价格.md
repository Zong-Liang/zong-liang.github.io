## 题目：1164. 指定日期的产品价格

**产品数据表:** `Products`

| Column Name | Type |
| ----------- | ---- |
| product_id  | int  |
| new_price   | int  |
| change_date | date |

- `(product_id, change_date)` 是此表的主键。
- 这张表的每一行分别记录了 某产品 在某个日期 更改后 的新价格。
- 一开始，所有产品价格都为 10。

**问题描述:**

编写一个解决方案，找出在 **2019-08-16** 所有产品的价格。

以 **任意顺序** 返回结果表。

**示例 1:**

**输入:**

`Products` 表:

| product_id | new_price | change_date |
| ---------- | --------- | ----------- |
| 1          | 20        | 2019-08-14  |
| 2          | 50        | 2019-08-14  |
| 1          | 30        | 2019-08-15  |
| 1          | 35        | 2019-08-16  |
| 2          | 65        | 2019-08-17  |
| 3          | 20        | 2019-08-18  |

**输出:**

| product_id | price |
| ---------- | ----- |
| 2          | 50    |
| 1          | 35    |
| 3          | 10    |

## 题解

这题看起来是在查某一天的产品价格，实际上真正要做的是：

**对每个产品，找到它在 `2019-08-16` 这一天之前最后一次生效的价格。**

因为表里记录的不是“每天的价格”，而是“价格发生变化时的新价格”，所以如果想知道某个产品在 `2019-08-16` 的价格，就不能只看当天有没有记录，而要看：

- 如果这个产品在 `2019-08-16` 之前改过价，那么取**最后一次改价后的价格**；
- 如果这个产品在 `2019-08-16` 之前从来没有改过价，那么它的价格就是题目给的**初始价格 10**。

所以这题的核心一共就两步：

1. 先找出每个产品在 `2019-08-16` 及之前的最后一次改价记录；
2. 再把那些在这一天之前从未改过价的产品，价格补成 `10`。

对应的 SQL 写法如下：

```sql
SELECT p.product_id,
       IFNULL(last_price.price, 10) AS price
FROM (SELECT DISTINCT product_id FROM Products) AS p
         LEFT JOIN
     (SELECT product_id,
             new_price AS price
      FROM Products
      WHERE (product_id, change_date) IN (SELECT product_id,
                                                 MAX(change_date)
                                          FROM Products
                                          WHERE change_date <= '2019-08-16'
                                          GROUP BY product_id)) AS last_price ON p.product_id = last_price.product_id;
```

可以顺着这个查询一步一步理解。

先看这一部分：

```sql
SELECT product_id,
       MAX(change_date)
FROM Products
WHERE change_date <= '2019-08-16'
GROUP BY product_id
```

它的作用是：  
对于每个产品，只保留在 `2019-08-16` 及之前**最后一次改价的日期**。

比如示例里：

- 产品 `1` 在 `2019-08-16` 前最后一次改价日期是 `2019-08-16`；
- 产品 `2` 在 `2019-08-16` 前最后一次改价日期是 `2019-08-14`；
- 产品 `3` 在 `2019-08-16` 前没有任何改价记录，所以这里不会出现它。

接着再看这一层：

```sql
SELECT product_id,
       new_price AS price
FROM Products
WHERE (product_id, change_date) IN (...)
```

这里的目的就是根据刚才找出来的 `(product_id, 最后改价日期)`，把对应的 `new_price` 取出来。  
也就是说，这一步真正拿到的是：**每个产品在目标日期之前最后一次生效的价格。**

然后主查询部分做的事情就比较简单了。

```sql
FROM (SELECT DISTINCT product_id FROM Products) AS p
LEFT JOIN ... AS last_price
ON p.product_id = last_price.product_id
```

这里先把所有产品的 `product_id` 都取出来，再去和刚才算出来的“最后有效价格”做左连接。

这样做的原因是：  
有些产品虽然在表里出现过，但在 `2019-08-16` 之前并没有改过价。  
如果不用 `LEFT JOIN`，这些产品就会直接丢掉；而题目要求的是返回**所有产品**在这一天的价格，所以必须把它们保留下来。

最后这一句是整题的收尾：

```sql
IFNULL(last_price.price, 10) AS price
```

意思是：

- 如果这个产品在 `2019-08-16` 之前有过价格记录，就用查到的那个价格；
- 如果没有，就说明它一直没改过价，此时用初始价格 `10`。

结合示例来看：

- 产品 `1` 在 `2019-08-16` 之前最后一次价格是 `35`，所以结果是 `35`；
- 产品 `2` 在 `2019-08-16` 之前最后一次价格是 `50`，所以结果是 `50`；
- 产品 `3` 到 `2019-08-16` 为止还没有改过价，所以结果是初始价格 `10`。

所以最后返回：

- `1 -> 35`
- `2 -> 50`
- `3 -> 10`

这题本质上考的是一种很典型的查询思路：

**先按条件找“每组最后一条记录”，再去补默认值。**

以后只要碰到“某个时刻的最新状态”“截止某天最后一次记录”“如果没有记录就给默认值”这类题，基本都可以往这个方向去想。
