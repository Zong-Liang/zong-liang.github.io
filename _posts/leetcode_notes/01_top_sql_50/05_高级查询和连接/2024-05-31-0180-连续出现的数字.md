## 题目：180. 连续出现的数字

**表:** `Logs`

| Column Name | Type    |
| ----------- | ------- |
| id          | int     |
| num         | varchar |

- 在 SQL 中, `id` 是该表的主键。
- `id` 是一个自增列。

**问题描述:**

找出所有至少连续出现三次的数字。

返回的结果表中的数据可以按 **任意顺序** 排列。

**示例 1:**

**输入:**

`Logs` 表:

| id  | num |
| --- | --- |
| 1   | 1   |
| 2   | 1   |
| 3   | 1   |
| 4   | 2   |
| 5   | 1   |
| 6   | 2   |
| 7   | 2   |

**输出:**

`Result` 表:

| ConsecutiveNums |
| --------------- |
| 1               |

**解释:** 1 是唯一连续出现至少三次的数字。

## 题解

这题的关键，不是怎么把三张表拼起来，而是先想清楚一句话：

**怎么判断一个数字连续出现了至少三次？**

最直接的办法，就是把相邻的三条记录拿出来一起看。  
如果这三条记录的 `id` 是连续的，并且 `num` 也都相同，那么这个数字就满足题意了。

题目里已经说明了 `id` 是主键，而且还是自增列，所以这里可以直接用 `id` 是否连续，来判断记录是不是连续出现的。

基于这个思路，我们可以把 `Logs` 表看成三个副本，分别表示连续的三行：

- `l1` 表示第 1 行；
- `l2` 表示紧挨着的下一行；
- `l3` 表示再下一行。

只要满足下面两个条件，就说明找到了一个连续出现至少三次的数字：

- 这三行的 `id` 是连续的；
- 这三行的 `num` 完全相同。

对应的 SQL 写法如下：

```sql
SELECT DISTINCT l1.num AS ConsecutiveNums
FROM Logs l1,
     Logs l2,
     Logs l3
WHERE l1.id = l2.id - 1
  AND l2.id = l3.id - 1
  AND l1.num = l2.num
  AND l2.num = l3.num;
```

这段代码可以这样理解：

`l1.id = l2.id - 1 AND l2.id = l3.id - 1`  
说明 `l1`、`l2`、`l3` 正好对应三条连续的记录。

`l1.num = l2.num AND l2.num = l3.num`  
说明这三条连续记录里的数字都一样。

这样一来，只要某个数字在表中连续出现了三次，它就会被筛出来。

这里还要注意一个细节：查询结果用了 `DISTINCT` 去重。  
因为如果一个数字连续出现了四次或者五次，那么在枚举连续三行的时候，它其实会被匹配到不止一次。

比如 `1` 连续出现 4 次：

- 第 1、2、3 行能匹配一次；
- 第 2、3、4 行还能再匹配一次。

但题目只需要返回这个数字本身，不需要重复返回，所以要加上 `DISTINCT`。

结合示例数据来看：

- `id = 1, 2, 3` 这三行的 `num` 都是 `1`，满足条件；
- 后面的数字虽然也有重复，但没有形成连续三次相同的情况，所以不会被选出来。

因此最终结果就是：

- `1`

这题本质上考的是**如何判断连续记录**。  
这类题的常见思路，就是把表和自己做自连接，用多个别名去表示“前一行、当前行、后一行”，再根据题意补上对应的相等关系或连续关系。
