## 题目：1907. 按分类统计薪水

**表:** `Accounts`

| 列名       | 类型 |
| ---------- | ---- |
| account_id | int  |
| income     | int  |

- 在 SQL 中, `account_id` 是这个表的主键。
- 每一行都包含一个银行帐户的月收入的信息。

**问题描述:**

查询每个工资类别的银行账户数量。工资类别如下：

- **"Low Salary"**: 所有工资 **严格低于** 20000 美元。
- **"Average Salary"**: **包含** 范围内的所有工资 `[$20000, $50000]`。
- **"High Salary"**: 所有工资 **严格大于** 50000 美元。

结果表 **必须** 包含所有三个类别。如果某个类别中没有帐户，则报告 0。

按 **任意顺序** 返回结果表。

**示例 1:**

**输入:**

`Accounts` 表:

| account_id | income |
| ---------- | ------ |
| 3          | 108939 |
| 2          | 12747  |
| 8          | 87709  |
| 6          | 91796  |

**输出:**

| category       | accounts_count |
| -------------- | -------------- |
| Low Salary     | 1              |
| Average Salary | 0              |
| High Salary    | 3              |

**解释:**

- 低薪：有一个账户 2。
- 中等薪水：没有。
- 高薪：有三个账户，他们是 3, 6 和 8。

## 题解

这题本身不难，关键是要先注意题目的一个要求：

**结果里必须包含三种工资类别，即使某一类一个账户都没有，也要返回这一行，并且数量写成 `0`。**

这就意味着，这题不能只对现有数据简单分组统计。  
因为如果某一类在表里根本没有数据，普通的 `GROUP BY` 很可能就不会返回这一类。

所以这里更直接的写法是：  
**把三种工资类别分别单独统计出来，再把结果拼到一起。**

对应的 SQL 如下：

```sql
SELECT 'Low Salary' AS category,
       COUNT(account_id) AS accounts_count
FROM Accounts
WHERE income < 20000

UNION ALL

SELECT 'Average Salary' AS category,
       COUNT(account_id) AS accounts_count
FROM Accounts
WHERE income BETWEEN 20000 AND 50000

UNION ALL

SELECT 'High Salary' AS category,
       COUNT(account_id) AS accounts_count
FROM Accounts
WHERE income > 50000;
```

这段查询其实就是把题目里的三种分类规则，原样翻译成了三个独立的统计查询。

第一部分统计低薪账户：

```sql
SELECT 'Low Salary' AS category,
       COUNT(account_id) AS accounts_count
FROM Accounts
WHERE income < 20000
```

这里的意思很直接：

- 类别名称固定写成 `'Low Salary'`；
- 只统计 `income < 20000` 的账户数量。

第二部分统计中等薪水账户：

```sql
SELECT 'Average Salary' AS category,
       COUNT(account_id) AS accounts_count
FROM Accounts
WHERE income BETWEEN 20000 AND 50000
```

这里要特别注意边界。  
题目明确说了中等薪水包含区间 `[20000, 50000]`，所以 `20000` 和 `50000` 都算在这一类里。  
`BETWEEN 20000 AND 50000` 正好就是包含两端的写法。

第三部分统计高薪账户：

```sql
SELECT 'High Salary' AS category,
       COUNT(account_id) AS accounts_count
FROM Accounts
WHERE income > 50000
```

也就是把所有严格大于 `50000` 的账户统计出来。

最后用 `UNION ALL` 把三部分结果合并：

```sql
UNION ALL
```

这里用 `UNION ALL` 而不是 `UNION`，原因也很简单：  
我们不是在做去重，而是明确要把这三行结果完整拼起来。每个查询本来就只会返回一行，所以直接用 `UNION ALL` 更合适。

这题最容易忽略的地方就在于：  
**不是“按现有分类统计”，而是“无论有没有数据，都必须把三类都返回出来”。**

比如示例里没有任何账户落在 `Average Salary` 这一类，  
但因为第二个查询本身仍然会执行，所以依然会返回：

- `Average Salary, 0`

这正好满足题意。

这题本质上考的是**按条件分类统计，并且要补全所有类别**。  
遇到这种“结果必须包含固定几类，即使某一类没有数据也要显示出来”的题，用多个独立查询配合 `UNION ALL`，通常会比直接 `GROUP BY` 更稳，也更容易写对。
