## 题目：0570. 至少有 5 名直接下属的经理

**表:** `Employee`

| Column Name | Type    |
| ----------- | ------- |
| id          | int     |
| name        | varchar |
| department  | varchar |
| managerId   | int     |

- `id` 是此表的主键（具有唯一值的列）。
- 该表的每一行表示雇员的名字、他们的部门和他们的经理的 id。
- 如果`managerId`为空，则该员工没有经理。
- 没有员工会成为自己的管理者。

**问题描述:**

编写一个解决方案，找出至少有五个 **直接下属** 的经理。

以 **任意顺序** 返回结果表。

**示例 1:**

**输入:**

`Employee` 表:

| id  | name  | department | managerId |
| --- | ----- | ---------- | --------- |
| 101 | John  | A          | Null      |
| 102 | Dan   | A          | 101       |
| 103 | James | A          | 101       |
| 104 | Amy   | A          | 101       |
| 105 | Anne  | A          | 101       |
| 106 | Ron   | B          | 101       |

**输出:**

| name |
| ---- |
| John |

## 题解

这题的关键在于先理解“至少有 5 名直接下属”这句话。

这里统计的不是一个经理名下所有层级的员工，而是只统计**直接汇报给他的人**。  
放到表里看，就是谁的 `managerId` 等于某个经理的 `id`，谁就是这位经理的直接下属。

所以这题可以分成两步来做：

- 先按 `managerId` 分组，统计每个经理有多少直接下属；
- 再把下属人数不少于 `5` 的经理找出来，返回他们的姓名。

对应的 SQL 写法如下：

```sql
SELECT name
FROM Employee
WHERE id IN (
    SELECT managerId
    FROM Employee
    WHERE managerId IS NOT NULL
    GROUP BY managerId
    HAVING COUNT(*) >= 5
);
```

这段 SQL 可以这样理解：

第一步，子查询先找出符合条件的经理编号：

```sql
SELECT managerId
FROM Employee
WHERE managerId IS NOT NULL
GROUP BY managerId
HAVING COUNT(*) >= 5
```

这里的意思是：

- `managerId` 相同的员工，说明他们都直接汇报给同一个经理；
- `GROUP BY managerId` 就是按经理分组；
- `HAVING COUNT(*) >= 5` 表示只保留下属数量不少于 `5` 的经理。

第二步，外层查询再根据这些经理编号，取出对应的姓名：

```sql
SELECT name
FROM Employee
WHERE id IN (...)
```

因为员工表里本身就有经理的信息，所以只要经理的 `id` 出现在刚才统计出来的结果里，就把他的 `name` 查出来即可。

结合示例来看：

- `Dan`、`James`、`Amy`、`Anne`、`Ron` 这五个人的 `managerId` 都是 `101`；
- 说明 `id = 101` 的员工有 `5` 个直接下属；
- 而 `id = 101` 对应的名字是 `John`。

所以最终返回：

- `John`

这题本质上考的是 **分组统计**。  
只要题目里出现“每个经理有多少人”“每个用户有多少条记录”“每个商品出现多少次”这种问法，通常就要往 `GROUP BY` 和 `HAVING` 这个方向去想。
