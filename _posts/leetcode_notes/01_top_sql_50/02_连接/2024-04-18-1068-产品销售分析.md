## 题目：1068. 产品销售分析 I

**销售表 `Sales`:**

| Column Name | Type |
| ----------- | ---- |
| sale_id     | int  |
| product_id  | int  |
| year        | int  |
| quantity    | int  |
| price       | int  |

- `(sale_id, year)` 是销售表 `Sales` 的主键。
- `product_id` 是关联到产品表 `Product` 的外键。
- 该表的每一行显示 `product_id` 在某一年的一些销售情况。
- 注意: `price` 表示每单位价格。

**产品表 `Product`:**

| Column Name  | Type    |
| ------------ | ------- |
| product_id   | int     |
| product_name | varchar |

- `product_id` 是表的主键。
- 该表的每一行表示每种产品的产品名称。

**问题描述:**

编写解决方案，以获取 `Sales` 表中所有 `sale_id` 对应的 `product_name` 以及该产品的所有 `year` 和 `price`。

返回结果表 **无顺序要求**。

**示例 1:**

**输入:**

`Sales` 表:

| sale_id | product_id | year | quantity | price |
| ------- | ---------- | ---- | -------- | ----- |
| 1       | 100        | 2008 | 10       | 5000  |
| 2       | 100        | 2009 | 12       | 5000  |
| 7       | 200        | 2011 | 15       | 9000  |

`Product` 表:

| product_id | product_name |
| ---------- | ------------ |
| 100        | Nokia        |
| 200        | Apple        |
| 300        | Samsung      |

**输出:**

| product_name | year | price |
| ------------ | ---- | ----- |
| Nokia        | 2008 | 5000  |
| Nokia        | 2009 | 5000  |
| Apple        | 2011 | 9000  |

## 题解

这题的目标很明确：`Sales` 表里已经有产品的销量信息、年份和价格，但没有产品名称；产品名称在 `Product` 表里。

所以这题本质上就是：**根据 `product_id`，把销售记录和对应的产品名称拼起来。**

也就是说，每一条销售记录都要去 `Product` 表里找到同一个 `product_id` 对应的 `product_name`，然后再把题目要求的三列取出来。

这种题就是最典型的**两表连接**。

```sql
SELECT p.product_name,
       s.year,
       s.price
FROM Sales s
JOIN Product p
    ON s.product_id = p.product_id;
```

这条 SQL 可以这样理解：

- `Sales s`：先从销售表出发，因为题目要的是每一条销售记录对应的信息；
- `JOIN Product p ON s.product_id = p.product_id`：根据 `product_id` 把销售记录和产品表关联起来；
- `SELECT p.product_name, s.year, s.price`：最后取出题目要求返回的产品名称、年份和价格。

结合示例来看：

- `sale_id = 1` 的 `product_id = 100`，去 `Product` 表中可以找到它对应的产品名称是 `Nokia`，所以这一行结果就是 `Nokia, 2008, 5000`；
- `sale_id = 2` 的 `product_id = 100`，对应的还是 `Nokia`，所以结果是 `Nokia, 2009, 5000`；
- `sale_id = 7` 的 `product_id = 200`，对应产品名称是 `Apple`，所以结果是 `Apple, 2011, 9000`。

因此最终返回的就是题目给出的结果。

这题考点比较基础，重点就是看到：

- 一张表里有编号；
- 另一张表里有这个编号对应的详细信息；

这时候通常就要想到用 `JOIN` 按照关联字段把两张表连起来。
