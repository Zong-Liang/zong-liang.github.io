## 题目：1581. 进店却未进行过交易的顾客

**表:** `Visits`

| Column Name | Type |
| ----------- | ---- |
| visit_id    | int  |
| customer_id | int  |

- `visit_id` 是该表中具有唯一值的列。
- 该表包含有关光临过购物中心的顾客的信息。

**表:** `Transactions`

| Column Name    | Type |
| -------------- | ---- |
| transaction_id | int  |
| visit_id       | int  |
| amount         | int  |

- `transaction_id` 是该表中具有唯一值的列。
- 此表包含 `visit_id` 期间进行的交易的信息。

**问题描述:**

有一些顾客可能光顾了购物中心但没有进行交易。请你编写一个解决方案，来查找这些顾客的 ID，以及他们只光顾不交易的次数。

返回以 **任意顺序** 排序的结果表。

**示例 1:**

**输入:**

`Visits` 表:

| visit_id | customer_id |
| -------- | ----------- |
| 1        | 23          |
| 2        | 9           |
| 4        | 30          |
| 5        | 54          |
| 6        | 96          |
| 7        | 54          |
| 8        | 54          |

`Transactions` 表:

| transaction_id | visit_id | amount |
| -------------- | -------- | ------ |
| 2              | 5        | 310    |
| 3              | 5        | 300    |
| 9              | 5        | 200    |
| 12             | 1        | 910    |
| 13             | 2        | 970    |

**输出:**

| customer_id | count_no_trans |
| ----------- | -------------- |
| 54          | 2              |
| 30          | 1              |
| 96          | 1              |

**解释:**

- ID = 23 的顾客曾经逛过一次购物中心，并在 ID = 12 的访问期间进行了一笔交易。
- ID = 30 的顾客曾经去过购物中心，并且没有进行任何交易。
- ID = 54 的顾客三度造访了购物中心。在 2 次访问中，他们没有进行任何交易，在 1 次访问中，他们进行了 3 次交易。
- ID = 96 的顾客曾经去过购物中心，并且没有进行任何交易。
- 如我们所见，ID 为 30 和 96 的顾客一次没有进行任何交易就去了购物中心。顾客 54 也两次访问了购物中心并且没有进行任何交易。

## 题解

这题最关键的地方，不是统计次数，而是先判断：**哪一次到店记录没有对应交易。**

题目要找的是“进店了，但没有发生任何交易”的顾客，而且最后还要统计这种情况一共出现了多少次。  
所以这题不能直接从 `Transactions` 表出发，因为没有交易的记录根本不会出现在这张表里。

正确做法应该是：

- 先以 `Visits` 表为主表，保留所有到店记录；
- 再去 `Transactions` 表里看这次到店有没有对应交易；
- 如果没有对应交易，就把这次访问记下来；
- 最后按 `customer_id` 分组统计次数。

这类题最常见的写法就是 `LEFT JOIN`。

```sql
SELECT v.customer_id,
       COUNT(v.visit_id) AS count_no_trans
FROM Visits v
LEFT JOIN Transactions t
       ON v.visit_id = t.visit_id
WHERE t.transaction_id IS NULL
GROUP BY v.customer_id;
```

这条 SQL 可以按顺序这样理解。

先看连接部分：

```sql
FROM Visits v
LEFT JOIN Transactions t
       ON v.visit_id = t.visit_id
```

这里的意思是：把每一次到店记录都保留下来，再根据 `visit_id` 去交易表里找有没有对应的交易。

如果找到了，说明这次访问发生过交易；  
如果没找到，那么连接后的 `t.transaction_id` 就会是 `NULL`，这正好表示“这次进店没有交易”。

接着看筛选条件：

```sql
WHERE t.transaction_id IS NULL
```

这一步就是把“没有交易的访问记录”筛出来。

然后再按顾客分组：

```sql
GROUP BY v.customer_id
```

因为题目最后要的不是某一次访问，而是每个顾客“只进店不交易”的次数，所以要按顾客编号汇总。

最后这一句：

```sql
COUNT(v.visit_id) AS count_no_trans
```

统计的就是每个顾客符合条件的访问次数，也就是题目要的结果。

结合示例来看：

- 顾客 `30` 到店一次，没有任何交易，所以计数是 `1`；
- 顾客 `96` 到店一次，也没有交易，所以计数是 `1`；
- 顾客 `54` 一共来了三次，其中一次有交易，两次没有交易，所以计数是 `2`；
- 顾客 `23` 虽然来过，但那次访问有交易，所以不出现在结果中。

因此最终返回的就是这些“只进店不交易”的顾客，以及他们对应的次数。

这题本质上考的是一类很典型的思路：

- 要找“左表有、右表没有”的数据时，优先考虑 `LEFT JOIN`；
- 再配合 `WHERE 右表某列 IS NULL` 把没有匹配成功的记录筛出来；
- 最后如果题目还要求统计次数，再加上 `GROUP BY` 和 `COUNT`。

这个套路在连接题里非常常见。
