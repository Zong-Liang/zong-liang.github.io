## 题目：0577. 员工奖金

**表:** `Employee`

| Column Name | Type    |
| ----------- | ------- |
| empId       | int     |
| name        | varchar |
| supervisor  | int     |
| salary      | int     |

- `empId` 是该表中具有唯一值的列。
- 该表的每一行都表示员工的 id 和姓名，以及他们经理的 id 和他们的工资。

**表:** `Bonus`

| Column Name | Type |
| ----------- | ---- |
| empId       | int  |
| bonus       | int  |

- `empId` 是该表具有唯一值的列。
- `empId` 是 `Employee` 表中 `empId` 的外键(reference 列)。
- 该表的每一行都包含一个员工的 id 和他们各自的奖金。

**问题描述:**

编写解决方案，报告每个奖金 **少于** 1000 的员工的姓名和奖金数额。

以 **任意顺序** 返回结果表。

**示例 1:**

**输入:**

`Employee` table:

| empId | name   | supervisor | salary |
| ----- | ------ | ---------- | ------ |
| 3     | Brad   | null       | 4000   |
| 1     | John   | 3          | 1000   |
| 2     | Dan    | 3          | 2000   |
| 4     | Thomas | 3          | 4000   |

`Bonus` table:

| empId | bonus |
| ----- | ----- |
| 2     | 500   |
| 4     | 2000  |

**输出:**

| name | bonus |
| ---- | ----- |
| Brad | null  |
| John | null  |
| Dan  | 500   |

## 题解

这题表面上是在查“奖金少于 `1000` 的员工”，但真正要注意的是示例里把**没有奖金记录的员工也查出来了**。

也就是说，题目要求保留两类人：

- 奖金小于 `1000` 的员工；
- 没有奖金记录的员工。

这就说明不能直接只查 `Bonus` 表，因为那样会把没有奖金记录的人漏掉。  
正确做法是以 `Employee` 表为主表，再去关联 `Bonus` 表，这样所有员工都会先被保留下来。

所以这里要用 `LEFT JOIN`。

```sql
SELECT e.name,
       b.bonus
FROM Employee e
LEFT JOIN Bonus b
    ON e.empId = b.empId
WHERE b.bonus < 1000 OR b.bonus IS NULL;
```

这条 SQL 可以分成两步理解。

第一步，先把员工表和奖金表连起来：

```sql
FROM Employee e
LEFT JOIN Bonus b
    ON e.empId = b.empId
```

这里用 `LEFT JOIN` 的原因是：

- `Employee` 表里的员工都要保留；
- 如果某个员工在 `Bonus` 表里没有对应记录，那么连接之后他的 `bonus` 就会是 `NULL`。

第二步，再筛选出符合要求的人：

```sql
WHERE b.bonus < 1000 OR b.bonus IS NULL
```

这里分成两种情况：

- `b.bonus < 1000`：奖金存在，而且小于 `1000`；
- `b.bonus IS NULL`：奖金不存在，也就是没有奖金记录。

这两个条件用 `OR` 连起来，就正好对应题目的要求。

结合示例来看：

- `Dan` 的奖金是 `500`，小于 `1000`，所以要保留；
- `Brad` 和 `John` 在 `Bonus` 表里没有记录，连接后奖金是 `NULL`，所以也要保留；
- `Thomas` 的奖金是 `2000`，不符合条件，所以不保留。

因此最终结果是：

- `Brad  null`
- `John  null`
- `Dan   500`

这题真正考的不是条件判断本身，而是**什么时候该用 `LEFT JOIN`**。  
只要题目里有“主表中的数据都要保留，即使另一张表里没有对应记录”这种意思，就要优先考虑 `LEFT JOIN`。
