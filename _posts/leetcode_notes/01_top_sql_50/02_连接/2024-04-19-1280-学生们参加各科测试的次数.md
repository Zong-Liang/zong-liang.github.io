## 题目：1280. 学生们参加各科测试的次数

**表结构:**

`Students`

| Column Name  | Type    |
| ------------ | ------- |
| student_id   | int     |
| student_name | varchar |

- `student_id` 是主键。
- 该表的每一行都记录有学校一名学生的信息。

`Subjects`

| Column Name  | Type    |
| ------------ | ------- |
| subject_name | varchar |

- `subject_name` 是主键。
- 每一行记录学校的一门科目名称。

`Examinations`

| Column Name  | Type    |
| ------------ | ------- |
| student_id   | int     |
| subject_name | varchar |

- 该表没有主键，可能包含重复数据。
- 学生表里的一个学生修读科目表里的一门科目。
- 这张考试表的每一行记录就表示学生表里的某个学生参加了一次科目表里某门科目的测试。

**问题描述:**

查询出每个学生参加每一门科目测试的次数，结果按 `student_id` 和 `subject_name` 排序。

**示例 1:**

**输入:**

`Students` table:

| student_id | student_name |
| ---------- | ------------ |
| 1          | Alice        |
| 2          | Bob          |
| 13         | John         |
| 6          | Alex         |

`Subjects` table:

| subject_name |
| ------------ |
| Math         |
| Physics      |
| Programming  |

`Examinations` table:

| student_id | subject_name |
| ---------- | ------------ |
| 1          | Math         |
| 1          | Physics      |
| 1          | Programming  |
| 2          | Programming  |
| 1          | Physics      |
| 1          | Math         |
| 13         | Math         |
| 13         | Programming  |
| 13         | Physics      |
| 2          | Math         |
| 1          | Math         |

**输出:**

| student_id | student_name | subject_name | attended_exams |
| ---------- | ------------ | ------------ | -------------- |
| 1          | Alice        | Math         | 3              |
| 1          | Alice        | Physics      | 2              |
| 1          | Alice        | Programming  | 1              |
| 2          | Bob          | Math         | 1              |
| 2          | Bob          | Physics      | 0              |
| 2          | Bob          | Programming  | 1              |
| 6          | Alex         | Math         | 0              |
| 6          | Alex         | Physics      | 0              |
| 6          | Alex         | Programming  | 0              |
| 13         | John         | Math         | 1              |
| 13         | John         | Physics      | 1              |
| 13         | John         | Programming  | 1              |

**解释:**

结果表需包含所有学生和所有科目（即便测试次数为 0）。

## 题解

这题最容易漏掉的一点，不是怎么计数，而是**结果必须包含每个学生和每一门科目的所有组合**，即使某个学生从来没参加过某门科目的考试，也要把这一行显示出来，并且次数记为 `0`。

所以这题不能直接从 `Examinations` 表出发统计。  
如果直接按考试表分组，那么那些从未参加过某门课考试的组合根本不会出现，结果就不完整了。

正确思路要分成两步：

- 先把“所有学生”和“所有科目”配成完整组合；
- 再去考试表里统计每个组合出现了多少次。

第一步要用 `CROSS JOIN`，第二步要用 `LEFT JOIN`。

```sql
SELECT st.student_id,
       st.student_name,
       su.subject_name,
       COUNT(ex.subject_name) AS attended_exams
FROM Students st
CROSS JOIN Subjects su
LEFT JOIN Examinations ex
       ON st.student_id = ex.student_id
      AND su.subject_name = ex.subject_name
GROUP BY st.student_id,
         st.student_name,
         su.subject_name
ORDER BY st.student_id,
         su.subject_name;
```

这条 SQL 可以按顺序这样理解。

先看这一句：

```sql
FROM Students st
CROSS JOIN Subjects su
```

这里会生成每个学生和每门科目的所有组合。  
比如有 4 个学生、3 门科目，就会先得到 `4 × 3 = 12` 行。这样就保证了结果不会漏掉任何一个“学生 - 科目”的组合。

接着再看：

```sql
LEFT JOIN Examinations ex
       ON st.student_id = ex.student_id
      AND su.subject_name = ex.subject_name
```

这一步是把刚才生成的每个组合，去 `Examinations` 表里找对应的考试记录。

- 如果找到了，说明这个学生参加过这门课的考试；
- 如果没找到，这一行依然会保留，只是 `ex` 表对应的字段会是 `NULL`。

这就是为什么这里必须用 `LEFT JOIN`，因为题目要求连考试次数为 `0` 的组合也要展示出来。

然后是计数：

```sql
COUNT(ex.subject_name) AS attended_exams
```

这里不能随便写成别的列，核心在于：`COUNT(列名)` 只统计非 `NULL` 的值。

所以：

- 如果某个学生参加过某门课 3 次，就会统计出 `3`；
- 如果一次都没参加，对应连接结果里 `ex.subject_name` 全是 `NULL`，`COUNT` 就会得到 `0`。

最后再按题目要求排序：

```sql
ORDER BY st.student_id,
         su.subject_name
```

结合示例来看：

- `Alice` 参加了 `Math` 3 次，所以是 `3`；
- `Bob` 没参加过 `Physics`，但这一行 still 要保留，所以是 `0`；
- `Alex` 没参加过任何考试，因此他和每门科目的组合都要出现，次数都是 `0`。

这题真正考的是一个很典型的套路：

- 题目要求“所有组合都要出现”时，先考虑用 `CROSS JOIN` 补全组合；
- 题目要求“没有匹配记录也要保留”时，再考虑用 `LEFT JOIN`；
- 最后再配合 `GROUP BY` 和 `COUNT` 做统计。

这类题一旦看懂这个思路，后面会很常见。
