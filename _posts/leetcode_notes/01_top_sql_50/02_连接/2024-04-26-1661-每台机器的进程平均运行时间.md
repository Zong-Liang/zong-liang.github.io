## 题目：1661. 每台机器的进程平均运行时间

**表:** `Activity`

| Column Name   | Type                 |
| ------------- | -------------------- |
| machine_id    | int                  |
| process_id    | int                  |
| activity_type | enum('start', 'end') |
| timestamp     | float                |

- `(machine_id, process_id, activity_type)` 是当前表的主键。
- `machine_id` 是一台机器的 ID 号。
- `process_id` 是运行在各机器上的进程 ID 号。
- `activity_type` 是枚举类型 ('start', 'end')。
- `timestamp` 是浮点类型, 代表当前时间(以秒为单位)。
- 'start' 代表该进程在这台机器上的开始运行时间戳, 'end' 代表该进程在这台机器上的终止运行时间戳。
- 同一台机器, 同一个进程都有一个开始时间戳和结束时间戳, 而且开始时间戳永远在结束时间戳前面。

**问题描述:**

现在有一个工厂网站由几台机器运行，每台机器上运行着相同数量的进程。编写解决方案，计算每台机器各自完成一个进程任务的平均耗时。

完成一个进程任务的时间指进程的 'end' 时间戳减去 'start' 时间戳。平均耗时通过计算每台机器上所有进程任务的总耗费时间除以机器上的总进程数量获得。

结果表必须必须包含 `machine_id` (机器 ID) 和对应的 `average time` (平均耗时) 别名 `processing_time`，且**四舍五入保留 3 位小数**。

以 **任意顺序** 返回结果表。

**示例 1:**

**输入:**

`Activity` table:

| machine_id | process_id | activity_type | timestamp |
| ---------- | ---------- | ------------- | --------- |
| 0          | 0          | start         | 0.712     |
| 0          | 0          | end           | 1.520     |
| 0          | 1          | start         | 3.140     |
| 0          | 1          | end           | 4.120     |
| 1          | 0          | start         | 0.550     |
| 1          | 0          | end           | 1.550     |
| 1          | 1          | start         | 0.430     |
| 1          | 1          | end           | 1.420     |
| 2          | 0          | start         | 4.100     |
| 2          | 0          | end           | 4.512     |
| 2          | 1          | start         | 2.500     |
| 2          | 1          | end           | 5.000     |

**输出:**

| machine_id | processing_time |
| ---------- | --------------- |
| 0          | 0.894           |
| 1          | 0.995           |
| 2          | 1.456           |

**解释:**

- 机器 0 的平均耗时: `((1.520 - 0.712) + (4.120 - 3.140)) / 2 = 0.894`
- 机器 1 的平均耗时: `((1.550 - 0.550) + (1.420 - 0.430)) / 2 = 0.995`
- 机器 2 的平均耗时: `((4.512 - 4.100) + (5.000 - 2.500)) / 2 = 1.456`

## 题解

这题看起来是在求“每台机器的平均运行时间”，但真正要先想清楚的是：**单个进程的运行时间怎么得到。**

题目已经告诉我们，一个进程的运行时间等于：

```sql
end - start
```

而表里每个进程都会有两条记录：

- 一条是 `activity_type = 'start'`
- 一条是 `activity_type = 'end'`

所以这题本质上就是先把同一台机器上、同一个进程的开始时间和结束时间合并起来，算出每个进程的耗时；然后再按 `machine_id` 分组，求平均值。

这里有一种很巧的写法，不需要先单独把 `start` 和 `end` 连接起来，而是直接利用条件求和来完成计算。

```sql
SELECT machine_id,
       ROUND(
           SUM(CASE
                   WHEN activity_type = 'end' THEN timestamp
                   ELSE -timestamp
               END) / COUNT(DISTINCT process_id),
           3
       ) AS processing_time
FROM Activity
GROUP BY machine_id;
```

这条 SQL 的核心在这一段：

```sql
SUM(CASE
        WHEN activity_type = 'end' THEN timestamp
        ELSE -timestamp
    END)
```

它的意思是：

- 如果这一行是 `end`，就把时间戳当正数加进去；
- 如果这一行是 `start`，就把时间戳当负数加进去。

这样对于同一个进程来说：

```sql
end + (-start) = end - start
```

也就是这个进程的运行时间。

把同一台机器上所有进程都这样加起来，就得到了这台机器的总运行时间。

接着再看这一句：

```sql
COUNT(DISTINCT process_id)
```

它表示统计这台机器上一共有多少个不同的进程。  
总运行时间除以进程数，自然就是平均运行时间。

最后再用：

```sql
ROUND(..., 3)
```

把结果四舍五入保留三位小数，这也是题目明确要求的。

结合示例来看：

- 机器 `0` 有两个进程，耗时分别是 `1.520 - 0.712 = 0.808` 和 `4.120 - 3.140 = 0.980`；
- 平均值就是 `(0.808 + 0.980) / 2 = 0.894`；
- 其他机器也是同样的计算方式。

所以最终返回每台机器的 `machine_id` 和对应的平均耗时即可。

这题的关键不只是分组求平均，更重要的是看出：  
**同一个进程的运行时间，可以通过 `end` 记正、`start` 记负，再用 `SUM` 一次性算出来。**

这种写法在聚合题里很常见，本质上就是一种**条件聚合**思路。以后遇到“同一组数据里，不同类型的记录要分别加减处理”时，也可以往这个方向想。
