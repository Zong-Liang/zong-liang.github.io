## 题目：197. 上升的温度

**表:** `Weather`

| Column Name | Type |
| ----------- | ---- |
| id          | int  |
| recordDate  | date |
| temperature | int  |

- `id` 是该表具有唯一值的列。
- 没有具有相同 `recordDate` 的不同行。
- 该表包含特定日期的温度信息。

**问题描述:**

编写解决方案，找出与之前（昨天）日期相比温度更高的所有日期的 `id`。

返回结果 **无顺序要求**。

**示例 1:**

**输入:**

`Weather` 表:

| id  | recordDate | Temperature |
| --- | ---------- | ----------- |
| 1   | 2015-01-01 | 10          |
| 2   | 2015-01-02 | 25          |
| 3   | 2015-01-03 | 20          |
| 4   | 2015-01-04 | 30          |

**输出:**

| id  |
| --- |
| 2   |
| 4   |

**解释:**

- 2015-01-02 的温度比前一天高 (10 -> 25)
- 2015-01-04 的温度比前一天高 (20 -> 30)

## 题解

这题的关键不是单独看某一天的温度，而是要把**当天和前一天放在一起比较**。

题目要求找出这样一些日期对应的 `id`：

- 这一天确实有前一天的数据；
- 并且当天温度比前一天更高。

这种“同一张表里的两行数据互相比较”的题，最常见的做法就是**自连接**。

我们可以把 `Weather` 表看成两个副本：

- `w1` 表示“今天”；
- `w2` 表示“昨天”。

然后先让它们按照“日期正好相差 1 天”匹配起来，再比较温度大小。

```sql
SELECT w1.id
FROM Weather w1
JOIN Weather w2
  ON DATEDIFF(w1.recordDate, w2.recordDate) = 1
WHERE w1.temperature > w2.temperature;
```

这条语句可以分成两步理解：

第一步，先找到连续两天的数据：

```sql
DATEDIFF(w1.recordDate, w2.recordDate) = 1
```

这表示 `w1` 的日期比 `w2` 晚一天，也就是 `w1` 是今天，`w2` 是昨天。

第二步，再判断今天是不是更热：

```sql
w1.temperature > w2.temperature
```

只有当天温度严格高于前一天，`w1.id` 才会被选出来。

结合示例来看：

- `2015-01-02` 和 `2015-01-01` 能配成连续两天，温度从 `10` 变成 `25`，升高了，所以返回 `id = 2`；
- `2015-01-03` 和 `2015-01-02` 也是连续两天，但温度从 `25` 变成 `20`，没有升高，所以不返回；
- `2015-01-04` 和 `2015-01-03` 能配成连续两天，温度从 `20` 变成 `30`，升高了，所以返回 `id = 4`。

因此最终结果是：

- `2`
- `4`

这题本质上考的是**自连接**。  
只要遇到“同一张表中，不同行之间要互相比较”的情况，就可以考虑把表起两个别名，当成两张表来用，再通过连接条件把它们对应起来。
