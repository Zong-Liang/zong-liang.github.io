## 题目：0585. 2016 年的投资

**表结构:** `Insurance`

| Column Name | Type  |
| ----------- | ----- |
| pid         | int   |
| tiv_2015    | float |
| tiv_2016    | float |
| lat         | float |
| lon         | float |

- `pid` 是这张表的主键。
- 表中的每一行都包含一条保险信息，其中：
  - `pid` 是投保人的投保编号。
  - `tiv_2015` 是该投保人在 2015 年的总投保金额。
  - `tiv_2016` 是该投保人在 2016 年的总投保金额。
  - `lat` 是投保人所在城市的纬度。题目数据确保 `lat` 不为空。
  - `lon` 是投保人所在城市的经度。题目数据确保 `lon` 不为空。

**问题描述:**

编写解决方案报告 2016 年 (`tiv_2016`) 所有满足下述条件的投保人的投保金额之和：

- 他在 2015 年的投保额 (`tiv_2015`) 至少跟一个其他投保人在 2015 年的投保额相同。
- 他所在的城市必须与其他投保人都不同（也就是说 `(lat, lon)` 不能跟其他任何一个投保人完全相同）。

`tiv_2016` 四舍五入的 **两位小数**。

**示例 1:**

**输入:**

`Insurance` 表:

| pid | tiv_2015 | tiv_2016 | lat | lon |
| --- | -------- | -------- | --- | --- |
| 1   | 10       | 5        | 10  | 10  |
| 2   | 20       | 20       | 20  | 20  |
| 3   | 10       | 30       | 20  | 20  |
| 4   | 10       | 40       | 40  | 40  |

**输出:**

| tiv_2016 |
| -------- |
| 45.00    |

**解释:**

- 表中的第一条记录和最后一条记录都满足两个条件。

  `tiv_2015` 值为 10 与第三条和第四条记录相同，且其位置是唯一的。

- 第二条记录不符合任何一个条件。其 `tiv_2015` 与其他投保人不同，并且位置与第三条记录相同，这也导致了第三条记录不符合题目要求。
- 因此，结果是第一条记录和最后一条记录的 `tiv_2016` 之和，即 45。

## 题解

这题真正麻烦的地方，不是求和，而是要先把题目的两个筛选条件拆清楚。

只有同时满足下面两件事的投保人，`tiv_2016` 才能被计入答案：

- 他的 `tiv_2015` 和至少另一个人相同；
- 他的地理位置 `(lat, lon)` 是唯一的，不能和其他人重复。

所以这题本质上不是直接算总和，而是：

**先筛出符合这两个条件的人，再把他们的 `tiv_2016` 加起来。**

对应的 SQL 写法如下：

```sql
SELECT ROUND(SUM(i.tiv_2016), 2) AS tiv_2016
FROM Insurance i
WHERE i.tiv_2015 IN (
          SELECT tiv_2015
          FROM Insurance
          GROUP BY tiv_2015
          HAVING COUNT(*) > 1
      )
  AND (i.lat, i.lon) IN (
          SELECT lat,
                 lon
          FROM Insurance
          GROUP BY lat, lon
          HAVING COUNT(*) = 1
      );
```

这段查询可以顺着题意分成两部分理解。

先看第一个条件：

```sql
i.tiv_2015 IN (
    SELECT tiv_2015
    FROM Insurance
    GROUP BY tiv_2015
    HAVING COUNT(*) > 1
)
```

这部分是在找：  
**哪些 `tiv_2015` 的值在表里出现了不止一次。**

因为题目要求的是“2015 年投保额至少和另一个人相同”，所以我们只需要把那些重复出现的 `tiv_2015` 找出来，再保留这些记录即可。

比如某个 `tiv_2015` 出现了 3 次，那么这 3 条记录在第一个条件上都算满足。

再看第二个条件：

```sql
(i.lat, i.lon) IN (
    SELECT lat,
           lon
    FROM Insurance
    GROUP BY lat, lon
    HAVING COUNT(*) = 1
)
```

这部分是在找：  
**哪些地理位置只出现过一次。**

题目要求某个投保人的城市必须和其他所有人都不同，也就是说 `(lat, lon)` 这一对坐标不能重复。  
所以这里按 `(lat, lon)` 分组，再用 `HAVING COUNT(*) = 1`，就能筛出位置唯一的记录。

最后把这两个条件用 `AND` 连起来：

```sql
WHERE ...
  AND ...
```

意思就是：  
只有同时满足“2015 投保额重复”以及“地理位置唯一”这两个条件的人，才会被选中。

等这些人筛出来之后，题目就只剩最后一步了：

```sql
SELECT ROUND(SUM(i.tiv_2016), 2) AS tiv_2016
```

把这些人的 `tiv_2016` 全部加起来，再按题目要求保留两位小数。

结合示例来看：

- 第 1 条记录的 `tiv_2015 = 10`，和别人重复；位置 `(10,10)` 也是唯一的，所以满足条件；
- 第 3 条记录虽然 `tiv_2015 = 10` 也重复，但它的位置 `(20,20)` 和第 2 条重复，所以不满足条件；
- 第 4 条记录的 `tiv_2015 = 10` 也重复，而且位置 `(40,40)` 唯一，所以也满足条件。

因此最后计入答案的是第 1 条和第 4 条记录：

- `5 + 40 = 45`

结果就是 `45.00`。

这题本质上考的是**分组筛选 + 再聚合**。  
也就是说，先通过 `GROUP BY ... HAVING ...` 找出满足某种“重复”或“唯一”条件的值，再回到原表继续筛选，最后做 `SUM`、`COUNT` 这类聚合。

以后只要碰到“先找出重复值”“先找出唯一值”“满足某种分组特征后再统计”的题，都可以往这个方向去想。
