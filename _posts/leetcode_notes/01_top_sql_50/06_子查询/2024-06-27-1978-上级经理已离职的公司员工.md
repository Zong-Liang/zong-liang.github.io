## 题目：1978. 上级经理已离职的公司员工

**表:** `Employees`

| Column Name | Type    |
| ----------- | ------- |
| employee_id | int     |
| name        | varchar |
| manager_id  | int     |
| salary      | int     |

- 在 SQL 中, `employee_id` 是这个表的主键。
- 这个表包含了员工，他们的薪水和上级经理的 id。
- 有一些员工没有上级经理（其 `manager_id` 是空值）。

**问题描述:**

查找这些员工的 id，他们的薪水严格少于 $30000 并且他们的上级经理已离职。当一个经理离开公司时，他们的信息需要从员工表中删除掉，但是表中的员工的 `manager_id` 这一列还是设置的离职经理的 id。

返回的结果按照 `employee_id` 从小到大排序。

**示例:**

**输入:**

`Employees` table:

| employee_id | name      | manager_id | salary |
| ----------- | --------- | ---------- | ------ |
| 3           | Mila      | 9          | 60301  |
| 12          | Antonella | null       | 31000  |
| 13          | Emery     | null       | 67084  |
| 1           | Kalel     | 11         | 21241  |
| 9           | Mikaela   | null       | 50937  |
| 11          | Joziah    | 6          | 28485  |

**输出:**

| employee_id |
| ----------- |
| 11          |

**解释:**

- 薪水少于 30000 美元的员工有 1 号(Kalel) 和 11 号(Joziah)。
- Kalel 的上级经理是 11 号员工，他还在公司上班(他是 Joziah )。
- Joziah 的上级经理是 6 号员工，他已经离职，因为员工表里面已经没有 6 号员工的信息了，它被删除了。

## 题解

这题真正要抓住的，不是“谁工资低”，而是题目里对“经理已离职”的定义：

**如果某个经理已经离职，那么他的信息会从 `Employees` 表里被删掉，但下属员工的 `manager_id` 还会保留原来的值。**

所以这题其实是在找这样一类员工：

- 自己工资小于 `30000`；
- 自己确实有经理；
- 但这个 `manager_id` 已经在员工表里找不到了。

对应的 SQL 写法如下：

```sql
SELECT employee_id
FROM Employees
WHERE salary < 30000
  AND manager_id IS NOT NULL
  AND manager_id NOT IN (
      SELECT employee_id
      FROM Employees
  )
ORDER BY employee_id ASC;
```

这段查询的逻辑可以顺着题意直接拆开来看。

先看第一个条件：

```sql
salary < 30000
```

这个没什么绕弯，就是按照题目要求，先把薪水严格小于 `30000` 的员工筛出来。

再看第二个条件：

```sql
manager_id IS NOT NULL
```

这一步也很重要。  
因为有些员工本来就没有上级经理，`manager_id` 是空值。  
这种人不能算“经理离职”，因为他一开始就没有经理，所以要先排除掉。

真正的关键在第三个条件：

```sql
manager_id NOT IN (
    SELECT employee_id
    FROM Employees
)
```

这里的意思是：

- 先把当前仍然在公司的所有员工 `employee_id` 都找出来；
- 再检查某个员工的 `manager_id` 是否不在这个列表里。

如果不在，就说明这个经理已经不在员工表中了，也就是已经离职了。

换句话说，这里不是去查“经理状态”，而是通过“员工表里已经没有这个人”来间接判断经理离职。

结合示例来看：

- `Kalel` 的工资小于 `30000`，经理是 `11`；
- 而 `11` 这个员工在表里还能找到，所以他的经理还在公司，不能入选。

再看 `Joziah`：

- 他的工资也是小于 `30000`；
- 经理是 `6`；
- 但表里已经没有 `employee_id = 6` 这条记录了；

所以说明他的经理已经离职，`Joziah` 就符合题意。

最后再按题目要求升序排序：

```sql
ORDER BY employee_id ASC
```

这样就能得到最终答案。

这题本质上考的是一种很常见的判断方式：

**通过“某个外键值在主表中已经不存在了”，反推出它关联的那条记录已经失效或被删除。**

以后碰到“表里保留了关联 ID，但对应记录已经没了”这种题，通常都可以往这个方向去想。
