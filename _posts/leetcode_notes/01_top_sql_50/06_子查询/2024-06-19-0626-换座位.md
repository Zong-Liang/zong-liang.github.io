## 题目：0626. 换座位

**表:** `Seat`

| Column Name | Type    |
| ----------- | ------- |
| id          | int     |
| student     | varchar |

- `id` 是该表的主键（唯一值）列。
- 该表的每一行都表示学生的姓名和 ID。
- ID 序列始终从 1 开始并连续增加。

**问题描述:**

编写解决方案来交换每两个连续的学生的座位号。如果学生的数量是奇数，则最后一个学生的 id 不交换。

按 `id` **升序** 返回结果表。

**示例 1:**

**输入:**

`Seat` 表:

| id  | student |
| --- | ------- |
| 1   | Abbot   |
| 2   | Doris   |
| 3   | Emerson |
| 4   | Green   |
| 5   | Jeames  |

**输出:**

| id  | student |
| --- | ------- |
| 1   | Doris   |
| 2   | Abbot   |
| 3   | Green   |
| 4   | Emerson |
| 5   | Jeames  |

**解释:**

请注意，如果学生人数为奇数，则不需要更换最后一名学生的座位。

## 题解

这题不要把重点放在“交换学生”，真正要处理的是：

**怎么给每一行生成交换之后的新 `id`。**

因为题目最后要求的是按 `id` 升序返回结果，所以这题其实不是去改学生名字，而是要想办法让每个学生对应到交换后的座位号上。

交换规则很简单：

- 如果当前是奇数座位，而且后面还有一个人，那么它要和下一位交换，所以新 `id` 变成 `id + 1`；
- 如果当前是偶数座位，那么它要和前一位交换，所以新 `id` 变成 `id - 1`；
- 如果当前是最后一个人，并且总人数是奇数，那么他没有人可以交换，`id` 保持不变。

对应的 SQL 写法如下：

```sql
SELECT CASE
           WHEN MOD(id, 2) != 0 AND id != COUNT(*) OVER() THEN id + 1
           WHEN MOD(id, 2) = 0 THEN id - 1
           ELSE id
       END AS id,
       student
FROM Seat
ORDER BY id ASC;
```

这段查询的核心就在 `CASE` 里。

先看第一种情况：

```sql
WHEN MOD(id, 2) != 0 AND id != COUNT(*) OVER() THEN id + 1
```

这里表示：

- `MOD(id, 2) != 0`：当前座位号是奇数；
- `id != COUNT(*) OVER()`：当前这一行不是最后一个学生。

如果这两个条件都满足，就说明当前学生可以和下一位交换，所以新的座位号就是 `id + 1`。

再看第二种情况：

```sql
WHEN MOD(id, 2) = 0 THEN id - 1
```

如果当前座位号是偶数，那么它一定是和前一个奇数座位成对交换的，所以新的座位号就是 `id - 1`。

最后剩下的情况就是：

```sql
ELSE id
```

这对应的就是“最后一个落单的学生”。  
只有当总人数是奇数时，才会出现这种情况。因为他后面没有人了，所以不能交换，只能保持原来的座位号不变。

这里还有一个比较容易忽略的点，就是：

```sql
COUNT(*) OVER()
```

它表示整张表的总行数，也就是学生总人数。  
有了这个值，我们就能判断当前这一行是不是最后一个学生。

结合示例来看：

原来的座位是：

- `1 -> Abbot`
- `2 -> Doris`
- `3 -> Emerson`
- `4 -> Green`
- `5 -> Jeames`

交换之后：

- `1` 和 `2` 交换；
- `3` 和 `4` 交换；
- `5` 是最后一个，而且没有配对的人，所以不动。

所以结果就是：

- `1 -> Doris`
- `2 -> Abbot`
- `3 -> Green`
- `4 -> Emerson`
- `5 -> Jeames`

这题本质上考的是**条件映射**。  
也就是根据不同的行，按规则生成一个新的结果列，再用这个结果列去排序。

以后遇到这种“不是改数据本身，而是按规则重新映射位置、编号、顺序”的题，通常都可以优先想到 `CASE WHEN`。
