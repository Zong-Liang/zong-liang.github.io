## 题目：1341. 电影评分

**表结构:**

`Movies`

| Column Name | Type    |
| ----------- | ------- |
| movie_id    | int     |
| title       | varchar |

- `movie_id` 是这个表的主键。
- `title` 是电影的名字。
- 每部电影都有一个唯一的 `title`。

`Users`

| Column Name | Type    |
| ----------- | ------- |
| user_id     | int     |
| name        | varchar |

- `user_id` 是表的主键。
- `name` 列具有唯一值。

`MovieRating`

| Column Name | Type |
| ----------- | ---- |
| movie_id    | int  |
| user_id     | int  |
| rating      | int  |
| created_at  | date |

- `(movie_id, user_id)` 是这个表的主键。
- 这个表包含用户在其评论中对电影的评分 `rating`。
- `created_at` 是用户的点评日期。

**问题描述:**

请你编写一个解决方案：

- 查找评论电影数量最多的用户名。如果出现平局，返回字典序较小的用户名。
- 查找在 **February 2020** 平均评分最高的电影名称。如果出现平局，返回字典序较小的电影名称。

返回结果格式如下例所示。

**示例 1:**

**输入:**

`Movies` 表:

| movie_id | title    |
| -------- | -------- |
| 1        | Avengers |
| 2        | Frozen 2 |
| 3        | Joker    |

`Users` 表:

| user_id | name   |
| ------- | ------ |
| 1       | Daniel |
| 2       | Monica |
| 3       | Maria  |
| 4       | James  |

`MovieRating` 表:

| movie_id | user_id | rating | created_at |
| -------- | ------- | ------ | ---------- |
| 1        | 1       | 3      | 2020-01-12 |
| 1        | 2       | 4      | 2020-02-11 |
| 1        | 3       | 2      | 2020-02-12 |
| 1        | 4       | 1      | 2020-01-01 |
| 2        | 1       | 5      | 2020-02-17 |
| 2        | 2       | 2      | 2020-02-01 |
| 2        | 3       | 2      | 2020-03-01 |
| 3        | 1       | 3      | 2020-02-22 |
| 3        | 2       | 4      | 2020-02-25 |

**输出:**

`Result` 表:

| results  |
| -------- |
| Daniel   |
| Frozen 2 |

**解释:**

- Daniel 和 Monica 都点评了 3 部电影 ("Avengers", "Frozen 2" 和 "Joker") 但是 Daniel 字典序比较小。
- Frozen 2 和 Joker 在 2 月的评分都是 3.5, 但是 Frozen 2 的字典序比较小。

## 题解

这题其实不是一道查询，而是**两道彼此独立的查询**：

- 第一问：找出评论电影数量最多的用户；
- 第二问：找出 **2020 年 2 月** 平均评分最高的电影。

最后只是因为题目要求把这两个结果放到同一列里返回，所以才需要把它们合并起来。

也就是说，这题的思路不是先想怎么一次查完，而是先把两个问题分别解决，再把结果拼起来。

对应的 SQL 写法如下：

```sql
(
    SELECT u.name AS results
    FROM MovieRating mr
    JOIN Users u
      ON mr.user_id = u.user_id
    GROUP BY u.user_id, u.name
    ORDER BY COUNT(mr.movie_id) DESC, u.name ASC
    LIMIT 1
)

UNION ALL

(
    SELECT m.title AS results
    FROM MovieRating mr
    JOIN Movies m
      ON mr.movie_id = m.movie_id
    WHERE DATE_FORMAT(mr.created_at, '%Y-%m') = '2020-02'
    GROUP BY m.movie_id, m.title
    ORDER BY AVG(mr.rating) DESC, m.title ASC
    LIMIT 1
);
```

这段查询可以分成两部分来看。

第一部分是找评论数量最多的用户：

```sql
SELECT u.name AS results
FROM MovieRating mr
JOIN Users u
  ON mr.user_id = u.user_id
GROUP BY u.user_id, u.name
ORDER BY COUNT(mr.movie_id) DESC, u.name ASC
LIMIT 1
```

这里的逻辑很直接：

先把评分表 `MovieRating` 和用户表 `Users` 连接起来，这样每条评分记录就能对应到用户名。  
然后按用户分组，统计每个用户一共评论了多少部电影。

最关键的是这一句：

```sql
ORDER BY COUNT(mr.movie_id) DESC, u.name ASC
```

它表示：

- 先按评论数从大到小排序；
- 如果评论数相同，就按用户名的字典序从小到大排序。

这样一来，排在最前面的就是题目要求的人。  
最后再 `LIMIT 1`，取第一名即可。

第二部分是找 2020 年 2 月平均评分最高的电影：

```sql
SELECT m.title AS results
FROM MovieRating mr
JOIN Movies m
  ON mr.movie_id = m.movie_id
WHERE DATE_FORMAT(mr.created_at, '%Y-%m') = '2020-02'
GROUP BY m.movie_id, m.title
ORDER BY AVG(mr.rating) DESC, m.title ASC
LIMIT 1
```

这一段和前一问思路类似，只不过对象从“用户”变成了“电影”。

先看筛选条件：

```sql
WHERE DATE_FORMAT(mr.created_at, '%Y-%m') = '2020-02'
```

这里先把评分时间限制在 `2020 年 2 月`。  
因为题目只关心这个月里的评分，所以其他月份的数据要先排除掉。

接着按电影分组，计算每部电影在这个月里的平均分：

```sql
ORDER BY AVG(mr.rating) DESC, m.title ASC
```

这里表示：

- 先按平均评分从高到低排序；
- 如果平均分相同，就按电影名字典序从小到大排序。

最后同样 `LIMIT 1`，取出答案。

两部分各自都会返回一行结果：

- 第一行是用户名；
- 第二行是电影名。

最后用 `UNION ALL` 把它们合并起来：

```sql
UNION ALL
```

这里用 `UNION ALL` 就够了，因为题目本来就要求返回两行结果，我们只是把两次查询的结果上下拼接起来，并不需要去重。

结合示例来看：

第一问里，Daniel 和 Monica 都评论了 3 部电影，数量一样；  
但 Daniel 的名字字典序更小，所以返回 Daniel。

第二问里，Frozen 2 和 Joker 在 2020 年 2 月的平均分都是 `3.5`；  
但 Frozen 2 的名字字典序更小，所以返回 Frozen 2。

因此最终结果就是：

- `Daniel`
- `Frozen 2`

这题本质上考的是**把一个题目拆成多个独立查询，再合并结果**。  
以后如果题目本身就在问两件不同的事，而且最后只是要求把结果放在一起返回，通常不需要硬凑成一条复杂 SQL，分开查再用 `UNION ALL` 合并，反而更清晰，也更不容易写错。
