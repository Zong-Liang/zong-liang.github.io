## 题目：0185. 部门工资前三高的所有员工

**表结构:**

`Employee`

| Column Name  | Type    |
| ------------ | ------- |
| id           | int     |
| name         | varchar |
| salary       | int     |
| departmentId | int     |

- `id` 是该表的主键列。
- `departmentId` 是 `Department` 表中 `id` 的外键。
- 该表的每一行都表示员工的 ID、姓名和工资。它还包含了他们部门的 ID。

`Department`

| Column Name | Type    |
| ----------- | ------- |
| id          | int     |
| name        | varchar |

- `id` 是该表的主键列。
- 该表的每一行表示部门 ID 和部门名。

**问题描述:**

公司的主管们感兴趣的是公司每个部门中谁赚的钱最多。一个部门的 **高收入者** 是指一个员工的工资在该部门的 **不同** 工资中 **排名前三**。

编写解决方案，找出每个部门中收入高的员工。

以 **任意顺序** 返回结果表。

**示例 1:**

**输入:**

`Employee` 表:

| id  | name  | salary | departmentId |
| --- | ----- | ------ | ------------ |
| 1   | Joe   | 85000  | 1            |
| 2   | Henry | 80000  | 2            |
| 3   | Sam   | 60000  | 2            |
| 4   | Max   | 90000  | 1            |
| 5   | Janet | 69000  | 1            |
| 6   | Randy | 85000  | 1            |
| 7   | Will  | 70000  | 1            |

`Department` 表:

| id  | name  |
| --- | ----- |
| 1   | IT    |
| 2   | Sales |

**输出:**

| Department | Employee | Salary |
| ---------- | -------- | ------ |
| IT         | Max      | 90000  |
| IT         | Joe      | 85000  |
| IT         | Randy    | 85000  |
| IT         | Will     | 70000  |
| Sales      | Henry    | 80000  |
| Sales      | Sam      | 60000  |

**解释:**

在 IT 部门:

- Max 的工资最高
- 兰迪和乔都赚取第二高的独特的薪水
- 威尔的薪水是第三高的

在销售部:

- 亨利的工资最高
- 山姆的薪水第二高
- 没有第三高的工资，因为只有两名员工

## 题解

这题最容易看错的地方，不是在“前三高”，而是在题目里这两个字：

**不同工资。**

也就是说，题目要找的不是“每个部门工资排前 3 名的 3 个人”，而是：

**每个部门里，工资属于前 3 个不同档位的所有员工。**

这两个说法差别很大。

比如某个部门工资从高到低是：

- 90000
- 85000
- 85000
- 70000

那么前 3 个**不同工资**是：

- 90000
- 85000
- 70000

此时工资是 `85000` 的两个人都应该保留下来，而不是只取其中一个。  
所以这题不能用 `ROW_NUMBER()`，因为那样会把并列工资强行拆开；更合适的是 `DENSE_RANK()`，因为它会给相同工资相同排名，并且排名不会跳号。

对应的 SQL 写法如下：

```sql
WITH RankedSalaries AS (
    SELECT e.name AS Employee,
           e.salary,
           e.departmentId,
           DENSE_RANK() OVER (
               PARTITION BY e.departmentId
               ORDER BY e.salary DESC
           ) AS salary_rank
    FROM Employee e
)
SELECT d.name AS Department,
       rs.Employee,
       rs.salary AS Salary
FROM RankedSalaries rs
JOIN Department d
  ON rs.departmentId = d.id
WHERE rs.salary_rank <= 3;
```

这段查询可以顺着思路分成两步理解。

第一步，先在每个部门内部按工资排名：

```sql
DENSE_RANK() OVER (
    PARTITION BY e.departmentId
    ORDER BY e.salary DESC
) AS salary_rank
```

这里的作用是：

- `PARTITION BY e.departmentId`：按部门分组，每个部门单独排名；
- `ORDER BY e.salary DESC`：工资从高到低排；
- `DENSE_RANK()`：如果工资相同，就给相同的排名，而且下一个不同工资会接着排下去。

比如某个部门工资是：

- 90000
- 85000
- 85000
- 70000

那么 `DENSE_RANK()` 的结果就是：

- 90000 -> 1
- 85000 -> 2
- 85000 -> 2
- 70000 -> 3

这正好符合题目的要求，因为题目看的是“不同工资中的前三高”，不是“前三行记录”。

第二步，再把排名前 3 的员工筛出来，并补上部门名：

```sql
SELECT d.name AS Department,
       rs.Employee,
       rs.salary AS Salary
FROM RankedSalaries rs
JOIN Department d
  ON rs.departmentId = d.id
WHERE rs.salary_rank <= 3;
```

这里做了两件事：

- 把前面算好的排名结果和 `Department` 表连接起来，拿到部门名称；
- 只保留 `salary_rank <= 3` 的员工，也就是工资属于本部门前 3 个不同档位的人。

结合示例来看，IT 部门的工资从高到低是：

- Max：90000
- Joe：85000
- Randy：85000
- Will：70000
- Janet：69000

这里前 3 个不同工资是：

- 90000
- 85000
- 70000

所以应该保留：

- Max
- Joe
- Randy
- Will

其中 Joe 和 Randy 虽然工资一样，但都属于第二高工资这一档，所以都要返回。

这也是这题最关键的地方。

这题本质上考的是**分组排名**，而且要特别注意“并列排名怎么处理”。  
以后只要题目里出现“每组前几名”“同分都要保留”“按不同值排名”这种描述，就要优先考虑窗口函数，并且先想清楚该用：

- `ROW_NUMBER()`
- `RANK()`
- 还是 `DENSE_RANK()`

这题显然是 `DENSE_RANK()` 最合适。
