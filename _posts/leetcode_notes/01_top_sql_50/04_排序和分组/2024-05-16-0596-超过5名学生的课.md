## 题目：0596. 超过 5 名学生的课

**表:** `Courses`

| Column Name | Type    |
| ----------- | ------- |
| student     | varchar |
| class       | varchar |

- `(student, class)` 是该表的主键（不同值的列的组合）。
- 该表的每一行表示学生的名字和他们注册的班级。

**问题描述:**

查询 **至少有 5 个学生** 的所有班级。

以 **任意顺序** 返回结果表。

**示例 1:**

**输入:**

`Courses` table:

| student | class    |
| ------- | -------- |
| A       | Math     |
| B       | English  |
| C       | Math     |
| D       | Biology  |
| E       | Math     |
| F       | Computer |
| G       | Math     |
| H       | Math     |
| I       | Math     |

**输出:**

| class |
| ----- |
| Math  |

**解释:**

- 数学课有 6 个学生，所以我们包括它。
- 英语课有 1 名学生，所以我们不包括它。
- 生物课有 1 名学生，所以我们不包括它。
- 计算机课有 1 个学生，所以我们不包括它。

## 题解

这题本质上就是在问：**每门课分别有多少学生，然后筛出人数达到要求的班级。**

题目要找的是“至少有 `5` 个学生”的课，所以思路很直接：

- 先按 `class` 分组；
- 再统计每个班级的人数；
- 最后只保留人数不少于 `5` 的班级。

对应的 SQL 写法如下：

```sql
SELECT class
FROM Courses
GROUP BY class
HAVING COUNT(student) >= 5;
```

这条 SQL 可以按顺序这样理解。

先看分组：

```sql
GROUP BY class
```

这里表示把相同班级的记录放到同一组里。  
因为表里的每一行都是“某个学生选了某门课”，所以按 `class` 分组之后，每一组就对应一门课的所有选课记录。

然后看筛选条件：

```sql
HAVING COUNT(student) >= 5
```

这里的 `COUNT(student)` 统计的是当前班级这一组里有多少个学生。  
题目要求的是“至少有 5 个学生”，所以直接保留 `COUNT(student) >= 5` 的班级即可。

这里要特别注意一下：  
这种条件不能写在 `WHERE` 里，因为 `WHERE` 是分组前筛选，而这道题是要先分组、再根据分组后的统计结果筛选，所以要用 `HAVING`。

结合示例来看：

- `Math` 一共有 6 个学生，所以满足条件，要保留；
- `English` 只有 1 个学生，不满足条件；
- `Biology` 只有 1 个学生，不满足条件；
- `Computer` 也只有 1 个学生，不满足条件。

所以最终结果只有：

- `Math`

这题的重点其实不是语法本身，而是分清楚：

- **`GROUP BY`** 是按什么维度分组；
- **`HAVING`** 是对分组后的统计结果做筛选。

以后只要看到“找出人数达到多少的班级”“找出订单数超过多少的用户”“找出出现次数不少于多少的商品”这种题，基本都可以往 `GROUP BY + HAVING` 这个方向去想。
