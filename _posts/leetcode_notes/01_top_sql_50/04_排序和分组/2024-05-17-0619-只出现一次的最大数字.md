## 题目：0619. 只出现一次的最大数字

**表:** `MyNumbers`

| Column Name | Type |
| ----------- | ---- |
| num         | int  |

- 该表可能包含重复项（换句话说，在 SQL 中，该表没有主键）。
- 这张表的每一行都含有一个整数。

**问题描述:**

**单一数字** 是在 `MyNumbers` 表中只出现一次的数字。

找出最大的 **单一数字**。如果不存在 **单一数字**，则返回 `null`。

**示例 1:**

**输入:**

`MyNumbers` 表:

| num |
| --- |
| 8   |
| 8   |
| 3   |
| 3   |
| 1   |
| 4   |
| 5   |
| 6   |

**输出:**

| num |
| --- |
| 6   |

**解释:** 单一数字有 1、4、5 和 6 。6 是最大的单一数字，返回 6 。

**示例 2:**

**输入:**

`MyNumbers` table:

| num |
| --- |
| 8   |
| 8   |
| 7   |
| 7   |
| 3   |
| 3   |

**输出:**

| num  |
| ---- |
| null |

**解释:** 输入的表中不存在单一数字，所以返回 null 。

## 题解

这题要找的不是最大数字本身，而是**只出现一次的数字里面的最大值**。

所以思路不能直接去 `MAX(num)`，而是要先做一步筛选：

- 先找出哪些数字只出现了一次；
- 再从这些数字里取最大值。

也就是说，这题其实分成两层：

- 第一层：按数字分组，筛出出现次数等于 `1` 的数字；
- 第二层：对这些“单一数字”再求最大值。

对应的 SQL 写法如下：

```sql
SELECT MAX(num) AS num
FROM (
    SELECT num
    FROM MyNumbers
    GROUP BY num
    HAVING COUNT(num) = 1
) AS single_numbers;
```

这条 SQL 可以按顺序这样理解。

先看子查询：

```sql
SELECT num
FROM MyNumbers
GROUP BY num
HAVING COUNT(num) = 1
```

这里的作用是找出所有只出现一次的数字。

- `GROUP BY num`：把相同的数字分到同一组；
- `COUNT(num) = 1`：只保留出现次数恰好为 `1` 的组。

经过这一步后，剩下的就是题目定义里的“单一数字”。

然后外层这句：

```sql
SELECT MAX(num) AS num
```

就是在这些单一数字里取最大值。

比如示例 1 里：

- `8` 出现了两次，不算；
- `3` 出现了两次，不算；
- `1`、`4`、`5`、`6` 都只出现了一次，算作单一数字。

所以最后在 `1`、`4`、`5`、`6` 里面取最大值，结果就是 `6`。

再看示例 2：

- 所有数字都出现了两次；
- 也就是说子查询结果是空的。

这时候 `MAX(num)` 对空结果集会返回 `NULL`，正好符合题目要求，所以这里不需要额外处理。

这题的关键在于先把问题拆开。  
只要题目里出现“满足某种频次条件的元素里，再找最大值/最小值”这种说法，通常都可以先 `GROUP BY + HAVING` 把目标元素筛出来，再在外层继续做 `MAX()`、`MIN()` 这样的聚合。
