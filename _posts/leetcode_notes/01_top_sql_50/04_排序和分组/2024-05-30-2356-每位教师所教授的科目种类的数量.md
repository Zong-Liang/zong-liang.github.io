## 题目：2356. 每位教师所教授的科目种类的数量

**表:** `Teacher`

| Column Name | Type |
| ----------- | ---- |
| teacher_id  | int  |
| subject_id  | int  |
| dept_id     | int  |

- 在 SQL 中, `(subject_id, dept_id)` 是该表的主键。
- 该表中的每一行都表示带有 `teacher_id` 的教师在系 `dept_id` 中教授科目 `subject_id`。

**问题描述:**

查询每位老师在大学里教授的科目种类的数量。

以 **任意顺序** 返回结果表。

**示例 1:**

**输入:**

`Teacher` 表:

| teacher_id | subject_id | dept_id |
| ---------- | ---------- | ------- |
| 1          | 2          | 3       |
| 1          | 2          | 4       |
| 1          | 3          | 3       |
| 2          | 1          | 1       |
| 2          | 2          | 1       |
| 2          | 3          | 1       |
| 2          | 4          | 1       |

**输出:**

| teacher_id | cnt |
| ---------- | --- |
| 1          | 2   |
| 2          | 4   |

**解释:**

- **教师 1:**
  - 他在 3、4 系教科目 2。
  - 他在 3 系教科目 3。
  - 总共教了科目 2 和 3，共 2 种。

- **教师 2:**
  - 他在 1 系教科目 1。
  - 他在 1 系教科目 2。
  - 他在 1 系教科目 3。
  - 他在 1 系教科目 4。
  - 总共教了科目 1, 2, 3, 4，共 4 种。

## 题解

这题乍一看像是在统计老师教了多少条记录，但题目真正要问的是：**每位老师教授了多少种不同的科目。**

这里“科目种类”是关键。  
也就是说，如果一个老师在不同院系教的是同一门课，这门课也只能算一次，不能重复计数。

所以这题的思路很直接：

- 先按 `teacher_id` 分组；
- 再统计每个老师教过多少个不同的 `subject_id`。

对应的 SQL 写法如下：

```sql
SELECT teacher_id,
       COUNT(DISTINCT subject_id) AS cnt
FROM Teacher
GROUP BY teacher_id;
```

这条 SQL 可以按顺序这样理解。

先看分组：

```sql
GROUP BY teacher_id
```

这里表示把同一个老师的所有授课记录放到同一组里。  
因为题目要求统计的是“每位老师”教授的科目种类数量，所以必须先按老师分组。

然后看核心统计：

```sql
COUNT(DISTINCT subject_id) AS cnt
```

这里有两个关键点：

- `COUNT(...)` 是在计数；
- `DISTINCT subject_id` 表示只统计不重复的科目编号。

也就是说，同一个老师如果在多个系教同一个 `subject_id`，最后也只算一门课。

结合示例来看：

- 教师 `1` 虽然有三条记录，但其中 `subject_id = 2` 出现了两次，只是分别在不同系里教，所以最终只算科目 `2` 和科目 `3`，总数是 `2`；
- 教师 `2` 教了 `1`、`2`、`3`、`4` 四门不同的课，所以结果是 `4`。

这题的重点其实不是分组本身，而是看清题目统计的是：

- 记录条数；
- 还是去重之后的种类数。

只要题目里出现“种类数”“不同的”“去重后数量”这种表述，就要优先想到 `COUNT(DISTINCT ...)`。这类题非常常见。
