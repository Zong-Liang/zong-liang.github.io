## 题目：1084. 销售分析 III

**表结构:**

`Product`

| Column Name  | Type    |
| ------------ | ------- |
| product_id   | int     |
| product_name | varchar |
| unit_price   | int     |

- `product_id` 是该表的主键（具有唯一值的列）。
- 该表的每一行显示每个产品的名称和价格。

`Sales`

| Column Name | Type |
| ----------- | ---- |
| seller_id   | int  |
| product_id  | int  |
| buyer_id    | int  |
| sale_date   | date |
| quantity    | int  |
| price       | int  |

- 这个表可能含有重复的行。
- `product_id` 是 `Product` 表的外键 (reference 列)。
- 该表的每一行包含关于一个销售的一些信息。

**问题描述:**

编写解决方案，报告 2019 年春季 **才** 售出的产品。即 **仅** 在 `2019-01-01` (含) 至 `2019-03-31` (含) 之间出售的商品。

以 **任意顺序** 返回结果表。

**示例 1:**

**输入:**

`Product` table:

| product_id | product_name | unit_price |
| ---------- | ------------ | ---------- |
| 1          | S8           | 1000       |
| 2          | G4           | 800        |
| 3          | iPhone       | 1400       |

`Sales` table:

| seller_id | product_id | buyer_id | sale_date  | quantity | price |
| --------- | ---------- | -------- | ---------- | -------- | ----- |
| 1         | 1          | 1        | 2019-01-21 | 2        | 2000  |
| 1         | 2          | 2        | 2019-02-17 | 1        | 800   |
| 2         | 2          | 3        | 2019-06-02 | 1        | 800   |
| 3         | 3          | 4        | 2019-05-13 | 2        | 2800  |

**输出:**

| product_id | product_name |
| ---------- | ------------ |
| 1          | S8           |

**解释:**

- id 为 1 的产品仅在 2019 年春季销售。
- id 为 2 的产品在 2019 年春季销售, 但也在 2019 年春季之后销售。
- id 为 3 的产品在 2019 年春季之后销售。
- 我们只返回 id 为 1 的产品, 因为它是 2019 年春季才销售的产品。

## 题解

这题最容易理解错的地方在于题目里的“**才** 售出”。

它不是在问“2019 年春季卖过的产品有哪些”，而是在问：  
**某个产品的所有销售记录，是否都落在 `2019-01-01` 到 `2019-03-31` 这个区间内。**

也就是说，只要一个产品：

- 在春季之前卖过，排除；
- 在春季之后卖过，排除；
- 只有销售记录全部都在春季，才保留。

这样一来，这题就不应该只筛“区间内的销售记录”，而是应该站在**每个产品整体的所有销售日期**上去判断。

最直接的做法就是：

- 先按产品分组；
- 再看这个产品最早的销售日期和最晚的销售日期；
- 如果最早销售日期都不早于 `2019-01-01`，并且最晚销售日期都不晚于 `2019-03-31`，那就说明这个产品所有销售都发生在春季。

对应的 SQL 写法如下：

```sql
SELECT p.product_id,
       p.product_name
FROM Product p
JOIN Sales s
     ON p.product_id = s.product_id
GROUP BY p.product_id, p.product_name
HAVING MIN(s.sale_date) >= '2019-01-01'
   AND MAX(s.sale_date) <= '2019-03-31';
```

这条 SQL 可以按顺序这样理解。

先看连接部分：

```sql
FROM Product p
JOIN Sales s
     ON p.product_id = s.product_id
```

这里是把产品表和销售表按 `product_id` 连接起来，这样每个产品就能拿到自己的所有销售日期，同时最后也能返回产品名称。

接着看分组：

```sql
GROUP BY p.product_id, p.product_name
```

这里表示把同一个产品的所有销售记录放到同一组里。  
因为题目要判断的是“这个产品是不是只在春季卖过”，所以必须把这个产品的全部销售记录放在一起看。

然后是核心判断：

```sql
HAVING MIN(s.sale_date) >= '2019-01-01'
   AND MAX(s.sale_date) <= '2019-03-31'
```

这两句合在一起，含义其实很明确：

- `MIN(s.sale_date) >= '2019-01-01'`：说明这个产品最早的一次销售也没有早于春季开始；
- `MAX(s.sale_date) <= '2019-03-31'`：说明这个产品最晚的一次销售也没有晚于春季结束。

如果最早和最晚都在春季范围内，那么中间所有销售日期自然也都在这个范围内。  
这就说明这个产品**只在 2019 年春季卖过**。

结合示例来看：

- 产品 `1` 的销售日期是 `2019-01-21`，最早和最晚都在春季范围内，所以保留；
- 产品 `2` 虽然在 `2019-02-17` 卖过，但后来又在 `2019-06-02` 卖过，所以 `MAX(s.sale_date)` 超出了 `2019-03-31`，排除；
- 产品 `3` 只在 `2019-05-13` 卖过，本身就在春季之后，也排除。

所以最终结果只剩下：

- `1  S8`

这题真正考的是一个很常见的分组判断思路：

- 如果题目要求“某个对象的所有记录都满足某条件”，
- 往往就要先按这个对象分组，
- 再通过 `MIN()`、`MAX()` 或者条件聚合去判断整组数据是否全部落在目标范围内。

这种“不是筛单条记录，而是判断整组记录是否合规”的题，后面会经常遇到。
