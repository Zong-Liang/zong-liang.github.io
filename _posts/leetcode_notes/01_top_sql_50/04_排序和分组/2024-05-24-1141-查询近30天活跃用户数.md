## 题目：1141. 查询近 30 天活跃用户数

**表:** `Activity`

| Column Name   | Type                                                               |
| ------------- | ------------------------------------------------------------------ |
| user_id       | int                                                                |
| session_id    | int                                                                |
| activity_date | date                                                               |
| activity_type | ENUM('open_session', 'end_session', 'scroll_down', 'send_message') |

- 该表没有包含重复数据。
- 该表记录社交媒体网站的用户活动。
- 注意，每个会话只属于一个用户。

**问题描述:**

编写解决方案，统计截至 `2019-07-27` (包含 2019-07-27)，近 **30 天** 的每日活跃用户数（当天只要有一条活动记录，即为活跃用户）。

以 **任意顺序** 返回结果表。

**示例 1:**

**输入:**

`Activity` table:

| user_id | session_id | activity_date | activity_type |
| ------- | ---------- | ------------- | ------------- |
| 1       | 1          | 2019-07-20    | open_session  |
| 1       | 1          | 2019-07-20    | scroll_down   |
| 1       | 1          | 2019-07-20    | end_session   |
| 2       | 4          | 2019-07-20    | open_session  |
| 2       | 4          | 2019-07-21    | send_message  |
| 2       | 4          | 2019-07-21    | end_session   |
| 3       | 2          | 2019-07-21    | open_session  |
| 3       | 2          | 2019-07-21    | end_session   |
| 4       | 3          | 2019-06-25    | open_session  |
| 4       | 3          | 2019-06-25    | end_session   |

**输出:**

| day        | active_users |
| ---------- | ------------ |
| 2019-07-20 | 2            |
| 2019-07-21 | 2            |

**解释:** 注意非活跃用户的记录不需要展示。

## 题解

这题看起来是在统计活跃用户数，但真正要先抓住两个限定条件：

- 只统计截至 `2019-07-27` 的近 `30` 天；
- 按**每天**统计活跃用户数，而且同一个用户一天内只算一次。

所谓“活跃用户”，题目也已经说得很清楚：  
**当天只要有任意一条活动记录，就算活跃。**

所以这题的思路其实很直接：

- 先把近 30 天范围内的记录筛出来；
- 再按日期分组；
- 最后统计每天有多少个不同的用户出现过。

对应的 SQL 写法如下：

```sql
SELECT activity_date AS day,
       COUNT(DISTINCT user_id) AS active_users
FROM Activity
WHERE activity_date BETWEEN DATE_SUB('2019-07-27', INTERVAL 29 DAY)
                        AND '2019-07-27'
GROUP BY activity_date;
```

这条 SQL 可以按顺序这样理解。

先看时间范围：

```sql
WHERE activity_date BETWEEN DATE_SUB('2019-07-27', INTERVAL 29 DAY)
                        AND '2019-07-27'
```

题目要求的是“截至 `2019-07-27` 的近 30 天”，而且包含 `2019-07-27` 当天。  
所以起点不是往前推 30 天，而是往前推 `29` 天，这样首尾加起来正好是 30 天。

也就是说，这里筛出来的实际日期范围是：

```sql
2019-06-28 ~ 2019-07-27
```

接着看分组：

```sql
GROUP BY activity_date
```

因为题目要求的是“每日活跃用户数”，所以要按日期分组。  
每一天的活动记录会被放到同一组里，后面才能单独统计这一天有多少活跃用户。

然后看核心统计：

```sql
COUNT(DISTINCT user_id) AS active_users
```

这里一定要用 `DISTINCT`，因为同一个用户在同一天可能有多条活动记录。  
比如打开会话、滚动、发消息、结束会话，都可能分别记成多行，但题目只要统计“这个用户今天活不活跃”，不是统计他今天做了几次操作。

所以：

- 同一天里一个用户出现一次也算活跃；
- 出现多次也还是只算一个人。

这正是 `COUNT(DISTINCT user_id)` 的作用。

再结合示例看：

- `2019-07-20` 这一天，用户 `1` 和用户 `2` 都有活动记录，所以活跃用户数是 `2`；
- `2019-07-21` 这一天，用户 `2` 和用户 `3` 都有活动记录，所以活跃用户数也是 `2`；
- `2019-06-25` 虽然也有记录，但它不在近 30 天范围内，所以不会被统计进去。

因此最终结果就是题目给出的那两天。

这题本质上是一个很标准的“时间范围内按天去重计数”问题。  
以后只要看到“某段时间内，每天有多少活跃用户 / 下单用户 / 登录用户”这种题，通常都是这套思路：

- 先用 `WHERE` 限定时间范围；
- 再用 `GROUP BY` 按天分组；
- 最后用 `COUNT(DISTINCT ...)` 统计每天的不重复人数。
