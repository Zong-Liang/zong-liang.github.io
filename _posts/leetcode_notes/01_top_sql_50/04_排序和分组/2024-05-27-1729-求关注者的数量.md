## 题目：1729. 求关注者的数量

**表:** `Followers`

| Column Name | Type |
| ----------- | ---- |
| user_id     | int  |
| follower_id | int  |

- `(user_id, follower_id)` 是这个表的主键（具有唯一值的列的组合）。
- 该表包含一个关注关系中关注者和用户的编号，其中关注者关注用户。

**问题描述:**

编写解决方案，对于每一个用户，返回该用户的关注者数量。

按 `user_id` 的顺序返回结果表。

**示例 1:**

**输入:**

`Followers` 表:

| user_id | follower_id |
| ------- | ----------- |
| 0       | 1           |
| 1       | 0           |
| 2       | 0           |
| 2       | 1           |

**输出:**

| user_id | followers_count |
| ------- | --------------- |
| 0       | 1               |
| 1       | 1               |
| 2       | 2               |

**解释:**

- 0 的关注者有 {1}
- 1 的关注者有 {0}
- 2 的关注者有 {0,1}

## 题解

这题本质上很直接：  
表里的每一行都表示“`follower_id` 关注了 `user_id`”，所以如果想知道某个用户有多少关注者，其实就是数一数：**有多少行记录的 `user_id` 是他。**

也就是说，题目要做的就是：

- 按 `user_id` 分组；
- 统计每组里有多少个 `follower_id`；
- 最后按 `user_id` 升序排序。

对应的 SQL 写法如下：

```sql
SELECT user_id,
       COUNT(follower_id) AS followers_count
FROM Followers
GROUP BY user_id
ORDER BY user_id ASC;
```

这条 SQL 可以按顺序这样理解。

先看分组：

```sql
GROUP BY user_id
```

这里表示把相同 `user_id` 的记录放到同一组里。  
因为题目要统计的是“每一个用户的关注者数量”，所以自然要按用户编号分组。

接着看计数：

```sql
COUNT(follower_id) AS followers_count
```

这里统计的就是当前这个用户一共有多少个关注者。  
因为 `Followers` 表中的一行就代表一个关注关系，所以某个 `user_id` 对应多少行，就说明这个用户有多少个关注者。

最后这一句：

```sql
ORDER BY user_id ASC
```

是题目明确要求的排序方式，也就是按 `user_id` 从小到大输出。

结合示例来看：

- `user_id = 0` 只被 `1` 关注，所以关注者数量是 `1`；
- `user_id = 1` 只被 `0` 关注，所以数量也是 `1`；
- `user_id = 2` 被 `0` 和 `1` 关注，所以数量是 `2`。

这题本身不难，关键就是把表意读清楚：  
**谁是被关注的人，谁是关注者。**

一旦读明白这一点，后面的写法就是最基础的 `GROUP BY + COUNT()`。以后遇到“每个用户有多少粉丝”“每个商品被买了多少次”“每门课有多少学生”这种题，通常也都是这一类分组计数题。
