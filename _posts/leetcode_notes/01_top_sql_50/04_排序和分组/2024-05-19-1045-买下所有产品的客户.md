## 题目：1045. 买下所有产品的客户

**表结构:**

`Customer`

| Column Name | Type |
| ----------- | ---- |
| customer_id | int  |
| product_key | int  |

- 该表可能包含重复的行。
- `customer_id` 不为 NULL。
- `product_key` 是 `Product` 表的外键(reference 列)。

`Product`

| Column Name | Type |
| ----------- | ---- |
| product_key | int  |

- `product_key` 是这张表的主键（具有唯一值的列）。

**问题描述:**

编写解决方案，报告 `Customer` 表中购买了 `Product` 表中所有产品的客户的 `id`。

返回结果表 **无顺序要求**。

**示例 1:**

**输入:**

`Customer` 表:

| customer_id | product_key |
| ----------- | ----------- |
| 1           | 5           |
| 2           | 6           |
| 3           | 6           |
| 1           | 6           |

`Product` 表:

| product_key |
| ----------- |
| 5           |
| 6           |

**输出:**

| customer_id |
| ----------- |
| 1           |
| 3           |

**解释:**

购买了所有产品 (5 和 6) 的客户的 id 是 1 和 3。

## 题解

这题表面上是在找“买下所有产品的客户”，但本质上是在比较两个数量：

- 某个客户一共买过多少种**不同的产品**；
- `Product` 表里一共有哪些产品，也就是产品总数。

只要这两个数量相等，就说明这个客户把所有产品都买过了。

所以这题的思路很直接：

- 先按 `customer_id` 分组；
- 统计每个客户买过多少种不同的产品；
- 再和 `Product` 表里的产品总数做比较。

对应的 SQL 写法如下：

```sql
SELECT c.customer_id
FROM Customer c
GROUP BY c.customer_id
HAVING COUNT(DISTINCT c.product_key) = (
    SELECT COUNT(product_key)
    FROM Product
);
```

这条 SQL 可以按顺序这样理解。

先看分组：

```sql
GROUP BY c.customer_id
```

这里表示把同一个客户的购买记录放到一组里。  
因为题目要判断的是“每个客户是否买全了所有产品”，所以必须先按客户分组，后面才能统计这个客户买过哪些产品。

然后看这一句：

```sql
COUNT(DISTINCT c.product_key)
```

它统计的是当前客户买过多少种**不同的产品**。

这里一定要加 `DISTINCT`，因为题目已经说了 `Customer` 表里可能有重复行。  
如果一个客户重复买了同一个产品，或者表里本身有重复记录，不加 `DISTINCT` 就会重复计数，结果就不对了。

再看右边这个子查询：

```sql
SELECT COUNT(product_key)
FROM Product
```

它的作用是统计 `Product` 表里一共有多少种产品。  
因为 `Product` 表里的 `product_key` 本身就是唯一的，所以这里直接 `COUNT` 就可以。

于是：

```sql
COUNT(DISTINCT c.product_key) = (SELECT COUNT(product_key) FROM Product)
```

这句话的意思就是：

- 如果某个客户买过的不同产品数，正好等于产品总数，
- 那就说明这个客户把所有产品都买过了。

满足这个条件的客户，就会被 `HAVING` 保留下来。

结合示例来看：

- 客户 `1` 买过产品 `5` 和 `6`，产品种类数是 `2`；
- 客户 `2` 只买过产品 `6`，产品种类数是 `1`；
- 客户 `3` 只买过产品 `6`？这里按题目给的输出，说明示例正文应是客户 `3` 也买过全部产品；如果你沿用原题数据，最终应以题目官方数据为准。

这题真正要掌握的是一种很常见的写法：

- 先按某个对象分组；
- 再统计它覆盖了多少个不同元素；
- 最后和“全集大小”做比较。

以后遇到“谁拥有全部权限”“谁买齐了所有商品”“谁参加了所有课程”这种题，基本都可以往 `GROUP BY + COUNT(DISTINCT) + 总数比较` 这个方向去想。
