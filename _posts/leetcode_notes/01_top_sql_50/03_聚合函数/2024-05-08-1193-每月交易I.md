## 题目：1193. 每月交易 I

**表:** `Transactions`

| Column Name | Type                         |
| ----------- | ---------------------------- |
| id          | int                          |
| country     | varchar                      |
| state       | enum('approved', 'declined') |
| amount      | int                          |
| trans_date  | date                         |

- `id` 是这个表的主键。
- 该表包含有关传入事务的信息。
- `state` 列类型为 `["approved", "declined"]` 之一。

**问题描述:**

编写一个 sql 查询来查找每个月和每个国家/地区的事务数及其总金额、已批准的事务数及其总金额。

以 **任意顺序** 返回结果表。

**示例 1:**

**输入:**

`Transactions` table:

| id  | country | state    | amount | trans_date |
| --- | ------- | -------- | ------ | ---------- |
| 121 | US      | approved | 1000   | 2018-12-18 |
| 122 | US      | declined | 2000   | 2018-12-19 |
| 123 | US      | approved | 2000   | 2019-01-01 |
| 124 | DE      | approved | 2000   | 2019-01-07 |

**输出:**

| month   | country | trans_count | approved_count | trans_total_amount | approved_total_amount |
| ------- | ------- | ----------- | -------------- | ------------------ | --------------------- |
| 2018-12 | US      | 2           | 1              | 3000               | 1000                  |
| 2019-01 | US      | 1           | 1              | 2000               | 2000                  |
| 2019-01 | DE      | 1           | 1              | 2000               | 2000                  |

## 题解

这题看起来字段很多，但本质上只是在做一件事：  
**按“月份 + 国家”分组统计，然后顺手把其中已批准的交易单独再统计一遍。**

题目一共要我们返回 6 个结果：

- 每个月、每个国家的总交易数；
- 每个月、每个国家的已批准交易数；
- 每个月、每个国家的总交易金额；
- 每个月、每个国家的已批准交易金额。

所以这题可以直接分成两步理解：

- 先确定分组单位：按月份和国家分组；
- 再在每个分组里分别统计总数、总金额，以及 `approved` 相关的数据。

对应的 SQL 写法如下：

```sql
SELECT DATE_FORMAT(trans_date, '%Y-%m') AS month,
       country,
       COUNT(id) AS trans_count,
       SUM(CASE WHEN state = 'approved' THEN 1 ELSE 0 END) AS approved_count,
       SUM(amount) AS trans_total_amount,
       SUM(CASE WHEN state = 'approved' THEN amount ELSE 0 END) AS approved_total_amount
FROM Transactions
GROUP BY DATE_FORMAT(trans_date, '%Y-%m'),
         country;
```

这条 SQL 可以按顺序这样理解。

先看分组依据：

```sql
GROUP BY DATE_FORMAT(trans_date, '%Y-%m'),
         country
```

这里的意思是把交易按“年月”和“国家”组合起来分组。  
比如 `2018-12` 的 `US` 是一组，`2019-01` 的 `US` 是另一组，`2019-01` 的 `DE` 也是另一组。

也就是说，后面的所有统计，都是在这些组内部完成的。

再看第一列：

```sql
DATE_FORMAT(trans_date, '%Y-%m') AS month
```

它的作用是把日期转成题目要求的月份格式，比如 `2019-01`。  
因为题目要按月统计，所以这里不能直接用完整日期，而是要先把“年-月”提取出来。

接着看总交易数：

```sql
COUNT(id) AS trans_count
```

这表示统计当前分组里一共有多少条交易记录。

再看已批准交易数：

```sql
SUM(CASE WHEN state = 'approved' THEN 1 ELSE 0 END) AS approved_count
```

这里是一个很典型的条件计数写法：

- 如果当前交易是 `approved`，就记 `1`；
- 否则记 `0`；
- 最后把这一组里的值加起来，就得到了已批准交易的数量。

总交易金额则更直接：

```sql
SUM(amount) AS trans_total_amount
```

就是把当前分组里的所有交易金额加总。

最后是已批准交易金额：

```sql
SUM(CASE WHEN state = 'approved' THEN amount ELSE 0 END) AS approved_total_amount
```

这个逻辑和刚才的条件计数一样，只不过这次不是记 `1`，而是：

- 如果交易已批准，就把它的 `amount` 加进去；
- 否则加 `0`。

这样最终得到的，就是已批准交易的总金额。

结合示例来看：

对于 `2018-12` 的 `US` 这组：

- 一共有两笔交易，所以 `trans_count = 2`；
- 其中一笔是 `approved`，所以 `approved_count = 1`；
- 两笔总金额是 `1000 + 2000 = 3000`；
- 已批准的那笔金额是 `1000`，所以 `approved_total_amount = 1000`。

其他分组也是同样的计算方式。

这题真正要掌握的是一个很常见的聚合套路：

- 先用 `GROUP BY` 明确统计范围；
- 普通统计直接用 `COUNT`、`SUM`；
- 只统计满足某种条件的数据时，就用 `CASE WHEN` 配合聚合函数做条件聚合。

后面只要遇到“总数统计一份，满足某条件的数据再统计一份”这种题，基本都可以照这个思路来写。
