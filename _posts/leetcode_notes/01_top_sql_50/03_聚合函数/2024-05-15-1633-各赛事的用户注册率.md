## 题目：1633. 各赛事的的用户注册率

**用户表:** `Users`

| Column Name | Type    |
| ----------- | ------- |
| user_id     | int     |
| user_name   | varchar |

- `user_id` 是该表的主键。
- 该表中的每行包括用户 ID 和用户名。

**注册表:** `Register`

| Column Name | Type |
| ----------- | ---- |
| contest_id  | int  |
| user_id     | int  |

- `(contest_id, user_id)` 是该表的主键。
- 该表中的每行包含用户的 ID 和他们注册的赛事。

**问题描述:**

编写解决方案统计出各赛事的的用户注册百分率，**保留两位小数**。

返回的结果表按 `percentage` 的 **降序** 排序，若相同则按 `contest_id` 的 **升序** 排序。

**示例 1:**

**输入:**

`Users` 表:

| user_id | user_name |
| ------- | --------- |
| 6       | Alice     |
| 2       | Bob       |
| 7       | Alex      |

`Register` 表:

| contest_id | user_id |
| ---------- | ------- |
| 215        | 6       |
| 209        | 2       |
| 208        | 2       |
| 210        | 6       |
| 208        | 6       |
| 209        | 7       |
| 209        | 6       |
| 215        | 7       |
| 208        | 7       |
| 210        | 2       |
| 207        | 2       |
| 210        | 7       |

**输出:**

| contest_id | percentage |
| ---------- | ---------- |
| 208        | 100.00     |
| 209        | 100.00     |
| 210        | 100.00     |
| 215        | 66.67      |
| 207        | 33.33      |

**解释:**

- 所有用户都注册了 208、209 和 210 赛事，因此这些赛事的注册率为 100%
- Alice 和 Alex 注册了 215 赛事，注册率为 ((2/3) \* 100) = 66.67%
- Bob 注册了 207 赛事，注册率为 ((1/3) \* 100) = 33.33%

## 题解

这题要算的是**每个赛事的注册率**，本质上就是：

```sql
某个赛事的注册人数 / 总用户数 * 100
```

所以这题先别急着写 SQL，先把分子和分母想清楚：

- 分子：注册了这个赛事的用户数；
- 分母：平台上的总用户数。

题目给了两张表：

- `Register` 表记录“谁报名了哪个赛事”；
- `Users` 表记录“平台一共有多少用户”。

因此做法其实很直接：  
先按 `contest_id` 统计每个赛事的报名人数，再除以 `Users` 表里的总人数，最后按要求排序即可。

```sql
SELECT r.contest_id,
       ROUND(
           COUNT(r.user_id) * 100.0 / (SELECT COUNT(*) FROM Users),
           2
       ) AS percentage
FROM Register r
GROUP BY r.contest_id
ORDER BY percentage DESC,
         r.contest_id ASC;
```

这条 SQL 可以按顺序这样理解。

先看分组：

```sql
GROUP BY r.contest_id
```

这里表示把 `Register` 表中相同 `contest_id` 的记录放到一组。  
因为题目要求统计“每个赛事”的注册率，所以自然要按赛事编号分组。

然后看分子：

```sql
COUNT(r.user_id)
```

它表示当前这个赛事一共有多少个用户报名。  
因为 `Register` 表里每一行就是一个“用户报名某个赛事”的记录，所以直接计数即可。

再看分母：

```sql
(SELECT COUNT(*) FROM Users)
```

这一步是在统计平台总用户数。  
因为注册率不是除以某个赛事相关的人数，而是除以**所有用户总数**，所以这里直接从 `Users` 表里统计总行数即可。

于是：

```sql
COUNT(r.user_id) * 100.0 / (SELECT COUNT(*) FROM Users)
```

就是当前赛事的注册百分比。

这里写成 `100.0` 而不是 `100`，是为了确保计算时按小数来处理，避免一些数据库里出现整数除法的问题。

然后再用：

```sql
ROUND(..., 2)
```

把结果四舍五入保留两位小数。

最后看排序部分：

```sql
ORDER BY percentage DESC,
         r.contest_id ASC
```

这正好对应题目的要求：

- 先按注册率从高到低排序；
- 如果注册率相同，再按 `contest_id` 从小到大排序。

结合示例来看：

- 赛事 `208`、`209`、`210` 都有 3 个用户注册，而总用户数也是 3，所以注册率都是 `100.00`；
- 赛事 `215` 有 2 个用户注册，所以注册率是 `2 / 3 * 100 = 66.67`；
- 赛事 `207` 只有 1 个用户注册，所以注册率是 `1 / 3 * 100 = 33.33`。

这题本质上就是一个很标准的“分组后求占比”问题。  
以后只要看到“某一类数量占总体数量的百分比”这种题，通常都可以先想清楚分子、分母分别是谁，再用 `GROUP BY + COUNT + 子查询总数` 这种结构来写。
