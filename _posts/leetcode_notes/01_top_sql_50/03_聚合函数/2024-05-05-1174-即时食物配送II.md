## 题目：1174. 即时食物配送 II

**配送表:** `Delivery`

| Column Name                 | Type |
| --------------------------- | ---- |
| delivery_id                 | int  |
| customer_id                 | int  |
| order_date                  | date |
| customer_pref_delivery_date | date |

- `delivery_id` 是该表中具有唯一值的列。
- 该表保存着顾客的食物配送信息，顾客在某个日期下了订单，并指定了一个期望的配送日期（和下单日期相同或者在那之后）。

**问题描述:**

- 如果顾客期望的配送日期和下单日期相同，则该订单称为 **「即时订单」**，否则称为 **「计划订单」**。
- **「首次订单」** 是顾客最早创建的订单。我们保证一个顾客只会有一个「首次订单」。

编写解决方案以获取即时订单在所有用户的首次订单中的比例。**保留两位小数**。

**示例 1:**

**输入:**

`Delivery` 表:

| delivery_id | customer_id | order_date | customer_pref_delivery_date |
| ----------- | ----------- | ---------- | --------------------------- |
| 1           | 1           | 2019-08-01 | 2019-08-02                  |
| 2           | 2           | 2019-08-02 | 2019-08-02                  |
| 3           | 1           | 2019-08-11 | 2019-08-12                  |
| 4           | 3           | 2019-08-24 | 2019-08-24                  |
| 5           | 3           | 2019-08-21 | 2019-08-22                  |
| 6           | 2           | 2019-08-11 | 2019-08-13                  |
| 7           | 4           | 2019-08-09 | 2019-08-09                  |

**输出:**

| immediate_percentage |
| -------------------- |
| 50.00                |

**解释:**

- 1 号顾客的 1 号订单是首次订单, 并且是计划订单。
- 2 号顾客的 2 号订单是首次订单, 并且是即时订单。
- 3 号顾客的 5 号订单是首次订单, 并且是计划订单。
- 4 号顾客的 7 号订单是首次订单, 并且是即时订单。
- 因此, 一半顾客的首次订单是即时的。

## 题解

这题表面上是在求“即时订单的比例”，但真正要先想清楚的是：**只能统计每个用户的首次订单**，后面的订单都不参与计算。

所以这题可以拆成两步：

- 先找出每个用户的首次订单；
- 再判断这些首次订单里，有多少是即时订单。

题目里已经说明了：

- 如果 `order_date = customer_pref_delivery_date`，就是即时订单；
- 否则就是计划订单。

因此，只要把所有用户的首次订单筛出来，再在这些订单里统计“即时订单”所占的比例即可。

```sql
SELECT ROUND(
           AVG(
               CASE
                   WHEN order_date = customer_pref_delivery_date THEN 1
                   ELSE 0
               END
           ) * 100,
           2
       ) AS immediate_percentage
FROM Delivery
WHERE (customer_id, order_date) IN (
    SELECT customer_id, MIN(order_date)
    FROM Delivery
    GROUP BY customer_id
);
```

这条 SQL 可以按顺序这样理解。

先看子查询：

```sql
SELECT customer_id, MIN(order_date)
FROM Delivery
GROUP BY customer_id
```

它的作用是找出每个用户最早的下单日期，也就是首次订单的日期。

因为同一个用户可能下过很多次单，所以要按 `customer_id` 分组；  
而 `MIN(order_date)` 取到的就是这个用户最早的下单时间。

接着外层这句：

```sql
WHERE (customer_id, order_date) IN (...)
```

表示只保留那些“用户编号 + 下单日期”正好等于首次订单的记录。  
经过这一步，剩下的就全部是每个用户的首次订单。

然后再看这一段：

```sql
CASE
    WHEN order_date = customer_pref_delivery_date THEN 1
    ELSE 0
END
```

这里是在判断首次订单是不是即时订单：

- 如果下单日期和期望送达日期相同，记为 `1`；
- 否则记为 `0`。

这样一来，所有首次订单就会被转成一串 `1` 和 `0`。

最后用：

```sql
AVG(...) * 100
```

求平均值，其实就是在求 `1` 的占比，也就是即时订单在所有首次订单中的比例。  
再乘上 `100`，就变成百分比。

最后的：

```sql
ROUND(..., 2)
```

则是按题目要求保留两位小数。

结合示例来看：

- 顾客 `1` 的首次订单不是即时订单，记为 `0`；
- 顾客 `2` 的首次订单是即时订单，记为 `1`；
- 顾客 `3` 的首次订单不是即时订单，记为 `0`；
- 顾客 `4` 的首次订单是即时订单，记为 `1`。

所以就是：

```sql
(0 + 1 + 0 + 1) / 4 * 100 = 50.00
```

这题的关键其实就一句话：  
**不是统计所有订单里有多少即时订单，而是只看每个用户的第一单。**

以后遇到这种“每个人先找第一次，再在这些第一次里做统计”的题，通常都可以先用 `GROUP BY + MIN()` 找出第一次，再在外层继续筛选和计算。
