## 题目：0620. 有趣的电影

**表:** `cinema`

| Column Name | Type    |
| ----------- | ------- |
| id          | int     |
| movie       | varchar |
| description | varchar |
| rating      | float   |

- `id` 是该表的主键（具有唯一值的列）。
- 每行包含有关电影名称、类型和评级的信息。
- 评级为 `[0,10]` 范围内的小数点后 2 位浮点数。

**问题描述:**

编写解决方案，找出所有影片描述为 **非** `boring` (不无聊) 的并且 `id` 为 **奇数** 的影片。

返回结果按 `rating` **降序排列**。

**示例 1:**

**输入:**

| id  | movie      | description | rating |
| --- | ---------- | ----------- | ------ |
| 1   | War        | great 3D    | 8.9    |
| 2   | Science    | fiction     | 8.5    |
| 3   | irish      | boring      | 6.2    |
| 4   | Ice song   | Fantacy     | 8.6    |
| 5   | House card | Interesting | 9.1    |

**输出:**

| id  | movie      | description | rating |
| --- | ---------- | ----------- | ------ |
| 5   | House card | Interesting | 9.1    |
| 1   | War        | great 3D    | 8.9    |

**解释:**

我们有三部电影，它们的 id 是奇数:1、3 和 5。id = 3 的电影是 boring 的，所以我们不把它包括在答案中。

## 题解

这题其实就是把题目里的三个要求直接翻译成 SQL：

- `id` 是奇数；
- `description` 不是 `'boring'`；
- 结果按 `rating` 降序排序。

前两个是筛选条件，最后一个是排序要求，所以直接写在 `WHERE` 和 `ORDER BY` 里就可以了。

```sql
SELECT *
FROM cinema
WHERE MOD(id, 2) = 1
  AND description != 'boring'
ORDER BY rating DESC;
```

这条 SQL 可以这样理解。

先看第一个条件：

```sql
MOD(id, 2) = 1
```

`MOD(id, 2)` 表示 `id` 除以 `2` 的余数。  
如果余数是 `1`，说明这个 `id` 是奇数；如果余数是 `0`，说明它是偶数。

再看第二个条件：

```sql
description != 'boring'
```

这表示只保留描述不是 `'boring'` 的电影。

因为题目要求这两个条件要同时满足，所以这里要用 `AND` 连起来。  
也就是说，只有既是奇数编号、又不是无聊电影的记录，才会进入结果集。

最后这一句：

```sql
ORDER BY rating DESC
```

表示按评分从高到低排序。  
`DESC` 就是降序，所以评分高的电影会排在前面。

结合示例来看：

- `id = 1` 是奇数，`description` 也不是 `boring`，保留；
- `id = 3` 虽然是奇数，但 `description = 'boring'`，排除；
- `id = 5` 是奇数，而且描述也不是 `boring`，保留。

最后剩下 `id = 1` 和 `id = 5` 两部电影，再按照 `rating` 降序排序，所以 `5` 会排在 `1` 前面。

这题本身不难，重点就是把题目要求拆清楚：

- 筛选条件写进 `WHERE`；
- 排序要求写进 `ORDER BY`；
- 多个条件同时满足时，用 `AND` 连接。
