## 题目：1075. 项目员工 I

**项目表 `Project`:**

| Column Name | Type |
| ----------- | ---- |
| project_id  | int  |
| employee_id | int  |

- 主键为 `(project_id, employee_id)`。
- `employee_id` 是员工表 `Employee` 表的外键。
- 这张表的每一行表示 `employee_id` 的员工正在 `project_id` 的项目上工作。

**员工表 `Employee`:**

| Column Name      | Type    |
| ---------------- | ------- |
| employee_id      | int     |
| name             | varchar |
| experience_years | int     |

- 主键是 `employee_id`。数据保证 `experience_years` 非空。
- 这张表的每一行包含一个员工的信息。

**问题描述:**

请写一个 SQL 语句，查询每一个项目中员工的 **平均** 工作年限，**精确到小数点后两位**。

以 **任意顺序** 返回结果表。

**示例 1:**

**输入:**

`Project` 表:

| project_id | employee_id |
| ---------- | ----------- |
| 1          | 1           |
| 1          | 2           |
| 1          | 3           |
| 2          | 1           |
| 2          | 4           |

`Employee` 表:

| employee_id | name   | experience_years |
| ----------- | ------ | ---------------- |
| 1           | Khaled | 3                |
| 2           | Ali    | 2                |
| 3           | John   | 1                |
| 4           | Doe    | 2                |

**输出:**

| project_id | average_years |
| ---------- | ------------- |
| 1          | 2.00          |
| 2          | 2.50          |

**解释:**

第一个项目中，员工的平均工作年限是 (3 + 2 + 1) / 3 = 2.00; 第二个项目中，员工的平均工作年限是 (3 + 2) / 2 = 2.50。

## 题解

这题的目标很明确：**统计每个项目中所有员工的平均工作年限。**

但 `Project` 表里只有：

- `project_id`
- `employee_id`

并没有员工的工作年限。  
真正的 `experience_years` 在 `Employee` 表里，所以这题第一步一定是先把两张表连起来，把“项目里有哪些员工”转换成“项目里这些员工各有多少年经验”。

这就是一个很典型的连接后再分组统计的问题。

```sql
SELECT p.project_id,
       ROUND(AVG(e.experience_years), 2) AS average_years
FROM Project p
JOIN Employee e
     ON p.employee_id = e.employee_id
GROUP BY p.project_id;
```

这条 SQL 可以按顺序这样理解。

先看连接部分：

```sql
FROM Project p
JOIN Employee e
     ON p.employee_id = e.employee_id
```

这里的作用是把项目表和员工表通过 `employee_id` 关联起来。  
这样每一条“某员工参与某项目”的记录，就都能拿到这个员工对应的 `experience_years`。

接着看分组：

```sql
GROUP BY p.project_id
```

因为题目要求的是“每一个项目”的平均工作年限，所以要按项目编号分组。  
同一个项目下的所有员工会被放到同一组里，后面才能在这一组内部求平均值。

然后是核心计算：

```sql
AVG(e.experience_years)
```

它会计算每个项目分组中，所有员工工作年限的平均值。

最后再用：

```sql
ROUND(..., 2)
```

把结果四舍五入保留两位小数，这也是题目要求的输出格式。

结合示例来看：

- 项目 `1` 对应员工 `1`、`2`、`3`，他们的工作年限分别是 `3`、`2`、`1`，平均值是 `(3 + 2 + 1) / 3 = 2.00`；
- 项目 `2` 对应员工 `1`、`4`，他们的工作年限分别是 `3`、`2`，平均值是 `(3 + 2) / 2 = 2.50`。

所以最终返回的就是：

- `1  2.00`
- `2  2.50`

这题本质上考的是一个很常见的套路：

- 一张表里有“关系”；
- 另一张表里有“属性”；
- 先通过 `JOIN` 把属性补全；
- 再用 `GROUP BY + AVG()` 做分组统计。

后面只要看到“某个分组里的平均值、总和、数量”这类题，基本都可以往这个方向想。
