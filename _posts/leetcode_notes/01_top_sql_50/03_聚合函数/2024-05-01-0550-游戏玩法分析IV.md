## 题目：0550. 游戏玩法分析 IV

**Table:** `Activity`

| Column Name  | Type |
| ------------ | ---- |
| player_id    | int  |
| device_id    | int  |
| event_date   | date |
| games_played | int  |

- `(player_id, event_date)` 是此表的主键（具有唯一值的列的组合）。
- 这张表显示了某些游戏的玩家的活动情况。
- 每一行是一个玩家的记录，他在某一天使用某个设备注销之前登录并玩了很多游戏（可能是 0）。

**问题描述:**

编写解决方案，报告在首次登录的第二天再次登录的玩家的 **比率**，**四舍五入到小数点后两位**。换句话说，你需要计算从首次登录后的第二天登录的玩家数量，并将其除以总玩家数。

**示例 1:**

**输入:**

`Activity` table:

| player_id | device_id | event_date | games_played |
| --------- | --------- | ---------- | ------------ |
| 1         | 2         | 2016-03-01 | 5            |
| 1         | 2         | 2016-03-02 | 6            |
| 2         | 3         | 2017-06-25 | 1            |
| 3         | 1         | 2016-03-02 | 0            |
| 3         | 4         | 2018-07-03 | 5            |

**输出:**

| fraction |
| -------- |
| 0.33     |

**解释:**

只有 ID 为 1 的玩家在第一天登录后才重新登录，所以答案是 1/3 = 0.33。

## 题解

这题表面上是在求一个比率，但真正要先拆清楚的是：**分子和分母分别是什么。**

题目要我们统计的是：

- 分母：总玩家数；
- 分子：那些在**首次登录后的第二天**又登录过的玩家数。

所以这题不能一上来就直接算比例，应该先分两步：

- 先找出每个玩家的首次登录日期；
- 再判断这个玩家有没有在首次登录后的第二天继续登录。

对应的 SQL 写法如下：

```sql
SELECT ROUND(
           COUNT(t.player_id) / (SELECT COUNT(DISTINCT player_id) FROM Activity),
           2
       ) AS fraction
FROM (
         SELECT player_id,
                MIN(event_date) AS first_login_date
         FROM Activity
         GROUP BY player_id
     ) AS t
WHERE (t.player_id, DATE_ADD(t.first_login_date, INTERVAL 1 DAY)) IN (
    SELECT player_id, event_date
    FROM Activity
);
```

这条 SQL 可以按顺序这样理解。

先看这个子查询：

```sql
SELECT player_id,
       MIN(event_date) AS first_login_date
FROM Activity
GROUP BY player_id
```

它的作用是找出每个玩家最早的登录日期，也就是首次登录日期。

因为同一个玩家可能登录很多次，所以要按 `player_id` 分组；  
而 `MIN(event_date)` 取到的就是最早那一天。

这样我们就得到了一个中间结果：每个玩家对应一个 `first_login_date`。

接着看外层的筛选条件：

```sql
WHERE (t.player_id, DATE_ADD(t.first_login_date, INTERVAL 1 DAY)) IN (
    SELECT player_id, event_date
    FROM Activity
)
```

这里是在做一件事：  
检查某个玩家在“首次登录后的第二天”这一天，是否真的有登录记录。

比如一个玩家首次登录是 `2016-03-01`，那么这里就会去看：

```sql
(player_id, 2016-03-02)
```

这个组合是否出现在 `Activity` 表里。  
如果存在，就说明这个玩家满足题意；如果不存在，就说明他没有在第二天继续登录。

因此，经过这一步筛选后，剩下的就是所有“次日再次登录”的玩家。

然后再看这一句：

```sql
COUNT(t.player_id)
```

它统计的就是满足条件的玩家数量，也就是分子。

分母则是：

```sql
(SELECT COUNT(DISTINCT player_id) FROM Activity)
```

也就是总玩家数。

最后再用：

```sql
ROUND(..., 2)
```

把结果四舍五入保留两位小数。

结合示例来看：

- 玩家 `1` 首次登录是 `2016-03-01`，并且在 `2016-03-02` 又登录了，所以算入分子；
- 玩家 `2` 只有一次登录，不算；
- 玩家 `3` 虽然后面也登录过，但不是首次登录后的第二天，所以也不算。

因此分子是 `1`，分母是 `3`，结果就是：

```sql
1 / 3 = 0.33
```

这题的关键不是语法，而是把题目里的“首次登录后的第二天再次登录”准确翻译成查询逻辑。  
以后遇到这种“先找每个人的第一次，再判断后续某个时间点是否发生过某事”的题，通常都可以先用 `GROUP BY + MIN()` 找第一次，再去原表里做存在性判断。
