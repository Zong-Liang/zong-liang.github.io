## 题目：1211. 查询结果的质量和占比

**表:** `Queries`

| Column Name | Type    |
| ----------- | ------- |
| query_name  | varchar |
| result      | varchar |
| position    | int     |
| rating      | int     |

- 此表可能存在重复的行。
- 此表包含了一些从数据库中收集的查询信息。
- “位置”（position）列的值为 1 到 500 。
- “评分”（rating）列的值为 1 到 5 。评分小于 3 的查询被定义为质量很差的查询。

**问题定义:**

- 将查询结果的 **质量** `quality` 定义为：各查询结果的评分与其位置之间比率的平均值。
- 将 **劣质查询百分比** `poor_query_percentage` 定义为：评分小于 3 的查询结果占全部查询结果的百分比。

编写解决方案，找出每次的 `query_name`、`quality` 和 `poor_query_percentage`。

`quality` 和 `poor_query_percentage` 都应 **四舍五入到小数点后两位**。

以 **任意顺序** 返回结果表。

**示例 1:**

**输入:**

`Queries` table:

| query_name | result           | position | rating |
| ---------- | ---------------- | -------- | ------ |
| Dog        | Golden Retriever | 1        | 5      |
| Dog        | German Shepherd  | 2        | 5      |
| Dog        | Mule             | 200      | 1      |
| Cat        | Shirazi          | 5        | 2      |
| Cat        | Siamese          | 3        | 3      |
| Cat        | Sphynx           | 7        | 4      |

**输出:**

| query_name | quality | poor_query_percentage |
| ---------- | ------- | --------------------- |
| Dog        | 2.50    | 33.33                 |
| Cat        | 0.66    | 33.33                 |

**解释:**

- **Dog** 查询结果的质量为 ((5 / 1) + (5 / 2) + (1 / 200)) / 3 = 2.50
- **Dog** 查询结果的劣质查询百分比为 (1 / 3) \* 100 = 33.33
- **Cat** 查询结果的质量为 ((2 / 5) + (3 / 3) + (4 / 7)) / 3 = 0.66
- **Cat** 查询结果的劣质查询百分比为 (1 / 3) \* 100 = 33.33

## 题解

这题要算两个指标，但本质上都是**按 `query_name` 分组之后做统计**。

题目要求的是：

- `quality`：`rating / position` 的平均值；
- `poor_query_percentage`：评分小于 `3` 的结果占全部结果的百分比。

也就是说，先把同一个 `query_name` 的所有记录放到一组里，然后分别计算这两个值就可以了。

对应的 SQL 写法如下：

```sql
SELECT query_name,
       ROUND(AVG(rating / position), 2) AS quality,
       ROUND(SUM(CASE WHEN rating < 3 THEN 1 ELSE 0 END) * 100 / COUNT(*), 2) AS poor_query_percentage
FROM Queries
GROUP BY query_name;
```

这条 SQL 可以按顺序这样理解。

先看分组：

```sql
GROUP BY query_name
```

这里表示相同 `query_name` 的记录归为一组。  
因为题目要求的是“每次查询的质量和劣质查询占比”，所以统计必须在每个 `query_name` 内部完成。

接着看第一个指标：

```sql
ROUND(AVG(rating / position), 2) AS quality
```

题目已经直接给出了 `quality` 的定义，就是每条记录的：

```sql
rating / position
```

然后再对这些值求平均数。

所以这里的逻辑很直接：

- 先对每一行算 `rating / position`；
- 再在同一个 `query_name` 组里取平均值；
- 最后用 `ROUND(..., 2)` 保留两位小数。

再看第二个指标：

```sql
ROUND(SUM(CASE WHEN rating < 3 THEN 1 ELSE 0 END) * 100 / COUNT(*), 2) AS poor_query_percentage
```

这里是在算“评分小于 `3` 的记录占总记录的百分比”。

可以拆成两部分看：

第一部分：

```sql
SUM(CASE WHEN rating < 3 THEN 1 ELSE 0 END)
```

这表示统计当前组里有多少条记录满足 `rating < 3`。  
满足条件就记 `1`，不满足就记 `0`，最后加总起来，就是劣质查询的数量。

第二部分：

```sql
COUNT(*)
```

这是当前组里的总记录数。

于是：

```sql
劣质记录数 / 总记录数 * 100
```

就是题目要的百分比。  
最后同样用 `ROUND(..., 2)` 保留两位小数。

结合示例来看：

对于 `Dog` 这组：

- `quality = ((5 / 1) + (5 / 2) + (1 / 200)) / 3 = 2.50`
- 评分小于 `3` 的只有一条，所以：
  ```sql
  1 / 3 * 100 = 33.33
  ```

对于 `Cat` 这组也是同样的计算方式。

这题的关键就是读懂定义，然后把定义原样翻译成聚合表达式。  
以后遇到这种“某个指标已经被题目明确定义好了”的题，通常不用绕，直接按定义写进 `AVG`、`SUM`、`COUNT` 里就可以了。
