## 题目：1484. 按日期分组销售产品

**表结构:** `Activities`

| 列名      | 类型    |
| --------- | ------- |
| sell_date | date    |
| product   | varchar |

该表没有主键，它可能包含重复项。此表的每一行都包含产品名称和在市场上销售的日期。

**问题描述:**

编写解决方案找出每个日期、销售的不同产品的数量及其名称。

每个日期的销售产品名称应按词典序排列。

返回按 `sell_date` 排序的结果表。

**示例 1:**

**输入:**

`Activities` 表:

| sell_date  | product    |
| ---------- | ---------- |
| 2020-05-30 | Headphone  |
| 2020-06-01 | Pencil     |
| 2020-06-02 | Mask       |
| 2020-05-30 | Basketball |
| 2020-06-01 | Bible      |
| 2020-06-02 | Mask       |
| 2020-05-30 | T-Shirt    |

**输出:**

| sell_date  | num_sold | products                     |
| ---------- | -------- | ---------------------------- |
| 2020-05-30 | 3        | Basketball,Headphone,T-shirt |
| 2020-06-01 | 2        | Bible,Pencil                 |
| 2020-06-02 | 1        | Mask                         |

**解释:**

- 对于 2020-05-30，出售的物品是 (Headphone, Basketball, T-shirt)，按词典序排列，并用逗号 ',' 分隔。
- 对于 2020-06-01，出售的物品是 (Pencil, Bible)，按词典序排列，并用逗号分隔。
- 对于 2020-06-02，出售的物品是 (Mask)，只需返回该物品名。

## 题解

这题要做的事情其实有两件，而且都要按“每天”来统计：

- 统计这一天卖出了多少种**不同**产品；
- 把这一天卖出的所有**不同**产品名称按字典序拼成一个字符串。

所以这题的关键不是普通分组，而是要注意两个点：

1. 表里**可能有重复记录**，所以产品既不能重复计数，也不能在拼接结果里重复出现；
2. 题目要求产品名称按字典序排列，所以在拼接字符串时还要额外排序。

对应的 SQL 写法如下：

```sql
SELECT sell_date,
       COUNT(DISTINCT product) AS num_sold,
       GROUP_CONCAT(DISTINCT product ORDER BY product ASC SEPARATOR ',') AS products
FROM Activities
GROUP BY sell_date
ORDER BY sell_date ASC;
```

这段查询可以顺着题意拆开来看。

先看第一列：

```sql
sell_date
```

因为题目要求的是“每个日期”的统计结果，所以这里自然要按日期分组，后面的所有统计都会围绕 `sell_date` 来做。

接着看不同产品数量：

```sql
COUNT(DISTINCT product) AS num_sold
```

这里的重点是 `DISTINCT`。  
因为题目明确说表中可能有重复项，所以同一天里同一个产品可能出现多次。  
但题目要的是“不同产品的数量”，不是记录条数，所以必须先去重，再计数。

比如示例里 `2020-06-02` 这一天，`Mask` 出现了两次，  
如果直接 `COUNT(product)`，结果会变成 `2`，这是错的；  
只有写成 `COUNT(DISTINCT product)`，结果才会是正确的 `1`。

再看产品名称拼接这一列：

```sql
GROUP_CONCAT(DISTINCT product ORDER BY product ASC SEPARATOR ',') AS products
```

这句其实就是把题目的要求原样翻译成 SQL。

它做了三件事：

- `DISTINCT product`：先去重，保证同一个产品不会重复出现；
- `ORDER BY product ASC`：按字典序排序；
- `SEPARATOR ','`：用逗号把这些产品名拼接起来。

所以如果某一天卖出的不同产品是：

- `Headphone`
- `Basketball`
- `T-shirt`

那么拼接前先排序，得到：

- `Basketball`
- `Headphone`
- `T-shirt`

最后再拼成：

- `Basketball,Headphone,T-shirt`

这正好符合题目要求。

最后两句：

```sql
GROUP BY sell_date
ORDER BY sell_date ASC
```

也很好理解：

- `GROUP BY sell_date`：按日期分组，分别统计每天的数据；
- `ORDER BY sell_date ASC`：按日期升序输出结果。

结合示例来看：

- `2020-05-30` 这一天卖了 `Headphone`、`Basketball`、`T-Shirt`，一共 3 种不同产品；
- `2020-06-01` 这一天卖了 `Pencil`、`Bible`，一共 2 种；
- `2020-06-02` 这一天虽然 `Mask` 出现了两次，但它仍然只算 1 种产品。

所以最终结果就是按日期返回每天的不同产品数量和排好序的产品列表。

这题本质上考的是**分组后同时做去重计数和去重拼接**。  
以后遇到“每组里有多少种不同值”以及“把每组里的值拼成字符串”这类题时，通常就会想到：

- `COUNT(DISTINCT ...)`
- `GROUP_CONCAT(...)`

而如果题目还强调“去重”和“排序”，就把这两个要求直接写进聚合函数里。
