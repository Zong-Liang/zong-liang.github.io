## 题目：1667. 修复表中的名字

**表:** `Users`

| Column Name | Type    |
| ----------- | ------- |
| user_id     | int     |
| name        | varchar |

`user_id` 是该表的主键（具有唯一值的列）。

该表包含用户的 ID 和名字。名字仅由小写和大写字符组成。

**问题描述:**

编写解决方案，修复名字，使得只有第一个字符是大写的，其余都是小写的。

返回按 `user_id` 排序的结果表。

**示例 1:**

**输入:**

`Users` table:

| user_id | name  |
| ------- | ----- |
| 1       | aLice |
| 2       | bOB   |

**输出:**

| user_id | name  |
| ------- | ----- |
| 1       | Alice |
| 2       | Bob   |

## 题解

这题本身不复杂，关键是把题目要求准确拆开：

**名字的第一个字符要变成大写，后面的所有字符都要变成小写。**

也就是说，这不是简单地整段转大写或整段转小写，而是要把名字拆成两部分分别处理：

- 第一个字符单独拿出来，转成大写；
- 剩下的部分单独拿出来，转成小写；
- 最后再把两部分拼回去。

对应的 SQL 写法如下：

```sql
SELECT user_id,
       CONCAT(UPPER(SUBSTRING(name, 1, 1)), LOWER(SUBSTRING(name, 2))) AS name
FROM Users
ORDER BY user_id;
```

这段查询的思路其实很直接。

先看名字的第一部分：

```sql
SUBSTRING(name, 1, 1)
```

这里表示从 `name` 里取出第一个字符。

接着：

```sql
UPPER(SUBSTRING(name, 1, 1))
```

把这个首字符转成大写。  
这样就满足了“名字首字母大写”的要求。

然后再看第二部分：

```sql
SUBSTRING(name, 2)
```

这里表示从第二个字符开始，一直取到字符串末尾。  
也就是把首字母之外的剩余部分全部取出来。

再配合：

```sql
LOWER(SUBSTRING(name, 2))
```

把后面的所有字符都转成小写。  
这样无论原来是大写、小写，还是大小写混杂，都会被统一规范成小写形式。

最后再把这两部分拼起来：

```sql
CONCAT(UPPER(SUBSTRING(name, 1, 1)), LOWER(SUBSTRING(name, 2)))
```

这样得到的新名字就符合题目要求了。

比如：

- `aLice` 会变成 `Alice`
- `bOB` 会变成 `Bob`

最后再按题目要求排序：

```sql
ORDER BY user_id
```

这样输出顺序就正确了。

这题本质上考的是**字符串拆分后分别处理，再重新拼接**。  
以后只要题目里出现“第一部分按一种规则处理，剩余部分按另一种规则处理”这种要求，通常都可以往 `SUBSTRING + UPPER/LOWER + CONCAT` 这个方向去想。
