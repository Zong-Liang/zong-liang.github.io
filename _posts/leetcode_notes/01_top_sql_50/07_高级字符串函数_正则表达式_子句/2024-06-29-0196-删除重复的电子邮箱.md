## 题目：0196. 删除重复的电子邮箱

**表结构:** `Person`

| Column Name | Type    |
| ----------- | ------- |
| id          | int     |
| email       | varchar |

`id` 是该表的主键列。该表的每一行包含一封电子邮件。电子邮件将不包含大写字母。

**问题描述:**

编写解决方案 **删除** 所有重复的电子邮件，只保留一个具有最小 `id` 的唯一电子邮件。

(对于 SQL 用户，请注意你应该编写一个 `DELETE` 语句而不是 `SELECT` 语句。)

运行脚本后，显示的答案是 `Person` 表。驱动程序将首先编译并运行你的代码片段，然后再显示 `Person` 表。`Person` 表的最终顺序无关紧要。

**示例 1:**

**输入:**

`Person` 表:

| id  | email            |
| --- | ---------------- |
| 1   | john@example.com |
| 2   | bob@example.com  |
| 3   | john@example.com |

**输出:**

| id  | email            |
| --- | ---------------- |
| 1   | john@example.com |
| 2   | bob@example.com  |

**解释:** john@example.com 重复两次。我们保留最小的 Id = 1。

## 题解

这题和前面那些“查数据”的题不一样，它不是让你找出重复邮箱，而是要求你**真正把重复数据删掉**。

而且删除的时候还不是随便留一条，题目要求非常明确：

**对于相同的 `email`，只保留 `id` 最小的那一条，其余都删除。**

所以这题的关键不是“怎么找重复”，而是：

**怎么准确找出那些应该被删掉的记录。**

对应的 SQL 写法如下：

```sql
DELETE p1
FROM Person p1, Person p2
WHERE p1.email = p2.email
  AND p1.id > p2.id;
```

这段 SQL 的核心思路是：  
把 `Person` 表和自己做比较，找出“邮箱相同，但 `id` 更大”的那一条，然后删掉它。

可以把这张表看成两个副本：

- `p1`：表示当前考虑要不要删除的这条记录；
- `p2`：表示拿来和它比较的另一条记录。

先看第一个条件：

```sql
p1.email = p2.email
```

这表示两条记录的邮箱相同。  
也就是说，我们只在“同一个邮箱”的记录之间做比较，不会把不同邮箱混在一起。

再看真正决定删除的条件：

```sql
p1.id > p2.id
```

这句才是关键。  
它表示：如果 `p1` 和 `p2` 的邮箱相同，但 `p1` 的 `id` 更大，那么 `p1` 就不是这一组里最小的那条记录，所以它应该被删除。

换句话说，只要某条记录能找到另一条“邮箱相同但 `id` 更小”的记录，它就一定是重复项，应该删掉。

结合示例来看：

表里有两条：

- `1, john@example.com`
- `3, john@example.com`

当这两条记录互相比对时：

- 邮箱相同；
- `3 > 1` 成立；

所以 `id = 3` 这一条会被删掉。

而 `id = 1` 这一条不会被删除，因为不存在另一条同邮箱且 `id` 比它更小的记录。

因此最后就只保留：

- `1, john@example.com`
- `2, bob@example.com`

这正好符合题意。

这题最值得记住的点是：

**删除重复数据时，通常不是直接删“重复值”，而是删“不是最优保留项”的那些记录。**

在这道题里，“最优保留项”就是同一邮箱下 `id` 最小的记录。  
所以判断一条记录该不该删，本质上就是看：**它是不是还能找到一个比自己更小的同邮箱记录。**

这题本质上考的是**自连接筛重复，再做删除**。  
以后如果遇到“保留每组最小值 / 最大值，其余删除”这种题，这类写法会很常见。
