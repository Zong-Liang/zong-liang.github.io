## 题目：1517. 查找拥有有效邮箱的用户

**表结构:** `Users`

| Column Name | Type    |
| ----------- | ------- |
| user_id     | int     |
| name        | varchar |
| mail        | varchar |

`user_id` 是该表的主键。该表包含了网站已注册用户的信息。有一些电子邮件是无效的。

**问题描述:**

编写一个解决方案，以查找具有有效电子邮件的用户。

一个有效的电子邮件具有前缀名称和域，其中：

1. **前缀名称** 是一个字符串，可以包含字母（大小写或小写）、数字、下划线 `_`、点 `.` 和（或）破折号 `-`。前缀名称 **必须** 以字母开头。
2. **域** 为 `'@leetcode.com'`。

可以以任何顺序返回结果表。

**示例 1:**

**输入:**

`Users` 表:

| user_id | name      | mail                    |
| ------- | --------- | ----------------------- |
| 1       | Winston   | winston@leetcode.com    |
| 2       | Jonathan  | jonathanisgreat         |
| 3       | Annabelle | bella-@leetcode.com     |
| 4       | Sally     | sally.come@leetcode.com |
| 5       | Marwan    | quarz#2020@leetcode.com |
| 6       | David     | david69@gmail.com       |
| 7       | Shapiro   | .shapo@leetcode.com     |

**输出:**

| user_id | name      | mail                    |
| ------- | --------- | ----------------------- |
| 1       | Winston   | winston@leetcode.com    |
| 3       | Annabelle | bella-@leetcode.com     |
| 4       | Sally     | sally.come@leetcode.com |

**解释:**

- 用户 2 的电子邮件没有域。
- 用户 5 的电子邮件带有不充许的 '#' 符号。
- 用户 6 的电子邮件没有 leetcode 域。
- 用户 7 的电子邮件以点开头。

## 题解

这题不是在考邮箱知识，而是在考：

**怎么把题目给出的邮箱规则，准确翻译成正则表达式。**

题目对有效邮箱的要求其实只有两部分：

1. `@` 前面的前缀部分：
   - 只能包含字母、数字、下划线 `_`、点 `.`、破折号 `-`
   - 并且**必须以字母开头**
2. 域名部分必须是：
   - `@leetcode.com`

所以这题的核心就是写出一个能同时限制这两部分的匹配规则。

对应的 SQL 写法如下：

```sql
SELECT user_id,
       name,
       mail
FROM Users
WHERE mail REGEXP '^[a-zA-Z][a-zA-Z0-9_.-]*@leetcode\\.com$'
  AND mail LIKE BINARY '%@leetcode.com';
```

这段查询的重点就在 `WHERE` 条件里。

先看正则这部分：

```sql
mail REGEXP '^[a-zA-Z][a-zA-Z0-9_.-]*@leetcode\\.com$'
```

它可以拆开理解。

开头的：

```sql
^
```

表示从字符串开头开始匹配。

接着这一段：

```sql
[a-zA-Z]
```

表示邮箱前缀的第一个字符必须是字母。  
这正好对应题目里“前缀必须以字母开头”的要求。

再往后：

```sql
[a-zA-Z0-9_.-]*
```

表示在第一个字母之后，还可以继续出现任意多个合法字符，范围包括：

- 字母
- 数字
- 下划线 `_`
- 点 `.`
- 破折号 `-`

这里的 `*` 表示前面的字符集合可以出现 0 次或多次。  
也就是说，前缀不一定很长，只要第一个字符合法，后面可以有，也可以没有。

再看域名部分：

```sql
@leetcode\\.com
```

这里要求邮箱必须以 `@leetcode.com` 结尾。  
其中 `.` 要写成 `\\.`，因为在正则里点号本身有特殊含义，表示“任意字符”，所以这里要转义，表示真正的点。

最后：

```sql
$
```

表示匹配到字符串结尾为止。  
这样就能保证整个邮箱完整符合规则，而不是只匹配其中一部分。

不过这里还多加了一句：

```sql
AND mail LIKE BINARY '%@leetcode.com';
```

这一句是为了把域名大小写也卡死。  
因为有些 SQL 环境下，`REGEXP` 可能会受到字符集或排序规则影响，大小写判断不一定严格。  
而题目要求的域名就是固定的 `@leetcode.com`，所以这里额外加一个 `LIKE BINARY`，可以确保域名部分必须是这个小写形式。

结合示例来看：

- `winston@leetcode.com` 合法；
- `bella-@leetcode.com` 合法，因为前缀以字母开头，后面带 `-` 也允许；
- `sally.come@leetcode.com` 也合法，因为点 `.` 在允许范围内；
- `jonathanisgreat` 不合法，因为连域名都没有；
- `quarz#2020@leetcode.com` 不合法，因为前缀里出现了 `#`；
- `david69@gmail.com` 不合法，因为域名不是 `leetcode.com`；
- `.shapo@leetcode.com` 不合法，因为前缀是以点开头，不是以字母开头。

所以能保留下来的，就是所有同时满足这两部分规则的用户。

这题本质上考的是**把字符串规则翻译成正则表达式**。  
以后只要题目里出现“必须以什么开头”“中间只能包含哪些字符”“必须以什么结尾”这种描述，基本都可以往正则去想。关键不是死记语法，而是先把规则拆清楚，再一段一段拼出来。
