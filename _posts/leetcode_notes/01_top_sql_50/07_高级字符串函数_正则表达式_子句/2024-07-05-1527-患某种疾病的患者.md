## 题目：1527. 患某种疾病的患者

**患者信息表:** `Patients`

| Column Name  | Type    |
| ------------ | ------- |
| patient_id   | int     |
| patient_name | varchar |
| conditions   | varchar |

在 SQL 中, `patient_id` (患者 ID) 是该表的主键。

`conditions` (疾病) 包含 0 个或以上的疾病代码，以空格分隔。

这个表包含医院中患者的信息。

**问题描述:**

查询患有 I 类糖尿病的患者 ID (`patient_id`)、患者姓名 (`patient_name`) 以及其患有的所有疾病代码 (`conditions`)。I 类糖尿病的代码总是包含前缀 `DIAB1`。

按 **任意顺序** 返回结果表。

**示例 1:**

**输入:**

`Patients` 表:

| patient_id | patient_name | conditions   |
| ---------- | ------------ | ------------ |
| 1          | Daniel       | YFEV COUGH   |
| 2          | Alice        |              |
| 3          | Bob          | DIAB100 MYOP |
| 4          | George       | ACNE DIAB100 |
| 5          | Alain        | DIAB201      |

**输出:**

| patient_id | patient_name | conditions   |
| ---------- | ------------ | ------------ |
| 3          | Bob          | DIAB100 MYOP |
| 4          | George       | ACNE DIAB100 |

**解释:** Bob 和 George 都患有代码以 `DIAB1` 开头的疾病。

## 题解

这题表面上是在查某种疾病，实际上关键是先看清楚 `conditions` 这一列的存储方式：

**它不是一个单独的疾病代码，而是一串用空格分隔的疾病代码。**

所以题目要找的，不是“字符串里包含 `DIAB1`”这么简单，而是：

**某个疾病代码本身要以 `DIAB1` 开头。**

这一点很重要。  
因为我们匹配的是一个完整的疾病代码位置，而不是随便在字符串中找一段子串。

对应的 SQL 写法如下：

```sql
SELECT patient_id,
       patient_name,
       conditions
FROM Patients
WHERE conditions LIKE 'DIAB1%'
   OR conditions LIKE '% DIAB1%';
```

这段查询的核心就在 `WHERE` 条件里。

先看第一部分：

```sql
conditions LIKE 'DIAB1%'
```

它表示：  
如果 `conditions` 这个字符串一开头就是 `DIAB1`，那么这条记录要保留下来。

这对应的是疾病代码出现在第一个位置的情况。  
比如：

- `DIAB100 MYOP`

这里第一个疾病代码就是 `DIAB100`，它以 `DIAB1` 开头，所以符合题意。

再看第二部分：

```sql
conditions LIKE '% DIAB1%'
```

它表示：  
如果字符串中某个位置出现了“空格 + `DIAB1` 开头的代码”，那么这条记录也要保留下来。

为什么前面要特意加一个空格？  
因为 `conditions` 里的多个疾病代码是用空格分隔的，所以只有当 `DIAB1` 前面是空格时，才能说明它是一个新的疾病代码，而不是前一个单词的一部分。

这就对应了疾病代码出现在中间或后面的位置。  
比如：

- `ACNE DIAB100`

这里 `DIAB100` 前面有一个空格，说明它是第二个疾病代码，而且它以 `DIAB1` 开头，所以也应该被选出来。

把这两个条件用 `OR` 连起来之后，就覆盖了两种可能：

- `DIAB1...` 出现在开头；
- `DIAB1...` 出现在后面的某个疾病代码位置。

结合示例来看：

- `Bob` 的 `conditions` 是 `DIAB100 MYOP`，开头就是 `DIAB100`，符合条件；
- `George` 的 `conditions` 是 `ACNE DIAB100`，`DIAB100` 出现在后面，但前面有空格，也符合条件；
- `Alain` 的 `conditions` 是 `DIAB201`，虽然也以 `DIAB` 开头，但不是 `DIAB1` 开头，所以不符合条件。

因此最终返回的就是 Bob 和 George。

这题本质上考的是**在一串由空格分隔的单词里，匹配满足前缀条件的某个单词**。  
以后只要碰到这种“字段里存的是多个值，用空格分开”的题，就不要把它当成普通字符串包含问题，而要先考虑：你匹配的是整段文本，还是其中某个单词的位置。这一步想清楚，条件就不容易写偏。
