## 题目：1327. 列出指定时间段内所有的下单产品

**表结构:**

`Products`

| Column Name      | Type    |
| ---------------- | ------- |
| product_id       | int     |
| product_name     | varchar |
| product_category | varchar |

`product_id` 是该表主键。该表包含该公司产品的数据。

`Orders`

| Column Name | Type |
| ----------- | ---- |
| product_id  | int  |
| order_date  | date |
| unit        | int  |

该表可能包含重复行。`product_id` 是表 `Products` 的外键。`unit` 是在日期 `order_date` 内下单产品的数目。

**问题描述:**

写一个解决方案，要求获取在 2020 年 2 月份下单的数量不少于 100 的产品的名字和数目。

返回结果表的顺序无要求。

**示例 1:**

**输入:**

`Products` 表:

| product_id | product_name          | product_category |
| ---------- | --------------------- | ---------------- |
| 1          | Leetcode Solutions    | Book             |
| 2          | Jewels of Stringology | Book             |
| 3          | HP                    | Laptop           |
| 4          | Lenovo                | Laptop           |
| 5          | Leetcode Kit          | T-shirt          |

`Orders` 表:

| product_id | order_date | unit |
| ---------- | ---------- | ---- |
| 1          | 2020-02-05 | 60   |
| 1          | 2020-02-10 | 70   |
| 2          | 2020-01-18 | 30   |
| 2          | 2020-02-11 | 80   |
| 3          | 2020-02-17 | 2    |
| 3          | 2020-02-24 | 3    |
| 4          | 2020-03-01 | 20   |
| 4          | 2020-03-04 | 30   |
| 5          | 2020-02-25 | 50   |
| 5          | 2020-02-27 | 50   |
| 5          | 2020-03-01 | 50   |

**输出:**

| product_name       | unit |
| ------------------ | ---- |
| Leetcode Solutions | 130  |
| Leetcode Kit       | 100  |

**解释:**

- 2020 年 2 月份下单 `product_id = 1` 的产品数目总和为 (60 + 70) = 130。
- 2020 年 2 月份下单 `product_id = 2` 的产品数目总和为 80。
- 2020 年 2 月份下单 `product_id = 3` 的产品数目总和为 (2 + 3) = 5。
- 2020 年 2 月份 `product_id = 4` 的产品并没有下单。
- 2020 年 2 月份下单 `product_id = 5` 的产品数目总和为 (50 + 50) = 100。

## 题解

这题看起来是在查“哪些产品符合条件”，但真正的思路其实很明确：

**先只看 2020 年 2 月的订单，再按产品统计这个月的下单总数，最后筛掉不足 100 的产品。**

也就是说，这题的核心不是连接两张表本身，而是先把题目的条件拆成三个动作：

1. 限定时间范围，只保留 2020 年 2 月的订单；
2. 按产品分组，统计每个产品在这个月的总下单量；
3. 只保留总数不少于 `100` 的产品。

对应的 SQL 写法如下：

```sql
SELECT p.product_name,
       SUM(o.unit) AS unit
FROM Products p
JOIN Orders o
  ON p.product_id = o.product_id
WHERE o.order_date BETWEEN '2020-02-01' AND '2020-02-29'
GROUP BY p.product_id, p.product_name
HAVING SUM(o.unit) >= 100;
```

这段查询可以顺着题意一步一步看。

先看连接部分：

```sql
FROM Products p
JOIN Orders o
  ON p.product_id = o.product_id
```

`Orders` 表里只有 `product_id`，并没有产品名称。  
而题目最后要求返回的是 `product_name` 和对应的数量，所以这里要先把 `Products` 和 `Orders` 连接起来，这样才能在统计完之后把产品名字一起查出来。

接着看时间筛选：

```sql
WHERE o.order_date BETWEEN '2020-02-01' AND '2020-02-29'
```

这一句的作用就是：  
只保留 **2020 年 2 月** 的订单记录，其他月份全部先排除掉。

这里一定要注意，题目统计的是“2020 年 2 月份下单的数量”，不是某个产品的总历史订单量。  
所以像示例里 `Leetcode Kit` 在 `2020-03-01` 还有一条 `50` 的订单，这一条是不能算进去的。

然后看分组统计：

```sql
GROUP BY p.product_id, p.product_name
```

因为题目要的是“每个产品”的下单总数，所以这里要按产品分组。  
分组之后，`SUM(o.unit)` 才能把同一个产品在 2 月里的多笔订单加起来。

比如示例里：

- `product_id = 1` 在 2 月有两条订单：`60` 和 `70`
- 那么分组之后，它的总数就是 `130`

最后再看分组后的筛选：

```sql
HAVING SUM(o.unit) >= 100
```

这一步表示：  
只保留 2 月总下单量不少于 `100` 的产品。

这里为什么是 `HAVING`，而不是 `WHERE`？  
因为 `SUM(o.unit)` 是分组聚合后的结果，只有分组完成之后才能拿来判断，所以必须用 `HAVING`。

结合示例来看：

- `Leetcode Solutions` 在 2 月的总下单量是 `60 + 70 = 130`，满足条件；
- `Jewels of Stringology` 在 2 月的总下单量是 `80`，不满足条件；
- `HP` 在 2 月的总下单量是 `2 + 3 = 5`，不满足条件；
- `Leetcode Kit` 在 2 月的总下单量是 `50 + 50 = 100`，满足条件；
- 它在 3 月还有一条 `50`，但这条不属于 2 月，所以不能算。

因此最终结果就是：

- `Leetcode Solutions, 130`
- `Leetcode Kit, 100`

这题本质上考的是一个很典型的处理顺序：

**先筛行，再分组，再用 `HAVING` 过滤聚合结果。**

以后只要题目里出现“某个时间段内，每个对象的总数 / 总金额 / 总销量达到多少”的描述，基本都可以往这个方向去想。
