## 题目：0176. 第二高的薪水

**表结构:** `Employee`

| Column Name | Type |
| ----------- | ---- |
| id          | int  |
| salary      | int  |

`id` 是这个表的主键。表的每一行包含员工的工资信息。

**问题描述:**

查询并返回 `Employee` 表中第二高的 **不同** 薪水。如果不存在第二高的薪水，查询应该返回 `null`。

**示例 1:**

**输入:**

`Employee` 表:

| id  | salary |
| --- | ------ |
| 1   | 100    |
| 2   | 200    |
| 3   | 300    |

**输出:**

| SecondHighestSalary |
| ------------------- |
| 200                 |

**示例 2:**

**输入:**

`Employee` 表:

| id  | salary |
| --- | ------ |
| 1   | 100    |

**输出:**

| SecondHighestSalary |
| ------------------- |
| null                |

## 题解

这题真正要注意的，不是“第二高”，而是题目里的这个限定词：

**不同。**

也就是说，我们要找的是**第二高的不同薪水**，而不是按员工记录排完序之后的第二行。

比如工资如果是：

- `300`
- `300`
- `200`

那么答案应该是 `200`，而不是第二条 `300`。  
所以这题第一步一定要先去重，再去找排名第二的薪水。

对应的 SQL 写法如下：

```sql
SELECT (
    SELECT DISTINCT salary
    FROM Employee
    ORDER BY salary DESC
    LIMIT 1 OFFSET 1
) AS SecondHighestSalary;
```

这段查询可以顺着思路拆开来看。

先看里面这一层：

```sql
SELECT DISTINCT salary
FROM Employee
ORDER BY salary DESC
LIMIT 1 OFFSET 1
```

它做的事情其实很明确。

第一步：

```sql
SELECT DISTINCT salary
FROM Employee
```

先把工资去重。  
因为题目要的是“不同薪水”中的第二高，所以相同工资只能算一次，不能重复参与排名。

第二步：

```sql
ORDER BY salary DESC
```

把去重后的工资从大到小排序。  
这样排完以后，第一名就是最高薪水，第二名就是第二高薪水。

第三步：

```sql
LIMIT 1 OFFSET 1
```

这里的意思是：

- `OFFSET 1`：先跳过第一行，也就是最高薪水；
- `LIMIT 1`：再取接下来这一行，也就是第二高薪水。

所以只要表里至少存在两个不同的工资值，这一层子查询就能准确取到答案。

外面再包一层：

```sql
SELECT (...) AS SecondHighestSalary
```

这一层的作用，是保证结果始终按题目要求返回一列，列名叫 `SecondHighestSalary`。

同时它还有一个很重要的效果：  
如果表里根本不存在第二高的不同薪水，比如只有一个工资值，那么里面的子查询查不到结果，这时外层 `SELECT` 会返回 `NULL`。

这正好符合题目的要求。

结合示例来看：

如果工资是：

- `100`
- `200`
- `300`

那么去重后还是：

- `300`
- `200`
- `100`

跳过第一行 `300`，再取一行，得到的就是 `200`。

如果表里只有：

- `100`

那么去重后也只有一个值。  
这时跳过第一行之后，后面已经没有数据了，所以结果就是 `NULL`。

这题本质上考的是一个很常见的处理顺序：

**先去重，再排序，最后取第 k 个值。**

以后只要题目里出现“第几高的不同值”“第几大的唯一结果”这类描述，都要先警惕重复值，别直接按原表排序去拿。
